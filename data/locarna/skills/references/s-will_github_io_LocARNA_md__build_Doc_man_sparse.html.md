USAGE: sparse [options] &lt;Input 1&gt; &lt;Input 2&gt; sparse is an even faster, stronger sparsified pairwise alignment algorithm than locarna. Like locarna, it performs fast simultaneous folding and alignment based on two RNA sequences (or alignments). In addition to the filtering of considered base pairs by their probabilities, it filters by conditional probabilities of bases and base pairs in their enclosing loops. Input, Constraints, and Output The usage, input, constraints-specifications and output are analogous to locarna; please refer to the help or man page of locarna for explanations and examples. OPTIONS -h , &ndash;help : Print this help. &ndash;galaxy-xml : Print galaxy xml wrapper. -V , &ndash;version : Print only version string. -v , &ndash;verbose : Be verbose. Prints input parameters, sequences and size information. -q , &ndash;quiet : Be quiet. Scoring parameters: -i , &ndash;indel= &lt;score&gt;( -150 ) : Indel score. Score contribution of each single base insertion or deletion. Indel opening score and indel score define the affine scoring of gaps. -i , &ndash;indel-loop= &lt;score&gt;( -300 ) : Score for insertions and deletions of loops per base &ndash;indel-opening= &lt;score&gt;( -750 ) : Indel opening score. Score contribution of opening an insertion or deletion, i.e. score for a consecutive run of deletions or insertions. Indel opening score and indel score define the affine scoring of gaps. &ndash;indel-opening-loop= &lt;score&gt;( -900 ) : Opening score for insertions and deletions of loops &ndash;ribosum-file= &lt;f&gt;(RIBOSUM85_60) : File specifying the Ribosum base and base-pair similarities. [default: use RIBOSUM85_60 without requiring a Ribosum file.] &ndash;use-ribosum= &lt;bool&gt;(true) : Use ribosum scores for scoring base matches and base pair matches; note that tau=0 suppresses any effect on the latter. -m , &ndash;match= &lt;score&gt;(50) : Set score contribution of a base match (unless ribosum scoring). -M , &ndash;mismatch= &lt;score&gt;(0) : Set score contribution of a base mismatch (unless ribosum scoring). &ndash;unpaired-penalty= &lt;score&gt;(0) : Penalty for unpaired bases -s , &ndash;struct-weight= &lt;score&gt;(200) : Maximal weight of 1/2 arc match. Balances structure vs. sequence score contributions. -e , &ndash;exp-prob= &lt;prob&gt; : Expected base pair probability. Used as background probability for base pair scoring [default: calculated from sequence length]. -t , &ndash;tau= &lt;factor&gt;(100) : Tau factor. Factor for contribution of sequence similarity in an arc match (in percent). tau=0 does not penalize any sequence information including compensatory mutations at arc matches, while tau=100 scores sequence similarity at ends of base matches (if a scoring matrix like ribosum is used, this adds the contributions for base pair match from the matrix). [default tau=0!] -E , &ndash;exclusion= &lt;score&gt;(0) : Score contribution per exclusion in structure local alignment. Set to zero for unrestricted structure locality. &ndash;stacking : Use stacking terms (requires stack-probs by RNAfold -p2 ) &ndash;new-stacking : Use new stacking terms (requires stack-probs by RNAfold -p2 ) Partition function representation (for sequence envelopes): &ndash;extended-pf : Use extended precision for the computation of sequence envelopes. This enables handling significantly larger instances. [default] &ndash;quad-pf : Use quad precision for partition function values. Even more precision than extended pf, but usually much slower (overrides extended-pf). Controlling_output: -w , &ndash;width= &lt;columns&gt;(120) : Width of alignment output. &ndash;clustal= &lt;file&gt; : Write alignment in ClustalW (aln) format to given file. &ndash;stockholm= &lt;file&gt; : Write alignment Stockholm format to given file. &ndash;pp= &lt;file&gt; : Write alignment in PP format to given file. &ndash;alifold-consensus-dp : Compute consensus dot plot by alifold (warning: this may fail for long sequences). &ndash;consensus-structure= &lt;type&gt;(alifold) : Type of consensus structures written to screen and stockholm output [alifold|mea|none] (default: none). &ndash;consensus-gamma= &lt;float&gt;(1.0) : Base pair weight for mea consensus computation. For MEA, base pairs are scored by their pair probability times 2 gamma; unpaired bases, by their unpaired probability. -L , &ndash;local-output : Output only local sub-alignment (to std out). &ndash;local-file-output : Write only local sub-alignment to output files. -P , &ndash;pos-output : Output only local sub-alignment positions. &ndash;write-structure : Write guidance structure in output. &ndash;special-gap-symbols : Special distinct gap symbols for loop gaps or gaps caused by sparsification &ndash;stopwatch : Print run time informations. Heuristics for speed accuracy trade off: -p , &ndash;min-prob= &lt;prob&gt;(0.001) : Minimal probability. Only base pairs of at least this probability are taken into account. &ndash;prob-unpaired-in-loop-threshold= &lt;threshold&gt;(0.00005) : Threshold for prob_unpaired_in_loop &ndash;prob-basepair-in-loop-threshold= &lt;threshold&gt;(0.0001) : Threshold for prob_basepair_in_loop &ndash;max-bps-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #base pairs divided by sequence length. This serves as a second filter on the "significant" base pairs. [default: 0.0 = no effect]. &ndash;max-uil-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #unpaired bases in loops divided by sequence length (def: no effect) &ndash;max-bpil-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #base pairs in loops divided by loop length (def: no effect) -D , &ndash;max-diff-am= &lt;diff&gt;( -1 ) : Maximal difference for sizes of matched arcs. [-1=off] -d , &ndash;max-diff= &lt;diff&gt;( -1 ) : Maximal difference for positions of alignment traces (and aligned bases). [-1=off] &ndash;max-diff-at-am= &lt;diff&gt;( -1 ) : Maximal difference for positions of alignment traces at arc match ends. [-1=off] &ndash;max-diff-aln= &lt;aln file&gt;() : Maximal difference relative to given alignment (file in clustalw format) &ndash;max-diff-pw-aln= &lt;alignment&gt;() : Maximal difference relative to given alignment (string, delim=AMPERSAND) &ndash;max-diff-relax : Relax deviation constraints in multiple aligmnent &ndash;min-trace-probability= &lt;probability&gt;(1e-5) : Minimal sequence alignment probability of potential traces (probability-based sequence alignment envelope) [default=1e-4]. MEA score: &ndash;mea-alignment : Perform maximum expected accuracy alignment (instead of using the default similarity scoring). &ndash;match-prob-method= &lt;int&gt;(0) : Select method for computing sequence-based base match probablities (to be used for mea-type alignment scores). Methods: 1=probcons-style from HMM, 2=probalign-style from PFs, 3=from PFs, local &ndash;probcons-file= &lt;file&gt; : Read parameters for probcons-like calculation of match probabilities from probcons parameter file. &ndash;temperature-alipf= &lt;int&gt;(300) : Temperature for the */sequence* alignment/ partition functions used by the probcons-like sequence-based match/trace probability computation (this temperature is different from the \'physical\' temperature of RNA folding!). &ndash;pf-struct-weight= &lt;weight&gt;(200) : Structure weight in PF computations (for the computation of sequence-based match probabilties from partition functions). &ndash;mea-gapcost : Use gap cost in mea alignment &ndash;mea-alpha= &lt;weight&gt;(0) : Weight alpha for MEA &ndash;mea-beta= &lt;weight&gt;(200) : Weight beta for MEA &ndash;mea-gamma= &lt;weight&gt;(100) : Weight gamma for MEA &ndash;probability-scale= &lt;scale&gt;(10000) : Scale for probabilities/resolution of mea score &ndash;write-match-probs= &lt;file&gt; : Write match probs to file (don\'t align!). &ndash;read-match-probs= &lt;file&gt; : Read match probabilities from file. &ndash;write-arcmatch-scores= &lt;file&gt; : Write arcmatch scores (don\'t align!) &ndash;read-arcmatch-scores= &lt;file&gt; : Read arcmatch scores. &ndash;read-arcmatch-probs= &lt;file&gt; : Read arcmatch probabilities (weighted by factor mea_beta/100) Constraints: &ndash;noLP : Disallow lonely pairs in prediction and alignment. &ndash;maxBPspan= &lt;span&gt;( -1 ) : Limit maximum base pair span [default=off]. &ndash;relaxed-anchors : Use relaxed semantics of anchor constraints [default=strict semantics]. Input files: The tool is called with two input files &lt;Input 1&gt; and &lt;Input 2&gt;, which specify the two input sequences or input alignments. Different input formats (Fasta, Clustal, Stockholm, LocARNA PP, ViennaRNA postscript dotplots) are accepted and automatically recognized (by file content); the two input files can be in different formats. Extended variants of the Clustal and Stockholm formats enable specifying anchor and structure constraints.
