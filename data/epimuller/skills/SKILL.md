# epimuller

## Overview
epimuller is a Python-based toolset designed to visualize the evolution and frequency of viral lineages over time. Unlike standard frequency plots, epimuller integrates phylogenetic hierarchy, allowing you to see how specific clades emerge from ancestral backgrounds. It is particularly useful for tracking Variants of Concern (VOCs) by defining clades based on specific amino acid mutations or traits like PANGO lineages.

The tool operates as a suite of three primary components:
1. **epimuller-define**: Assigns samples to clades based on ancestral reconstructions and calculates abundance.
2. **epimuller-draw**: Generates the actual Muller plots using the hierarchy and abundance data.
3. **epimuller**: A convenient wrapper that executes both the define and draw steps in a single command.

## Installation
Install via Bioconda or pip:
```bash
conda install bioconda::epimuller
# OR
pip3 install epimuller
```

## Core Workflow and CLI Patterns

### 1. Standard Wrapper Usage
The `epimuller` command is the most common entry point. It requires either a Nextstrain results directory or an annotated TreeTime nexus file.

**Using Nextstrain Input:**
```bash
epimuller \
  -n path/to/nextstrain_results/ \
  -m metadata.tsv \
  -oDir output_folder \
  -oP experiment_prefix \
  -c epidemiology_data.tsv
```

**Using TreeTime Nexus Input:**
```bash
epimuller \
  -a timetree.nexus \
  -g gene_boundaries.json \
  -m metadata.tsv \
  -oDir output_folder \
  -oP experiment_prefix
```

### 2. Defining Clades by Mutations
To track specific amino acid changes, use the `-mut` flag. The format is `[GENE][*ORAncAA][site][*ORtoAA]`.
*   Example: `S*501*` (any mutation at Spike site 501)
*   Example: `SE484K` (Spike Glutamic acid to Lysine at 484)

```bash
epimuller -n ./results -m meta.tsv -oP voc_tracking -mut 'S*501*' 'SE484K' -min 50
```

### 3. Visualizing Traits (e.g., PANGO Lineages)
If you want to group by a specific trait key found in your JSON/Nexus file:
```bash
epimuller \
  -n ./results \
  -m meta.tsv \
  -oP lineage_plot \
  --traitOfInterstKey lineage \
  -lp Max \
  -min 100
```

## Input Requirements
*   **Metadata (-m)**: A TSV file containing at least `strain` and `date` columns.
*   **Nextstrain (-n)**: A directory containing `tree.nwk` and the trait JSON (default `aa_muts.json`).
*   **Annotated Tree (-a)**: A Nexus file with `[&!traitOfInterst=value]` annotations.
*   **Gene Boundaries (-g)**: Required when using TreeTime nexus files with amino acid mutations to map coordinates.

## Expert Tips and Best Practices
*   **Filtering Noise**: Use `-min <int>` (minimum total count) and `-mt <int>` (minimum time points) to prevent the plot from being cluttered by low-frequency or transient clades.
*   **Label Positioning**: Use `-lp {Right, Max, Start, End}` to adjust where clade labels appear. `Max` is often best for readability as it places the label at the point of highest frequency.
*   **Custom Colors**: To specify colors, edit the `parentHierarchy` file generated by the define step. Add a `color` column with hex values (e.g., `#FF0000`) before running `epimuller-draw`.
*   **Time Windows**: Adjust the temporal resolution using `-t <int>` (time window in days). The default is often 7 or 14 days depending on the dataset density.
*   **GISAID Pre-processing**: If starting with raw GISAID downloads, use `epimuller-parse` to convert the fasta headers into a usable metadata TSV.
