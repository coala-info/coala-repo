# bigmag

## Overview
BIgMAG (Board InteGrating Metagenome-Assembled Genomes) is a visualization dashboard designed to consolidate multiple metagenome quality metrics and taxonomic annotations into a single interface. It processes the unique summary files generated by the MAGFlow or nf-core/mag pipelines. This skill provides the necessary commands and best practices for launching the interactive Dash-based dashboard or generating standalone HTML reports for MAG quality assessment.

## Installation
The tool can be installed via Bioconda or Pip:

```bash
# Via Bioconda
conda install bioconda::bigmag

# Via Pip
pip install bigmag
```

## Core Usage Patterns

### Launching the Interactive Dashboard
The primary use case is launching a local web server to explore MAG metrics interactively.

```bash
# Basic execution (default port 8050)
bigmag 'path/to/final_df.tsv'

# Specifying a custom port
bigmag -p 8888 'path/to/final_df.tsv'
```

### Generating Static HTML Reports
For sharing results or viewing data without a Python backend, use the "lite" version to export an HTML file. Note that interactive callbacks (dynamic filtering) are disabled in this mode.

```bash
# Generate a standalone HTML report
bigmag-lite path/to/final_df.tsv -o /path/to/output/report.html
```

### Customizing the Visualization
When using `bigmag-lite` or specific CLI configurations, you can set thresholds and parameters for the plots:

*   **CheckM2 Completeness**: `--cl_checkm2 <N>` (Default: 10)
*   **BUSCO Completeness**: `--csco_busco <N>` (Default: 10)
*   **GUNC Metric**: `--param_gunc <STRING>` (Default: clade_separation_score)
*   **QUAST Metric**: `--param_quast <STRING>` (Default: "GC (%)")
*   **Taxonomy Level**: `--tax_level <STRING>` (Default: Phylum)

## Expert Tips and Best Practices

### Input Preparation
*   **nf-core/mag Integration**: To generate the required input file (`bigmag_summary.tsv`), run the nf-core/mag pipeline with the following flags: `--generate_bigmag_file --run_gunc --run_checkm2`. Ensure that BINQC, GTDB-TK, QUAST, and BUSCO processes are not skipped.
*   **Sample Limits**: For optimal visualization and color distinctness, limit each run to **15-20 samples**.
*   **Naming Conventions**: Keep sample names short (**5-10 characters**) to prevent labels from crowding the plot area.

### Dashboard Optimization
*   **Layout Adjustment**: If the dashboard appears disorganized, reload the page. On smaller screens (e.g., 13-inch laptops), zooming the browser to **67%** often provides the most aesthetic layout.
*   **Exporting Plots**: Use the camera icon in the top-right corner of any individual plot to download it as a high-quality image.
*   **Taxonomic Hover**: If GTDB-Tk2 data is included in the input, hovering over CheckM2 or BUSCO plot points will reveal the specific taxonomic classification for that bin.
