# bioconductor-zenith

## Overview

The `zenith` package implements gene set analysis (GSA) specifically designed to work with linear mixed models. It extends the `camera` method from the `limma` package to be compatible with `dream` (from the `variancePartition` package). This allows for gene set testing that accounts for the correlation between gene expression traits in complex experimental designs, such as longitudinal studies or those with repeated measures.

## Core Workflow

### 1. Differential Expression with Dream
Before using `zenith`, you must process your RNA-seq data through the `dream` workflow to account for precision weights and random effects.

```r
library(zenith)
library(variancePartition)
library(edgeR)

# Standard dream workflow
dge <- DGEList(counts = geneCounts)
dge <- calcNormFactors(dge)
vobj <- voomWithDreamWeights(dge, ~ condition + (1|subject), metadata)
fit <- dream(vobj, ~ condition + (1|subject), metadata)
fit <- eBayes(fit)
```

### 2. Loading Gene Sets
`zenith` provides convenient wrappers to fetch gene sets and map them to specific identifiers (e.g., ENSEMBL, SYMBOL, or ENTREZ).

```r
# Load MSigDB Hallmark gene sets mapped to ENSEMBL
gs.msigdb <- get_MSigDB("H", to = "ENSEMBL")

# Load Gene Ontology Biological Process
gs.go <- get_GeneOntology("BP", to = "SYMBOL")

# Load from Enrichr (requires EnrichmentBrowser)
library(EnrichmentBrowser)
gs.allen <- getGenesets(org = "hsa", db = "enrichr", lib = "Allen_Brain_Atlas_10x_scRNA_2021")
```

### 3. Running Zenith GSA
Use `zenith_gsa` to test for enrichment. You must specify the coefficient or contrast from your `dream` fit.

```r
# Run analysis for a specific coefficient
res.gsa <- zenith_gsa(fit, gs.msigdb, 'conditionTreatment')

# View results
head(res.gsa)
```

### 4. Visualizing Results
The package includes a specialized plotting function to summarize the top results across multiple coefficients or cell types.

```r
# Plot top gene sets by t-statistic
plotZenithResults(res.gsa)
```

## Key Functions

- `zenith_gsa()`: The primary function for gene set analysis. It takes a `dream` fit object, a gene set collection, and the coefficient to test.
- `get_MSigDB()`: Shortcut to download and cache Molecular Signatures Database collections.
- `get_GeneOntology()`: Shortcut to download and cache Gene Ontology collections.
- `plotZenithResults()`: Generates a heatmap-style summary of enrichment results.

## Tips for Success

- **ID Mapping**: Ensure the gene IDs in your `dream` fit (row names) match the `to` argument used in `get_MSigDB` or `get_GeneOntology`.
- **Caching**: The first time you download gene sets, it may take ~1 minute. Subsequent calls are nearly instantaneous as results are cached on disk.
- **Mixed Models**: Always use `zenith` instead of standard `camera` when your differential expression model includes random effects (mixed models), as standard methods may produce biased p-values in these contexts.
