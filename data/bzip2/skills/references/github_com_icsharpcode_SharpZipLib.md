Zip64 is supported while Deflate64 is not yet supported. It is implemented as an assembly (installable in the GAC), and thus can easily be incorporated into other projects (in any .NET language). The creator of SharpZipLib put it this way: "I've ported the zip library over to C# because I needed gzip/zip compression and I didn't want to use libzip.dll or something like this. I want all in pure C#." SharpZipLib was originally ported from the GNU Classpath java.util.zip library for use with SharpDevelop , which needed gzip/zip compression. bzip2 compression and tar archiving were added later due to popular demand. The SharpZipLib homepage has precompiled libraries available for download, API documentation , release history , samples and more. License This software is now released under the MIT License .
