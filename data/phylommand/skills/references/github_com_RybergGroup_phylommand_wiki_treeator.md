Treeator is a command line program to construct trees. The program take either a left triangular similarity matrix (neighbour joining) or a data matrix of fasta, nexus, or relaxed phylip format (not interleaved; parsimony/maximum likelihood) as input through standard in/ last argument/ as given below. For character data an alphabet is also needed, default is DNA (A T C G -). Usage treeator [arguments] data_file.txt treeator [arguments] -d data_file.txt tree_file.tree Arguments --alphabet_file / -a Give file with character alphabet, or dna, protein, or binary for dna, amino acid, respectively binary (0 1) alphabets (default: dna). See example_files/dna.alphabet for an example of how an alphabet file can look. --data_file / -d Give the data file. --fixed / -e Give parameter to fix. First parameter is indexed 0. Several parameters can be given in a comma separated string, e.g.: treeator -e 0,2,3 -m 0,1,2,3,4,5,6 -d data_file.fst -t tree_file.tree -a alphabet.file --file / -f Give data file name, or if data file name already given, then tree file name. If nexus format and no tree file name is given, tree is assumed to be in same file as data. --format Give the format of the input files. For character file fasta, phylip and nexus are the options. For the tree file the options are newick and nexus. Give the character file format first, and the tree file format after a comma, e.g.: treeator --format phylip,newick -d data_file.phy -t tree_file.tree Fasta is the default character file format, and newick is the default tree file format, unless character file is set to nexus, then nexus is also default for tree file, e.g.: treeator --format nexus -f tree_and_data.nex --get_state_at_nodes Will give the states at internal nodes as comments (readable in FigTree). --help / -h Print help. --likelihood / -l Calculate likelihood for data given tree. --model / -m Give the model by numbering the rate parameters for different transition, e.g.: treeator -m 0,1,0,2,1,2 -d data_file.phy -t tree_file.tree -a alphabet.file The order is by row, i.e. from parameter 0 to parameter 1 first then, 0 to 2, and so on to all other parameters, then 1 to 0, and so forth. --neighbour_joining / -n Compute neighbour joining tree for given data. The data should be a left triangular similarity matrix. --no_branch_length / -0 Do not print branch lengths and do not calculate branch lengths for parsimony trees. --no_label / -L Will tell treeator that there are no taxon labels in the similarity matrix. --no_optim / -N Calculate likelihood for given parameters. No optimization. --output Give tree format for output, nexus (nex or x for short) or newick (new or w for short), e.g: treeator --output x Default w. --parameters / -P Give corresponding parameter values for parameters. If optimizing these will be starting values, e.g.: treeator -P 0.1,0.01,0.05 -m 0,1,0,2,1,2 -d data_file.phy -t tree_file.tree -a alphabet.file --parsimony / -p Calculate parsimony score for given tree and data. --random / -r Do stepwise addition in random order. --tree_file / -t Give tree file name. --step_wise / -s Do parsimony stepwise addition. --verbose / -v Get additional output.
