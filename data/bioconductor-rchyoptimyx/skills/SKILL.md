# bioconductor-rchyoptimyx

## Overview
RchyOptimyx (cellular hieraRCHY OPTIMization) is an R package designed to find the most efficient gating strategies for flow cytometry data. It takes phenotypes (often generated by `flowType`) and their associated importance scores (e.g., p-values from clinical outcomes) to construct a Directed Acyclic Graph (DAG). The tool optimizes the path from a root population (e.g., "All Cells") to a target population, identifying the most significant intermediate markers and reducing redundancy in high-dimensional panels.

## Core Workflow

### 1. Data Preparation
RchyOptimyx typically requires input from `flowType`. You need:
- **Phenotype Codes**: Encoded representations of cell populations.
- **Phenotype Scores**: A numeric vector (e.g., -log10 p-values) where higher values indicate more "interesting" populations.

```r
library(RchyOptimyx)
library(flowType)

# Example: Assuming ResList is a flowType result and Pvals are calculated
pheno_codes <- ResList[[1]]@PhenoCodes
pheno_scores <- -log10(Pvals)
target_phenotype <- pheno_codes["CD3+CD27+"] # Example target
```

### 2. Optimizing the Hierarchy
The `RchyOptimyx` function is the primary constructor. It finds the best paths to a specific target population.

```r
# Optimize hierarchy for a single target
# pathCount: Number of paths to keep (e.g., 1 for the single best path)
res <- RchyOptimyx(pheno.codes = pheno_codes, 
                   phenotypeScores = pheno_scores, 
                   startPhenotype = target_phenotype, 
                   pathCount = 1, 
                   trim = FALSE)
```

### 3. Merging Multiple Targets
To visualize multiple significant populations in one hierarchy, create individual objects and merge them.

```r
# Initialize with first target
res_combined <- RchyOptimyx(pheno_codes, pheno_scores, target1, 1, FALSE)

# Merge subsequent targets
res_temp <- RchyOptimyx(pheno_codes, pheno_scores, target2, 1, FALSE)
res_combined <- merge(res_combined, res_temp)
```

### 4. Visualization
The `plot` method provides a specialized view of the hierarchy, coloring nodes by their scores.

```r
plot(res_combined, 
     phenotypeScores = pheno_scores, 
     phenotypeCodes = pheno_codes, 
     marker.names = MarkerNames, 
     ylab = "-log10(Pvalue)")
```

## Key Parameters
- `pathCount`: Controls the complexity of the result. Setting this to `factorial(number_of_markers)` shows all possible paths; setting it to 1 or 2 shows only the most optimized routes.
- `trim.level`: Used to limit the depth of the hierarchy (e.g., `trim.level = 4` only shows populations with up to 4 markers).
- `startPhenotype`: Despite the name, this is the **target** population (the leaf node) you want to reach from the root.

## Tips for Success
- **Score Direction**: RchyOptimyx maximizes the score. If using raw p-values, transform them (e.g., `-log10(p)`) so that significant results have higher values.
- **Marker Names**: Ensure `marker.names` matches the order of markers used in the `flowType` analysis to ensure correct node labeling.
- **Memory Management**: For very high-dimensional data (many markers), start with a low `pathCount` to keep the graph legible and the computation fast.
