# spinningjenny

## Overview
nf-core/spinningjenny is a computational pipeline designed to simulate the first industrial revolution using Agent Based Models (ABM). It leverages NetLogo for model execution and R for statistical visualization, allowing researchers to explore historical economic scenarios through parallelized simulations.

The pipeline transforms experimental variables into XML configurations to run multiple model iterations across compute infrastructures. Users receive a suite of results including raw model outputs and graphical reports detailing various economic parameters generated during the simulation.

## Data preparation
The pipeline requires three primary input components to define the simulation environment and experimental variables.

*   **Samplesheet**: A file containing information about the variables used in the experiment. While the internal schema suggests a CSV format, the documentation specifies a tab-separated file with 4 columns and no header row.
*   **XML Template**: An experiment file (`.xml`) containing the number of simulation steps and initial values for the variables.
*   **NetLogo Model**: A `.nlogo` file that defines the logic and agents of the Agent Based Model.

Example samplesheet structure (as shown in general usage):
```csv
sample,fastq_1,fastq_2
CONTROL_REP1,AEG588A1_S1_L002_R1_001.fastq.gz,AEG588A1_S1_L002_R2_001.fastq.gz
```
*Note: The documentation contains conflicting information regarding samplesheet columns; ensure the file matches the specific ABM variables required for your simulation.*

## How to run
Run the pipeline using the `nextflow run` command. It is recommended to test the setup with the `test` profile before using production data.

```bash
nextflow run nf-core/spinningjenny \
   -profile <docker/singularity/conda/institute> \
   --input samplesheet.csv \
   --template experiment.xml \
   --nlogo model.nlogo \
   --outdir <OUTDIR>
```

Key parameters:
*   `-profile`: Choose a configuration profile (e.g., `docker`, `singularity`).
*   `--batches`: Specify the number of batches for running chunks of experiments (default: 1).
*   `-resume`: Use this Nextflow flag to restart a pipeline from the last successful step.
*   `-r`: Pin a specific pipeline release version.

## Outputs
Results are saved in the directory specified by the `--outdir` parameter.

*   **Model Runs**: Data generated by the NetLogo simulation engines.
*   **Economic Plots**: Graphical reports created in R that visualize different economic parameters from the simulation.
*   **MultiQC**: A summary report aggregating pipeline statistics and run metadata.

For a detailed description of all output files and how to interpret the economic visualizations, refer to the [official output documentation](https://nf-co.re/spinningjenny/output).
