Технологический стек В данном проекте автотесты написаны на Java с использованием библиотеки REST Assured JUnit 5 используется для модульного тестирования Gradle используется для автоматизированной сборки проекта Jenkins выполняет запуск тестов Allure Report формирует отчет о запуске тестов Автотесты интегрируются с тест-менеджмент системой Allure TestOps и таск-трекер системой Jira В Telegram отправляются уведомления о пройденном прогоне с помощью библиотеки allure-notifications Запуск тестов из терминала Подробнее 🚀 Локальный запуск тестов gradle clean ${task} ❗ Для запуска тестов необходимы файлы, в которых прописаны определенные параметры: api.properties apiUrl – адрес сервера, который будет использоваться в тестах credentials.properties userName – логин пользователя для авторизации в приложении Book Store password – пароль пользователя для авторизации в приложении Book Store 🚀 Параметры сборки TASK - список тестов, сгруппированных по параметру тега. В зависимости от выбранного параметра, будут запускаться определенные группы тестов test - запуск всех тестов high_priority_tests - запуск тестов с высоким приоритетом Формирование отчета Allure Подробнее ❗ Предварительно необходимо установить Allure allure serve build/allure-results Запуск тестов в Jenkins Подробнее 🚩 Для запуска тестов в Jenkins необходимо выполнить следующие шаги: Открыть сборку Jenkins Нажать на таск "Собрать с параметрами" Указать значения параметров Нажать на кнопку "Собрать" 🚩 Для формирования отчета о прохождении тестов в Allure Report необходимо выполнить следующий шаг: После выполнения сборки нажать на любую ссылку/иконку "Allure Report" Отчет о результатах тестирования в Allure Report Подробнее Allure-framework используется в качестве инструмента для построения отчетов о прогоне автотестов. Он позволяет получить информацию о ходе выполнения тестов, а также прикрепить скриншоты, логи и видео к формируемому отчету. Имеется возможность указать различные теги, приоритеты и прочую сопутствующую информацию для тестов. 🎯 Главная страница Allure-отчета 🎯 Информация о тестовом прогоне в графическом виде 🎯 Группировка тестов по проверяемому функционалу Интеграция тестов c тест-менеджмент системой Allure TestOps Подробнее Allure TestOps - это платформа управления качеством программного обеспечения, объединяющая автоматическое и ручное тестирование, которая позволяет управлять всем, что связано с тестированием, в одном месте. 🧪 Основной дашборд 🧪 Дашборд для отображения успешности и длительности тестов 🧪 Дашборд по стендам 🧪 Дашборд по членам команды 🧪 Запуски тестов 🧪 Результаты запуска тестов 🧪 Сгруппированные тест-кейсы по проверяемому функционалу Интеграция тестов c таск-трекер системой Jira Подробнее Интеграция с Jira позволяет добавлять в задачи тест-кейсы, запуски и их результаты.
