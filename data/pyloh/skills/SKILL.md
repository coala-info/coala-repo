# pyloh

## Overview
PyLOH is a specialized bioinformatics tool designed to solve the "identifiability problem" in cancer genomics, where different combinations of tumor purity and ploidy can explain sequencing data equally well. It achieves this by integrating somatic copy number alterations (SCNA) and loss of heterozygosity (LOH) data. The tool operates through a three-stage workflow: preprocessing raw alignment data, executing a probabilistic model to estimate purity and allele-specific copy numbers, and generating BAF (B-Allele Frequency) heat maps for visualization.

## CLI Usage and Best Practices

### 1. Preprocessing
The `preprocess` module extracts allelic counts from heterozygous loci.

```bash
PyLOH.py preprocess REFERENCE.fasta NORMAL.bam TUMOUR.bam BASENAME \
    --segments_bed SEGMENTS.bed \
    --min_depth 20 \
    --process_num 10
```

**Expert Tips:**
- **Segmentation:** While PyLOH can default to 22 autosomes, it is highly recommended to provide a `--segments_bed` file generated by external algorithms like BICseq (for WGS) or DNAcopy (for WES).
- **WES vs. WGS:** If working with Whole Exome Sequencing data, you **must** include the `--WES` flag.
- **Indexing:** Ensure your reference fasta is indexed (`samtools faidx`) and your BAM files have corresponding `.bai` files in the same directory.
- **Performance:** Use `--process_num` to leverage multi-core processing, as `pysam`-based extraction is computationally intensive.

### 2. Model Execution
The `run_model` module performs the actual deconvolution using an Expectation-Maximization (EM) algorithm.

```bash
PyLOH.py run_model BASENAME \
    --allelenumber_max 2 \
    --max_iters 100 \
    --stop_value 1e-7
```

**Expert Tips:**
- **Priors:** Using the default uniform prior (by omitting the `--priors` flag) is generally recommended unless specific biological constraints are known.
- **Allele Number:** The `--allelenumber_max` parameter defines the search space for copy states. Increasing this beyond 2 increases complexity and runtime.
- **Convergence:** If the model fails to converge, consider increasing `--max_iters` or adjusting the `--stop_value`.

### 3. Visualization (Postprocessing)
Generate BAF heat maps to visualize LOH cluster patterns.

```bash
PyLOH.py BAF_heatmap BASENAME
```

**Requirements:**
- Requires `matplotlib` to be installed in the Python environment.
- This command uses the `.BAF_heatmap` file generated during the `preprocess` step.

## Troubleshooting and Constraints
- **Python Version:** PyLOH is built for Python 2.7. Ensure the environment matches this requirement.
- **Pysam Version:** Pysam 0.7.x is significantly faster for this tool than 0.8.x. If preprocessing is taking days rather than hours, check the `pysam` version.
- **Memory:** For large WGS datasets, ensure the system has sufficient RAM when using high `--process_num` values.
