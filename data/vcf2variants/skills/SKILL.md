# vcf2variants

## Overview
The `vcf2variants` tool is a specialized utility within the Varda2 preprocessing pipeline. It transforms standard Variant Call Format (VCF) data into a simplified, tabular format optimized for the Varda2 database. This process is not a direct conversion; it requires specific upstream filtering using `bcftools` to ensure that each line represents a single, clean variant. The tool is essential for researchers and bioinformaticians submitting genomic data to Varda2-compatible repositories.

## Preprocessing Workflow
Before running `vcf2variants`, the input VCF must be normalized and filtered. Failure to do so will result in incompatible data structures.

### 1. Clean and Normalize VCF
Execute these `bcftools` commands in sequence to prepare the file:

```bash
# Trim alternative alleles and exclude uncalled genotypes
bcftools view --trim-alt-alleles --exclude-uncalled --output-file trimmed.vcf input.vcf

# Split multiallelic entries into single records
bcftools norm --multiallelics - --output normalized.vcf trimmed.vcf

# Exclude symbolic alleles (ALT="*")
bcftools view --exclude 'ALT=="*"' --output-file final_preprocessed.vcf normalized.vcf
```

### 2. Execute Conversion
The tool reads from standard input and writes to standard output.

```bash
vcf2variants < final_preprocessed.vcf > variants.varda
```

## Output Format Specification
The resulting file is a tab-separated values (TSV) file with the following columns:

1.  **CHROM**: Chromosome/Contig ID.
2.  **START**: Start position.
3.  **END**: End position.
4.  **PLOIDY**: Sample ploidy.
5.  **PHASE SET**: Phase information.
    *   `-1`: Homozygous.
    *   `0`: Unphased.
6.  **INSERTED LENGTH**: Length of the insertion.
7.  **INSERTED SEQUENCE**: The actual sequence inserted.
    *   `.`: Represents no insertion (e.g., in the case of a pure deletion).

## Expert Tips
*   **RefSeq Mapping**: If your VCF uses chromosome aliases (like `1` or `chr1`) instead of proper RefSeq accessions (like `NC_000001.10`), you must use `cthreepo` after running `vcf2variants` to map the IDs.
*   **Piping**: For efficiency, you can pipe the `bcftools` output directly into `vcf2variants` to avoid creating large intermediate files.
*   **Varda2 Submission**: Once the `.varda` file is generated, use the `varda2-client` to submit the variants along with the corresponding coverage file generated by `gvcf2coverage`.
