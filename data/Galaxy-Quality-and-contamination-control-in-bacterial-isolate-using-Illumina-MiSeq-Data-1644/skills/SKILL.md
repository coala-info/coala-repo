# Quality and contamination control in bacterial isolate using Illumina MiSeq Data

## Overview

This workflow provides a standardized pipeline for the quality assessment and taxonomic profiling of bacterial isolate sequencing data generated by Illumina MiSeq. It begins by evaluating raw paired-end reads using [Falco](https://toolshed.g2.bx.psu.edu/repos/iuc/falco/falco/1.2.4+galaxy0), a high-performance alternative to FastQC, to identify sequencing artifacts and quality issues.

Following the initial assessment, [fastp](https://toolshed.g2.bx.psu.edu/repos/iuc/fastp/fastp/0.23.2+galaxy0) is utilized for automated adapter trimming and quality filtering. The processed reads are then analyzed by [Kraken2](https://toolshed.g2.bx.psu.edu/repos/iuc/kraken2/kraken2/2.1.1+galaxy1) to assign taxonomic labels, which is essential for detecting potential sample contamination or verifying the identity of the bacterial isolate.

To refine the taxonomic results, [Bracken](https://toolshed.g2.bx.psu.edu/repos/iuc/bracken/est_abundance/2.8+galaxy0) estimates species-level abundance based on the Kraken2 classifications. The final stage uses [Recentrifuge](https://toolshed.g2.bx.psu.edu/repos/iuc/recentrifuge/recentrifuge/1.12.1+galaxy0) to generate interactive visualizations and detailed statistical reports, offering a clear overview of the sample's purity and composition.

This [GTN](https://training.galaxyproject.org/)-related workflow is licensed under GPL-3.0-or-later and is tagged for sequence-analysis, providing a robust framework for microbial genomics quality control.

Ensure your input files are in fastq.gz format and correctly paired to maintain data integrity during the quality control and taxonomic classification steps. While individual datasets are used here, organizing large batches into paired-end collections can significantly streamline the workflow execution. Refer to the README.md for comprehensive details on parameter settings and expected contamination thresholds. You can automate the setup of these inputs by using `planemo workflow_job_init` to generate a `job.yml` file. Always verify that your Kraken2 database matches the target bacterial taxa to ensure accurate Bracken abundance estimations.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run quality-and-contamination-control-in-bacterial-isolate-using-illumina-miseq-data.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run quality-and-contamination-control-in-bacterial-isolate-using-illumina-miseq-data.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run quality-and-contamination-control-in-bacterial-isolate-using-illumina-miseq-data.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init quality-and-contamination-control-in-bacterial-isolate-using-illumina-miseq-data.ga -o job.yml`
- Lint: `planemo workflow_lint quality-and-contamination-control-in-bacterial-isolate-using-illumina-miseq-data.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
