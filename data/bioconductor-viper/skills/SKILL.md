# bioconductor-viper

## Overview
The `viper` package implements the VIPER (Virtual Inference of Protein-activity by Enriched Regulon analysis) algorithm. It uses the expression of known downstream targets of a protein (its "regulon") as a reporter for that protein's activity. This allows for the computational inference of protein activity—including transcription factors and signal transduction proteins—on an individual sample basis, even when mRNA levels of the regulator itself are not predictive of its functional state.

## Core Workflow

### 1. Data Preparation and Regulon Generation
VIPER requires a gene expression signature (or matrix) and a cell context-specific regulatory network (a `regulon` object).

```R
library(viper)
library(bcellViper)

# Load example data
data(bcellViper)

# Generate a regulon object from an ARACNe .adj file
```

### 2. Master Regulator Analysis (msVIPER)
Use `msVIPER` for multi-sample analysis (e.g., comparing cases vs. controls) to identify top master regulators.

```R
# 1. Generate a Gene Expression Signature (GES) using Student's t-test
signature <- rowTtest(dset, "description", c("CB", "CC"), "N")
# Convert p-values to z-scores
ges <- (qnorm(signature$p.value/2, lower.tail = FALSE) * sign(signature$statistic))[, 1]

# 2. Generate a NULL model by sample permutations (to account for gene correlations)
nullmodel <- ttestNull(dset, "description", c("CB", "CC"), "N", per = 1000)

# 3. Run msVIPER
mrs <- msviper(ges, regulon, nullmodel)

# 4. Summarize and Plot
summary(mrs)
plot(mrs)
```

### 3. Single Sample VIPER
Transform a gene expression matrix into a protein activity matrix.

```R
# Basic implementation (scaling genes)
vpres <- viper(dset, regulon)

# Advanced: Using a reference set for signatures
vpsig <- viperSignature(dset, "description", "N")
vpres <- viper(vpsig, regulon)

# The result is a matrix of Normalized Enrichment Scores (NES)
# Rows = Regulators, Cols = Samples
activity_matrix <- exprs(vpres)
```

## Advanced Analyses

### Shadow Analysis
Determines if a regulator's activity is simply a "shadow" of another regulator due to shared targets.
```R
mrshadow <- shadow(mrs, regulators = 25)
summary(mrshadow)
```

### Synergy Analysis
Identifies pairs of regulators that synergistically regulate a gene expression signature.
```R
mrs <- msviperCombinatorial(mrs, regulators = 25)
mrs <- msviperSynergy(mrs)
plot(mrs)
```

### Leading-Edge Analysis
Identifies the specific target genes driving the enrichment score for a regulator.
```R
mrs <- ledge(mrs)
summary(mrs) # Shows 'Ledge' column with driving genes
```

## Tips for Success
*   **Regulon Quality:** The accuracy of VIPER depends heavily on the quality of the regulatory network. Use ARACNe-inferred networks specific to the tissue/cell type being studied.
*   **Signature Scaling:** When running `viper()` on single samples, ensure the data is properly normalized. Using `viperSignature` with a set of reference (normal) samples is recommended over simple row-scaling.
*   **Distance Metrics:** For protein activity matrices generated by VIPER, use `viperSimilarity()` to compute sample similarities. This is more robust than standard correlation for NES values.
