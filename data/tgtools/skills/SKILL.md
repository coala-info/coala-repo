# tgtools

## Overview

tgtools is a command-line utility designed to facilitate the creation and manipulation of `transjson` (.trjson) files. These files serve as a standardized format for storing transmission and relatedness networks in pathogen genomics. The tool acts as a converter, taking raw outputs from various bioinformatic pipelines and structuring them into a node-and-edge JSON format that includes metadata for both samples and their relationships.

## Installation

Install via Bioconda for the most stable environment:

```bash
conda install -c bioconda tgtools
```

## Core Workflows

### Importing SNP Distance Matrices
Convert a tab-delimited matrix of pairwise SNP distances into a transmission network. The input file must have sample names in the first row and first column.

```bash
tgtools import snp-distance -i distances.tab -o network.trjson -d <MAX_DISTANCE>
```
*   **Expert Tip**: Set `-d` to the maximum SNP threshold considered "related" (e.g., 12 for M. tuberculosis) to prevent the creation of an over-saturated network.

### Importing TransPhylo Results
Convert transmission probability matrices generated by TransPhylo.

```bash
tgtools import transphylo -i probabilities.csv -o network.trjson -p <MIN_PROBABILITY>
```
*   **Expert Tip**: Use a higher `-p` value (e.g., 0.5 or 0.8) to focus on high-confidence transmission events and reduce visual clutter in downstream analysis.

### Importing hmmIBD (Identity By Descent) Data
Process outputs from `hmmIBD` to represent relatedness based on IBD values.

```bash
tgtools import isorelate -i hmmibd_output.txt -o network.trjson -d <MIN_IBD>
```

## Data Structure Reference

The resulting `.trjson` file follows a specific schema that Claude can use for further analysis or visualization scripts:

*   **Nodes**: Contain a unique `id` (sample name) and a `metadata` dictionary (e.g., geographic location, collection date).
*   **Edges**: Contain `source`, `target`, and `metadata` (e.g., `probability` or `snp_distance`).

## Best Practices

1.  **Input Validation**: Ensure your input matrices are strictly tab-delimited for `snp-distance`. Non-numeric values in the distance matrix (other than headers) will cause the import to fail.
2.  **Metadata Enrichment**: While the basic importers create the structure, the `.trjson` format is designed to hold arbitrary metadata. After conversion, you can programmatically add epidemiological data to the `nodes` list to enhance the network.
3.  **Filtering**: Always apply thresholds (`-d`, `-p`) during the import phase rather than trying to filter the resulting JSON, as this keeps the file sizes manageable and the networks interpretable.
