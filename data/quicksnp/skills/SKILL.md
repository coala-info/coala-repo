# quicksnp

## Overview

QuickSNP is a streamlined Python utility used to construct Neighbor Joining trees from SNP distance data. Unlike complex phylogenetic suites that handle alignment and distance calculation, QuickSNP focuses exclusively on the tree-building step. It takes a pre-computed square distance matrix as input and outputs a standard Newick tree file. This makes it an ideal tool for rapid iterations or for use in pipelines where distance matrices are generated by external tools.

## Usage and Best Practices

### Basic Command Syntax
The tool follows a simple positional argument structure:
```bash
QuickSNP <distance_matrix> <output_tree>
```

### Input Requirements
To ensure successful tree generation, the input distance matrix must adhere to the following:
*   **Format**: Must be a CSV or TSV file.
*   **Structure**: The matrix must be "square" (N x N), meaning the number of rows must match the number of columns.
*   **Labels**: The file must include both column headers and row names. These labels will be used as the leaf nodes in the resulting Newick tree.
*   **Values**: Ensure the matrix contains numerical SNP distances.

### Output Handling
*   The output is a standard Newick format tree file.
*   This file can be directly imported into visualization tools such as FigTree, iTOL, or R packages like `ggtree` and `ape`.

### Expert Tips
*   **Environment**: If running via Conda, ensure the `bioconda` channel is enabled. The tool relies on `scikit-bio` for the underlying NJ algorithm implementation.
*   **Data Cleaning**: Before running QuickSNP, verify that there are no missing values (NaN) in your distance matrix, as this will cause the script to fail during the tree construction phase.
*   **Large Datasets**: While "quick," the Neighbor Joining algorithm has a computational complexity of O(n³). For extremely large matrices (thousands of samples), ensure the environment has sufficient memory to load the matrix into a pandas DataFrame.
