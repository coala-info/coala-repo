## Overview
The treekin skill is designed to compute the dynamics of molecular folding on a coarse-grained energy landscape. It achieves this by numerically integrating a Markov process. This is particularly useful for analyzing complex systems like RNA secondary structures, where understanding the energy landscape, local minima, and energy barriers is crucial for predicting folding pathways and dynamics. The tool can also be used to generate barrier trees and compute transition rates between different conformational states.

## Usage Instructions

Treekin is a command-line tool used for analyzing energy landscapes and computing folding dynamics. It typically takes an energy-sorted list of conformations as input and outputs information about local minima, energy barriers, and potentially transition rates.

### Core Functionality

The primary function of `treekin` is to process an energy landscape to identify key features like local minima and the barriers between them. This is often achieved by providing an input file that lists conformations and their corresponding energies.

### Input Data Format

The input to `treekin` is an "energy sorted list of conformations." For RNA secondary structures, this input is often generated by tools like `RNAsubopt`. The exact format may vary, but it generally includes the conformation (e.g., a sequence or structural representation) and its associated energy.

### Key Command-Line Options

Consult the man page for a comprehensive list of options. Here are some commonly used ones:

*   **Input File**: Specify the landscape data file as an argument.
    ```bash
    treekin [options] [FILE]
    ```
*   **Verbose Output (`-v`, `--verbose`)**: Provides more detailed information during execution.
*   **Quiet Output (`-q`, `--quiet`)**: Suppresses standard output, including the PostScript tree drawing.
*   **Graph Type (`-G`, `--graph`)**: Defines the type of configuration space (e.g., 'RNA', 'RNA-noLP', 'Q2', 'T', 'P'). The default is 'RNA'.
    ```bash
    treekin -G RNA input.dat
    ```
*   **Move Set (`-M`, `--moves`)**: Selects the move-set for generating neighbors of a configuration. This depends on the chosen graph. For 'RNA' graphs, common options are `[no]Shift` and `ligand`.
    ```bash
    treekin -G RNA -M Shift input.dat
    ```
*   **Compute Rates (`--rates`)**: Computes transition rates between macro states (basins). This generates `rates.out` (ASCII) and `rates.bin` (binary) files.
    ```bash
    treekin --rates input.dat
    ```
*   **Temperature (`-T`, `--temp`)**: Sets the temperature in degrees Celsius, used for calculating Boltzmann factors when `--rates` is enabled.
    ```bash
    treekin --rates -T 37.0 input.dat
    ```
*   **Print Basin Sizes (`--bsize`)**: Includes the size of each basin in the output.
*   **Print Saddle Point Conformations (`--saddle`)**: Outputs the saddle point conformations.
*   **Max Minima (`--max`)**: Computes only the lowest `<num>` local minima.
*   **Min Height (`--minh`)**: Filters output to include only minima with an energy barrier greater than `<delta>`.

### Example Workflow

1.  **Generate input data**: Use a tool like `RNAsubopt` to generate an energy-sorted list of RNA secondary structures.
    ```bash
    RNAsubopt -p 100 -e -o structures.out -T 37.0 -o seq.dbn
    ```
    *(Note: `RNAsubopt` command is illustrative; actual parameters may vary.)*

2.  **Run treekin**: Process the generated data to find minima and barriers.
    ```bash
    treekin -v --rates --temp 37.0 structures.out
    ```
    This command will:
    *   Read `structures.out` as the input landscape.
    *   Provide verbose output (`-v`).
    *   Compute transition rates between basins (`--rates`).
    *   Use a temperature of 37.0 degrees Celsius (`--temp 37.0`).

3.  **Analyze output**: Examine the standard output for minima and barrier information, and the generated `rates.out` and `rates.bin` files for transition rates. A `tree.ps` file may also be generated for visualization.

### Expert Tips

*   **Input File Generation**: Ensure your input file is correctly formatted and energy-sorted. Tools like `RNAsubopt` are designed for this purpose with RNA structures.
*   **Graph and Move Set Selection**: The choice of graph (`-G`) and move set (`-M`) is critical and depends on the nature of the system you are modeling. For RNA, 'RNA' and 'RNA-noLP' are common graph types. Consult the documentation for available options for other systems.
*   **Transition Rates**: The `--rates` option is powerful for understanding the kinetics of folding. Remember to set the appropriate temperature (`-T`) for your system.
*   **Visualization**: Treekin can generate a PostScript file (`tree.ps`) visualizing the barrier tree. This is invaluable for understanding the landscape topology. Use a PostScript viewer to inspect it.
*   **Memory Usage**: For very large landscapes, consider the `--with-hash-bits` configure flag and `CFLAGS+="-mcmodel=large"` during compilation if memory becomes a bottleneck, as detailed in the documentation.
