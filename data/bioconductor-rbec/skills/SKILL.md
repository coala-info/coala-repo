# bioconductor-rbec

## Overview

Rbec (Reference-based error correction) is a specialized tool for analyzing amplicon sequencing data from synthetic microbial communities (SynComs). Unlike de novo OTU or ASV pickers, Rbec leverages the known reference sequences of the strains used in the SynCom to accurately correct PCR and sequencing errors. It is particularly effective at distinguishing between true intra-species variations and technical artifacts, while also providing a robust mechanism for identifying environmental contaminants.

## Core Workflow

### 1. Reference-Based Error Correction
The primary function `Rbec()` processes raw FASTQ files against a set of known reference sequences.

```r
library(Rbec)

# Define paths to input files
fq_file <- "path/to/sample_reads.fastq.gz"
ref_fasta <- "path/to/reference_strains.fasta"
output_dir <- "results/rbec_output"

# Run the error correction
# Parameters: fastq, reference, outdir, kmer_size, max_contig_len, phred64_offset
Rbec(fq_file, ref_fasta, output_dir, 1, 500, 33)
```

**Key Parameters:**
- `kmer_size`: Size of k-mers used for initial alignment (default is often 1).
- `max_contig_len`: Maximum length of the amplicon sequences.
- `phred`: The FASTQ quality encoding (usually 33 for modern Illumina data).

### 2. Community Profiling
Rbec generates a composition matrix for the samples. The output directory will contain:
- Corrected sequences.
- Abundance tables for each strain in the reference.
- Log files detailing the recruitment of reads to each reference.

### 3. Contamination Detection
Rbec identifies potential contaminants by analyzing the "recruitment ratio"—the proportion of reads that could be successfully mapped and corrected against the provided reference database.

```r
# 1. Collect paths to all log files generated by Rbec()
log_paths <- list.files("results/rbec_output", pattern = ".log", full.names = TRUE)

# 2. Write paths to a temporary file
log_list_file <- tempfile()
writeLines(log_paths, log_list_file)

# 3. Run detection
# This generates a distribution plot and a list of predicted contaminated samples
Contam_detect(log_list_file, "results/contamination_analysis")
```

**Interpretation:**
- **Clean Samples:** Typically have >90% of reads successfully corrected/recruited.
- **Contaminated Samples:** Show a significant drop in the recruitment ratio, suggesting the presence of DNA not matching the SynCom references.

## Usage Tips
- **Reference Quality:** Ensure the `ref.fasta` contains all possible strains in the SynCom, including known intra-species variants if high resolution is required.
- **Log Files:** Always retain the `.log` files produced by the `Rbec()` function, as they are mandatory inputs for the contamination detection workflow.
- **Memory/Time:** For large datasets, ensure the `outdir` is on a filesystem with sufficient space, as Rbec generates detailed intermediate log and sequence files.
