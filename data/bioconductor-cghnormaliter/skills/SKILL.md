# bioconductor-cghnormaliter

name: bioconductor-cghnormaliter
description: Normalization of two-channel array Comparative Genomic Hybridization (aCGH) data using the CGHnormaliter strategy. Use this skill when analyzing aCGH data with many copy number imbalances (aberrations) that require a "local-LOWESS" normalization approach to prevent gains and losses from biasing the normalization of normal regions.

# bioconductor-cghnormaliter

## Overview

`CGHnormaliter` provides an iterative strategy for normalizing aCGH data. Standard LOWESS normalization can be biased when a genome contains many copy number aberrations (CNAs). This package addresses this by:
1. Segmenting the data (using `DNAcopy`).
2. Calling aberrations (using `CGHcall`) to identify "normal" regions.
3. Applying LOWESS normalization using only the identified normal regions.
4. Iterating these steps until convergence to refine the normalization.

## Data Preparation

The input must be a `data.frame` or a tab-separated text file with a header.

### Required Columns
1. **ID**: Unique identifier for array elements.
2. **Chromosome**: Chromosome number.
3. **Start**: Chromosomal start position (numeric, bp).
4. **End**: Chromosomal end position (numeric, bp).

### Sample Columns
Following the first four columns, each sample must have **two adjacent columns**:
- `Sample.test`: Test intensity.
- `Sample.ref`: Reference intensity.

Missing values should be denoted as `NA`.

## Core Workflow

### 1. Loading the Package and Data
```r
library(CGHnormaliter)
# Example dataset
data(Leukemia)
```

### 2. Running Normalization
The primary function is `CGHnormaliter`. It automates the iterative segmentation, calling, and normalization process.

```r
# Basic usage
# nchrom: number of chromosomes to process (optional)
# cellularity: fraction of tumor cells in the sample (default is 1.0)
result <- CGHnormaliter(Leukemia, nchrom=23, cellularity=1.0)
```

### 3. Accessing Results
The output is a `cghRaw`, `cghSeg`, or `cghCall` object (depending on the stage) from which data can be extracted:

```r
# Extract normalized log2 ratios
normalized.data <- copynumber(result)

# Extract segmented data
segmented.data <- segmented(result)

# Extract copy number calls (-1: loss, 0: normal, 1: gain, 2: amplification)
called.data <- calls(result)
```

### 4. Visualization
The package provides built-in plotting to assess normalization quality.

```r
# MA-plots are generated by default during the run (MAplot.pdf)

# Density plot to check centralization around zero
plot(density(normalized.data[, 1]), main="Sample 1 Density")
abline(v=0, lty=2)

# Profile plot of results
# ylimit: range of log2 ratios
# dotres: plot every Nth probe (1 for full resolution)
plot(result[, 1], ylimit=c(-2, 2), dotres=1)
```

### 5. Exporting Data
Use the specialized write function to save results to text files.

```r
# Save normalized log2 ratios
CGHnormaliter.write.table(result, data.type="normalized")

# Save segmented data
CGHnormaliter.write.table(result, data.type="segmented")

# Save calls
CGHnormaliter.write.table(result, data.type="called")
```

## Tips and Best Practices
- **Convergence**: The algorithm typically converges in 2-5 iterations. If it fails to converge, check if the `cellularity` parameter needs adjustment.
- **MA-Plots**: Always inspect the `MAplot.pdf` generated by the tool. In the "After normalization" plots, the black dots (identified normals) should be centered around the M=0 line.
- **Memory**: For very large arrays, processing many samples simultaneously can be memory-intensive due to the underlying `CGHcall` and `DNAcopy` steps.
