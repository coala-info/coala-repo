# Qimba Developer Documentation

## Overview

Qimba is a modular bioinformatics toolkit focused on amplicon processing. This document provides technical details for developers who want to understand or extend Qimba's functionality.

## Core Components

### Configuration Management

The configuration system uses Python's `configparser` and manages settings through an INI file:

* Default location: `~/.config/qimba.ini`
* Default settings:
  + `default_output_dir = .`
  + `threads = 4`

### Job Execution System

The `Job` class (`qimba.core.Job`) handles external command execution with built-in validation and logging:

```
job = Job(
    command=['program', '--arg', 'value'],
    required_input=['input.txt'],
    required_output=['output.txt'],
    log_stderr='error.log',
    log_stdout='output.log'
)
```

Key features:

* Input/output file validation
* Automatic log directory creation
* Command logging with timestamps
* Proper file handle management
* Shell and non-shell command execution support

### Sample Management

⚠️ Mapping and SampleSheet must be merged

#### Mapping Class

`qimba.core.Mapping` handles sample metadata:

* Required columns: Sample ID, Forward, Reverse
* Validates file existence
* Supports arbitrary additional columns as sample attributes
* Methods for attribute access and filtering

#### SampleSheet Class

`qimba.formats.SampleSheet` provides a more flexible sample management system:

* Supports optional reverse reads
* Includes sorting capabilities (by sample ID or forward read path)
* Methods for file I/O with absolute/relative path options
* Sample attribute management

```
sheet = SampleSheet()
sheet.add_sample("sample1", "reads_R1.fq", "reads_R2.fq", {"group": "A"})
sheet.save_to_file("mapping.tsv", absolute_paths=True)
```

### FASTA Processing

The `parse_fasta` utility (`qimba.utils`) provides a memory-efficient FASTA parser:

```
for seq_id, comments, sequence in parse_fasta("input.fasta"):
    # Process each sequence
```

## Utility Functions

### Temporary Directory Management

`make_temp_dir` creates managed temporary directories:

```
temp_dir = make_temp_dir(parent_dir='/tmp', prefix='qimba_')
```

### Log Analysis

`extract_from_log` parses log files using regex patterns:

```
results = extract_from_log("logfile.txt", r"Pattern with (\d+) (groups)")
```
