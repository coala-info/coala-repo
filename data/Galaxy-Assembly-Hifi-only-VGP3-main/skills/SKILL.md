# genome-assembly-from-hifi-reads-vgp3

## Overview

This workflow generates high-quality genome assemblies from PacBio HiFi reads as part of the Vertebrate Genomes Project (VGP) suite. It is designed as the third stage (VGP3) in the pipeline and requires outputs from the VGP1 k-mer profiling workflow—specifically a Meryl database and GenomeScope parameters—to guide the assembly process.

The core assembly is performed using [Hifiasm](https://github.com/chhylp123/hifiasm), which produces both primary and alternate contigs in FASTA and GFA formats. The pipeline incorporates [Cutadapt](https://github.com/marcelm/cutadapt) for initial read processing and utilizes [Bandage](https://github.com/rrwick/Bandage) to visualize the raw unitig graph, providing immediate insight into the assembly's connectivity and complexity.

Comprehensive quality control is integrated throughout the process to evaluate the resulting contigs. The workflow generates [Merqury](https://github.com/marbl/merqury) k-mer spectra plots and completeness statistics, [Compleasm](https://github.com/huangnengCSU/compleasm) reports for gene conservation, and detailed assembly statistics via [gfastats](https://github.com/GFA-spec/gfastats). These outputs, including Nx and size plots, ensure a rigorous evaluation of both the primary and alternate assembly sets.

| 4 | Genomescope Summary | data_input |  |
| 5 | Bits for Hifiasm bloom filter | parameter_input | Defaults to 37 if not specified. For genomes much larger than human, applying -f38 or even -f39 is preferred to save memory on k-mer counting. |
| 6 | Homozygous Read Coverage | parameter_input | If empty, read coverage will be estimated from the Genomescope parameters. |
| 7 | Genomescope Model Parameters | data_input | GenomeScope model parameters generated by K-mer profiling workflow |

| 9 | Lineage | parameter_input | Taxonomic lineage for the organism being assembled for Busco analysis |
| 10 | Name of primary assembly | parameter_input |  |
| 11 | Name of alternate assembly | parameter_input |  |


Ensure PacBio HiFi reads are organized into a dataset collection in FASTQ format, while the k-mer database must be provided in the specific `meryldb` format. You will need the summary and model parameter files generated during the VGP1 k-mer profiling stage to correctly configure the assembly parameters. For a complete breakdown of all input requirements and parameter definitions, please consult the README.md. Automated job configuration can be streamlined by using `planemo workflow_job_init` to create a `job.yml` file.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Assembly-Hifi-only-VGP3.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Assembly-Hifi-only-VGP3.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Assembly-Hifi-only-VGP3.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Assembly-Hifi-only-VGP3.ga -o job.yml`
- Lint: `planemo workflow_lint Assembly-Hifi-only-VGP3.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
