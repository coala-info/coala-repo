# tbpore

## Overview

tbpore is a specialized bioinformatics pipeline designed to transform raw Nanopore sequencing data from *Mycobacterium tuberculosis* into actionable genomic insights. It automates the complex workflow of read decontamination, variant calling against the canonical H37Rv reference genome, and drug susceptibility testing (DST). Additionally, it provides utilities for molecular epidemiology by clustering samples based on a user-defined SNP distance threshold.

## Core Workflows

### 1. Initial Setup
Before processing samples, you must download and validate the decontamination database index. By default, this is stored in `${HOME}/.tbpore/`.

```bash
tbpore download
```
*   Use `-o <path>` to specify a custom location.
*   Use `-f` to force an overwrite if the database already exists.

### 2. Single-Sample Processing
The `process` command handles decontamination, variant calling (via bcftools), and resistance prediction (via Mykrobe).

```bash
tbpore process --name SampleID --outdir ./results/ input_reads.fastq.gz
```

**Expert Tips for Processing:**
*   **Input Handling**: You can provide multiple FASTQ files or a directory. Use `--recursive` (`-r`) to search directories for FASTQ files automatically.
*   **Subsampling**: Use `--coverage` (`-c`) to subsample reads to a specific depth (e.g., `-c 50`). This is highly recommended for high-depth Nanopore runs to reduce runtime without sacrificing accuracy. Set to `0` to disable.
*   **Custom Databases**: If you downloaded the decontamination DB to a non-default path, you must specify it using `--db <path>`.
*   **Performance**: The tool defaults to 1 thread. Increase this with `--threads` (`-t`) for faster mapping and variant calling. Expect ~13GB RAM usage per sample.

### 3. Sample Clustering
Cluster consensus sequences generated by `tbpore process` to identify potential transmission chains or outbreaks.

```bash
tbpore cluster --threshold 6 --outdir ./clusters/ output/sample_*/*.consensus.fa
```

**Expert Tips for Clustering:**
*   **Thresholds**: The default SNP threshold is 6. Adjust this based on the specific epidemiological context of your investigation.
*   **Inputs**: Use glob patterns to easily include multiple consensus files from different sample directories.

## Common CLI Patterns

| Task | Command Pattern |
| :--- | :--- |
| **Fast Processing** | `tbpore process -t 8 -c 50 --name MySample reads/` |
| **Outbreak Analysis** | `tbpore cluster -T 12 -o outbreak_analysis/ */*.consensus.fa` |
| **Clean Run** | `tbpore process --cleanup --name SampleA sampleA.fq` |
| **Verbose Debugging** | `tbpore -v process [ARGS]` |

## Best Practices
*   **Sample Naming**: If `--name` is not provided, `tbpore` derives the name from the first input file. Explicitly naming samples prevents confusion in multi-sample projects.
*   **Cleanup**: Use the `--cleanup` (`-d`) flag to remove large temporary files (like intermediate BAMs) upon successful completion to save disk space.
*   **Mykrobe Reports**: To see all resistance calls (including low-frequency variants), use the `--report_all_mykrobe_calls` (`-A`) flag.
