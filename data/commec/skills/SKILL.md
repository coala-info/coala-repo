# commec

## Overview

The `commec` tool is a specialized biosecurity utility designed to screen DNA sequences for potential risks. It is particularly useful for synthesis providers or researchers who need to ensure sequences do not match regulated pathogens or known pathogenicity factors. The tool operates by running sequences through a multi-stage pipeline involving HMM-based biorisk searches, BLAST-based taxonomy searches against NCBI databases, and a filtering step to remove "low concern" matches (common or conserved sequences).

## Command Line Usage

### Core Commands

- **`commec setup`**: Run this first to download the required reference databases. Note that a full installation for regulatory compliance requires significant storage (275-650 GB).
- **`commec screen <input.fasta>`**: The primary command to analyze DNA sequences. It processes the FASTA file through the biorisk and taxonomy pipelines.
- **`commec flag`**: Used to parse the `.screen.json` output files generated by the screen command. It aggregates results into a CSV file for easier review of outcomes.
- **`commec split <multi-record.fasta>`**: A utility to break a large multi-record FASTA file into individual files, which can be useful for parallel processing or granular analysis.

### Screening Workflow

1.  **Initialization**: Ensure databases are ready using `commec setup`.
2.  **Execution**: Run `commec screen` on your target sequences.
    - *Lightweight Mode*: Focuses on biorisk searches (HMM-based). Requires < 1 GB of databases.
    - *Comprehensive Mode*: Includes taxonomy searches (BLASTX/DIAMOND against NCBI nr and BLASTN against core_nt). Requires high-performance computing (20 GB+ RAM).
3.  **Reporting**: Use `commec flag` on the output directory to generate a summary CSV of flagged sequences.

### Best Practices

- **Sequence Length**: `commec` is optimized to screen sequences as short as 50 base pairs.
- **Environment**: For full taxonomy screening, use a high-performance bioinformatics environment due to the size of the NCBI reference databases.
- **Interpretation**: Results are output as `.screen.json` files. Always use the `flag` command to convert these into a human-readable CSV format for final assessment.
