# ratt

## Overview
RATT (Rapid Annotation Transfer Tool) is designed to efficiently transfer existing gene annotations from a reference genome to a query genome. This is invaluable when you have a well-annotated reference and want to quickly annotate a closely related, but unannotated, genome. It supports transfers between different assemblies of the same organism, between different strains, and even between closely related species. RATT aims to preserve detailed annotation information, such as systematic IDs and notes, which are crucial for comparative genomics and functional analysis.

## Usage Instructions

RATT automates the process of transferring annotations by leveraging synteny-based alignment. The core command structure involves specifying the reference annotation directory, the query genome sequence, and a transfer type.

### Basic Command Structure

```bash
ratt \
  { -p | --prefix } result-prefix \
  { -t | --type } transfer-type \
  [{ -r | --refseq } reference.fasta ] \
  reference-annotation-directory \
  query.fasta
```

### Key Arguments:

*   **`reference-annotation-directory`**: (Required) The directory containing all reference annotation files. These files *must* be in EMBL format.
*   **`query.fasta`**: (Required) The nucleotide sequence (in FASTA format) to which the reference annotations will be mapped.
*   **`-p, --prefix result-prefix`**: (Required) A prefix for all output files generated by RATT.
*   **`-t, --type transfer-type`**: (Required) Specifies the type of annotation transfer. Choose the type that best reflects the relationship between your reference and query genomes:
    *   **`Assembly`**: For transferring annotations between different assembly versions of the same genome.
    *   **`Assembly.Repetitive`**: Use if the `Assembly` type misses too many annotations due to a highly repetitive genome.
    *   **`Strain`**: For transferring annotations between different strains of the same species (typically 95-99% similarity).
    *   **`Strain.Repetitive`**: Use if the `Strain` type misses too many annotations due to a highly repetitive genome.
    *   **`Strain.Global`**: An alternative for strain transfer if the assembly has few gaps and rearrangements.
    *   **`Species`**: For transferring annotations between different species (typically 50-94% similarity).
    *   **`Species.Repetitive`**: Use if the `Species` type misses too many annotations due to a highly repetitive genome.
    *   **`Species.Global`**: An alternative for species transfer if the assembly has few gaps and rearrangements.
    *   **`Multiple`**: Use when the query genome has many insertions compared to multiple annotated reference strains. RATT will select the best regions from each reference.
    *   **`Free`**: Allows manual specification of all underlying `nucmer` parameters via environment variables (e.g., `RATT_l`, `RATT_minInd`, `RATT_c`, `RATT_g`, `RATT_anchor`, `RATT_rearrange`).
*   **`-r, --refseq reference.fasta`**: (Optional) A multi-FASTA file containing the reference sequences. **Crucially**, the FASTA header names in this file must exactly match the corresponding EMBL annotation filenames in the `reference-annotation-directory`.

### Environment Variables

*   **`RATT_HOME`**: (Required) Must be set to the path of your RATT installation directory.
*   **`RATT_CONFIG`**: (Optional) Path to a custom configuration file. Default configuration files (`RATT.config_bac`, `RATT.config_euk`) are provided for bacteria and eukaryotes, respectively, and may need adjustment for specific start codons or splice sites.
*   **`RATT_VERBOSE`**: (Optional) If defined, RATT will report its invocations of external programs (like `nucmer`).
*   **`RATT_DOTRANSLATION`**: (Optional) Set to `1` to include protein amino acid sequences in the output.

### Expert Tips and Best Practices

*   **EMBL Format is Key**: Ensure your reference annotation files are strictly in EMBL format. RATT relies on this format for parsing annotation details.
*   **Header Matching**: When using the `--refseq` option, meticulously ensure that the FASTA headers in `reference.fasta` precisely match the filenames of the corresponding EMBL annotation files. Any mismatch will lead to errors.
*   **Choosing the Right `--type`**: The selection of the `transfer-type` is critical for performance and accuracy. Start with the most specific type (e.g., `Assembly` for different assemblies) and only move to more general or repetitive types if results are insufficient. The `Multiple` type is powerful but requires careful selection of reference strains.
*   **Configuration Files**: For optimal results, especially with non-standard genomes, customize the `RATT.config` file. Pay attention to start codons and splice site definitions.
*   **Output Analysis**: RATT provides statistics on transferred tags, differences, and new insertions. Analyze these outputs using tools like Artemis and ACT for a comprehensive understanding of the annotation transfer.
*   **`RATT_HOME` Setup**: Always ensure the `RATT_HOME` environment variable is correctly set to the RATT installation directory before running the tool.
