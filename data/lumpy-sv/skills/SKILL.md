# lumpy-sv

## Overview

Lumpy-sv is a probabilistic framework designed to detect structural variants by integrating multiple signals from high-throughput sequencing data. It is particularly effective because it can combine evidence from various libraries and discovery types (like split-reads and discordant pairs) into a single statistical model. While the core `lumpy` tool offers high flexibility for advanced users, the `lumpyexpress` wrapper provides a streamlined, best-practice workflow for standard analyses.

## Installation

The most reliable way to install lumpy-sv is via Bioconda:

```bash
conda install -c bioconda lumpy-sv
```

Alternatively, it can be built from source using `make`.

## Pre-processing Requirements

Lumpy-sv requires specific input signals extracted from your primary BAM files. The input BAMs must be coordinate-sorted and contain proper `@RG` (Read Group) tags in the header.

### Extracting Evidence
If you are not using a wrapper like `speedseq`, you must manually extract discordant and split reads:

1. **Discordant Reads**: Pairs that do not map with the expected orientation or distance.
   ```bash
   samtools view -b -F 1294 sample.bam > sample.discordants.unsorted.bam
   ```
2. **Split Reads**: Reads that map to non-contiguous genomic locations.
   ```bash
   samtools view -h sample.bam | extractSplitReads_BwaMem -i stdin | samtools view -Sb - > sample.splitters.unsorted.bam
   ```
3. **Sort Evidence**:
   ```bash
   samtools sort sample.discordants.unsorted.bam -o sample.discordants.bam
   samtools sort sample.splitters.unsorted.bam -o sample.splitters.bam
   ```

## Common CLI Patterns

### Standard Analysis (lumpyexpress)
For most users, `lumpyexpress` is the recommended interface. It automates the calculation of library statistics and runs the core engine.

```bash
lumpyexpress \
    -B sample.bam \
    -S sample.splitters.bam \
    -D sample.discordants.bam \
    -o output.vcf
```

### Joint Calling (Multi-sample)
To call variants across multiple samples simultaneously, provide comma-separated lists to the input flags.

```bash
lumpyexpress \
    -B s1.bam,s2.bam \
    -S s1.splitters.bam,s2.splitters.bam \
    -D s1.discordants.bam,s2.discordants.bam \
    -o multi_sample.vcf
```

### Advanced Customization (lumpy)
The traditional `lumpy` command allows for fine-grained control over probability curves and library parameters.

```bash
lumpy \
    -mw 4 \
    -tt 0 \
    -pe id:sample,bam_file:sample.discordants.bam,histo_file:sample.lib.histo,mean:300,stdev:50,read_length:101,min_non_overlap:101,discordant_z:3,back_distance:10,weight:1 \
    -sr id:sample,bam_file:sample.splitters.bam,back_distance:10,weight:1 \
    > output.vcf
```

## Expert Tips and Best Practices

- **Use Smoove**: The authors recommend `smoove` as the primary way to run lumpy. It simplifies the workflow further, handles parallelization, and implements additional filters to reduce false positives.
- **Exclude Problematic Regions**: Use the `-x` flag to provide a BED file of "blacklisted" regions (e.g., telomeres, centromeres, or high-signal regions) to significantly improve performance and accuracy.
- **Library Statistics**: If using the traditional `lumpy` command, ensure your `histo_file` (generated by `pairend_distro.py`) is accurate, as the probabilistic model relies heavily on the insert size distribution.
- **CRAM Support**: While `lumpyexpress` can accept CRAM files for the main alignment (-B), the splitters (-S) and discordants (-D) must still be in BAM format.
