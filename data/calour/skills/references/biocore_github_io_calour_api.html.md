They can be called in two manners equivalently: called as normal functions with an Experiment object as first parameter; called as a method upon the Experiment object. For example: &gt;&gt;&gt; from calour import Experiment &gt;&gt;&gt; exp = Experiment ( np . array ([[ 1 , 2 ], [ 3 , 4 ]]), sparse = False , ... sample_metadata = pd . DataFrame ({ &#39;category&#39; : [ &#39;A&#39; , &#39;B&#39; ], ... &#39;ph&#39; : [ 6.6 , 7.7 ]}, ... index = [ &#39;s1&#39; , &#39;s2&#39; ]), ... feature_metadata = pd . DataFrame ({ &#39;motile&#39; : [ &#39;y&#39; , &#39;n&#39; ]}, index = [ &#39;otu1&#39; , &#39;otu2&#39; ])) Let’s filter samples: &gt;&gt;&gt; new1 = exp . filter_samples ( &#39;category&#39; , &#39;A&#39; ) &gt;&gt;&gt; new1 Experiment ---------- data dimension: 1 samples, 2 features sample IDs: Index([&#39;s1&#39;], dtype=&#39;object&#39;) feature IDs: Index([&#39;otu1&#39;, &#39;otu2&#39;], dtype=&#39;object&#39;) Equivalently, we can filter in this way: &gt;&gt;&gt; from calour.filtering import filter_samples &gt;&gt;&gt; new2 = filter_samples ( exp , &#39;category&#39; , &#39;A&#39; ) &gt;&gt;&gt; new2 Experiment ---------- data dimension: 1 samples, 2 features sample IDs: Index([&#39;s1&#39;], dtype=&#39;object&#39;) feature IDs: Index([&#39;otu1&#39;, &#39;otu2&#39;], dtype=&#39;object&#39;) &gt;&gt;&gt; new1 == new2 True Main classes and utility functions ¶ experiment ( calour.experiment ) amplicon experiment ( calour.amplicon_experiment ) read &amp; write ( calour.io ) utilities ( calour.util ) Functions operating on Experiment object ¶ heatmap ( calour.heatmap ) other plotting functions ( calour.plotting ) filtering ( calour.filtering ) sorting ( calour.sorting ) transforming ( calour.transforming ) analysis ( calour.analysis ) manipulation ( calour.manipulation ) export_html ( calour.export_html ) machine learning ( calour.training ) database access functions ( calour.database ) Back to top &copy; Copyright 2016-2018, calour development team.
