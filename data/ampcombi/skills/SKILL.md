# ampcombi

## Overview

AMPcombi is a specialized command-line toolkit designed to harmonize the outputs of various AMP prediction programs (such as ampir, macrel, and amplify). It transforms raw prediction data into a curated dataset by performing functional alignment, calculating physiochemical properties, and clustering similar sequences into families. This skill provides the necessary procedural knowledge to execute the four primary submodules: parsing, completing, clustering, and signal peptide prediction.

## Core Workflows and CLI Patterns

### 1. Parsing and Filtering (`parse_tables`)
This is the primary entry point. It aligns sequences to reference databases and filters hits based on length and probability.

**Standard Multi-Sample Pattern:**
```bash
ampcombi parse_tables \
    --amp_results path/to/results_folder/ \
    --faa path/to/faa_files/ \
    --gbk path/to/gbk_files/ \
    --sample_list sample_1 sample_2 \
    --amp_database 'DRAMP' \
    --threads 10
```

**Expert Tips for Parsing:**
*   **Directory Structure**: When using `--amp_results`, ensure your folder is organized by `tool_name/sample_name/file`.
*   **Tool Suffixes**: Use `--<tool>_file` (e.g., `--macrel_file '.txt'`) if your prediction tools produced non-default file extensions.
*   **Database Selection**: DRAMP is the default. Use `--amp_database_dir` to point to a custom folder containing `*.fasta` files if the built-in options (DRAMP, APD, UniRef100) are insufficient.
*   **Filtering**: Adjust `--amp_cutoff` (probability) or `--aminoacid_length` (default 100) to refine your hit list.

### 2. Aggregating Results (`complete`)
Use this to merge individual sample tables generated by `parse_tables` into a single master summary.

```bash
ampcombi complete --summaries_directory path/to/parse_tables_output/
```
*   **Output**: Generates `Ampcombi_summary.tsv`, which is the required input for the clustering module.

### 3. Sequence Clustering (`cluster`)
Clusters the aggregated AMP hits into families using MMSeqs2.

```bash
ampcombi cluster --ampcombi_summary Ampcombi_summary.tsv
```

**Key Parameters:**
*   `--cluster_seq_id`: Adjust sequence identity threshold (default 0.4).
*   `--cluster_remove_singletons`: Set to `False` if you want to keep unique peptides that do not form clusters.
*   `--cluster_min_member`: Minimum number of sequences to define a cluster (default 3).

### 4. Signal Peptide Prediction (`signal_peptide`)
Predicts signal peptides on the filtered/clustered hits. Note: Requires a separate installation of SignalP.

```bash
ampcombi signal_peptide \
    --signalp_model path/to/signalp_model/ \
    --ampcombi_cluster Ampcombi_summary_cluster.tsv
```

## Installation and Environment
*   **Conda**: `conda create -n ampcombi python==3.11 mmseqs2==15.6f452 ampcombi`
*   **Docker/Singularity**: Use the pre-built image `ampcombi:0.2.2--pyhdfd78af_0`.



## Subcommands

| Command | Description |
|---------|-------------|
| ampcombi cluster | Clustering of AMPs using mmseqs2 based on the Ampcombi summary results. |
| ampcombi complete | Complete the ampcombi analysis by aggregating summary files from a directory or a list of files. |
| ampcombi parse_tables | Parse and summarize results from various AMP (Antimicrobial Peptide) prediction tools. |
| ampcombi signal_peptide | Identify signal peptides using SignalP models and Ampcombi cluster summaries. |
