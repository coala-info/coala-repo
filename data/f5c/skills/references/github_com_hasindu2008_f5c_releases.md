## What's Changed

* updated slow5lib to 1.3 to support ex-zd compressed blow5
* fix unnecessary tab being printed in resquiggle tsv output [see [#180](https://github.com/hasindu2008/f5c/issues/180) ]
* improvements to error/warning messages
* **f5c now supports AMD GPUs (ROCM/HIP)** - implemented with [@BonsonW](https://github.com/BonsonW) in [#206](https://github.com/hasindu2008/f5c/pull/206)
  Experimental ROCM binaries are also available, which require an O/S with GLIBC >=2.27 (e.g., Ubuntu 22, Red Hat 8). ROCM runtime v5.7 is included in the binaries, so a driver compatible with this would work. Tested on GPUs MI250X, Radeon 7900XTX and Radeon 780M. Help us test by running the commands below and reporting any issues:

  ```
  wget https://github.com/hasindu2008/f5c/releases/download/v1.6/f5c-v1.6-rocm-binaries-experimental.tar.gz && tar xf f5c-v1.6-rocm-binaries-experimental.tar.gz
  cd f5c-v1.6 && mv f5c_x86_64_linux_rocm f5c
  scripts/test.sh
  ```

**Full Changelog**: [v1.5...v1.6](https://github.com/hasindu2008/f5c/compare/v1.5...v1.6)

[f5c-v1.3](/hasindu2008/f5c/releases/tag/v1.3)

Changes/new features from f5c-v1.3-beta are:

1. adding a [draft 5-mer pore model for upcoming RNA004 chemistry](https://raw.githubusercontent.com/hasindu2008/f5c/v1.3/test/rna004-models/rna004.nucleotide.5mer.model). To use this model:

   First download the model file

   ```
   wget https://raw.githubusercontent.com/hasindu2008/f5c/v1.3/test/rna004-models/rna004.nucleotide.5mer.model
   ```

   Then execute event align with `--kmer-model` pointing to the path to the downloaded k-mer model as follows:

   ```
   f5c eventalign --rna -b reads.bam -r reads.fastq -g transciptome.fa -o eventalign.tsv --kmer-model /path/to/rna004.nucleotide.5mer.model  # --slow5
   reads.blow5  if using S/BLOW5 input
   ```

   Thanks [@GoekeLab](https://github.com/GoekeLab) for help with this model generation and testing.
2. Fixing a bug that affected the last line of SAM output being truncated sometimes (--sam in eventalign).

---

Compared f5c-v1.2, this f5c-v1.3 version (introduced in f5c-v1.3-beta) adds a new feature to f5c eventalign to write the output in PAF and SAM formats with signal-to-reference alignment information embedded as tags. This output is much more compact than the default TSV output, yet is sufficient for reconstructing the alignment (Note: As of this version signal samples corresponding to insertion are yet collapsed to the previous base. This will be improved in future versions). These SAM and PAF formats with signal alignment tags are the heart of the pileup view being implemented in [Squigualiser](https://github.com/hiruna72/squigualiser).

* eventalign PAF output which is explained [here](https://hasindu2008.github.io/f5c/docs/output#eventalign-paf-output) (`-c` option)
* default eventalign SAM output changed (`--sam` option). The new SAM format is explained [here](https://hasindu2008.github.io/f5c/docs/output#eventalign-sam-output). To revert to the old SAM format (same format as when Nanopolish --sam is provided), please provide `--sam-out-version 1`. The reason for the change in the SAM format is to support the upcoming pileup view in [Squigualiser](https://github.com/hiruna72/squigualiser). See the screenshots below.
* `-a` shorthand option for `--sam`
