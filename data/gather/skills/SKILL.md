# gather

## Overview
GATHeR (Graph-based Accurate Tool for Immunoglobulin HEavy- and Light-chain Reconstruction) is a specialized toolkit designed to assemble paired BCR heavy and light chains from sequence data. By utilizing de Bruijn graphs, it reconstructs full-length sequences and extends them into constant regions, enabling accurate isoform, subclass, and allele assignment. It is optimized for both plate-based (Smart-seq2/3) and droplet-based (10x Genomics Chromium 5' GEX) single-cell technologies.

## Installation and Environment Setup
The tool is primarily distributed via Bioconda.

```bash
# Create a dedicated environment
conda create -n gather-env -c bioconda gather
conda activate gather-env
```

**Critical Dependency Note:** For full functionality (annotation and phylogeny), you must run the provided setup scripts to configure external tools like IgBLAST and reference databases (IMGT):
- `setup_gather_dependencies.sh`: Installs R packages and external tools (BLAST+, SPAdes, IgPhyML).
- `setup_igblast_env.sh`: Configures the IgBLAST environment and downloads reference data.

## Command Line Usage

### 1. Single-Cell Assembly (Smart-seq)
Use `sc_asm.py` for standard paired-end or single-end FASTQ files representing individual cells.

**Paired-End:**
```bash
sc_asm.py --seq_1 {cell}_R1.fastq.gz --seq_2 {cell}_R2.fastq.gz --output_dir ./output_folder
```

**Single-End:**
```bash
sc_asm.py --seq_1 {cell}_R1.fastq.gz --output_dir ./output_folder
```

### 2. 10x Genomics Chromium 5' GEX Processing
For 10x data, use `tenx_asm.py`. This requires the `barcodes.tsv` file generated by Cell Ranger.

**Workflow:**
1. Concatenate all R1 reads into `Merged_R1.fastq.gz`.
2. Concatenate all R2 reads into `Merged_R2.fastq.gz`.
3. Run the assembly:
```bash
tenx_asm.py --barcode_file barcodes.tsv \
            --r1_fastq Merged_R1.fastq.gz \
            --r2_fastq Merged_R2.fastq.gz \
            --output_dir_name cells_output \
            --num_jobs 8
```

### 3. Post-processing and Clonality
After assembly, use `postproc.py` to perform V(D)J gene assignment, CDR3 parsing, and productivity assessment.

```bash
# Analyze Light Chains
postproc.py --bcrs_dir ./BCRs_DIR --chain light

# Analyze Heavy Chains
postproc.py --bcrs_dir ./BCRs_DIR --chain heavy
```

## Expert Tips and Best Practices

### K-mer Selection
The default k-mer size is `k=25`. 
- **Rule of thumb:** `k` should be less than `read length - 20`.
- For very short reads (e.g., 30bp), use values between 21 and 25.
- Increasing `k` can resolve ambiguities in complex regions but requires longer reads and higher coverage.

### Performance Optimization
- **CPU Allocation:** Always set `--num_jobs` to the maximum number of available CPU cores when running `tenx_asm.py` to handle the high volume of cell-specific assemblies.
- **Memory:** Assembly via SPAdes (internal to GATHeR) can be memory-intensive; ensure at least 16GB RAM is available for standard runs.

### Output Interpretation
- `*_merged.unitigs.fa`: The raw unitigs from the de Bruijn graph.
- `*_merged.contigs.fa`: The final assembled transcriptome contigs.
- `*_merged.BCR.fa`: The annotated BCR sequences. Check the headers for the algorithm source (`algo_1=SPAdes`, `algo_2=GATHeR`).
