# stranger

## Overview
The `stranger` tool bridges the gap between raw STR calls and clinical interpretation. While tools like ExpansionHunter and TRGT identify the number of repeat units at specific loci, `stranger` applies a curated catalog of known pathogenic repeats to these results. It calculates whether a specific expansion falls within normal, pre-mutation, or full-mutation ranges based on literature-derived thresholds, making it essential for clinical genomics workflows focusing on repeat expansion disorders.

## Usage Guidelines

### Basic Command Structure
The primary command follows this pattern:
```bash
stranger [OPTIONS] <INPUT_VCF> > <OUTPUT_VCF>
```

### Processing TRGT Outputs
If your VCF was generated by TRGT (Tandem Repeat Genotyping Tool), you must use the specific flag to ensure correct parsing:
```bash
stranger --trgt input_trgt.vcf > annotated_trgt.vcf
```

### Custom Repeat Catalogs
By default, `stranger` uses an internal resource for GRCh37. For GRCh38 or custom research needs, provide a specific JSON definition file:
```bash
stranger --repeats-file path/to/variant_catalog.json input.vcf
```

### Key Annotation Fields
After running `stranger`, the VCF INFO field will be populated with several critical tags:
- `STR_STATUS`: Classification of the repeat (e.g., normal, pathologic).
- `STR_DISEASE`: The name of the associated clinical condition.
- `STR_INHERITANCE`: The mode of inheritance (AD, AR, XR, etc.).
- `STR_NORMAL_MAX`: The maximum number of repeats considered "normal".
- `STR_PATHOLOGIC_MIN`: The minimum number of repeats considered "pathologic".

### Implementation Tips
- **Family ID**: Use the `-i, --family_id` option when processing multi-sample VCFs to maintain pedigree context in the annotations.
- **Log Levels**: For troubleshooting complex VCF structures, increase verbosity using `--loglevel DEBUG`.
- **Resource Selection**: Always verify that your `--repeats-file` matches the reference genome (GRCh37 vs GRCh38) used for the initial STR calling to avoid coordinate or locus ID mismatches.
