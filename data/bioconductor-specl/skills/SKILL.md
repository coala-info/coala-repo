# bioconductor-specl

name: bioconductor-specl
description: Expert guidance for the specL Bioconductor package to generate spectral libraries for targeted proteomics (SWATH/DIA). Use this skill when you need to read BiblioSpec files, annotate peptide spectrum matches (PSMs) with FASTA protein information, normalize retention times using iRT peptides, or export assay panels for Spectronaut.

## Overview
The `specL` package is designed to bridge the gap between shotgun proteomics discovery and targeted data extraction. It provides a streamlined workflow to convert peptide identification results (from Mascot, TPP, or Skyline) into high-quality spectral libraries. Its core strengths include retention time normalization via iRT peptides and the ability to optimize SWATH extraction windows.

## Core Workflow

### 1. Loading Data
The package primarily consumes `.blib` files generated by BiblioSpec (often via Skyline).

```r
library(specL)

# Read non-redundant and redundant BiblioSpec files
# These contain the best representative spectra and all PSMs respectively
my_data <- read.bibliospec("path/to/library.blib")
my_data_redundant <- read.bibliospec("path/to/library.redundant.blib")

summary(my_data)
```

### 2. Protein Annotation
BiblioSpec files often lack protein metadata. Use a FASTA file to re-annotate the PSMs.

```r
library(seqinr)
fasta <- read.fasta("path/to/database.fasta", seqtype = "AA", as.string = TRUE)

# Annotate the psmSet object
my_data <- annotate.protein_id(my_data, fasta = fasta)
```

### 3. Generating the Ion Library
The `genSwathIonLib` function is the central command. It filters fragment ions and performs iRT normalization if redundant data is provided.

```r
# Generate library with default settings
# topN: number of fragment ions per precursor
# iRT: data frame of iRT peptides (defaults to specL::iRTpeptides)
specLibrary <- genSwathIonLib(
  data = my_data,
  data.fit = my_data_redundant,
  topN = 6,
  fragmentIonMzRange = c(300, 1250)
)

summary(specLibrary)
plot(specLibrary)
```

### 4. Exporting Results
Export the generated library for use in downstream software like Spectronaut.

```r
write.spectronaut(specLibrary, file = "my_assay_panel.csv")
```

## Advanced Features

### Dynamic SWATH Windows (CDSW)
To optimize SWATH acquisition, `specL` can compute "Computed Dynamic SWATH Windows" to ensure an even distribution of precursors across windows.

```r
# Compute windows based on precursor density
# n: target number of windows
# massrange: Q1 range
windows <- cdsw(my_data, n = 35, massrange = c(400, 1200))
```

### Retention Time Prediction (SSRC)
Predict peptide retention times using the Sequence Specific Retention Calculator (SSRC) algorithm.

```r
# Predict RT for a specific sequence
predicted_rt <- ssrc("PEPTIDESEQUENCE")
```

## Tips and Best Practices
- **iRT Normalization**: Always provide `data.fit` (redundant blib) to `genSwathIonLib` to enable robust linear regression for retention time normalization.
- **Fragment Ion Selection**: You can customize fragment ion types (e.g., only y-ions) by passing a custom function to the `fragmentIonFUN` parameter in `genSwathIonLib`.
- **Memory Management**: For very large libraries, use `summary()` to check memory usage of the `specLSet` object.
