# nwkit mark

## Overview

`nwkit mark` adds texts to node labels by identifying the targets with a leaf name regex. This function is useful, for example, when specifying the foreground branch to run [PAML](https://github.com/abacus-gene/paml)'s `codeml` analysis with the two-ratio mode (with `--insert_txt "#1"`) and to run [HyPhy](https://github.com/veg/hyphy)'s RELAX analysis (with `--insert_txt "{Foreground}"`).

## Examples

### Command

```
nwkit mark --infile input.nwk --pattern "A.*|B.*" --insert_txt "#1" --target clade --target_only_clade yes --outfile output.nwk
```

### input.nwk

```
(((A1:2.0,(B1:1.0,B2:1.0):1.0):1.0,(A2:1.0,C1:1.0):2.0):1.0,C2:4.0):0.25;
```

### output.nwk

```
(((A1#1:2,(B1#1:1,B2#1:1)#1:1)#1:1,(A2#1:1,C1:1):2):1,C2:4):0.25;
```

The `--target` option controls which nodes are marked:

### --target leaf

Only the matched leaves are marked. Internal nodes are not affected.

```
nwkit mark --pattern "B.*" --insert_txt "#1" --target leaf
```

```
(((A1:2,(B1#1:1,B2#1:1):1):1,(A2:1,C1:1):2):1,C2:4):0.25;
```

`B1` and `B2` are marked. Their parent node and other internal nodes are not marked.

### --target mrca

Only the MRCA node of all matched leaves is marked.

```
nwkit mark --pattern "B.*" --insert_txt "#1" --target mrca --target_only_clade yes
```

```
(((A1:2,(B1:1,B2:1)#1:1):1,(A2:1,C1:1):2):1,C2:4):0.25;
```

Only the parent node of `B1` and `B2` is marked. The leaves themselves are not marked.

### --target clade

All nodes in the clade containing the matched leaves are marked (MRCA + all its descendants).

```
nwkit mark --pattern "B.*" --insert_txt "#1" --target clade --target_only_clade yes
```

```
(((A1:2,(B1#1:1,B2#1:1)#1:1):1,(A2:1,C1:1):2):1,C2:4):0.25;
```

The MRCA of `B1`/`B2` and the leaves `B1`/`B2` are all marked.

### --target\_only\_clade

Controls the scope of the MRCA/clade:

* `--target_only_clade yes` (default): Finds the MRCA whose clade contains **only** target leaves.
* `--target_only_clade no`: Finds the MRCA that contains **all** target leaves (even if non-target leaves are also included).

## Other options

* `--insert_pos prefix|suffix` (default: `suffix`): Whether to insert the text before or after the existing label.
* `--insert_sep` (default: `""`): Separator between the existing label and the inserted text.
