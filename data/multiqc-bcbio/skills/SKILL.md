# multiqc-bcbio

## Overview
The `multiqc-bcbio` skill provides specialized support for the MultiQC plugin that handles outputs from the bcbio-nextgen bioinformatics framework. While standard MultiQC supports many individual tools, this plugin is essential for correctly interpreting the aggregated metrics, sample metadata, and specific directory structures produced by bcbio pipelines. It allows researchers to visualize complex variant calling, RNA-seq, or small RNA-seq workflows in a single, interactive HTML report.

## Installation and Setup
To use this plugin, it must be installed in the same environment as MultiQC. It is primarily distributed via Bioconda.

```bash
# Recommended installation via conda
conda install -c bioconda multiqc-bcbio

# Alternative installation via pip
pip install multiqc-bcbio
```

## Common CLI Patterns
Since this is a plugin, it integrates directly into the standard `multiqc` command. You do not run a separate `multiqc-bcbio` executable; instead, you point MultiQC to your bcbio project folders.

### Basic Report Generation
Run MultiQC in the final project directory or the `final` folder generated by bcbio:
```bash
multiqc .
```

### Targeting bcbio-specific Directories
If your bcbio output is organized into specific subdirectories, you can point MultiQC directly to the relevant paths to speed up parsing:
```bash
multiqc path/to/bcbio_project/final/
```

### Handling Large bcbio Projects
bcbio runs can generate thousands of files. Use the following flags to optimize performance:
- `--ignore`: Skip large temporary directories (e.g., `work/`) to prevent unnecessary file scanning.
- `--max-table-rows`: Increase this if you have a very large number of samples in your bcbio run.

```bash
multiqc . --ignore work/ --ignore intermediate/
```

## Expert Tips
- **Metadata Integration**: bcbio often includes sample metadata (e.g., batch, phenotype). The `multiqc-bcbio` plugin attempts to map these into the General Stats table. Ensure your bcbio sample configuration is well-defined to get the most out of the visualization.
- **Module Conflicts**: If you find that standard MultiQC modules are misidentifying bcbio logs, you can force specific modules or exclude others using `-m` or `-x`.
- **Search Patterns**: The plugin looks for specific filenames like `project-summary.yaml` or `programs.txt`. If MultiQC isn't picking up your bcbio data, verify these files exist in your target directory.
