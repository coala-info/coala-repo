# svtyper

## Overview
SVTyper is a tool designed to provide genotype calls for structural variants by analyzing the alignment patterns of Illumina reads. It is particularly useful for processing SV candidates generated by discovery tools like LUMPY. By assessing both concordant and discordant reads, it calculates the likelihood of homozygous reference, heterozygous, or homozygous variant genotypes at specific breakpoints.

## Command Line Usage

### Standard Genotyping
The primary command genotypes a VCF of SV sites against a specific alignment file.

```bash
svtyper \
    -i input_svs.vcf \
    -B sample_aligned.bam \
    -l library_metrics.json \
    > genotyped_svs.vcf
```

### Single-Sample Optimization (svtyper-sso)
For faster processing of a single sample, use the `svtyper-sso` variant which supports multi-core parallelization.

```bash
svtyper-sso \
    --cores 4 \
    --batch_size 1000 \
    -i input_svs.vcf \
    -B sample_aligned.bam \
    -l library_metrics.json \
    > genotyped_svs.vcf
```

### Generating Library Statistics
SVTyper requires information about the sequencing library (insert size distribution, etc.). You can pre-calculate this to save time or troubleshoot alignment issues.

```bash
# Generate the JSON metrics file only
svtyper -B sample_aligned.bam -l sample_metrics.json
```

## Expert Tips and Best Practices

- **Library Metrics**: Always use the `-l` flag to generate or provide a JSON file. This file contains essential metrics on the BAM file's insert size distribution. If SVTyper is struggling, check these metrics.
- **Input Requirements**: Ensure your BAM/CRAM files are aligned with BWA-MEM and contain both paired-end and split-read alignments for maximum accuracy.
- **Memory and Performance**: When using `svtyper-sso`, the `--batch_size` parameter controls how many SVs are processed in a single batch. Adjust this based on available system memory.
- **High Coverage Regions**: Use the `--max_reads` flag (default 1000 in SSO) to skip genotyping at sites with excessive depth, which often represent repetitive regions or assembly artifacts that can slow down processing.
- **Python Environment**: SVTyper is traditionally built for Python 2.7. Ensure your environment has `pysam` (>= 0.15.0), `numpy`, and `scipy` installed.
- **Limitations**: SVTyper currently does not support the `TRA` (translocation) SV type. It focuses on deletions, duplications, inversions, and breakends (BND).
