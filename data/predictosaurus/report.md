### Original Help Text
```text
Build a full variant graph out of VCF files and store it

Usage: predictosaurus build [OPTIONS] --calls <CALLS> --output <OUTPUT>

Options:
  -c, --calls <CALLS>
          Path to the calls file
  -o, --observations <OBSERVATIONS>
          One or more observation files in the format `sample=observations.vcf`. Make sure the sample names match the sample names in the calls file
  -m, --min-prob-present <MIN_PROB_PRESENT>
          Minimum probability for a variant to be considered in the graph [default: 0.8]
      --min-vaf <MIN_VAF>
          Minimum VAF for a variant to be kept. The maximum VAF across samples must meet or exceed this threshold [default: 0.05]
      --output <OUTPUT>
          Path to the output file containing the impact graph
  -v, --verbose
          
  -h, --help
          Print help
```


## predictosaurus_process

### Tool Description
Retrieve subgraphs for individual features from the given GFF file

### Metadata
- **Docker Image**: quay.io/biocontainers/predictosaurus:0.8.4--hbcba35e_0

### Original Help Text
```text
Retrieve subgraphs for individual features from the given GFF file

Usage: predictosaurus process [OPTIONS] --features <FEATURES> --reference <REFERENCE> --graph <GRAPH> --output <OUTPUT>

Options:
  -f, --features <FEATURES>
          Path to the gff file containing the features of interest
  -r, --reference <REFERENCE>
          Path to reference genome fasta file
  -g, --graph <GRAPH>
          Path to the graph file generated by the build command
  -d, --distance-metric <DISTANCE_METRIC>
          Distance metric to use for calculating distances between amino acids [default: grantham] [possible values: grantham, sneath, epstein, miyata]
      --haplotype-metric <HAPLOTYPE_METRIC>
          Metric to use for haplotype quantification [default: minimum] [possible values: product, geometric-mean, minimum]
  -v, --verbose
          
  -o, --output <OUTPUT>
          Path to the output file containing the paths for the features given via the GFF file
      --max-cds-length <MAX_CDS_LENGTH>
          Maximum CDS length to consider for processing. Transcripts containing longer CDSs will be ignored with a warning [default: 5000]
  -h, --help
          Print help
```


## predictosaurus_peptides

### Tool Description
Output all distinct peptides from the given features to a fastq file per given CDS in the feature file

### Metadata
- **Docker Image**: quay.io/biocontainers/predictosaurus:0.8.4--hbcba35e_0
- **Homepage**: https://github.com/fxwiegand/predictosaurus
- **Package**: https://anaconda.org/channels/bioconda/packages/predictosaurus/overview
- **Validation**: PASS

### Original Help Text
```text
Output all distinct peptides from the given features to a fastq file per given CDS in the feature file

Usage: predictosaurus peptides [OPTIONS] --features <FEATURES> --reference <REFERENCE> --graph <GRAPH> --sample <SAMPLE> --min-event-prob <MIN_EVENT_PROB> --max-background-event-prob <MAX_BACKGROUND_EVENT_PROB> --output <OUTPUT>

Options:
  -f, --features <FEATURES>
          Path to the gff file containing the features of interest
  -r, --reference <REFERENCE>
          Path to reference genome fasta file
  -g, --graph <GRAPH>
          Path to the graph file generated by the build command
  -i, --interval <INTERVAL>
          Interval for peptide lengths to be generated in the format `start-end` [default: 8-11]
  -s, --sample <SAMPLE>
          Sample name used to retrieve allele frequencies from the graph
  -v, --verbose
          
  -e, --events <EVENTS>
          
      --min-event-prob <MIN_EVENT_PROB>
          
  -b, --background-events <BACKGROUND_EVENTS>
          
      --max-background-event-prob <MAX_BACKGROUND_EVENT_PROB>
          
  -o, --output <OUTPUT>
          Path to the output directory for the fastq files
      --max-cds-length <MAX_CDS_LENGTH>
          Maximum CDS length to consider for processing. Transcripts containing longer CDSs will be ignored with a warning [default: 5000]
  -h, --help
          Print help
```
