A form consists of a sequence of inputs that usually have names, and which can take on various values. The state of a form can be tweaked and it can then be asked to provide HTTP::Request objects that can be passed to the request() method of LWP::UserAgent . The following methods are available: @forms = HTML::Form-&gt;parse( $html_document, $base_uri ) @forms = HTML::Form-&gt;parse( $html_document, base =&gt; $base_uri, %opt ) @forms = HTML::Form-&gt;parse( $response, %opt ) The parse() class method will parse an HTML document and build up HTML::Form objects for each &lt;form&gt; element found. If called in scalar context only returns the first &lt;form&gt;. Returns an empty list if there are no forms to be found. The required arguments is the HTML document to parse ($html_document) and the URI used to retrieve the document ($base_uri). The base URI is needed to resolve relative action URIs. The provided HTML document should be a Unicode string (or US-ASCII). By default HTML::Form assumes that the original document was UTF-8 encoded and thus encode forms that don&#39;t specify an explicit accept-charset as UTF-8. The charset assumed can be overridden by providing the charset option to parse(). It&#39;s a good idea to be explicit about this parameter as well, thus the recommended simplest invocation becomes: my @forms = HTML::Form-&gt;parse( Encode::decode($encoding, $html_document_bytes), base =&gt; $base_uri, charset =&gt; $encoding, ); If the document was retrieved with LWP then the response object provide methods to obtain a proper value for base and charset : my $ua = LWP::UserAgent-&gt;new; my $response = $ua-&gt;get(&quot;http://www.example.com/form.html&quot;); my @forms = HTML::Form-&gt;parse($response-&gt;decoded_content, base =&gt; $response-&gt;base, charset =&gt; $response-&gt;content_charset, ); In fact, the parse() method can parse from an HTTP::Response object directly, so the example above can be more conveniently written as: my $ua = LWP::UserAgent-&gt;new; my $response = $ua-&gt;get(&quot;http://www.example.com/form.html&quot;); my @forms = HTML::Form-&gt;parse($response); Note that any object that implements a decoded_content(), base() and content_charset() method with similar behaviour as HTTP::Response will do. Additional options might be passed in to control how the parse method behaves. The following are all the options currently recognized: base =&gt; $uri This is the URI used to retrieve the original document. This option is not optional ;-) charset =&gt; $str Specify what charset the original document was encoded in. This is used as the default for accept_charset. If not provided this defaults to &quot;UTF-8&quot;. verbose =&gt; $bool Warn (print messages to STDERR) about any bad HTML form constructs found. You can trap these with $SIG{__WARN__}. The default is not to issue warnings. strict =&gt; $bool Initialize any form objects with the given strict attribute. If the strict is turned on the methods that change values of the form will croak if you try to set illegal values or modify readonly fields. The default is not to be strict. $form-&gt;push_input( $type, \%attr, $verbose ) This method adds additional inputs to the form. The first argument is the type of input (e.g. hidden, option, etc.). The second argument is a reference to a hash of the input attributes. The third argument is optional, and will issue warnings about unknown input types. Example: push_input( &#39;hidden&#39;, { name =&gt; &#39;NewFormElement&#39;, id =&gt; &#39;NewFormElementId&#39;, value =&gt; &#39;some value&#39;, }); $method = $form-&gt;method $form-&gt;method( $new_method ) This method is gets/sets the method name used for the HTTP::Request generated. It is a string like &quot;GET&quot; or &quot;POST&quot;. $action = $form-&gt;action $form-&gt;action( $new_action ) This method gets/sets the URI which we want to apply the request method to. $enctype = $form-&gt;enctype $form-&gt;enctype( $new_enctype ) This method gets/sets the encoding type for the form data. It is a string like &quot;application/x-www-form-urlencoded&quot; or &quot;multipart/form-data&quot;. $accept = $form-&gt;accept_charset $form-&gt;accept_charset( $new_accept ) This method gets/sets the list of charset encodings that the server processing the form accepts. Current implementation supports only one-element lists. Default value is &quot;UNKNOWN&quot; which we interpret as a request to use document charset as specified by the &#39;charset&#39; parameter of the parse() method. $value = $form-&gt;attr( $name ) $form-&gt;attr( $name, $new_value ) This method give access to the original HTML attributes of the &lt;form&gt; tag. The $name should always be passed in lower case. Example: @f = HTML::Form-&gt;parse( $html, $foo ); @f = grep $_-&gt;attr(&quot;id&quot;) eq &quot;foo&quot;, @f; die &quot;No form named &#39;foo&#39; found&quot; unless @f; $foo = shift @f; $bool = $form-&gt;strict $form-&gt;strict( $bool ) Gets/sets the strict attribute of a form. If the strict is turned on the methods that change values of the form will croak if you try to set illegal values or modify readonly fields. The default is not to be strict. @inputs = $form-&gt;inputs This method returns the list of inputs in the form. If called in scalar context it returns the number of inputs contained in the form. See &quot;INPUTS&quot; for what methods are available for the input objects returned. $input = $form-&gt;find_input( $selector ) $input = $form-&gt;find_input( $selector, $type ) $input = $form-&gt;find_input( $selector, $type, $index ) @inputs = $form-&gt;find_input( $selector ) @inputs = $form-&gt;find_input( $selector, $type ) This method is used to locate specific inputs within the form. All inputs that match the arguments given are returned. In scalar context only the first is returned, or undef if none match. If $selector is not undef , then the input&#39;s name , id or class attribute must match. A selector prefixed with &#39;#&#39; must match the id attribute of the input. A selector prefixed with &#39;.&#39; matches the class attribute. A selector prefixed with &#39;^&#39; or with no prefix matches the name attribute. my @by_id = $form-&gt;find_input( &#39;#some-id&#39; ); my @by_class = $form-&gt;find_input( &#39;.some-class&#39; ); my @by_name = $form-&gt;find_input( &#39;^some-name&#39; ); my @also_by_name = $form-&gt;find_input( &#39;some-name&#39; ); If you want to find an input that has no name at all, pass in a reference to undef
