# merqury

## Overview

Merqury is a k-mer based toolset designed to evaluate genome assemblies without requiring a high-quality reference genome. By comparing the k-mer spectrum of raw sequencing reads (typically Illumina) to the k-mers found in an assembly, it provides quantitative metrics for consensus quality (QV) and completeness. It is the standard tool for generating spectra-cn plots, which visualize how k-mers from the reads are represented in the assembly (e.g., missing, correctly represented, or duplicated). For trio-based projects, Merqury can also assess phasing accuracy, identify switch errors, and generate hap-mer blob plots.

## Core Workflow

### 1. Preparation: Finding the Best k-mer Size
Before running Merqury, you must determine the optimal k-mer size for your genome size and error rate.
```bash
$MERQURY/best_k.sh <genome_size>
```

### 2. Building Meryl Databases
Merqury requires k-mer count databases generated by `meryl`.
```bash
# Count k-mers in reads
meryl count k=<k_size> output read_db.meryl reads.fastq.gz

# For trios, build hap-mer databases (maternal/paternal)
# See documentation for specific hap-mer extraction workflows
```

### 3. Running Merqury
The main script `merqury.sh` handles most evaluation tasks.

**Single Assembly (No Phasing)**
```bash
merqury.sh read_db.meryl asm.fasta out_prefix
```

**Diploid Assembly (Two Haplotypes, No Phasing)**
```bash
merqury.sh read_db.meryl hap1.fasta hap2.fasta out_prefix
```

**Trio-based Evaluation (With Hap-mers)**
```bash
merqury.sh read_db.meryl mat.meryl pat.meryl hap1.fasta hap2.fasta out_prefix
```

## Key Output Files

- **.completeness.stats**: Reports the fraction of k-mers in the reads that are found in the assembly.
- **.qv**: Consensus quality scores (log-scaled error probability) for each scaffold/contig and the overall assembly.
- **spectra-cn.png**: A plot showing the k-mer distribution. Look for the "missing" k-mer peak (black) to identify under-assembly and the "duplicated" peaks to identify artificial duplications.
- **.hapmers.blob.png**: (Trio only) Visualizes the separation of maternal and paternal k-mers in the assembly.

## Expert Tips and Best Practices

- **Database Naming**: Merqury strictly assumes all meryl databases are directories ending with the `.meryl` extension.
- **Memory Management**: For large genomes or high k-mer sizes (k > 30), ensure you have significant RAM available. If the tool crashes during the `union-sum` step, increase the memory allocation for `meryl`.
- **Input Formats**: While originally designed for `.fasta`, modern versions of Merqury also accept `.fa` and `.fasta.gz` extensions.
- **Environment Variables**: Ensure `$MERQURY` is set to the installation directory and `meryl` is in your `$PATH`.
- **IGV Integration**: Merqury generates `.wig` files instead of the older `.tdf` format. These can be loaded directly into IGV or the UCSC Genome Browser to track error-prone regions or haplotype blocks across the assembly.
- **Parallelization**: Merqury can be parallelized by running the `eval/` and `trio/` modules independently if you are building custom pipelines, though `merqury.sh` is sufficient for most users.
