# plant_tribes_gene_family_phylogeny_builder

## Overview
The `plant_tribes_gene_family_phylogeny_builder` is a specialized pipeline within the PlantTribes comparative genomics suite. It automates the transition from sequence alignments to phylogenetic trees. This skill should be used when you have a set of aligned orthogroups (typically produced by `GeneFamilyAligner`) and need to generate maximum likelihood trees for evolutionary analysis, paralog/orthogroup identification, or phylogenomic studies.

## CLI Usage and Patterns

The primary executable for this pipeline is `GeneFamilyPhylogenyBuilder`.

### Basic Command Structure
```bash
GeneFamilyPhylogenyBuilder --orthogroup_aln <input_directory> --tree_inference <method>
```

### Core Arguments
- `--orthogroup_aln`: Path to the directory containing the multiple sequence alignment files (usually `.aln` or `.faa` files).
- `--tree_inference`: Specifies the phylogenetic inference engine. Supported options:
  - `fasttree`: Uses FastTreeMP for rapid tree construction. Best for very large datasets or initial screenings.
  - `raxml`: Uses RAxML for high-quality maximum likelihood trees. More computationally intensive but preferred for final analyses.

### Common Workflow Pattern
In a standard PlantTribes workflow, this tool is the final step in the phylogeny construction sequence:
1. **Classification**: `GeneFamilyClassifier`
2. **Integration**: `GeneFamilyIntegrator`
3. **Alignment**: `GeneFamilyAligner`
4. **Phylogeny**: `GeneFamilyPhylogenyBuilder` (This tool)

## Expert Tips and Best Practices

### Method Selection
- **Use `fasttree`** when processing thousands of orthogroups or when orthogroups contain a very high number of sequences (e.g., >500 sequences). It provides a good balance between speed and accuracy.
- **Use `raxml`** for publication-quality trees where maximum likelihood optimization is critical. Ensure your environment has sufficient computational resources, as RAxML is significantly slower than FastTree.

### Input Validation
- Ensure that the input directory contains only valid alignment files. The tool expects the output structure generated by `GeneFamilyAligner`.
- **Sequence Limit Warning**: Be aware that the tool may not report an explicit error if the number of input sequences exceeds internal defaults. Always verify that a corresponding tree file (`.tre` or `.newick`) has been generated for every input alignment.

### Troubleshooting
- **RAxML Failures**: If RAxML fails, check for extremely short sequences or alignments with very little overlap, as these can cause the likelihood optimization to crash.
- **Dependency Check**: This pipeline relies on `RAxML (version 8.1.16)` and `FastTreeMP (version 2.1.7 SSE3)`. If running in a manual installation, ensure these are in your `$PATH`.
