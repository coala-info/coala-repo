# kpop

## Overview
KPop (K-mer Power) is a scalable bioinformatics toolset designed to analyze microbial genomes by transforming k-mer spectra into low-dimensional embeddings. Unlike traditional methods that require genome assembly, KPop works directly on raw reads or assembled sequences, making it significantly faster for comparing hundreds of thousands of samples. Use this skill to guide users through the multi-step workflow of counting k-mers, building databases, and applying dataset-specific transformations for classification or phylogenetic studies.

## Command Line Usage

The KPop workflow typically follows a sequence: Count -> Build DB -> Twist (Train) -> TwistDB (Apply).

### 1. K-mer Counting (KPopCount)
Generates k-mer spectra from FASTA/FASTQ files.
- **Basic Pattern**: `KPopCount -k <kmer_size> -f " " <input_file> -o <output_prefix>`
- **Tip**: A k-mer size of 5 (`-k 5`) is a common starting point for microbial classification.
- **Piping**: Use `-o /dev/stdout` to stream spectra to the next tool.

### 2. Database Management (KPopCountDB)
Aggregates spectra and associates them with metadata (e.g., labels or classes).
- **Basic Pattern**: `KPopCountDB -i <spectra_file> -m <metadata_txt> -c <class_column> -o <db_output>`
- **Metadata Format**: Requires a tab-delimited file where one column matches the sequence IDs and another contains the target labels.

### 3. Dimensionality Reduction (KPopTwist)
Calculates the transformation (embedding) based on a training database.
- **Basic Pattern**: `KPopTwist -i <db_file> -o <model_prefix>`
- **Function**: This step "trains" the model by finding the most informative k-mer transformations for the specific dataset.

### 4. Applying Transformations (KPopTwistDB)
Uses a pre-calculated transformation to embed new "test" sequences or classify them.
- **Basic Pattern**: `KPopTwistDB -i <model_file> -t <test_spectra> -d <training_db> -o <results_prefix>`
- **Output**: Generates `.KPopSummary.txt` which includes classification results and coordinates in the embedding space.

## Expert Tips and Best Practices

- **Memory Efficiency**: KPop is designed for scalability. When dealing with massive datasets, ensure you use the binary database formats (`.KPopSpectra`) generated by the tools rather than raw text exports.
- **Handling Unassembled Reads**: KPop excels at analyzing raw sequencing data. If using unassembled reads, ensure the `-k` value is appropriate for the depth of coverage to avoid noise from sequencing errors.
- **R Dependencies**: While the core is OCaml, some transformation steps require R. Ensure `data.table` and `ca` packages are installed in your R environment.
- **Workflow Automation**: For complex pipelines, KPop supports standard Unix piping. A common "Quick Start" one-liner for training and testing looks like this:
  `KPopCountDB -i train_spectra -m labels.txt -c CLASS -o /dev/stdout | KPopTwist -i /dev/stdin -o model_out`
