IRMA dynamically detects its install directory and requires the IRMA_RES folder in the same directory, which is how it is distributed by default. IRMA is pre-packaged with LABEL , which cannot be removed because IRMA uses some of LABEL's packaged scripts. Return to the IRMA homepage IRMA was designed for use with Linux and Mac OS X . IRMA requires at least Perl version 5 and BASH version 3, which is standard on most Linux &amp Mac OS X systems. If a particular standard unix utility is missing, you may receive a warning message or else running IRMA will fail. This is unlikely. Download the latest version of IRMA &amp; LABEL and unzip the archive in the desired &lt;install_path&gt;. Install R . R must be available on any computer which runs IRMA processes. If you are running IRMA on a computer grid, you'll need a compatible grid engine such as Open Grid Scheduler . You can use IRMA without a computer grid, but you must make sure to set GRID_ON=0 in your configuration , which is default. You must add &lt;install_path&gt;/flu-amd to your system's PATH variable.
