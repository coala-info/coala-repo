The software contained here comes equipped with modules for processing sequencing reads, reconstructing &amp; plotting trees, analyzing these trees, and benchmarking new algorithms. You can find all of our documentation here . We also have provided tutorials for three modules: processing fastqs reconstructing trees simulating trees and benchmarking plotting trees with our local library You can also find our originally describing Cassiopeia published in Genome Biology . Free Software: MIT License Installation For users: pip install git+https://github.com/YosefLab/Cassiopeia@master#egg=cassiopeia-lineage For developers: Clone the package as so: git clone https://github.com/YosefLab/Cassiopeia.git Ensure that you have Python &gt;= 3.10 installed. (Due to dependencies and ongoing updates to Python, we will not guarantee support for python &lt;= 3.9) We prefer using miniconda . [Optional] If you intend to use the CassiopeiaILP solver, please be sure that Gurobi is installed. You can follow the instructions listed here . To verify that it's working correctly, use the following tests: Run the command gurobi.sh from a terminal window From the Gurobi installation directory (where there is a setup.py file), use python setup.py install --user [Optional] To use fast versions of Neighbor-Joining and UPGMA, install CCPhylo . Then copy the file ./data/ccphylo_config.ini to your ./cassiopeia directory, rename it config.ini and set the Path variable to point to your CCPhylo installation. Install Cassiopeia by first changing into the Cassiopeia directory and then pip3 install . or make install . To install dev and docs requirements, you can run pip3 install .[dev,docs] . [Optional] To use tools built for the analysis of spatial lineage tracing datasets, you can install Cassiopeia with pip install .[spatial] . Please note that we recommend using Python &gt;= 3.9 for these analyses as some features might not be available otherwise, due to package dependencies (especially 3D visualization). To verify that it installed correctly, install pytest ( pip install pytest ) and try running our tests with make test .
