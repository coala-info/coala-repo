# dindel

## Overview
Dindel is a specialized variant caller designed to identify small indels in Illumina sequencing data. Unlike simple pileup-based callers, Dindel extracts candidate indels and performs a local realignment of reads to candidate haplotypes within ~120 bp windows. This specific TGI fork is updated to handle supplementary alignments (0x800 flag) found in modern SAM/BAM specifications, making it more robust for current bioinformatics pipelines.

## Core Workflow
The Dindel process typically follows a multi-step procedure to move from raw alignments to a VCF:

1. **Candidate Extraction**: The tool scans the BAM file to identify potential indel sites based on existing gaps or mismatches in the alignments.
2. **Windowing**: Candidate sites are grouped into processing windows (typically ~120bp) using helper scripts like `makeWindows.py`.
3. **Haplotype Realignment**: Reads (both mapped and unmapped) are realigned to potential haplotypes containing the candidate indels.
4. **Variant Calling**: Bayesian inference is used to calculate genotype likelihoods and produce the final VCF output.

## CLI Usage Patterns
- **Compilation**: Ensure `libboost-dev`, `libboost-program-options-dev`, and `libncurses5-dev` are installed. Run `make` in the root directory to build the `dindel` binary.
- **Input Requirements**: Requires a coordinate-sorted BAM file with a valid index (.bai) and the reference FASTA file.
- **Handling Supplementary Alignments**: This fork automatically processes 0x800 flags; no additional configuration is needed to include these reads in the realignment process.
- **Python Integration**: Use the scripts in the `python/` directory to manage the workflow, particularly for generating the windows required for the realignment stage.

## Expert Tips
- **Unmapped Reads**: Dindel can realign unmapped reads to candidate haplotypes. Ensure your input BAM includes unmapped reads that were placed near candidate regions by the primary aligner to maximize sensitivity.
- **SNP Calling Improvement**: Use the realigned BAM files generated by Dindel to improve the accuracy of SNP calls near indels. Standard alignments often misplace SNPs in the presence of an indel; Dindel's realignment corrects these artifacts.
- **External Candidates**: Dindel can test candidate indels discovered via other methods (such as split-read or assembly). Providing these as input allows Dindel to use its realignment model to validate or reject the candidates based on read support.
- **Memory Management**: For large-scale projects, process the genome in chunks using the windowing approach to keep memory usage within limits.
