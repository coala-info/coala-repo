# scatac-fragment-tools

## Overview
The `scatac-fragment-tools` suite provides high-performance utilities written in Rust with a Python interface for managing single-cell ATAC-seq data. It is specifically designed to handle the large, tabix-indexed BED-like fragment files generated by pipelines like CellRanger. This skill enables efficient generation of genomic coverage tracks (bigWig) and the ability to split or modify fragment files based on cell barcodes or sample identifiers.

## Installation and Setup
To use the full functionality, ensure the tool and its high-performance dependencies are installed:

```bash
# Basic installation
pip install scatac-fragment-tools

# Recommended for fast bigWig generation
pip install scatac-fragment-tools[pybigtools]
```

## Common CLI Patterns

### Generating bigWig Tracks
The `bigwig` subcommand creates coverage tracks from fragment files. It is optimized for speed when `pybigtools` is present.

```bash
# Generate a bigWig file from fragments
scatac_fragment_tools bigwig input.fragments.tsv.gz output.bw --chrom_sizes hg38.chrom.sizes
```

**Expert Tips for bigWig:**
- **Chromosome Prefixes**: Use `--chrom_prefix` if your fragment file and reference genome have different naming conventions (e.g., adding "chr").
- **Normalization**: Check help for scaling options to ensure tracks are comparable across samples.

### Splitting Fragment Files
The `split` subcommand is used to de-multiplex or organize fragments based on cell barcodes.

```bash
# Split fragments by cell barcode
scatac_fragment_tools split input.fragments.tsv.gz --output_dir split_fragments/ --barcode_file barcodes.txt
```

**Key Features:**
- **Sample ID Prefixing**: Use the option to prefix cell barcodes with a sample ID when merging datasets from different runs to avoid barcode collisions.
- **CellRanger Compatibility**: The tool supports reading fragments from CellRanger-ATAC (2.2.x) and CellRanger-ARC (2.1.x).

## Best Practices
- **Sorting**: Ensure fragment files are properly sorted and indexed with `tabix`. Recent versions of the tool include memory-efficient sorting logic, but pre-sorted input is generally required for most subcommands.
- **Memory Management**: When working with extremely large datasets, the Rust-backed implementation handles memory more efficiently than pure Python scripts, but always monitor peak RSS during `split` operations.
- **Dependency Choice**: Prefer `pybigtools` over `pybigwig` for significantly faster bigWig creation.
