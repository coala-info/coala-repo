# biopet-extractadaptersfastqc

## Overview
`biopet-extractadaptersfastqc` is a utility designed to bridge the gap between quality control and data cleaning. It parses the `fastqc_data.txt` file generated by FastQC to identify sequences flagged in the "Adapter Content" module. By converting these findings into plain text or FASTA formats, it allows for a seamless transition from QC to adapter trimming without manual sequence copying.

## Usage Instructions

### Installation
The tool is available via Bioconda:
```bash
conda install -c bioconda biopet-extractadaptersfastqc
```

### Basic Command Line Patterns
The tool requires the raw data file from a FastQC run (usually found inside the unzipped FastQC output folder).

**Extract to Plain Text**
By default, the tool outputs sequences separated by newlines:
```bash
biopet-extractadaptersfastqc --fastqcData path/to/fastqc_data.txt
```

**Extract to FASTA Format**
To generate a FASTA file for use with tools that require formatted adapter databases:
```bash
biopet-extractadaptersfastqc --fastqcData path/to/fastqc_data.txt --outputFasta --output adapters.fasta
```

### Integration with Trimming Workflows
The primary use case for this tool is to feed sequences into `cutadapt`. You can capture the output in a variable or a temporary file to build your trimming command dynamically.

**Example Workflow:**
1. Run FastQC on your raw reads.
2. Unzip the output: `unzip sample_fastqc.zip`.
3. Extract adapters: `biopet-extractadaptersfastqc --fastqcData sample_fastqc/fastqc_data.txt > adapters.txt`.
4. Pass to Cutadapt:
   ```bash
   ADAPTERS=$(cat adapters.txt)
   for seq in $ADAPTERS; do
     # Append each found adapter to the cutadapt command
     ADAPTER_ARGS="$ADAPTER_ARGS -a $seq"
   done
   cutadapt $ADAPTER_ARGS -o trimmed.fastq raw.fastq
   ```

## Expert Tips
- **Input File**: Ensure you are pointing to `fastqc_data.txt`. The tool cannot parse the `.html` report or the `.zip` archive directly.
- **Empty Reports**: If FastQC did not detect any adapters, the tool will produce an empty output. Always check if the output is non-empty before proceeding to a trimming step in an automated pipeline.
- **Standardization**: Use the `--outputFasta` flag if you plan to maintain a record of adapters found across a large project, as it provides a more structured format than plain text.
