### Changelog

The prefix option *-p* for *metrics* does not add a '\_' after the prefix anymore, and the default value is now nothing instead of "metrics". Here's a quick overview of the changes in practice:

**Before**:

* `metrics test.maf` would generate the output files *metrics\_alignability.wig* and *metrics\_identity.wig*
* `metrics test.maf -p test` would generate the output files *test\_alignability.wig* and *test\_identity.wig*

**Now**:

* `metrics test.maf` generates the output files *alignability.wig* and *identity.wig*
* `metrics test.maf -p test_` generates the output files *test\_alignability.wig* and *test\_identity.wig*

[First beta release](/RomainFeron/mwga-utils/releases/tag/0.1.0)

First beta release of mwga\_utils. Currently implemented:

* **metrics**: compute base metrics from a MAF and produce WIG files
* **missing\_regions**: add regions from the reference assembly that are missing from a MAF file
* **single\_coverage**: check that the each sequence in the reference species is only covered once in a MAF
* **stats**: compute a series of statistics on a MAF file
