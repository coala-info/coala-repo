# variantbreak

## Overview
variantbreak is a structural variant analyzer designed to consolidate SV data from multiple samples into a unified format. It specializes in intersecting and merging breakends from NanoVar-generated VCFs or standard BED files. The tool provides automated annotation using GTF/GFF/BED files and allows for custom filtering against specific genomic regions. Its primary output is an HDF5 file optimized for graphical visualization in VariantMap (Dash Bio) and a master CSV for manual data inspection.

## Command Line Usage
The basic syntax for variantbreak requires paths for annotations, filters, input variants, and a working directory.

```bash
variantbreak [Options] -a <annotation_file_or_dir> -f <filter_file_or_dir> <variant_path> <working_dir>
```

### Key Arguments
- `-a`: Path to a single annotation file or a directory containing GTF, GFF, or BED files.
- `-f`: Path to a single BED filter file or a directory of BED files. Variants overlapping these regions are labeled "HIT" or "MISS".
- `variant_path`: Path to the directory containing your NanoVar VCF files or variant BED files.
- `working_dir`: The destination directory for output files.

### Output Files
- `output.h5`: The HDF5 master dataframe required for VariantMap visualization.
- `output.csv`: A delimited text file containing the merged and annotated SV data.
- `legend.txt`: A reference file for sample labels used during the analysis.

## Python API Usage
For integration into bioinformatics pipelines, variantbreak can be called directly within Python.

```python
from variantbreak import variantbreak, write_to_files

# Process samples and return a pandas DataFrame
df = variantbreak("/path/to/variants/", "/path/to/annotations/", "/path/to/filters/")

# Export results to disk
write_to_files(df, "/path/to/output_prefix", sep="\t")
```

## Best Practices and Tips
- **Input Compatibility**: Currently, VCF input is specifically optimized for files generated by NanoVar (v1.3.6+). For SVs called by other tools, convert them to BED format before processing.
- **Batch Processing**: Point the tool to directories rather than individual files when working with cohorts; variantbreak will automatically aggregate all compatible files in the folder.
- **Dependency Management**: Ensure `bedtools` is installed and available in your system PATH, as the underlying `pybedtools` library requires the binary for genomic intersections.
- **Performance**: Processing speed is roughly 30 minutes per 100,000 merged SVs. For very large cohorts, ensure the system has sufficient RAM to handle the master pandas dataframe.
- **Visualization**: Use the `.h5` output specifically with the VariantMap component in Dash Bio for interactive structural variant exploration.
