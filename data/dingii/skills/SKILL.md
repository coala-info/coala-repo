# dingii

## Overview

The `dingii` tool provides a specialized framework for calculating evolutionary distances between genomes that do not follow simple permutation patterns. Unlike basic rearrangement models, `dingii` accounts for the complexities of "natural" genomes, such as varying gene content. The tool follows a decoupled workflow where it first generates a mathematical optimization model (ILP) and subsequently interprets the results after they have been processed by an external solver.

## Installation

Install `dingii` via the Bioconda channel:

```bash
conda install bioconda::dingii
```

## Command Line Usage

The tool uses a unified command-line interface with two primary subcommands: `ilp` and `sol`.

### 1. ILP Construction
Use the `ilp` subcommand to generate the Integer Linear Program formulation based on your input genomes.

```bash
dingii ilp [OPTIONS] [GENOME_FILES]
```

*   **Purpose**: Translates the biological rearrangement problem into a mathematical format.
*   **Output**: Typically produces a `.lp` or `.mps` file compatible with standard optimization solvers (e.g., Gurobi, CPLEX, or SCIP).

### 2. Solution Parsing
After running an external solver on the ILP, use the `sol` subcommand to interpret the results.

```bash
dingii sol [SOLVER_OUTPUT_FILE]
```

*   **Purpose**: Extracts the rearrangement distance and specific evolutionary events from the solver's technical output.
*   **Note**: Ensure you use the solution file generated by the solver as the input for this command.

## Expert Tips and Best Practices

*   **Python Integration**: `dingii` is implemented as a Python module. For complex pipelines or custom scripts, you can access its functionality directly within Python:
    ```python
    import dingii
    ```
*   **Handling Indels**: When working with genomes that have significant differences in gene content, look for experimental flags related to "indel min/max" functionality, which allows for more nuanced distance calculations.
*   **Workflow Automation**: Since `dingii` requires an external solver, the standard workflow is:
    1.  `dingii ilp ... > model.lp`
    2.  `solver model.lp > solution.sol`
    3.  `dingii sol solution.sol`
*   **Bug Mitigation**: If you encounter issues with self-edges in the underlying graph representation, ensure you are using version 0.0.1 or later, as specific fixes for edge duplication were implemented in recent updates.
