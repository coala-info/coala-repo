# decom

## Overview
decOM is a high-accuracy microbial source tracking tool specifically designed for the contamination assessment of ancient oral samples. It utilizes a k-mer-based approach to compare input metagenomic data against a reference matrix containing signatures for Ancient Oral (aOral), Modern Oral (mOral), Skin, and Sediment/Soil environments. The tool represents each input sample as a presence/absence vector of k-mers and estimates the contribution of each source environment, providing both raw counts and proportional estimates.

## Installation and Setup
Install decOM via bioconda or by cloning the repository.
```bash
conda install bioconda::decom
```

Before running the tool, you must download and decompress the required reference source matrix:
```bash
wget https://zenodo.org/record/6513520/files/decOM_sources.tar.gz
tar -xf decOM_sources.tar.gz
```

## Input Preparation
decOM requires a "File of Files" (.fof) key for each sink to map sample names to their respective sequence files.

### Key File Format (.fof)
Create a text file where each line follows this structure:
- **Single-end**: `sample_name : path/to/file_1.fastq.gz`
- **Paired-end**: `sample_name : path/to/file_1.fastq.gz; path/to/file_2.fastq.gz`

Note: Ensure the sample name used in the `-s` flag matches the name defined in the `.fof` file.

## Common CLI Patterns

### Analyzing a Single Sink
Use this pattern to process one metagenomic sample.
```bash
decOM -s <sample_name> -p_sources <path_to_decOM_sources> -k <path_to_key.fof> -mem 10GB -t 5
```

### Analyzing Multiple Sinks
To process multiple samples in batch, provide a list of sinks and a directory containing their respective key files.
```bash
decOM -p_sources <path_to_decOM_sources> -p_sinks <sinks.txt> -p_keys <directory_with_keys/> -mem 10GB -t 5
```

## Resource Management and Optimization
- **Memory Allocation**: The total memory used is calculated as `-mem` multiplied by the number of threads `-t`. For example, `-mem 10GB -t 5` will allocate 50GB total.
- **Minimum Requirements**: Use at least 10GB of memory per core. If using a custom k-mer matrix, allocate memory equal to or greater than the size of the matrix file.
- **Monitoring**: decOM uses Dask for parallelization. Monitor the process via the local diagnostic dashboard at `http://127.0.0.1:8787/status` if running locally.

## Output Interpretation
The tool generates a `decOM_output/` directory containing:
- `decOM_output.csv`: A dataframe with raw k-mer counts per source, running time, and estimated proportions (e.g., `p_aOral`, `p_Skin`).
- `result_plot_sinks.pdf/html`: Static and interactive visualizations of source environment proportions.
- `{s}_vector/`: Intermediate k-mer vectors generated by kmtricks.

## Best Practices
- **Permissions**: Ensure you have both read and write permissions in the directory where decOM is executed and where the source matrix is stored.
- **Environment**: decOM is designed for Linux-like systems.
- **Unknowns**: The "Unknown" source contribution feature is currently in development; focus on the raw counts for Soil, Skin, aOral, and mOral for the most reliable assessment.
