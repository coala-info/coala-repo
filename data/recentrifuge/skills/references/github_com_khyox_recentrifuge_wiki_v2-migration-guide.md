# Version 2.0.0 migration guide

This guide provides detailed information about the changes in Recentrifuge 2.0.0 and how to migrate from previous versions.

## Python version requirements

### Breaking change

Recentrifuge 2.0.0 requires **Python 3.12 or higher**. Support for older Python versions (3.6-3.11) has been dropped.

### Migration steps

1. **Check Current Python Version**

   ```
   python --version
   # or
   python3 --version
   ```
2. **Upgrade Python** (if needed)

   * **macOS**: `brew install python@3.14`
   * **Ubuntu/Debian**: `sudo apt update && sudo apt install python3.14`
   * **Windows**: Download from [python.org](https://www.python.org/downloads/)
   * **Conda**: `conda install python=3.14`
3. **Verify Installation**

   ```
   python3.14 --version
   ```
4. **Reinstall Recentrifuge**

   ```
   pip install recentrifuge
   # or if using a specific Python version
   python3.14 -m pip install recentrifuge
   ```

### Python 3.12 deprecation warning

Note that Recentrifuge 2.0.0 includes a warning that support for Python 3.12 will be dropped soon, so if you work with Python 3.12, plan to upgrade to Python 3.13 or higher in the future.

## HTML file size optimization

### What changed

Recentrifuge 2.0.0 introduces a new optimized XML format (Version 2) that significantly reduces the size of generated HTML files. This optimization uses:

* **Delimited format**: Comma-separated values instead of repeated XML tags
* **Sparse representation**: For mostly-zero data, only non-zero values are stored with their indices

### Impact

* HTML files are typically 50-90% smaller
* Faster loading times for large datasets
* Reduced memory usage in browsers
* Backward compatibility maintained

### No action required

This change is completely transparent to users. Existing workflows will continue to work without modification. The generated HTML files will simply be smaller.

## Taxonomic resolution changes

### Breaking change

The default taxonomic resolution has been changed from species-level to strain-level analysis.

### Migration steps

If you want to maintain the previous behavior (species-level only):

```
rcf --no-strain [other options]
```

If you want to use the new default behavior (strain-level included), equivalent to previous `--strain` experimental flag, no changes are needed.

## Parallel processing control

### New feature

Recentrifuge 2.0.0 introduces fine-grained control over parallel processing with the new `--threads` argument.

### Usage

```
# Use default behavior (legacy mode - automatically determine threads)
rcf [options]

# Specify exact number of threads
rcf --threads 4 [options]

# Disable parallel processing
rcf --sequential [options]
```

## New features [experimental, under development]

### Regentrifuge (rgf)

Recentrifuge 2.0.0 introduces a new tool for functional metagenomics analysis:

```
rgf --help
```

### Gene Ontology support

Support for functional metagenomics results based on Gene Ontology (GO).

## API changes

### KronaTree.node() method

The internal API for generating Krona-style nodes has been modified to add support for the new sparse/delimited format. This change does not affect user-facing APIs. Backward compatibility in the XML format has been maintained.

### Command Line Interface (CLI)

* All existing CLI options remain compatible
* New options have been added but are optional
* Default behavior has changed for taxonomic resolution (now includes strain level)

### Configuration files

* Existing configuration files will continue to work
* New options have been added but are optional

## Dependency updates

### What changed

All dependencies have been updated to versions compatible with Python 3.12:

* biopython: 1.84
* numpy: 2.3.4+
* openpyxl: 3.1.2+
* matplotlib: 3.3.4+
* pandas: 2.3.3+

### Migration steps

Simply reinstalling Recentrifuge will automatically install the correct versions of all dependencies:

```
pip install recentrifuge
```

## Troubleshooting

### Common issues

1. **Python Version Error**

   ```
   ImportError: Recentrifuge requires Python 3.12 or later
   ```

   **Solution**: Upgrade to Python 3.12 or higher as described above.
2. **Dependency Conflicts**

   ```
   pkg_resources.ContextualVersionConflict
   ```

   **Solution**: Create a fresh virtual environment:

   ```
   python3.14 -m venv rcf_env
   source rcf_env/bin/activate  # On Windows: rcf_env\Scripts\activate
   pip install recentrifuge
   ```
3. **Permission Errors**

   ```
   ERROR: Could not install packages due to an OSError: [Errno 13] Permission denied
   ```

   **Solution**: Use a virtual environment or install with --user flag:

   ```
   pip install --user recentrifuge
   ```

### Reporting issues

If you encounter any issues not covered in this guide, please:

1. Check the GitHub Issues
2. Create a new issue attending to the instructions in the issue page:
   * Your Python version (`python --version`)
   * Your operating system
   * The error message
   * Steps to reproduce

## Testing your installation

After upgrading, you can verify your installation works correctly:

```
# Check version
rcf --version

# Run the simplest test
rcf -h  # Show help

# Run the CI test suite for comprehensive testing (please visit [[testing and more|Testing-and-more]] for details)
retest -d -l -r
```

* Request support through GitHub Issues
