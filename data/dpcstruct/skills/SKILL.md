# dpcstruct

## Overview

DPCstruct is a specialized pipeline for the unsupervised identification and classification of protein domains. It operates on local structural alignments (typically generated by Foldseek) to group similar structural motifs across different proteins. The tool is particularly useful for analyzing large-scale structural databases, such as AlphaFold2 predictions, to discover recurring domain architectures without relying on predefined libraries or hidden Markov models.

## Core Pipeline Workflow

The DPCstruct pipeline consists of five sequential modules. Each step depends on the output of the previous one.

### 1. Prefiltering
Apply initial quality and relevance filters to the raw Foldseek alignments.
```bash
dpcstruct prefilters -i <alignments.tsv> -m <protein_lookup.tsv> -o <filtered_alns.tsv>
```
*   **Input**: TSV of alignments and a lookup table mapping protein names to indices.
*   **Tip**: Ensure your Foldseek output includes `query,target,qstart,qend,tstart,tend,qlen,tlen,alnlen,pident,evalue,bits,alntmscore,lddt`.

### 2. Primary Clustering
Cluster local alignments for each individual query sequence.
```bash
dpcstruct primarycluster -i <filtered_alns.tsv> -o <pcs.bin> -t <threads>
```
*   **Output**: A binary file (`.bin`) containing primary clusters.

### 3. Secondary Clustering
Perform all-vs-all clustering of the primary clusters to identify global structural similarities. This is a two-part process.

**Step A: Calculate Distance Matrix**
```bash
dpcstruct secondarycluster distance -i <pcs.bin> -j <pcs.bin> -o <distance_matrix.bin> -p 1 -c <threads>
```

**Step B: Classification**
```bash
dpcstruct secondarycluster classify <distance_matrix.bin> -o <sc_classification.txt>
```

### 4. Traceback
Map the clusters back to the original protein sequences to define domain boundaries.
```bash
dpcstruct traceback <pcs.bin> -l <sc_classification.txt> -o <output_dir_binary>
```

### 5. Postfiltering
Remove redundancies and finalize the domain assignments.
```bash
dpcstruct postfilters -i <output_dir_binary>/sequence-labels_1.bin -o <final_output.txt>
```

## Input Preparation (Foldseek)

DPCstruct requires specific Foldseek output formats. Use the following pattern to generate compatible input:

```bash
foldseek convertalis <queryDB> <targetDB> <alns> <output.tsv> \
    --format-mode 4 \
    --format-output query,target,qstart,qend,tstart,tend,qlen,tlen,alnlen,pident,evalue,bits,alntmscore,lddt
```

## Expert Tips

*   **AlphaFold pLDDTs**: If working with AlphaFold structures, use the provided utility script `download_plddts.sh` (found in the `util` directory) to fetch per-residue confidence scores, which can be used during prefiltering to ignore low-confidence regions.
*   **Protein Indexing**: DPCstruct prefers numeric identifiers. Convert your PDB filenames to a numeric index table (`proteins.tsv`) before starting the pipeline to ensure consistent mapping.
*   **Parallelization**: Use the `-t` or `-c` flags in the clustering modules to utilize multiple CPU cores, as structural comparison is computationally intensive.



## Subcommands

| Command | Description |
|---------|-------------|
| dpcstruct prefilters | Filters alignments based on quality metrics. |
| dpcstruct primarycluster | Identifies primary clusters given a set of query proteins. |
| dpcstruct-secondarycluster | Perform secondary clustering operations. |
| dpcstruct_postfilters | filters the results produced by traceback.cc. |
| dpcstruct_traceback | Assign a metacluster label to each domain inside a primary cluster. |
