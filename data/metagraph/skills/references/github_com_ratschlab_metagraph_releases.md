## What's Changed

### Performance improvements

* map sequences to graph with dynamic omp schedule for better performance by [@karasikov](https://github.com/karasikov) in [#587](https://github.com/ratschlab/metagraph/pull/587)
* refactoring and optimizations in annotation queries by [@karasikov](https://github.com/karasikov) in [#582](https://github.com/ratschlab/metagraph/pull/582)
* updated sdsl-lite: removed memory overhead in select\_support\_mcl loaded with mmap by [@karasikov](https://github.com/karasikov) in [#603](https://github.com/ratschlab/metagraph/pull/603)

### Workflow and API changes

* changed default query mode: report hits with num k-mer matches by [@karasikov](https://github.com/karasikov) in [#588](https://github.com/ratschlab/metagraph/pull/588)
* fix graph augmentation and added tests by [@karasikov](https://github.com/karasikov) in [#580](https://github.com/ratschlab/metagraph/pull/580)
* Inline indexing of suffix ranges in BOSS construction by [@karasikov](https://github.com/karasikov) in [#590](https://github.com/ratschlab/metagraph/pull/590)
* Server queries in multi-graph regime with full loading to RAM by [@karasikov](https://github.com/karasikov) in [#569](https://github.com/ratschlab/metagraph/pull/569)

### Other updates

* Fix catching errors when lsof doesn't exist or fails by [@karasikov](https://github.com/karasikov) in [#583](https://github.com/ratschlab/metagraph/pull/583)
* Don't compile sshash executable by [@karasikov](https://github.com/karasikov) in [#584](https://github.com/ratschlab/metagraph/pull/584)
* Add citation section to README by [@adamant-pwn](https://github.com/adamant-pwn) in [#585](https://github.com/ratschlab/metagraph/pull/585)
* check workflows on linux by [@karasikov](https://github.com/karasikov) in [#592](https://github.com/ratschlab/metagraph/pull/592)
* updated KMC: removed unused variables by [@karasikov](https://github.com/karasikov) in [#602](https://github.com/ratschlab/metagraph/pull/602)
* Upgrade sshash to v5.0.0 with API compatibility changes by [@adamant-pwn](https://github.com/adamant-pwn) in [#594](https://github.com/ratschlab/metagraph/pull/594)

**Full Changelog**: [v0.5.0...v0.5.1](https://github.com/ratschlab/metagraph/compare/v0.5.0...v0.5.1)

## What's Changed

### Major or API updates:

* Support parsing k-mer abundances from headers by [@adamant-pwn](https://github.com/adamant-pwn) in [#558](https://github.com/ratschlab/metagraph/pull/558)
* Optimization: don't serialize the inverse label mapping by [@karasikov](https://github.com/karasikov) in [#561](https://github.com/ratschlab/metagraph/pull/561)
  **Note:** The new version can load both old and new annotations, but annotations generated with the new version require this version or later. (Only backward compatibility is ensured. Make sure to use the latest version of metagraph.)
* Aggregate k-mer counts with --count-kmers in transform\_anno by [@adamant-pwn](https://github.com/adamant-pwn) in [#568](https://github.com/ratschlab/metagraph/pull/568)
* Support mapping annotation label+coordinates to accessions by [@hmusta](https://github.com/hmusta) and [@karasikov](https://github.com/karasikov) in [#549](https://github.com/ratschlab/metagraph/pull/549)

### Reliability improvements

* Improvements in threading, parallel integration tests by [@karasikov](https://github.com/karasikov) in [#538](https://github.com/ratschlab/metagraph/pull/538)
* Optimization of single-threaded RowDiff queries by [@karasikov](https://github.com/karasikov) in [#551](https://github.com/ratschlab/metagraph/pull/551)
* Workflow testing on MacOS and full parallelization of non-rectangular loops by [@karasikov](https://github.com/karasikov) in [#556](https://github.com/ratschlab/metagraph/pull/556)
* Don't use range-based loops with omp by [@karasikov](https://github.com/karasikov) in [#557](https://github.com/ratschlab/metagraph/pull/557)
* Server changes: load all graphs, don't compute num\_relations in stats by [@karasikov](https://github.com/karasikov) in [#563](https://github.com/ratschlab/metagraph/pull/563)
* a workaround for cases where the outfile is on a different filesystem by [@karasikov](https://github.com/karasikov) in [#578](https://github.com/ratschlab/metagraph/pull/578)

### Build and CI improvements

* Make jemalloc optional: allow compiling without it by [@karasikov](https://github.com/karasikov) in [#555](https://github.com/ratschlab/metagraph/pull/555)
* Support add\_subdirectory and ExternalProject\_Add in third-party projects, improve CI by [@adamant-pwn](https://github.com/adamant-pwn) in [#565](https://github.com/ratschlab/metagraph/pull/565)
* Consolidate workflows with YAML anchors, put examples into repo by [@adamant-pwn](https://github.com/adamant-pwn) in [#566](https://github.com/ratschlab/metagraph/pull/566)
* Pass CMAKE\_POSITION\_INDEPENDENT\_CODE to SDSL by [@adamant-pwn](https://github.com/adamant-pwn) in [#573](https://github.com/ratschlab/metagraph/pull/573)
* updated sdsl-lite (removed unused variable) by [@karasikov](https://github.com/karasikov) in [#579](https://github.com/ratschlab/metagraph/pull/579)

**Note:** This release changes the label encoder format used in all annotations. Backward compatibility is preserved and this version can load annotations constructed with older versions of metagraph. However, older releases cannot load annotations created with the new format. For this reason, **always use the latest version**.

**Full Changelog**: [v0.4.5...v0.5.0](https://github.com/ratschlab/metagraph/compare/v0.4.5...v0.5.0)
