# metawrap-binning

## Overview
MetaWRAP is a modular suite designed to handle the entire metagenomic analysis pipeline, but its core strength lies in the "binning" and "bin processing" stages. It provides a unified interface to run multiple binning tools simultaneously and includes a sophisticated refinement module that consolidates different binning predictions into a single, superior set of Metagenome-Assembled Genomes (MAGs). Use this skill to guide the transition from a metagenomic assembly to high-quality, refined, and reassembled genomic bins.

## Installation and Setup
The preferred method for installing MetaWRAP is via Mamba to handle the complex dependency tree efficiently.

```bash
# Create environment and install metawrap
mamba create -y --name metawrap-env --channel ursky metawrap-mg=1.3.2
conda activate metawrap-env

# Fix potential CONCOCT/MKL conflicts if they arise
conda install -y blas=2.5=mkl
```

Before running modules, ensure the `config-metawrap` file is edited to point to the correct paths for databases (CheckM, Kraken, etc.).

## Core Binning Workflow

### 1. Initial Binning
Run the `binning` module to generate initial bin sets using one or more algorithms. It is recommended to use all three supported binners to provide the best input for the refinement step.

```bash
metawrap binning -a assembly.fasta -o binning_out --maxbin2 --metabat2 --concoct reads_1.fastq reads_2.fastq
```

### 2. Bin Refinement
This is MetaWRAP's most powerful feature. It takes the outputs from the previous step and uses a hybrid approach to identify the best version of each bin, significantly outperforming individual tools or simple consolidation programs like DAS_Tool.

```bash
metawrap bin_refinement -o refined_out -A binning_out/maxbin2_bins -B binning_out/metabat2_bins -C binning_out/concoct_bins -c 50 -x 10
```
*   `-c 50`: Minimum completion percentage.
*   `-x 10`: Maximum contamination percentage.

### 3. Bin Reassembly
After refinement, use the `reassemble_bins` module. This extracts the reads belonging to each bin and reassembles them using a non-metagenomic assembler (like SPAdes), which typically improves N50 and reduces contamination.

```bash
metawrap reassemble_bins -o reassembled_out -b refined_out/metawrap_50_10_bins -1 reads_1.fastq -2 reads_2.fastq
```

## Expert Tips and Best Practices
*   **Resource Allocation**: MetaWRAP is resource-intensive. Ensure at least 8 cores and 64GB of RAM are available, especially for the Kraken and Assembly modules.
*   **Refinement Flexibility**: The `bin_refinement` module is standalone. You can feed it bins generated by any software or even the same software run with different parameters.
*   **Abundance Estimation**: Use the `quant_bins` module after refinement to estimate the abundance of your MAGs across different samples.
*   **Blobology**: For troubleshooting or manual inspection of binning quality, use the `blobology` module to visualize the GC content, coverage, and taxonomic affiliation of contigs.
*   **Python Version**: MetaWRAP traditionally relies on Python 2.7 for several of its wrapped dependencies; always use the dedicated conda environment to avoid system-wide conflicts.
