# coconet-binning

## Overview
CoCoNet (Composition and Coverage Network) is a specialized bioinformatic tool for binning viral contigs. Unlike general metagenomic binners, CoCoNet leverages a neural network to learn dataset-specific features from k-mer composition and read coverage. It predicts the likelihood that pairs of contigs originated from the same genome and uses these probabilities to infer clusters. This approach is particularly effective for capturing the diversity of viral populations in complex environmental samples.

## Command Line Usage

The tool is executed via the `coconet` command-line interface.

### Core Command
To execute the standard binning workflow, use the `run` subcommand:

```bash
coconet run --fasta assembly.fasta --bam sample1.bam sample2.bam --output results_dir
```

### Essential Arguments
- `--fasta`: The input assembly file containing viral contigs.
- `--bam`: One or more BAM files generated by mapping reads from your samples back to the assembly. Providing multiple samples improves binning accuracy through differential coverage.
- `--output`: The directory where CoCoNet will store intermediate files (like the coverage matrix) and the final binning results.
- `--threads`: Specify the number of CPU cores to use for the composition analysis and coverage calculations.

### Verification
To ensure the environment is correctly configured, you can run the help menu or the internal test suite:

```bash
coconet run -h
# Or run the test if the source is available
make test
```

## Expert Tips and Best Practices

- **Contig Length Filtering**: Viral signal is often noisy on short fragments. For optimal deep learning performance, it is recommended to filter your assembly to contigs $\ge$ 1,500 bp.
- **GPU Acceleration**: CoCoNet utilizes PyTorch for its neural network. If a CUDA-enabled GPU is available, the training phase will be significantly faster. Ensure your environment has the appropriate version of `torch` installed to recognize the GPU.
- **Coverage Calculation**: The tool calculates coverage in windows. If you have extremely high coverage or very long contigs, you may need to monitor disk space in the output directory, as the `coverage.h5` files can become large.
- **Unsupervised Learning**: Because CoCoNet learns from the data itself, the quality of the bins is directly related to the quality of the input assembly and the depth of the mapping. Ensure your BAM files are sorted and indexed.
- **Pre-clustering**: For extremely large datasets, consider pre-filtering or pre-clustering highly similar contigs to reduce the complexity of the adjacency matrix that the neural network must process.
