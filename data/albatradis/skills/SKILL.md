# albatradis

## Overview

AlbaTraDIS is a specialized bioinformatics tool for the comparative analysis of TraDIS data. It identifies genomic regions where transposon insertion patterns differ significantly between a control group and a test condition. By utilizing statistical methods to compare insert site plot files against an annotated reference genome, it helps researchers pinpoint genes involved in survival or fitness under specific environmental stresses. It is particularly effective at identifying both large-scale changes (like gene essentiality) and subtle regulatory shifts.

## Installation and Setup

The tool is most reliably installed via Bioconda or used via Docker to manage its dependencies (specifically the QuaTradis toolkit and R 3.6+).

```bash
# Installation via Conda
conda install -c conda-forge -c bioconda albatradis
```

## Core Command Line Usage

The primary command requires an annotated reference in EMBL format followed by the condition plot files and then the control plot files.

```bash
albatradis [options] <reference.embl> <condition_plot1 condition_plot2> <control_plot1 control_plot2>
```

### Positional Arguments
- **EMBL file**: An annotated reference genome (e.g., from EBI).
- **Plot files**: Insert site plot files generated by `bacteria_tradis`. You must provide an equal number of condition and control files (minimum 1 of each, though 2+ is recommended for statistical power).

### Key Analysis Options
- `--use_annotation, -a`: Switches from the default sliding window analysis to using the reference annotation directly. Use this when you are specifically interested in known gene features.
- `--span_gaps, -s <int>`: Merges fragmented blocks of significant change. Useful for large genes where signals might be interrupted, but setting this too high can erroneously merge distinct biological mechanisms.
- `--iterations, -i <int>`: Performs multiple rounds of analysis. In each round, the strongest signals are identified and masked, allowing the tool to find weaker signals in subsequent passes.
- `--minimum_logfc, -f <float>`: Sets the threshold for log fold change (default: 1).
- `--prefix, -o <string>`: Defines the output directory prefix.

## Expert Tips and Best Practices

- **Replicate Consistency**: Always aim for at least two replicates per condition. AlbaTraDIS uses statistical methods that benefit significantly from the variance information provided by replicates.
- **Window Tuning**: If not using `--use_annotation`, the default window size is 100bp (`-w 100`) with a 25bp interval (`-l 25`). For very dense libraries, you may decrease the window size for higher resolution; for sparse libraries, increase it to capture enough signal.
- **Iterative Discovery**: If your data has a very dominant signal (e.g., a single gene providing massive resistance), use `--iterations 3` or higher to ensure that secondary, less obvious fitness factors are not overlooked.
- **Normalisation**: By default, the tool normalises input plots. If you have already pre-normalised your data using external scripts, use the `--dont_normalise_plots` flag to prevent double-normalisation.
- **Memory Management**: For large-scale comparative datasets with many replicates, ensure the environment has sufficient RAM, as the tool loads multiple plot files into memory for comparison.
