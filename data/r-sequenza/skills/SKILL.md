# r-sequenza

## Overview
`sequenza` is an R package designed to analyze Next Generation Sequencing (NGS) data from tumor samples (paired with normal controls). It provides functions to estimate tumor purity, average ploidy, and chromosome-specific copy number profiles by modeling the relationship between depth ratio and B-allele frequency.

## Installation
```R
install.packages("sequenza")
```
Note: The package requires pre-processed input files (typically generated by the `sequenza-utils` Python tool) in the `.seqz` format.

## Core Workflow

### 1. Data Loading and Pre-processing
The entry point is a `.seqz` file. For large files, use `sequenza.extract` to downsample and calculate statistics.

```R
library(sequenza)

# Extract data from a seqz file
test_data <- sequenza.extract("sample.seqz.gz", verbose = FALSE)
```

### 2. Parameter Estimation
Estimate tumor purity (cellularity) and ploidy by scanning a grid of possible values.

```R
# Run the grid-search for cellularity and ploidy
CP_results <- sequenza.fit(test_data)

# Export results and plots to a directory
sequenza.results(sequenza.extract = test_data, 
                 cp.table = CP_results, 
                 sample.id = "Sample01", 
                 out.dir = "sequenza_results")
```

### 3. Visualization
The package provides several plotting functions to inspect the model fit and genomic profiles.

```R
# Plot the CP (Cellularity/Ploidy) likelihood surface
cp.plot(CP_results)

# Plot genome-wide copy number and BAF
genome.view(test_data, info = CP_results$max.res, main = "Sample01")
```

## Key Functions

- `sequenza.extract()`: Reads the seqz file, performs normalization, and calculates log-ratios and BAF.
- `sequenza.fit()`: Performs the Bayesian inference to find the best-fit cellularity and ploidy.
- `sequenza.results()`: Generates a comprehensive report including plots and text files.
- `get.model.stats()`: Extracts specific statistics from the fitted model.
- `chromosome.view()`: Provides a detailed view of a specific chromosome.

## Tips for Success
- **Input Quality**: Ensure the input `.seqz` file is correctly formatted. It should contain columns for chromosome, position, base in reference, depth in normal, depth in tumor, and BAF.
- **Normalization**: `sequenza.extract` handles GC-content normalization by default.
- **Memory Management**: For whole-genome sequencing (WGS), the `sequenza.extract` function is essential to reduce the data size into manageable windows.
- **Manual Override**: If the automated grid search picks a sub-optimal peak, you can manually specify cellularity and ploidy in the `sequenza.results` function using the `cellularity` and `ploidy` arguments.
