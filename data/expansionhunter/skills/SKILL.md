# expansionhunter

## Overview
ExpansionHunter is a sequence-graph based tool designed to genotype short tandem repeats (STRs). It excels at identifying repeat expansions that are significantly longer than the sequencing read length by analyzing reads that span, flank, or are fully contained within the repeat region. Use this skill to guide the execution of repeat-sizing workflows, from preparing variant catalogs to interpreting VCF and JSON outputs.

## Installation
The tool is most easily installed via Bioconda:
```bash
conda install -c bioconda expansionhunter
```

## Core Command Line Usage
The basic execution requires an alignment file (BAM/CRAM), a reference genome, and a variant catalog defining the target loci.

```bash
ExpansionHunter \
  --reads sample.bam \
  --reference reference.fasta \
  --variant-catalog variant_catalog.json \
  --output-prefix sample_results \
  --sex male
```

### Required Arguments
- `--reads`: Path to the BAM or CRAM file. The file must be indexed.
- `--reference`: Path to the reference genome FASTA file used for alignment.
- `--variant-catalog`: A JSON file defining the STR loci to be analyzed.
- `--output-prefix`: The prefix used for all output files (VCF, JSON, and BAM).

### Important Optional Arguments
- `--sex`: Specify `male` or `female`. This is critical for the correct genotyping of repeats located on the X and Y chromosomes.
- `--log-level`: Set to `debug` or `info` for troubleshooting execution issues.

## Best Practices and Expert Tips
- **Input Data Quality**: For the most accurate results, use PCR-free whole-genome sequencing (WGS) data. PCR-based libraries can introduce artifacts that complicate repeat size estimation.
- **Variant Catalogs**: Always ensure your `variant_catalog.json` matches the coordinate system of your reference genome (e.g., GRCh37 vs. GRCh38). Using a catalog from a different assembly will result in incorrect or empty calls.
- **Duplicate Marking**: While standard pipelines mark duplicates, ExpansionHunter is designed to handle the high-depth nature of repeat regions. Ensure your input BAM is the original alignment; excessive filtering or recalibration (BQSR) is generally not required but the file must be sorted and indexed.
- **Output Interpretation**:
    - **VCF File**: Contains the genotypes and confidence intervals for each locus.
    - **JSON File**: Provides more granular details, including the number of spanning and flanking reads identified.
- **Visualization**: To validate suspicious expansions, use the companion tool **REViewer** to visualize the read alignments against the sequence graphs generated by ExpansionHunter.

## Common Troubleshooting
- **Missing Index**: If the tool fails immediately, verify that the `.bai` (for BAM) or `.crai` (for CRAM) index files are in the same directory as the reads.
- **Locus Structure Errors**: If a specific locus fails, check the `LocusStructure` field in the variant catalog. It must correctly define the repeat unit (motif) and any flanking sequences.
