We incorporated optimizations from mm2-fast (v1.0) and implemented parallel algorithms for efficient genome-to-genome alignment. The optimizations include: Parallel chaining Faster interval tree-based algorithm for selecting primary chains AVX2/AVX512 SIMD alignment (from mm2-fast ) AVX2/AVX512 SIMD chaining (from Intel TAL ) Parallel sorting (GNU parallel stable sort std::stable_sort ) mm2-plus serves as a drop-in replacement for minimap2 , providing near-identical output while significantly reducing execution time, especially for genome-to-genome alignments. Prerequisites mm2-plus depends on the following: GCC 9 or later - GCC Zlib - zlib Jemalloc - jemalloc We provide an automated script to simplify the dependency installation. Assuming you already have GCC, you can use the command make deps to install both zlib and jemalloc automatically. Get mm2-plus # Build and Install locally git clone https://github.com/at-cg/mm2-plus cd mm2-plus # build jemalloc and zlib make deps # make mm2plus make # Install with bioconda conda install bioconda::mm2plus # Install with docker docker pull quay.io/biocontainers/mm2plus:1.2--h9ee0642_0 # test run ./mm2plus -cx asm20 test/MT-human.fa test/MT-orang.fa &gt; out.paf Usage mm2-plus offers same command line interface as minimap2. Therefore, users can refer to minimap2 README . Here are some test examples for mm2-plus run # test read alignment ./mm2plus -cx map-ont test/MT-human.fa test/reads.fq &gt; out.paf # test genome alignment ./mm2plus -cx asm20 test/MT-human.fa test/MT-orang.fa &gt; out.paf Accuracy evaluation 1) Read Alignment The accuracy of mm2-plus can be validated against minimap2 (v2.30). Ensure the --max-chain-skip=1000000 parameter is used during the chaining phase for high precision. Follow these steps: # Run mm2-plus git clone https://github.com/at-cg/mm2plus.git cd mm2-plus &amp;&amp; make deps &amp;&amp; make ./mm2plus -ax map-ont test/MT-human.fa test/reads.fq --max-chain-skip=1000000 &gt; mm2plus.sam # Run minimap2 git clone https://github.com/lh3/minimap2.git -b v2.30 cd minimap2 &amp;&amp; make ./minimap2 -ax map-ont test/MT-human.fa test/reads.fq --max-chain-skip=1000000 &gt; mm2.sam Compare the outputs: # Get the difference (skip the line with tools name and version) diff &lt;( grep -v ' ^@PG ' mm2.sam ) &lt;( grep -v ' ^@PG ' mm2plus.sam ) The diff command should return null output, indicating no differences (0 lines). 2) Genome Alignment To compare genome alignments, use the following commands: # Run mm2-plus git clone https://github.com/at-cg/mm2-plus.git cd mm2-plus &amp;&amp; make deps &amp;&amp; make ./mm2plus -cx asm20 test/MT-human.fa test/MT-orang.fa &gt; mm2plus.paf # Run minimap2 git clone https://github.com/lh3/minimap2.git -b v2.30 cd minimap2 &amp;&amp; make ./minimap2 -cx asm20 test/MT-human.fa test/MT-orang.fa &gt; mm2.paf Compare the outputs: # Get the difference diff mm2.paf mm2plus.paf Both diff commands should return null output. For large genomes, there may be negligible differences in the PAF output. Refer to the paper for more information. Performance We observed up to 7x speedup for genome-to-genome alignment across all tested datasets. The speedup for long-read alignment is comparable to that achieved by mm2-fast . To see a thorough evaluation, please refer to our paper . Performance comparison of whole-genome alignment between minimap2 and mm2-plus across various datasets, including plant, human, and primate genomes. The experiments were conducted using 48 threads on a Cascade Lake processor.
