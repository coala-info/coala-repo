# ezaai

## Overview

EzAAI is a high-throughput computational suite designed for comparative genomics. It provides a streamlined workflow for calculating Average Amino Acid Identity (AAI), a key metric for defining microbial species and genera. Use this skill to process raw genomic sequences into protein databases, compute similarity matrices between multiple organisms, and visualize these relationships through hierarchical clustering. It is optimized for performance by leveraging Prodigal for gene prediction and MMseqs2 for sequence alignment.

## Installation

The most efficient way to deploy EzAAI is via Bioconda:

```bash
conda install -c bioconda ezaai
```

## Core Workflow and CLI Patterns

### 1. Protein Extraction
Before calculating AAI, you must extract protein sequences from your genome files (FASTA). This module uses Prodigal internally.

```bash
ezaai extract -i <INPUT_GENOMES> -o <OUTPUT_DIRECTORY>
```
- **Input**: Can be a single genome file or a directory containing multiple prokaryotic genomes.
- **Output**: A directory where the generated protein databases will be stored.

### 2. AAI Calculation
Calculate the identity values between two protein databases or two directories of databases.

```bash
ezaai calculate -i <DB_DIR_1> -j <DB_DIR_2> -o <RESULT_FILE>
```
- **Note**: To perform an all-vs-all comparison, point both `-i` and `-j` to the same directory of protein databases.
- **Output**: A text file containing the pairwise AAI values.

### 3. Hierarchical Clustering
Transform the AAI results into a hierarchical structure for taxonomic analysis.

```bash
ezaai cluster -i <AAI_RESULT_FILE> -o <CLUSTER_OUTPUT>
```
- **Input**: The result file generated by the `calculate` module.
- **Output**: Clustering results and dendrogram representation data.

### 4. Database Conversion
If you already have CDS (Coding Sequence) profiles in FASTA format, convert them directly to the EzAAI/MMseqs2 format.

```bash
ezaai convert -i <IN_CDS_FASTA> -s <SEQ_TYPE> -o <OUT_DB>
```
- **Sequence Type (`-s`)**: Specify `nucl` for nucleotide or `prot` for protein sequences.

## Expert Tips and Best Practices

- **Batch Processing**: When dealing with hundreds of genomes, always use directory paths for the `extract` and `calculate` modules rather than individual files to leverage the tool's internal parallelization.
- **Memory Management**: Since EzAAI runs on the JVM, ensure your environment has sufficient heap space for large-scale all-vs-all calculations.
- **Input Validation**: Ensure genome headers are unique across your dataset, as EzAAI uses these identifiers for the final AAI matrix and clustering labels.
- **Sequence Type**: When using the `convert` module, double-check the `-s` flag. Providing `nucl` for a protein file (or vice versa) will result in calculation errors or nonsensical AAI values.
