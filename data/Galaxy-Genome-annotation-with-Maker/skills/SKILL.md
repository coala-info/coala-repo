# Genome annotation with Maker

## Overview

This workflow provides a comprehensive pipeline for structural genome annotation using the [Maker](https://toolshed.g2.bx.psu.edu/repos/iuc/maker/maker/2.31.11) ecosystem. It integrates evidence from EST/cDNA and protein sequences with a target genome sequence to identify and refine gene models. Initial quality control is performed using [Fasta Statistics](https://toolshed.g2.bx.psu.edu/repos/iuc/fasta_stats/fasta-stats/1.0.1) and [BUSCO](https://toolshed.g2.bx.psu.edu/repos/iuc/busco/busco/4.1.4) to establish a baseline for the assembly's completeness.

The core of the process is an iterative training loop designed to optimize gene prediction accuracy. After an initial Maker run, the workflow uses the resulting gene models to [Train Augustus](https://toolshed.g2.bx.psu.edu/repos/bgruening/augustus_training/augustus_training/3.3.3) and [Train SNAP](https://toolshed.g2.bx.psu.edu/repos/iuc/snap_training/snap_training/2013_11_29+galaxy1). These species-specific models are then utilized in subsequent Maker rounds to progressively improve the annotation of gene structures.

Throughout the iterative cycles, [Genome annotation statistics](https://toolshed.g2.bx.psu.edu/repos/iuc/jcvi_gff_stats/jcvi_gff_stats/0.8.4) and BUSCO are employed to monitor the quality and biological completeness of each version. The final outputs include a refined GFF file with standardized IDs via [Map annotation ids](https://toolshed.g2.bx.psu.edu/repos/iuc/maker_map_ids/maker_map_ids/2.31.11), extracted CDS and protein sequences generated by [gffread](https://toolshed.g2.bx.psu.edu/repos/devteam/gffread/gffread/2.2.1.2), and a [JBrowse](https://toolshed.g2.bx.psu.edu/repos/iuc/jbrowse/jbrowse/1.16.10+galaxy0) instance for interactive visualization.

This workflow is licensed under GPL-3.0-or-later and follows the best practices established by the Galaxy Training Network (GTN) for genome annotation.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | EST and/or cDNA | data_input | It contains the sequences used by Maker to construct gene models. |
| 1 | Protein sequences | data_input | A set of protein sequences, usually from closely related species or from a curated sequence database like UniProt/SwissProt. |
| 2 | Genome sequence | data_input | This file corresponds to the sequence to be annotated. |


Ensure all input files for the genome, protein sequences, and ESTs are in FASTA format, verifying that the genome assembly is properly masked to improve annotation accuracy. While individual datasets are standard, using collections for evidence files can streamline the iterative training rounds required for SNAP and Augustus. Consult the README.md for specific instructions on parameterizing the three successive Maker runs and interpreting the BUSCO quality metrics. You may use `planemo workflow_job_init` to create a `job.yml` file for configuring these inputs for automated execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run main-workflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run main-workflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run main-workflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init main-workflow.ga -o job.yml`
- Lint: `planemo workflow_lint main-workflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
