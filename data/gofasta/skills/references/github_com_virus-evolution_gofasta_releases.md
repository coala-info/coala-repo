[v1.2.0](/virus-evolution/gofasta/releases/tag/v1.2.0)

#### Notes

This release incorporates the following extra functionality:

* The ability to annotate amino acid changes in genes which are on the reverse strand using `variants` and `sam variants` ([#37](https://github.com/virus-evolution/gofasta/issues/37))
* The ability to restrict the output of `variants` and `sam variants` by genomic position
* The ability to wrap the output fastas produced by `sam toMultiAlign` and `sam toPairAlign` to a set number of bases wide

And fixes the following bugs:

* Fasta files with records up to 1MB on a single line can now be read without overflowing the buffer ([#37](https://github.com/virus-evolution/gofasta/issues/37))
* Genbank and gff protein-coding features on the reverse strand are now handled (see above)

#### Full changelog:

[v1.1.0...v1.2.0](https://github.com/virus-evolution/gofasta/compare/v1.1.0...v1.2.0)

[v1.1.0](/virus-evolution/gofasta/releases/tag/v1.1.0)

#### Notes

This release incorporates the following extra functionality:

* The ability to provide a genome annotation in gff (version 3) format to `variants` and `sam variants`
* The ability to push node distances outwards for non-empty bins in `updown topranking`
* The ability to write long-form outputs including the distances between targets and queries using the `--table` flag for `closest` and `updown topranking`
* Extra distance measures (number of snps and tn93 evolutionary distance) to `closest`

And fixes the following bugs:

* A reference fasta file is no longer required for `updown topranking` if both target and query input files are csvs
* The same amino acid mutation with different underlying nucleotide mutations is no longer represented twice in the output of `variants` and `sam variants` when using the `--aggregate` flag unless `--append-snps` is also called.

#### Full changelog:

[v1.0.0...v1.1.0](https://github.com/virus-evolution/gofasta/compare/v1.0.0...v1.1.0)
