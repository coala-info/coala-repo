# cooler

## Overview

`cooler` is a specialized toolkit for managing genomic interaction matrices, such as those generated by Hi-C experiments. It utilizes a sparse, compressed binary format based on HDF5 to store large-scale contact data efficiently. This skill enables the transformation of raw genomic pairs into structured matrices, the application of normalization (balancing) to remove experimental biases, and the extraction of specific genomic regions for analysis or visualization.

## Common CLI Patterns

### Creating Cooler Files
To create a `.cool` file from a list of genomic bins and a sorted contact list (e.g., a `.pairs` file):

```bash
# Create a cooler from a pairs file
cooler cload pairs -c1 2 -p1 3 -c2 4 -p2 5 <chrom.sizes>:<bin_size> <input.pairs> <output.cool>

# Create a cooler from a pre-calculated 3-column pixel file (bin1_id, bin2_id, count)
cooler load <chrom.sizes>:<bin_size> <pixels.tsv> <output.cool>
```

### Matrix Balancing (Normalization)
Normalization is critical for Hi-C analysis to correct for biases like GC content or mappability. The `balance` command implements the Iterative Correction and Evaluation (ICE) algorithm.

```bash
# Apply ICE normalization to a cooler file
cooler balance <input.cool>

# Balance using multiple cores for speed
cooler balance -p 8 <input.cool>
```

### Multi-resolution Files (.mcool)
For visualization in tools like HiGlass, you often need multiple resolutions (bin sizes) stored in a single file.

```bash
# Generate a multi-resolution .mcool file from a high-resolution .cool file
cooler zoomify <input.cool>
```

### Data Extraction and Querying
Use `dump` to export matrix data to text formats or `show` for quick terminal-based visualization.

```bash
# Export pixels to a TSV file
cooler dump --join <input.cool> > contacts.tsv

# Extract a specific genomic region (e.g., Chromosome 1)
cooler dump --range chr1:10,000,000-20,000,000 <input.cool>

# View a heatmap of a region in the terminal
cooler show <input.cool> chr2:10,000,000-15,000,000
```

### Matrix Manipulation
```bash
# Coarsen a matrix (increase bin size)
cooler coarsen -k 4 -o <output_coarse.cool> <input_fine.cool>

# Merge multiple cooler files (must have same bins)
cooler merge <output.cool> <input1.cool> <input2.cool>
```

## Expert Tips

- **Storage Efficiency**: Always use `.mcool` for large datasets intended for visualization; it prevents the need to manage dozens of individual resolution files.
- **Normalization Check**: After running `cooler balance`, the resulting file will contain a `weight` column in the `bins` table. Downstream tools use these weights to provide "balanced" counts.
- **Input Sorting**: When using `cload`, ensure your input pairs file is sorted by chromosome and position to avoid processing errors.
- **Parallelization**: Many `cooler` commands support the `-p` or `--nproc` flag. Use this on high-performance computing clusters to significantly reduce processing time for large matrices.
