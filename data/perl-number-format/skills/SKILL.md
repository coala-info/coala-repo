# perl-number-format

## Overview
The `perl-number-format` skill enables the transformation of raw numbers into localized or custom-formatted strings. It provides a flexible engine for handling decimal points, thousands separators, and negative number representations. This is particularly useful for bioinformaticians and data scientists who need to generate clean reports or logs from numeric output generated by command-line tools.

## Installation
If the environment does not already have the module, install it via Bioconda:
```bash
conda install bioconda::perl-number-format
```

## Common CLI Patterns
While primarily a Perl library, you can use it effectively in shell pipelines via Perl one-liners.

### Basic Number Formatting
Add thousands separators (default is a comma) to a number:
```bash
echo "1234567.89" | perl -MNumber::Format=:subs -ne 'print format_number($_)'
# Output: 1,234,567.89
```

### Formatting Bytes
Convert large integers (e.g., file sizes from `stat`) into human-readable formats:
```bash
# Traditional (powers of 1024, suffixes K, M, G)
perl -MNumber::Format=:subs -le 'print format_bytes(1073741824)'
# Output: 1G

# IEC Standard (KiB, MiB, GiB)
perl -MNumber::Format=:subs -le 'print format_bytes(1048576, 2, "iso60027")'
# Output: 1.00MiB
```

### Rounding in Pipelines
Round values to a specific precision:
```bash
echo "3.14159" | perl -MNumber::Format=:subs -ne 'print round($_, 2)'
# Output: 3.14
```

## Expert Usage and Best Practices

### Object-Oriented vs. Functional Interface
- **Functional (`:subs`)**: Best for quick one-liners or simple scripts where default settings (US-style) are sufficient.
- **Object-Oriented (`new`)**: Best for complex scripts or when you need to handle multiple locales simultaneously.

### Customizing Separators (International Formats)
To use European formatting (dot for thousands, comma for decimal):
```perl
use Number::Format;
my $de = new Number::Format(
    -thousands_sep => '.',
    -decimal_point => ',',
    -int_curr_symbol => 'EUR'
);
print $de->format_number(1234.56); # 1.234,56
```

### Handling Negative Numbers
Use `NEG_FORMAT` to control how negative values appear. The 'x' acts as a placeholder for the absolute value:
- `x-` : Trailing minus
- `(x)` : Parentheses (accounting style)
- `-x` : Leading minus (default)

### Precision and Padding
- Use `DECIMAL_DIGITS` to set default rounding.
- Set `DECIMAL_FILL => 1` to force trailing zeros (e.g., 1.5 becomes 1.50), which is essential for financial alignment.

### Unformatting
To convert a formatted string back into a raw number for calculation:
```perl
my $raw = $x->unformat_number("1,234,567.89"); # 1234567.89
```
