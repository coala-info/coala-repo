Most modules require specifying the result folder from the PGAP2 main run, such as `stat`, `singletree`, `baps`, and `tajimas_d`, because PGAP2 needs to read various output files from the entire analysis. However, the `profile` module only requires the **PAV file**, meaning users can provide PAV results generated by other software. PGAP2's post-profile function will process the PAV file for statistical analysis and visualization, similar to tools like PanGP.

> PGAP2 provides several practical downstream analysis pipelines that integrate other software required for these analyses. If users utilize any of these pipelines, in addition to citing PGAP2, they must also \*\*cite the other software \*\*included in PGAP2's workflow. The software used in each downstream analysis pipeline of PGAP2 is listed [here](https://github.com/bucongfan/PGAP2/wiki/Installation#postprocessing)

# profile

## Quick start

```
pgap2 post profile --pav your.pav.file --outdir .
```

## Parameters

```
usage: main.py post profile [-h] --pav PAV [--outdir OUTDIR] [--nodraw] [--single_file] [--S S] [--N N] [--R R]
                            [--K K] [--method {TR,DG}] [--threads THREADS] [--disable] [--verbose] [--debug]

options:
  -h, --help            show this help message and exit
  --pav PAV, -p PAV     The pav matrix (default: None)
  --outdir OUTDIR, -o OUTDIR
                        Output directory (default: .)
  --nodraw              Only output flat file, but no drawing plot (default: False)
  --single_file, -s     Output each vector plot as a single file (default: False)
  --S S                 Number of strains to sample in each bin (default: 100)
  --N N                 Number of bins to sample (default: 100)
  --R R                 Number of repeats for every sample (default: 1)
  --K K                 Inflated cof for DG method (default: 3)
  --method {TR,DG}      Sampling method. TR: total random; DG: diversity guided (default: TR)
  --threads THREADS, -t THREADS
                        Number of threads (default: 4)
  --disable             Disable progress bar (default: False)
  --verbose             Verbose output (default: False)
  --debug               Debug mode. Note: very verbose (default: False)
```

## Output files

1. postprocess.new\_clusters.tsv: New cluster number as the increasing of genome number
2. postprocess.rarefaction.tsv: Core and Pan cluster number as the increasing of genome number
3. postprocess.clust\_strain\_freq.tsv: Strain number and proportion of each cluster
4. postprocess.pan\_group\_stat.tsv: Statictics of pan-group
5. postprocess.para\_stat.tsv: Duplicated gene ratio in each cluster
6. postprocess.curve\_fit.txt: the core gene reduction curve and pan gene expansion curve follow exponential decay and Heap's law
7. postprocess\_profile.html: Interactive visualization result of profile result

# stat

## Quick start

```
pgap2 post stat --indir your.main_result.dir --outdir .
```

## Parameters

```
usage: main.py post stat [-h] --indir INDIR [--outdir OUTDIR] [--nodraw] [--single_file] [--S S] [--N N] [--R R]
                         [--K K] [--method {TR,DG}] [--threads THREADS] [--verbose] [--debug] [--disable]

options:
  -h, --help            show this help message and exit
  --indir INDIR, -i INDIR
                        Input directory generated by partition step
  --outdir OUTDIR, -o OUTDIR
                        Output directory
  --nodraw              Only output flat file, but no drawing plot
  --single_file, -s     Output each vector plot as a single file
  --S S                 Number of strains to sample in each bin
  --N N                 Number of bins to sample
  --R R                 Number of repeats for every sample
  --K K                 Inflated cof for DG method
  --method {TR,DG}      Sampling method. TR: total random; DG: diversity guided
  --threads THREADS, -t THREADS
                        threads used in parallel
  --verbose, -V         Verbose output
  --debug               Debug mode. Note: very verbose
  --disable             Disable progress bar
```

## Output files

The output of the `stat` module is generally similar to that of `profile`. Please refer to [here](https://github.com/bucongfan/PGAP2/wiki/Postprocessing#output-files) for details. The only difference is that PGAP2 performs additional statistical analysis and visualization based on its unique quantitative parameters.

# singletree

**SingleTree** is a tool used for generating a phylogenetic tree based on a **single-copy core gene** alignment. The method involves identifying and using genes that are universally present in all or most genomes of interest, excluding genes that may have undergone duplication or loss in specific strains. The result is a tree that reflects the evolutionary relationships between genomes based on a conserved set of core genes.

## Quick start

```
pgap2 post singletree --indir your.main_result.dir --outdir .
```

## Workflows

1. Extract core cds and prot
2. Multiple sequence alignment using --msa\_method assigned.
3. Codon alignment: Trans multiple protein alignment to corresponding nucleotide alignment.
4. Trim alignment using ClipKit.
5. Core alignment concatenation.
6. Phylogeny tree construction using tree\_method assigned software.
