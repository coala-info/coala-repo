# Coverage computations

## Overview

To get started run the command `kmertools cov --help` and you will see the following help output.

```
Generates coverage histogram based on the reads

Usage: kmertools cov [OPTIONS] --input <INPUT> --output <OUTPUT>

Options:
  -i, --input <INPUT>
          Input file path

  -a, --alt-input <ALT_INPUT>
          Input file path, for k-mer counting

  -o, --output <OUTPUT>
          Output directory path

  -k, --k-size <K_SIZE>
          K size for the coverage histogram

          [default: 15]

  -p, --preset <PRESET>
          Output type to write

          [default: spc]

          Possible values:
          - csv: Comma separated format
          - tsv: Tab separated format
          - spc: Space separated format

  -s, --bin-size <BIN_SIZE>
          Bin size for the coverage histogram

          [default: 16]

  -c, --bin-count <BIN_COUNT>
          Number of bins for the coverage histogram

          [default: 16]

  -m, --memory <MEMORY>
          Max memory in GB

          [default: 6]

      --counts
          Disable normalisation and output raw counts

  -t, --threads <THREADS>
          Thread count for computations 0=auto

          [default: 0]

  -h, --help
          Print help (see a summary with '-h')
```

## Notes

The algorithm has the following steps. Each formed vector can be thought of as a histogram (k-mer coverage histogram).

* For a given k-mer size and a sequence file, build a counts table
* Iterate the target sequences
  + For each k-mer in the sequence
    - Get count from table
    - Decide the bin (count/bin-size)
    - Increment the bin value by 1
  + Normalise the counts vector by total sum of all entries

Note that bin-count variable clip the vector size and all high abundance k-mers are grouped in the last index.

## Options

### Input and alt input

You can specify input for which the k-mers are counted for and vectors are generated. However, if you wanted to perform a multi-sample analysis by computing coverage of a read from one sample against another you can use alt-input. In this scenario, k-mers are counted from `alt-input` path and sequences are generated for `input` path sequences.

### Bin size and count

This decides the size of the vector. Bin size is the width of a histogram index. For example, `--bin-size 10` creates a histogram of each index holding 10 counts, first being from `0-9`, second `10-19`, and so on. Here, `--bin-count 15` would limit the captured count to 15 bins. Maximum coverage meaningfully captured will be $15\times10=150$. Higher coverage k-mers will be counted into the last bin.

### Memory

This is the maximum memory used for k-mer counting. This value is in GB and it it used to decide on the number of chunks for k-mer counting. The algorithm is similar to that of DSK, but not exactly. But this gets the required input for coverage histogram creation in one-shot and in correct formatting.

### Counts

This disables normalisations and return vectors of counts.

## Example application

### K-mer coverage histogram comparison of viruses *Escherichia phage phiX174* and *Anellovirus*

Download [`NC_001422.1`](https://www.ncbi.nlm.nih.gov/nuccore/NC_001422.1?report=fasta) and [`OR148954.1`](https://www.ncbi.nlm.nih.gov/nuccore/OR148954.1?report=fasta) from NCBI. These are provided here; [`NC_001422.1`](./examples/cov/NC_001422.1.fasta) and [`OR148954.1`](./examples/cov/OR148954.1.fasta).

Generate reads using [InSilicoSeq](https://insilicoseq.readthedocs.io/en/latest/) sequencing simulator. We will generate MiSeq reads with 50x coverage from `NC_001422.1` and 100x coverage from `OR148954.1` (The `coverage.txt` file can be found [here](./examples/cov/coverage.txt)). The generated read files are provided here; [`reads_R1.fastq`](./examples/cov/reads_R1.fastq) and [`reads_R2.fastq`](./examples/cov/reads_R2.fastq).

```
iss generate --genomes examples/cov/*.fasta --coverage_file examples/cov/coverage.txt --model miseq --output examples/cov/reads --seed 3
```

Run `kmertools` to get the k-mer counts and k-mer coverage histograms. We will give a bin size of 5 and bin count of 20.

```
kmertools cov -p csv -s 5 -c 20 -i examples/cov/reads_R1.fastq -o examples/cov/kmer_cov_reads
```

Load from python as follows ([script](./examples/cov/cov.1.py)).

```
import numpy as np
import pandas as pd
import seaborn as sns

import matplotlib.pyplot as plt

# Get read labels
with open("examples/cov/reads_R1.fastq", "r") as myfile:
    read_labels = ["_".join(line.strip().split()[0][1:].split("_")[:-2]) for i, line in enumerate(myfile) if i % 4 == 0]

# Read vectors
with open("examples/cov/kmer_cov_reads/kmers.vectors", "r") as myfile:
    data = myfile.read()

data = data.strip().split('\n')
data = [list(map(float, row.split(','))) for row in data]
np_array = np.array(data)

# Define custom headers
headers = [f"bin_{i+1}" for i in range(20)]

# Convert the numpy array to a pandas DataFrame with custom headers
df = pd.DataFrame(np_array, columns=headers)

# Adding the new column
df['Label'] = read_labels

# Melt the filtered DataFrame for better plotting with seaborn
df_melted = df.melt(id_vars='Label', var_name='Feature', value_name='Value')

# Create boxen plot
plt.figure(figsize=(15, 6))
sns.boxenplot(data=df_melted, x="Feature", y="Value", hue="Label")
plt.title("Boxen plot for k-mer coverage histograms")
plt.xlabel("Bins")
plt.ylabel("Normalised frequency")
plt.savefig("examples/cov/kmer_cov_hist.png", dpi=150, format="png", bbox_inches='tight')
plt.show()
```
