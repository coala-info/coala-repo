# masurca

## Overview
MaSuRCA is a powerful genome assembly toolkit that uses the "super-read" and "mega-read" algorithms to combine the high accuracy of Illumina data with the long-range information of third-generation sequencing. This skill provides guidance on running the standard assembly pipeline, using the simplified CLI for small projects, and applying specialized tools like SAMBA for scaffolding and `close_scaffold_gaps.sh` for gap filling.

## Core Workflows

### 1. Simplified CLI (Small Projects)
For small eukaryotes or bacteria using a single Illumina run and/or a single long-read file, use the direct CLI instead of a configuration file.
- **Illumina-only**: `masurca -t <threads> -i <R1.fastq>,<R2.fastq>`
- **Hybrid (Nanopore/PacBio)**: `masurca -t <threads> -i <R1.fastq>,<R2.fastq> -r <long_reads.fastq.gz>`

### 2. Standard Assembly (Large Projects)
For complex projects, MaSuRCA requires a configuration file (typically `sr_config.txt`).
1. **Generate Template**: Install MaSuRCA and locate `sr_config_example.txt`.
2. **Configure**: Define libraries (PE, JUMP, NANOPORE, PACBIO) and parameters (GRAPH_KMER_SIZE, USE_LINKING_MATES).
3. **Execute**:
   ```bash
   masurca config.txt
   ./assemble.sh
   ```

### 3. SAMBA Scaffolder
Use SAMBA to improve an existing assembly using long reads or another assembly.
- **Command**: `samba.sh -r <draft.fasta> -q <long_reads.fastq> -t <threads> -d <type>`
- **Data Types (-d)**: `ont` (default), `pbclr`, or `asm`.
- **Minimum Match (-m)**:
  - 2000-2500: Small genomes (100-400Mb).
  - 5000: Large eukaryotes (2-3Gbp).
  - 9000-10000: Highly repetitive plant genomes (5Gbp+).

### 4. Gap Filling
To close intra-scaffold gaps without merging scaffolds:
```bash
close_scaffold_gaps.sh -r <scaffolds.fasta> -q <long_reads.fastq> -t <threads>
```
*Note: This tool allows contig "flips" within scaffolds, making it ideal for fixing HiC-scaffolded assemblies.*

## Expert Tips
- **No Polishing Needed**: MaSuRCA's hybrid technique produces high-quality consensus; additional polishing (like Pilon) is generally unnecessary.
- **Memory Efficiency**: Version 4.0.1+ uses a k-unitig pre-correction that reduces memory requirements and speeds up mammalian assemblies by ~6x.
- **Input Formats**: Illumina files must be FASTQ (can be gzipped). Long reads can be FASTA or FASTQ.
- **SLURM Integration**: The `assemble.sh` script generated by the config phase is compatible with cluster schedulers.
