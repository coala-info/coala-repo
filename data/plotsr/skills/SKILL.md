# plotsr

## Overview
plotsr is a specialized visualization tool designed to generate high-quality comparative genomics plots. It takes structural annotations—typically generated by SyRI or provided in a specific BEDPE format—and maps them across multiple genomes in a stacked or interleaved orientation. It is the standard choice for researchers needing to illustrate complex structural variations and syntenic blocks between related species or accessions.

## Installation and Setup
The most reliable way to install plotsr is via Bioconda:
```bash
conda install -c bioconda plotsr
```

## Core Command Line Usage
The basic execution requires structural rearrangement (SR) files and a genomes configuration file.

### Basic Plotting
```bash
plotsr --sr genome1_vs_genome2.syri.out \
       --sr genome2_vs_genome3.syri.out \
       --genomes genomes.txt \
       -o comparative_plot.png
```

### Advanced Plotting with Tracks and Markers
To add functional data (GFF/BED) and specific highlights:
```bash
plotsr --sr A_B.out --sr B_C.out \
       --genomes genomes.txt \
       --tracks tracks.txt \
       --markers markers.bed \
       --cfg base.cfg \
       -S 0.5 -W 12 -H 8 -f 10 \
       -o final_plot.pdf
```

## Input File Specifications

### Genomes File (`genomes.txt`)
A tab-separated file defining the genomes in the order they should be plotted.
- **Column 1**: Path to FASTA file or chromosome length file.
- **Column 2**: Name to display on the plot.
- **Column 3 (Optional)**: Visual tags (e.g., `lw:1.5` for line width, `lc:red` for color, `ft:cl` if using length files).

### Structural Annotation (BEDPE)
If not using SyRI output, provide a 7-column BEDPE file:
1. Reference chromosome
2. Start
3. End
4. Query chromosome
5. Start
6. End
7. Annotation type (SYN, INV, TRA, INVTR, DUP, INVDP)

## Expert Tips and Best Practices
- **Order Consistency**: The order of genomes in `genomes.txt` must strictly match the reference-query order used in the `--sr` files. If A is reference for B, and B is reference for C, the file must list A, then B, then C.
- **Synteny Requirement**: plotsr requires syntenic (SYN) regions to group and align homologous chromosomes. If your data lacks SYN annotations, you must manually add a full-chromosome SYN entry to the BEDPE file to allow clustering.
- **Performance Optimization**: Use chromosome length files (`.cl`) instead of full FASTA files for the `--genomes` input to significantly reduce memory usage and startup time.
- **Visual Tuning**:
    - Use `-S` to scale the thickness of SR ribbons.
    - Use `--nosyn` to hide syntenic blocks if you only want to highlight rearrangements.
    - Use `--itx` mode for interleaved chromosome arrangements when comparing many genomes.
- **Zooming**: Use `--chr` to plot specific chromosomes or `--reg` to focus on a specific genomic coordinate range.
