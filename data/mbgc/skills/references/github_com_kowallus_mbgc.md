It performs efficiently in terms of compression ratio and speed for various collections but is tailored for bacteria species. The implementation obtained &gt;6.6 GB/s (both compression- and decompression-wise) squeezing ~4.9GB collection of 1k E. coli genomes to ~4.5MB, tested in RAM disk on: Intel Core i9-10940X (14 cores) 3.3 GHz CPU, 128 GB of DDR4-RAM (2666 MHz, CL 16). Major features: supports gzipped (.gz) archives as input and output, preserves folder structure of compressed files, preserves "well-formed" sequence formatting (defined in NAF specification), and offers lossy compression when sequences are malformed, supports standard input and output during (de)compression, enables appending FASTA to existing archives, supports decompression (or repacking to new archive) of selected files, handles FASTA with non-bacteria species and amino acids as well. mbgc is available through bioconda repository . More information can be found in the bioconda page . Manual build The following steps create mbgc executable. mbgc build requires cmake version &gt;= 3.5 installed (check using cmake --version ). git clone https://github.com/kowallus/mbgc.git cd mbgc mkdir build cd build cmake .. make mbgc Basic usage Usage for multiple file compression (list of files given as input): mbgc c [-m &lt;compressionMode&gt;] [-L] &lt;fastaFileList&gt; &lt;archiveFile&gt; Usage for single file compression: mbgc c [-m &lt;compressionMode&gt;] [-L] -i &lt;inputFastaFile&gt; &lt;archiveFile&gt; Usage for decompression: mbgc d [-z &lt;gzLevel&gt;] &lt;archiveFile&gt; [&lt;outputPath&gt;] Usage for partial decompression (list of patterns given as input): mbgc d [-E &lt;patternsFile&gt;] &lt;archiveFile&gt; [&lt;outputPath&gt;] &lt;fastaFileList&gt; name of text file with a list of FASTA files (raw or gz) (given in separate lines) for compression &lt;inputFastaFile&gt; name of a FASTA file (raw or gz) for compression &lt;archiveFile&gt; mbgc archive filename &lt;patternsFile&gt; name of text file with list of patterns (in separate lines) excludes files not matching any pattern (does not invalidate -e option) &lt;outputPath&gt; extraction target path root (current directory by default) Basic options (for compression, decompression and commons): [-m &lt;compressionMode&gt;] (speed: 0; default: 1; repo: 2; max: 3) [-L] allow lossy compression [-z &lt;gzLevel&gt;] extract FASTA files to gz archives (compression level: 1 &lt;= z &lt;= 12, recommended: 3) [-l &lt;basesPerRow&gt;] custom format of decoded sequences (0 - unlimited) [-f] overwrite an existing output file [-e &lt;pattern&gt;] exclude files with names not containing pattern [-E &lt;patternsFile&gt;] exclude files not matching any pattern [-t &lt;noOfThreads&gt;] set limit of used threads [-I] ignore FASTA files paths (use only filenames) [-h] print full command help and exit [-v] print version number and exit Compression modes description: (0) speed - for speed (fastest compression and decompression) (1) default - regular mode (good ratio, fast) (2) repo - for public repositories (better ratio, good speed) (3) max - for long-term storage (best ratio, memory-frugal) compression of FASTA files (raw or gzipped) listed in fastaList.txt file (one FASTA file per line): ./mbgc c fastaList.txt comp.mbgc compression of a single FASTA file (in FASTA or gzipped FASTA format): ./mbgc c -i input.fasta comp.mbgc decompression to out folder (which is created if it does not exist): ./mbgc d comp.mbgc out decompression to gz archives of files containing at least one pattern specified in patterns.txt file (one pattern per line) to out folder (which is created if it does not exist): ./mbgc d -z2 -E patterns.txt comp.mbgc out Please note that since version 2.1 MBGC does not overwrite existing files! To force overwriting use -f option, for example: ./mbgc d -f -comp.mbgc out Exemplary data and scripts demonstrating usages of MBGC in basic compression scenarios are located in example-scripts folder. More test datasets and scripts are available here . Lossless and lossy compression modes Since version 2.1 MBGC ensures lossless compression by default. Lossless compression is supported for multi-FASTA files: with headers starting with the '&gt;' symbol, with a fixed line length for sequence lines within each file (except that the last line of a sequence may be shorter), without empty lines (yet, empty sequences are supported), without comments starting with the ';' symbol. If a FASTA file cannot be compressed losslessly, lossy compression mode can be enabled using -L option. ./mbgc c -L fastaList.txt comp.mbgc When sequences are not "well-formed" (see NAF specification) and line-length cannot be preserved losslessly, MBGC will store the length of the longest sequence line within each file. For lossy single-file compression this may differ, because the file is compressed in blocks and each block stores the length of the longest sequence line within that block. Basic usage with standard I/O Following POSIX convention, a single hyphen character can be used to specify input from or output to the standard input and output streams. for standard input set &lt;inputFastaFile&gt; to - for standard input (resp. output) in compression (resp. decompression) set &lt;archiveFile&gt; to - for standard output set &lt;outputPath&gt; to - (all files are concatenated) compression of FASTA in standard input data stream (in raw or gzipped FASTA format): ./mbgc c -i - comp.mbgc decompression to standard output without EOLs symbols within sequences: ./mbgc d -l0 comp.mbgc - Usage with other commands MBGC offers following commands: Available commands (i - default): c compress FASTA file(/s) into archive d decompress FASTA file(/s) from archive i info about contents (FASTA file names &amp; headers) of archive a append FASTA file(/s) to the given archive r repack selected FASTA files from existing archive to new archive listing filenames in given archive: ./mbgc i comp.mbgc or using default command syntax: ./mbgc comp.mbgc listing headers (using convention: "&gt;sequencename&gt;filename") in filenames containing ASM17 pattern: ./mbgc i -e ASM17 comp.mbgc appending FASTA files (raw or gzipped) listed in fastaList.txt file (one FASTA file per line) to archive: ./mbgc a fastalist.txt comp.mbgc Please note that appending ignores FASTA with file names already existing in the archive. repacking archive (in current or older mbgc version) to max compression mode: ./mbgc r -m3
