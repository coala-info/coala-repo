# bioconductor-hgc

## Overview

HGC (Hierarchical Graph-based Clustering) is an R package designed for hierarchical clustering of large-scale single-cell RNA-seq (scRNA-seq) data. Unlike traditional hierarchical clustering which can be computationally expensive, HGC builds a dendrogram based on a shared nearest neighbor (SNN) graph, offering linear time complexity and high scalability. It produces standard `hclust` objects that can be used with existing R visualization and analysis tools.

## Core Workflow

### 1. Data Preparation
HGC requires a matrix where rows are cells and columns are features (typically PCs).
```r
library(HGC)
data(Pollen)
Pollen.PCs <- Pollen[["PCs"]]
```

### 2. Graph Construction
Build a graph (KNN, SNN, or CKNN) from the reduced dimensionality matrix.
```r
# SNN Construction
Pollen.SNN <- SNN.Construction(mat = Pollen.PCs, k = 25, threshold = 0.15)
```

### 3. Dendrogram Construction
Build the hierarchical tree directly from the graph.
```r
Pollen.ClusteringTree <- HGC.dendrogram(G = Pollen.SNN)
```

### 4. Clustering and Evaluation
Cut the tree into clusters and evaluate using Adjusted Rand Index (ARI) if labels are known.
```r
# Cut tree
clusters <- cutree(Pollen.ClusteringTree, k = 5)

# Evaluate ARI
Pollen.labels <- data.frame(Tissue = Pollen[["Tissue"]])
ARI.mat <- HGC.PlotARIs(tree = Pollen.ClusteringTree, labels = Pollen.labels)
```

## Pipeline Integration

### Seurat Integration
HGC can use graphs stored within a Seurat object.
```r
library(Seurat)
# Assuming pbmc is a Seurat object with FindNeighbors already run
pbmc <- FindClusteringTree(pbmc, graph.type = "SNN")
pbmc.tree <- pbmc@graphs$ClusteringTree
```

### scran/igraph Integration
HGC can process `igraph` objects generated by `scran`.
```r
library(scran)
# Assuming sce is a SingleCellExperiment object
g <- buildSNNGraph(sce, use.dimred="PCA")
cluster.tree <- HGC.dendrogram(G = g)
```

## Visualization

HGC provides built-in functions for dendrogram visualization, often benefiting from log-transformation of heights for better clarity.
```r
# Log-transform for better visualization
Pollen.ClusteringTree$height = log(log(Pollen.ClusteringTree$height + 1) + 1)

# Plot with labels
HGC.PlotDendrogram(tree = Pollen.ClusteringTree, k = 5, plot.label = TRUE, labels = Pollen.labels)

# Heatmap integration
library(pheatmap)
pheatmap(mat = Pollen.PCs, cluster_rows = Pollen.ClusteringTree, cluster_cols = FALSE)
```

## Tips and Best Practices
- **Preprocessing**: Always perform normalization and dimension reduction (e.g., PCA) before graph construction to ensure the graph captures the underlying manifold.
- **Graph Types**: While SNN is the default, HGC supports `KNN.Construction` and `CKNN.Construction`.
- **Scalability**: For very large datasets, HGC is significantly faster than standard `hclust` because it operates on the sparse adjacency matrix of the cell graph.
- **Custom Graphs**: Any graph stored as a `dgCMatrix` (Matrix package) can be passed to `HGC.dendrogram`.
