# capc-map

## Overview

Capture-C is a specialized chromosome conformation capture (3C) variant used to study high-resolution interactions for specific target genomic loci. The `capc-map` suite automates the complex process of trimming adapters, performing in silico digestion, aligning fragments, and generating interaction pile-ups. This skill enables the efficient execution of the pipeline, from initial genome preparation to the generation of binned and normalized interaction profiles suitable for visualization.

## Core Workflows

### 1. Genome Preparation
Before running the pipeline, you must generate a restriction fragment map and a Bowtie index for your reference genome.

*   **Generate Fragment Map**: Use `genomedigest` to create a BED file of all restriction sites.
    ```bash
    capC-MAP genomedigest -f reference.fasta -r DpnII -o genome_fragments.bed
    ```
*   **Build Bowtie Index**: `capc-map` requires Bowtie 1 (not Bowtie 2).
    ```bash
    bowtie-build reference.fasta reference_index
    ```

### 2. Target Fragment Identification
If you have genomic coordinates for your capture oligos but need the specific restriction fragments they map to:
```bash
capC-MAP capClocation2fragment genome_fragments.bed targets_locations.bed > targets.bed
```
*Note: Ensure the `targets.bed` file contains unique entries and descriptive names in the 4th column.*

### 3. Running the Pipeline
The `run` command executes the full pipeline using a configuration file.
```bash
capC-MAP run -c config.txt -o output_directory
```

**Configuration File Best Practices:**
The configuration file is a space-separated text file. Key parameters include:
- `FASTQ1` / `FASTQ2`: Paths to paired-end reads.
- `INDEX`: Path to the Bowtie 1 index.
- `RESTFRAGS`: The BED file generated by `genomedigest`.
- `ENZYME`: The enzyme name (e.g., `DpnII`) or recognition sequence.
- `BIN S W`: Set `S` for step size and `W` for window size (e.g., `BIN 200 1000` for 200bp bins with 1kb smoothing).
- `PARALLEL N`: Set `N` to the number of CPU cores to speed up alignment and sorting.

### 4. Post-processing and Normalization
To generate new binned profiles or normalize to Reads Per Million (RPM) without re-running the alignment:
```bash
capC-MAP postprocess -c config.txt -o output_directory
```
Ensure `NORMALIZE TRUE` is set in the config file to enable RPM scaling.

### 5. Combining Replicates
To merge data from multiple biological replicates:
```bash
capC-MAP combinereps -c combine_config.txt -o combined_output
```

## Expert Tips

*   **Alignment Mode**: Use `ALIGNMODE CONSERVATIVE` (default) to ensure high-quality mappings. If mapping rates are unexpectedly low, `RELAXED` can be used, but may increase noise.
*   **Exclusion Zones**: By default, fragments within 1000bp of a target are excluded (`EXCLUDE 1000`) to remove self-ligation and un-cleaved products. Adjust this if working with extremely high-resolution data or very small restriction fragments.
*   **Interchromosomal Data**: While `capc-map` always saves interchromosomal pairs in `validinterchrom` files, it does not generate pile-ups for them by default. Set `INTERCHROM TRUE` in the config if these profiles are required.
*   **Disk Space Management**: After a successful run, the in silico digested FastQ and intermediate SAM files can be deleted; the `validpairs` and `bedGraph` files are the primary outputs for downstream analysis.
