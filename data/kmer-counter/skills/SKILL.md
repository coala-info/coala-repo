# kmer-counter

## Overview

The `kmer-counter` tool is a Rust-based utility optimized for speed and minimal memory footprint. It transforms nucleotide sequences into a fixed-size feature matrix stored as a NumPy `.npy` file. Unlike tools designed for k-mer discovery or assembly, this counter is specifically built for feature extraction, where every possible k-mer of a specified length is assigned a deterministic index in the output array, facilitating direct loading into TensorFlow or NumPy.

## Usage Instructions

### Basic Command Pattern
To generate k-mer counts for a sequencing file, use the following structure:
```bash
kmer-counter --file <input.fasta> --ids <ids.txt> --klength <int> --out <output.npy> --collapse <0|1>
```

### Parameter Guidance
- `--file`: Path to the input nucleotide sequences. Supports both plain FASTA and gzipped (`.gz`) files.
- `--klength`: The length of the k-mers to count. The default is 4. Note that memory usage scales exponentially with this value ($4^k$).
- `--collapse`: 
    - `1` (Default): Canonicalizes k-mers, meaning a k-mer and its reverse complement are counted together under the same key.
    - `0`: Counts k-mers and their reverse complements separately.
- `--ids`: Generates a plain text file containing the sequence identifiers in the same order as the rows in the output matrix.
- `--out`: The path for the resulting `.npy` file containing the 32-bit integer count matrix.

### Interpreting the Output
The output matrix rows correspond to the sequence records in the input file. The columns correspond to the k-mer keys generated by the Cartesian product of "ACGT" in alphabetical order.

To reconstruct the k-mer keys in Python for a given `k`:
```python
from itertools import product
k = 4
keys = ["".join(i) for i in product("ACGT", repeat=k)]
```

## Expert Tips and Best Practices

- **Memory Efficiency**: The tool is designed to handle hundreds of millions of reads with very low peak memory (e.g., ~7MB for $k=4$). Memory requirements are determined by the k-mer space ($4^k$), not the number of sequences.
- **Dense vs. Sparse Data**: This tool is not recommended for very large k-mer sizes (e.g., $k > 12$) where the resulting matrix would be extremely sparse. It allocates a slot for every possible k-mer regardless of whether it is observed.
- **Downstream Integration**: Since the output is a standard `.npy` file, use `numpy.load('output.npy')` in Python to immediately begin analysis. The row order is guaranteed to match the input FASTA and the generated IDs file.
- **Canonicalization**: Always use `--collapse 1` unless your specific biological question requires distinguishing between a sequence and its reverse complement (e.g., strand-specific analysis).
