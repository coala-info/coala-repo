# 10x_bamtofastq

## Overview

The `10x_bamtofastq` tool is a specialized utility designed to reconstruct original FASTQ sequences from the BAM files generated by 10x Genomics pipelines. It extracts sequences and quality scores from BAM tags (like RX, QX, TR, and TQ) to recreate the R1, R2, and index reads. The resulting directory structure is compatible with 10x Genomics' `mkfastq` format, allowing the output to be used directly as input for re-running pipelines like `cellranger count`.

## Installation

Install the tool via Bioconda for the most stable environment:

```bash
conda install -c bioconda 10x_bamtofastq
```

## Basic Usage

To convert a BAM file to FASTQ, provide the path to the BAM file and a destination directory:

```bash
bamtofastq /path/to/input_alignments.bam /path/to/output_directory
```

## Common CLI Patterns

### Processing Specific Genomic Regions
If you only need to re-analyze a specific locus (e.g., for troubleshooting or targeted assembly), use the `--locus` flag. This significantly reduces processing time and output size.

```bash
bamtofastq --locus=chr1:1000000-2000000 input.bam ./output_dir
```

### Handling Legacy 10x Pipelines
BAM files produced by modern pipelines (Cell Ranger 1.2+ or Long Ranger 2.1+) contain header metadata that allows `bamtofastq` to work automatically. For older data, you must specify the source pipeline:

- **Cell Ranger 1.0 - 1.1**: Use `--cr11`
- **Long Ranger 2.0**: Use `--lr20`
- **GemCode (Long Ranger 1.0 - 1.3)**: Use `--gemcode`

### Filtering by Barcodes
To extract reads belonging to specific cell barcodes (BX tags), provide a text file containing the barcodes of interest. Note: This requires the BAM file to be BX-sorted and indexed.

```bash
bamtofastq --bx-list=barcodes.txt input.bam ./output_dir
```

### Adjusting Output Chunk Size
By default, the tool creates FASTQ chunks of 50 million reads. You can adjust this based on your filesystem's performance or downstream requirements:

```bash
bamtofastq --reads-per-fastq=10000000 input.bam ./output_dir
```

## Expert Tips and Best Practices

- **SRA Data Warning**: When downloading 10x data from the Sequence Read Archive (SRA), ensure you download the "Original format" BAM (Type = TenX). Standard SRA-processed BAMs often strip the tags required to reconstruct the original FASTQs.
- **Read Order**: While the tool recovers the exact sequences and quality scores, the original order of reads in the FASTQ is not preserved. This does not affect downstream 10x Genomics pipelines.
- **Disk Space**: Ensure the output partition has significantly more space than the BAM file, as the uncompressed or GZ-compressed FASTQs can be larger than the coordinate-sorted BAM.
- **Performance**: The tool is written in Rust and is highly efficient. If building from source, always use `cargo build --release` to ensure optimal conversion speed.
