# kat

## Overview

The K-mer Analysis Toolkit (KAT) is a suite of tools designed to characterize and compare genomic datasets through k-mer frequencies. It is primarily used for reference-free quality control of sequencing reads and the validation of de novo assemblies. By analyzing k-mer spectra, KAT helps identify issues like contamination, sequencing bias, and assembly errors (e.g., collapsed repeats or missing content) without requiring a gold-standard reference genome.

## Core Commands and Usage

KAT follows a standard CLI pattern: `kat <tool> <options>`.

### Generating K-mer Histograms
Use `hist` to create a distribution of k-mer occurrences. This is the first step for estimating genome size and heterozygosity.
- **Basic usage**: `kat hist -t <threads> -m <kmer_size> -o <output_prefix> <input.fastq>`
- **Tip**: Use `-m 21` as a starting point for most eukaryotic genomes.

### Comparing Datasets
Use `comp` to find shared k-mers between two or three sequence files or Jellyfish hashes. This is essential for comparing reads to assemblies or checking consistency between different libraries.
- **Basic usage**: `kat comp -t <threads> -o <output_prefix> <reads.fastq> <assembly.fasta>`
- **Output**: Generates a matrix file used for downstream plotting (e.g., `spectra-cn`).

### Estimating Sequence Coverage
Use `sect` to estimate the coverage of sequences in one file using k-mers derived from another.
- **Basic usage**: `kat sect -t <threads> -o <output_prefix> <reference.fasta> <reads.fastq>`

### Filtering Sequences
Use `filter` to clean datasets based on k-mer properties.
- **Filter hash**: `kat filter kmer -o <output_hash> --min-count <min> --max-count <max> <input_hash>`
- **Filter sequences**: `kat filter seq -o <output_file> -k <hash_file> <input.fastq>`

## Plotting and Visualization

KAT includes several plotting tools to visualize the matrices and histograms generated by the core tools. Note that these require a functional Python environment with `matplotlib`.

- **Spectra Copy Number (spectra-cn)**: Compare a read set hash to an assembly hash to visualize missing content and copy number variation.
  - `kat plot spectra-cn -o <output.png> <matrix_file>`
- **Density Plots**: Visualize the relationship between two k-mer hashes.
  - `kat plot density -o <output.png> <matrix_file>`
- **K-mer GC Processor (gcp)**: Analyze GC content relative to k-mer frequency to identify contamination or bias.
  - `kat gcp -t <threads> -o <output_prefix> <input.fastq>`
  - `kat plot density -o <output_gc.png> <gcp_matrix_file>`

## Expert Tips

- **Pre-compute Hashes**: For large datasets or iterative analyses, use Jellyfish to create `.jf` hashes first. KAT can consume these directly, saving significant processing time compared to re-counting k-mers from raw FASTQ files.
- **Memory Management**: K-mer counting is memory-intensive. Monitor RAM usage, especially when using large k-mer sizes (k > 31) or processing high-coverage datasets.
- **Assembly Validation**: When using `spectra-cn`, look for the "black" peak (k-mers present in reads but absent in the assembly). A large black peak at the expected homozygous coverage indicates significant missing genomic content.
- **Thread Scaling**: Most KAT tools scale well with threads (`-t`). Match the thread count to your available CPU cores to expedite the counting and matrix generation phases.
