**Where can I get help? Is there a mailing list?**

* `emirge.py --help`
* The **EMIRGE users** google group is located here: <https://groups.google.com/group/emirge-users>

---

**Does EMIRGE work with 454 data?**
No, not currently. EMIRGE has not been tested with 454 reads, mainly because 454 homopolymer errors introduce indels and EMIRGE currently uses a mapper (the original bowtie) which doesn't deal with indels. Also, the parameters of EMIRGE are tuned for Illumina-length reads. There has been some work done towards incorporating indels into the EMIRGE algorithm, but this is still a work in progress.

If you do try EMIRGE with 454 data, please let me know how it works for you. You may also be interested in these publications, which describe assembly of 16S genes from 454 data:
[Metatranscriptomics of the marine sponge Geodia barretti: tackling phylogeny and function of its microbial community.](http://www.ncbi.nlm.nih.gov/pubmed/22364353)
Environ Microbiol. 2012 May;14(5):1308-24
Radax R, Rattei T, Lanzen A, Bayer C, Rapp HT, Urich T, Schleper C.

[Reconstruction of Ribosomal RNA Genes from Metagenomic Data]
(<http://www.ncbi.nlm.nih.gov/pubmed/22761935>)
PLoS One. 2012;7(6):e39948. Epub 2012 Jun 27.
Fan L, McElroy K, Thomas T.

---

**Does EMIRGE work with single-end data?**
Yes. Simply omit the -2 parameter on the command line, providing only a single fastq file. EMIRGE was designed for and tested on paired-end sequencing reads. Although I have done some basic testing on single-end reads, runs with single reads have **NOT** been as extensively tested as runs with paired reads. Please let me know how it works for you if you try EMIRGE with single-end reads.

---

**Does EMIRGE work with transcriptome or amplicon data?**
Yes, but with the caveat that the standard emirge.py version may take 23 years to run. EMIRGE was developed on metagenomic data, where the fraction of reads coming from rRNA genes is small. If you have data where most of the sequencing effort is expected to be from rRNA genes (e.g. total RNA-Seq, rRNA PCR amplicons), then this version of EMIRGE might be too slow. A version of EMIRGE that will work with this scale of data is now available as emirge\_amplicon.py, and a paper describing its use has been submitted. In practice, up to a few million rRNA reads at a time (from a multiplexed lane of Illumina sequencing) have been analyzed with emirge\_amplicon.py. Note that emirge\_amplicon.py loads all reads in memory to speed calculation time, and thus requires a few GB of RAM, depending on how many reads you input. Eventually, the two versions of the code will be merged together again, so a user will not have to choose which version to run.

---

**Does EMIRGE work with SOLiD / colorspace data?**
Not directly. At least one user has converted SOLiD data to standard FASTQ and run EMIRGE with encouraging results. If you try this and it works for you, please let me know.

---

**Is EMIRGE described in a publication? How do I cite EMIRGE?**
Miller, C.S., B. J. Baker, B. C. Thomas, S. W. Singer and J. F. Banfield (2011). "EMIRGE: reconstruction of full-length ribosomal genes from microbial community short read sequencing data." Genome Biology 12(5): R44.
[Genome Biology full text (Open access)](http://genomebiology.com/2011/12/5/R44)
[PubMed](http://www.ncbi.nlm.nih.gov/pubmed/21595876)

A manuscript describing the application of EMIRGE to rRNA amplicon sequencing has been submitted.
