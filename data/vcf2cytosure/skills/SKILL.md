# vcf2cytosure

## Overview

The `vcf2cytosure` tool bridges the gap between standard bioinformatics variant calling and commercial cytogenetic analysis software. It transforms structural variant (SV) data into an emulated probe-based XML format (.CGH) that CytoSure uses to display oligo array measurements. This allows users to leverage the visualization and aberration-calling features of CytoSure for data derived from Next-Generation Sequencing (NGS) rather than traditional microarrays.

## Core Usage Patterns

### Basic Conversion
To convert a standard VCF containing structural variants:
```bash
vcf2cytosure --vcf input.vcf --out output.cgh
```

### Visualizing Coverage and Copy Number
To display actual coverage ratios instead of fixed-height markers, provide a coverage BED file (typically generated by TIDDIT):
```bash
vcf2cytosure --vcf input.vcf --coverage coverage.bed --sex male --out output.cgh
```
*   **Note**: When using `--coverage`, probe heights represent log2 ratios. Probes are clamped at +/- 4.

### Integrating SNV Data
To include Single Nucleotide Variant data for B-allele frequency visualization:
```bash
vcf2cytosure --vcf input.vcf --snv snv.vcf --out output.cgh
```

## Expert Tips and Best Practices

### Managing Resolution and Performance
CytoSure can become unresponsive if the number of probes is too high. Use the `--bins` parameter to pool coverage data:
```bash
vcf2cytosure --vcf input.vcf --coverage coverage.bed --bins 50 --out output.cgh
```
*   A higher bin value (e.g., 50) pools more coverage points into a single probe, reducing file size and improving software performance at the cost of resolution.

### Structural Variant Representation
The tool handles SV types with specific visual logic:
*   **DEL (Deletions)**: Probes at height -1.
*   **DUP (Duplications)**: Probes at height +1.
*   **TDUP (Tandem Duplications)**: Probes at height +1.5.
*   **Large SVs**: For variants spanning large regions, the tool generates probes spaced 100 kbp apart to ensure the variant is visible across the genome browser.

### Metadata Mapping
`vcf2cytosure` maps VCF INFO fields to CytoSure attributes:
*   **Confirmation Method**: Populated with the SV type (e.g., INV, INS).
*   **# Probes**: Populated from the `OCC` (occurrence) INFO field if present.
*   **CopyNumber**: Populated from the `RankScore` INFO field.
*   **Comments**: Any INFO attributes not explicitly mapped are concatenated into the CytoSure comment field.

### Genome Build Support
The tool includes specific handling for hg38 N-intervals. Ensure your input coordinates match the intended genome assembly. If your VCF uses "chr" prefixes, recent versions (v0.9.2+) automatically handle their removal for compatibility.

### Troubleshooting Output
The resulting `.cgh` file is XML. If you need to inspect the raw data, use `xmllint` to make it human-readable:
```bash
xmllint --format output.cgh > pretty.xml
```
