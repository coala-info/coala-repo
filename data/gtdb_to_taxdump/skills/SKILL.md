# gtdb_to_taxdump

## Overview
The `gtdb_to_taxdump` toolset bridges the gap between the flat TSV taxonomy files provided by GTDB and the hierarchical `taxdump` format used by many legacy and modern bioinformatics classifiers. While GTDB provides a more accurate bacterial and archaeal phylogeny, many tools are hardcoded to expect the NCBI format. This tool generates the required `nodes.dmp` and `names.dmp` files. 

**Important Note:** The TaxIDs generated by this tool are arbitrarily assigned and are **not stable** between different GTDB releases or different runs on different subsets of data. If your workflow requires persistent TaxIDs, consider using the `GTDB-taxdump` project instead.

## Common CLI Patterns

### Basic Taxonomy Conversion
To generate a taxdump from standard GTDB release files (Archaea and Bacteria), provide the TSV files as positional arguments. The tool supports remote URLs and gzipped files.

```bash
gtdb_to_taxdump.py \
  https://data.gtdb.ecogenomic.org/releases/release202/202.0/ar122_taxonomy_r202.tsv.gz \
  https://data.gtdb.ecogenomic.org/releases/release202/202.0/bac120_taxonomy_r202.tsv.gz \
  > taxID_info.tsv
```

### Integrating TaxIDs into Metadata
If you need to append the newly generated TaxIDs directly to a GTDB metadata table for downstream analysis, use the `--table` parameter.

```bash
gtdb_to_taxdump.py \
  --table ar122_metadata_r202.tsv \
  ar122_taxonomy_r202.tsv.gz \
  bac120_taxonomy_r202.tsv.gz \
  > taxID_info.tsv
```

### Mapping NCBI to GTDB (and vice versa)
Use `ncbi-gtdb_map.py` to translate classifications between the two systems. This is useful when comparing results from different databases.

```bash
# Map GTDB classifications to NCBI
ncbi-gtdb_map.py --query-taxonomy GTDB --table gtdb_metadata.tsv > mapping_results.tsv
```

### Preparing Diamond Databases
To create a Diamond-compatible database using GTDB taxonomy for protein-level classification:

```bash
gtdb_to_diamond.py --out gtdb_diamond_db [options]
```

### Generating Accession-to-Taxid Maps
For tools like Kraken2 that require a mapping of sequence accessions to TaxIDs:

```bash
acc2gtdb_tax.py --taxid-map taxID_info.tsv --out accession2taxid.tsv
```

## Expert Tips
- **TaxID Stability:** Always run the conversion on the full set of taxa you plan to use in a project simultaneously. If you run the script on Archaea and then later on Bacteria separately, the TaxIDs will conflict.
- **Fuzzy Mapping:** When using `ncbi-gtdb_map.py`, you can enable "fuzzy" mapping for cases where a 1-to-1 taxonomic match does not exist between NCBI and GTDB.
- **Lineage Resolution:** If you have a list of GTDB lineages (e.g., from GTDB-Tk) and need their corresponding TaxIDs from a previous `gtdb_to_taxdump` run, use `lineage2taxid.py`.
- **Dependencies:** Ensure `taxonkit` is in your PATH, as it is a core dependency for the formatting process.
