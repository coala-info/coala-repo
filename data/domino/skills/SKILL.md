# domino

## Overview
DOMINO (Discovery of Modules In Networks using Omic) is a specialized algorithm for identifying active modules in biological networks. Unlike many other AMI algorithms that tend to over-report enrichment, DOMINO is designed to ensure that identified modules have a high rate of empirically validated GO terms. The tool operates in two primary phases: network partitioning (slicing) and module discovery. It requires a gene network in SIF format and a list of active genes (typically Ensembl IDs) as input.

## Installation and Environment
DOMINO is best run in a Python 3.8 environment. It can be installed via Bioconda or pip:

```bash
# Via Conda
conda install bioconda::domino

# Via Pip
pip install domino-python
```

## Core Workflow

### 1. Network Partitioning (Slicer)
Before running the main algorithm, you must partition the network using the Louvain algorithm. This generates a "slices" file.

```bash
slicer --network_file <network.sif> --output_file <slices.txt>
```
*   **Input**: A simplified SIF file. The first row must be headers, and the first and last columns should contain single node IDs.
*   **Output**: A slices file used in the next step.

### 2. Module Discovery (Domino)
Run the discovery algorithm using the partitioned network and your active gene sets.

```bash
domino --active_genes_files <genes.txt> \
       --network_file <network.sif> \
       --slices_file <slices.txt> \
       --output_folder <output_dir> \
       --visualization true
```

## CLI Parameters and Best Practices

### Essential Arguments
*   `-a, --active_genes_files`: A comma-separated list of paths to files containing active genes. Genes must be in Ensembl format, one per line.
*   `-n, --network_file`: Path to the network SIF file.
*   `-s, --slices_file`: Path to the output generated by the `slicer` command.
*   `-o, --output_folder`: Directory where results will be stored.

### Optimization and Tuning
*   **Caching**: Use `-c true` or `--use_cache true` to use auto-generated `.pkl` cache files from previous runs on the same network. This significantly speeds up repeated executions.
*   **Parallelization**: Use `-p <threads>` to allocate multiple threads, though a single thread is often sufficient for standard datasets.
*   **Thresholds**:
    *   `-sth, --slices_threshold`: Adjust the relevance threshold for considering a slice.
    *   `-mth, --module_threshold`: Adjust the threshold for a putative module to be considered a final module.
*   **Visualization**: Always include `-v true` to generate interactive HTML visualizations (`module_i.html`) for each discovered module.

## Output Interpretation
The tool generates a sub-directory for each active gene file provided:
*   `modules.out`: A text file listing the final discovered modules and their constituent genes.
*   `module_i.html`: Interactive visualizations of the $i$-th module, allowing for manual inspection of network topology.
