# mnaseseq

## Overview
nf-core/mnaseseq is a bioinformatics analysis pipeline designed for DNA sequencing data obtained via micrococcal nuclease (MNase) digestion. It automates the transition from raw sequencing reads to high-level analysis of nucleosome positioning and occupancy across the genome.

The pipeline handles quality control, adapter trimming, and alignment to a reference genome. It produces normalized coverage files, nucleosome calls, and meta-profiles to help researchers visualize and quantify chromatin structure across different samples and replicates.

## Data preparation
The pipeline requires a design CSV file and a reference genome to function. It is designed to handle multiple libraries per sample and multiple replicates, merging them at appropriate stages of the analysis.

- **Input CSV**: Specified with `--input`, this file maps sample identifiers to their corresponding FASTQ files.
- **Reference Genome**: Specified with `--genome` (e.g., `GRCh37`), which provides the necessary FASTA and index files for alignment.
- **Blacklist Regions**: The pipeline can filter reads mapping to specific blacklisted regions if provided.

A minimal execution requires a design file structured for the pipeline's requirements:
```csv
sample,fastq_1,fastq_2
sample1,s1_R1.fastq.gz,s1_R2.fastq.gz
```

## How to run
Run the pipeline using the `nextflow run` command. It is recommended to specify a container profile (such as `docker` or `singularity`) and a specific pipeline release with `-r`.

```bash
nextflow run nf-core/mnaseseq \
    -profile docker \
    --input design.csv \
    --genome GRCh37 \
    --outdir ./results
```

- Use `-resume` to restart a pipeline run from the last successful process.
- Use `-profile test` to run a minimal dataset to verify the installation and environment setup.
- Pipeline-specific parameters use double dashes (e.g., `--input`), while Nextflow runtime parameters use a single dash (e.g., `-profile`).

## Outputs
All results are saved to the directory specified by the `--outdir` parameter.

- **MultiQC**: The `multiqc_report.html` file summarizes raw read quality control and alignment statistics.
- **Nucleosome Calls**: Positions and occupancy data generated by DANPOS2, including smoothed and normalized coverage bigWig files.
- **Coverage Tracks**: BigWig files scaled to 1 million mapped reads for genome-wide coverage assessment.
- **Visualization**: An IGV session file containing bigWig tracks for direct data visualization.
- **Meta-profiles**: Gene-body meta-profiles generated from smoothed bigWig files using deepTools.

For a complete list of output files and their interpretation, refer to the `docs/output.md` documentation.
