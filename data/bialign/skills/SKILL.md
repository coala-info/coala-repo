# bialign

## Overview
`bialign` is a specialized bioinformatics tool designed to handle "bi-alignments" of RNAs and proteins. Unlike traditional alignment methods that assume sequence and structure evolve in strict synchronization, `bialign` models evolutionary "shifts." This allows the tool to align molecules where structural elements have moved relative to the primary sequence. It is particularly effective for identifying structural homologs in cases of incongruent evolution, supporting affine gap costs and integration with structure prediction tools.

## Installation
The tool is best installed via Conda to ensure all dependencies (like the Vienna RNA package for RNA structure prediction) are correctly configured:

```bash
conda install -c bioconda bialign
```

## Command Line Usage
The primary interface is the `bialign.py` script.

### RNA Bi-alignment
When aligning RNA, you can provide sequences and their corresponding dot-bracket structures. If structures are not provided, the tool will attempt to predict them using the Vienna RNA package.

```bash
bialign.py <sequence_A> <sequence_B> \
  --strA "<structure_A>" \
  --strB "<structure_B>" \
  --max_shift 1 \
  --shift_cost -150 \
  --gap_opening_cost -200 \
  --gap_cost -50
```

### Protein Bi-alignment
For proteins, specify the type and an appropriate similarity matrix (e.g., BLOSUM62).

```bash
bialign.py <sequence_A> <sequence_B> \
  --strA "<secondary_structure_A>" \
  --strB "<secondary_structure_B>" \
  --type Protein \
  --simmatrix BLOSUM62 \
  --structure_weight 800 \
  --outmode sorted
```

### File-Based Input
You can read input directly from files, such as those generated by secondary structure prediction servers (e.g., CFSSP for proteins):

```bash
bialign.py --filein protein_A.cfssp protein_B.cfssp --type Protein --max_shift 1
```

## Expert Tips and Best Practices
- **Shift Configuration**: The core of the bialignment model is controlled by `--max_shift` and `--shift_cost`. A higher `max_shift` allows for larger structural displacements but increases computational complexity.
- **Output Visualization**: Use `--outmode sorted` to generate a BLAST-like text output. This view is highly useful for human inspection as it displays the consensus sequence and structure alongside the alignment and shift indicators.
- **Gap Penalties**: Always use affine gap costs (`--gap_opening_cost` and `--gap_cost`) for biological sequences to better model insertions and deletions.
- **Structure Weighting**: Use `--structure_weight` to balance the influence of structural similarity versus sequence similarity in the final alignment score.
- **Environment**: Note that full functionality (specifically RNA structure prediction via Vienna RNA) is supported on Linux and macOS, but may be limited on Windows.
