# bioconductor-top

## Overview

The `TOP` package implements a transfer learning framework designed to build predictive models that remain effective across different omics datasets. It addresses the challenge of batch effects and inter-dataset variation by leveraging information from multiple sources simultaneously. Using lasso regression, `TOP` identifies common biomarkers and constructs transferable models, particularly for survival analysis and regression tasks.

## Core Workflow

### 1. Data Preparation
To use `TOP`, you must ensure that all input datasets share the same feature names (e.g., Gene Symbols).

```r
# Identify common features across a list of expression matrices
raw_gene_list <- lapply(list_of_matrices, rownames)
common_genes <- Reduce(intersect, raw_gene_list)

# Subset matrices to common features
list_exprs <- lapply(list_of_matrices, function(x) x[common_genes, ])
```

### 2. Building a Survival Model
The primary function for survival analysis is `TOP_survival`. It requires a list of feature matrices (samples as rows) and a list of survival objects.

```r
library(TOP)
library(survival)

# x_list: List of matrices (Samples x Features)
# y_list: List of data frames with 'time' and 'event' (or Surv objects)
surv_model <- TOP_survival(
  x_list = list_of_matrices, 
  y_list = list_of_surv_data, 
  nFeatures = 10
)
```

### 3. Evaluating Performance
Use `Surv_TOP_CI` to calculate the concordance index (C-index) for the model on new or existing data.

```r
# Calculate C-index for a specific dataset
results <- Surv_TOP_CI(
  surv_model,
  newx = test_matrix, 
  newy = test_surv_data
)

print(results$concordance)
```

## Key Functions

- `TOP_survival()`: Fits a transferable survival model using lasso penalty across multiple datasets.
- `Surv_TOP_CI()`: Computes the concordance index for survival models generated by TOP.
- `TOP_regression()`: (Analogous to survival) Fits transferable linear regression models.

## Implementation Tips

- **Feature Alignment**: Always perform the intersection of row names across all training datasets before starting the modeling process.
- **Data Scaling**: While `TOP` handles variation, ensuring datasets are on similar scales (e.g., log2 transformation for expression data) is recommended.
- **Feature Selection**: The `nFeatures` parameter is a key hyperparameter; use cross-validation or domain knowledge to determine the optimal number of biomarkers to include.
- **Input Format**: Ensure that the `x_list` matrices have samples as rows and features as columns when passing them to the modeling functions.
