## Overview
This skill is designed to process the output files generated by Roary, a tool for pangenome analysis, and transform them into a format compatible with FRIpan, a tool for core genome analysis. It's specifically useful when you have pan-genome data from Roary and need to prepare it for downstream core genome studies using FRIpan.

## Usage Instructions

The `roary2fripan.py` script is a command-line tool. The primary function is to convert Roary's output, typically the `gene_presence_absence.csv` file, into a format that FRIpan can ingest.

### Core Functionality

The script's main purpose is to take the Roary gene presence/absence matrix and filter it to include only genes that are present in a user-defined minimum number of isolates. This filtered set of genes is then considered the "core genome" for FRIpan.

### Command Line Usage

The general syntax for using `roary2fripan.py` is as follows:

```bash
roary2fripan.py -p <path_to_gene_presence_absence.csv> -o <output_file_prefix> -c <min_isolates_for_core_genome>
```

**Key Arguments:**

*   `-p` or `--presence_absence`: **Required**. Path to the Roary `gene_presence_absence.csv` file. This file contains the gene presence/absence matrix for your isolates.
*   `-o` or `--output_prefix`: **Required**. A prefix for the output files. The script will generate files like `<output_file_prefix>_core_genome.tsv` and `<output_file_prefix>_accessory_genome.tsv`.
*   `-c` or `--core_threshold`: **Required**. An integer specifying the minimum number of isolates a gene must be present in to be considered part of the core genome. For example, if you have 10 isolates and set this to `10`, only genes present in all 10 isolates will be included in the core genome output. If set to `9`, genes present in 9 or more isolates will be considered core.

### Example Workflow

Let's say you have a Roary output file named `my_roary_output/gene_presence_absence.csv` and you want to identify genes present in at least 95% of your 20 isolates as the core genome. You would run the script like this:

1.  **Determine the core threshold:** If you have 20 isolates and want genes present in 95%, the threshold is `20 * 0.95 = 19`.
2.  **Execute the command:**
    ```bash
    roary2fripan.py -p my_roary_output/gene_presence_absence.csv -o my_fripan_input -c 19
    ```

This command will generate two files:
*   `my_fripan_input_core_genome.tsv`: Contains the list of genes identified as core genome, formatted for FRIpan.
*   `my_fripan_input_accessory_genome.tsv`: Contains the list of genes identified as accessory genome.

### Expert Tips

*   **Understand your core genome definition:** The `-c` (core\_threshold) parameter is crucial. Carefully consider what constitutes your "core genome" based on your biological question and the number of isolates in your dataset. A common approach is to define it as genes present in >95% of isolates.
*   **File Naming:** Use a descriptive output prefix (`-o`) to easily identify the generated files, especially if you are running this for multiple datasets or with different core genome definitions.
*   **Input File Format:** Ensure your input file is indeed the `gene_presence_absence.csv` generated by Roary. The script expects this specific format, including the header row with isolate names.
