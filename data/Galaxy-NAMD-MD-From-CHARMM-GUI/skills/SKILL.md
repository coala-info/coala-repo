# namd-md-from-charmm-gui

## Overview

This Galaxy workflow facilitates molecular dynamics (MD) simulations using the NAMD engine, specifically designed to process outputs generated by the [CHARMM-GUI](https://www.charmm-gui.org/) interface. It streamlines the transition from initial system preparation to high-performance simulation execution within a standardized computational environment.

The workflow requires four primary inputs typically provided by CHARMM-GUI: the PDB structure file, the XPLOR-formatted PSF topology file, the FFT configuration string, and the water box parameter file. These components define the molecular system's coordinates, connectivity, and periodic boundary conditions necessary for accurate force field calculations.

The simulation pipeline consists of two critical equilibration phases. First, it performs an NVT ensemble simulation (constant Number of particles, Volume, and Temperature) to stabilize the system's thermal energy. This is followed by an NPT ensemble simulation (constant Number of particles, Pressure, and Temperature) to adjust the simulation cell volume and ensure the density is correctly equilibrated.

By automating these steps, the workflow provides a reproducible path for researchers in computational chemistry to move from system building to production-ready MD. It is particularly useful for those following [GTN](https://training.galaxyproject.org/) training materials or managing complex NAMD simulation protocols.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run main-workflow-charmmgui.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run main-workflow-charmmgui.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run main-workflow-charmmgui.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init main-workflow-charmmgui.ga -o job.yml`
- Lint: `planemo workflow_lint main-workflow-charmmgui.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
