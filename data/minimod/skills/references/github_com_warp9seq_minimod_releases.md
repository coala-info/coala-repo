## Improvements

* support for more types of modifications on ONT platforms

  + Fix a bug that affected the negative strand reads not being output (CG was not affected by this before).
  + Status of skipped bases (encoded as . or ? in MM tag) is considered now. Previously, the status of skipped bases (eg: the MM tag starting with C+m. means low probability and starting with C+m? means unknown) was not handled. We have fixed it in this version. When skipped as low probability, minimod considers them to have 0 modification probability. Otherwise, when skipped as unknown, they are ignored.
  + All modification codes can be requested using a star wildcard.
  + Treat U, T bases as one
  + Now, a warning is printed if a certain modification and a context are untested. See **Important NOTE** below for details.
* New options added: `--allow-secondary` and `--skip-supplementary`. From this version, secondary alignments will be skipped by default unless `--allow-secondary` is specified. Supplementary alignments are included by default similar to before unless `--skip-supplementary` is specified.
* `freq` output is now sorted.
* `view` output is ordered in the same order as reads appear in the input BAM file, and for each read, entries are sorted by reference contig, reference position, strand, and modification code.
* New `summary` subtool.
* Remove the index file requirement for the BAM file.
* fix a bug where erroneous reference positions were included when `--insertions` option is used.
* slight change to how the modification probability is converted from an unsigned 8-bit in BAM (N) to a float probability (p). Previously, it was p = N/255.0. Now we use p = (N+0.5)/256.0 as the conversion function.
* From this version, when a context is provided (ex: -c h[CG]) option, we output modifications where context matches in reference and the modified read base matches the aligned reference base. However, when the context is \* (ex: -c a[\*]), both of those comparisons are ignored, and modifications in any context and at both matched and mismatched positions are output.

## **Important NOTE**

The previous versions only had the following supported and tested:

* For PacBio: m[CG] for DNA (aligned to genome)
* For ONT:
  + m[CG] for DNA (aligned to genome) basecalled using models with CG context (5hmc\_5mc\_cg)
  + a[A] for direct RNA (aligned to transcriptome) basecalled using models with DRACH context (m6A\_DRACH).

In this version, we have supported and tested the following in addition to the above:

* For ONT:
  + a[A] for direct RNA (aligned to genome) basecalled using models with DRACH context (m6A\_DRACH)
  + various other modification combinations as summarised [here](https://github.com/warp9seq/minimod/tree/fe9912267cb665443af78585c5f8e30512b826ae#ont)
