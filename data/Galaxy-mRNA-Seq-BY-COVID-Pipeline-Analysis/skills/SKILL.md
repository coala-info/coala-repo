# mRNA-Seq BY-COVID Pipeline: Analysis

## Overview

This Galaxy workflow is designed for the differential expression analysis of bulk mRNA-Seq data, specifically tailored for the [BY-COVID](https://by-covid.org/) project. It processes gene counts and length data—typically generated by `featureCounts`—alongside a factor metadata file to identify significantly regulated genes and prepare data for downstream systems biology applications.

The pipeline utilizes the [limma-voom](https://bioconductor.org/packages/release/bioc/html/limma.html) engine to perform statistical modeling and generate comprehensive differential expression reports. It incorporates ID annotation via `annotateMyIDs` and performs functional enrichment analysis using `goseq` to provide biological context to the results.

A key feature of this workflow is the generation of a formatted data table specifically structured for [MINERVA](https://minerva.pages.uni.lu/) platform integration. This allows researchers to transition from raw count statistics to interactive pathway analysis and disease map visualizations.

The workflow is licensed under [GPL-3.0-or-later](https://spdx.org/licenses/GPL-3.0-or-later.html) and is associated with the [Galaxy Training Network (GTN)](https://training.galaxyproject.org/). Detailed documentation regarding specific tool parameters and usage can be found in the [README.md](./README.md) file in the Files section.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | featureCounts: Counts | data_collection_input | count data collection with two column datasets (gene_id, count) |
| 1 | factordata | data_input | A two column factor table with (Sample Identifier, Condition)  This workflow assumes a 1 factor, 2 level analysis, and was specifically designed around SARS-CoV-2 analysis with two levels, e.g. ``` SampleName 	Group SRR16683284 	COVID SRR16683283 	COVID  SRR16683271 	healthy SRR16683270 	healthy  ``` |
| 2 | featureCounts: Lengths | data_collection_input | featureCounts Lengths collection |


Ensure that the count and length inputs are provided as paired data collections to maintain sample associations, while the factor data should be a single tabular file defining your experimental design. The factor data must accurately map sample IDs to their respective conditions to ensure the limma tool correctly identifies differential expression. For comprehensive formatting requirements and metadata specifications, refer to the included README.md file. You may also use `planemo workflow_job_init` to create a `job.yml` for streamlined execution and testing.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run galaxy-workflow-mrna-seq-by-covid-pipeline--analysis.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run galaxy-workflow-mrna-seq-by-covid-pipeline--analysis.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run galaxy-workflow-mrna-seq-by-covid-pipeline--analysis.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init galaxy-workflow-mrna-seq-by-covid-pipeline--analysis.ga -o job.yml`
- Lint: `planemo workflow_lint galaxy-workflow-mrna-seq-by-covid-pipeline--analysis.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
