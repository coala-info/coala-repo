To disable it, use --masking 0 . DIAMOND is optimized for large input files of &gt;1 million proteins. Naturally the tool can be used for smaller files as well, but the algorithm will not reach its full efficiency. The program may use quite a lot of memory and also temporary disk space. Should the program fail due to running out of either one, you need to set a lower value for the block size parameter -b . The sensitivity can be adjusted using the options --faster , --fast , --mid-sensitive , --sensitive , --more-sensitive , --very-sensitive and --ultra-sensitive .
