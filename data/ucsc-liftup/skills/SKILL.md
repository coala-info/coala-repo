# ucsc-liftup

## Overview

The `liftUp` utility is a specialized tool from the UCSC Genome Browser suite designed for bulk coordinate remapping. While often confused with `liftOver`, `liftUp` is distinct because it uses a simple tab-separated `.lift` file rather than complex `.chain` files. It is primarily used during genome assembly pipelines to transform coordinates from smaller sequence units (like contigs) to larger structures (like scaffolds or chromosomes). It supports a wide variety of common bioinformatics formats including PSL, GFF, GTF, and AGP.

## Command Line Usage

The basic syntax for `liftUp` requires specifying the destination file, the mapping file, the data type, and the source file.

```bash
liftUp [options] destFile liftFile sourceType sourceFile
```

### Supported Source Types
You must specify the format of the input file as one of the following:
- `psl`: Protein/Sequence Line (alignment)
- `gff` / `gtf`: Gene annotation formats
- `agp`: Assembly Golden Path
- `gap`: Gap specifications
- `gl`: Gap list
- `out`: RepeatMasker output
- `align`: Alignment files

### Common Options
- `-nohead`: Use this if your input file does not have a header line.
- `-type=X`: Explicitly set the type if the positional argument is ambiguous.

## The .lift File Format

The `liftFile` is the core of the operation. It is a tab-separated file that tells the tool how to translate coordinates. Each line typically follows this structure:
`[offset] [oldName] [oldSize] [newName] [newSize]`

- **offset**: The starting position of the old sequence within the new sequence.
- **oldName**: The identifier in the current file (e.g., contig_01).
- **oldSize**: The full length of the old sequence.
- **newName**: The identifier for the target coordinate system (e.g., chr1).
- **newSize**: The full length of the new sequence.

## Expert Tips and Best Practices

1. **Intra-Assembly Mapping**: Use `liftUp` for simple linear transformations (contig -> scaffold). If you are mapping between different assembly versions (e.g., hg19 to hg38) or different species, use `liftOver` instead.
2. **Permissions**: If running the binary directly from the UCSC download server, ensure it is executable: `chmod +x liftUp`.
3. **GFF/GTF Specifics**: When lifting GFF or GTF files, `liftUp` correctly adjusts the start and end coordinates of features based on the offset provided in the lift file.
4. **Validation**: Always verify a few records manually after a lift, especially checking that features near the end of a contig do not exceed the `newSize` specified in your lift file.
5. **Batch Processing**: `liftUp` is highly efficient for large PSL files generated by BLAT. It is often faster to lift the results of a BLAT search than to re-run the search against a different assembly level.
