# mea

## Overview
The `mea` toolset provides a specialized approach to RNA secondary structure prediction by focusing on maximum expected accuracy rather than minimum free energy. It is particularly useful for addressing "the trouble with long-range base pairs" by allowing fine-grained control over base pair penalties based on their span. You should use this skill to process ViennaRNA-formatted dot plots (`dp.ps`), compare predicted structures to references using F1-score and MCC, or mix multiple dot plots to improve prediction reliability.

## Command Line Usage

### Core Prediction
Predict a secondary structure from a ViennaRNA dot plot file:
```bash
mea [options] <dotplot.dp.ps>
```
*   **Parameter Tuning**: Control the base pair penalty and accuracy trade-offs using `--alpha`, `--beta`, `--gamma`, and `--delta`.
*   **Input Format**: Ensure input files are in the standard `dp.ps` format generated by tools like `RNAfold -p`.

### Structure Evaluation
Compare a predicted structure against a known reference to calculate accuracy metrics:
```bash
mea --structure "<predicted_bracket_notation>" --reference "<reference_bracket_notation>"
```
*   **Metrics**: Outputs a confusion matrix, F1-score, and Matthews Correlation Coefficient (MCC).
*   **Comparison Rules**: By default, version 0.6.x uses "slide rule" and "conflicts rule" for more nuanced structural comparisons.

### Mixing Dot Plots
Combine information from two different dot plots (e.g., from different models or experimental sources) to predict a single consensus-style MEA structure:
```bash
mea_mix [options] <dotplot1.dp.ps> <dotplot2.dp.ps>
```
*   **Weighting**: Use `--gamma1` and `--gamma2` to control the relative influence of each input dot plot on the final prediction.

### Parameter Optimization
For large-scale evaluation of how parameters affect prediction quality across a dataset:
```bash
mea_eval [options] <reference_data>
```

## Expert Tips
*   **Long-Range Pairs**: If your biological context involves significant long-range interactions, experiment with the span-dependent correction parameters to prevent the model from over-penalizing distant base pairs.
*   **R Integration**: If performing complex statistical optimization, the package can compile a shared library (`mea-eval-R.so`) for direct use within R environments.
*   **Help Flag**: Use `mea --help` to see the specific default values for alpha, beta, and gamma, as these may vary between versions.
