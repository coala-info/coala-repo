## Overview

The `spacerextractor` tool is designed to identify and extract CRISPR spacer sequences from short reads generated from metagenomic samples. It helps researchers investigate the diversity and function of CRISPR-Cas systems in microbial communities by pinpointing the specific DNA sequences that bacteria and archaea use to defend against foreign genetic elements like viruses.

## Usage Instructions

`spacerextractor` is a command-line tool. The primary function is to process short-read sequencing data (e.g., FASTQ files) to identify and extract CRISPR spacers.

### Core Functionality

The main command involves specifying input read files and an output directory.

**Basic Command Structure:**

```bash
spacerextractor -i <input_fastq_files> -o <output_directory> [options]
```

**Key Arguments:**

*   `-i, --input`: Path to the input FASTQ file(s). Can be a single file or a comma-separated list for paired-end reads.
*   `-o, --output`: Directory where the extracted spacers and associated files will be saved.
*   `--db`: Path to a pre-built database for matching spacers. If not provided, the tool may build one or use a default.
*   `--min_len`: Minimum length of a spacer to be considered.
*   `--max_len`: Maximum length of a spacer to be considered.
*   `--threads`: Number of CPU threads to use for processing.

### Common Workflows and Tips

1.  **Processing Paired-End Reads:**
    If you have paired-end reads (e.g., `read1.fastq.gz` and `read2.fastq.gz`), provide them as a comma-separated list:
    ```bash
    spacerextractor -i read1.fastq.gz,read2.fastq.gz -o ./spacers_output --threads 8
    ```

2.  **Specifying a Custom Database:**
    For more accurate results, especially when dealing with specific microbial communities or known CRISPR arrays, you can provide a custom database. Ensure the database is in the correct format (refer to the tool's documentation for specifics on database creation).
    ```bash
    spacerextractor -i input.fastq -o ./output --db /path/to/custom_spacers.db --min_len 20 --max_len 60
    ```

3.  **Adjusting Spacer Length Filters:**
    CRISPR spacers typically fall within a certain length range. Adjusting `--min_len` and `--max_len` can help filter out non-specific matches. Common lengths are between 20 and 60 base pairs.
    ```bash
    spacerextractor -i reads.fq -o ./results --min_len 24 --max_len 50
    ```

4.  **Optimizing Performance:**
    Utilize the `--threads` option to speed up processing, especially on large datasets. The optimal number of threads depends on your system's CPU cores.
    ```bash
    spacerextractor -i sample.fastq.gz -o ./output --threads 16
    ```

5.  **Understanding Output:**
    The output directory will typically contain files such as:
    *   Extracted spacer sequences (e.g., in FASTA format).
    *   Information about the source read and its location.
    *   Potentially, files related to database building or matching statistics.

### Expert Tips

*   **Quality Control:** Always perform quality control on your raw sequencing reads (e.g., using FastQC) before running `spacerextractor` to ensure high-quality input data.
*   **Database Management:** If you frequently work with specific types of metagenomes, consider building and maintaining a curated database of known CRISPR arrays or spacer sequences to improve the accuracy and speed of your analyses.
*   **Parameter Tuning:** Experiment with `--min_len` and `--max_len` based on the known characteristics of CRISPR spacers in your target organisms or environments.
*   **Error Handling:** Pay attention to any error messages or logs generated by the tool. Issues with database creation or file paths are common.



## Subcommands

| Command | Description |
|---------|-------------|
| spacerextractor build_database | add new repeats identified by CRISPR-Cas typer to the repeat database |
| spacerextractor create_target_db | create a database of potential targets to map spacers to from a fasta file |
| spacerextractor download_database | extract spacers from metagenomic reads using a database of known repeats |
| spacerextractor extract_spacers | extract spacers from metagenomic reads using a database of known repeats |
| spacerextractor filter_hq_spacers | filter results of 'extract_spacers' to only retain high-quality spacers |
| spacerextractor map_to_target | map spacers to a database of potential targets |
| spacerextractor run_cctyper | run CRISPR-Cas typer on a (set of) new genomes to identify new CRISPR repeats |
