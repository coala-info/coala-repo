# gtftools

## Overview
GTFtools is a specialized command-line utility designed to parse gene models and provide functional genomic data. It transforms complex GTF (Gene Transfer Format) files into more accessible formats like BED or plain text. Use this skill when you need to calculate non-overlapping exonic lengths, extract specific genomic elements (UTRs, introns, intergenic regions), or define promoter/TSS regions for downstream analysis like transcription factor binding site scanning.

## Common CLI Patterns

### Feature Extraction (BED Output)
Extract specific genomic components into BED format for use with tools like bedtools:
- **Exons**: `gtftools -e exons.bed input.gtf`
- **Introns**: `gtftools -i introns.bed input.gtf`
- **UTRs**: `gtftools -u utr.bed input.gtf`
- **Intergenic Regions**: `gtftools -b intergenic.bed input.gtf`
- **Isoform Coordinates**: `gtftools -s isoforms.bed input.gtf`

### Length Calculations
Essential for RNA-seq normalization (e.g., FPKM/TPM):
- **Gene Lengths**: `gtftools -l gene_lengths.txt input.gtf`
  - *Note*: This outputs four values: mean, median, and max isoform lengths, plus the merged exonic length.
- **Isoform Lengths**: `gtftools -r isoform_lengths.txt input.gtf`

### Advanced Genomic Regions
- **Merged Exons**: `gtftools -m merged_exons.bed input.gtf` (Useful for calculating non-overlapping exonic length).
- **Independent Introns**: `gtftools -d independent_introns.bed input.gtf` (Introns that do not overlap any exons; critical for intron retention studies).
- **Splice Sites**: `gtftools -q splice_regions.bed input.gtf` (Extracts 5' donor and 3' acceptor sites based on MaxEntScan regions).
- **TSS-Flanking Regions**: `gtftools -t tss.bed -w 1000-300 input.gtf` (Extracts 1000bp upstream and 300bp downstream of the TSS).

### Filtering and Mapping
- **Chromosome Filtering**: Use the `-c` flag to limit analysis to specific chromosomes.
  - Example: `gtftools -g genes.bed -c 1-5,X,Y input.gtf`
- **Cis-SNP Search**: Find SNPs within a specific range of genes.
  - Example: `gtftools -p snp_list.txt input.gtf > cis_snps.bed`
  - Use `-f` to set the range (e.g., `-f 2000-1000` for 2kb upstream and 1kb downstream).

## Expert Tips
- **Coordinate Systems**: Be aware that GTF files are typically 1-based, while the BED files generated by gtftools are 0-based, making them immediately compatible with most genome browsers and the bedtools suite.
- **Gene Length Selection**: For FPKM/TPM calculations, the "merged exon length" (the fourth column in `-l` output) is generally the most accurate representation of a gene's total exonic sequence.
- **Non-Human Models**: While often used for human data, gtftools is agnostic to the organism and works effectively with mouse or other well-annotated gene models.
- **Memory Efficiency**: If processing very large GTF files, use the `-c` flag to process chromosomes individually to reduce memory overhead.
