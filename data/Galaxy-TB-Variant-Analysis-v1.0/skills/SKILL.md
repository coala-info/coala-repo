# TB Variant Analysis v1.0

## Overview

This Galaxy workflow provides a comprehensive pipeline for the genomic analysis of *Mycobacterium tuberculosis* (TB) from Illumina sequencing samples. It automates the identification of genetic variants and the prediction of drug resistance, facilitating rapid pathogen characterization. The workflow accepts raw sequencing reads and a reference genome, allowing users to fine-tune analysis through parameters like minimum allele frequency and coverage depth.

The pipeline begins with data preprocessing using [fastp](https://github.com/OpenGene/fastp) for quality control and [Kraken2](https://github.com/google/kraken2) for taxonomic classification to ensure sample purity. Variant calling is performed by [snippy](https://github.com/tseemann/snippy), followed by specialized filtering via [TB Variant Filter](https://github.com/iuc/tb_variant_filter) to isolate high-confidence mutations. For clinical relevance, [TB-Profiler](https://github.com/jodyphelan/TB-Profiler) is utilized to cross-reference detected variants against known drug resistance markers.

The final outputs include annotated VCF files, consensus genomes generated by [bcftools](https://samtools.github.io/bcftools/), and detailed HTML reports summarizing both variants and drug resistance profiles. Comprehensive quality metrics are aggregated using [MultiQC](https://multiqc.info/) and [QualiMap](http://qualimap.conesalab.org/), providing a clear overview of the sequencing performance and the pathogen's genomic landscape. This workflow is tagged for [GTN](https://training.galaxyproject.org/) (Galaxy Training Network) and is licensed under AGPL-3.0-or-later.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Reads | data_collection_input | List of paired Illumina reads (set format to fastqsanger or fastqsanger.gz as appropriate) |
| 1 | Reference Genome | data_input | M. tuberculosis reference genome (must be in H37Rv coordinates. The M. tuberculosis inferred ancestral genome is recommended) |
| 2 | Minimum variant allele frequency | parameter_input | Minimum proportion of reads that must support a variant for it to be called |
| 3 | Minimum depth of coverage | parameter_input | Minimum depth of coverage for a position on the genome |
| 4 | Additional BWA-MEM options | parameter_input | Increasing the minimum seed length (-k) parameter can help eliminate matches from contaminants. If used this should be set to 2/3rds of the read length, i.e. the string "-k 100" should be used for 150 base pair reads |


For optimal results, provide Illumina sequencing reads as a paired-end list collection (fastqsanger) and the *M. tuberculosis* reference genome in FASTA format. Ensure that the reference sequence naming matches the expected nomenclature for TB-Profiler and Snippy to avoid alignment errors. You can use `planemo workflow_job_init` to generate a `job.yml` file for automated parameter configuration and batch execution. Refer to the README.md for comprehensive details on adjusting the minimum variant allele frequency and depth of coverage thresholds.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run tb-variant-analysis.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run tb-variant-analysis.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run tb-variant-analysis.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init tb-variant-analysis.ga -o job.yml`
- Lint: `planemo workflow_lint tb-variant-analysis.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
