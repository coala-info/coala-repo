# bioconductor-genomation

name: bioconductor-genomation
description: Toolkit for the annotation and visualization of genomic intervals. Use this skill when working in R to process tabular genomic data (BED, GFF, GTF), BAM, or BigWig files. It is specifically designed for extracting signal enrichment (e.g., ChIP-seq, methylation) over predefined regions, generating heatmaps and meta-plots, and annotating genomic intervals with gene structures like promoters, exons, and introns.

# bioconductor-genomation

## Overview
The `genomation` package is a powerful R toolkit for integrating diverse genomic datasets with annotations. It allows users to visualize "scored" genomic intervals (like read depths or methylation percentages) across "windows" of interest (like transcription start sites or CpG islands). Its primary strength lies in converting raw genomic signals into matrices for bulk visualization and providing streamlined workflows for overlapping peaks with gene models.

## Data Input
The package provides flexible functions to read genomic data into `GRanges` objects:
* `readGeneric()`: The primary function for tabular data. Use `chr`, `start`, and `end` arguments to specify columns. Use `meta.col` to keep scores.
* `gffToGRanges()`: Specifically for GFF/GTF files. Use `split.group = TRUE` to parse the attributes column.
* `readBroadPeak()` / `readNarrowPeak()`: Convenience wrappers for ENCODE peak formats.
* `readTranscriptFeatures()`: Reads a BED12 file and returns a `GRangesList` containing exons, introns, promoters, and TSSes.

## Extraction of Data (ScoreMatrix)
To visualize signal over regions, you must first create a `ScoreMatrix`:
* `ScoreMatrix(target, windows)`: Use when all windows have the same width (e.g., +/- 1kb around TSS). `target` can be a BAM file, BigWig, or `GRanges`.
* `ScoreMatrixBin(target, windows, bin.num)`: Use for windows of unequal width (e.g., exons). It scales all regions to a fixed number of bins.
* `ScoreMatrixList(targets, windows)`: Use to process multiple experiments (e.g., several ChIP-seq tracks) simultaneously against the same set of windows.

## Visualization
Once a matrix is created, use these functions for plotting:
* `heatMatrix(sm)`: Generates a heatmap of the signal.
* `plotMeta(sm)`: Generates a line plot (meta-plot) showing the average signal across all windows.
* `multiHeatMatrix(sml)`: Plots multiple `ScoreMatrix` objects side-by-side.
* **Scaling**: Use `scaleScoreMatrix()` or `scaleScoreMatrixList()` before plotting if the dynamic range of the signal is very high.

## Annotation
To understand where genomic intervals (like peaks) fall relative to other features:
* `annotateWithFeature(target, feature)`: Basic overlap analysis between two sets of ranges.
* `annotateWithGeneParts(target, gene.parts)`: Overlaps targets with promoters, exons, and introns. Requires a `GRangesList` typically generated by `readTranscriptFeatures()`.
* `plotGeneAnnotation(annot.list)`: Visualizes the percentage of overlaps in a bar plot or heatmap.

## Typical Workflow
1.  **Load Annotations**: `genes = readTranscriptFeatures("hg19_refseq.bed")`
2.  **Load Signal**: `peaks = readBroadPeak("sample_peaks.broadPeak")`
3.  **Annotate**: `annot = annotateWithGeneParts(peaks, genes)`
4.  **Extract Signal for Heatmap**: 
    `sm = ScoreMatrix(target = "sample_coverage.bw", windows = genes$TSS)`
5.  **Visualize**: `heatMatrix(sm, xcoords = c(-1000, 1000))`

## Tips
* **Strand Awareness**: Always set `strand.aware = TRUE` in `ScoreMatrix` functions if your windows (like TSSes) have strand information; this ensures the signal is oriented correctly (5' to 3').
* **Memory**: For large BAM files, `genomation` works efficiently with indexed files, but ensure `Rsamtools` is available.
* **Filtering**: `ScoreMatrixBin` will drop windows shorter than the `bin.num`. Check warnings to see if regions were excluded.
