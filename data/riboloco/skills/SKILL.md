# riboloco

## Overview
riboloco is a specialized tool designed for the precise analysis of Riboseq data. It addresses the unique challenges of ribosome profiling by calculating P-site offsets—the distance from the end of a sequencing read to the actual site of translation—and using these offsets to map reads to specific codons. This allows for high-resolution monitoring of protein synthesis across the transcriptome.

## CLI Usage and Best Practices

### Core Workflow
The primary entry point is the `riboloco` command. The tool typically requires a transcriptome-aligned BAM file and a corresponding annotation file (GTF/GFF).

```bash
# Basic execution pattern
riboloco --bam input.bam --gtf annotation.gtf --out_dir results/
```

### P-site Offset Determination
One of the most critical steps in Riboseq analysis is determining the offset for different read lengths.
- **Automatic Detection**: riboloco can automatically calculate offsets by looking at the distribution of reads around known Start and Stop codons.
- **Manual Overrides**: If the automatic detection fails due to low coverage, you can provide a custom offset file using the `--offsets` parameter.

### Common Command Line Options
- `--length_range`: Specify the range of read lengths to include (typically 25-35 nucleotides for eukaryotes).
- `--min_reads`: Set a threshold for the minimum number of reads required to process a transcript, filtering out low-confidence data.
- `--parallel`: Utilize multiple CPU cores to speed up the quantification process for large datasets.

### Expert Tips
- **Quality Control**: Always check the `metagene_profiles` generated by the tool. A successful run should show a clear 3-nucleotide periodicity (triplet periodicity) in the CDS region, which confirms that the P-site offsets are accurate.
- **Alignment Strategy**: Ensure your BAM files are sorted and indexed. riboloco performs best with reads aligned to the transcriptome rather than the whole genome to avoid complications with splicing.
- **Multimappers**: Decide on a strategy for multi-mapping reads before running riboloco. It is generally recommended to either filter them out or use a tool that assigns them weights, as riboloco's precision depends on unambiguous mapping.
