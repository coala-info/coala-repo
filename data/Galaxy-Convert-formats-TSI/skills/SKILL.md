# Convert formats - TSI

## Overview

This Galaxy workflow is designed to bridge the gap between TransDecoder results and the requirements of the Fgenesh gene prediction suite. It takes two primary inputs: peptide sequences (`transdecoder-peptides.fasta`) and nucleotide sequences (`transdecoder-nucleotides.fasta`). The main objective is to reformat these files into the specific `.pro`, `.dat`, and `.cdna` formats required for downstream analysis.

The workflow employs an extensive series of text processing and sequence manipulation tools to restructure the data. It utilizes [grep](https://toolshed.g2.bx.psu.edu/repos/bgruening/text_processing/tp_grep_tool/9.3+galaxy0), [sed](https://toolshed.g2.bx.psu.edu/repos/bgruening/text_processing/tp_sed_tool/9.3+galaxy0), and [awk](https://toolshed.g2.bx.psu.edu/repos/bgruening/text_processing/tp_awk_tool/9.3+galaxy0) for data extraction and reformatting, alongside [seqtk](https://toolshed.g2.bx.psu.edu/repos/iuc/seqtk/seqtk_subseq/1.4+galaxy0) for sequence handling. These steps ensure that headers and sequence data are correctly parsed and structured for compatibility.

By automating the conversion of FASTA files into tabular formats and back again, the workflow handles complex tasks such as sequence length computation and multi-step string replacements. This automation reduces manual errors in data preparation, streamlining the transition from transcriptomic ORF prediction to structural gene annotation.

You can use `planemo workflow_job_init` to generate a `job.yml` file for automated execution and testing.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Convert-formats-TSI.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Convert-formats-TSI.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Convert-formats-TSI.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Convert-formats-TSI.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Convert-formats-TSI.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
