## Overview
nf-core/oncoanalyser is designed for the comprehensive analysis of cancer DNA and RNA sequencing data. It leverages the WiGiTS toolkit from the Hartwig Medical Foundation to provide a unified workflow for identifying somatic variants, structural variants, copy number alterations, and immune-related features.

The pipeline processes various data modalities including Whole Genome Sequencing (WGS), Whole Transcriptome Sequencing (WTS), and targeted panels. It produces integrated reports that synthesize genomic and transcriptomic findings, making it suitable for clinical research and longitudinal studies where purity estimates across multiple timepoints are required.

## Data preparation
The pipeline requires a comma-separated (CSV) samplesheet specified with the `--input` parameter. This file must contain information about the subjects, samples, and their respective data files.

Required columns in the samplesheet:
- `group_id`: Groups sample entries into a single analysis unit.
- `subject_id`: Patient identifier used for internal sanity checks.
- `sample_id`: Unique identifier for the specific sample.
- `sample_type`: Must be one of `normal`, `tumor`, `donor`, or `tumor_normal`.
- `sequence_type`: Must be one of `dna`, `rna`, or `dna_rna`.
- `filetype`: The format of the input file (e.g., `bam`, `fastq`, `cram`).
- `filepath`: Absolute path to the sequencing file.

Example samplesheet:
```csv
group_id,subject_id,sample_id,sample_type,sequence_type,filetype,filepath
PATIENT1_WGTS,PATIENT1,PATIENT1-N,normal,dna,bam,/path/to/PATIENT1-N.dna.bam
PATIENT1_WGTS,PATIENT1,PATIENT1-T,tumor,dna,bam,/path/to/PATIENT1-T.dna.bam
PATIENT1_WGTS,PATIENT1,PATIENT1-T-RNA,tumor,rna,bam,/path/to/PATIENT1-T.rna.bam
```

Reference data must be provided via the `--genome` parameter (e.g., `GRCh37_hmf` or `GRCh38_hmf`). Custom panels or exomes require additional reference data preparation as described in the official documentation.

## How to run
The pipeline is executed using the `nextflow run` command. You must specify a workflow mode and a reference genome.

```bash
nextflow run nf-core/oncoanalyser \
  -profile <docker/singularity/conda> \
  -r 2.3.0 \
  --mode <wgts/targeted/purity_estimate> \
  --genome <GRCh37_hmf/GRCh38_hmf> \
  --input samplesheet.csv \
  --outdir <results_directory>
```

Key parameters:
- `-profile`: Defines the software container or environment (e.g., `docker`).
- `--mode`: Selects the analysis workflow (`wgts` for WGS/WTS, `targeted` for panels, or `purity_estimate`).
- `-r`: Pins a specific version of the pipeline for reproducibility.
- `-resume`: Allows the pipeline to restart from the last successful step if interrupted.

## Outputs
Results are saved in the directory specified by `--outdir`. The primary deliverables include:

- **Summary Reports**: Integrated analysis reports generated by ORANGE and linxreport.
- **Variant Calls**: SNV, MNV, and INDEL calls from SAGE/PAVE, and SV calls from ESVEE.
- **Copy Number**: CNV and purity estimates from AMBER, COBALT, and PURPLE.
- **RNA Analysis**: Transcript counts and fusion detection from ISOFOX.
- **Immune Analysis**: HLA typing and immune microenvironment characterization from LILAC and CIDER.

For a detailed description of all output files, refer to the [official output documentation](https://nf-co.re/oncoanalyser/output).
