# bioconductor-frenchfish

## Overview

The `frenchFISH` package provides methods to correct DNA copy-number estimates derived from FISH imaging of tissue sections. Standard spot counting often underestimates copy numbers in solid tumors due to truncated or overlapping nuclei in thin tissue slices. `frenchFISH` addresses this using a nuclear volume correction method combined with Poisson models to provide more accurate estimates of the true biological copy number.

## Core Workflows

### 1. Correcting Automated Counts
Use this workflow when you have spot counts generated by automated software like `FISHalyseR`.

```r
library(frenchFISH)

# 1. Load or convert data
# If using FISHalyseR CSV output:
csv_path <- "path/to/FISHalyseR_output.csv"
automatic_counts <- convertFishalyserCsvToCountMatrix(csv_path)

# 2. Set parameters
# nuclear_radius: average radius of the nuclei in the sample
# section_height: thickness of the tissue section (microns)
set.seed(366) # For reproducibility
nuclear_radius <- 40
section_height <- 5

# 3. Run Poisson Point Process model
corrected_auto <- getAutomaticCountsEstimates(
  automatic_counts,
  nuclear_radius,
  section_height
)

# 4. View results (includes lowCI, median, highCI)
print(corrected_auto)
```

### 2. Correcting Manual Counts
Use this workflow for manually counted spots where area measurements might not be available for every nucleus.

```r
library(frenchFISH)

# 1. Prepare count matrix (Rows = Nuclei, Columns = Probes)
# Example: 20 nuclei, 3 probes (red, green, blue)
manual_counts <- matrix(
  sample(0:5, 60, replace = TRUE), 
  ncol = 3, 
  dimnames = list(NULL, c("red", "green", "blue"))
)

# 2. Run Markov chain Monte Carlo (MCMC) model
nuclear_radius <- 8
section_height <- 4
corrected_manual <- getManualCountsEstimates(
  manual_counts,
  nuclear_radius,
  section_height
)

# 3. View results (percentiles X2.5 to X97.5)
print(corrected_manual)
```

## Key Functions

- `convertFishalyserCsvToCountMatrix()`: Formats FISHalyseR output for processing.
- `getAutomaticCountsEstimates()`: Implements a homogenous Poisson Point Process model for automated data.
- `getManualCountsEstimates()`: Implements a Poisson model via MCMC for manual counts without requiring control probes.

## Usage Tips

- **Parameter Accuracy**: The `nuclear_radius` and `section_height` are critical for the volume correction calculation. Ensure these match your experimental setup (e.g., FFPE section thickness).
- **Reproducibility**: Always use `set.seed()` before running estimation functions, as the underlying models involve stochastic processes.
- **Input Format**: For manual counts, the input should be a matrix where columns represent different color probes and rows represent individual nuclei.
