# nanocomp

## Overview
`nanocomp` is a specialized utility for the comparative analysis of long-read sequencing data, specifically optimized for Oxford Nanopore Technologies (ONT). It allows researchers to visualize differences between multiple datasets by generating interactive HTML reports and static images. Key metrics compared include read lengths, Phred quality scores, and alignment identities.

## Common CLI Patterns

### Comparing FASTQ Files
Compare multiple raw or compressed FASTQ files and assign custom labels:
`nanocomp --fastq run1.fastq.gz run2.fastq.gz --names Sample1 Sample2 -o output_dir`

### Comparing Alignments (BAM)
Analyze sorted BAM files to compare percent identity and alignment quality:
`nanocomp --bam align1.bam align2.bam --names WildType Mutant -o comparison_results`

### Using Sequencing Summary Files
For faster processing, use the `sequencing_summary.txt` files generated by basecallers (Guppy/Albacore):
`nanocomp --summary summary_runA.txt summary_runB.txt --names Flowcell_1 Flowcell_2`

### Handling Barcoded Data
If using a single summary file containing multiple barcodes, use the `--barcoded` flag to split the analysis:
`nanocomp --summary sequencing_summary.txt --barcoded`

## Expert Tips and Best Practices

- **Performance**: Always specify the number of threads with `-t` or `--threads` to speed up the processing of large datasets.
- **Plot Selection**: The default plot type is `violin`. Use `--plot box` for standard box plots or `--plot ridge` for joyplots, which are often clearer when comparing a large number of samples.
- **Static Images**: `nanocomp` creates dynamic HTML plots by default. To generate static `.png` versions automatically, ensure `orca` is installed in your environment.
- **Data Extraction**: Use the `--raw` flag to save the extracted metrics in a tab-separated file. This is useful if you need to perform custom statistical tests or create bespoke plots outside of the tool.
- **Color Customization**: Use the `-c` flag followed by a list of colors (e.g., `-c red blue green`) to maintain consistent color-coding across different figures or to match experimental groups.
- **Filtering**: If working with summary files, you can specify the read type to extract (1D, 2D, or 1D2) using the `--readtype` flag.
