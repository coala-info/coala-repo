# pisad

## Overview

PISAD (Phased Intraspecies Sample Anomalies Detection) is a specialized tool for identifying sample swaps and relationship errors in genomic cohorts without requiring a reference genome. The tool operates in two primary stages: first, it performs reference-free SNP calling on high-quality "target" samples to build a variant sketch; second, it compares k-mer frequencies from the rest of the cohort against these sketches to calculate genetic relationships. This approach is highly efficient for large-scale sequencing projects where reference-based alignment is computationally expensive or a reference genome is unavailable.

## Workflow and CLI Usage

### Stage 1: SNP Calling and Sketch Construction

Identify a low-error sequencing dataset from your cohort to serve as the baseline for SNP calling.

1.  **Call SNPs**: Use `run.sh` to perform rapid, reference-free SNP calling.
    ```bash
    run.sh -i sample.fastq.gz -m 0 -t 16
    ```
    *   `-i`: Input FASTQ or FASTQ.GZ.
    *   `-m`: Heterozygosity parameter. Use `0` for <1.2% heterozygosity, `1` otherwise.
    *   `-t`: Number of threads (default is 8).

2.  **Create Variant Sketch**: Convert the resulting `.snp` file into a k-mer variant sketch.
    ```bash
    create -i sample_pairex.snp -k 21 -o output_prefix
    ```
    *   `-i`: The `.snp` file generated in the previous step.
    *   `-k`: k-mer size (default 21).
    *   `-l`: Filtering threshold (default 21).

### Stage 2: K-mer Counting and Evaluation

Compare other samples in the cohort against the variant sketch to detect anomalies.

1.  **Count k-mers**: Compare target samples against the sketch.
    ```bash
    pisadCount -s sketch.fa -t 12 other_sample.fastq.gz
    ```
    *   `-s`: Path to the variant sketch (FASTA). Multiple sketches can be comma-separated (e.g., `-s s1.fa,s2.fa`).
    *   `-m`: For high-coverage data, set a k-mer coverage threshold (e.g., `-m 5`) to terminate early and save time.

2.  **Evaluate Relationships**: Process the statistics to detect swaps.
    ```bash
    pisadEval results.txt > summary.tsv
    ```
    *   Input the text file generated by `pisadCount`. The output `summary.tsv` will contain the inferred relationships.

## Expert Tips and Best Practices

*   **Target Selection**: For Stage 1, always choose the sample with the highest sequencing depth and lowest error rate to ensure the variant sketch is accurate.
*   **Performance Optimization**: When dealing with high-coverage cohorts, the `-m` parameter in `pisadCount` is critical. Setting a low threshold (like 5 or 10) allows the tool to gather enough statistics for relationship inference without processing the entire file.
*   **Multi-Sketch Analysis**: You can evaluate a sample against multiple sketches simultaneously by providing a comma-separated list to `pisadCount -s`. This is useful for checking a sample against several potential "correct" identities at once.
*   **Memory Management**: PISAD relies on `gperftools` and `boost`. Ensure your environment has sufficient RAM when processing large k-mer sets, especially if increasing the k-mer size (`-k`) beyond the default 21.
