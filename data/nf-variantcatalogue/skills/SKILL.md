# variantcatalogue

## Overview
nf-core/variantcatalogue is a bioinformatics best-practice analysis pipeline designed to generate variant catalogues from whole genome sequences. It produces a list of variants and their frequencies in a population, facilitating the study of genetic variation across large cohorts.

The pipeline processes raw sequencing reads to produce high-quality variant calls and frequency data. It integrates standard alignment and variant calling tools with specialized frameworks like Hail for large-scale genomic data analysis and VEP for functional annotation.

## Data preparation
The pipeline requires a comma-separated samplesheet provided via the `--input` parameter. This file must contain three columns with a header row: `sample`, `fastq_1`, and `fastq_2`.

Example `samplesheet.csv`:
```csv
sample,fastq_1,fastq_2
CONTROL_REP1,AEG588A1_S1_L002_R1_001.fastq.gz,AEG588A1_S1_L002_R2_001.fastq.gz
CONTROL_REP2,AEG588A2_S2_L002_R1_001.fastq.gz,AEG588A2_S2_L002_R2_001.fastq.gz
```

Reference genome files are also required. You can specify a genome using the iGenomes ID (e.g., `--genome GRCh38`) or provide a path to a FASTA file using `--fasta`. If a BWA index is not provided, the pipeline will generate one automatically.

## How to run
The pipeline is executed using Nextflow with the `run` command. It is recommended to use a specific release version with `-r` and a container profile for reproducibility.

```bash
nextflow run nf-core/variantcatalogue \
    -r 1.0.0 \
    -profile docker \
    --input samplesheet.csv \
    --outdir ./results \
    --genome GRCh38
```

For testing purposes, a minimal dataset profile is available:
```bash
nextflow run nf-core/variantcatalogue -profile test,docker --outdir ./results
```

Key parameters include:
- `-profile`: Choose a configuration profile (e.g., `docker`, `singularity`, `conda`).
- `--outdir`: Path to the results directory.
- `-resume`: Restart a pipeline from the last successful step if it was interrupted.

## Outputs
Results are saved in the directory specified by `--outdir`. The primary deliverables include:

- **MultiQC Report**: Located in the `multiqc/` folder, this provides a summary of quality control metrics from FastQC, Picard, and Mosdepth.
- **Variant Catalogues**: VCF files and frequency tables generated by Hail and GLnexus.
- **Annotations**: VEP-annotated variant files.
- **Mitochondrial Results**: Specific variant calls and frequencies for the mitochondrial genome.

For a detailed description of all output files, refer to the [official output documentation](https://nf-co.re/variantcatalogue/output).
