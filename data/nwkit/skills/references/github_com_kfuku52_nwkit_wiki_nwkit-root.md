# nwkit root

## Overview

`nwkit root` places or transfers the tree root.

Available methods are `outgroup`, `transfer`, `mad`, `mv`, `midpoint`, and `taxonomy`.

## 1. Outgroup rooting with a single tip label

Outgroup rooting places the root node at the midpoint of the branch specified by the user with tip label(s).

### Command

```
nwkit root --method outgroup --outgroup a --infile input.nwk --outfile output.nwk
```

### input.nwk

```
(((a:1,b:1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224145035-b2b9df79-8059-4ab2-b2b2-4dd4be35cbef.png)

### output.nwk

```
(a:0.5,(b:1,(c:1,((d:1,e:1):1,f:1):2):1):0.5):0;
```

![image](https://user-images.githubusercontent.com/19270515/224145096-053448db-339a-42b6-b260-c03f3df6b16e.png)

## 2. Outgroup rooting with multiple tip labels

### Command

```
nwkit root --method outgroup --outgroup a,b --infile input.nwk --outfile output.nwk
```

### input.nwk

```
(((a:1,b:1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224145035-b2b9df79-8059-4ab2-b2b2-4dd4be35cbef.png)

### output.nwk

```
((a:1,b:1):0.5,(c:1,((d:1,e:1):1,f:1):2):0.5):0;
```

![image](https://user-images.githubusercontent.com/19270515/224146079-bd8ad5c4-4d82-40c6-b120-bb091873b625.png)

## 3. Transferring the root position from one tree to another

`--infile` and `--infile2` do not have to be identical in tree topology, but the [bipartition](https://en.wikipedia.org/wiki/Split_%28phylogenetics%29) of the branch where the root is placed should be the same. The root position is transferred from `--infile2` to `--infile`, and therefore, the branch lengths are inherited from `--infile` rather than `--infile2`. The length of the branch where the new root node is placed is split into two at the midpoint.

### Command

```
nwkit root --method transfer --infile input1.nwk --infile2 input2.nwk --outfile output.nwk
```

### input1.nwk

```
(((a:1,b:1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224141349-47393b91-4599-4738-b9be-779e934311b9.png)

### input2.nwk

```
((((a:1,b:1):1,c:1),f:1):1,(d:3,e:3):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224142149-12dd4b9a-f8c8-41c6-90b9-e97397a60fbd.png)

### output.nwk

```
((d:1,e:1):0.5,(f:1,(c:1,(b:1,a:1):1):2):0.5)Root:0;
```

![image](https://user-images.githubusercontent.com/19270515/224142181-700e9ef7-4dd2-4f38-974f-35af5cbbad72.png)

## 4. MAD rooting (Minimal Ancestor Deviation)

MAD rooting is an outgroup-free method that determines the root position by minimizing the deviation of all pairwise midpoint root positions. It does not require prior knowledge of outgroup taxa. See [Tria et al. 2017, Nature Ecology & Evolution](https://doi.org/10.1038/s41559-017-0193) for details.

### Command

```
nwkit root --method mad --infile input.nwk --outfile output.nwk
```

### input.nwk

```
((A:1,B:2):1,(C:3,(D:1,E:2):1):1);
```

### output.nwk

```
((A:1,B:2):1.58461,(C:3,(D:1,E:2):1):0.415389);
```

The root is placed at the position that minimizes the ancestor deviation across all pairs of taxa. MAD statistics (MAD score, Ambiguity Index, Clock CV) are reported to stderr.

## 5. MV rooting (Minimum Variance)

MV rooting is an outgroup-free method that determines the root position by minimizing the variance of root-to-tip distances. It does not require prior knowledge of outgroup taxa. See [Mai, Saeedian & Mirarab 2017, PLoS ONE](https://doi.org/10.1371/journal.pone.0182238) for details.

### Command

```
nwkit root --method mv --infile input.nwk --outfile output.nwk
```

### input.nwk

```
((A:1,B:2):1,(C:3,(D:1,E:2):1):1);
```

### output.nwk

```
((C:3,(D:1,E:2):1):0.416667,(A:1,B:2):1.58333);
```

The root is placed at the position that minimizes the variance of root-to-tip distances across all taxa. MV rooting variance is reported to stderr.

## 6. Midpoint rooting

Midpoint rooting places the root node in the midpoint of the longest path between two tips.

### Command

```
nwkit root --method midpoint --infile input.nwk --outfile output.nwk
```

### input.nwk

```
((((a:5,b:1):1,c:3),f:1):1,(d:1,e:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224147187-aac8190f-7135-491d-9aa1-c211f4b4000c.png)

### output.nwk

```
((a:5,b:1):0.5,(c:3,(f:1,(d:1,e:1):2):1):0.5):0;
```

![image](https://user-images.githubusercontent.com/19270515/224147270-65912e97-4d23-457c-b3ed-087ff638be92.png)

## 7. Taxonomy rooting

`--method taxonomy` derives a reference root split from external taxonomic or guide-tree sources and transfers that split onto `--infile`. Sources are tried in the order specified by `--taxonomy_source`. The default is `ncbi,opentree,timetree`.

When leaf taxids are inferred from tip labels, species-aware parsing expects `GENUS_SPECIES` or `GENUS_SPECIES_SUFFIX` by default. Fields after the species epithet are ignored when resolving species names. If your labels use another convention, provide `--species_regex`. For `timetree` and `opentree`, repeated labels for the same species are allowed only when all such tips form a monophyletic clade in `--infile`.

This mode is strict. A source is accepted only when:

* the source-derived guide tree has exactly two children at the root
* the corresponding root [bipartition](https://en.wikipedia.org/wiki/Split_%28phylogenetics%29) exists in `--infile`

If a source fails these conditions, `nwkit root` tries the next source in `--taxonomy_source`. The command exits with an error only when all specified sources fail.

### Command

```
nwkit root --method taxonomy --infile input.nwk --outfile output.nwk
```

### input.nwk

```
((Arabidopsis_thaliana:1,Oryza_sativa:1):1,(Saccharomyces_cerevisiae:1,(Homo_sapiens:1,Pan_troglodytes:1):1):1);
```

### output.nwk

```
((Homo_sapiens:1,Pan_troglodytes:1):0.5,((Arabidopsis_thaliana:1,Oryza_sativa:1):2,Saccharomyces_cerevisiae:1):0.5)Root:0;
```

In this example, taxonomy rooting moves the root to the branch separating the animal clade (`Homo_sapiens`, `Pan_troglodytes`) from the remaining taxa.

### Optional inputs and cache control

* `--taxonomy_source` accepts a comma-separated source chain such as `ncbi,opentree,timetree`.
* `--taxid_tsv` can be used with the `ncbi` source to provide exact leaf-label-to-taxid mappings. The `leaf_name` column must match the leaf labels in `--infile` exactly.
* `--rank` limits the highest retained NCBI rank when building the NCBI-derived reference tree.
* `--download_dir` controls where the ETE4 NCBI taxonomy database is cached when the `ncbi` source is used. `inferred` stores files under `<outfile_dir>/downloads`, or `./downloads` when writing to STDOUT.

### Example with explicit NCBI taxids

```
nwkit root \
--method taxonomy \
--taxonomy_source ncbi \
--taxid_tsv taxid.tsv \
--rank no \
--download_dir ./shared_downloads \
--infile input.nwk \
--outfile output.nwk
```

## Custom species parsing

```
nwkit root --method taxonomy --infile input.nwk --outfile output.nwk \
  --species_regex '^([A-Za-z]+)\.([A-Za-z]+)\|'
```
