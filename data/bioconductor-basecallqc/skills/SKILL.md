# bioconductor-basecallqc

## Overview

The `basecallQC` package provides a suite of tools to manage the input and output of Illumina's bcl2fastq software. It streamlines the preprocessing of Sample Sheets—ensuring they meet the strict formatting requirements of modern bcl2fastq versions—and provides functions to parse and visualize post-run quality control metrics.

## Core Workflow

### 1. Initializing Parameters
All workflows begin by capturing run parameters from the `runParameters.xml` file generated by the sequencer.

```r
library(basecallQC)

# Define paths to required files
runXML <- "path/to/runParameters.xml"
config <- "path/to/config.ini" # Usually provided with the package or run

# Create the parameter object
bcl2fastqparams <- BCL2FastQparams(runXML, config, runDir = getwd(), verbose = FALSE)
```

### 2. Sample Sheet Validation and Cleaning
Illumina Sample Sheets often contain invalid characters or outdated formats. `validateBCLSheet` fixes common issues like whitespaces in names, numeric-starting IDs, and incorrect headers.

```r
# Clean a sample sheet (fixes headers, sample IDs, and splits dual indexes)
cleanedSheet <- validateBCLSheet("SampleSheet.csv", param = bcl2fastqparams)

# The result is a data.frame ready to be written back to CSV
write.csv(cleanedSheet, "SampleSheet_cleaned.csv", row.names = FALSE, quote = FALSE)
```

### 3. Generating bcl2fastq Commands
You can generate the exact shell command needed to run bcl2fastq, including the complex `--use-bases-mask` strings.

```r
# 1. Create base masks
baseMasks <- createBasemasks(cleanedSheet, param = bcl2fastqparams)

# 2. Generate the command string
bclCommand <- createBCLcommand(bcl2fastqparams, cleanedSheet, baseMasks)
cat(bclCommand)
```

### 4. Analyzing Run Metrics
After bcl2fastq has finished, use `basecallQC` to parse the XML output files (`ConversionStats.xml` and `DemultiplexingStats.xml`).

```r
# Run the full pipeline to create a basecallQC object
# outDir should point to the bcl2fastq output directory containing the 'Stats' folder
bclQC <- basecallQC(bcl2fastqparams, sampleSheet = "SampleSheet_cleaned.csv")

# Extract specific metrics if needed
bclMetrics <- baseCallMetrics(bcl2fastqparams)
demuxMetrics <- demultiplexMetrics(bcl2fastqparams)
```

### 5. Visualization and Reporting
The package includes functions to generate summary tables and plots for quality control.

```r
# Generate interactive HTML tables
summaryConvStatsTable(bclQC, output = "html")
summaryDemuxTable(bclQC, output = "html")

# Plotting
passFilterBar(bclQC, groupBy = "Sample", metricToPlot = "Yield")
passFilterTilePlot(bclQC, metricToPlot = "Yield")
demuxBarplot(bclQC, groupBy = "Sample")

# Generate a comprehensive HTML report
reportBCL(bclQC)
```

## Tips and Best Practices
- **Sample Sheet Versions**: If you have an old Sample Sheet (bcl2fastq < 1.8.9), `validateBCLSheet` will automatically convert it to the new format, splitting combined index columns into `Index` and `Index2`.
- **Base Masking**: The `createBasemasks` function is particularly useful for runs where the number of cycles sequenced does not match the length of the index used, as it handles the "I" and "N" masking logic automatically.
- **Directory Structure**: Ensure the `runDir` and `outDir` provided to `BCL2FastQparams` are correct, as the package looks for specific subdirectories (like `Data/Intensities/BaseCalls`) to locate XML stats.
