# nwkit validate

## Overview

`nwkit validate` checks one or more Newick trees and writes a tab-separated report.

It can summarize structural issues such as duplicated leaf labels, empty labels, negative branch lengths, missing support values, non-binary topology, format ambiguity, and inconsistent leaf sets across a tree collection.

## Basic example

### Command

```
nwkit validate -i trees.nwk -o validate.tsv
```

The input may contain multiple Newick trees in one file.

### Example input (`trees.nwk`)

```
((A:-1,A:1):1,B:1);
((A:1,B:1):1,(C:1,E:1):1);
```

### Example output excerpt (`validate.tsv`)

| tree\_id | status | num\_duplicate\_leaf\_names | num\_negative\_branch\_nodes | leaf\_set\_matches\_first | issues |
| --- | --- | --- | --- | --- | --- |
| 1 | invalid | 1 | 1 | True | duplicate\_leaf\_names,negative\_branch\_length |
| 2 | invalid | 0 | 0 | False | leaf\_set\_mismatch |

The actual report also includes parser information, support counts, rooted/ultrametric flags, and other structural summary columns.

## Requiring rooted, ultrametric, binary trees

### Command

```
nwkit validate -i trees.nwk -o validate.tsv \
  --require_rooted yes \
  --require_ultrametric yes \
  --require_binary yes
```

## Checking support and parser-related issues

### Command

```
nwkit validate -i trees.nwk -o validate.tsv \
  --require_all_support yes \
  --require_unambiguous_format yes \
  --require_unquoted_names yes
```

## Species-aware validation

### Command

```
nwkit validate -i trees.nwk -o validate.tsv \
  --check_species yes
```

This checks whether tip labels can be parsed as species labels and whether repeated species labels are monophyletic.

## Other options

* `--require_same_leaf_set yes|no`: for multiple trees, require all trees to share the same leaf set
* `--require_same_rooting yes|no`: for multiple trees, require all trees to have the same rooted/unrooted status
* `--fail_on_issue yes|no`: exit with a non-zero status when any issue is reported
