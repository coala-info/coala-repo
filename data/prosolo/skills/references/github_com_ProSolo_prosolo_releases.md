[Version 0.6.0 of ProSolo](/ProSolo/prosolo/releases/tag/v0.6.0)

Includes further runtime optimisations, originally aimed at `prosic2`, but also benefiting `prosolo` (compare issue [#2](https://github.com/ProSolo/prosolo/issues/2)). It also incorporates `libprosic 0.7.2` bringing back the usage of non-binarised `mapq` for `SNV` and `None`-alt calls, versus the binarization of `mapq` (0 vs. maximum observed) for `indel` calling.

[Version 0.5.0 of ProSolo](/ProSolo/prosolo/releases/tag/v0.5.0)

New version of ProSolo, pulling in a lot of changes in libprosic (see the [libprosic changelog](https://github.com/PROSIC/libprosic/blob/master/CHANGELOG.md) infos for `libprosic 0.7.0` ), especially performance optimisations of more than an order of magnitude (also see issue [#2](https://github.com/ProSolo/prosolo/issues/2)), enabled by [perf flamegraphing](https://gist.github.com/dlaehnemann/df31787c41bd50c0fe223df07cf6eb89) of the tool. Also, FDR control is now simplified and directly outputs filtered BCF files.
