# bioconductor-pathostat

## Overview
PathoStat is a Bioconductor package designed for the statistical analysis of metagenomic data. It provides a comprehensive suite of tools for processing microbiome abundance data, typically starting from PathoScope outputs or similar taxonomic profiling reports. While it features a Shiny-based interactive interface, it also provides the underlying R infrastructure for data manipulation and statistical visualization.

## Core Workflow

### 1. Installation and Loading
```r
if (!requireNamespace("BiocManager", quietly=TRUE))
    install.packages("BiocManager")
BiocManager::install("PathoStat")

library(PathoStat)
```

### 2. Launching the Interactive Interface
The primary way to use PathoStat is through its integrated Shiny application, which handles the end-to-end analysis workflow.
```r
runPathoStat()
```

### 3. Data Input and Objects
PathoStat typically works with `Phyloseq` objects or PathoScope generated report files.
- **PathoScope Reports**: The package is optimized to ingest `.csv` or `.txt` reports generated by the PathoScope pipeline.
- **Metadata**: Ensure your sample metadata (mapping file) is prepared with categorical variables for group comparisons (e.g., Treatment vs. Control).

### 4. Key Analysis Capabilities
- **Data Filtering**: Remove low-abundance taxa or samples with low sequencing depth.
- **Relative Abundance**: Generate stacked bar plots and heatmaps to visualize taxonomic composition across samples.
- **Diversity Analysis**:
    - **Alpha Diversity**: Calculate and plot Shannon, Simpson, or Observed richness indices.
    - **Beta Diversity**: Perform PCoA or NMDS using various distance metrics (Bray-Curtis, Jaccard, etc.).
- **Differential Expression**: Identify significantly different taxa between groups using `DESeq2` or `edgeR` integrations.
- **Biomarker Identification**: Use statistical methods to find specific taxa that characterize different biological states.

## Tips for Success
- **Taxonomic Levels**: PathoStat allows you to aggregate data at different levels (Kingdom, Phylum, Class, Order, Family, Genus, Species). Always verify which level is most appropriate for your biological question.
- **Normalization**: Before performing differential expression or diversity analysis, ensure your data is appropriately normalized (e.g., rarefaction or log-transformation) within the tool's settings.
- **Visualization**: Use the "Multiple species boxplot" feature to drill down into specific taxa identified as significant in the DE analysis.
