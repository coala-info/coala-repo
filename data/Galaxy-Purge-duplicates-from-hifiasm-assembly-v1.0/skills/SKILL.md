# purge-duplicates-from-hifiasm-assembly-v10

## Overview

This Galaxy workflow is designed to refine genome assemblies produced by hifiasm by identifying and removing haplotypic duplications. Using the [purge_dups](https://github.com/dfguan/purge_dups) tool suite, it processes primary assemblies to ensure a more accurate representation of a collapsed haploid genome, which is essential for downstream analysis in diploid or polyploid organisms.

The pipeline begins by mapping raw HiFi reads back to the assembly using [minimap2](https://github.com/lh3/minimap2) to calculate read depth and coverage statistics. These metrics are used to determine optimal cutoffs for identifying redundant sequences. The workflow then executes a series of `purge_dups` steps to identify overlaps, calculate coverage histograms, and generate a BED file of regions to be purged.

The final outputs include a cleaned haploid assembly, a set of purged sequences, and comprehensive [Fasta Statistics](https://github.com/galaxyproject/tools-iuc/tree/master/tools/fasta_stats) to evaluate the assembly quality. For detailed configuration and usage recommendations, refer to the [Genome assembly with hifiasm on Galaxy Australia](https://australianbiocommons.github.io/how-to-guides/genome_assembly/hifi_assembly) guide.

| 0 | HiFiASM 1o assembly | data_input |  |
| 1 | HiFi reads as FASTQ | data_input |  |


Ensure your primary assembly is in FASTA format and your HiFi reads are provided as FASTQ files, ideally organized into a dataset collection for efficient processing. Verify that the assembly input corresponds to the primary contigs (p_ctg) generated by hifiasm to ensure accurate overlap purging. For automated execution and parameter configuration, you can initialize your environment using `planemo workflow_job_init` to generate a `job.yml` file. Detailed instructions on data preparation and parameter tuning are available in the accompanying README.md. Always consult the linked Australian BioCommons guide for specific recommendations on handling large-scale HiFi datasets.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Purge_duplicates_from_hifiasm_assembly_v1.0.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Purge_duplicates_from_hifiasm_assembly_v1.0.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Purge_duplicates_from_hifiasm_assembly_v1.0.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Purge_duplicates_from_hifiasm_assembly_v1.0.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Purge_duplicates_from_hifiasm_assembly_v1.0.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
