## [Introduction](#id1)

Debarcer facilitates many different kinds of UMI library preparations. To specify your own library preps, a small config file with entries corresponding to each specific library prep must be created. A sample file is located in `config/library_prep_types.ini` and it is recommended that you add all future library preps to this file. Each entry looks like this:

```
; One library entry
[LIBRARY_NAME]
INPUT_READS=3
OUTPUT_READS=2
UMI_LOCS=2
UMI_LENS=10
SPACER=FALSE
```

## [Fields](#id2)

### [INPUT\_READS](#id3)

This is an integer between 1 and 3 representing the number of fastq files. For single read libraries, this would typically be one file, and for paired-end libaries this would typically be two. In some cases, additional fastq files containing only the UMI are used, in which case a single read library could have two input reads and a paired-end library could have three.

### [OUTPUT\_READS](#id4)

This is simply the number of reads after reheadering. A single read library will have one output read and a paired-end library will have two.

### [UMI\_LOCS](#id5)

This is a comma-separated list of integers specifying the reads which contain UMIs. For example, if the UMI is located only on r1, the value of `UMI_LOCS` should be `1`. If there are two UMIs on both r1 and r2, `UMI_LOCS` should be `1,2`.

### [UMI\_LENS](#id6)

This is a comma-separated list of integers corresponding to `UMI_LOCS` that specifies the bp length of each UMI. For example, if there are 10bp UMIs on r1 and r2, `UMI_LOCS` should be `1,2` and `UMI_LENS` should be `10,10`.

### [SPACER](#id7)

A boolean. Set this to `TRUE` if spacers are used in your library prep, and `FALSE` otherwise.

### [SPACER\_SEQ (optional)](#id8)

If a spacer is used, this will be set to the base sequence of your spacer. For example, if there is a 2bp spacer with sequence `GT`, then `SPACER_SEQ` should be `GT`.
