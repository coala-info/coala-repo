# scnanoseq

## Overview
The scnanoseq pipeline provides a best-practice bioinformatics workflow for analyzing single-cell or single-nuclei RNA-seq data generated by Oxford Nanopore Technologies, specifically optimized for R10.4 flow cells and Q20+ chemistry. It addresses the unique challenges of long-read single-cell data by performing barcode detection and UMI identification directly from Nanopore reads without requiring supplemental Illumina data.

The workflow transforms raw FASTQ files into filtered, deduplicated, and quantified feature-barcode matrices. These outputs are formatted for immediate ingestion into downstream analysis packages such as Seurat or Scanpy for clustering and differential expression studies.

## Data preparation
The pipeline requires a CSV samplesheet and specific reference files depending on the chosen quantification method.

### Samplesheet
The samplesheet must contain three columns: `sample`, `fastq`, and `cell_count`. Technical replicates (multiple FASTQ files for the same sample) are automatically merged.

```csv
sample,fastq,cell_count
CONTROL_REP1,sample1_L001.fastq.gz,5000
CONTROL_REP1,sample1_L002.fastq.gz,5000
TREATMENT_REP1,sample2.fastq.gz,3000
```

### Reference files
*   **GTF**: A gene annotation file is mandatory for all runs.
*   **Genome FASTA**: Required if using the `isoquant` quantifier.
*   **Transcriptome FASTA**: Required if using the `oarfish` quantifier.
*   **Whitelist**: Optional; the pipeline provides default 10X whitelists based on the `--barcode_format` parameter (e.g., `10X_3v3`).

## How to run
Run the pipeline using the `nextflow run` command. You must specify the input samplesheet, an output directory, the barcode format, the deduplication tool, and the quantifier.

```bash
nextflow run nf-core/scnanoseq \
    -r 1.1.0 \
    -profile docker \
    --input samplesheet.csv \
    --outdir ./results \
    --gtf genes.gtf \
    --genome_fasta genome.fasta \
    --barcode_format 10X_3v3 \
    --dedup_tool umitools \
    --quantifier isoquant
```

For large datasets, use the `--split_amount` parameter (e.g., `500000`) to split FASTQ files for faster parallel processing of barcode extraction and correction. Use `-resume` to restart a run from the last successful step.

## Outputs
Results are saved in the directory specified by `--outdir`.

*   **Feature-Barcode Matrices**: Located in quantifier-specific folders (`isoquant/` or `oarfish/`), these matrices are the primary deliverables for downstream analysis.
*   **MultiQC Report**: A summary report (`multiqc/multiqc_report.html`) aggregating QC metrics from FastQC, NanoPlot, BLAZE, and alignment steps.
*   **Alignment Files**: BAM files from minimap2, including tagged and deduplicated versions.
*   **Seurat Objects**: Preliminary QC objects if Seurat processing is not skipped.

For a detailed description of all output files, refer to the [official output documentation](https://nf-co.re/scnanoseq/output).
