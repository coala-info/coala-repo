A general purpose Python data validator. Schema format based on JSON Schema Proposal ( http://json-schema.org ) Contains code derived from jsonschema, by Ian Lewis and Yusuke Muraoka. Usage JSON documents and schema must first be loaded into a Python dictionary type before it can be validated. Parsing a simple JSON document: &gt;&gt;&gt; import validictory &gt;&gt;&gt; &gt;&gt;&gt; validictory.validate("something", {"type":"string"}) Parsing a more complex JSON document: &gt;&gt;&gt; import json &gt;&gt;&gt; import validictory &gt;&gt;&gt; &gt;&gt;&gt; data = json.loads('["foo", {"bar":["baz", null, 1.0, 2]}]') &gt;&gt;&gt; schema = { ... "type":"array", ... "items":[ ... {"type":"string"}, ... {"type":"object", ... "properties":{ ... "bar":{ ... "items":[ ... {"type":"string"}, ... {"type":"any"}, ... {"type":"number"}, ... {"type":"integer"} ... ] ... } ... } ... } ... ] ... } &gt;&gt;&gt; validictory.validate(data,schema) Catch ValueErrors to handle validation issues: &gt;&gt;&gt; import validictory &gt;&gt;&gt; &gt;&gt;&gt; try: ... validictory.validate("something", {"type":"string","minLength":15}) ... except ValueError, error: ... print(error) ... Length of value 'something' for field '_data' must be greater than or equal to 15 You can read more in the official documentation at Read the Docs .
