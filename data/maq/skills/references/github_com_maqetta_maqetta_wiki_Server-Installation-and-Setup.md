# Server Installation and Setup

## Prerequisites

* Java 1.6 or higher
* [daemonize](http://software.clapper.org/daemonize/) (if using the maqetta.daemonize.sh start script)

## Installation

### Linux Setup

This is an example on how to install and setup the Maqetta server to run as a Linux service. Due to differences between the many Linux distributions, you may need to make changes appropriate to your distro.

1. Download the [latest Maqetta release](http://maqetta.org/downloads) and install to `/usr/local/maqetta/latest`.
2. Save the [maqetta.daemonize.sh](https://gist.github.com/2822045#file-maqetta-daemonize-sh) file to `/etc/init.d/maqetta`.
   * Make script executable: `chmod a+x /etc/init.d/maqetta`
   * Setup as a system service:

     ```
       # on Ubuntu:
       sudo update-rc.d maqetta defaults
       # on CentOS:
       sudo chkconfig --add maqetta
       sudo chkconfig maqetta on
     ```
3. Save [maqetta.conf](https://gist.github.com/2822045#file-maqetta-conf) to `/etc/maqetta.conf`.
4. Edit *maqetta.conf* with the appropriate configuration (see "Configuration" section below).
5. Start Maqetta by running `service maqetta start` as *root* user and load **http://[hostname:port]/maqetta/** in your browser.

### Install in a Servlet Container (i.e. Apache Tomcat)

1. Download the [WAR file for the latest release](http://maqetta.org/downloads) and deploy to Tomcat.
   * NOTE: If you are updating from a previous release of the maqetta.war, please delete the `[tomcat]/work/Catalina/localhost/maqetta/` directory -- Tomcat doesn't seem to always update this during a new deployment, and continues to serve up old code.
2. Install the configuration files:
   * Copy the *maqetta.conf* file (in the "setup" directory in the downloaded archive) to the Tomcat configuration directory (`[tomcat]/conf/Catalina/localhost/`).
   * Copy the *siteConfig* directory (also in "setup") somewhere safe. Alongside *maqetta.conf* is a good location.
3. Edit *maqetta.conf* with the appropriate configuration (see "Configuration" section below).
4. Set Tomcat environment variables: The properties `osgi.instance.area` and `orion.core.configFile` must be set in the environment when running Tomcat. This can be done a couple of ways (update paths to match your setup):
   * Run the following command before running Tomcat:

     ```
       export CATALINA_OPTS="-Dorion.core.configFile=/PATH/TO/TOMCAT/conf/Catalina/localhost/maqetta.conf -Dosgi.instance.area=/tmp/maqetta/users"
     ```
   * If starting Tomcat from a script (catalina.sh, daemon.sh, etc), set CATALINA\_OPTS directly in those files.
5. Recommended: edit the `conf/server.xml` file in the Tomcat installation to [use UTF-8 encoding of URLs](http://wiki.apache.org/tomcat/FAQ/CharacterEncoding#Q2) to match the browser. Without this, non-ascii characters in filenames will be read incorrectly. Set the URIEncoding attribute on the element in server.xml as follows: `URIEncoding="UTF-8"`
6. Start Tomcat and load **http://[hostname:port]/maqetta/maqetta/** in your browser.

## Configuration

The Maqetta server is configured using the `/etc/maqetta.conf` file. Please read the comments in that file for information on each option.

Here are some of the more common options:

* `maqetta.user`: The user under which the Maqetta server will run.
* `maqetta.baseDirectory`: The directory where users' workspaces are stored. Must be owned by same user that runs the Maqetta server instance.
* `maqetta.siteConfigDirectory`: The location of the "siteConfig" data directory, copied from the WAR.
* `orion.auth.name`: Sets the user authentication method that the server will use. For LDAP support, set this to "ldaplogin". For more info, read the [Orion Server docs](http://wiki.eclipse.org/Orion/Server_admin_guide#Authentication_type).
* `maqetta.auth.ldap.*`: Many options for controlling LDAP login. Follow the comments in "maqetta.conf" to set correctly.
* `smtp.mailServer`: The SMTP mail server used by Maqetta, mainly for sending review & commenting emails. If there is a mail transfer agent (MTA) running on the same machine as the Maqetta server, set this to "localhost" -- if no port is specified, defaults to "25".

## Debugging

If you run into any issues, here are several things you can do to get extra information which may help you debug the problem:

* `maqetta.serverDebug`: Set this param to "true" in *maqetta.conf*. It will output extra information to standard output.
* **OSGi console**: For the standalone build, add "-console" to `APP_ARGS` in `/etc/init.d/maqetta`; for the WAR file, add that string to the `commandline` param in `WEB-INF/web.xml`. This will enable the OSGi console, which allows you to debug OSGi bundle loading issues.
