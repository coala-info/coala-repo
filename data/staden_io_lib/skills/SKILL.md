# staden_io_lib

## Overview
The Staden io_lib (often distributed as `libstaden-read`) is a specialized library and toolset designed for the efficient handling of DNA sequence data. While it serves as a foundational C library for many bioinformatics applications, its primary user-facing utility is the `scramble` tool. This tool is highly effective for converting between SAM, BAM, and CRAM formats, offering advanced control over CRAM compression profiles and versions (including support for CRAM 3.1 and experimental 4.0). It also provides unique support for reading and writing various chromatogram "trace" formats used in Sanger sequencing.

## Installation
The easiest way to install the library and its associated tools is via Bioconda:

```bash
conda install bioconda::staden_io_lib
```

## Common CLI Patterns with Scramble
The `scramble` utility is the primary command-line interface for file conversion and CRAM optimization.

### Basic Format Conversion
Convert between sequence formats by specifying the output format with `-O`.

```bash
# Convert BAM to CRAM (defaults to CRAM 3.0)
scramble -O cram input.bam output.cram

# Convert CRAM to BAM
scramble -O bam input.cram output.bam

# Convert SAM to BAM with specific compression level (1-9)
scramble -O bam -9 input.sam output.bam
```

### CRAM Version Selection
Use the `-V` flag to target specific GA4GH CRAM specifications.

```bash
# Use CRAM 3.1 (improved compression over 3.0)
scramble -O cram -V3.1 input.bam output.cram

# Use experimental CRAM 4.0 (for evaluation of 64-bit sizes and new codecs)
scramble -O cram -V4.0 input.bam output.cram
```

### Performance Optimization Profiles
The `-X` flag allows you to select a profile that balances file size against CPU time.

*   **fast**: Minimal compression, very fast encoding/decoding.
*   **normal**: The default balance.
*   **small**: Better compression using bzip2/arithmetic codecs; slower.
*   **archive**: Maximum compression for long-term storage; slowest.

```bash
# Create a highly compressed archive CRAM
scramble -O cram -V3.1 -X archive input.bam output.cram

# Create a CRAM optimized for fast processing
scramble -O cram -V3.1 -X fast input.bam output.cram
```

## Expert Tips
*   **Library Usage**: For C/C++ development, use the `scram_*()` API for SAM/BAM/CRAM and `read_reading()` for trace files. Include `<io_lib/Read.h>` for the generic Read structure.
*   **Codec Awareness**: CRAM 3.1 and 4.0 utilize advanced codecs like rANS++, tok3, and fqzcomp. Ensure downstream tools are compatible with these versions before converting production data.
*   **Validation**: Use `scram_flagstat -b` to check the integrity and basic statistics of a BAM/CRAM file generated by the library.
*   **Trace Files**: This library is one of the few modern tools that still provides robust support for legacy formats like ALF, ABI, and ZTR.
