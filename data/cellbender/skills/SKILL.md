# cellbender

## Overview
CellBender is a deep learning-based tool designed to clean up single-cell transcriptomics data. It specifically targets "ambient RNA"—the background soup of transcripts found in droplets that don't actually contain a cell—and barcode swapping artifacts. By removing this noise, it helps in identifying true cell populations and clearer gene expression profiles. It is particularly effective for datasets where high background noise obscures low-abundance transcripts or makes different cell types look more similar than they actually are.

## Installation
CellBender can be installed via pip or conda. It is recommended to use a dedicated environment.

```bash
# Via pip
pip install cellbender

# Via conda
conda install -c bioconda cellbender
```

## Core Workflow: remove-background
The primary module is `remove-background`. This module takes a raw count matrix and produces a denoised version.

### Basic Command Pattern
```bash
cellbender remove-background \
    --cuda \
    --input raw_feature_bc_matrix.h5 \
    --output denoised_matrix.h5
```

### Key Arguments
- `--input`: Path to the raw UMI-based count matrix. Supports `.h5` (10x Genomics format) and AnnData (`.h5ad`) files.
- `--output`: Path where the denoised `.h5` file will be saved.
- `--cuda`: Flag to enable GPU acceleration. Highly recommended as CPU-only runs are significantly slower.
- `--expected-cells`: (Optional) The number of cells expected in the experiment.
- `--total-droplets`: (Optional) The number of droplets to include in the model (should be larger than expected cells).
- `--fpr`: (Optional) Target false positive rate. Default is 0.01.

## Expert Tips and Best Practices

### Input Selection
Always use the **raw** (unfiltered) count matrix from your preprocessing pipeline (e.g., `raw_feature_bc_matrix.h5` from CellRanger). Do not use the "filtered" matrix, as CellBender needs the empty droplets to model the background ambient RNA profile.

### Version Safety
Avoid using version **v0.3.1** due to a known integer overflow bug that causes incorrect (sometimes negative) output counts. Use **v0.3.0** or **v0.3.2+**.

### Salvaging Runs with Checkpoints
If a run is interrupted or if you need to re-run the count estimation without retraining the model, use the checkpoint file:
```bash
cellbender remove-background \
    --input raw_matrix.h5 \
    --output new_output.h5 \
    --checkpoint path/to/ckpt.tar.gz \
    --force-use-checkpoint
```
The `--force-use-checkpoint` flag is particularly useful if you are using a checkpoint generated by a slightly different version of the software.

### Memory Management
If you encounter Out-of-Memory (OOM) errors on your GPU:
1. Reduce the `--low-count-threshold`.
2. Decrease the `--total-droplets` if it was set excessively high.
3. Ensure you are not running other GPU-intensive processes simultaneously.

### Output Files
CellBender produces several files. The most important is `[output_name].h5`, which contains the denoised count matrix. It also generates a PDF report; always inspect the "Log Probabilities" and "Rank-BCC" plots in this report to ensure the model has successfully distinguished cells from empty droplets.
