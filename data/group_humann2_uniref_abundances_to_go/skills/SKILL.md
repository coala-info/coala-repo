# group_humann2_uniref_abundances_to_go

## Overview
The `group_humann2_uniref_abundances_to_go` tool is a specialized utility designed to transform granular UniRef50 gene family abundances into broader, more interpretable Gene Ontology (GO) slim terms. While HUMAnN2 provides basic regrouping capabilities, this tool specifically targets the "metagenomics" GO slim subset (developed by Jane Lomax and the InterPro group) to provide a functional summary better suited for community-level microbiota analysis. It automates a complex workflow involving HUMAnN2 regrouping, GoaTools mapping, and output formatting to generate three distinct abundance files for the three primary GO ontologies.

## CLI Usage and Patterns

### Basic Execution
The primary entry point is the shell script. You must specify the input UniRef50 abundance file and three separate output paths for the different GO categories.

```bash
group_humann2_uniref_abundances_to_GO.sh \
    -i humann2_gene_families_abundance.tsv \
    -m molecular_function_abundance.tsv \
    -b biological_process_abundance.tsv \
    -c cellular_component_abundance.tsv
```

### Command Line Options
- `-i`: Input file containing UniRef50 gene family abundances (standard HUMAnN2 output).
- `-m`: Output file path for Molecular Function (MF) abundances.
- `-b`: Output file path for Biological Process (BP) abundances.
- `-c`: Output file path for Cellular Component (CC) abundances.
- `-h`: Display help and usage information.

## Expert Tips and Best Practices

### Input Preparation
- **UniRef Version**: Ensure your input is specifically UniRef50. While the tool is designed for UniRef50, version 1.3.0+ includes improved support for taxon identifiers and `UNMAPPED` entries.
- **File Format**: The tool typically expects tab-separated or comma-separated values as generated by the `humann2` main pipeline.

### Dependency Management
- **Conda Installation**: It is highly recommended to install via Bioconda (`conda install -c bioconda group_humann2_uniref_abundances_to_GO`) to ensure that `GoaTools` (>0.6.4) and `HUMAnN2` dependencies are correctly linked in the environment path.
- **Manual Setup**: If running from source, you must run `install_dependencies.sh` to set up the required virtual environment and fetch the necessary GO mapping files.

### Interpreting Results
- **GO Slim Subset**: Be aware that this tool uses a specific "metagenomics" slim. If your research requires a different GO slim (e.g., yeast or plant-specific), this tool's internal mapping logic may not be appropriate without modification of the source mapping files.
- **Taxonomic Stratification**: If your input file contains taxonomically stratified data (e.g., `UniRef50_XYZ|g__Genus.s__Species`), the tool (v1.3.0+) can handle these identifiers and transfer them to the resulting GO abundance tables.

### Troubleshooting
- **Missing Terms**: If many abundances are not mapping, verify that your UniRef50 IDs are standard and that the `map_infogo1000_uniref50` correspondence file is present in the tool's data directory.
- **GoaTools Version**: Ensure `GoaTools` is version 0.6.4 or higher; older versions may cause failures during the GO term mapping phase.
