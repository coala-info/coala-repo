# comparative-gene-analysis

## Overview

This workflow is designed for the comparative analysis of genes across annotated genomes, supporting research in evolutionary biology and genome annotation. It is particularly relevant for projects associated with the [Vertebrate Genomes Project (VGP)](https://vertebrategenomesproject.org/) and follows methodologies outlined by the [Galaxy Training Network (GTN)](https://training.galaxyproject.org/). The pipeline facilitates the identification and comparison of orthologous sequences by integrating structural annotation data with high-throughput sequence alignment.

The process begins by taking exon data and ORF predictions (from ORFipy) and aligning them against a reference database using [Diamond](https://github.com/bbuchfink/diamond). A series of data transformation steps—including regex-based filtering, coordinate intersection, and column manipulation—are employed to refine gene models and isolate specific genomic regions of interest. These steps ensure that the comparative analysis is performed on high-quality, filtered datasets.

In the final stages, the workflow performs multiple sequence alignment using [MAFFT](https://mafft.cbrc.jp/alignment/software/) to assess sequence conservation. It then utilizes [RapidNJ](https://birc.au.dk/software/rapidnj/) to calculate phylogenetic distances and reconstruct neighbor-joining trees. The resulting outputs provide a comprehensive view of gene relationships, offering both distance matrices and formatted tabular data for downstream evolutionary interpretation.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Exons | data_input | Amino acid sequences of CDS exons from the gene of interest |
| 1 | Diamond makedb | data_collection_input | Diamond DB created from ORF predicted from genomes used in the analysis |
| 2 | ORFipy BED | data_collection_input | BED dataset containing information about ORFs predicted in genomes of interest |


Ensure the Exons input is provided as a BED or GTF file, while the Diamond database and ORFipy BED files must be organized into data collections to enable efficient batch processing across the comparative steps. Using collections for these inputs is essential for the downstream tools to correctly map and compare multiple gene sets in parallel. For comprehensive instructions on data formatting and specific parameter settings, please consult the README.md file. You may also use `planemo workflow_job_init` to create a `job.yml` for streamlined execution and testing.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run comparative-gene-analysis.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run comparative-gene-analysis.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run comparative-gene-analysis.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init comparative-gene-analysis.ga -o job.yml`
- Lint: `planemo workflow_lint comparative-gene-analysis.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
