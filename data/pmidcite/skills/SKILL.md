# pmidcite

## Overview

`pmidcite` is a specialized command-line utility that bridges PubMed IDs (PMIDs) with NIH Open Citation Collection (NIH-OCC) data. It allows researchers to bypass manual "Cited by" searches on web interfaces by programmatically fetching citation counts, reference lists, and performance metrics directly from the terminal. It is particularly useful for bibliometric analysis, identifying high-impact papers within a specific field, and automating the discovery phase of systematic reviews.

## Core CLI Usage

The primary command for this tool is `icite`.

### Basic Citation Lookup
To get a quick summary of a paper's impact, including citation counts, reference counts, and author numbers:
```bash
icite -H <PMID>
```

### Citation Snowballing
*   **Forward Snowballing (Cited By):** Find all papers that have cited a specific PMID.
    ```bash
    icite <PMID> -c | sort -k6 -r
    ```
*   **Backward Snowballing (References):** Find all papers referenced by a specific PMID.
    ```bash
    icite <PMID> -r | sort -k6 -r
    ```
*Note: Sorting by `-k6 -r` typically organizes the output by the NIH percentile or citation rank.*

### Batch Processing
For large-scale literature searches, process a list of PMIDs stored in a text file:
```bash
icite -i input_pmids.txt -o citation_results.txt
```

### Summarizing Results
Use the `sumpaps` command to analyze a group of citations generated by `icite`. This provides a distribution of the "NIH Percentile" (0-4), where 4 represents top-performing papers:
```bash
sumpaps citation_results.txt
```

## Expert Tips and Best Practices

*   **NIH Percentile Grouping:** Pay attention to the `G` column (Group) and `%` column (Percentile). `pmidcite` uses a novel grouping (0-4) to highlight better-performing papers. Groups 2, 3, and 4 are generally considered high-impact.
*   **Handling New Papers:** New publications appear at the beginning of sorted lists regardless of citation count to ensure researchers find the latest discoveries.
*   **API Keys:** For high-volume searches, obtain an NCBI API key to avoid rate limiting. Configure this in your environment or via the `.pmidciterc` file.
*   **Configuration:** Generate a default configuration file to manage persistent settings:
    ```bash
    icite --generate-rcfile > ~/.pmidciterc
    ```
*   **Data Filtering:** Use standard Unix tools like `grep` and `sort` on the output files. For example, to find only the top-performing papers in a batch result:
    ```bash
    grep "TOP" citation_results.txt | sort -k6
    ```



## Subcommands

| Command | Description |
|---------|-------------|
| icite | Run NIH's iCite given PubMed IDs |
| sumpaps | Summarize NIH's citation data on a set(s) of papers |
