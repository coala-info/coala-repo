# artic-tools

## Overview

`artic-tools` is a high-performance C++ toolkit designed to handle the specialized file processing requirements of the ARTIC bioinformatic pipeline. It is primarily used in viral genomics (such as SARS-CoV-2 or Ebola sequencing) to manage the relationship between primer schemes and sequencing data. The tool provides deterministic, efficient methods for preparing reference data and cleaning alignment files to ensure that primer-derived artifacts do not interfere with variant calling.

## Core CLI Usage

The `artic-tools` suite is typically accessed via a single binary with specific subcommands.

### Primer Scheme Management
Use these commands to retrieve and verify the integrity of amplicon primer schemes.

*   **Download Schemes**: Retrieve standardized primer schemes and reference sequences directly from the ARTIC repository.
    ```bash
    artic-tools get_scheme <scheme_name>
    ```
*   **Validation**: Check a primer scheme (BED format) for overlaps, formatting errors, or inconsistencies that could break downstream pipelines.
    ```bash
    artic-tools validate <scheme.bed>
    ```

### Alignment Processing (Softmasking)
Softmasking is a critical step in the ARTIC workflow. It identifies read segments that originated from primers and masks them in the BAM file so they are not used for variant calling, while still preserving the full read for visualization.

*   **Masking Primers**:
    ```bash
    artic-tools mask --pe --scheme <scheme.bed> <input.bam> <output.masked.bam>
    ```
    *   **Tip**: Use the `--pe` flag for paired-end data to ensure both mates are handled correctly relative to the primer coordinates.

### Variant Filtering
Refine VCF outputs to remove low-confidence calls or artifacts common in amplicon sequencing.

*   **VCF Check**: Validate VCF records against the primer sites to identify potential false positives at primer binding locations.
    ```bash
    artic-tools vcf_check --vcf <input.vcf> --scheme <scheme.bed>
    ```
*   **Filtering**: Apply quality filters to VCF files generated by the pipeline.
    ```bash
    artic-tools filter <input.vcf> <output.filtered.vcf>
    ```

## Best Practices and Expert Tips

*   **Scheme Consistency**: Always ensure the version of the primer scheme used for `masking` matches the version used during the initial alignment. Using a V3 scheme to mask V4 data will result in significant variant calling artifacts.
*   **Case Sensitivity**: The scheme downloader is case-insensitive, but it is best practice to use the exact naming convention found in the ARTIC field guide (e.g., `SARS-CoV-2`).
*   **Softmasking vs. Hard-clipping**: Prefer `artic-tools mask` (softmasking) over tools that hard-clip reads. Softmasking allows you to retain the original sequence data in the BAM file, which is essential for troubleshooting problematic samples in genome browsers like IGV.
*   **Empty VCF Handling**: Be aware that older versions of the tool may have issues with empty VCF files. Ensure you are using version 0.3.1 or later to avoid segfaults when processing samples with no detected variants.
*   **Performance**: Because `artic-tools` is written in C++ and utilizes HTSlib, it is significantly faster than equivalent Python scripts. Use it in high-throughput production environments where processing time per sample is a constraint.
