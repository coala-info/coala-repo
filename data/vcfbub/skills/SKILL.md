# vcfbub

## Overview

`vcfbub` is a specialized utility for pangenomics designed to clean up complex VCF outputs from the Variation Graph (vg) toolkit. When `vg deconstruct` is run with the `--all-snarls` flag, it produces a hierarchy of nested variants. `vcfbub` simplifies this hierarchy by filtering based on tree depth (level) and allele length. This process ensures a set of non-overlapping sites that are more compatible with standard genomic tools while retaining high-quality nested information that would otherwise be lost in a simple top-level filter.

## Installation

The tool is available via Bioconda:

```bash
conda install bioconda::vcfbub
```

## Core CLI Usage

The tool requires a VCF file as input and typically outputs to stdout.

### Basic Level Filtering
To keep only the sites rooted on the main linear axis of the pangenome graph (top-level bubbles):

```bash
vcfbub --max-level 0 input.vcf > filtered.vcf
```

### Size-Based Bubble Popping
To remove large, complex structural variants (SVs) but keep the smaller variants nested inside them, use the reference length filter. This is the most common "bubble popping" workflow:

```bash
# Keep top-level sites (-l 0) unless they are >10kb, 
# in which case "pop" them and keep their nested children.
vcfbub -l 0 -r 10000 input.vcf > filtered.vcf
```

### Parameter Reference
- `-l, --max-level`: The maximum depth in the snarl tree to retain. Level 0 is the root.
- `-r, --max-ref-length`: Maximum length of the reference allele. Sites exceeding this are popped, and their children are promoted.
- `-a, --max-allele-length`: Maximum length of any allele (reference or alternative).

## Expert Tips and Best Practices

### 1. Upstream Requirements
`vcfbub` relies on specific metadata generated by `vg deconstruct`. Ensure your upstream command includes the `-a` (or `--all-snarls`) flag to expose the nesting information:
```bash
vg deconstruct -e -a -H '#' graph.vg > var.vcf
```

### 2. Handling Complex SVs
In pangenome graphs, large inversions or segmental duplications often appear as massive bubbles that are difficult to genotype or impute. By setting a `--max-ref-length` (e.g., 10kb or 100kb), you can remove these problematic "parent" sites while automatically retaining the SNPs and small indels nested within them.

### 3. Logic of Promotion
When a site is "popped" due to size constraints (`-r` or `-a`), `vcfbub` ensures that the output contains the bubbles nested inside that popped bubble, even if those children are at a level greater than the specified `--max-level`. This prevents the loss of genetic information in complex regions.

### 4. Downstream Compatibility
Standard VCF tools often struggle with the overlapping records produced by pangenome deconstruction. Running `vcfbub` is a critical preprocessing step before using the VCF for:
- Haplotype panel construction
- Imputation references (e.g., Beagle)
- Standard variant effect predictors
