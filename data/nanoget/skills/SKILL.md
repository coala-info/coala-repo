# nanoget

## Overview
The `nanoget` skill provides a streamlined workflow for retrieving read-level information from Oxford Nanopore Technologies (ONT) outputs. It standardizes disparate data sources—such as alignments, raw reads, and summary text files—into a consistent format. This is particularly useful for researchers building custom analysis pipelines or preparing data for tools like NanoPlot, as it handles the complexities of parsing rich FASTQ headers and multi-threaded BAM processing.

## Core Functions and Usage
`nanoget` is primarily used as a Python module to generate pandas DataFrames. Below are the standard patterns for data extraction:

### Processing Alignments (BAM)
Extract metrics from a sorted BAM file. This function supports multi-threading to speed up the parsing of large alignment files.
```python
import nanoget
df = nanoget.process_bam("input.bam", threads=4)
```

### Processing FASTQ Files
There are two primary ways to handle FASTQ data depending on the available metadata:

*   **Plain FASTQ**: Use for standard files without specialized ONT metadata.
    ```python
    df = nanoget.process_fastq_plain("reads.fastq.gz", threads=4)
    ```
*   **Rich FASTQ**: Use for files generated by MinKNOW or Albacore that contain additional metadata in the header.
    ```python
    df = nanoget.process_fastq_rich("rich_reads.fastq.gz")
    ```

### Processing Sequencing Summaries
Extract data from the `sequencing_summary.txt` file produced during basecalling (e.g., by Guppy or Albacore).
```python
df = nanoget.process_summary("sequencing_summary.txt", readtype='1D')
```

## Best Practices and Tips
*   **Compression Support**: `nanoget` natively handles compressed FASTQ files. You can pass `.gz`, `.bz2`, or `.bgz` files directly to the processing functions without manual decompression.
*   **Standardized Headers**: The resulting DataFrames use consistent column names (e.g., `lengths`, `quals`, `readIDs`) regardless of the input source, allowing you to concatenate data from BAM and FASTQ files easily.
*   **Feather Serialization**: For very large datasets, use the included script to serialize the extracted DataFrame to a Feather file. This significantly speeds up loading times for subsequent analysis steps.
*   **Thread Optimization**: When using `process_bam` or `process_fastq_plain`, match the `threads` argument to your available CPU cores to minimize processing time, especially for high-coverage datasets.
