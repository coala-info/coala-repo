# perl-archive-zip

## Overview
The `perl-archive-zip` skill provides a specialized interface for handling ZIP archives through the Perl `Archive::Zip` module. While standard system utilities like `zip` and `unzip` are designed for simple file-to-archive operations, this tool allows for complex manipulation of archive contents. It enables agents to treat ZIP files as objects, allowing for the insertion of virtual files from memory, renaming members without re-zipping, and precise error checking during the archival process.

## Common Usage Patterns

### Basic Archive Creation
To create a new ZIP file and add members from disk:
```perl
use Archive::Zip qw( :ERROR_CODES :CONSTANTS );

my $zip = Archive::Zip->new();
$zip->addFile('data.txt');
$zip->addDirectory('logs/');

unless ( $zip->writeToFileNamed('output.zip') == AZ_OK ) {
    die "Write error";
}
```

### Adding Content from Strings
Useful for creating "README" or metadata files on the fly without writing to a temporary file first:
```perl
my $string_member = $zip->addString('Generated by AI Agent', 'metadata.txt');
$string_member->desiredCompressionMethod( COMPRESSION_DEFLATED );
```

### Reading and Extracting
To read an existing archive and extract a specific file:
```perl
my $zip = Archive::Zip->new();
die "Read error" unless $zip->read('archive.zip') == AZ_OK;

# Extract a specific member by name
$zip->extractMember('data.txt', 'extracted_data.txt');
```

### Modifying Existing Archives
You can update a ZIP file by reading it, adding/removing members, and writing it back:
```perl
my $zip = Archive::Zip->new('existing.zip');
$zip->removeMember('old_file.log');
$zip->addFile('new_file.log');
$zip->overwrite();
```

## Expert Tips and Best Practices

- **Error Handling**: Always check return codes against `:ERROR_CODES` (e.g., `AZ_OK`). The module does not throw exceptions by default for most I/O operations.
- **Memory Management**: For very large archives, be aware that `Archive::Zip` keeps the directory structure in memory. If you are only extracting a single file from a massive archive, ensure the environment has sufficient overhead.
- **Compression Levels**: Use `desiredCompressionLevel()` to balance speed and size. Level 0 is no compression (Store), 1 is fastest, and 9 is best compression.
- **File Naming**: Always use forward slashes (`/`) as path separators within the ZIP internal structure to ensure cross-platform compatibility, regardless of the host OS.
- **Member Names**: When adding files, you can specify a different name for the file inside the archive: `$zip->addFile('local_path.txt', 'internal_name.txt')`.
