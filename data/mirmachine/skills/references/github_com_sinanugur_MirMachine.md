Installation To install this package with conda run: conda install mirmachine -c bioconda -c conda-forge Please add conda-forge as a channel. Installing via mamba is also strongly recommended for a faster installation. You can install mamba and later MirMachine like this: conda install mamba -c conda-forge mamba install mirmachine -c bioconda -c conda-forge Check if the installation works by calling the main script. MirMachine.py --help Note: You have to install dependencies if you prefer GitHub or PyPi installation. Quick start example Create a new directory and run MirMachine there after the installation. MirMachine will create the required directories while running. MirMachine.py -n Caenorhabditis -s Caenorhabditis_elegans --genome sample/genomes/ce11.fa --cpu 20 --model proto See our documentation for detailed explanations: https://mirmachine.readthedocs.io/ Options and Arguments Usage: MirMachine.py --node &lt;text&gt; --species &lt;text&gt; --genome &lt;text&gt; [--model &lt;text&gt;] [--evalue &lt;float&gt;] [--cpu &lt;integer&gt;] [--add-all-nodes|--single-node-only] [--unlock|--remove][--touch] [--dry] MirMachine.py --species &lt;text&gt; --genome &lt;text&gt; --family &lt;text&gt; [--model &lt;text&gt;] [--evalue &lt;float&gt;] [--cpu &lt;integer&gt;] [--unlock|--remove] [--touch] [--dry] MirMachine.py --node &lt;text&gt; [--add-all-nodes] MirMachine.py --print-all-nodes MirMachine.py --print-all-families MirMachine.py --print-ascii-tree MirMachine.py (-h | --help) MirMachine.py --version Arguments: -n &lt;text&gt;, --node &lt;text&gt; Node name. (e.g. Caenorhabditis) -s &lt;text&gt;, --species &lt;text&gt; Species name. (e.g. Caenorhabditis_elegans) -g &lt;text&gt;, --genome &lt;text&gt; Genome fasta file location (e.g. data/genome/example.fasta) -m &lt;text&gt;, --model &lt;text&gt; Model type: deutero, proto, combined [default: combined] -f &lt;text&gt;, --family &lt;text&gt; Run only a single miRNA family (e.g. Let-7). -e &lt;text&gt;, --evalue &lt;float&gt; Inclusion E-value. May inflate low quality hits. [default: 0.2] -c &lt;integer&gt;, --cpu &lt;integer&gt; CPUs. [default: 2] Options: -a, --add-all-nodes Move on the tree both ways. NOT required most of the time. -o, --single-node-only Run only on the given node for miRNA families. -p, --print-all-nodes Print all available node options and exit. -l, --print-all-families Print all available families in this version and exit. -t, --print-ascii-tree Print ascii tree of the tree file. -u, --unlock Rescue stalled jobs (Try this if the previous job ended prematurely). -r, --remove Clear all output files (this won't remove input files). -d, --dry Dry run. -h, --help Show this screen. --touch Touch output files (mark them up to date without really changing them). --version Show version. Output The MirMachine main executable will generate GFF annotations (filtered and unfiltered) and some other files. You will see results/predictions/ directory which contains: gff/ All predicted miRNA families. filtered_gff/ High confidence miRNA family predictions after bitscore filtering. (This file is what you need in most cases) fasta/ Both high and low confidence predictions in FASTA format.
