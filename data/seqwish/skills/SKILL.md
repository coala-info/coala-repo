# seqwish

## Overview
seqwish is a high-performance tool designed to "squish" sequences and their alignments into a unified variation graph. It implements a lossless conversion algorithm that transforms pairwise alignments into the nodes, edges, and paths of a pangenome graph. By utilizing disk-backed arrays and succinct data structures, it can handle massive datasets—such as those encountered in human pangenomics—that would otherwise exceed available RAM.

## Usage Patterns and Best Practices

### Basic Command Structure
The primary workflow involves providing a FASTA file of sequences and a PAF file of alignments to generate a GFA file.
```bash
seqwish -s sequences.fa -p alignments.paf -g output.gfa
```

### Input Requirements
- **Sequences (-s, --seqs)**: A FASTA or FASTQ file containing all sequences involved in the alignments.
- **Alignments (-p, --paf)**: Pairwise alignments, typically generated by tools like `wfmash` or `minimap2`. These alignments define which parts of the sequences are merged into single nodes in the graph.

### Performance and Resource Management
- **Disk-Backed Sorting**: seqwish performs heavy disk I/O. Ensure the directory where you run the command (or the designated temporary directory) has sufficient space for intermediate files, which can be several times the size of the input alignments.
- **Temporary Storage**: For faster execution, use high-speed storage for temporary files. If available, `/dev/shm` (RAM disk) can significantly speed up the sorting phases.
- **Multithreading**: Use the `-t` or `--threads` flag to specify the number of compute threads. The algorithm benefits from parallelization during the transitive closure and link-sorting phases.

### Output Formats
- **GFAv1**: The standard output format (`-g`). This is compatible with downstream pangenome tools like `vg`, `odgi`, and `GCSA2`.
- **Unitig Compression**: seqwish naturally produces single-base nodes. Downstream tools are often used to compress these into unitigs to reduce the identifier space and improve performance in analysis.

### Expert Tips
- **All-versus-all Alignments**: While seqwish can work with any alignment set, it is most commonly used with all-versus-all alignments of a set of related genomes to build a comprehensive pangenome.
- **Memory vs. Disk**: If you encounter performance bottlenecks, check disk latency. Because seqwish uses succinct rank/select dictionaries, its RAM footprint is relatively low, but its speed is highly dependent on disk throughput.
- **Validation**: After induction, the resulting GFA contains "Paths" (P-lines) that represent the original input sequences. You can verify the lossless nature of the graph by tracing these paths to reconstruct the input FASTA.
