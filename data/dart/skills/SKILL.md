# dart

## Overview

Dart (Divide-and-conquer Alignment of RNA-seq Reads) is a specialized mapper designed to handle the complexities of transcriptomic data. Unlike traditional aligners that extend seeds using dynamic programming, Dart utilizes a divide-and-conquer strategy, breaking reads into segments to identify optimal alignments and splice sites. It is particularly effective for identifying novel splice junctions and handling large-scale RNA-seq datasets with high accuracy and minimal computational overhead.

## Core Workflows

### 1. Genome Indexing
Before mapping, you must create a BWT-based index of the reference genome. Dart can use its internal indexing tool or indexes generated by BWA.

```bash
# Using Dart's internal tool
dart index reference.fa index_prefix

# Alternative using bwa (if dart index is unavailable)
bwa index -p index_prefix reference.fa
```

### 2. Read Mapping
Dart supports single-end, paired-end (separate files), and interleaved paired-end reads.

**Single-end mapping:**
```bash
dart -i index_prefix -f reads.fastq -o output.sam
```

**Paired-end mapping (separate files):**
```bash
dart -i index_prefix -f R1.fastq -f2 R2.fastq -o output.sam
```

**Paired-end mapping (interleaved file):**
```bash
dart -i index_prefix -f interleaved.fastq -p -o output.sam
```

**Direct BAM output:**
To save disk space and skip SAM-to-BAM conversion, use the `-bo` flag.
```bash
dart -i index_prefix -f R1.fq -f2 R2.fq -bo output.bam
```

## Parameter Optimization

### Performance and Threading
*   **Threads:** Use `-t` to specify CPU cores. The default is 16. For high-performance clusters, match this to your allocated slots.
*   **Memory:** Dart is efficient, but ensure your system has enough RAM to load the BWT index (typically ~4-8GB for the human genome).

### Alignment Sensitivity
*   **Mismatches:** Control the stringency of alignments with `-mis [INT]`.
*   **Repetitive Sequences:** Use `-max_dup [INT]` (range 100-10000) to limit how many repetitive fragments are processed.
*   **Unique Mapping:** Use the `-unique` flag to only output reads that map to a single location.

### Splice Junction Detection
*   **Junction Output:** Dart outputs predicted splice junctions to `junctions.tab` by default. Change this with `-j filename`.
*   **Discovery Mode:** Use `-all_sj` to detect all possible splice junctions regardless of the mapping quality (MAPQ) score.
*   **Intron Size:** The default maximum intron size is 500,000 bp. Adjust this with `-intron [INT]` for organisms with exceptionally long or short introns.

## Input/Output Specifications

*   **Reference:** Must be in FASTA format.
*   **Reads:** Supports FASTA or FASTQ. FASTQ files can be GZIP compressed (`.gz`), but FASTA files must be uncompressed.
*   **Case Sensitivity:** Read sequences should be in capital letters.
*   **Strandness:** For reads aligned to the reverse strand, Dart automatically converts them to the obverse strand in the SAM/BAM output.
