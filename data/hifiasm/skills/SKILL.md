# hifiasm

## Overview
Hifiasm is a high-performance assembler designed to produce phased, haplotype-resolved genome assemblies from PacBio HiFi reads. Unlike traditional assemblers that collapse heterozygosity into a single consensus sequence, hifiasm maintains the distinction between maternal and paternal haplotypes. It is highly efficient, capable of assembling a human genome in under a day, and supports hybrid workflows using Hi-C, trio-binning (via yak), and Oxford Nanopore (ONT) reads to achieve near-complete, telomere-to-telomere results without requiring external polishing tools.

## Common CLI Patterns

### Basic Assembly (HiFi only)
For standard diploid genomes with default duplication purging:
`hifiasm -o <output_prefix> -t <threads> <input_reads.fq.gz>`

### Homozygous or Inbred Genomes
Disable the built-in duplication purging to prevent over-purging of similar sequences in genomes with low heterozygosity:
`hifiasm -o <output_prefix> -t <threads> -l0 <input_reads.fq.gz>`

### Hi-C Integrated Phasing
Use paired-end Hi-C reads to guide the phasing of HiFi contigs and resolve switch errors:
`hifiasm -o <output_prefix> -t <threads> --h1 <hic_r1.fq.gz> --h2 <hic_r2.fq.gz> <input_reads.fq.gz>`

### Trio Binning
Requires parental k-mer counts generated by `yak`. This produces the most accurate haplotype separation by binning reads before assembly:
`hifiasm -o <output_prefix> -t <threads> -1 <paternal.yak> -2 <maternal.yak> <input_reads.fq.gz>`

### Telomere-to-Telomere (T2T) Hybrid Assembly
Combine HiFi, Ultra-Long (UL) ONT, and Hi-C reads for maximum contiguity and chromosomal-scale scaffolding:
`hifiasm -o <output_prefix> -t <threads> --dual-scaf --h1 <hic_r1.fq.gz> --h2 <hic_r2.fq.gz> --ul <ultralong_ont.fq.gz> <input_reads.fq.gz>`

## Expert Tips and Best Practices

- **Memory Management**: For small genomes, use `-f0` to disable the initial bloom filter and save 16GB of RAM. For very large genomes (e.g., >10Gb), increase the bloom filter size (e.g., `-f38` or `-f39`) to optimize memory usage.
- **Reusing Intermediate Files**: Hifiasm saves corrected reads and overlaps as `.bin` files. If a run is interrupted or you want to tweak assembly parameters, hifiasm will automatically reload these bins to skip the time-consuming overlap calculation. Use `-i` to force a recalculation if the input data has changed.
- **Telomere Preservation**: When assembling human or similar genomes, use `--telo-m CCCTAA` to specifically protect telomeric repeats from being clipped or misassembled.
- **Output Conversion**: Hifiasm outputs assemblies in GFA format. To convert the primary contigs to FASTA for downstream analysis:
  `awk '/^S/{print ">"$2;print $3}' <output>.bp.p_ctg.gfa > <output>.p_ctg.fa`
- **Adapter Trimming**: If working with older HiFi data that might contain adapter residuals, use `-z20` to trim 20bp from both ends of every read.
- **Partially Phased Outputs**: Since v0.15, hifiasm produces two sets of partially phased contigs (`.hap1.p_ctg.gfa` and `.hap2.p_ctg.gfa`) even without Hi-C or Trio data, though these may contain switch errors depending on the sample's heterozygosity.
