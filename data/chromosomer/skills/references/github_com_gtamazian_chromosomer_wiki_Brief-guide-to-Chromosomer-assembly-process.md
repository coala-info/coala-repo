# Chromosomer assembly process

Chromosomer assembles putative chromosomes from genome fragments (e.g., contigs or scaffolds) in two major steps. First, it processes alignments of the fragments to reference genome chromosomes; the result is a map of the fragments on the chromosomes. Second, Chromosomer produces the reconstructed chromosome sequences from the fragment map and the fragment sequences. These steps are implemented by the *chromosomer fragmentmap* and *chromosomer assemble* routines, respectively.

*chromosomer fragmentmap* requires two input files: a file of fragment-to-chromosome alignments in the BLAST tabular format and a file of fragment lengths. The alignment file can be obtained using the *blastn* tool from the *NCBI BLAST+* package (use the *-outfmt 6* option to get output in the proper format). The fragment length file can be obtained either with the *chromosomer fastalength* routine or the *faSize* program from Jim Kent’s tools.

Also *fragmentmap* has the *gap\_size* argument that is the size in bp of a gap inserted between adjacent fragments. I recommend to set *gap\_size* equal to the maximum insert size of the mate-pair library used to obtain the scaffolds being assembled. The motivation is the following: if we have two separate scaffolds, then there were no mate-pair reads connecting then, because otherwise these two scaffolds would be joined to a single one by a scaffolder. So, the gap of the maximum mate-pair library insert size would be the minimal possible size of a gap separating the scaffolds.

*chromosomer assembler* requires two input files: a fragment map obtained with *chromosomer fragmentmap* and the FASTA file of fragment sequences. Its output is the FASTA file of putative chromosome sequences.

## Example: assembling a simulated genome

To demonstrate how to use Chromosomer, I will use a set of simulated genomic fragments obtained with the *chromosomer simulator* routine:

```
chromosomer simulator --gap_size 100 --unplaced 15 --prefix demo_ 200 5000 4 .
```

This command generates **215** genomic fragments of length **5000 bp** of which **200** fragments are organized into **4 chromosomes** with gaps of **100 bp** between them and **15 fragments** are not located on any chromosome. Two FASTA files with the **demo\_** prefix in their names are created: *demo\_fragments.fa* and *demo\_chromosomes.fa*. These files contain sequences of the fragments and the chromosomes, respectively. Also the fragment map file *demo\_map.txt* is created.

### Step 1: obtaining fragment lengths

The following command produces the fragment length file in the suitable format:

```
chromosomer fastalength demo_fragments.fa demo_fragments.length
```

### Step 2: aligning fragments to a reference genome

In this example, we will use the generated chromosome sequences from *demo\_chromosomes.fa* as a reference genome. To obtain alignments, we will use *blastn*, but first we should create a BLAST database of the chromosome sequences.

```
makeblastdb -in demo_chromosomes.fa -dbtype nucl
```

Note that we did not mask the reference genome here since it was generated as a random concatenation of randomly generated fragment sequences. However, it is strongly recommended to mask real chromosome and fragment sequences for both interspersed and low-complexity repeats. For example, *RepeatMasker* (<http://www.repeatmasker.org>) or *WindowMasker* (<http://bioinformatics.oxfordjournals.org/content/22/2/134.full>) can be used to mask interspersed repeats; *Tandem Repeat Finder* (TRF; <https://tandem.bu.edu/trf/trf.html>) and DustMasker (<https://www.ncbi.nlm.nih.gov/pubmed/16796549>) can be applied to detect low-complexity repeats. One may introduce masks to the BLAST database using the *convert2blastmask* tool from the NCBI BLAST+ package and the *-mask\_data*, *-mask\_id* and *-mask\_desc* options of *makeblastdb*.

Next, we launch *blastn*:

```
blastn -query demo_fragments.fa -db demo_chromosomes.fa -outfmt 6 -out fragment_alignments.txt
```

Note the *-outfmt 6* option: it specifies the output format suitable for Chromosomer. You may pass additional options to *blastn* like *-perc\_identity* (the alignment identity threshold), *-evalue* (the e-value threshold) or *-word\_size* (the alignment seed size).

### Step 3: deriving a fragment map from the fragment alignments

To get the fragment map, we use the *chromosomer fragmentmap* subroutine:

```
chromosomer fragmentmap fragment_alignments.txt 100 demo_fragments.length fragment_map.txt
```

It will show statistics on the produced fragment map like this:

```
2016-10-07 02:28:55 - 200 mapped fragments of total length 1000000 bp
2016-10-07 02:28:55 - 0 unlocalized fragments of total length 0 bp
2016-10-07 02:28:55 - 0 unplaced fragments of total length 0 bp
2016-10-07 02:28:55 - 4 chromosomes formed
2016-10-07 02:28:55 - 20000 bp of gaps inserted
```

### Step 4: obtaining sequences of the assembled chromosomes

Finally, we use *chromosomer assemble* to get the FASTA-file of the assembled sequences:

```
chromosomer assemble fragment_map.txt demo_fragments.fa assembled_chromosomes.fa
```

The output file **assembled\_chromosomes.fa** will contain the assembled chromosome sequences having the same names as the reference genome chromosomes.
