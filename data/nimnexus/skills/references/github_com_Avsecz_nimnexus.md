Commands Trim $ nimnexus trim --help nimnexus version:0.1.0 Trim the fastq reads Usage: nimnexus trim [options] &lt; barcode &gt; Arguments: &lt; barcode &gt; Barcode sequences (comma-separated) that follow random barcode Options: -t --trim &lt; int &gt; Pre-trim all reads by this length before processing [default: 0] -k --keep &lt; int &gt; Minimum number of bases required after barcode to keep read [default: 18] -r --randombarcode &lt; int &gt; Number of bases at the start of each read used for random barcode [default: 5] Example: zcat input.fastq.gz | nimnexus trim -t 1 CTGA,TGAC,GACT,ACTG | gzip -c &gt; output.fastq.gz # Using pigz to (de-)compress in parallel pigz -cd input.fastq.gz | nimnexus trim -t 1 CTGA,TGAC,GACT,ACTG | pigz -c &gt; output.fastq.gz Example: zcat tests/data/mesc_pbx_raw_sample.fastq.gz | ./nimnexus trim -t 1 CTGA,TGAC,GACT,ACTG &gt; /tmp/output.fastq Dedup $ nimnexus dedup --help nimnexus version:0.1.0 Remove duplicate reads from the sorted bam file Usage: nimnexus dedup [options] &lt; BAM &gt; Arguments: &lt; BAM &gt; sorted BAM file Options: -t --threads &lt; int &gt; number of BAM decompression threads [default: 2] Example: nimnexus dedup -t 10 file.bam | samtools view -b &gt; file.dedup.bam Note: nimnexus dup writes the output in the SAM format to stdout. Hence samtools view -b is used to convert SAM-&gt;BAM. Credit the original script written in R was implemented by by Melanie Weilert and Jeff Johnston.
