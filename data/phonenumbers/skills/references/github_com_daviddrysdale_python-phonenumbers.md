Documentation Installation Install using pip with: pip install phonenumbers Example Usage The main object that the library deals with is a PhoneNumber object. You can create this from a string representing a phone number using the parse function, but you also need to specify the country that the phone number is being dialled from (unless the number is in E.164 format, which is globally unique). &gt;&gt;&gt; import phonenumbers &gt;&gt;&gt; x = phonenumbers.parse( " +442083661177 " , None ) &gt;&gt;&gt; print (x) Country Code: 44 National Number: 2083661177 Leading Zero: False &gt;&gt;&gt; type (x) &lt;class 'phonenumbers.phonenumber.PhoneNumber'&gt; &gt;&gt;&gt; y = phonenumbers.parse( " 020 8366 1177 " , " GB " ) &gt;&gt;&gt; print (y) Country Code: 44 National Number: 2083661177 Leading Zero: False &gt;&gt;&gt; x == y True &gt;&gt;&gt; z = phonenumbers.parse( " 00 1 650 253 2222 " , " GB " ) # as dialled from GB, not a GB number &gt;&gt;&gt; print (z) Country Code: 1 National Number: 6502532222 Leading Zero(s): False The PhoneNumber object that parse produces typically still needs to be validated, to check whether it's a possible number (e.g. it has the right number of digits) or a valid number (e.g. it's in an assigned exchange). &gt;&gt;&gt; z = phonenumbers.parse( " +120012301 " , None ) &gt;&gt;&gt; print (z) Country Code: 1 National Number: 20012301 Leading Zero: False &gt;&gt;&gt; phonenumbers.is_possible_number(z) # too few digits for USA False &gt;&gt;&gt; phonenumbers.is_valid_number(z) False &gt;&gt;&gt; z = phonenumbers.parse( " +12001230101 " , None ) &gt;&gt;&gt; print (z) Country Code: 1 National Number: 2001230101 Leading Zero: False &gt;&gt;&gt; phonenumbers.is_possible_number(z) True &gt;&gt;&gt; phonenumbers.is_valid_number(z) # NPA 200 not used False The parse function will also fail completely (with a NumberParseException ) on inputs that cannot be uniquely parsed, or that can't possibly be phone numbers. &gt;&gt;&gt; z = phonenumbers.parse( " 02081234567 " , None ) # no region, no + =&gt; unparseable Traceback (most recent call last): File "phonenumbers/phonenumberutil.py", line 2350, in parse "Missing or invalid default region.") phonenumbers.phonenumberutil.NumberParseException: (0) Missing or invalid default region. &gt;&gt;&gt; z = phonenumbers.parse( " gibberish " , None ) Traceback (most recent call last): File "phonenumbers/phonenumberutil.py", line 2344, in parse "The string supplied did not seem to be a phone number.") phonenumbers.phonenumberutil.NumberParseException: (1) The string supplied did not seem to be a phone number. Once you've got a phone number, a common task is to format it in a standardized format. There are a few formats available (under PhoneNumberFormat ), and the format_number function does the formatting. &gt;&gt;&gt; phonenumbers.format_number(x, phonenumbers.PhoneNumberFormat. NATIONAL ) '020 8366 1177' &gt;&gt;&gt; phonenumbers.format_number(x, phonenumbers.PhoneNumberFormat. INTERNATIONAL ) '+44 20 8366 1177' &gt;&gt;&gt; phonenumbers.format_number(x, phonenumbers.PhoneNumberFormat.E164) '+442083661177' If your application has a UI that allows the user to type in a phone number, it's nice to get the formatting applied as the user types. The AsYouTypeFormatter object allows this. &gt;&gt;&gt; formatter = phonenumbers.AsYouTypeFormatter( " US " ) &gt;&gt;&gt; formatter.input_digit( " 6 " ) '6' &gt;&gt;&gt; formatter.input_digit( " 5 " ) '65' &gt;&gt;&gt; formatter.input_digit( " 0 " ) '650' &gt;&gt;&gt; formatter.input_digit( " 2 " ) '650 2' &gt;&gt;&gt; formatter.input_digit( " 5 " ) '650 25' &gt;&gt;&gt; formatter.input_digit( " 3 " ) '650 253' &gt;&gt;&gt; formatter.input_digit( " 2 " ) '650-2532' &gt;&gt;&gt; formatter.input_digit( " 2 " ) '(650) 253-22' &gt;&gt;&gt; formatter.input_digit( " 2 " ) '(650) 253-222' &gt;&gt;&gt; formatter.input_digit( " 2 " ) '(650) 253-2222' Sometimes, you've got a larger block of text that may or may not have some phone numbers inside it. For this, the PhoneNumberMatcher object provides the relevant functionality; you can iterate over it to retrieve a sequence of PhoneNumberMatch objects. Each of these match objects holds a PhoneNumber object together with information about where the match occurred in the original string. &gt;&gt;&gt; text = " Call me at 510-748-8230 if it's before 9:30, or on 703-4800500 after 10am. " &gt;&gt;&gt; for match in phonenumbers.PhoneNumberMatcher(text, " US " ): ... print (match) ... PhoneNumberMatch [11,23) 510-748-8230 PhoneNumberMatch [51,62) 703-4800500 &gt;&gt;&gt; for match in phonenumbers.PhoneNumberMatcher(text, " US " ): ... print (phonenumbers.format_number(match.number, phonenumbers.PhoneNumberFormat.E164)) ... +15107488230 +17034800500 You might want to get some information about the location that corresponds to a phone number. The geocoder.description_for_number does this, when possible. &gt;&gt;&gt; from phonenumbers import geocoder &gt;&gt;&gt; ch_number = phonenumbers.parse( " 0431234567 " , " CH " ) &gt;&gt;&gt; geocoder.description_for_number(ch_number, " de " ) 'Zürich' &gt;&gt;&gt; geocoder.description_for_number(ch_number, " en " ) 'Zurich' &gt;&gt;&gt; geocoder.description_for_number(ch_number, " fr " ) 'Zurich' &gt;&gt;&gt; geocoder.description_for_number(ch_number, " it " ) 'Zurigo' For mobile numbers in some countries, you can also find out information about which carrier originally owned a phone number. &gt;&gt;&gt; from phonenumbers import carrier &gt;&gt;&gt; ro_number = phonenumbers.parse( " +40721234567 " , " RO " ) &gt;&gt;&gt; carrier.name_for_number(ro_number, " en " ) 'Vodafone' You might also be able to retrieve a list of time zone names that the number potentially belongs to. &gt;&gt;&gt; from phonenumbers import timezone &gt;&gt;&gt; gb_number = phonenumbers.parse( " +447986123456 " , " GB " ) &gt;&gt;&gt; timezone.time_zones_for_number(gb_number) ('Atlantic/Reykjavik', 'Europe/London') For more information about the other functionality available from the library, look in the unit tests or in the original libphonenumber project .
