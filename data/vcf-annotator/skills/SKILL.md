# vcf-annotator

## Overview
The `vcf-annotator` tool is a specialized utility for microbial genomics that transforms raw variant data into biologically meaningful information. By cross-referencing VCF coordinates with a GenBank reference file, it identifies whether mutations are genic or intergenic, determines their effect on protein sequences, and extracts metadata like product descriptions and protein IDs. It is most effective for small to medium-sized microbial genomes (e.g., ~3-5MB) and is compatible with VCF files generated by callers like GATK.

## Usage Instructions

### Basic Command Syntax
The tool requires two positional arguments: the VCF file and the GenBank file.
```bash
vcf-annotator [VCF_FILE] [GENBANK_FILE] [OPTIONS]
```

### Common Patterns
*   **Annotate and stream to file**: By default, the tool outputs to STDOUT.
    ```bash
    vcf-annotator variants.vcf reference.gb > annotated_variants.vcf
    ```
*   **Specify output file explicitly**:
    ```bash
    vcf-annotator variants.vcf reference.gb --output annotated_variants.vcf
    ```

### Annotation Fields Added
The tool appends several tags to the `INFO` column of the VCF. Key fields include:
*   **AminoAcidChange**: The specific change (e.g., S123P).
*   **IsSynonymous**: 1 for synonymous, 0 for nonsynonymous.
*   **LocusTag / Gene**: Identifiers for the affected feature.
*   **RefCodon / AltCodon**: The underlying DNA triplets.
*   **Product**: The functional description of the gene.

## Expert Tips and Best Practices
*   **File Compression**: Ensure the input VCF is **uncompressed**. The tool does not natively support `.vcf.gz` files; use `gunzip` or `zcat` if necessary before processing.
*   **Reference Matching**: The GenBank file must exactly match the reference genome used during the alignment and variant calling steps. Discrepancies in chromosome/contig names between the VCF `CHROM` column and the GenBank `LOCUS` field will cause errors.
*   **Microbial Focus**: While the tool is highly efficient for bacteria and viruses, it is not optimized for large eukaryotic genomes. For a typical 3MB microbial genome with ~20k variants, expect processing to take approximately 10 seconds.
*   **Multi-sequence GenBank Files**: The tool supports GenBank files containing multiple sequences (e.g., multiple contigs or plasmids), provided the identifiers match the VCF.
*   **VCF Compatibility**: While tested primarily on GATK output, any VCF following standard formatting should work. If using non-standard callers, verify that the `CHROM` and `POS` columns are correctly populated.
