# bubblegun

## Overview

BubbleGun is a specialized bioinformatics utility for navigating and analyzing the topology of Graphical Fragment Assembly (GFA) graphs. It focuses on identifying "bubble chains"—sequences of simple bubbles and complex superbubbles that often correspond to biological variations or assembly ambiguities. The tool allows users to simplify graphs through compaction, isolate the largest connected components, and export specific structural features for downstream analysis like haplotyping or variant calling.

## Core Workflows and CLI Patterns

### 1. Detecting Bubble Chains
The primary function of BubbleGun is identifying nested bubble structures.

*   **Standard Detection**: Generate a JSON report of all bubbles and chains.
    ```bash
    BubbleGun -g input_graph.gfa bchains --bubble_json output_chains.json
    ```
*   **Memory Efficient Mode**: For very large graphs, use `--save_memory` to store only the topology and skip reading sequences.
    ```bash
    BubbleGun -g large_graph.gfa bchains --save_memory --bubble_json output.json
    ```
*   **Extracting Sequences**: Output bubble branches as a FASTA file (useful for simple bubbles).
    ```bash
    BubbleGun -g input_graph.gfa bchains --only_simple --fasta branches.fasta
    ```

### 2. Graph Simplification and Pre-processing
Before running complex analyses, it is often beneficial to clean the graph.

*   **Compaction**: Merge linear stretches of single nodes into single segments to reduce graph size.
    ```bash
    BubbleGun -g input_graph.gfa compact compacted_graph.gfa
    ```
    *Note: Compaction may remove some optional tags from S-lines in the GFA.*
*   **Isolating Components**: Extract only the largest connected component to remove small, disconnected assembly artifacts.
    ```bash
    BubbleGun -g input_graph.gfa biggestcomp biggest_component.gfa
    ```

### 3. Targeted Extraction
Once chains are identified in the JSON output, you can extract specific regions.

*   **Extracting Specific Chains**: Use the IDs provided in the `bchains` JSON output to isolate specific genomic regions.
    ```bash
    BubbleGun -g input_graph.gfa chainout --json_file output_chains.json --chain_ids 10 15 22 --output_chain targeted_regions.gfa
    ```
*   **Neighborhood Extraction (BFS)**: Extract a local subgraph around a specific node.
    ```bash
    BubbleGun -g input_graph.gfa bfs --start_node node_id --neighborhood_size 3 --output_bfs local_subgraph.gfa
    ```

### 4. Haplotyping Support
BubbleGun can assist in generating haplotypes from assembly graphs.

*   **Random Path Extraction**: Generate two random paths for each identified chain to simulate haplotyping.
    ```bash
    BubbleGun -g input_graph.gfa bchains --out_haplos
    ```

## Expert Tips and Best Practices

*   **Input Formats**: While GFA is the standard, BubbleGun also supports VG formats. Ensure your GFA is "bluntified" (overlaps removed or handled) for the most accurate bubble detection in de Bruijn graphs.
*   **Logging**: Use the `--log` global argument (e.g., `--log DEBUG`) if the tool fails to identify expected structures or if you are troubleshooting complex superbubbles.
*   **Chain IDs**: Always keep the JSON file generated by `bchains` if you plan to use `chainout` later; the IDs are specific to that specific graph and run.
*   **Sequence Handling**: If you only care about the graph structure (e.g., counting bubbles), always use `--save_memory` to significantly reduce the RAM footprint.
