# miniasm

## Overview
Miniasm is a minimalist, high-speed assembler designed for long-read sequencing data. It follows the Overlap-Layout-Consensus (OLC) paradigm but uniquely omits the consensus step to achieve "ultrafast" performance. The tool takes all-vs-all read self-mappings (typically generated by minimap2) and raw reads to produce an assembly graph in GFA format. Because it does not perform error correction, the resulting unitigs have the same error rate as the raw input reads and must be polished using downstream tools.

## Core Workflow
Miniasm requires a two-step process: overlapping and layout.

1. **Overlapping (using minimap2)**
   Generate the required PAF (Pairwise Mapping Format) file.
   * **PacBio:** `minimap2 -x ava-pb -t8 reads.fq reads.fq | gzip -1 > overlaps.paf.gz`
   * **Nanopore:** `minimap2 -x ava-ont -t8 reads.fq reads.fq | gzip -1 > overlaps.paf.gz`

2. **Layout (using miniasm)**
   `miniasm -f reads.fq overlaps.paf.gz > assembly.gfa`

## Common CLI Patterns
* **Standard Assembly:**
  `miniasm -f <reads.fastq> <mappings.paf> > <output.gfa>`
* **Memory Optimization (2-pass mode):**
  Use `-R` to reduce peak memory usage, which is helpful for larger genomes.
  `miniasm -R -f reads.fq overlaps.paf > assembly.gfa`
* **GFA1 Compatibility:**
  Use `-p` to specify the output type. The `sg` (string graph) option is often used for GFA1 compatibility.
  `miniasm -p sg -f reads.fq overlaps.paf > assembly.gfa`

## Expert Tips and Best Practices
* **Mandatory Polishing:** Since miniasm does not have a consensus step, the output GFA contains raw read sequences. You **must** use a tool like Racon, Medaka, or Pilon after assembly to achieve acceptable base accuracy.
* **Input Requirements:** Miniasm expects all-vs-all mappings. If the PAF file only contains partial mappings or is missing self-mappings, the assembly will be fragmented or empty.
* **Repeat Resolution:** Miniasm is prone to collapsing repeats or segmental duplications that are longer than the input reads. It is most effective for high-coverage bacterial genomes where long reads can span most repeat regions.
* **Filtering:** For very high coverage datasets, you can improve speed and potentially assembly quality by filtering for the longest reads (e.g., top 30-50x coverage) before running the overlap step.
* **Circularization:** For bacterial genomes, look for "circular" tags in the GFA output or use tools like `minidot` (included in the repository) to visualize the assembly graph and check for circularity.
