# rna-velocity-analysis-velocyto-for-10x-data-from-bundled-out

## Overview

This Galaxy workflow automates the quantification of spliced and unspliced RNA transcripts from 10X Genomics single-cell data. It is specifically designed to integrate with the output of 10X preprocessing pipelines, such as the [fastq-to-matrix-10x](https://usegalaxy.org/) workflows, by accepting bundled matrices and BAM files containing cell (CB) and molecular (UB) barcodes.

The process begins by extracting filtered cell barcodes from the provided data bundles. These barcodes, along with a genome annotation (GTF) file and the aligned BAM files, are passed to the [velocyto](https://github.com/velocyto-team/velocyto.py) command-line tool. The workflow generates a loom file for each sample, containing separate count matrices for spliced exons, unspliced introns, and ambiguous regions.

The resulting loom files are essential for downstream RNA velocity analysis, enabling researchers to infer cellular trajectories and predict the future state of individual cells. This workflow is released under the [MIT license](https://opensource.org/licenses/MIT) and is tagged for single-cell analysis. Detailed instructions on file preparation and input requirements can be found in the [README.md](README.md) in the Files section.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | BAM files with CB and UB | data_collection_input | This can be output of CellRanger or STARsolo |
| 1 | filtered matrices in bundle | data_collection_input | This can be output of STARsolo or DropletUtils |
| 2 | gtf file | data_input | gtf file |


Ensure your input BAM files contain the necessary CB and UB tags, typically generated by CellRanger or STARsolo, and provide a GTF file that matches your reference genome. For the bundled input, use a list of lists collection where each sample contains the barcodes, genes, and matrix files; the workflow will automatically isolate the barcodes for processing. Using data collections instead of individual datasets is recommended to maintain sample organization and streamline the parallel execution of the Velocyto CLI. Refer to the README.md for comprehensive details on file preparation and specific tag requirements. You can use `planemo workflow_job_init` to generate a `job.yml` for automated testing and execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Velocyto-on10X-from-bundled.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Velocyto-on10X-from-bundled.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Velocyto-on10X-from-bundled.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Velocyto-on10X-from-bundled.ga -o job.yml`
- Lint: `planemo workflow_lint Velocyto-on10X-from-bundled.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
