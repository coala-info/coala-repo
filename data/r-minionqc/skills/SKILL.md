# r-minionqc

name: r-minionqc
description: Quality control for Oxford Nanopore MinION sequencing data. Use this skill to analyze sequencing_summary.txt files, generate summary statistics, and produce QC plots including length distributions, Q-scores, and throughput over time.

## Overview
`minionqc` is an R-based tool designed for the quality control of Oxford Nanopore MinION sequencing data. It processes the `sequencing_summary.txt` files generated by basecallers (such as Guppy or Albacore) to produce a standardized set of plots and a summary CSV file. It helps in evaluating run performance, flow cell health, and library quality.

## Installation
To install the package from CRAN, use:
`install.packages("r-minionqc")`

## Core Workflow
The package typically operates on a directory containing sequencing summary files. It identifies all summary files in the path, filters reads based on a quality score (Q-score) threshold, and generates analytical visualizations.

### Basic Usage
The main entry point for the package is the processing of a summary file or a directory of files.

```r
library(minionqc)

# Define the input directory containing sequencing_summary.txt
input_dir <- "path/to/sequencing_data"

# Run the QC analysis
# q = 7 is the standard threshold for 'pass' reads in Nanopore sequencing
minionQC(input = input_dir, q = 7)
```

### Key Parameters
- `input`: The path to a single `sequencing_summary.txt` file or a directory containing one or more such files.
- `output`: (Optional) The directory where plots and the summary CSV will be saved. Defaults to the input directory.
- `q`: The Q-score threshold (default is 7). Reads with a score greater than or equal to this value are categorized as "pass".
- `processors`: (Optional) Number of processors to use for parallel processing of multiple flow cells.

## Generated Outputs
The package produces several standard plots:
1.  **Length vs Quality**: A 2D histogram showing the relationship between read length and mean Q-score.
2.  **Throughput**: Cumulative yield of bases over the duration of the run.
3.  **Read Length Distribution**: Histograms of read lengths for both "pass" and "all" reads.
4.  **Flow Cell Overview**: A map of the flow cell showing the total output per pore.
5.  **Summary CSV**: A file containing key metrics like total bases, total reads, N50, and median read length.

## Tips for Success
- **File Naming**: Ensure your basecaller has finished and the `sequencing_summary.txt` file is complete before running the analysis.
- **Large Datasets**: For runs with millions of reads, ensure the system has sufficient RAM, as the package loads the summary table into memory.
- **Multiple Flow Cells**: If the input directory contains subdirectories for different flow cells, `minionqc` will detect them and generate separate reports for each, as well as a combined report.
