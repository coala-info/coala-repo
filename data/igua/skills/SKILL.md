# igua

## Overview

IGUA (Iterative Gene clUster Analysis) is a high-throughput, content-agnostic method for grouping gene clusters into families. Unlike traditional methods that rely on Pfam domain annotations, IGUA utilizes unsupervised clustering of protein sequences. This allows for the inclusion of proteins without known domains and eliminates the need for a costly annotation step. The tool operates through three iterations: fragment mapping, nucleotide deduplication, and protein representation.

## Installation and Setup

IGUA requires MMseqs2 as a core dependency.

- **Conda/Pixi**: `conda install bioconda::igua` or `pixi add igua`
- **Pip**: `pip install igua` (Note: MMseqs2 must be installed separately and available in your PATH).
- **Containerized**: Use `docker://quay.io/biocontainers/igua:0.1.0--py39h5b94c0b_0` for reproducible environments.

## Common CLI Patterns

### Basic GCF Identification
To assign gene clusters to families from one or more GenBank files:
`igua -i clusters1.gbk -i clusters2.gbk --output gcf_assignments.tsv`

### Comprehensive Output Generation
To extract representative protein sequences and the compositional matrix alongside the GCF table:
`igua -i input.gbk --output results.tsv --features proteins.fasta --compositions matrix.h5ad`

### Resource and Performance Optimization
For large datasets, manage memory and scratch space:
`igua -i input.gbk --output results.tsv --workdir /path/to/fast/ssd --precision single`

## Expert Tips and Best Practices

- **Input Requirements**: Ensure input GenBank files contain gene annotations within `CDS` features. IGUA uses the `locus_tag` or locus identifier as the unique name for each cluster.
- **Handling Duplicates**: If multiple clusters share the same identifier, IGUA will process the first one encountered and issue a warning. Pre-process your GenBank files to ensure unique identifiers if working with merged datasets.
- **Clustering Sensitivity**: The default `--clustering-distance` is `0.8`, representing roughly 20% amino-acid level difference. Decrease this value (e.g., `0.6`) for stricter, more conserved families, or increase it for broader groupings.
- **Scratch Space**: MMseqs2 generates significant intermediate data. Always point `--workdir` to a high-speed local drive (like an NVMe SSD) rather than network-attached storage to avoid I/O bottlenecks.
- **Memory Management**: If running into OOM (Out of Memory) errors on large datasets, use `--precision single` or `--precision half` to reduce the memory footprint of the distance matrix.
- **Annotation Transfer**: Use the FASTA file generated by `--features` to perform downstream functional annotation (e.g., via eggNOG-mapper or InterProScan). These annotations can then be mapped back to the GCFs using the representative protein IDs.
