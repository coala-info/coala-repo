# Oligo nucleotide computations

## Overview

To get started run the command `kmertools comp oligo --help` and you will see the following help output.

```
Generate oligonucleotide frequency vectors

Usage: kmertools comp oligo [OPTIONS] --input <INPUT> --output <OUTPUT>

Options:
  -i, --input <INPUT>
          Input file path

  -o, --output <OUTPUT>
          Output vectors path

  -c, --counts
          Disable normalisation and output raw counts

  -k, --k-size <K_SIZE>
          Set k-mer size

          [default: 3]

  -r, --raw-count
          Raw counts

  -p, --preset <PRESET>
          Output type to write

          Possible values:
          - csv: Comma separated format
          - tsv: Tab separated format
          - spc: Space separated format

          [default: spc]

  -H, --header
          Include header (with k-mer in ACGT format)

  -t, --threads <THREADS>
          Thread count for computations 0=auto

          [default: 0]

  -h, --help
          Print help (see a summary with '-h')
```

## Notes

* K-mers are computed by retaining the lexicographically minumum k-mer and its reverse compliment.
* Counts are normalised by the total number of valid k-mers of the sequence
* Invalid k-mers are non ACGT letters k-mers (ACCN, AWCT, etc). Such windows are ignored automatically.

## Options

This specifies the format of the output. For loading into panas consider using `csv`.

```
import pandas as pd

df = pd.read_csv('output.kmers', index=None, header=None)
```

```
import pandas as pd

df = pd.read_csv('output.kmers', index=None)
```

### K-size

K-mer sizes between 3 and 7 are allowed. Note that larger k-mers results in very large outputs. The shape of the resulting vector will be;

* For even k sizes

$$|N| = \frac{4^k + 4^{\frac{k}{2}}}{2}$$

* For odd k sizes

$$|N| = \frac{4^k}{2}$$

### Raw-count

This option skips the computation of counts for minimum complement k-mer. Using this changes the header to output all possible k-mers in ACGT format.

### Counts

This option disables normalisation and returns the array of k-mer counts. This is useful if you do not wish to normalise data but wants to do some other analysis.

## Example application

### K-mer count comparison of bacteria *Enterococcus faecalis* and *Pseudomonas aeruginosa*

Download [`NZ_CP050491.1`](https://www.ncbi.nlm.nih.gov/nuccore/NZ_CP050491.1?report=fasta) and [`CP007224.1`](https://www.ncbi.nlm.nih.gov/nuccore/CP007224.1?report=fasta) from NCBI. These are provided here; [`NZ_CP050491.1`](./examples/oligo/NZ_CP050491.1.fasta.gz) and [`CP007224.1`](./examples/oligo/CP007224.1.fasta.gz).

Run `kmertools` to get k-mer counts in a `.csv` file with headers.

```
# Enterococcus faecalis
kmertools comp oligo -H -p csv -i examples/oligo/NZ_CP050491.1.fasta.gz -o examples/oligo/NZ_CP050491.1.csv
# Pseudomonas aeruginosa
kmertools comp oligo -H -p csv -i examples/oligo/CP007224.1.fasta.gz -o examples/oligo/CP007224.1.csv
```

Load from python as follows ([script](./examples/oligo/oligo.1.py)).

```
import matplotlib.pyplot as plt
import pandas as pd

df_ef = pd.read_csv("examples/oligo/NZ_CP050491.1.csv")
df_pa = pd.read_csv("examples/oligo/CP007224.1.csv")

trimers = list(df_ef.columns)

plt.figure(figsize=(10,3))

plt.plot(trimers, df_ef.loc[0, :].values.tolist(), linewidth=1, color="blue", label="Enterococcus faecalis strain EGM183")
plt.plot(trimers, df_pa.loc[0, :].values.tolist(), linewidth=1, color="red", label="Pseudomonas aeruginosa PA96")

plt.title('Normalised 3-mer count comparison of NZ_CP050491.1 and CP007224.1')
plt.ylabel('Normalised 3-mer counts')
plt.xlabel('3-mers')
plt.xticks(rotation=60)
plt.legend()

plt.savefig("examples/oligo/NZ_CP050491.1_CP007224.1_kmer_comp.png", dpi=150, format="png", bbox_inches='tight')
plt.show()
```
