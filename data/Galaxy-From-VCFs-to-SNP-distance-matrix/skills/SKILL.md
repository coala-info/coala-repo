# From VCFs to SNP distance matrix

## Overview

This Galaxy workflow is designed for evolutionary analysis, specifically to calculate pairwise SNP distances from genomic data. It processes a collection of VCF files alongside a reference genome—such as the MTBC ancestor—to reconstruct sample-specific sequences and quantify genetic divergence.

The pipeline begins by filtering variants using [TB Variant Filter](https://toolshed.g2.bx.psu.edu/repos/iuc/tb_variant_filter/tb_variant_filter/0.1.3+galaxy0) and generating consensus sequences via [bcftools consensus](https://toolshed.g2.bx.psu.edu/repos/iuc/bcftools_consensus/bcftools_consensus/1.9+galaxy2). These individual sequences are then concatenated into a single multifasta file representing the entire sample set.

In the final stages, the workflow utilizes [snp-sites](https://toolshed.g2.bx.psu.edu/repos/iuc/snp_sites/snp_sites/2.5.1+galaxy0) to extract polymorphic positions and [snp-dists](https://toolshed.g2.bx.psu.edu/repos/iuc/snp_dists/snp_dists/0.6.3+galaxy0) to compute the final distance matrix. This automated approach is particularly useful for [GTN](https://training.galaxyproject.org/) training materials and evolutionary studies requiring standardized SNP-based comparisons.

Ensure your input VCF files are organized into a dataset collection to allow the workflow to process multiple samples simultaneously against the single FASTA reference genome. Verify that all VCFs were called against the exact reference provided to ensure coordinate consistency during the consensus sequence generation. For comprehensive details on parameter settings and specific filtering criteria, refer to the `README.md` file. You can also use `planemo workflow_job_init` to create a `job.yml` for automated execution.

## Steps (tools)

| Step | Name | Tool ID | Notes |
| --- | --- | --- | --- |
| 2 | TB Variant Filter | toolshed.g2.bx.psu.edu/repos/iuc/tb_variant_filter/tb_variant_filter/0.1.3+galaxy0 | We will ensure at this step that variants to build the MSA are fixed variants and that we low-confidence filter repetitive regions of the MTB genome |
| 3 | bcftools consensus | toolshed.g2.bx.psu.edu/repos/iuc/bcftools_consensus/bcftools_consensus/1.9+galaxy2 | The complete genome of each of the samples is generated by inserting the SNPs defined in the respective VCF in the reference genome that was used for mapping and SNP calling |
| 4 | Concatenate datasets | toolshed.g2.bx.psu.edu/repos/bgruening/text_processing/tp_cat/0.1.1 | All genomes are concatenated in a single multifasta file. Because all o them have the same length, this may be seen as a multiple sequence alignment. |
| 5 | Finds SNP sites | toolshed.g2.bx.psu.edu/repos/iuc/snp_sites/snp_sites/2.5.1+galaxy0 | Discard invariant positions from the MSA to simplify the file so only contains positions with at least one SNP in at least one strain. |
| 6 | SNP distance matrix | toolshed.g2.bx.psu.edu/repos/iuc/snp_dists/snp_dists/0.6.3+galaxy0 | From the MSA. Calculate pairwise SNP distances between samples. |

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run main-workflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run main-workflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run main-workflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init main-workflow.ga -o job.yml`
- Lint: `planemo workflow_lint main-workflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
