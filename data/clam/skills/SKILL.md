# clam

## Overview

`clam` is a bioinformatics tool designed for the classification of callable loci intervals and the accurate estimation of population genetic statistics. It solves a common problem in genomics: calculating diversity metrics accurately without the storage burden of "all-sites" VCF files. By processing sequencing depth data (often in D4 format) to create a "callable loci" map, `clam` allows researchers to calculate statistics from standard variant-only VCFs while correctly accounting for missing data and varying coverage across samples.

## Command Line Usage

### 1. Generating Callable Loci Intervals
The `loci` command identifies genomic regions with sufficient sequencing depth. It aggregates depth data from multiple samples to determine how many individuals are "callable" at each position.

```bash
# Basic usage with multiple depth files
clam loci -t 16 -m 10 sample1.d4.gz sample2.d4.gz sample3.d4.gz
```

**Key Arguments:**
- `-t, --threads`: Number of threads to use for parallel processing.
- `-m, --min-depth`: The minimum depth threshold for a site to be considered callable.
- Input files: Typically `.d4.gz` or GVCF files.

### 2. Estimating Population Statistics
The `stat` command calculates population genetic metrics in sliding windows across the genome. It requires the Zarr output generated by the `loci` command and a VCF file containing the variants.

```bash
# Calculate statistics in 10kb windows
clam stat -o results/ -w 10000 -c callable.zarr variants.vcf.gz
```

**Key Arguments:**
- `-o, --output`: Directory where results will be saved.
- `-w, --window-size`: Size of the genomic windows for calculation (e.g., 10000 for 10kb).
- `-c, --callable`: Path to the Zarr directory created by `clam loci`.
- `--samples`: (Optional) Specify population groupings or specific samples to include in the analysis.

## Best Practices and Expert Tips

- **Input Formats**: While `clam` can process GVCFs, using D4 files (`.d4` or `.d4.gz`) for depth data is significantly more efficient for large-scale population studies.
- **Window Selection**: When using `clam stat`, choose a window size (`-w`) that balances resolution with the density of your variant data. 10kb to 50kb is standard for many eukaryotic population analyses.
- **Memory Management**: `clam` is written in Rust and designed for high performance, but when processing hundreds of samples in `loci`, ensure your system has sufficient I/O bandwidth, as depth file reading can become a bottleneck.
- **VCF Indexing**: Always ensure your VCF files are compressed with `bgzip` and indexed with `tabix` before running `clam stat`.
- **Handling Missing Data**: One of `clam`'s primary strengths is its ability to produce accurate diversity estimates even with missing data. It uses the Zarr metadata to know exactly how many sites were callable in each window, providing a more accurate denominator for statistics like π (Pi) than simply using the window size.
