Minimal definition of a metabolic model requires: List of reactions. A reaction is minimally defined by reactants and products. Enzymes and cellular compartment, tissue, species are additional. List of compounds with chemical formula. Exact mass can be calculated on formula. List of pathways. A pathway is defined by the reactions in the pathway. This can extend to simple ontological terms, such as chemical classes, lipid categories, etc. Mummichog works on user input metabolomics data and a corresponding metabolic model. The metabolic model is prior knowledge and not changed during data analysis here. The key output of mummichog is 1) mapping relationship btw empCpd and Compounds, and 2) enrichment patterns on metabolic pathways and networks. User input data are modeled following the experimental concepts. The ideal input data would be in empCpd format, which is the result of "annotation". But the software should have a fallback annotation function to deal with user data without annotation per se. mapping relationship btw empCpd and Compounds is in empCpd Note that an empCpd can be a mixture of real compounds, e.g. (compound w, compound z). input data in empCpd format {"neutral_base_mass": 0, "list_of_features": [ # feature, ion, [m/z, rtime, mean_intensity] {'feature': '', 'ion': 'M-H[1-]', 'm/z': 169.0013, 'rtime': 55}, {}, # ... ], "identity": { # compound(mixtures): probability (compound x): 0.6, (compound y, compound z): 0.2, } } Work in progress to implement these into MetDataModel. Mummichog and Azimuth will follow the data models in MetDataModel.
