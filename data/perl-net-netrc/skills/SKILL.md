# perl-net-netrc

## Overview

The `Config::Netrc` module provides a specialized parser for .netrc files, which are commonly used to store login credentials for remote machines (like FTP or autogenerated API tokens). This skill enables the programmatic extraction of these credentials into a structured Raku hash. It is particularly useful for scripts that need to authenticate against multiple services without hardcoding sensitive information, as it handles both the file-level parsing and the extraction of inline comments.

## Core Usage Patterns

### Basic Parsing
To load and parse a specific netrc file, use the `parse-file` function.

```raku
use Config::Netrc;

my $config = Config::Netrc::parse-file('path/to/.netrc');

if $config {
    for $config<entries>.list -> $entry {
        say "Machine: " ~ $entry<machine><value>;
        say "Login: "   ~ $entry<login><value> if $entry<login>;
    }
}
```

### Parsing Strings
If the netrc content is already in memory (e.g., fetched from a secure vault or environment variable), use the `parse` function.

```raku
my $netrc-string = "machine github.com login myuser password mypass # primary account";
my $result = Config::Netrc::parse($netrc-string);
```

## Data Structure Reference

The parser returns a hash with two primary keys: `comments` and `entries`.

1.  **comments**: An array containing all top-level comment strings (with the leading `#` removed).
2.  **entries**: An array of hashes representing each machine entry. Each entry contains:
    *   `machine`: A hash with `value` and `comment`.
    *   `login`: (Optional) A hash with `value` and `comment`.
    *   `password`: (Optional) A hash with `value` and `comment`.

### Accessing Specific Values
Because the structure is nested to include comments for every field, access values using the following pattern:
`$hash<entries>[index]<field><value>`

## Expert Tips

*   **Error Handling**: Both `parse` and `parse-file` return `Nil` if the parser fails. Always validate the result before attempting to iterate over entries.
*   **Comment Preservation**: This module is unique in that it captures comments on the same line as the machine, login, or password tokens. Use `$entry<machine><comment>` to retrieve metadata associated with a specific credential.
*   **Optional Fields**: The netrc specification allows for machines without logins or passwords. The parser handles these as missing keys in the entry hash; use the `exists` or `defined` checks in Raku to avoid TypeCheck errors.
*   **Home Directory**: While the module currently requires a path, the standard location for these files is `~/.netrc` on Unix-like systems or `_netrc` on Windows. Use `$*HOME.child('.netrc')` to find the default file.
