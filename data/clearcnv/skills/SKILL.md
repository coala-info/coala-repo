# clearcnv

## Overview

ClearCNV is a specialized tool for detecting Copy Number Variations (CNVs) in clinical sequencing panels. It is particularly effective in environments where noise or sample-to-panel ambiguity exists. The tool provides two primary paths: a direct CNV calling workflow for well-defined batches and a reassignment workflow that uses clustering to group samples with the most compatible sequencing panels before calling.

## Installation and Setup

Install ClearCNV via bioconda:

```bash
conda create -n clearcnv clearcnv -c conda-forge -c bioconda
conda activate clearcnv
```

## Core Workflows

### 1. Standard CNV Calling
Use this when you have a single gene panel and a collection of BAM files.

1. **Prepare Meta-file (`meta.tsv`)**: A tab-separated file with the following structure:
   `genepanel_name    /path/to/bams.txt    /path/to/genepanel.bed`
   *Note: `bams.txt` should contain absolute paths to all BAM files, one per line.*

2. **Execute Workflow**:
   ```bash
   clearCNV workflow_cnv_calling --metafile /path/to/meta.tsv --workdir ./output_dir
   ```

### 2. Sample Reassignment and Batching
Use this when you have multiple panels or are unsure if BAM files are correctly assigned to specific panels.

1. **Initialize Reassignment**:
   ```bash
   clearCNV workflow_reassignment --workdir ./reassign_work --reference ref.fa --metafile meta.tsv --coverages cov.tsv --bedfile union.bed
   ```

2. **Visualize and Adjust**:
   Launch the interactive browser tool to refine panel assignments:
   ```bash
   clearCNV visualize_reassignment --metafile meta.tsv --coverages cov.tsv --bedfile union.bed --new_panel_assignments_directory ./assigned_batches
   ```

3. **Final Calling**:
   After reassignment, run `workflow_cnv_calling` on each newly created batch file found in the output directory.

## Manual Command Execution

For granular control or custom pipelines, use the underlying commands directly:

### Calculate Match Scores
Creates a match score matrix used to identify the best reference samples for normalization.
```bash
clearCNV matchscores -p panel_name -c coverages.tsv -m matchscores.tsv
```

### Execute CNV Calling
```bash
clearCNV cnv_calling -p panel_name -c coverages.tsv -a ./analysis_dir -m matchscores.tsv -C cnv_calls.tsv -r rscores.tsv -z zscores.tsv
```

## Expert Tips and Best Practices

- **Path Management**: Always use absolute paths in your `meta.tsv` and `bams.txt` files to prevent execution errors in sub-workflows.
- **Parameter Tuning**: In the `cnv_calling` command, avoid using the example parameters `-g 15 -u 3` for real clinical data; these are intended for tiny test datasets only.
- **HPC Integration**: If running on a cluster, ClearCNV supports DRMAA. Use `--drmaa_mem` (e.g., 1600 for 16GB) and `--drmaa_time` (e.g., 4:00) to manage resources.
- **Quality Control**: Always inspect the QC files generated by `workflow_cnv_calling` before interpreting results, especially the Z-score and R-score distributions.
