# bioconductor-scruff

## Overview

The `scruff` package is a comprehensive pipeline for processing scRNA-seq data generated by CEL-Seq and CEL-Seq2 protocols. It handles the transition from raw FASTQ files to a `SingleCellExperiment` object containing UMI-filtered counts. Key features include automated demultiplexing based on cell barcodes, alignment using the `Rsubread` wrapper, UMI correction (Hamming distance), and extensive QC visualization.

## Core Workflow

### 1. Preparation
Before running the pipeline, you need the paths to your Read 1 (barcodes/UMIs) and Read 2 (transcripts) FASTQ files, a reference FASTA, and a GTF annotation file.

```r
library(scruff)
library(Rsubread)

# Define paths
r1 <- "path/to/read1.fastq.gz"
r2 <- "path/to/read2.fastq.gz"
fasta <- "path/to/genome.fa"
gtf <- "path/to/annotation.gtf"

# Build Rsubread index
indexBase <- "genome_index"
Rsubread::buildindex(basename = indexBase, reference = fasta)
```

### 2. All-in-One Processing
The `scruff()` function executes the entire pipeline (demultiplexing, alignment, and counting) in one command.

```r
# Load predefined barcodes if applicable
data(barcodeExample, package = "scruff")

sce <- scruff(
    project = "my_project",
    experiment = "exp1",
    lane = "L001",
    read1Path = r1,
    read2Path = r2,
    bc = barcodeExample,
    index = indexBase,
    reference = gtf,
    bcStart = 1, bcStop = 8,    # CEL-Seq barcode positions
    umiStart = 9, umiStop = 12, # UMI positions
    keep = 75,                  # Trim reads to this length
    cores = 4
)
```

### 3. Stepwise Execution
For more control, you can run the stages individually:

*   **Demultiplexing**: `demultiplex()` splits FASTQ files by cell barcode.
*   **Alignment**: `alignRsubread()` wraps `Rsubread` to map demultiplexed reads.
*   **Counting**: `countUMI()` performs UMI correction and generates the count matrix.

```r
# Example Stepwise Counting
sce <- countUMI(
    al,                         # Object from alignRsubread
    reference = gtf,
    umiEdit = 1,                # Hamming distance for UMI correction
    cellPerWell = c(rep(1, 96)) # Specify expected cells per well
)
```

### 4. Quality Control and Visualization
`scruff` provides built-in functions to assess data quality.

```r
# Generate standard QC plots
qc <- qcplots(sce)
qc # Displays boxplots for reads, alignment rates, and gene detection

# Visualize read mapping at specific genomic coordinates
# Requires a GAlignments object (e.g., from GenomicAlignments::readGAlignments)
g1 <- rview(bamObj, chr = "MT", start = 1000, end = 2000)
g2 <- gview(gtf, chr = "MT", start = 1000, end = 2000)
library(patchwork)
g1 / g2
```

## 10X BAM Quality Assessment
`scruff` can also be used to assess BAM files generated by the 10X Cell Ranger pipeline.

```r
sce_10x <- tenxBamqc(
    bam = "cellranger_output.bam",
    experiment = "10x_sample",
    filter = "filtered_barcodes.tsv"
)
qcplots(sce_10x)
```

## Tips and Best Practices
*   **Barcode Positions**: Always verify the `bcStart/Stop` and `umiStart/Stop` positions based on your specific protocol version (CEL-Seq vs CEL-Seq2).
*   **Parallelization**: Use the `cores` argument in major functions to speed up processing on multi-core systems.
*   **Empty Wells**: Use the `cellPerWell` argument to flag empty well controls (set to 0) or bulk samples (set to >1) to ensure they are handled correctly in QC metrics.
*   **Memory**: Alignment and counting can be memory-intensive; ensure your R session has sufficient allocation for the genome index.
