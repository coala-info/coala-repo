# fq

## Overview

`fq` is a high-performance utility designed for the rapid manipulation of FASTQ files, specifically those generated by Illumina platforms. It provides a suite of subcommands to validate the structural integrity of sequencing data, filter reads based on name allowlists or sequence motifs, and subsample large datasets while maintaining pair synchronization. It natively supports both raw and gzipped inputs and outputs, making it an efficient choice for bioinformatic preprocessing pipelines.

## Core Workflows

### Validating FASTQ Files (lint)
Use the `lint` command to check for common formatting errors or inconsistencies in paired-end data.

*   **Basic validation**: `fq lint r1.fastq.gz r2.fastq.gz`
*   **Comprehensive error reporting**: By default, `fq` panics on the first error. Use `--lint-mode log` to see all validation issues in a single run.
*   **Customizing strictness**: Adjust validation levels (low, medium, high) for single or paired reads using `--single-read-validation-level` or `--paired-read-validation-level`.
*   **Disabling specific checks**: If a specific validator (e.g., S007 for duplicate names) is too restrictive for your use case, disable it with `--disable-validator S007`.

### Filtering Reads (filter)
Filter records based on an allowlist of names or regular expression patterns.

*   **By Name**: `fq filter --names allowlist.txt --dsts out.fastq in.fastq`
*   **By Sequence Motif**: Keep records matching a specific pattern (e.g., a specific start sequence):
    `fq filter --sequence-pattern ^TC --dsts out.1.fq --dsts out.2.fq in.1.fq in.2.fq`
*   **Paired-end Filtering**: When filtering pairs, the match is typically determined by the first input's records and applied to both to maintain synchronization.

### Subsampling Datasets (subsample)
Create smaller datasets for testing or downsampling.

*   **By Probability**: To get approximately 10% of the data:
    `fq subsample --probability 0.1 --r1-dst out.r1.fq --r2-dst out.r2.fq in.r1.fq in.r2.fq`
*   **By Exact Count**: To get exactly 10,000 records (requires two passes over the first input):
    `fq subsample --record-count 10000 --r1-dst out.r1.fq in.r1.fq`
*   **Reproducibility**: Always provide a seed with `-s <SEED>` to ensure the same subset is generated across different runs.

## Expert Tips

*   **Automatic Compression**: `fq` determines output compression based on the file extension. Use `.gz` in your destination paths (e.g., `--r1-dst output.fq.gz`) to automatically trigger gzip compression.
*   **Validation Codes**:
    *   **S002 (Alphabet)**: Ensures sequences only contain ACGTN.
    *   **S005 (ConsistentSeqQual)**: Verifies sequence and quality string lengths match.
    *   **P001 (Names)**: Critical for paired-end data; ensures R1 and R2 headers match.
*   **Performance**: `fq` is written in Rust and optimized for speed. For the fastest processing of gzipped files, ensure you are using the latest version which utilizes optimized compression backends.
