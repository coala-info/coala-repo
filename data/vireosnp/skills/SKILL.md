# vireosnp

## Overview

The `vireosnp` skill provides a workflow for using Vireo (Variational Inference for Reconstructing Ensemble Origin), a Bayesian model designed to demultiplex single-cell RNA-seq data. In multiplexed experiments where cells from multiple individuals are sequenced in a single pool, `vireosnp` uses the genetic variation (SNPs) captured in the reads to identify which donor each cell belongs to. It can operate in two primary modes: supervised (using known donor genotypes) or unsupervised (inferring donor profiles de novo).

## Installation and Setup

The tool is available via Python's package manager or Conda:

```bash
pip install -U vireoSNP
# OR
conda install -c bioconda vireosnp
```

The primary executable command is `vireo`.

## Common CLI Patterns

Vireo requires input files typically generated by `cellSNP` or `cellSNP-lite`, specifically the cell-by-variant matrices for alternative allele depth (AD) and total depth (DP).

### 1. Demultiplexing Without Reference Genotypes
Use this pattern when you know the number of donors in the pool but do not have their individual genotype profiles.

```bash
vireo -c <cell_data_directory> -N <number_of_donors> -o <output_directory>
```
- `-c`: Path to the directory containing `cellSNP` output (AD.mtx, DP.mtx, variants.tsv).
- `-N`: The expected number of donors in the pool.

### 2. Demultiplexing With Reference Genotypes
Use this pattern for higher accuracy when donor genotypes (from WGS, WES, or SNP arrays) are available in VCF format.

```bash
vireo -c <cell_data_directory> -d <donor_genotypes.vcf.gz> -o <output_directory>
```
- `-d`: Path to the donor VCF file. Ensure the VCF is indexed.

### 3. Additional Useful Arguments
- `-t`: Specify the field in the VCF to use for genotypes (default is `GT`, but `GP` or `PL` can be used if available).
- `--randSeed`: Set a seed for reproducibility of the variational inference.
- `--nproc`: Number of processors for parallel computation.

## Expert Tips and Best Practices

- **Input Quality**: The accuracy of Vireo is highly dependent on the number of informative SNPs. Use `cellSNP-lite` with a list of common SNPs (e.g., from 1000 Genomes) to ensure sufficient coverage.
- **Doublet Identification**: Vireo outputs a `donor_ids.tsv` file. Pay close attention to the `label` column (identifying the donor) and the `prob_doublet` column. Cells with high doublet probability should typically be excluded from downstream analysis.
- **Ambient RNA**: High levels of ambient RNA (soup) can introduce "noise" SNPs. If demultiplexing quality is low, consider pre-processing data with ambient RNA removal tools or increasing the minimum depth threshold for SNPs.
- **Missing Data**: Vireo handles missing SNP data (where a cell has no reads for a specific variant) using a Bayesian framework, but a minimum number of detected SNPs per cell is usually required for a confident assignment (typically >50-100 SNPs).
