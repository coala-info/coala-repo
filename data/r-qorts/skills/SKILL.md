# r-qorts

## Overview
The QoRTs (Quality of RNA-Seq Toolset) R package is designed for the analysis, quality control, and data management of high-throughput sequencing datasets. It provides a comprehensive suite of tools to visualize quality metrics across multiple samples, identify batch effects, and prepare data for downstream analysis tools like DESeq2 or EdgeR.

## Installation
To install the QoRTs R package from CRAN:
```R
install.packages("QoRTs")
```

## Core Workflow
The typical R workflow involves loading data generated by the QoRTs Java companion tool and generating diagnostic plots.

### 1. Loading Data
Use `read.QC.resultsset` to load data for an entire experiment.
```R
library(QoRTs)

# Define the directory containing sample subdirectories
input.dir <- "/path/to/qorts_output/"

# Load the dataset
res <- read.QC.resultsset(input.dir)
```

### 2. Generating Quality Control Reports
QoRTs can generate a comprehensive PDF report containing dozens of quality metrics.
```R
makeMultiPlot.all(res, 
                  outfile = "QC_Report.pdf", 
                  plot.device.height = 11, 
                  plot.device.width = 8.5)
```

### 3. Individual Metric Visualization
You can plot specific metrics to investigate potential issues:
- **Read Counts**: `makePlot.countSummary(res)`
- **Insert Size**: `makePlot.insert.size(res)`
- **Genomic Mapping**: `makePlot.genomicRegion(res)`
- **NVC (Nucleotide Versus Cycle)**: `makePlot.nvc(res)`

### 4. Data Export
Export normalized counts or summary tables for downstream analysis.
```R
# Get gene-level counts
counts <- get.count.matrix(res)

# Export summary metrics to a table
write.QC.summary(res, outfile = "summary_metrics.txt")
```

## Key Functions
- `read.QC.resultsset()`: Primary function to load data from a directory of samples.
- `makeMultiPlot.all()`: Generates a multi-page PDF with all standard QC plots.
- `get.count.matrix()`: Extracts gene-level or exon-level counts into a matrix.
- `build.plotter.obj()`: Creates a plotter object for customized visualization.

## Tips for Success
- **Sample Metadata**: Ensure your input directory contains a `decoder.txt` file if you want to group samples by condition or batch in the plots.
- **Memory Management**: For very large datasets (hundreds of samples), use the `debugMode = TRUE` or `skip.files` parameters in `read.QC.resultsset` to load only essential metrics.
- **Consistency**: Ensure the annotation (GTF/GFF) used during the data generation phase matches the one used in downstream differential expression analysis.
