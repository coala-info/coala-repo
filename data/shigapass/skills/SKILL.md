# shigapass

## Overview

ShigaPass is an *in silico* command-line tool designed for the high-resolution taxonomic classification of *Shigella* strains. It processes assembled genomes to provide serotype predictions and distinguishes *Shigella* from closely related Enteroinvasive *E. coli* (EIEC) and other non-*Shigella* species. This skill is essential for genomic epidemiology and clinical research involving *Shigella* surveillance.

## Installation and Setup

ShigaPass is available via Bioconda or GitHub.

```bash
# Installation via Conda
conda install bioconda::shigapass

# Manual installation
git clone https://github.com/imanyass/ShigaPass.git
chmod +x ShigaPass/ShigaPass.sh
```

## Core Workflow

### 1. Prepare Input List
ShigaPass requires a text file containing the absolute or relative paths to your FASTA files (one per line).

```bash
# Generate the input list for all FASTA files in the current directory
readlink -e *.fasta > fasta_list.txt
```

### 2. Initialize Databases (First Run Only)
The first time you run ShigaPass, or after updating the tool, you must initialize the internal databases using the `-u` flag. This flag must be used in conjunction with a full command.

```bash
ShigaPass.sh -l fasta_list.txt -o output_dir -p path/to/ShigaPass_DataBases -u
```

### 3. Standard Execution
For subsequent runs, omit the `-u` flag to save time.

```bash
ShigaPass.sh -l fasta_list.txt -o output_dir -p path/to/ShigaPass_DataBases -t 4
```

## Command Line Options

| Option | Description | Requirement |
| :--- | :--- | :--- |
| `-l` | Path to the text file listing input FASTA files | Mandatory |
| `-o` | Output directory name | Mandatory |
| `-p` | Path to the `ShigaPass_DataBases` directory | Mandatory |
| `-t` | Number of threads (default: 2) | Optional |
| `-u` | Initialize/update BLAST databases | Mandatory for 1st run |
| `-k` | Keep intermediate subdirectories for each genome | Optional |
| `-v` | Display version | Optional |

## Interpreting Results

The tool generates two primary semicolon-delimited summary files in the output directory:

1.  **ShigaPass_summary.csv**: Contains the main prediction results, including:
    *   `rfb` type and coverage.
    *   MLST profile and `fliC` type.
    *   Presence of `ipaH` (virulence marker).
    *   Predicted Serotype (SB: *S. boydii*; SD: *S. dysenteriae*; SF: *S. flexneri*; SS: *S. sonnei*).
2.  **ShigaPass_Flex_summary.csv**: Provides detailed phage and plasmid-encoded O-antigen modification (POAC) genes specifically for *S. flexneri* predictions.

## Expert Tips

*   **Assembly Quality**: ShigaPass is designed for assembled genomes. For best results, use assemblies generated by SPAdes (e.g., using `--careful` and `--cov-cutoff auto`).
*   **Database Pathing**: Ensure the `-p` flag points to the actual directory containing the database files (e.g., `RFB`, `FLIC`, `MLST` folders), not just the parent ShigaPass folder.
*   **Intermediate Files**: Use the `-k` flag if you need to inspect the raw BLAST outputs (`blastout.txt`) or specific hit records (`records.txt`) for troubleshooting ambiguous serotype calls.
*   **Unzip Inputs**: Ensure all input FASTA files are unzipped (`.fasta` or `.fa`) before running the tool, as it does not natively handle `.gz` files.
