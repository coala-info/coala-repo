# chiron

## Overview
Chiron is a deep neural network-based tool designed to translate raw electrical signals from nanopore sequencers directly into nucleotide sequences. It utilizes a CNN+RNN+CTC architecture to provide end-to-end basecalling. This skill provides the necessary command-line patterns to execute basecalling, optimize performance through decoder selection and batching, and manage the multi-directory output structure generated by the tool.

## Basecalling Workflows

### DNA Basecalling
To basecall DNA samples using the default model and preset:
```bash
chiron call -i <input_fast5_folder> -o <output_folder> -m chiron/model/DNA_default -p dna-pre
```

### RNA Basecalling
For direct-RNA sequencing, ensure the `--mode rna` flag is used along with the RNA-specific preset:
```bash
chiron call -i <input_fast5_folder> -o <output_folder> -m chiron/model/RNA_default -p rna-pre --mode rna
```

## Performance Optimization

### Decoder Selection
Chiron supports two decoding methods that trade off speed for accuracy:
- **Beam Search (Default)**: Higher accuracy. Increase `--beam <width>` for better results (e.g., 50), though this significantly slows down processing.
- **Greedy Decoder**: Much faster but less accurate. Enable by setting beam width to zero:
  ```bash
  chiron call -i <input> -o <output> --beam 0
```

### Memory and GPU Management
If you encounter Out-Of-Memory (OOM) errors on your GPU or system RAM, reduce the batch size (default is 400 for DNA, 300 for RNA):
```bash
chiron call -i <input> -o <output> -p dna-pre -b 200
```

## Output Management
Chiron creates five distinct subdirectories in the output folder:
- `result/`: Individual fastq/fasta files for each read.
- `raw/`: Raw signal data as integer lists.
- `segments/`: Basecalled segments.
- `meta/`: Metadata including read length and basecalling rate.
- `reference/`: Reference sequences if provided.

### Merging Results
Because Chiron outputs one file per read, use the following pattern to create a single dataset for downstream tools:
```bash
# For Fasta
paste --delimiter=\\n --serial result/*.fasta > merged.fasta

# For Fastq
paste --delimiter=\\n --serial result/*.fastq > merged.fastq
```

### Output Format
Toggle between FASTQ (default, includes quality scores) and FASTA using the `-e` flag:
```bash
chiron call -i <input> -o <output> -e fasta
```

## Expert Tips
- **Environment**: Chiron requires TensorFlow 1.15. It is highly recommended to run this within a dedicated Conda environment or virtualenv to avoid dependency conflicts.
- **RNA Preprocessing**: For improved RNA results, consider pre-segmenting files with BoostNano to remove polyA and adapter tails before running Chiron.
- **Preset Overrides**: Arguments provided after the `-p` preset flag will override the preset's defaults (e.g., changing `jump` or `segment_len`).
