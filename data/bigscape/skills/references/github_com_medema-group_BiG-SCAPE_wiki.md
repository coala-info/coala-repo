## How BiG-SCAPE Cluster works, in a nutshell

BiG-SCAPE 2 can be run in 4 [workflows](wiki/02.-BiG-SCAPE-Workflows): [Cluster](wiki/02.-BiG-SCAPE-Workflows#big-scape-cluster),
[Query](wiki/02.-BiG-SCAPE-Workflows#big-scape-query), [Dereplicate](wiki/02.-BiG-SCAPE-Workflows#big-scape-dereplicate) and [Benchmark](wiki/02.-BiG-SCAPE-Workflows#big-scape-benchmark) (See figure).

`bigscape cluster` performs clustering of BGCs into GCFs. This is the equivalent of running BiG-SCAPE 1’s
bigscape.py. With `bigscape query` you can search for BGCs that show similarity to a user provided query
BGC/.gbk. `bigscape dereplicate` performs a protein sequence based redundancy analysis to remove near-identical sequences, and `bigscape benchmark` compares the results of a BiG-SCAPE 2 Cluster mode run, BiG-SCAPE 1
run or [BiG-SLiCE](https://github.com/medema-group/bigslice) run against a user-provided set
of BGC <-> GCF assignments.

![](https://github.com/medema-group/BiG-SCAPE/wiki/figures/Fig_1_scaled.png)

BiG-SCAPE Cluster reads BGC information stored in [antiSMASH](https://antismash.secondarymetabolites.org)
processed GenBank files, and uses a phmm([profile hidden Markov Models](https://www.ebi.ac.uk/training/online/courses/pfam-creating-protein-families/what-are-profile-hidden-markov-models-hmms/)) database (commonly, [Pfam](http://pfam.xfam.org/))
and `hmmscan` from the [HMMER](http://hmmer.org/) suite to predict protein protein domains in each sequence,
thus summarizing each BGC as a linear set of protein domains. BGCs can be grouped in bins by their given
[antiSMASH class/category](https://docs.antismash.secondarymetabolites.org/glossary/#protocluster-types),
or grouped into a single mixed bin (by using --mix). For every pair of BGCs in the bin, the pair is first
aligned (based on a user-defined alignment mode) which allows the [region to compare](wiki/08.-Defining-the-comparable-region)
to be defined. Then, the pairwise [distance](wiki/09.-Distance-Calculation) between the pair of BGCs is calculated as the
weighted combination of the Jaccard, Adjacency Index (AI) and Domain Sequence Similarity (DSS) indices
(more detail [here](wiki/09.-Distance-Calculation#Weight_Distribution)). A cutoff is then applied to these distance
values, and clustering with [Affinity propagation](https://scikit-learn.org/stable/modules/generated/sklearn.cluster.AffinityPropagation.html#sklearn.cluster.AffinityPropagation) is
applied to generate Gene Cluster Families (GCFs). BiG-SCAPE 2 also allows more than one cutoff to be provided
per run, in which case a set of GCFs will be generated for each cutoff.

Learn more about the BiG-SCAPE modes and options with python `bigscape.py --help`, the [tutorials](wiki/12.-Tutorials) or by reading through this wiki. We suggest going through this wiki in the following order:
