## New features

* Build from source is now available - [`fc3578b`](https://github.com/steineggerlab/ufcg/commit/fc3578b6f852b0b7d397fe1ed375a84fc3f36f21)
  + Any builds including commits between stable releases will be marked as unstable, and will be warned - [`63240fd`](https://github.com/steineggerlab/ufcg/commit/63240fde6bfebe399eb4e6a2e2249fd630c06d01)
  + Powered by [appassembler](https://github.com/mojohaus/appassembler) by [mojohaus](https://github.com/mojohaus)
* `profile` - Validation sensitivity adjustment feature - [#19](https://github.com/steineggerlab/ufcg/issues/19), [`6dbbe3d`](https://github.com/steineggerlab/ufcg/commit/6dbbe3d8e441e8296c1b713e6cbe381f6bd81dab)
  + `--sensitivity 3` (default): Use original algorithm (cov. 80%, 50%, e-value only)
  + `--sensitivity 2` : 2-step search (cov. 80%, 50%)
  + `--sensitivity 1` : 1-step search (cov. 80%)
* New mascot! Special thanks to [Gallois](https://blog.naver.com/profile/intro.naver?blogId=trex97) - [Image source](https://github.com/steineggerlab/ufcg/assets/49298377/7aa5a1d7-96b0-4f1e-b151-60220f796d94), [`3e2e9a9`](https://github.com/steineggerlab/ufcg/commit/3e2e9a91df95def5e30e64c66ed1146835db764c)

## Adjustments

* `profile`, `profile-rna`, `profile-pro` - Pipeline will automatically download config files for its initial run - [#16](https://github.com/steineggerlab/ufcg/issues/16), [`75265d5`](https://github.com/steineggerlab/ufcg/commit/75265d5d9549e6aac43c0d93da21550d3d102cf5)
* `profile` - Use default MMseqs easy-search parameters for `--search-type` and `--cov-mode` flags - [`3a1fabf`](https://github.com/steineggerlab/ufcg/commit/3a1fabf9eba2d17af6ff3b0c22e05856aa691ce7)
* `download` - Use the current working directory for target `sample` - [`64ac3ee`](https://github.com/steineggerlab/ufcg/commit/64ac3ee7e01328510332226530cb9fc0df77408c)
* `align` - Dependency can now be internally set through arguments - [`03f11ae`](https://github.com/steineggerlab/ufcg/commit/03f11ae52e2a6dbab0ccc65ccb9558920dd90c67)

## Bug fixes

* `profile`, `profile-pro` - Completely synchronized output for multi-threading - [`1f8a7d7`](https://github.com/steineggerlab/ufcg/commit/1f8a7d7b012ce912f7d6098ce33ab57ae98185a9)
* Minor fixes including typos

## New features

* Provided better usability to `tree` and `align` - [`557de11`](https://github.com/steineggerlab/ufcg/commit/557de11892ff78cb9164ab4a339e56e7365a1098), [`e384942`](https://github.com/steineggerlab/ufcg/commit/e38494276cd073422737948f28c6f5961b1c225e), [`2d9fbfd`](https://github.com/steineggerlab/ufcg/commit/2d9fbfd477c31eaded25921473054ea2c2a7539e)
  + The modules will no longer produce a sub-directory with random hexadecimal string names even when the run name is not given.
  + Instead, outputs will be directly written in the given output directory.
  + `tree` module will no longer require `-l` option. `label` will be used as a default parameter.
* Introduced file logging based checkpoint feature - [#12](https://github.com/steineggerlab/ufcg/issues/12), [`a1e9177`](https://github.com/steineggerlab/ufcg/commit/a1e9177281b1740a3911066059d9a48598f24097)
  + Currently applied to `tree` and `align`

## Bug fixes

* Fixed `mmseqs` detection in interactive `profile` mode - [#9](https://github.com/steineggerlab/ufcg/issues/9), [`35e52bb`](https://github.com/steineggerlab/ufcg/commit/35e52bbfa061901fe354beb2d5ccda2c86578922)
* Faster and more reliable reverse complement generator - [#13](https://github.com/steineggerlab/ufcg/issues/13), [`fe699d1`](https://github.com/steineggerlab/ufcg/commit/fe699d19a4ef8c16edece33966f6dbcedcc15361)
  + Thanks to [@Changwanseo](https://github.com/Changwanseo)

* Minor fixes
