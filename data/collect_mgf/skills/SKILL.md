# collect_mgf

## Overview

The `collect_mgf` tool is a specialized utility for proteomics workflows that use the XMass software environment. In these experiments, MS/MS spectra are often stored in individual files across a nested directory structure. This skill enables the automated collection of these spectra into a single file, ensuring that precursor mass and charge state information—often missing or incomplete in the raw individual MGFs—is correctly mapped from the experiment's `dd_results` file.

## Usage Instructions

### Command Syntax
The tool follows a strict positional argument structure:
`collect_mgf <data_directory> <dd_results_file> <start_index> <end_index> <output_mgf>`

### Arguments
- **data_directory**: The base path containing the numbered experiment folders (e.g., `1`, `2`, `3`).
- **dd_results_file**: The path to the `dd_results` file generated by the XMass experiment.
- **start_index**: The integer representing the first experiment number (EXPNO) to include.
- **end_index**: The integer representing the last experiment number (EXPNO) to include.
- **output_mgf**: The desired filename for the resulting consolidated MGF file.

### Example Pattern
To collect data from folders 2 through 256 in a directory named `project_alpha`:
`collect_mgf project_alpha dd_results 2 256 consolidated_data.mgf`

## Expert Tips and Best Practices

### Directory Structure Requirements
The tool expects a specific Bruker-style directory hierarchy. For a given index `i`, it looks for the source MGF at:
`<data_directory>/<i>/pdata/1/<data_directory>_<i>_1.mgf`
Ensure your data directory name matches the prefix of the MGF files within the `pdata` folders.

### Data Filtering Logic
- **MS/MS Only**: The tool is hardcoded to process only even-numbered scans, which typically correspond to MS/MS spectra in these specific experiment types.
- **Charge State Filter**: Only precursors with a successfully deconvoluted charge state (where CHARGE > 0 in the `dd_results` file) are included in the final output. Spectra with undetermined charge states are automatically skipped.

### Troubleshooting
- **File Not Found Errors**: If the tool fails to open an MGF file, verify that the `<data_directory>` argument exactly matches the folder name and the file prefix inside the `pdata` directory.
- **Empty Output**: If the output file is created but empty, check the `dd_results` file to ensure it contains valid precursor information and that the experiment indices provided in the command line match the indices listed in the results file.
