# pyloh

## Overview

PyLOH is a bioinformatics tool used to analyze next-generation sequencing data from tumor samples that contain a mixture of normal and cancerous cells. It helps researchers identify the specific genomic reads associated with tumor cells or subclones. By combining copy number data with LOH patterns, PyLOH provides a robust estimation of tumor purity (the fraction of tumor cells in the sample) and ploidy (the number of sets of chromosomes).

The tool operates in three primary stages: preprocessing raw sequence alignments, running a probabilistic model to estimate parameters, and post-processing for visualization.

## Usage Guidelines

### 1. Data Preparation
Before running PyLOH, ensure your input files are properly indexed. PyLOH requires indexed BAM files and an indexed reference genome.
*   **Reference Index**: Run `samtools faidx REFERENCE_GENOME.fasta`
*   **BAM Index**: Run `samtools index NORMAL.bam` and `samtools index TUMOUR.bam`

### 2. Preprocessing
The `preprocess` module extracts allelic counts from heterozygous loci in the normal genome.

```bash
PyLOH.py preprocess REFERENCE_GENOME.fasta NORMAL.bam TUMOUR.bam BASENAME \
    --segments_bed SEGMENTS.bed \
    --min_depth 20 \
    --process_num 10
```

**Key Parameters:**
*   `--segments_bed`: Highly recommended. Use a BED file generated by segmentation algorithms like BICseq (for WGS) or DNAcopy (for WES).
*   `--WES`: Use this flag if working with Whole Exome Sequencing data.
*   `--process_num`: Increase this to utilize multiple CPU cores for faster processing.

### 3. Running the Model
The `run_model` module performs the actual deconvolution using the files generated during preprocessing.

```bash
PyLOH.py run_model BASENAME --allelenumber_max 2 --max_iters 100
```

**Key Parameters:**
*   `--allelenumber_max`: The maximum copy number allowed for each allele.
*   `--priors`: Optional. If not provided, the tool uses a uniform prior (recommended for most cases).
*   `--stop_value`: The convergence threshold for the EM algorithm (default is 1e-7).

### 4. Visualization (Post-processing)
To visualize the cluster patterns of LOH, generate a BAF (B-Allele Frequency) heatmap.

```bash
PyLOH.py BAF_heatmap BASENAME
```

## Expert Tips and Best Practices

*   **Segmentation Quality**: The accuracy of PyLOH is heavily dependent on the quality of the input genome segmentation. Always prefer using specialized segmentation tools (BICseq/DNAcopy) over the default 22-autosome segmentation.
*   **Performance**: If processing speed is a concern, note that `pysam` version 0.7.x is significantly faster for PyLOH's preprocessing tasks than version 0.8.x.
*   **Environment**: PyLOH is built for Python 2.7. Ensure your environment (e.g., Conda) is configured for Python 2.7.3 to avoid compatibility issues with `numpy` and `scipy`.
*   **Output Interpretation**:
    *   `*.PyLOH.counts`: Contains allelic counts for heterozygous loci.
    *   `*.PyLOH.segments`: Contains genomic information for each segment.
    *   The model output will provide the estimated tumor purity and the copy number state for each segment.
