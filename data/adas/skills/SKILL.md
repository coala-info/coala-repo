# adas

## Overview

ADAS (Advanced Database Search) is a high-performance sequence search engine optimized for long biological sequences. It utilizes a combination of Minimizers, Order MinHash, and Hierarchical Navigable Small World (HNSW) graphs to enable fast searches based on edit distance. The tool is designed to handle large-scale genomic data by creating a searchable graph index, allowing for rapid retrieval of nearest neighbors and approximate alignments without the computational overhead of traditional exhaustive search methods.

## Installation

The recommended way to install adas is via Bioconda:

```bash
conda install -c bioconda -c conda-forge adas
```

## Core Workflows

### 1. Building a Database Index
Before searching, you must build an HNSW graph index from your reference sequences.

```bash
adas-build -i reference.fasta -k 8 -s 512 -t 8 --max_nb_connection 128 --hnsw-ef 800
```

**Key Parameters:**
- `-k, --kmer-size`: Size of k-mers (must be ≤ 14, default is 8).
- `-s, --sketch-size`: Size of the MinHash sketch (default 512).
- `--hnsw-capacity`: Maximum number of sequences the index can hold.
- `--max_nb_connection`: Maximum number of connections per node in the HNSW graph.
- `--scale_modify_f`: Scale modification factor (0.2 to 1.0). Lower values increase speed but may reduce sensitivity.

### 2. Searching the Database
Search query sequences against a directory containing pre-built HNSW index files.

```bash
adas-search -i query.fasta -b ./database_dir -n 50 -t 8
```

**Key Parameters:**
- `-b, --hnsw`: The directory containing the index files generated by `adas-build`.
- `-n, --nbng`: Number of search answers (top-K neighbors) to return (default 128).

### 3. Updating an Existing Index
Add new sequences to an existing database without rebuilding the entire graph.

```bash
adas-insert -i new_sequences.fasta -b ./database_dir -t 8
```

### 4. Long Read Alignment (Chaining)
Perform seed-chain-extend alignment similar to minimap2 for overlaps or mapping.

```bash
adas-chain -q query.fasta -r reference.fasta -t 8 -o output.paf
```

### 5. Extracting K-Nearest Neighbors
Extract the actual sequence IDs for the nearest neighbors of every sequence within a pre-built database.

```bash
adas-knn -b ./database_dir -n 32 -o neighbors_list.txt
```

## Expert Tips and Best Practices

- **K-mer Constraints**: Ensure `-k` is 14 or less. For highly divergent sequences, smaller k-mer sizes (e.g., 8-10) are generally more effective for MinHash-based sketching.
- **Index Persistence**: `adas-build` generates multiple files (usually 5). Always point `-b` or `--hnsw` to the directory containing these files, not a specific file.
- **Memory Management**: For extremely large databases, monitor the `--hnsw-capacity` parameter. Setting this too high can lead to unnecessary memory allocation, while setting it too low will prevent the index from growing.
- **Performance Tuning**: Increasing `--hnsw-ef` (entry factor) during the build phase improves search accuracy but increases indexing time.
- **Output Format**: `adas-chain` produces results in PAF (Pairwise Mapping Format), making it compatible with downstream bioinformatics pipelines like Flye or Racon.
