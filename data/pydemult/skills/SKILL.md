# pydemult

## Overview
pydemult is a specialized tool for the streamed and parallel demultiplexing of FASTQ files. It identifies samples by parsing read names using regular expressions rather than searching the sequence data itself. By separating the CPU-bound task of barcode matching from the I/O-bound task of writing compressed output, it achieves high throughput on multi-core systems.

## Usage Guidelines

### Barcode File Format
The tool requires a barcode mapping file (typically tab-separated) with at least two columns:
- **Sample**: The name used for the output FASTQ file.
- **Barcode**: The sequence to match against the extracted string.

Example `barcodes.txt`:
```text
Sample    Barcode
sample1   CTTCAA
sample2   CAACAA
```

### Regular Expression Syntax
pydemult uses Python's `re` module. You must define a named capturing group `(?P<CB>...)` for the cell barcode. An optional `(?P<UMI>...)` group can be used for Unique Molecular Identifiers.

- **Standard Illumina header**: `"(.*):(?P<CB>[ATGCN]{6})"`
- **DropSeq (with UMI)**: `"(.*):CELL_(?P<CB>[ATGCN]{10}):UMI_(?P<UMI>[ATGCN]{8})"`

### Performance Optimization
The tool splits work into demultiplexing (CPU) and writing (I/O). Writing is generally the bottleneck.
- **Thread Ratio**: For optimal performance, use a **1:3 ratio** of `--threads` to `--writer-threads`.
- **Example**: `pydemult --threads 4 --writer-threads 12 ...`

### Common CLI Pattern
```bash
pydemult --fastq input.fastq.gz \
         --barcodes barcodes.txt \
         --barcode-regex "(.*):(?P<CB>[ATGCN]{6})" \
         --threads 4 \
         --writer-threads 12 \
         --output-dir ./demultiplexed_results
```

## Expert Tips
- **Regex Testing**: Always verify your regex against a single read header from your FASTQ file using a Python interpreter before starting a large run.
- **I/O Bottlenecks**: If running on a cluster, ensure the `--output-dir` is on a high-performance filesystem (like Lustre or GPFS) to support the high number of concurrent write operations.
- **Empty Entries**: If you need to preserve entries that do not match any barcode, check for the `--keep-empty` flag (available in later versions) or inspect the unmatched output file usually generated by the tool.
