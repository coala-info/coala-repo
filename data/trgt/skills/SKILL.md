# trgt

## Overview

TRGT (Tandem Repeat Genotyping Tool) is a specialized utility designed to characterize tandem repeats using high-fidelity (HiFi) long-read sequencing data. Unlike traditional tools that only estimate repeat length, TRGT analyzes internal sequence composition, detects mosaicism, and extracts CpG methylation patterns directly from the reads. It is the primary tool for identifying pathogenic expansions and conducting genome-wide repeat variation studies on the PacBio platform.

## Core Workflows

### Genotyping Repeats

Run the primary genotyping command to generate VCF files and spanning BAMs (BAMlets).

```bash
trgt genotype \
  --genome reference.fasta \
  --reads input.hifi.bam \
  --repeats repeat_catalog.bed \
  --output-prefix sample_name
```

*   **Input BAM**: Must be indexed.
*   **Repeat Catalog**: A BED file defining the repeat regions and motifs.
*   **Output**: Produces `sample_name.vcf.gz` (genotypes) and `sample_name.spanning.bam` (reads overlapping repeats).

### Visualizing Loci

Generate waterfall plots to inspect repeat structure and read alignments.

```bash
trgt plot \
  --genome reference.fasta \
  --reads sample_name.spanning.bam \
  --vcf sample_name.vcf.gz \
  --repeats repeat_catalog.bed \
  --locus "chr1:1000-2000" \
  --output-prefix locus_plot
```

*   **Note**: You must use the `.spanning.bam` generated by the `genotype` command, not the original input BAM.
*   **Locus**: Can be specified as a coordinate string or a repeat ID from the catalog.

### Joint Analysis and Merging

Combine VCFs from multiple samples into a single cohort VCF.

```bash
# Using a list of files
trgt merge --vcf-list sample_vcf_list.txt --output cohort_merged.vcf.gz

# Using explicit flags
trgt merge --vcf sample1.vcf.gz --vcf sample2.vcf.gz --output cohort_merged.vcf.gz
```

*   **Streaming Mode**: For very large cohorts, use `--no-index` to reduce memory usage (requires identical contig ordering across inputs).
*   **Indexing**: Use `--write-index` to automatically generate a tabix/CSI index for the output.

### Locus Deepdive

Perform detailed downstream analysis for a specific locus by realigning reads to consensus sequences.

```bash
trgt deepdive \
  --genome reference.fasta \
  --reads input.hifi.bam \
  --vcf sample_name.vcf.gz \
  --repeats repeat_catalog.bed \
  --locus "repeat_id"
```

## Expert Tips and Best Practices

*   **Performance Optimization**: Ensure the repeat catalog BED file is sorted by contig and start position. TRGT 4.0+ uses read streaming, which is significantly faster (2-4x) when the catalog is sorted.
*   **Targeted Sequencing**: When analyzing PureTarget or other enrichment data, always use the `--preset targeted` flag. This prioritizes reads with higher repeat purity between flanks.
*   **Phasing Support**: TRGT 5.0+ supports phased genotypes. If the input BAM contains HP/PS tags (e.g., from HiPhase), TRGT will output phased genotypes (e.g., `1|0`).
*   **Methylation**: TRGT automatically extracts CpG methylation from BAM records containing base modifications (Mm/Ml tags). This is reflected in the VCF and visualization plots.
*   **Catalog Validation**: Before running large jobs, use `trgt validate --repeats catalog.bed` to check for formatting errors or overlapping definitions in your repeat file.
*   **Memory Management**: When merging thousands of samples, use the `--threads` flag to speed up (de)compression and the `--no-index` flag to keep memory footprints low.
