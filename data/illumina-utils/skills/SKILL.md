# illumina-utils

## Overview

The `illumina-utils` suite provides specialized tools for the initial stages of genomic data processing. It is particularly effective for handling paired-end FASTQ files where high-precision merging or specific quality control standards are required. The toolset relies heavily on a standardized configuration file format to manage input/output paths and sequence patterns, ensuring reproducible workflows for complex sequencing runs.

## Core Workflows

### 1. Demultiplexing Raw Data
Use `iu-demultiplex` when you have raw FASTQ files and an index file.
- **Required inputs**: R1, R2, Index FASTQ files, and a TAB-delimited barcode-to-sample mapping.
- **Command**: `iu-demultiplex -s barcode_to_sample.txt --r1 r1.fastq --r2 r2.fastq --index index.fastq -o output_dir/`

### 2. Configuration Management
Most scripts require a `.ini` config file. Instead of manual creation, use `iu-gen-configs`.
- **From Demultiplexing**: Run `iu-gen-configs [path_to_report]` using the report generated by `iu-demultiplex`.
- **From Existing FASTQs**: Create a TAB-delimited file with columns `sample`, `r1`, and `r2`, then run `iu-gen-configs [mapping_file]`.

### 3. Merging Paired-End Reads
Use `iu-merge-pairs` to merge overlapping reads.
- **Mechanism**: Uses Levenshtein distance to calculate the best overlap (default minimum 15 nt).
- **Command**: `iu-merge-pairs [config_file]`
- **Output**: Generates FASTA files for successfully merged reads and those that failed.

### 4. Quality Filtering
The suite implements two primary filtering philosophies:
- **Minoche et al.**: Focuses on B-tail trimming and Q30 thresholds. Use `iu-analyze-illumina-quality-minoche`.
- **Bokulich et al.**: Focuses on quality trimming and ambiguous base counts. Use `iu-analyze-illumina-quality-bokulich`.
- **Tip**: Use the `--visualize-quality-curves` flag to generate PNG reports of quality scores per tile.

## Configuration File Structure
Ensure your `.ini` files follow this pattern:
- `[general]`: Define `project_name`, `input_directory`, and `output_directory`.
- `[files]`: List comma-separated filenames for `pair_1` and `pair_2`. The order must match.
- `[prefixes]`: (Optional) Use regex to specify primers or UMIs for trimming (e.g., `^....TACGC...`).

## Expert Tips
- **Directory Safety**: The software will not automatically create the `output_directory`. You must create it manually before running scripts, or the process will fail.
- **Overwrite Protection**: Scripts will not overwrite existing files in the output directory. Clear the directory or use a new path for iterative runs.
- **TAB Completion**: After installation, type `iu-` and press TAB twice to see the full list of available utilities (e.g., `iu-trim-fastq`, `iu-filter-quality-minoche`).



## Subcommands

| Command | Description |
|---------|-------------|
| iu-demultiplex | Demultiplex a paired-end Illumina Run |
| iu-gen-configs | Generate config files for demultiplexed FASTQ files |
| iu-merge-pairs | Merge Overlapping Paired-End Illumina Reads |
| iu-trim-fastq | Trim Illumina reads |
