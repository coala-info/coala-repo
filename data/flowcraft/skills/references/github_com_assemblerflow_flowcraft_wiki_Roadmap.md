# Roadmap

[Jump to bottom](#wiki-pages-box)

Diogo Silva edited this page Sep 27, 2018
·
[9 revisions](/assemblerflow/flowcraft/wiki/Roadmap/_history)

This is an overview of the main features and releases that are planned for FlowCraft:

* [x]  1.2.x: **Add the `inspect` run mode** to track the progress of nextflow pipelines in the terminal and broadcasting to a web app. Also the refactor of the name from assemblerflow to flowcraft.
* [x]  1.3.x: **Add the `report` run mode** to visualize the reports of the pipeline using the flowcraft web app.

  + [x]  Limit the size and compress the `report.json` output.
  + [x]  Create a REST API in the flowcraft web service to receive report data.
  + [x]  Add `report` run mode to send report data to the web service.
  + [x]  Create the frontend of the reports (<https://github.com/assemblerflow/flowcraft-webapp>).
  + [x]  Add option to export pipeline parameters as JSON
* [ ]  1.4.x: **Revamp the `params` system** to allow process specific parameters and the injection of *extra parameters*.

  + [x]  Process specific parameters
  + [ ]  Add support for automatic handling of paired and single end FastQ
  + [ ]  Custom extra parameters
  + [ ]  Add long-read support
* [x]  1.5.x: **Revamp the `recipes` system** to take advantage of the `params` implementation.
* [ ]  1.6.x: Add `build` mode to webservice ??
