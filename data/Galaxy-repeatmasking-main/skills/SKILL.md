# Repeat masking with RepeatModeler and RepeatMasker

## Overview

This Galaxy workflow provides an automated pipeline for the *de novo* identification and masking of transposable elements (TEs) and low-complexity DNA sequences. It integrates two primary tools to transform a raw genome assembly into an annotated and masked sequence ready for downstream genomic analysis.

The process begins with [RepeatModeler](https://toolshed.g2.bx.psu.edu/repos/csbl/repeatmodeler/repeatmodeler/2.0.4+galaxy1), which uses a combination of programs like RECON and RepeatScout to identify repeat element boundaries and model repeat families from the input FASTA file. This step generates essential consensus sequences and seed alignments that define the repeat landscape of the specific organism.

In the second stage, [RepeatMasker](https://toolshed.g2.bx.psu.edu/repos/bgruening/repeat_masker/repeatmasker_wrapper/4.1.5+galaxy0) uses the library generated by RepeatModeler to screen the genome. It produces a masked version of the query sequence along with a detailed repeat catalog, GFF3 annotations, and comprehensive statistics regarding the mutation sites and repeat content.

This workflow is released under the MIT license and is designed for researchers requiring a standardized method for repeat annotation. For specific details on file preparation and input requirements, refer to the README.md in the Files section.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run RepeatMasking-Workflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run RepeatMasking-Workflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run RepeatMasking-Workflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init RepeatMasking-Workflow.ga -o job.yml`
- Lint: `planemo workflow_lint RepeatMasking-Workflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
