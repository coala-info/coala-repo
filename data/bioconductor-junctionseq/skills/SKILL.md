# bioconductor-junctionseq

## Overview

JunctionSeq is a Bioconductor package designed to detect differential usage of transcript subunits (exons and splice junctions) in RNA-Seq experiments. Unlike gene-level differential expression, "differential usage" refers to changes in the relative expression of a specific feature compared to the gene as a whole. This is a powerful proxy for identifying alternative splicing, alternative promoter usage, and other isoform-regulation phenomena.

The package provides a robust statistical framework based on negative binomial GLMs (extending the DEXSeq methodology) and includes a comprehensive visualization suite for generating gene profile plots and genome browser tracks.

## Core Workflow

### 1. Data Preparation
JunctionSeq requires count data generated by the QoRTs software. You need:
- A flattened GFF file (generated by QoRTs from a standard GTF).
- Count files for each sample (containing exon and junction counts).
- A design data frame or decoder file.

### 2. Simple Analysis Pipeline
The most efficient way to run a standard analysis is using the wrapper function:

```r
library(JunctionSeq)

# Define paths to count files and GFF
countFiles <- c("sample1.txt.gz", "sample2.txt.gz", ...)
sampleNames <- c("S1", "S2", ...)
condition <- factor(c("Control", "Control", "Test", "Test"))

# Run full analysis
jscs <- runJunctionSeqAnalyses(
  sample.files = countFiles,
  sample.names = sampleNames,
  condition = condition,
  flat.gff.file = "annotation.gff.gz",
  analysis.type = "junctionsAndExons",
  nCores = 1
)
```

### 3. Advanced Step-by-Step Pipeline
For more control over dispersion estimation or model parameters:

```r
# 1. Load data
jscs <- readJunctionSeqCounts(countfiles = countFiles, samplenames = sampleNames, 
                              design = data.frame(condition), flat.gff.file = gff.file)

# 2. Estimate size factors and dispersions
jscs <- estimateJunctionSeqSizeFactors(jscs)
jscs <- estimateJunctionSeqDispersions(jscs)
jscs <- fitJunctionSeqDispersionFunction(jscs)

# 3. Perform testing and effect size estimation
jscs <- testForDiffUsage(jscs)
jscs <- estimateEffectSizes(jscs)
```

## Visualization and Results

### Exporting Results
```r
# Write results to text files and BED tracks
writeCompleteResults(jscs, outfile.prefix = "./results/", save.jscs = TRUE)
```

### Generating Plots
JunctionSeq is known for its highly interpretable gene profile plots.

```r
# Build all plots for significant genes
buildAllPlots(jscs = jscs, 
              outfile.prefix = "./plots/", 
              use.plotting.device = "png", 
              FDR.threshold = 0.01)

# Plot a specific gene of interest
plotJunctionSeqResultsForGene(jscs, geneID = "ENSRNOG00000009281")
```

## Key Functions and Parameters

- `runJunctionSeqAnalyses`: The primary high-level wrapper.
- `analysis.type`: Set to `"junctionsAndExons"` (default), `"exonsOnly"`, or `"junctionsOnly"`.
- `method.countVectors`: Defaults to `"gene-minus-feature"`, ensuring no read is counted twice in a single model (improving on the DEXSeq "sum-of-other-bins" approach).
- `geneWisePadj`: A critical result column representing the q-value for the hypothesis that at least one feature in the gene is differentially used.

## Tips for Interpretation
- **Relative Expression**: A feature can be "differentially used" even if its absolute expression is flat, provided the rest of the gene changes expression.
- **Novel Junctions**: If QoRTs was configured to find novel junctions, JunctionSeq will test them. These are labeled with an "N" prefix (e.g., N001) and are vital for identifying unannotated splicing events.
- **Visualization**: Use the `buildAllPlots` function to generate HTML-navigable reports, which are much easier to parse than raw text tables.
