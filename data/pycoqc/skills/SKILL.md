# pycoqc

## Overview
pycoqc is a specialized tool designed for the post-sequencing analysis of Oxford Nanopore data. It transforms raw sequencing summary files (generated by basecallers like Guppy or MinKNOW) into comprehensive, interactive HTML reports. This skill provides the necessary command-line patterns to evaluate library quality, identify potential run issues, and generate publication-ready visualizations of sequencing yield and accuracy.

## Core Usage Patterns

### Basic Report Generation
The primary command requires a sequencing summary file as input.
```bash
pycoqc -f sequencing_summary.txt -o pycoqc_report.html
```

### Handling Multiple Summary Files
If a run was split into multiple summary files, provide them as a space-separated list or use a wildcard.
```bash
pycoqc -f summary_1.txt summary_2.txt -o combined_report.html
```

### Integration with Alignment Data
To include alignment-based metrics (like coverage and identity), provide a BAM file. Note that the BAM must be indexed.
```bash
pycoqc -f sequencing_summary.txt -a alignment.bam -o full_qc_report.html
```

### Filtering and Configuration
You can refine the analysis by applying filters directly in the CLI:
- **Minimum Read Length**: `--min_len 500`
- **Minimum Quality Score**: `--min_qual 7`
- **Limit Reads**: `--max_reads 1000000` (useful for quick previews of large datasets)

## Expert Tips
- **Memory Management**: For extremely large datasets, use the `--skip_per_read_plots` flag to reduce the HTML file size and memory consumption during report generation.
- **Barcoding**: If the run was barcoded, pycoqc automatically detects barcode columns in the summary file and generates per-barcode statistics.
- **JSON Output**: For automated pipelines where you need machine-readable data instead of a visual report, use `-j metrics.json`.
