# panisa

## Overview
The panisa skill enables the identification of insertion sequences in bacterial genomes using a signature-based approach. Instead of relying on a database of known sequences, it analyzes BAM files to find clusters of clipped reads that indicate the boundaries of an insertion. It then attempts to reconstruct the left and right inverted repeats (IRL and IRR) to validate the potential IS. This tool is essential for genomic epidemiology and studying bacterial evolution where mobile genetic elements play a significant role.

## Installation and Setup
PanISa requires Python (pysam, requests) and the EMBOSS package.

```bash
# Recommended installation via Conda
conda install -c bioconda panisa

# Alternative installation via Pip
pip install panisa
```

## Core Usage Patterns

### Basic IS Detection
To run the standard analysis on a coordinate-sorted BAM file:
```bash
python panISa.py input.bam > results.tab
```

### High-Sensitivity Detection
If you suspect low-frequency insertions or are working with lower coverage data, adjust the clipped read threshold:
```bash
# Lowering -m from default 10 to 5 to catch more candidates
python panISa.py -m 5 input.bam -o results_sensitive.tab
```

### Filtering for High-Quality Alignments
To reduce false positives caused by mapping noise, increase the minimum alignment quality:
```bash
# Increasing -q from default 20 to 30
python panISa.py -q 30 input.bam -o results_high_quality.tab
```

### Validating Results with ISFinder
After generating a results table, use the companion script to search for homology against the ISFinder database to identify known IS families:
```bash
python ISFinder_search.py results.tab > annotated_results.tab
```

## Expert Tips and Best Practices
- **Alignment Choice**: PanISa is optimized for and performs best with alignments generated by `bwa`.
- **Direct Repeat Constraints**: The default maximum size for direct repeats (`-s`) is 20bp. If you are looking for specific elements known to have longer target site duplications, increase this value.
- **Consensus Threshold**: The `-p` parameter (default 0.8) controls the stringency of the consensus sequence reconstruction for the IS boundaries. If your output contains many "N" characters in the IRL/IRR sequences, consider slightly lowering this value or checking your read depth.
- **Input Preparation**: Ensure your BAM file is indexed (`samtools index input.bam`) before running panISa to ensure efficient processing.
