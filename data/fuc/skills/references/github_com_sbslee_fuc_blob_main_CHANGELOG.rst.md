# Changelog

## 0.38.0 (2024-06-16)

* Update [:meth:`pyvcf.has\_chr\_prefix`](#id2) method to ignore the HLA contigs for GRCh38.
* [:issue:`71`](#id4): Deprecate [:meth:`common.plot\_cytobands`](#id6) method.

## 0.37.0 (2023-09-09)

* [:issue:`67`](#id9): Fix bug in [:meth:`pymaf.MafFrame.plot\_waterfall`](#id11) method where `count=1` was causing color mismatch.
* Add new submodule `pychip`.
* Add new method [:meth:`common.reverse\_complement`](#id13).
* Fix bug in [:meth:`common.extract\_sequence`](#id15) method where a long DNA sequence output was truncated.
* [:issue:`68`](#id17): Refresh the variant consequences database from Ensembl VEP. The database's latest update was on May 31, 2021.

## 0.36.0 (2022-08-12)

* `fuc` now has a citation! Please refer to the publication “[ClinPharmSeq: A targeted sequencing panel for clinical pharmacogenetics implementation](https://doi.org/10.1371/journal.pone.0272129)” by Lee et al., 2022 (Steven is the first author). Fore more details, see the Citation section in README.
* Update `pyvcf` submodule to accept "sites-only" VCF.
* Add new method [:meth:`pyvcf.VcfFrame.filter\_gsa`](#id20).
* Add new method [:meth:`pyvcf.VcfFrame.duplicated`](#id22).
* Add new optional argument `to_csv` to [:meth:`pymaf.MafFrame.plot\_regplot\_tmb`](#id24) method.
* Add new optional argument `count` to [:meth:`pymaf.MafFrame.plot\_mutated\_matched`](#id26) method.

## 0.35.0 (2022-07-12)

* Fix bug in [:meth:`pyvcf.VcfFrame.pseudophase`](#id29) method.
* Add new methods [:meth:`pyvcf.VcfFrame.diploidize`](#id31) and [:meth:`pyvcf.gt\_diploidize`](#id33).
* Update [:meth:`pyvcf.VcfFrame.get\_af`](#id35) method to handle situations where there are multiple records with the same `REF` allele.
* Add new method [:meth:`pymaf.MafFrame.plot\_regplot\_tmb`](#id37).
* Rename [:meth:`pyvcf.VcfFrame.plot\_regplot`](#id39) method to [:meth:`pyvcf.VcfFrame.plot\_regplot\_tmb`](#id41) and [:meth:`pymaf.MafFrame.plot\_regplot`](#id43) method to [:meth:`pymaf.MafFrame.plot\_regplot\_gene`](#id45).

## 0.34.0 (2022-06-08)

* Add new optional argument `--stranded` to [:command:`ngs-quant`](#id48) command.
* Add new method [:meth:`pycov.CovFrame.merge`](#id50).
* Add new method [:meth:`pycov.merge`](#id52).
* [:issue:`61`](#id54): Update [:meth:`pymaf.MafFrame.from\_vcf`](#id56) method to automatically detect CSQ field in INFO column (thanks [@lbeltrame](https://github.com/lbeltrame)).
* [:issue:`63`](#id58): Update [:meth:`pyvcf.VcfFrame.sort`](#id60) method to handle contigs that are not pre-defined.

## 0.33.1 (2022-05-03)

* Add new method [:meth:`pybam.index`](#id63) which simply wraps [:meth:`pysam.index`](#id65) method.
* Update [:command:`bam-index`](#id67) command to use [:meth:`pybam.index`](#id69) method.
* Add new method [:meth:`pybam.slice`](#id71).
* Update [:command:`bam-slice`](#id73) command to use [:meth:`pybam.slice`](#id75) method.
* Update [:command:`ngs-bam2fq`](#id77) and [:command:`ngs-fq2bam`](#id79) commands to allow users to run in local environment.
* Update [:command:`ngs-fq2bam`](#id81) command to handle cases where input FASTQ does not have information on flowcell and barcode.
* Update [:meth:`pyvcf.call`](#id83) method to run more efficiently.

## 0.32.0 (2022-04-02)

* Add new optional argument `filter_off` to [:class:`pykallisto.KallistoFrame`](#id86) constructor, which is useful for generating a simple count or tpm matrix.
* Add new optional argument `--dir-path` to [:command:`vcf-call`](#id88) command for storing intermediate files.
* Add new optional argument `--gap_frac` to [:command:`vcf-call`](#id90) command so that users can control indel calling sensitivity.
* Add new optional argument `--group-samples` to [:command:`vcf-call`](#id92) command so that users can group samples into populations and apply the HWE assumption within but not across the populations.
* Fix minor bug in [:meth:`pyvcf.call`](#id94) method when `pybed.BedFrame` object is given as `regions`.

## 0.31.0 (2022-03-01)

* Fix bug in [:meth:`pykallisto.KallistoFrame.compute\_fold\_change`](#id97) method.
* Add new method [:meth:`pyvcf.call`](#id99) and new command [:command:`vcf-call`](#id101).
* Combine optional arguments `bam` and `fn` into single positional argument `bams` for [:meth:`pycov.CovFrame.from\_bam`](#id103) method. The same goes for [:command:`bam-depth`](#id105) command (combine `--bam` and `--fn` into `bams`).
* Combine optional arguments `bed` and `region` into single optional argument `regions` for [:meth:`pycov.CovFrame.from\_bam`](#id107) method. The same goes for [:command:`bam-depth`](#id109) command (combine `--bed` and `--region` into `--regions`).
* Update [:meth:`pycov.CovFrame.from\_bam`](#id111) method and [:command:`bam-depth`](#id113) command to automatically handle the 'chr' string.
* Rename [:meth:`pyvcf.VcfFrame.variants`](#id115) method to [:meth:`pyvcf.VcfFrame.to\_variants`](#id117).
* Add new optional arguments `force` and `missing` to [:meth:`pyvcf.row\_updateinfo`](#id119) method.
* Add new method [:meth:`pyvcf.gt\_ploidy`](#id121).
* Update [:meth:`pyvcf.gt\_polyp`](#id123) method to use [:meth:`pyvcf.gt\_ploidy`](#id125) method internally.
* [:issue:`53`](#id127): Add new methods to compute AC/AN/AF in the INFO column: [:meth:`pyvcf.row\_computeinfo`](#id129) and [:meth:`pyvcf.VcfFrame.compute\_info`](#id131).
* [:issue:`54`](#id133): Update [:meth:`pyvcf.VcfFrame.cfilter\_empty`](#id135) method so that users can control missingness threshold for filtering samples.
* Rename [:meth:`pyvcf.VcfFrame.cfilter\_empty`](#id137) method to [:meth:`pyvcf.VcfFrame.empty\_samples`](#id139).
* Update [:meth:`common.sort\_regions`](#id141) method to support regions containing an ALT contig (e.g. chr16\_KI270854v1\_alt).

## 0.30.0 (2022-02-05)

* Update [:command:`fuc-find`](#id144) command to allow users to control whether to use recursive retrieving.
* Add new command [:command:`ngs-trim`](#id146).
* Add new command [:command:`ngs-quant`](#id148).
* Add new submodule `pykallisto`.
* Update [:meth:`pycov.CovFrame.from\_bam`](#id150) method to use filename as sample name when the SM tag is missing.
* Add new method [:meth:`pyvcf.row\_phased`](#id152). From now on, it's used to get the `pyvcf.VcfFrame.phased` property.
* Add new method [:meth:`pyvcf.split`](#id154) and [:command:`vcf-split`](#id156) command for splitting VCF by individual.
* Update [:meth:`pyvcf.merge`](#id158) method, [:meth:`pyvcf.VcfFrame.merge`](#id160) method, and [:command:`vcf-merge`](#id162) command to automatically handle the 'chr' string.

## 0.29.0 (2021-12-19)

* Add new property `pyvcf.VcfFrame.phased`.
* Update [:meth:`pyvcf.VcfFrame.slice`](#id165) method to automatically handle the 'chr' string.
* Add new argument `--thread` to [:command:`ngs-hc`](#id167) command. This argument will be used to set `--native-pair-hmm-threads` for GATK's [:command:`HaplotypeCaller`](#id169) command, `--reader-threads` for GATK's [:command:`GenomicsDBImport`](#id171) command, and `-XX:ParallelGCThreads` and `-XX:ConcGCThreads` for Java.
* Add new argument `--batch` to [:command:`ngs-hc`](#id173) command. This argument will be used to set `--batch-size` for GATK's [:command:`GenomicsDBImport`](#id175) command.
* Update [:command:`ngs-bam2fq`](#id177) command to fix the SGE issue that outputs an error like `Unable to run job: denied: "XXXXX" is not a valid object name (cannot start with a digit)`.
* Update [:command:`ngs-hc`](#id179) command so that when `--posix` is set, it will use `--genomicsdb-shared-posixfs-optimizations` argument from GATK's [:command:`GenomicsDBImport`](#id181) command in addition to exporting relevant shell variable (i.e. `export TILEDB_DISABLE_FILE_LOCKING=1`).
* Add new argument `--job` to [:command:`ngs-fq2bam`](#id183) command.
