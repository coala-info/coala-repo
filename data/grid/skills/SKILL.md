# grid

## Overview
GRiD (Growth Rate Index) is a specialized tool for measuring the *in situ* replication rates of bacteria. By analyzing the coverage gradient across a genome—specifically the ratio of DNA at the origin of replication (*ori*) to the terminus (*ter*)—it provides a proxy for growth speed. It is highly effective for analyzing metagenomic bins and draft-quality genomes, even when sequencing depth is extremely shallow.

## Usage Guidelines

### Input Requirements
- **Read Format**: Accepts `.fq`, `.fastq`, `.fastq.gz`, and `.sam` (SAM is only valid for the `single` module).
- **Single-End Only**: GRiD requires single-end reads. If you have paired-end data, use only one of the mate pairs or concatenate both pairs into a single file before processing.
- **SAM Files**: For the `single` module, SAM files must be generated by mapping reads to the specific reference genome being analyzed.

### Module Selection
- **`grid single`**: Use when you have a specific reference genome (FASTA) and want to analyze growth for that organism specifically.
- **`grid multiplex`**: Use for high-throughput analysis of all bacteria in a sample. This requires a pre-indexed database (comprehensive or environment-specific).

### Common CLI Patterns

#### Single Genome Analysis
```bash
grid single -r <samples_dir> -g <reference.fasta> -o <output_dir> -n <threads>
```
*Key flags:*
- `-e`: Specify extension (default: fastq).
- `-l`: Path to a file listing a subset of samples to analyze.

#### High-Throughput Multiplex Analysis
```bash
grid multiplex -r <samples_dir> -d <database_dir> -o <output_dir> -c 0.2 -p
```
*Key flags:*
- `-c`: Coverage cutoff (default is 1, but can go as low as 0.2).
- `-p`: Enables reassignment of ambiguous reads using Pathoscope2 (recommended for complex samples).
- `-m`: Merges all output tables into a single matrix file (`merged_table.txt`).

### Expert Tips and Best Practices
- **Database Selection**: To reduce runtime in `multiplex` mode, use an environment-specific database (e.g., stool microbes) rather than the comprehensive database if the niche is known.
- **Reliability Metric**: Pay close attention to the `Species heterogeneity` value in the output. Values **< 0.3** indicate highly reliable growth estimates.
- **Validation Ratios**: In the `single` module output, check the `dnaA/ori` and `ter/dif` ratios. The closer these values are to **1**, the more accurate the GRiD score is likely to be.
- **Performance**: For very large datasets, subsample the reads to reduce runtime without significantly impacting the index calculation.
- **Output Interpretation**: 
    - The `.pdf` output in `single` mode provides a visual coverage plot across the genome.
    - The `multiplex` heatmap displays the 70 most abundant species.
