# pirate

## Overview

PIRATE (Pangenome analysis and threshold evaluation toolbox) is a scalable framework for the rapid interrogation of bacterial pangenomes. It distinguishes itself by clustering genes over a wide range of amino-acid or nucleotide identity thresholds rather than a single fixed value. This approach allows for the identification of orthologous groups across divergent taxa while maintaining the resolution needed to distinguish closely related paralogs. It is specifically optimized to handle complex genomic events such as gene fissions, fusions, and duplications, providing a pangenome graph and measures of allelic variance.

## Usage Guidelines

### Input Requirements
*   **Format**: PIRATE requires GFF3 files that include the nucleotide sequence at the end of the file (FASTA-embedded GFF).
*   **Source**: It is highly recommended to use GFF3 files generated by **Prokka**.
*   **Validation**: If using GFF3 files from RAST or NCBI, ensure the fasta header matches the first field in the annotation and that `locus_tag` or `ID` fields are present.

### Core Command Patterns

**Basic pangenome construction:**
```bash
PIRATE -i /path/to/gff_directory/ -o /path/to/output_dir/
```

**Customizing identity thresholds:**
By default, PIRATE uses steps of 50, 60, 70, 80, 90, 95, and 98% identity. You can modify this to be more or less stringent:
```bash
PIRATE -i ./gffs -s 70,80,90,95,98,99
```

**Analyzing specific features:**
While `CDS` is the default, you can specify other annotated features:
```bash
PIRATE -i ./gffs -f CDS,rRNA,tRNA
```

**Nucleotide-based clustering:**
For very closely related strains where protein translation might mask variation, use the nucleotide sequence directly:
```bash
PIRATE -i ./gffs --nucl
```

### Post-Processing and Tools

**Restoring original Locus Tags:**
PIRATE renames tags to a standardized format (`genome_locusnumber`). To map these back to your original nomenclature in the final output:
```bash
subsample_outputs.pl -i PIRATE.gene_families.ordered.tsv -o renamed_output.tsv --field "prev_locus"
```

**Generating Alignments:**
To produce unaligned multifasta files for specific features (useful for downstream phylogenetics):
```bash
align_feature_sequences.pl --no-align
```

### Expert Tips
*   **Iterative Refinement**: If you have already run the pangenome construction and only want to adjust the classification or downstream analysis, use the `--pan-off` flag to save time by skipping the initial clustering steps.
*   **Paralog Handling**: If your dataset is extremely large and paralog classification is not a priority, use `--para-off` to significantly reduce computation time.
*   **Memory Management**: PIRATE relies on `mcxdeblast` and `mcl`. For datasets with thousands of genomes, ensure your environment has sufficient RAM for the inflation steps of the MCL algorithm.
*   **Standardization**: If your GFF files are inconsistent, use `ncbi-genome-download` to get the FASTA files and re-annotate everything with Prokka to ensure the GFF3 attributes are perfectly formatted for PIRATE.
