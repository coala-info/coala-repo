# Metatranscriptomics analysis using microbiome RNA-seq data - Workflow 3: Functional Information

## Overview

This Galaxy workflow is designed for the functional characterization of microbiome RNA-seq data, serving as the third stage in a comprehensive metatranscriptomics pipeline. It takes interlaced reads and a taxonomic profile as primary inputs to determine the metabolic potential and active functional pathways within a microbial community.

The core of the analysis is driven by [HUMAnN](https://toolshed.g2.bx.psu.edu/repos/iuc/humann/humann/3.9+galaxy0), which profiles gene families and metabolic pathways. The workflow automates several critical post-processing steps, including regrouping gene families, renaming features for better interpretability, and renormalizing abundances to ensure data comparability across different samples.

The final outputs provide a deep dive into the community's functional landscape, generating stratified and unstratified tables for gene families and pathway abundances. Additionally, the workflow extracts and organizes Gene Ontology (GO) information, specifically categorizing results into Cellular Component (CC), Biological Process (BP), and Molecular Function (MF) terms.

Developed as part of the [Galaxy Training Network (GTN)](https://training.galaxyproject.org/), this tool is released under an MIT license. It is essential for researchers transitioning from taxonomic identification to understanding the actual biological activities occurring within a microbiome sample.

## Inputs and data preparation

| 0 | Interleaced reads | data_input | Metatranscriptomics interleaced reads |
| 1 | Taxonomic profile | data_input | Taxonomic profile generated by MetaPhlAn |


Ensure your interlaced reads are in fastq format and the taxonomic profile is provided as a tabular file to ensure compatibility with the HUMAnN tool suite. For efficient batch processing, it is recommended to use dataset collections rather than individual files when handling multiple samples. Detailed instructions on database dependencies and specific tool parameters are available in the README.md. You may also utilize `planemo workflow_job_init` to create a `job.yml` for structured job submission.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run workflow3-functional-information.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run workflow3-functional-information.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run workflow3-functional-information.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init workflow3-functional-information.ga -o job.yml`
- Lint: `planemo workflow_lint workflow3-functional-information.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
