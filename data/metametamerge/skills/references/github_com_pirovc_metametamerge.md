Piro ( vitorpiro@gmail.com ) This tool is part of the MetaMeta Pipeline ( https://github.com/pirovc/metameta ) but can also be used as a standalone tool. Input: MetaMetaMerge integrates profiling and binning tools. MetaMetaMerge accepts BioBoxes format directly ( https://github.com/bioboxes/rfc/tree/master/data-format ) or a .tsv file in the following format: Profiling: rank, taxon name or taxid, abundance Example: genus Methanospirillum 0.0029 genus Thermus 0.0029 genus 568394 0.0029 species Arthrobacter sp. FB24 0.0835 species 195 0.0582 species Mycoplasma gallisepticum 0.0536 Binning: readid, taxon name or taxid, lenght of sequence assigned Example: M2|S1|R140 354 201 M2|S1|R142 195 201 M2|S1|R145 457425 201 M2|S1|R146 562 201 M2|S1|R147 1245471 201 M2|S1|R150 354 201 Database profiles: Database profiles can be generated based on a list of accession codes and two scripts (acc2tab.bash and tab2count.bash): https://github.com/pirovc/metameta/blob/master/scripts/acc2tab.bash https://github.com/pirovc/metameta/blob/master/scripts/tab2count.bash cat accession_list.txt | xargs --max-procs=12 -I '{}' bash acc2tab.bash '{}' &gt; output_tax tab2count.bash output_tax &gt; output_dbprofile Example: ./MetaMetaMerge.py -i binning_out.tsv profile1.tsv profile2.out -d dbprofile1.out dbprofile2.out dbprofile3.out -t 'tool1,tool2,tool3' -c 'b,p,p' -n names.dmp -e nodes.dmp -m merged.dmp -o output_profile.out Parameters: usage: MetaMetaMerge.py [-h] -i [&lt;input_files&gt; [&lt;input_files&gt; ...]] -d [&lt;database_profiles&gt; [&lt;database_profiles&gt; ...]] -t &lt;tool_identifier&gt; -c &lt;tool_method&gt; -n &lt;names_file&gt; -e &lt;nodes_file&gt; -m &lt;merged_file&gt; [-b &lt;bins&gt;] [-r &lt;cutoff&gt;] [-f &lt;mode&gt;] [-s &lt;ranks&gt;] -o &lt;output_file&gt; [-p &lt;output_type&gt;] [--output-parsed-profiles] [--detailed] [--verbose] [-v] MetaMetaMerge by Vitor C. Piro (vitorpiro@gmail.com, http://github.com/pirovc) optional arguments: -h, --help show this help message and exit -i [&lt;input_files&gt; [&lt;input_files&gt; ...]], --input-files [&lt;input_files&gt; [&lt;input_files&gt; ...]] Input (binning or profiling) files. Bioboxes or tsv format (see README) -d [&lt;database_profiles&gt; [&lt;database_profiles&gt; ...]], --database-profiles [&lt;database_profiles&gt; [&lt;database_profiles&gt; ...]] Database profiles on the same order of the input files (see README) -t &lt;tool_identifier&gt;, --tool-identifier &lt;tool_identifier&gt; Comma-separated identifiers on the same order of the input files -c &lt;tool_method&gt;, --tool-method &lt;tool_method&gt; Comma-separated methods on the same order of the input files (p -&gt; profiling / b -&gt; binning) -n &lt;names_file&gt;, --names-file &lt;names_file&gt; names.dmp from the NCBI Taxonomy database -e &lt;nodes_file&gt;, --nodes-file &lt;nodes_file&gt; nodes.dmp from the NCBI Taxonomy database -m &lt;merged_file&gt;, --merged-file &lt;merged_file&gt; merged.dmp from the NCBI Taxonomy database -b &lt;bins&gt;, --bins &lt;bins&gt; Number of bins. Default: 4 -r &lt;cutoff&gt;, --cutoff &lt;cutoff&gt; Minimum abundance/Maximum results for each taxonomic level (0: off / 0-1: minimum relative abundance / &gt;=1: maximum number of identifications). Default: 0.0001 -f &lt;mode&gt;, --mode &lt;mode&gt; Result mode (precise, very-precise, linear, sensitive, very-sensitive, no-cutoff). Default: linear -s &lt;ranks&gt;, --ranks &lt;ranks&gt; Comma-separated list of ranks to be independently merged (superkingdom,phylum,class,order,family,genus,s pecies,all). Default: species -o &lt;output_file&gt;, --output-file &lt;output_file&gt; Output file -p &lt;output_type&gt;, --output-type &lt;output_type&gt; Output type (tsv, bioboxes). Default: bioboxes --output-parsed-profiles Output parsed and converted profiles for all input files (without cutoff) --detailed Generate an additional detailed output with individual normalized abundances for each tool, where: 0 -&gt; not identified but present in the database, -1 not present in the database. --verbose Verbose output log -v, --version show program's version number and exit About No description, website, or topics provided.
