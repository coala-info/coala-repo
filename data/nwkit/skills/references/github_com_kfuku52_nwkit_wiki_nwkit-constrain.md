## Overview

`nwkit constrain` generates a species-tree-like Newick file for topological constraint. This is a function created to introduce species-tree-like constraints in phylogenetic trees where horizontal gene transfers involve (see [Fukushima & Pollock (2023)](https://www.nature.com/articles/s41559-022-01932-7)). However, this function should also be useful for a standard species tree inference because it can effectively limit the tree search space, if the rank classification is well-established and unequivocal in taxonomic monophyly.

For species-aware matching, leaf labels may be written as `GENUS SPECIES`, `GENUS_SPECIES`, or `GENUS_SPECIES_SUFFIX`. Only the genus and species fields are used for taxonomic lookup; any remaining suffix is preserved in the output but ignored during species-name parsing.

## 1. Generating a constrained tree using the rank information from NCBI Taxonomy

### Command

```
nwkit constrain --backbone ncbi --species_list gene_list.txt --rank family --outfile output.nwk
```

### gene\_list.txt

Expected formats are `GENUS SPECIES`, `GENUS_SPECIES`, or `GENUS_SPECIES_OTHERINFO`. `OTHERINFO` is not used as a query for NCBI Taxonomy.

### Adjusting the taxonomic resolution with --rank

In the above example, `--rank family` was specified. If this is changed to `--rank genus`, the output tree has a higher resolution because of the split of *Eutrema*, *Capsella*, and *Arabidopsis*.

## Combining NCBI Taxonomy and user-provided tree to improve phylogenetic resolution

In the above example, `--backbone user` was specified. By specifying `--backbone ncbi_user` instead (and combined with `--rank`), multifurcations may be resolved by additionally introducing the rank information in NCBI Taxonomy.
