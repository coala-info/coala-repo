# calib

## Overview
Calib is a bioinformatics tool designed for clustering and error-correcting barcode-tagged paired-end reads. Unlike traditional aligners, it utilizes Locality Sensitive Hashing (LSH) and MinHashing to group reads that originated from the same DNA molecule without the computational overhead of full alignment. The workflow typically involves two stages: first, clustering reads based on their barcodes and sequence similarity, and second, generating a consensus sequence from those clusters to eliminate PCR and sequencing errors.

## Clustering Reads
The `calib` executable performs the initial clustering. It requires paired-end FASTQ files and the length of the barcode tag.

### Basic Usage
```bash
calib -f R1.fastq -r R2.fastq -l 8 -o output_prefix
```
*   `-f`: Forward reads FASTQ.
*   `-r`: Reverse reads FASTQ.
*   `-l`: Length of the barcode tag on one mate.
*   `-o`: Prefix for the output cluster file (creates `output_prefixcluster`).

### Advanced Clustering Options
*   **Gzipped Input**: Use `-g` or `--gzip-input` if your FASTQ files are compressed.
*   **Multithreading**: Use `-c` or `--threads` (max 8 recommended). Note that memory usage increases significantly with more threads.
*   **Ignore Sequence Prefixes**: Use `-p` to specify the number of bases to ignore immediately following the barcode tag.
*   **Sorting**: Use `--sort` to group reads of the same cluster together in the output file. This is optional and may slow down the process, but it is not required for the consensus module.

### Tuning Parameters
If default clustering is insufficient, adjust these parameters:
*   `-e`: Error tolerance (positive integer ≤ barcode length).
*   `-k`: K-mer size.
*   `-m`: Minimizer count.
*   `-t`: Minimizer threshold (≤ minimizer count).

## Error Correction (Consensus)
The `calib_cons` executable takes the output from the clustering step to generate high-quality consensus sequences.

### Basic Usage
```bash
calib_cons -c output_prefixcluster -q R1.fastq R2.fastq -o R1_cons R2_cons
```
*   `-c`: The cluster file generated by the `calib` command.
*   `-q`: Space-separated list of the original input FASTQ files.
*   `-o`: Space-separated list of output prefixes for the consensus files.

### Consensus Parameters
*   `-m`: Minimum reads per cluster (default is 2). Clusters with fewer reads than this threshold will not produce a consensus sequence.
*   `-t`: Number of threads to use (default is 4).

## Output Formats
*   **Clustering Output**: A tab-separated file containing `read_cluster_id`, `read_node_id`, `read_id`, and the original FASTQ names, sequences, and quality scores for both mates.
*   **Consensus Output**: For every input FASTQ, two files are generated:
    1.  A FASTQ file containing the consensus records.
    2.  A file containing the Multiple Sequence Alignment (MSA) of the reads within each cluster.
