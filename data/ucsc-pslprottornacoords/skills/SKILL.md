# ucsc-pslprottornacoords

## Overview

The `pslProtToRnaCoords` utility is a specialized tool within the UCSC Genome Browser "Kent" suite designed to bridge the gap between protein-level alignments and nucleotide-level coordinates. When proteins are aligned to genomic or transcriptomic sequences, the resulting PSL files often require transformation to be compatible with RNA-based analysis pipelines. This tool automates the conversion, ensuring that the alignment blocks are correctly scaled from amino acid positions to their corresponding triplet nucleotide positions.

## Usage Instructions

### Basic Command Line
The tool follows the standard UCSC utility pattern of taking an input file and an output file as positional arguments:

```bash
pslProtToRnaCoords input.psl output.psl
```

### Input Requirements
- **Format**: The input must be a standard PSL file.
- **Content**: The input should represent protein-to-nucleotide alignments (typically generated by `blat` using the `-t=dnax -q=prot` or similar protein-query settings).

### Best Practices and Tips
- **Coordinate Scaling**: The tool automatically handles the 3x multiplier required to convert protein residues to nucleotide bases. You do not need to pre-process the coordinates.
- **Permissions**: If you are using a binary downloaded directly from the UCSC server rather than through Conda, ensure the file is executable:
  ```bash
  chmod +x pslProtToRnaCoords
  ```
- **Validation**: It is recommended to run `pslCheck` on the resulting output file to verify that the coordinate transformations have not introduced any format inconsistencies.
- **Pipeline Context**: This tool is most frequently used as a post-processing step for BLAT protein alignments before the data is loaded into a genome browser or used for transcript-level variant mapping.
