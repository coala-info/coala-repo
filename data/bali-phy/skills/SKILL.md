# bali-phy

## Overview
BAli-Phy is a specialized tool for joint Bayesian inference of alignment and phylogeny. Unlike traditional methods that treat alignment as a fixed preprocessing step, BAli-Phy uses Markov Chain Monte Carlo (MCMC) to average over alternative alignments, reducing bias in branch length and positive selection estimates. It is particularly powerful for datasets with significant insertion/deletion (indel) activity or alignment ambiguity.

## Common CLI Patterns

### Basic Joint Estimation
To run a joint estimation of alignment and phylogeny on a single dataset:
```bash
bali-phy dataset.fasta
```

### Specifying Substitution Models
Use the `-m` flag to define the substitution model. BAli-Phy supports complex models including GTR, Gamma distribution for rate heterogeneity, and codon models:
```bash
# DNA with GTR + Gamma
bali-phy dataset.fasta -m GTR+gamma

# Codon model (M3) for protein-coding DNA
bali-phy dataset.fasta -m M3
```

### Multi-gene Analysis
For multiple loci, provide multiple files. BAli-Phy automatically estimates relative rates for each gene:
```bash
bali-phy gene1.fasta gene2.fasta gene3.fasta
```

### Fixed Alignment Inference
If the alignment is already known and only the tree/parameters need estimation:
```bash
bali-phy dataset.fasta --fix-alignment
```

### Ancestral Sequence Reconstruction
BAli-Phy automatically reconstructs ancestral sequences. To ensure these are saved during the MCMC run:
```bash
bali-phy dataset.fasta --ancestors
```

## Expert Tips & Best Practices

*   **Alignment Uncertainty**: Use the Alignment Uncertainty (AU) plots generated by BAli-Phy to identify regions where the alignment is "certain" vs. "uncertain." This is critical for downstream selection analysis.
*   **MCMC Convergence**: Bayesian analysis requires checking for convergence. Always run multiple independent chains and compare the posterior distributions of the log-likelihood and tree topologies.
*   **Indel Models**: BAli-Phy's strength lies in its likelihood-based indel models. If your sequences have many gaps, joint estimation is significantly more accurate than using MUSCLE or MAFFT followed by a tree-builder.
*   **Resource Management**: Joint estimation is computationally intensive. For large datasets, consider starting with a fixed alignment to estimate initial parameters before enabling full joint MCMC.
