# mageck

## Overview
MAGeCK is a computational pipeline designed for the analysis of genome-scale CRISPR-Cas9 knockout screens. It enables researchers to transition from raw FASTQ files to ranked lists of genes that are significantly enriched or depleted in a screen. The tool is particularly effective at handling the noise and overdispersion typical of sequencing data, providing robust statistical rankings (RRA) and visualization capabilities for publication-quality figures.

## Core Workflows

### 1. Count Generation (mageck count)
The first step converts raw sequencing data into a count table.
- **Input**: FASTQ files and a library file (CSV/TSV) containing sgRNA sequences and their target genes.
- **Best Practice**: Use the `--sample-label` flag to provide meaningful names for your replicates to simplify downstream analysis.
- **Command Pattern**:
  ```bash
  mageck count -l library.csv -n output_prefix --sample-label Control,Treatment -sample1 control.fastq -sample2 treatment.fastq
  ```

### 2. Statistical Analysis (mageck test)
Used for simple comparisons between two conditions (e.g., Control vs. Treatment).
- **Method**: Uses the Robust Rank Aggregation (RRA) algorithm.
- **Key Parameters**:
  - `-c`: Control samples (comma-separated indices or labels).
  - `-t`: Treatment samples (comma-separated indices or labels).
- **Command Pattern**:
  ```bash
  mageck test -k count_table.txt -c 0,1 -t 2,3 -n comparison_results
  ```

### 3. Complex Comparisons (mageck mle)
Used for experiments with multiple conditions, time points, or complex designs.
- **Method**: Maximum Likelihood Estimation (MLE) to calculate "beta scores" (similar to log-fold change).
- **Input**: Requires a design matrix file specifying the relationship between samples.
- **Command Pattern**:
  ```bash
  mageck mle -k count_table.txt -d design_matrix.txt -n mle_results
  ```

## Expert Tips
- **Quality Control**: Always check the `.countsummary.txt` file generated by the count command. High "Gini Index" values or low "Mapped Percentage" indicate potential issues with library preparation or sequencing quality.
- **Normalization**: MAGeCK defaults to median normalization. If your screen has a high percentage of active guides (e.g., a focused library), consider using `--norm-method control` with a set of non-targeting controls.
- **Visualization**: Use `mageck-vis` (if available in the environment) or the R-based `MAGeCKFlute` package to generate volcano plots and pathway enrichment analyses from the output files.
