# riblast

## Overview
RIblast is a high-performance bioinformatics tool that utilizes a seed-and-extension algorithm to predict interactions between RNA molecules. It calculates interaction energy by considering both the hybridization energy of the two sequences and the accessibility energy required to open the local secondary structures. The workflow is split into two distinct phases: indexing target sequences into a formatted database and performing the search using query sequences.

## Installation
The recommended method for installation is via Bioconda:
```bash
conda install bioconda::riblast
```

## Workflow and CLI Usage

### 1. Database Construction
Before searching, you must convert your target RNA sequences (e.g., a transcriptome) into the RIblast database format.

**Basic Command:**
```bash
RIblast db -i targets.fa -o target_db
```

**Key Parameters:**
- `-r [0|1|2]`: Repeat masking style. `0`: hard-masking (default), `1`: soft-masking, `2`: no-masking.
- `-w INT`: Maximal span (distance between base pairs). Default is 70; must be at least 20.
- `-d INT`: Minimum accessible length for accessibility approximation. Default is 5.

### 2. RNA Interaction Search (RIS)
Once the database is created, use the `ris` command to find interactions for your query RNA.

**Basic Command:**
```bash
RIblast ris -i query.fa -o output.txt -d target_db
```

**Key Parameters:**
- `-e DBL`: Hybridization energy threshold for the initial seed search. Default is -6.0.
- `-f DBL`: Interaction energy threshold for filtering candidates before gapped extension. Default is -4.0.
- `-g DBL`: Final energy threshold for the reported output. Default is -8.0.
- `-s [0|1]`: Output format style. `0`: simplified (default), `1`: detailed (includes base pair information).

## Output Format
The default output (`-s 0`) provides five columns:
1. **Id**: Interaction identifier.
2. **Query name**: Name of the query RNA.
3. **Target name**: Name of the target RNA from the database.
4. **Interaction Energy**: The calculated total energy (lower is more stable).
5. **Interacted Regions**: Coordinates in the format `(query_start-query_end:target_start-target_end)`.

## Expert Tips and Best Practices
- **Performance**: RIblast is optimized for speed. If running on a large transcriptome, ensure you have sufficient disk space for the database files generated by the `db` command.
- **Sensitivity Tuning**: If you are getting too few hits, consider raising the energy thresholds (e.g., changing `-g -8.0` to `-g -5.0`), though this will increase the number of potential false positives.
- **Repeat Masking**: Use hard-masking (`-r 0`) if your sequences contain many repetitive elements that might lead to non-specific binding predictions.
- **Detailed Analysis**: Use `-s 1` if you need to visualize the specific base-pairing patterns of the predicted interaction, though this will result in significantly larger output files.
