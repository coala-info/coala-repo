# makehub

## Overview

MakeHub is a specialized automation tool that transforms genomic data into a structured format compatible with the UCSC Genome Browser. It eliminates the manual labor of creating "Big" file formats (BigBed, BigWig) and writing the associated configuration files (`hub.txt`, `genomes.txt`, `trackDb.txt`). This skill provides guidance on executing the `make_hub.py` script, preparing compatible input data, and managing dependencies like Samtools and UCSC Kent utilities.

## Core Workflows

### 1. Environment Preparation
MakeHub relies on several external binaries. Ensure the following are in your `$PATH`:
- **UCSC Tools**: `bedToBigBed`, `faToTwoBit`, `gtfToGenePred`, `hgGcPercent`, `ixIxx`, `twoBitInfo`, `wigToBigWig`.
- **Bioinformatics**: `samtools` (version 1.8+ required for `-o` support), `gzip`, and `python3` with `biopython`.
- **Optional**: `bam2wig` (from the AUGUSTUS suite) to accelerate BAM to Wiggle conversion.

### 2. Data Preparation Best Practices
- **Genome FASTA**: Use simple headers. Avoid complex metadata in the header line that might break downstream UCSC tools.
- **Gene Models**: MakeHub supports GTF and GFF3. If using NCBI-sourced GTF files, ensure they are "sane" (standardized) to avoid parsing errors.
- **RNA-Seq (BAM)**: Ensure BAM files are indexed. MakeHub will perform internal sorting and indexing using `samtools sort -@ [threads]` if necessary.
- **Repeat Masking**: MakeHub uses Biopython to identify soft-masked regions (lowercase letters) in your FASTA to automatically generate repeat tracks.

### 3. Common CLI Patterns
While specific flags depend on the version, the standard execution follows this pattern:

```bash
# Basic hub generation
python3 make_hub.py --genome genome.fa --labels labels.txt [options]

# Adding tracks to an existing hub
python3 make_hub.py --genome genome.fa --existing_hub path/to/hub --track_file new_data.gtf
```

### 4. Expert Tips
- **Automated Downloads**: If UCSC binaries are missing from your path, MakeHub attempts to download them automatically during execution. For production environments, it is better to pre-install these to avoid network-related failures.
- **Samtools Compatibility**: Verify your samtools version by checking `samtools sort --help`. If it lists `-o FILE`, it is compatible with MakeHub's subprocess calls.
- **Web Hosting**: Remember that the output directory generated by MakeHub must be placed on a publicly accessible web server (or a password-protected one accessible via URL) for the UCSC Genome Browser to load the hub.
- **GeMoMa Integration**: For GeMoMa GFF3 outputs, use the provided `gemoma_to_augustus_like_gtf.py` script to convert data before hub generation.
