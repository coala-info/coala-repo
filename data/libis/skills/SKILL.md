# libis

## Overview

LiBis (Low input Bisulfite sequencing alignment) is an ultrasensitive alignment method designed to overcome the low mapping ratios typically associated with Whole Genome Bisulfite Sequencing (WGBS) of limited DNA quantities. It is particularly effective for clinical liquid biopsy samples where DNA is fragmented (often around 120bp). The tool functions by dynamically clipping unmapped reads and remapping the fragments to rescue informative reads that standard aligners might discard. Use this skill to execute the LiBis pipeline, optimize clipping parameters, and generate methylation calls and reports.

## Installation and Environment

The preferred method for installing LiBis and its dependencies (FastQC, Bedtools, Trim-galore, MOABS, and Samtools) is via Conda:

```bash
conda config --add channels bioconda
conda config --add channels conda-forge
conda install libis
```

Verify the installation by running `LiBis --help`.

## Command Line Usage

### Basic Alignment
To run a standard alignment with paired-end reads and a reference genome:

```bash
LiBis -n sample1_R1.fq.gz,sample1_R2.fq.gz -r /path/to/reference.fa
```

### Handling Multiple Samples
Separate mate pairs with a comma and separate different samples with a space:

```bash
LiBis -n s1_R1.fq,s1_R2.fq s2_R1.fq,s2_R2.fq -r reference.fa -l Label1 Label2
```

### Optimization and Clipping Parameters
LiBis uses a sliding window to clip and remap reads. You can tune these for specific fragment lengths:

- **Clip Mode (`-c`)**: Enabled by default (1). If you have already performed aggressive trimming, you might disable it (0), though it is the core strength of LiBis.
- **Window Length (`-w`)**: The length of the clipping window (default: 40).
- **Step Size (`-s`)**: The distance the window moves (default: 5).
- **Filter (`-ft`)**: Minimal length for recombined reads (default: 46).

### Downstream Analysis
LiBis can automate methylation calling and reporting:

- **`-mcall`**: Runs the `mcall` module to extract methylation status for each CpG.
- **`-plot`**: Generates a final report with visualization.
- **`-g`**: Specify the genome build (e.g., `hg38`, `mm10`) for plotting purposes.

## Expert Tips and Best Practices

- **Trimming vs. Clipping**: If you use the default Clip Mode (`-c 1`), you generally do not need to perform independent adapter trimming (`-t 0`). Clipping effectively handles the removal of non-mapping sequences.
- **Computational Resources**: Use the `-p` parameter to specify the number of processes. Note that the underlying BSMAP process typically consumes 8 CPUs per instance; plan your `-p` value accordingly to avoid overloading the system.
- **Resuming Workflows**: If you already have BAM files generated by BSMAP, you can skip the initial mapping stage using the `-bam` flag.
- **Overwriting Results**: By default, LiBis checks for existing result folders. Use `-nc` (no check) to skip this and overwrite previous outputs if necessary.
- **Temporary Files**: Use `-gz` if your temporary files are in Gzip format to save disk space, or `--fullmode` if you need to inspect all intermediate files for troubleshooting.
