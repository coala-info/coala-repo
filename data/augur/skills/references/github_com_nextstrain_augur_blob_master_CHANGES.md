# CHANGELOG

## **NEXT**

## 33.0.1 (11 March 2026)

### Bug fixes

* filter, frequencies, refine, subsample: A date value of `XXXX-XX-XX` is now treated the same as no value (empty string). [#1894](https://github.com/nextstrain/augur/issues/1894) @victorlin
* filter, merge: Fixed formatting of the error message shown when there are duplicate sequence ids. [#1954](https://github.com/nextstrain/augur/pull/1954) @victorlin
* filter: Adjusted the error message shown when there are missing weights to mention the option of updating values in metadata. [#1956](https://github.com/nextstrain/augur/pull/1956) @victorlin
* frequencies: Added a proper error message for missing or invalid dates. [#1960](https://github.com/nextstrain/augur/issues/1960) @victorlin
* merge: Added a workaround for a bug in SQLite version 3.52.0. [#1969](https://github.com/nextstrain/augur/pull/1969) @victorlin
* merge: Improved error messages from SQLite. [#1969](https://github.com/nextstrain/augur/pull/1969) @victorlin

## 33.0.0 (26 January 2026)

### Major Changes

* Dropped support for Python 3.9 and added support for Python 3.14. [#1938](https://github.com/nextstrain/augur/pull/1938), [#1939](https://github.com/nextstrain/augur/issues/1939) (@victorlin)
* `augur.utils.write_json` is deprecated and will be removed in a future major version. Users should use `augur.io.write_json` instead. [#1943](https://github.com/nextstrain/augur/pull/1943) (@victorlin)

### Features

* filter, frequencies, refine: Added support in metadata for precise date ranges in `YYYY-MM-DD/YYYY-MM-DD` format. [#1304](https://github.com/nextstrain/augur/issues/1304) (@victorlin)
* refine: Added a new option `--keep-ids` to keep certain tips in the tree regardless of clock filtering. This allows force-inclusion similar to `augur filter`'s `--include` option, and the same file can be used for both. [#1768](https://github.com/nextstrain/augur/issues/1768) (@victorlin)
* `augur.io.write_json` is a new function serving as a replacement for `augur.utils.write_json`. Output minification is controlled by two new parameters `minify` and `minify_threshold_mb`. [#1943](https://github.com/nextstrain/augur/pull/1943) (@victorlin)

### Bug fixes

* export v2: Improved the error message that is displayed when a deprecated coloring key is used. [#1882](https://github.com/nextstrain/augur/issues/1882) (@corneliusroemer)
* export v2: `--no-minify-json` now properly overrides any truthy value in `AUGUR_MINIFY_JSON`. [#1943](https://github.com/nextstrain/augur/pull/1943) (@victorlin)
* export v2: Skip unhashable node attr values with warning message to avoid previously unhandled `TypeError`. [#1948](https://github.com/nextstrain/augur/pull/1948) (@joverlee521)

## 32.1.0 (18 November 2025)

### Features

* augur.io.read\_metadata: Added a new parameter `keep_id_as_column` to keep the resolved id column as a column in addition to setting it as the DataFrame index. [#1917](https://github.com/nextstrain/augur/pull/1917) (@victorlin)
* subsample: Filepaths in the config file can now be relative to the config file's parent directory in addition to the current working directory. Custom directories can also be specified using a new command line option `--search-paths` or environment variable `AUGUR_SEARCH_PATHS`. [#1897](https://github.com/nextstrain/augur/issues/1897) (@victorlin)
* A helper function – `augur.subsample.get_referenced_files` – has been added to optimize usage of `augur subsample` in Snakemake workflows. This is experimental and not yet part of the public API. [#1918](https://github.com/nextstrain/augur/pull/1918) (@victorlin)

### Bug fixes

* filter: Previously, `--query`, `--exclude-where`, and `--include-where` did not work for the id column (`strain`, `name`, or other from `--metadata-id-columns`). This has been fixed. [#1915](https://github.com/nextstrain/augur/issues/1915) (@corneliusroemer, @victorlin)
* export v2: Support export of URLs for non-string values. [#1926](https://github.com/nextstrain/augur/pull/1926) (@joverlee521)

## 32.0.0 (21 October 2025)

### Major Changes

* ancestral, translate: These will now error when the length of any reference gene is indivisible by 3, instead of silently padding with N to translate to 'X'. [#1895](https://github.com/nextstrain/augur/issues/1895) (@victorlin)
* `augur.utils.load_features` is deprecated and will be removed in a future major version. Users should use `augur.io.load_features` instead. [#1912](https://github.com/nextstrain/augur/pull/1912) (@victorlin)

### Features

* `augur curate apply-record-annotations` will now warn if an annotation was unnecessary, often indicative of the upstream data being updated. [#1893](https://github.com/nextstrain/augur/pull/1893) (@jameshadfield)

## 31.5.0 (17 September 2025)

* A new command, `augur subsample`, supports complex subsampling using file-based configuration. See the updated [Filtering and Subsampling guide](https://docs.nextstrain.org/en/latest/guides/bioinformatics/filtering-and-subsampling.html) for a comparison with `augur filter`. [#635](https://github.com/nextstrain/augur/issues/635) (@victorlin)

## 31.4.0 (14 August 2025)

### Features

* schema: Allow parentheses (`()`) in gene names. [#1819](https://github.com/nextstrain/augur/pull/1819) (@kimandrews)
* geolocation rules: Add rules to define region per country to ensure that regions are labelled for all countries. This is especially useful for data sources that do not include region in the metadata. [#1844](https://github.com/nextstrain/augur/pull/1844) (@joverlee521)
* support numpy v2 in addition to v1. [#1855](https://github.com/nextstrain/augur/pull/1855) (@corneliusroemer)
* support for Python 3.13. [#1857](https://github.com/nextstrain/augur/pull/1857) (@corneliusroemer)
* tree: Prefer `iqtree3` binary over `iqtree2` and `iqtree` when available. [#1875](https://github.com/nextstrain/augur/pull/1875) (@joverlee521)
* export v2: URLs encoded in metadata (both TSV and node-data JSONs) will be associated with the value in the exported JSON. Given a column/key `<X>` then a valid URL in a column/key named `<X>__url` will be automatically used. This allows values to be a clickable link when viewed in Auspice. [#1852](https://github.com/nextstrain/augur/pull/1852) (@jameshadfield)

### Bug fixes

* filter: Improved speed of using `--group-by month` on large datasets. [#1845](https://github.com/nextstrain/augur/pull/1845) (@victorlin)
* merge: Added validation to require at least two sequence inputs for merging, consistent with metadata merging behavior. [#1865](https://github.com/nextstrain/augur/issues/1865) (@victorlin)
* validate: Send all log messages to `stderr`. [#1869](https://github.com/nextstrain/augur/pull/1869) (@victorlin)
* validate: only print the entire merged Auspice config to `stderr` when there's a validation error. [#1878](https://github.com/nextstrain/augur/pull/1878)(@joverlee521)

## 31.3.0 (3 July 2025)

### Features

* traits: Added new options `--branch-labels` and `--branch-confidence` to export branch labels for nodes which have a corresponding state change. These are useful for creating streamtrees which convey geographic jumps. [#1814](https://github.com/nextstrain/augur/pull/1814) (@jameshadfield)
* filter, merge: Added a new option `--nthreads` to configure parallelism. Right now, it is only passed to SeqKit, but it may be used for other internal optimizations in the future. [#1833](https://github.com/nextstrain/augur/issues/1833) (@victorlin)
* filter: Added a new option `--skip-checks` to bypass checks for duplicates in sequences and whether ids in metadata have a sequence entry. Mainly useful when working with larger files. [#1833](https://github.com/nextstrain/augur/issues/1833) (@victorlin)
* Added a new `AUG
