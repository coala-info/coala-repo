PhyloRank is intended to aid the manual taxonomic curation of trees inferred from genomes spanning the bacterial or archaeal domain. PhyloRank provides functionality for calculating the relative evolutionary divergence (RED) of taxa in a tree and for finding the best placement of taxonomic labels in a tree. Other functionality is in development and is currently unsupported. Install The simplest way to install this package is through Bioconda: conda install -c bioconda phylorank Alternatively, it can be installed through PyPI: pip install phylorank This package makes use of the numpy , matplotlib , jinja2 (&gt;=2.7.3) , and dendropy (&gt;=4.1.0) Python libraries. These must be install seperately. PhyloRank also uses mpld3 (&gt;=0.2) which has been explicitly added to this package. It also required biolib (&gt;=0.1.0), but this will be install with PhyloRank if you are doing the installation through pip. PhyloRank requires Python 3 starting with v0.1.0 Calculating RED PhyloRank can calculate the relative evolutionary divergence (RED) of taxa in a tree in order to identify taxa at a given taxonomic rank that have conspicuous placements in the tree. This information can be used to refine the placement of taxa in the tree with the aim of taxa at the same rank having similar RED. RED values can be calculated using: &gt;phylorank outliers &lt;input_tree&gt; &lt;taxonomy_file&gt; &lt;output_dir&gt; where &lt;input_tree&gt; is a tree in Newick format that is decorated with taxonomic information, the &lt;taxonomy_file&gt; indicates the taxonomic assignment of each genome in the tree (see File Formats below), and &lt;output_dir&gt; is the location to store all results. This command assumes the taxonomically-decorated tree and taxonomy file are congruent. The taxonomy file is required in order to establish taxonomic affiliations that could not be provided in the tree (e.g., a species with a single representative and consequently no internal node in the tree). The output directory contains a table indicating the RED value calculated for all taxa along with a plot indicating the RED value of taxa at each rank. Decorating a Tree PhyloRank can decorate a tree with taxonomic information. This is accomplished by determining the placement for each taxa that results in the highest F-measure (see McDonald et al., 2012 ). If the provided taxonomy is incongruent with the topology of the tree, only the most cohesive lineage of a polyphyletic group will be labelled and it is possible that taxa at the same rank may be nested within each other (e.g., a lineage may contain two genus labels). The resulting decorated tree can be manually curated to obtain a taxonomy that is consistent with the tree topology. Alternatively, the resulting F-measure scores can be examined to assess the congruency between the taxonomy and tree topology. A tree can be decorated using: &gt;phylorank decorate &lt;input_tree&gt; &lt;taxonomy_file&gt; &lt;output_tree&gt; --skip_rd_refine where &lt;input_tree&gt; is a tree in Newick format, &lt;taxonom_file&gt; indicates the taxonomic assignment of each genome in the tree (see File Formats below), and &lt;output_tree&gt; is the desired name of the decorated tree. The --skip_rd_refine flag indicates that the placement of taxa in the tree should not be adjusted to account for their RED. Adjusting for RED is only recommended once the taxonomy and tree topology have been established as being largely congruent. The file &lt;output_tree&gt;-table indicates the F-measure, precision, and recall for each taxa in the tree and the file &lt;output_tree&gt;-taxonomy gives the assigned taxonomy of each genome in the tree. File formats The taxonomy file is a simple tab-separated values file with two columns indicating the genome ID and Greengenes-style taxonomy string, e.g.: &gt;GCF_001687105.1 d__Bacteria;p__Proteobacteria;c__Alphaproteobacteria;o__Rhodobacterales;f__Rhodobacteraceae;g__Yangia;s__ Example data Example data is provided in the example_data directory.
