# dia_analysis_msstats

## Overview

This workflow is designed for the statistical analysis of Data-Independent Acquisition (DIA) proteomics data using [MSstats](https://toolshed.g2.bx.psu.edu/repos/galaxyp/msstats/msstats/3.20.1.0). It processes two primary inputs: a tabular MSstats-compatible file, typically generated by PyProphet, and a comparison matrix that defines the experimental groups and contrasts for differential expression analysis.

The core of the pipeline utilizes the MSstats tool to perform data normalization and calculate protein-level significance. It generates a comprehensive suite of outputs, including volcano plots, quality control (QC) plots, and comparison plots, alongside the raw processed data and statistical results. These visualizations allow researchers to identify differentially abundant proteins across various biological conditions.

Following the statistical analysis, the workflow includes post-processing steps to refine and visualize specific subsets of the data. Using Grep and Filter tools, it isolates specific protein groups—such as *E. coli* spike-ins—and generates a histogram to analyze their distribution. This makes the workflow particularly useful for benchmarking and training purposes within the [Galaxy Training Network (GTN)](https://training.galaxyproject.org/).

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run galaxy-workflow-dia-msstats-training-msstats-input-tabular.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run galaxy-workflow-dia-msstats-training-msstats-input-tabular.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run galaxy-workflow-dia-msstats-training-msstats-input-tabular.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init galaxy-workflow-dia-msstats-training-msstats-input-tabular.ga -o job.yml`
- Lint: `planemo workflow_lint galaxy-workflow-dia-msstats-training-msstats-input-tabular.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
