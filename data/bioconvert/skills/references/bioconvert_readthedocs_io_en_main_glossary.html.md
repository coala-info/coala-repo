# 9. Glossary[](#glossary "Link to this heading")

Note that formats mentionned here below have dedicated description in the
[Formats](formats.html#formats) section.

ABI[](#term-ABI "Link to this term")
:   File format produced by ABI sequencing machines. Contains the trace data
    which includes probabilities of the four nucleotides. See
    the [ABI](formats.html#format-abi) format page for details.

ASQG[](#term-ASQG "Link to this term")
:   The ASQG format describes an assembly graph. Each line is a tab-delimited
    record. The first field in each record describes the record type. See the
    [ASQG](formats.html#format-asqg) page for details.

BAI[](#term-BAI "Link to this term")
:   The index file related to file generated in the BAM format. (This is a
    non-standard file type.) See the [BAI](formats.html#format-bai) page for details.

BAM[](#term-BAM "Link to this term")
:   Binary version of the Sequence Alignment Map (SAM) format. See the
    [BAM](formats.html#format-bam) format page for details.

BCF[](#term-BCF "Link to this term")
:   Binary version of the Variant Call Format (VCF).
    See [BCF](formats.html#format-bcf) page for details.

BCL[](#term-BCL "Link to this term")
:   BCL is the raw format used by Illumina sequencers. See the [BCL](formats.html#format-bcl) format
    page for details.

BED[](#term-BED "Link to this term")
:   BEDGRAPH/BED format is line-oriented and allows display of continuous-valued
    data. Similar to WIG format.
    See the [BED](formats.html#format-bed) format page for details.

BED3[](#term-BED3 "Link to this term")
:   Variants of the BED format with 4 columns storing the track name,
    start and end positions and values.
    See the [BED4](formats.html#format-bed4) format page for details.

BED4[](#term-BED4 "Link to this term")
:   Variants of the BED format with 4 columns storing the track name,
    start and end positions and values.
    See the [BED4](formats.html#format-bed4) format page for details.

BEDGRAPH[](#term-BEDGRAPH "Link to this term")
:   BEDGRAPH/BED format is line-oriented and allows display of continuous-valued
    data. Similar to WIG format.
    See the [BED](formats.html#format-bed) format page for details.

BIGBED[](#term-BIGBED "Link to this term")
:   An indexed binary version of a BED file
    See [BIGBED](formats.html#format-bigbed) page for details.

BIGWIG[](#term-BIGWIG "Link to this term")
:   Indexed binary version of the Wiggle format.
    See [BIGWIG](formats.html#format-bigwig) page for details.

BPLINK[](#term-BPLINK "Link to this term")
:   Binary version of the PlINK forat used for analyzing genotypic data
    for Genome-wide Association Studies (GWAS).
    See [PLINK binary files (BED/BIM/FAM)](formats.html#format-plink-binary) page for details.

BZ2[](#term-BZ2 "Link to this term")
:   **bzip2** is a file compression program that uses the Burrows–Wheeler algorithm.
    Extension is usually .bz2 See [BZ2](formats.html#format-bz2) page for details.

CLUSTAL[](#term-CLUSTAL "Link to this term")
:   The alignment format of Clustal X and Clustal W. See
    [CLUSTAL](formats.html#format-clustal) page for details.

COV[](#term-COV "Link to this term")
:   A bioconvert format to store coverage in the form of a 3 column
    tab-tabulated file. See [COV](formats.html#format-cov) page for details.

CRAM[](#term-CRAM "Link to this term")
:   A more compact version of BAM files used to store Sequence Alignment
    Map (SAM) format. See [CRAM](formats.html#format-cram) page for details.

CSV[](#term-CSV "Link to this term")
:   A comma-separated values format is a delimited text file that uses a
    comma to separate values. See [CSV](formats.html#format-csv) format page for
    details.

DSRC[](#term-DSRC "Link to this term")
:   A compression tool dedicated to FastQ files
    See [DSRC](formats.html#format-dsrc) page for details.

EMBL[](#term-EMBL "Link to this term")
:   EMBL Flat File Format.
    See [EMBL](formats.html#format-embl) page for details.

FAA[](#term-FAA "Link to this term")
:   FASTA-formatted sequence files containing amino acid sequences
    See [FAA](formats.html#format-faa) page for details.

FASTA[](#term-FASTA "Link to this term")
:   FASTA-formatted sequence files contain either nucleic acid sequence
    (such as DNA) or protein sequence information. FASTA files can also store multiple
    sequences in a single file. See [FASTA](formats.html#format-fasta) page for details.

FASTQ[](#term-FASTQ "Link to this term")
:   FASTQ-formatted sequence files are used to represent high-throughput
    sequencing data, where each read is described by a name, its sequence,
    and its qualities. See [FastQ](formats.html#format-fastq) page for details.

GENBANK[](#term-GENBANK "Link to this term")
:   GenBank Flat File Format.
    See [GENBANK](formats.html#format-genbank) page for details.

GFA[](#term-GFA "Link to this term")
:   Graphical Fragment Assembly format. <https://github.com/GFA-spec/GFA-spec>

GFF2[](#term-GFF2 "Link to this term")
:   General Feature Format, used for describing genes and other features
    associated with DNA, RNA and Protein sequences.
    See [GTF](formats.html#format-gff) page for details.

GFF3[](#term-GFF3 "Link to this term")
:   General Feature Format, used for describing genes and other features
    associated with DNA, RNA and Protein sequences.
    <http://genome.ucsc.edu/FAQ/FAQformat#format3>
    See [GTF](formats.html#format-gff) page for details.

GZ[](#term-GZ "Link to this term")
:   **gzip** is a file compression program based on the DEFLATE algorithm.
    See [GZ](formats.html#format-gz) page for details.

JSON[](#term-JSON "Link to this term")
:   A human-readable data serialization language commonly used in
    configuration files. See [JSON](formats.html#format-json) page for details.

MAF[](#term-MAF "Link to this term")
:   A human-readable multiple alignment format.
    See [MAF (Multiple Alignement Format)](formats.html#format-maf) page for details.

NEWICK[](#term-NEWICK "Link to this term")
:   Plain text minimal format used to store phylogenetic tree.
    See [NEWICK](formats.html#format-newick) page for details.

NEXUS[](#term-NEXUS "Link to this term")
:   Plain text minimal format used to store multiple alignment and
    phylogenetic trees. See [NEXUS](formats.html#format-nexus) page for details.

PAF[](#term-PAF "Link to this term")
:   PAF is a text format describing the approximate mapping positions
    between two set of sequences.

PHYLIP[](#term-PHYLIP "Link to this term")
:   Plain text format to store a multiple sequence alignment.
    See [PHYLIP](formats.html#format-phylip) page for details.

PHYLOXML[](#term-PHYLOXML "Link to this term")
:   XML format to store a multiple sequence alignment.
    See [PHYLOXML](formats.html#format-phyloxml) page for details.

PLINK[](#term-PLINK "Link to this term")
:   Format used for analyzing genotypic data for Genome-wide Association
    Studies (GWAS). See [PLINK flat files (MAP/PED)](formats.html#format-plink-flat) page for details.

QUAL[](#term-QUAL "Link to this term")
:   Sequence of qualities associated with a sequence of nucleotides.
    Associated with FastA file, the original FastQ file can be built back.
    See [QUAL](formats.html#format-qual) page for details.

SAM[](#term-SAM "Link to this term")
:   Sequence Alignment Map is a generic nucleotide alignment format that
    describes the alignment of query sequences or sequencing reads to a reference
    sequence or assembly. See [SAM](formats.html#format-sam) page for details.

SCF[](#term-SCF "Link to this term")
:   Standard Chromatogram Format, a binary
    chromatogram format described in Staden package documentation SCF file format.

SRA[](#term-SRA "Link to this term")
:   The Sequence Read Archive (SRA) is a website that stores
    sequencing data at <https://www.ncbi.nlm.nih.gov/sra>
    It is not a format per se. See [SRA](formats.html#format-sra) page for details.

STOCKHOLM[](#term-STOCKHOLM "Link to this term")
:   Stockholm format is a multiple sequence alignment format used to store
    multiple sequence alignment. See [STOCKHOLM](formats.html#format-stockholm) page for details.

TSV[](#term-TSV "Link to this term")
:   A tab-separated values format is a delimited text file that uses a
    tab character to separate values. See [TSV](formats.html#format-tsv) format page for
    details.

TWOBIT[](#term-TWOBIT "Link to this term")
:   **2bit** file stores multiple DNA sequences (up to 4 Gb total) in a
    compact randomly-accessible format. The file contains masking information
    as well as the DNA itself. See [TWOBIT](formats.html#format-twobit) format page for
    details.

VCF[](#term-VCF "Link to this term")
:   Variant Call Format (VCF) is a flexible and extendable format for
    storing variation in sequences such as single nucleotide variants,
    insertions/deletions, copy number variants and structural variants.
    See [VCF](formats.html#format-vcf) page for details.

WIG[](#term-WIG "Link to this term")
:   Synonym for the wiggle (WIG) format. See [WIG](formats.html#format-wig).

WIGGLE[](#term-WIGGLE "Link to this term")
:   The wiggle (WIG) format stores dense, continuous data such as GC percent,
    probability scores, and transcriptome data. See [WIG](formats.html#format-wig) page
    for details.

XLS[](#term-XLS "Link to this term")
:   Spreadsheet file format (Microsoft Excel file format).
    See [XLS](formats.html#format-xls) page for details.

XLSX[](#term-XLSX "Link to this term")
:   Spreadsheet file format defined in the Office Open XML specification.
    See [XLSX](formats.html#format-xlsx) page for details.

XMFA[](#term-XMFA "Link to this term")
:   TODO

YAML[](#term-YAML "Link to this term")
:   A human-readable data serialization language commonly used in
    configuration files. See <https://en.wikipedia.org/wiki/YAML>
    See [YAML](formats.html#format-yaml) page for details.
