# tablet

## Overview
Tablet is a high-performance graphical viewer designed for the visual exploration of next-generation sequencing data. It provides a lightweight interface for navigating large-scale genomic alignments, supporting both packed and stacked read formats. Beyond visualization, it includes utilities for converting legacy assembly formats and generating coverage statistics, making it a vital tool for validating variants and assessing assembly quality.

## Command Line Usage and Best Practices

### Opening Assemblies
While Tablet is primarily a GUI application, it can be launched from the command line to open specific datasets immediately.
- **Basic Launch**: `tablet [assembly_file] [reference_file]`
- **Specific Position**: You can pass coordinates to jump to a specific contig or base position upon opening.
- **macOS Multiple Instances**: To run more than one instance on macOS, use:
  `open -n /Applications/Tablet.app`

### Memory Management
Tablet's performance on large datasets is heavily dependent on the Java Heap size.
- **64-bit Systems**: Default is usually 4GB.
- **Adjustment**: If encountering "Out of Memory" errors on massive BAM files, adjust the `-Xmx` parameter in the executable's configuration or via the command line if running the JAR directly:
  `java -Xmx8g -cp tablet.jar tablet.io.utils.MaqToAce [options]`

### Assembly Conversion Patterns
Tablet provides or supports several workflows to bring data into its preferred SAM/BAM format.

#### Maq to ACE Conversion
If working with Maq alignments, use the `maqtoace` utility included in the Tablet `utils` directory:
```bash
maqtoace -maqtxt=assembly.txt -fastq=consensus.fastq -dir=. -filename=assembly.ace
```

#### SAM to BAM Preparation
For optimal performance in Tablet, SAM files must be converted to indexed BAM files.
1. **Index Reference**: `samtools faidx reference.fasta`
2. **Convert to BAM**: `samtools view -b -S -t reference.fasta.fai -o assembly.bam assembly.sam`
3. **Sort**: `samtools sort assembly.bam -o assembly_sorted.bam`
4. **Index BAM**: `samtools index assembly_sorted.bam`

### Coverage Analysis
Use the `coveragestats.py` script (often provided in the additional downloads) to process coverage export files generated by Tablet:
- **Usage**: `python coveragestats.py [tablet_coverage_file]`
- This is useful for generating summary statistics of a mapping run after visual inspection.

## Expert Tips
- **BAM Indexing**: If Tablet shows "?" for read counts in a BAM assembly, ensure the index (.bai) was generated with a modern version of samtools (0.1.8 or later).
- **CIGAR Features**: Tablet automatically generates "CIGAR-I" features for insertions. Use the Features Table to filter and jump to these events quickly.
- **Sticky Highlights**: For variant validation, enable "Sticky Highlights" in the overlay options to keep regions of interest marked while switching between different contigs.
- **Large Scale Navigation**: Use the "Superzoom" feature (zoom levels supporting hundreds of thousands of bases) to identify coverage gaps or anomalies across entire contigs before diving into base-level reads.
