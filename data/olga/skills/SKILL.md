# olga

## Overview
The `olga` skill provides a specialized interface for the Optimized Likelihood estimate of immunoGlobulin Amino-acid sequences (OLGA) software. It allows for the calculation of how likely a specific immune receptor sequence is to be generated by the random V(D)J recombination process. This is essential for identifying "public" clones, detecting convergent evolution in repertoires, and distinguishing between generation bias and antigen-driven selection.

## Core CLI Commands

### Computing Generation Probabilities (Pgen)
Use `olga-compute_pgen` to calculate the probability of specific sequences.

- **Single Sequence**:
  `olga-compute_pgen --humanTRB CASSLGRDGGHEQYF`
- **Batch Processing**:
  `olga-compute_pgen --humanTRB -i input_sequences.tsv -o output_pgens.tsv`
- **Performance Optimization**:
  The tool uses `numba` for JIT compilation by default. If processing large datasets, ensure `numba` is installed for up to 127x speedup. Use `--skip_fast_pgen` only if troubleshooting numerical stability.

### Generating Synthetic Sequences
Use `olga-generate_sequences` to create random sequences based on established genomic models.

- **Generate to Stdout**:
  `olga-generate_sequences --humanTRB -n 5`
- **Save to File**:
  `olga-generate_sequences --humanTRB -o synthetic_repertoire.tsv -n 1000`

## Model Selection
You must specify a model flag for every command.

| Flag | Description |
| :--- | :--- |
| `--humanTRA` / `--humanTRB` | Human T-cell Alpha (VJ) or Beta (VDJ) |
| `--mouseTRA` / `--mouseTRB` | Mouse T-cell Alpha (VJ) or Beta (VDJ) |
| `--humanIGH` / `--humanIGK` / `--humanIGL` | Human B-cell Heavy, Kappa, or Lambda |
| `--mouseIGH` / `--mouseIGK` / `--mouseIGL` | Mouse B-cell Heavy, Kappa, or Lambda |

### Custom Models
To use models inferred from specific datasets (e.g., via IGoR), point to the directory containing `model_params.txt`, `model_marginals.txt`, and the V/J anchor CSV files:
- `olga-compute_pgen --set_custom_model_VDJ path/to/model/ ...`

## Best Practices
- **Input Formats**: When using `-i`, OLGA expects a TSV/text file. For amino acid sequences, ensure they are in-frame and follow standard CDR3 definitions (starting with the conserved Cysteine and ending with the conserved Phenylalanine/Tryptophan).
- **Memory Management**: For extremely large repertoires, process files in chunks using the `-m` flag to limit the number of lines read at once.
- **Nucleotide vs. Amino Acid**: OLGA can compute Pgen for both. Note that amino acid Pgen is the sum of all possible underlying nucleotide recombinations that result in that specific peptide.
