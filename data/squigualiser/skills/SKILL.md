# squigualiser

## Overview
squigualiser is a specialized tool for the interactive visualization of raw nanopore sequencing signals. It bridges the gap between raw electrical current data (stored in POD5/FAST5/SLOW5 formats) and genomic alignments. It allows researchers to inspect how specific DNA/RNA sequences correspond to the raw signal, which is essential for debugging basecalling errors, identifying modifications, and verifying structural variants.

## Installation and Setup
The most efficient way to install squigualiser is via Conda or using precompiled binaries for Linux/macOS.

```bash
# Conda installation
conda install bioconda::squigualiser

# Python pip installation (requires Python >= 3.8)
pip install squigualiser
```

**Note:** Google Chrome is the recommended browser for viewing the interactive HTML plots generated by the tool.

## Common CLI Patterns

### 1. Signal-to-Reference Pileup View
Use `plot_pileup` to visualize multiple raw signals aligned to a specific genomic region. This is the most common command for inspecting variants or modifications.

```bash
squigualiser plot_pileup \
    -f reference.fasta \
    -s signals.blow5 \
    -a alignment.bam \
    -o output_dir \
    --region chr1:92,778,040-92,782,120 \
    --tag_name "sample_id"
```

### 2. Single Read Signal-to-Read Visualization
Use `plot` to inspect the signal-to-base alignment for a specific read ID. This requires a signal-to-read alignment file (typically a PAF file generated by `f5c resquiggle`).

```bash
squigualiser plot \
    -f reads.fastq \
    -s signals.blow5 \
    -a resquiggle.paf \
    -r "read_id_string" \
    -o output_dir
```

### 3. Signal-to-Reference (Single Read)
To visualize how a single read's signal aligns to the reference genome (using `f5c eventalign` or `uncalled4` output):

```bash
squigualiser plot \
    -f reference.fasta \
    -s signals.blow5 \
    -a eventalign.bam \
    -r "read_id_string" \
    -o output_dir
```

## Expert Tips and Best Practices

### Data Preparation
*   **Signal Format:** While FAST5 and POD5 are supported, converting to SLOW5/BLOW5 using `slow5tools` is highly recommended for significantly faster random access and smaller file sizes.
*   **Alignment Requirements:** squigualiser does not perform the initial signal-to-base alignment. You must provide alignment files generated by external tools:
    *   For signal-to-read: Use `f5c resquiggle`.
    *   For signal-to-reference: Use `f5c eventalign`, `uncalled4 align`, or basecaller move tables (from Dorado/Guppy).

### Visualization Enhancements
*   **Signal Scaling:** If the signal appears compressed or offset, use the `--scale_hi` or `--scale_low` flags to adjust the Y-axis.
*   **Base Shifting:** If the signal peaks do not align perfectly with the expected base positions, use `--base_shift` to manually adjust the offset.
*   **BED Annotations:** You can overlay genomic features (like methylation sites or genes) onto the pileup view using the `--bed` flag.

### Performance
*   When plotting large regions in `plot_pileup`, the resulting HTML file can become very large. Limit the `--region` size or the number of reads to maintain browser responsiveness.
