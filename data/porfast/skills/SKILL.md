## Overview
pORFast is a bioinformatics tool designed to efficiently extract Open Reading Frames (ORFs) from paired-end sequencing data in FASTQ format. It is particularly useful for identifying potential protein-coding sequences within genomic or transcriptomic datasets generated by Illumina sequencing.

## Usage Instructions

pORFast is a command-line tool. The primary function is to process FASTQ files and identify ORFs.

### Basic Usage

The most common usage involves specifying the input FASTQ files and an output directory.

```bash
porfast -f1 <forward_fastq_file> -f2 <reverse_fastq_file> -o <output_directory>
```

### Key Options

*   `-f1 <forward_fastq_file>`: Path to the forward read FASTQ file.
*   `-f2 <reverse_fastq_file>`: Path to the reverse read FASTQ file.
*   `-o <output_directory>`: Directory where the results will be saved.
*   `--min-len <length>`: Minimum length of the ORF to report. Defaults to 100 bp.
*   `--max-len <length>`: Maximum length of the ORF to report. Defaults to 1000 bp.
*   `--min-score <score>`: Minimum score for an ORF to be considered. Defaults to 0.
*   `--threads <number>`: Number of threads to use for parallel processing.

### Expert Tips

*   **Minimum and Maximum Lengths**: Adjust `--min-len` and `--max-len` based on your expected gene sizes. For example, if you are looking for small coding sequences, you might decrease `--min-len`.
*   **Scoring**: The `--min-score` parameter can be used to filter out less confident ORFs. Experiment with this value if you have a high number of spurious hits.
*   **Parallel Processing**: For large datasets, utilize the `--threads` option to significantly speed up the analysis.
*   **Output**: The output directory will contain files detailing the identified ORFs, including their start and end positions, strand, and length.
