# SNP indel variant quality metrics and filtering (v2.4,v2.5)

[Jump to bottom](#wiki-pages-box)

Hannes Pétur Eggertsson edited this page Jul 21, 2021
·
[5 revisions](/DecodeGenetics/graphtyper/wiki/SNP-indel-variant-quality-metrics-and-filtering-%28v2.4%2Cv2.5%29/_history)

## Overview

The graphtyper output VCFs comply with the variant-call format (VCF) specifications (v4.2). The output VCF files include a simple filter, specified in the FILTER field of the VCF. For a more strict filtering threshold, it is possible to additionally filter based on the INFO/LOGF field, a pre-trained logistic regression classifier based on the GIAB truth sets. The scores of the classifier are between 0 and 1, higher value indicates higher variant quality. For single sample calling it is recommended to keep only on the "FILTER=PASS" variants and additionally, in larger genotyping runs, we suggest to keep only alternative allelese with INFO/LOGF > 0.5. This threshold might be adjusted for different applications.

For training INFO/LOGF, we genotyped a set of 1250 individuals, the seven GIAB samples and 1243 Icelanders. If a variant was in the GIAB high confidence regions and was called in any of the GIAB samples, it was labelled either as a true positive or false negative depending if it was present in the truth sets. A few variants were labelled as both true positive and a false negative and we removed those from the training. The calculation for the full trained LOGF model is:

```
abhet_bins = [-6.03446, -6.03446, -1.35948, -0.84956, -0.28956, 0, -1.05013, -1.35024, -1.34475, -3.74512]
sbalt_bins = [-0.32486, -0.32486, -0.25342, -0.32696, 0.02442, 0, -0.33522, -0.41332, -0.74043, -1.60844]

pwr = -29.28908 + ABHom * 23.12909 + CRBySeqDepth * (-10.22658) + MQ * 0.01024 + PASS_ratio * 0.85320 + GTYield * 4.91178 + QD * 0.23215 + abhet_bins[ABHet * 10] + sbalt_bins[SBAlt * 10]
logf = 1.0 / (1.0 + exp(-pwr))
```

GTYield is defined NumberOfCallsGenotyped/NumberOfSamples. Other metrics used are defined below.

### INFO fields

The output includes several other metrics which may be used for further filtering or to develop new classifiers, these include:

* **ABHet**: Heterozygous allele balance. Calculated as read count of c2/(c1+c2) where the called genotype is c1/c2 and c1 != c2. The value is -1 if there are no heterozygous calls.
* **ABHom**: Homozygous allele balance. Calculated as read count of c1/(c1+O) where c1 is the called allele and O is the total depth supporting all other alleles. The value is -1 if there are no homozygous calls.
* **CR**: Total number of clipped reads in graph alignment.
* **QD**: Variant quality by depth. Here, quality is the PHRED likelihood of the call being homozygous reference weighted and depth is the total depth supporting alternative alleles. Alt. depth is at most 10 and quality is at most 25\*depth, thus maximum QD is 25. These limits were since the PHREDs only goes up to 255.
* **SB**: read strand bias. Calculated as SB=F/(F+R) where F and R and forward and reverse oriented reads, respectively.
* **SBAlt**: read strand bias of reads supporting one of the alternative alleles. Calculated the same as SB (but using only alternative supporting reads).
* **SeqDepth**: Total accumulated sequencing depth over all the samples.
* **MQ**: root-mean square mapping quality of reads supporting at least one allele.
* **PASS\_AC**, **PASS\_AN**, **PASS\_ratio**: A call is PASSed if GQ>40 (SNP/indel calling only). PASS\_AC is allele count of PASSed called for each alternative allele, PASS\_AN is the number of allele calls with a PASS genotype, and PASS\_ratio is the ratio of PASSed calls.

Then there are several other metrics are defined in the VCF header.

### FORMAT fields

* **GT**: Genotype call as specified in the VCF specifications. All graphtyper genotypes are unphased. If a site has a zero depth, the output genotype will be "./.". Otherwise, the called genotype is the genotype with the PL value "0" (see PL). In case there are more than one genotype with PL==0, the first genotyping the list of PL values is given as the genotype call. These ambigous calls can be filtered on the GQ field.
* **AD**: Allele depth. AD is a list of value of length equal to the number of alleles (REF+ALTs). Number of reads given in the AD field specify how many reads graphtyper considers to uniquely support the allele.
* **MD**: Multiple allele depth. A single number indicating how many reads supported more than one allele. These reads do not affect AD counts but will affect the called genotype.
* **DP**: Depth. Total number of reads that support at least one allele. DP=sum(AD)+MD. Different from some other variant callers, all reads overlapping the site affect the genotype likelihoods, however, they have different weights in the model depending on their quality metrics (base quality, mapping quality, number of mismatches between read graph alignment, clipping in graph alignment).
* **GQ**: Conditional genotype quality as specified in the VCF specifications. −10log10 p(genotype call is wrong, conditioned on the site’s being variant). This is the approximated in graphtyper to be the second lowest PL value. If more than one PL value is zero, then GQ=0.
* **PL**: Phred scaled normalized genotype likelihoods rounded to the closest integer. If A is the allele in REF and B,C,... are the alleles as ordered in ALT, the ordering of genotypes for the likelihoods is given by: F(j/k) = (k\*(k+1)/2)+j.
