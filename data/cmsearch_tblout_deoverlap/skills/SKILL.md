# cmsearch_tblout_deoverlap

## Overview

The `cmsearch_tblout_deoverlap` tool provides a specialized Perl script, `cmsearch-deoverlap.pl`, designed to resolve hit redundancies in Infernal (v1.1x) tabular output. When searching genomic sequences with Covariance Models (CMs), it is common for multiple hits to overlap on the same strand and coordinates. This tool automates the "de-overlapping" process by comparing bit scores of overlapping hits and retaining only the highest-scoring one. It is particularly useful for cleaning up Rfam-based searches where hits from different models within the same "clan" (related RNA families) need to be reconciled.

## Command Line Usage

The primary script is `cmsearch-deoverlap.pl`. It typically operates on the tabular output (`--tblout`) generated by Infernal's `cmsearch` or `cmscan`.

### Basic Syntax
```bash
perl cmsearch-deoverlap.pl [options] <tblout_file>
```

### Common Options
- `--cmscan`: Use this flag if the input file was generated by `cmscan` rather than `cmsearch`.
- `--claninfo <file>`: Provide a clan information file (e.g., `ribo.claninfo`). This allows the script to treat hits from different models within the same clan as competing for the same locus, keeping only the best hit across the entire clan.
- `--overlapout <file>`: Directs the script to write details about the removed (lower-scoring) overlapping hits to a specified file for auditing.
- `--invclan`: Modifies how same-clan criteria are checked during the de-overlapping process.
- `-v`: Verbose mode; prints detailed information about each overlap found and resolved to standard output.
- `--fcmsearch`: Forces the script to treat the input as `cmsearch` format, even if the header suggests otherwise.

## Expert Tips and Best Practices

- **Input Preparation**: Ensure your Infernal search was run with the `--tblout` option. The script cannot process the standard alignment output; it requires the fixed-column tabular format.
- **Clan-Aware Filtering**: When annotating a genome using the full Rfam database, always use the `--claninfo` option. Without it, the script will only de-overlap hits from the *same* model. With it, it will correctly resolve overlaps between related models (e.g., different SSU rRNA models).
- **Handling cmscan**: Note that `cmscan` and `cmsearch` have slightly different tabular formats. If you are processing `cmscan` results, the `--cmscan` flag is mandatory to prevent parsing errors.
- **Sorting**: The script internally handles sorting by bit score to ensure the highest-scoring hit is prioritized. You do not need to pre-sort your `--tblout` file.
- **Validation**: Use the `-v` (verbose) flag during initial pipeline development to verify that the overlaps being removed are indeed the ones you expect based on your bit score thresholds.
