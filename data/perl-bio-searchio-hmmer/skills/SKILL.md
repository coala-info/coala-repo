# perl-bio-searchio-hmmer

## Overview
The `Bio::SearchIO::hmmer` module is a Perl-based interface designed to parse the complex text output generated by the HMMER software suite. It provides a standardized way to extract hits, scores, E-values, and alignments from HMMER2 and HMMER3 reports, allowing for automated bioinformatics pipeline integration.

## Usage Guidelines

### Basic Parsing Pattern
To use this parser within a Perl environment, initialize the `Bio::SearchIO` object with the `hmmer` format specification.

```perl
use Bio::SearchIO;

# Initialize the parser for a HMMER result file
my $searchio = Bio::SearchIO->new(
    -format => 'hmmer',
    -file   => 'results.hmmscan'
);

# Iterate through results (Result objects)
while (my $result = $searchio->next_result) {
    print "Query: ", $result->query_name, "\n";
    
    # Iterate through hits (Hit objects)
    while (my $hit = $result->next_hit) {
        print "  Hit: ", $hit->name, " E-value: ", $hit->significance, "\n";
        
        # Iterate through High-Scoring Pairs (HSP objects)
        while (my $hsp = $hit->next_hsp) {
            print "    Score: ", $hsp->score, "\n";
        }
    }
}
```

### Version Compatibility
*   **HMMER3 Support**: This parser is compatible with the default output of HMMER3 (e.g., `hmmsearch`, `hmmscan`).
*   **HMMER2 Support**: It maintains backward compatibility for older HMMER2 `hmmpfam` and `hmmsearch` outputs.
*   **Format Detection**: While `Bio::SearchIO` often guesses formats, explicitly setting `-format => 'hmmer'` is recommended for HMMER-specific files to ensure the correct parser engine is engaged.

### Best Practices
*   **Memory Management**: For very large HMMER output files, use the iterative `next_result` approach shown above rather than loading the entire file into memory.
*   **Data Extraction**: Use `$hit->significance()` to retrieve E-values and `$hit->bits()` for bit scores, as these are the primary metrics for HMMER result validation.
*   **Output Handling**: If the HMMER output was generated with the `--tblout` or `--domtblout` flags, ensure you are using the standard human-readable output file with this parser, as tabular formats may require different `SearchIO` handlers.
