# VVV2_align_PE

## Overview

The **VVV2_align_PE** workflow is a comprehensive pipeline designed for paired-end viral sequence analysis. It performs read preprocessing, reference alignment, variant calling, and functional annotation. The process begins with quality control and trimming via [fastp](https://toolshed.g2.bx.psu.edu/repos/iuc/fastp/fastp/0.23.2+galaxy0), followed by alignment to a reference genome using [BWA-MEM](https://toolshed.g2.bx.psu.edu/repos/devteam/bwa/bwa_mem/0.7.12.1).

Variant calling is handled through a combination of [bcftools](https://toolshed.g2.bx.psu.edu/repos/iuc/bcftools_call/bcftools_call/1.4.0) and [VarDict](https://toolshed.g2.bx.psu.edu/repos/iuc/vardict_java/vardict_java/1.8.3+galaxy1), allowing for the identification of SNPs and complex mutations. A critical requirement for this workflow is the manual import of a **Ploidy file** (available in "Données partagées") to ensure bcftools correctly processes haploid viral data.

The workflow also integrates [VADR](https://toolshed.g2.bx.psu.edu/repos/ftouzain/vadr/vadr/0.1.1) for viral annotation, which requires the selection of an appropriate model based on the target virus. Final outputs include consensus sequences, detailed variant reports, and visual summaries of SNP proportions and CDS locations generated by the `vvv2_display` tool.

Ensure your input paired-end reads are organized as a dataset collection to streamline processing through fastp and BWA-MEM. You must manually import the haploid ploidy file from "Shared Data" into your history to satisfy the bcftools call requirement, and remember to select the specific VADR model that matches your target virus. For automated testing or execution, you can initialize a job template using `planemo workflow_job_init` to generate a `job.yml` file. Refer to the `README.md` for comprehensive details on parameter tuning and specific tool configurations.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run 20231016_Galaxy-Workflow-VVV2_align_PE.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run 20231016_Galaxy-Workflow-VVV2_align_PE.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run 20231016_Galaxy-Workflow-VVV2_align_PE.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init 20231016_Galaxy-Workflow-VVV2_align_PE.ga -o job.yml`
- Lint: `planemo workflow_lint 20231016_Galaxy-Workflow-VVV2_align_PE.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
