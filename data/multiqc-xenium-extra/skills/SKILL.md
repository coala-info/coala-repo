# multiqc-xenium-extra

## Overview
The `multiqc-xenium-extra` plugin extends the core MultiQC functionality to provide deeper insights into Xenium spatial transcriptomics experiments. While standard MultiQC can parse basic XOA summary files, this "extra" module is designed to capture more granular data points that are critical for assessing the success of a spatial run, including probe performance, cell segmentation quality, and transcript density across the tissue section. It is essential for bioinformaticians performing high-throughput spatial analysis who require a single, unified report to identify batch effects or sample-specific failures in Xenium datasets.

## Usage Guidelines

### Installation
The plugin is typically installed via Bioconda to ensure all dependencies (including MultiQC itself) are correctly resolved.
```bash
conda install -c bioconda multiqc-xenium-extra
```

### Basic Execution
Once installed, the plugin registers itself with MultiQC. You do not need to call a specific "xenium" command; instead, run MultiQC on the directory containing your Xenium output folders (e.g., the `outs/` directories).

```bash
multiqc .
```

### Targeted Analysis
If your directory structure is complex, point MultiQC specifically to the Xenium output directories to speed up the search process:
```bash
multiqc path/to/xenium_run1/outs/ path/to/xenium_run2/outs/
```

### Expert Tips and Best Practices
- **Input Detection**: The plugin looks for specific files generated by the Xenium pipeline, such as `experiment.xenium`, `metrics_summary.csv`, and cell-level metadata. Ensure you are running MultiQC on the full `outs/` folder to allow the plugin to find all necessary components.
- **Sample Naming**: MultiQC uses the directory name or the internal experiment ID for sample naming. If your reports show generic names, use the `--sample-names` or a `multiqc_config.yaml` file to map file paths to human-readable sample IDs.
- **Combining with Other Tools**: In a typical spatial workflow, you may have upstream sequencing data (e.g., for H&E image processing or orthogonal validation). Run MultiQC on the root of your project to combine Xenium metrics with FastQC or Star results in one report.
- **Exporting Data**: Use the `-m xenium_extra` flag if you want to force the module to run or check for its specific output. To get the raw aggregated data for downstream R/Python analysis, check the `multiqc_data/` folder for `multiqc_xenium_extra.txt`.
