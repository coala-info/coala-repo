# Differential peak analysis with SnapATAC2

## Overview

This workflow performs differential peak analysis on single-cell ATAC-seq data using [SnapATAC2](https://toolshed.g2.bx.psu.edu/repos/iuc/snapatac2_peaks_and_motif/snapatac2_peaks_and_motif/2.6.4+galaxy1). It requires a cell-type-annotated AnnData object and a chromosome sizes file as primary inputs. The process begins by executing peak calling via MACS3 specifically for the defined cell types within the dataset.

Following peak calling, the workflow constructs a cell-by-peak matrix to facilitate downstream statistical analysis. It performs differential accessibility tests to compare either two specific cell types or a single cell type against a background of all other cells. This allows for the precise identification of genomic regions that are uniquely accessible in specific populations.

The final stages of the workflow focus on identifying and visualizing differentially accessible marker regions. The primary outputs include a comprehensive list of marker regions and associated plots generated by [SnapATAC2 Plotting](https://toolshed.g2.bx.psu.edu/repos/iuc/snapatac2_plotting/snapatac2_plotting/2.6.4+galaxy1), providing a clear overview of the epigenetic signatures defining each cell cluster.

Ensure the input AnnData file is in `.h5ad` format and contains the necessary cell-type annotations within the metadata to allow for accurate peak calling and differential testing. The chromosome sizes file must be a tab-delimited `.txt` file matching the reference genome used during the initial mapping steps. For automated execution and reproducibility, you can generate a `job.yml` template using `planemo workflow_job_init`. Please refer to the `README.md` for comprehensive details on specific parameter requirements and cell-type naming conventions.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Differential_peak_analysis_with_SnapATAC2.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Differential_peak_analysis_with_SnapATAC2.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Differential_peak_analysis_with_SnapATAC2.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Differential_peak_analysis_with_SnapATAC2.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Differential_peak_analysis_with_SnapATAC2.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
