# ultraheatmap

## Overview

`ultraheatmap` is a specialized tool designed to enhance the production of heatmaps typically generated by deepTools. It simplifies the process of creating "ordered" matrices—where genomic regions are clustered based on the signal of specific reference samples—and allows for the integration of orthogonal data, such as gene expression levels, directly into the genomic signal matrix. This tool is particularly useful for researchers who need to visualize ChIP-seq or ATAC-seq data alongside associated transcriptomic data in a single, cohesive figure.

## Installation and Setup

The tool is best managed via Conda to ensure all deepTools dependencies are correctly resolved.

```bash
conda create -n ultraheatmap python=3.10
conda activate ultraheatmap
conda install -c bioconda -c conda-forge ultraheatmap
```

## Core CLI Usage Patterns

### 1. Computing an Ordered Matrix
The `computeOrderedMatrix` command generates a deepTools-compatible matrix but adds the ability to cluster regions based on a subset of the provided samples.

*   **Basic Clustering:** Use `--kmeans` or `--hclust` to define the number of clusters.
*   **Reference-Based Sorting:** Use the `-g` (or `--groupUsingSamples`) flag to specify 1-based indices of the bigwig files that should be used for the initial sorting/clustering.
*   **Quick Visualization:** Use `-op` to generate an intermediate heatmap image (png, pdf, svg) to verify clustering results before proceeding to final publication-quality plots.

**Example Pattern:**
```bash
computeOrderedMatrix -S signal1.bw signal2.bw signal3.bw \
  -R regions.bed \
  -o clustered_matrix.gz \
  -g 1 2 \
  --kmeans 3 \
  -a 1000 -b 1000 \
  -op check_clustering.png
```
*In this example, only the first two bigwigs (signal1 and signal2) drive the 3-way k-means clustering, but the signal for all three is included in the final matrix.*

### 2. Adding Orthogonal Features
The `addFeatureToMatrix` command appends external tabular data (like a CSV of gene expression values) to an existing deepTools matrix.

*   **Requirement:** You must have a pre-computed matrix and a table where IDs match the genomic regions in the matrix.
*   **Annotation:** Use `--genomeGtf` to provide gene models if the matrix needs to be linked to the closest genes for expression mapping.

## Expert Tips and Best Practices

*   **Sample Indexing:** Remember that the `-g` flag uses **1-based indexing**. If you have 5 bigwig files and want to cluster based on the last two, use `-g 4 5`.
*   **Region Handling:** While you can provide multiple BED files to `-R`, be aware that specific "per-file" group information may be merged or lost during the clustering process. It is often better to provide a single merged BED file if you intend to use `--kmeans`.
*   **Flanking Regions:** Use `-a` (after/downstream) and `-b` (before/upstream) to define the window around your regions of interest. For TSS-centered heatmaps, 1000-3000bp is standard.
*   **Performance:** Use the `-p` flag to specify the number of processors. You can use `-p max` to utilize all available cores for matrix computation.
*   **Matrix Reuse:** Always save the sorted regions using `--outFileSortedRegions`. This allows you to generate additional heatmaps for different marks while maintaining the exact same row order and clustering.
