# pypairs

## Overview
`pypairs` is a Python-based implementation of the Pairs algorithm, specifically optimized for single-cell transcriptomics. It transforms gene expression data into a robust classification model by identifying pairs of genes that exhibit consistent relative expression patterns within specific cell states. While widely recognized for cell cycle phase prediction, it serves as a general-purpose supervised classifier that integrates seamlessly with the `Scanpy` ecosystem and `AnnData` structures.

## Core Workflow
The tool follows a two-step supervised learning process:

### 1. Marker Identification (Sandbagging)
The `sandbag` process identifies gene pairs that characterize target classes (e.g., different phases of the cell cycle).
- **Input**: Labeled training data. This can be an `AnnData` object or a count matrix with associated class labels.
- **Logic**: It searches for gene pairs (Gene A, Gene B) where Gene A is consistently expressed higher than Gene B in one class, but not in others.
- **Key Parameter**: `fraction`. This defines the stringency of the marker selection. For example, `fraction=0.6` means a pair is only kept if the relationship holds true in at least 60% of the cells in that class.

### 2. Classification (Cyclone)
The `cyclone` process applies the identified markers to new, unlabeled data.
- **Input**: Unlabeled testing data and the marker pairs generated by `sandbag`.
- **Logic**: For every cell, it calculates the proportion of marker pairs that match the training pattern.
- **Output**: A score for each class (between 0 and 1) and a final assignment based on these scores.

## Best Practices and Expert Tips
- **AnnData Integration**: Always prefer passing `AnnData` objects. `pypairs` is designed to read directly from and write to `AnnData` slots, making it compatible with standard `Scanpy` workflows.
- **Handling Sparse Data**: Ensure you are using version 3.2.3 or later when working with large-scale experiments, as these versions include specific optimizations for sparse matrix handling to reduce memory consumption.
- **Performance Optimization**: `pypairs` utilizes `Numba` for JIT compilation and `guvectorize` for parallelization. If performance is slow, verify that `numba` is correctly configured to use all available CPU cores.
- **Marker Filtering**: If the classification step fails or returns "N/A" for classes, it often indicates that the `fraction` used in `sandbag` was too high, resulting in no valid marker pairs. Try lowering the `fraction` or verifying that the gene identifiers (e.g., Ensembl IDs vs. Gene Symbols) are consistent across both datasets.
- **Reproducibility**: Use the seeding feature (introduced in v3.2.2) to ensure that results remain deterministic across different environments.
