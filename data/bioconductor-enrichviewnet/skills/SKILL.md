# bioconductor-enrichviewnet

## Overview

The `enrichViewNet` package bridges the gap between tabular functional enrichment results and network-based visualization. It specifically supports results generated by the `gprofiler2` package. It offers two primary visualization strategies:
1.  **Gene-Term Networks**: Representing both terms (GO, Pathways) and genes as nodes, with edges connecting genes to their associated terms. These are designed for export to **Cytoscape**.
2.  **Enrichment Maps**: Representing terms as nodes where edges connect terms that share a significant number of genes (Jaccard similarity), helping identify functional modules.

## Core Workflow

### 1. Prepare Enrichment Data
The package requires a `gost` object from `gprofiler2`.

```r
library(gprofiler2)
library(enrichViewNet)

# Example: Run enrichment
gostres <- gost(query = c("TP53", "ATM", "ATR"), 
                organism = "hsapiens", 
                sources = c("REAC", "WP", "KEGG"),
                evcodes = TRUE) # evcodes=TRUE is often required for gene mapping
```

### 2. Create Gene-Term Networks (Cytoscape)
Use `createNetwork()` to generate a network. If Cytoscape is running locally with the CyREST API enabled, the network loads automatically. Otherwise, it saves a `.cx` file.

```r
# Create network for a specific source (e.g., Reactome)
createNetwork(gostObject = gostres, 
              source = "REAC", 
              removeRoot = TRUE, 
              title = "My_Reactome_Network")

# Create network for specific Term IDs
createNetwork(gostObject = gostres, 
              source = "TERM_ID", 
              termIDs = c("REAC:R-HSA-9614085", "REAC:R-HSA-9617828"))
```

### 3. Create Enrichment Maps (ggplot2)
Enrichment maps cluster similar terms. `createEnrichMap()` returns a `ggplot` object.

```r
# Basic ggplot enrichment map
p <- createEnrichMap(gostObject = gostres, 
                     query = "query_1", 
                     source = "KEGG")
print(p)

# Customizing colors (it's a standard ggplot)
library(ggplot2)
p + scale_color_continuous(low = "blue", high = "red")
```

### 4. Create Enrichment Maps (igraph)
For more programmatic control or complex layouts, use the `igraph` variants.

```r
library(igraph)
# Generate igraph object
ig <- createEnrichMapAsIgraph(gostObject = gostres, 
                              query = "query_1", 
                              source = "REAC", 
                              similarityCutOff = 0.3)

# Plot using igraph
plot(ig, layout = layout_with_fr, vertex.label.cex = 0.7)
```

### 5. Multi-Analysis Comparisons
To compare results from different experiments or different sources in one map:

*   **Basic Multi-Map**: `createEnrichMapMultiBasic()` (ggplot) or `createEnrichMapMultiBasicAsIgraph()`.
*   **Complex Multi-Map**: Use `createEnrichMapMultiComplex()` with a `queryInfo` data frame to define specific subsets and group names.

```r
# queryInfo structure
queryInfo <- data.frame(
  queryName = c("query_1", "query_1"),
  source = c("KEGG", "REAC"),
  removeRoot = c(TRUE, TRUE),
  termIDs = c("", ""),
  groupName = c("Kegg_Group", "Reactome_Group"),
  stringsAsFactors = FALSE
)

createEnrichMapMultiComplex(gostObjectList = list(gostres, gostres), 
                            queryInfo = queryInfo)
```

## Tips for Better Visualizations
*   **Seed Setting**: When using `igraph` or `ggtangle` layouts, use `set.seed()` to ensure the network layout is reproducible.
*   **Similarity Threshold**: Adjust `similarityCutOff` (default 0.2). Higher values result in sparser networks with only very closely related terms connected.
*   **Category Limit**: Use `showCategory` to limit the number of terms displayed (e.g., top 10) to avoid "hairball" networks.
*   **Cytoscape**: If the network is too dense in R, use `createNetwork()` and perform layout adjustments (like Force Directed) within Cytoscape.
