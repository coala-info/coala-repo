# scvelo

## Overview
scVelo is a scalable toolkit used to recover directed dynamic information from single-cell transcriptomics by leveraging splicing kinetics. Unlike traditional methods that assume a steady-state, scVelo utilizes a dynamical model to solve the full transcriptional dynamics for each gene. This allows for the characterization of transient cell states, the estimation of reaction rates (transcription, splicing, and degradation), and the inference of a "latent time" that represents the internal clock of a cell's differentiation process.

## Installation

```bash
# Using Conda

# Using Pip
pip install scvelo
```

## Core Workflow Patterns
The standard scVelo pipeline is typically executed within a Python environment. The following sequence represents the high-utility path for velocity estimation:

### 1. Data Preparation
Ensure your AnnData object (`adata`) contains `spliced` and `unspliced` layers. These are typically generated by tools like velocyto or kallisto/bustools.

```python
import scvelo as scv

# Load example data or your own .h5ad file
adata = scv.datasets.bonemarrow()

# Show spliced/unspliced proportions
scv.pl.proportions(adata)
```

### 2. Preprocessing
Filtering and normalization are essential to ensure counts are comparable across cells.

```python
# Filter genes and normalize
scv.pp.filter_and_normalize(adata, min_shared_counts=20, n_top_genes=2000)

# Compute moments for velocity estimation (required)
scv.pp.moments(adata, n_pcs=30, n_neighbors=30)
```

### 3. Velocity Estimation
Choose the model based on your biological system.

*   **Stochastic/Steady-State Model**: Faster, assumes a constant ratio of splicing/degradation.
*   **Dynamical Model**: More powerful, captures non-steady-state dynamics.

```python
# Run the dynamical model (recommended for complex lineages)
scv.tl.recover_dynamics(adata)
scv.tl.velocity(adata, mode='dynamical')

# Alternative: Stochastic model
# scv.tl.velocity(adata, mode='stochastic')
```

### 4. Velocity Graph and Visualization
Project the velocities onto a low-dimensional embedding (UMAP/TSNE).

```python
# Compute the velocity graph
scv.tl.velocity_graph(adata)

# Visualize as a stream plot on UMAP
scv.pl.velocity_embedding_stream(adata, basis='umap')
```

## Expert Tips and Best Practices
*   **Latent Time**: Use `scv.tl.latent_time(adata)` after running the dynamical model. It is often more accurate than standard pseudotime because it is based on the gene-specific splicing kinetics rather than global transcriptomic similarity.
*   **Gene-Specific Analysis**: Use `scv.pl.velocity(adata, var_names=['GeneA', 'GeneB'])` to inspect phase portraits. A "circle" or "almond" shape in the spliced-unspliced plot indicates well-captured dynamics.
*   **Kinetic Clusters**: Identify groups of genes that share similar kinetic regimes using `scv.tl.kinetic_clusters(adata)`.
*   **Data Cleaning**: If observation names are messy (e.g., from different loom file merges), use `scv.core.clean_obs_names(adata)` to standardize them.
*   **Memory Management**: For very large datasets, use `scv.pp.filter_and_normalize` with a smaller `n_top_genes` count to reduce the computational load of the `recover_dynamics` step.
