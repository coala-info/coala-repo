# Basic info

MDMcleaner can download and process its own database using the most recent datasets at GTDB, NCBI-RefSeq (eukaryote genomes) and Silva. However the process takes quite a long while (aproximately 18h for download and processing, but can be even longer depending on your download-speed). To make this a bit faster for you, we are planning to provide regularly updated versions of the MDMcleaner database on our institutes FTP-server sometime in the future, so keep looking out here for when that option becomes available.

For reproducability and testing purposes, the database version used during the work on our publication is also [available via zenodo](https://zenodo.org/record/5698995). The current version of the `mdmcleaner.py makedb` command even has an option to download this "publication reference dataset" directly: `--get_pub_data`. **However, this database is already outdated by now and you will definitively get better results using the most recent reference datasets!**

In order to download the most recent reference dataset you should use the `makedb` command of mdmcleaner.py. **Because the download and processing takes so long, it is highly recommended to run the makedb process with [nohup](https://en.wikipedia.org/wiki/Nohup) or [screen](https://en.wikipedia.org/wiki/GNU_Screen) and as a background job!** (see examples below).

The downloaded database will be located in the specified target folder, in the form of a "gtdb" subfolder (This may seem strange considering that the database combines datasets from Silva and Refseq as well, however it is based on the GTDB taxonomy, which is indicated by the subfolder name. We plan to optionally allow the use of the ncbi-taxonomy as well, which will require a seperate "ncbi" database subfolder at the same base-location. So we are sticking with this system for now)

# Usage

```
usage: mdmcleaner.py makedb [-h] [-o OUTDIR] [-c CONFIGFILE] [--verbose] [--quiet]

optional arguments:
  -h, --help            show this help message and exit
  -o OUTDIR, --outdir OUTDIR
                        target base directory for reference-data. may not be the current working directory. Needs >100GB space! Default = './db'
  -c CONFIGFILE, --config CONFIGFILE
                        provide a local config file with the target location to store database-files. default: looks for config files named 'mdmcleaner.config' in current working directory. settings in
                        the local config file will override settings in the global config file '/home/ww5070/temp_binsaga/sagabin_refiner/lib/mdmcleaner.config'
  --get_pub_data        get the ready made reference dataset used during the work on the mdmcleaner publication (WARNING: is outdated and only provided for reproducability and/or testing purposes)
  --verbose             verbose output (download progress etc)
  --quiet               quiet mode (suppress any status messages except Errors and Warnings)
```

# Examples

**In order to download the most recent database to a "mdmcleanerdb/" folder in your home directory you could use:**

```
nohup mdmcleaner.py makedb -o ~/mdmcleanerdb &> nohup_mdmcleaner_download.log&
```

(remember to specify this folder in a local config file, though)

**In order to download the database to a folder already specified in a config file (called "myconfig.cfg") you could use:**

```
nohup mdmcleaner.py makedb -c myconfig.cfg &> nohup_mdmcleaner_download.log&
```

**In order to download the publication reference dataset (which is already outdated) you could use:**

```
nohup mdmcleaner.py --get_pub_data -o ~/old_publication_database &> nohup_mdmcleaner_download.log &
```
