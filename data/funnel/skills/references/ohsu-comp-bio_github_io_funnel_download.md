Quick Start The following command will automatically download and verify the latest version of Funnel for your operating system. Funnel requires that Docker be installed in order to run commands within a sandboxed environment. /bin/bash -c &#34; $( curl -fsSL https://raw.githubusercontent.com/ohsu-comp-bio/funnel/refs/heads/develop/install.sh ) &#34; funnel server run 2. Containers The following commands show examples of running Funnel via Docker, Podman, and Singularity. Each command demonstrates how to mount an optional example config ( example.yaml ) for use by the Funnel Server. # example.yaml ➜ local Funnel Server config Database : boltdb Compute : local Logger : Level : debug Warning When given no config, Funnel will simply run in the default &ldquo;local&rdquo; mode. This can be helpful for testing and development, but production deployments are recommended to use the more robust database and compute backends available. Docker Tip Docker Image → quay.io/repository/ohsu-comp-bio/funnel:latest docker run -p 8000:8000 -v example.yaml:/example.yaml quay.io/ohsu-comp-bio/funnel:latest server run --config /example.yaml curl localhost:8000/service-info # { # &#34;description&#34;: &#34;Funnel is a toolkit for distributed task execution via a simple, standard API.&#34;, # &#34;documentationUrl&#34;: &#34;https://ohsu-comp-bio.github.io/funnel/&#34;, # ... # } Podman Tip Podman: Running a container podman machine init # Machine init complete podman machine start # Machine &#34;podman-machine-default&#34; started successfully podman run -p 8000:8000 -v ./example.yaml:/example.yaml quay.io/ohsu-comp-bio/funnel:latest server run --config /example.yaml # {&#34;httpPort&#34;: &#34;8000&#34;, &#34;msg&#34;: &#34;Server listening&#34;, &#34;rpcAddress&#34;: &#34;:9090&#34;} Singularity Tip Singularity and Docker singularity run --bind example.yaml:/example.yaml docker://quay.io/ohsu-comp-bio/funnel:latest server run --config /example.yaml # INFO: Converting OCI blobs to SIF format # INFO: Starting build... # INFO: Creating SIF file... # server Server listening # httpPort 8000 # rpcAddress :9090 3.
