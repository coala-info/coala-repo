# ntstat

## Overview
ntStat is a specialized toolkit for the statistical analysis of k-mer frequency and depth. It utilizes Bloom filters to efficiently track k-mer information for downstream applications. The tool is primarily used to model k-mer count histograms using mixture models, which allows for the de novo inference of genome size, heterozygosity, and the probability of a k-mer being erroneous. It is a critical tool for quality control and preliminary genome characterization before assembly or complex variant calling.

## Core Modules and CLI Patterns

### 1. K-mer Counting (`count`)
Calculates TF-IDF (Term Frequency-Inverse Document Frequency) information. It requires a k-mer spectrum file previously generated by `ntCard`.

*   **Basic Usage:**
    `ntstat count -k <kmer_size> -f <ntcard_hist> -o <output_prefix> <reads.fastq>`
*   **Optimization:** Use the `--long` flag when working with long-read data (PacBio/ONT) to adjust for different error profiles.
*   **Memory Management:** By default, ntStat targets a false positive rate of 0.0001 (`-e`). If memory is constrained, manually set the Bloom filter size in bytes using `-b`.

### 2. K-mer Filtering (`filter`)
Extracts k-mers within a specific frequency range. This is highly effective for removing sequencing errors (low frequency) or masking repeats (high frequency).

*   **Basic Usage:**
    `ntstat filter -k 31 -f <ntcard_hist> -cmin 2 -cmax 100 -o filtered.bf <reads.fastq>`
*   **Counting Bloom Filter:** Use `--counts` to save actual k-mer counts rather than just presence/absence. Note that this increases the output file size by approximately 8x.
*   **Automatic Thresholds:** It is best practice to run the `hist` module first to identify the "first minimum" in the histogram, then use that value for `-cmin`.

### 3. Histogram Analysis (`hist`)
Fits a mixture model to the k-mer spectrum to estimate biological and technical parameters.

*   **Basic Usage:**
    `ntstat hist <ntcard_hist>`
*   **Visualizing the Fit:** Generate a plot of the model fit using `-o plot.png`.
*   **Exporting Probabilities:** Use `--probs <file.tsv>` to output the probability of a k-mer being erroneous given its count, which is useful for custom filtering scripts.

### 4. Querying Results (`query`)
Converts the binary Bloom filter files into human-readable TSV format for inspection or custom analysis.

*   **Basic Usage:**
    `ntstat query -b <filter.bf> -o <output.tsv> <query_data.fasta>`

## Expert Tips and Best Practices

*   **ntCard Dependency:** ntStat is designed to work in tandem with `ntCard`. You must run `ntCard` first to generate the k-mer spectrum file (`-f`) required by the `count`, `filter`, and `hist` modules.
*   **Consistency is Key:** Ensure the k-mer length (`-k`) used in `ntStat` matches the k-mer length used in the preceding `ntCard` run.
*   **Spaced Seeds:** For increased sensitivity in certain comparative genomics tasks, use the `-s` flag to provide a file containing spaced seed patterns (1s for care, 0s for don't care) instead of standard k-mers.
*   **Thread Scaling:** Use the `-t` flag to specify the number of threads. K-mer counting is a parallelizable task; increasing threads significantly reduces wall-clock time on high-core systems.
*   **Handling Compressed Data:** The `query` module natively supports compressed FASTA/FASTQ/BAM files, eliminating the need for manual decompression.
