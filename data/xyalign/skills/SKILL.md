# xyalign

name: xyalign
description: |
  Command-line tool and Python library for inferring ploidy, correcting for sex chromosome complement, and working with Next-Generation Sequencing (NGS) data.
  Use when dealing with genomic data where sex chromosome complement (e.g., XY, XX) needs to be accounted for, or when analyzing data with potential biases related to sex chromosomes.
  This tool is particularly useful for identifying and correcting technical biases in sequencing data that arise from differences in coverage or mapping quality between sex chromosomes.
```
## Overview
XYalign is a bioinformatics tool designed to address challenges in analyzing sequencing data related to sex chromosomes. It helps in determining the ploidy of sex chromosomes, correcting for variations in sequencing data that are specific to the sex chromosome complement (like XY or XX), and generally working with Next-Generation Sequencing (NGS) data where sex chromosome biases might be present. This is crucial for accurate genomic analysis, especially in species with complex sex determination systems or when studying sex-linked traits.

## Usage

XYalign can be used as a command-line tool or integrated as a Python library. The primary function involves processing sequencing data (e.g., BAM files) to infer ploidy and correct for sex chromosome biases.

### Command-Line Interface (CLI)

The core functionality of XYalign is accessed through its command-line interface. The general structure involves specifying input files and desired operations.

**Basic Usage:**

The most common use case involves providing an input BAM file and specifying the output prefix.

```bash
xyalign <input.bam> --output <output_prefix>
```

**Key Options:**

*   `--output <output_prefix>`: Specifies the prefix for all output files generated by XYalign. This is a mandatory argument.
*   `--reference_mask <reference_mask.bed>`: (Optional) A BED file defining regions to mask. This can be useful for excluding specific genomic regions from analysis.
*   `--ploidy <ploidy_file.tsv>`: (Optional) A tab-separated file specifying ploidy information for specific samples or regions.
*   `--threads <num_threads>`: (Optional) Number of threads to use for parallel processing.

**Example Workflow:**

To infer ploidy and correct for sex chromosome biases in a BAM file named `sample1.bam`, saving the output with the prefix `sample1_xyalign`:

```bash
xyalign sample1.bam --output sample1_xyalign
```

This command will generate several output files, including those related to inferred ploidy and corrected read counts or balances.

**Advanced Usage and Considerations:**

*   **Input File Formats**: While BAM is the primary input, the tool may support other formats like CRAM or SAM (refer to specific version documentation if needed).
*   **Sex Chromosome Specificity**: The tool is designed to handle both XY and XX individuals, inferring and correcting based on the detected sex chromosome complement.
*   **Output Interpretation**: The output files will typically contain metrics for read balance, ploidy inference, and potentially corrected coverage values. Understanding these outputs is key to interpreting the results.

## Expert Tips

*   **Parallel Processing**: For large datasets, utilize the `--threads` option to significantly speed up processing.
*   **Reference Masking**: If you have known problematic regions or specific areas you wish to exclude from the analysis (e.g., due to repetitive sequences or known assembly issues), use the `--reference_mask` option with a properly formatted BED file.
*   **Ploidy Specification**: For known ploidy states, providing a `--ploidy` file can guide the analysis and potentially improve accuracy.
*   **Version Management**: Ensure you are using a recent version of XYalign, as improvements and bug fixes are regularly released. Check the bioconda channel for the latest stable versions.
