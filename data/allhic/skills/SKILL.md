# allhic

## Overview
ALLHIC is a specialized genome scaffolding tool designed to handle the complexities of polyploid and highly heterozygous diploid genomes. Unlike standard scaffolders that may struggle with homologous chromosomes, ALLHIC utilizes Hi-C contact data to partition contigs into linkage groups and then employs a Genetic Algorithm to determine the most likely linear order and orientation of those contigs. It is best used when you have a draft assembly (contigs) and high-quality Hi-C sequencing data.

## Core Workflow and CLI Patterns

### 1. Link Extraction
The first step processes the raw alignments to identify Hi-C contacts.
```bash
allhic extract <sample.bam> <ref.fasta>
```
*   **Input**: A BAM file of Hi-C reads mapped to your contigs and the corresponding FASTA file.
*   **Output**: Generates a `.clm` file (compact link matrix) and a distribution of intra-contig links.

### 2. Allelic Pruning (Polyploid Specific)
For polyploid genomes, use `prune` to remove links between allelic contigs that might cause mis-partitioning.
```bash
allhic prune <Allele.ctg.table> <sample.pairs.txt>
```
*   **Tip**: This step is optional for simple inbred diploids but critical for polyploids to ensure contigs are assigned to the correct homologous group.

### 3. Partitioning
Divide contigs into $k$ groups (typically the number of haploid chromosomes).
```bash
# Standard usage
allhic partition <counts_GATC.txt> <sample.pairs.txt>

# If pruning was performed
allhic partition <counts_GATC.txt> <sample.pairs.prune.txt>
```
*   **Optimization**: This uses hierarchical clustering based on average links to separate contigs into chromosomal clusters.

### 4. Optimization
Reconstruct the order and orientation of contigs within each partition using a Genetic Algorithm.
```bash
allhic optimize <partition.counts_GATC.txt> <sample.clm>
```
*   **Note**: You must run this for each partition generated in the previous step. It produces a `.tour` file containing the optimized path.

### 5. Final Assembly Building
Convert the optimized tours into standard genomic formats.
```bash
allhic build <tour_files> <ref.fasta> <output.fasta>
```
*   **Output**: Generates both `.agp` and `.fasta` files representing the final scaffolded assembly.

### 6. Visualization
Generate heatmaps to assess the quality of the scaffolding.
```bash
allhic plot <sample.bam> <sample.tour>
```

## Best Practices
*   **Scripted Pipelines**: For large-scale assemblies, it is highly recommended to wrap these commands in a shell script to iterate through all partitions during the `optimize` phase.
*   **Restriction Sites**: Ensure the counts file (e.g., `counts_GATC.txt`) correctly reflects the restriction enzyme used in your Hi-C library preparation.
*   **Genetic Algorithm**: The `optimize` step is stochastic. If results are inconsistent, consider running multiple iterations or checking the `.clm` file for link density.



## Subcommands

| Command | Description |
|---------|-------------|
| alleles | Identify and classify allelic contigs from a PAF file to be used for 'allhic prune'. Generates 'alleles.table'. |
| assess | Assess Hi-C data using BAM and BED files |
| build | Build a chromosome-level assembly from tour files and contigs |
| extract | Extract information from BAM and FASTA files for ALLHIC |
| optimize | Given a set of Hi-C contacts between contigs, as specified in the clmfile, reconstruct the highest scoring ordering and orientations for these contigs. Optimize run on a specific partition in "clusters.txt" as generated by "partition" sub-command, with the group_number matching the order appearing in "clusters.txt". |
| partition | Separate all the contigs into separate clusters using a hierarchical clustering algorithm based on average links. Requires counts_RE.txt and pairs.txt generated by the extract sub-command. |
| pipeline | A pipeline for Hi-C scaffolding using the ALLHIC algorithm. |
| plot | Plot BAM and tour files for ALLHIC |
| prune | Prune inter-allelic links from contig pairing data to enable allele-separated assemblies. It removes Hi-C links between contigs identified as allelic and filters links to keep only the single-best edge when multiple contigs link to the same allelic group. |
