# pirate

## Overview

PIRATE (Pangenome Iterative Refinement and Threshold Evaluation) is a comprehensive toolbox designed to map the genetic landscape of bacterial populations. While traditional pangenome tools often rely on a single sequence identity cutoff, PIRATE employs an iterative clustering approach across a wide range of thresholds. This allows for a more granular classification of orthologous groups and provides specific insights into evolutionary events like gene duplications or fissions. It produces a pangenome graph and provides measures of allelic variance, making it a robust choice for researchers studying genomic adaptation and diversity.

## Usage Guidelines

### Input Requirements
* **Format**: PIRATE requires GFF3 files.
* **Best Practice**: Use GFF3 files generated by **Prokka**. Files from RAST or NCBI may lack the required matching nucleotide sequences at the end of the file or consistent `locus_tag` fields.
* **Validation**: PIRATE automatically verifies GFF3 formatting and will discard non-compliant files.

### Core Command Patterns

**Basic Pangenome Construction**
Run PIRATE with default settings (identity thresholds: 50, 60, 70, 80, 90, 95, 98%):
```bash
PIRATE -i /path/to/gff_directory/ -o output_results/
```

**Customizing Identity Thresholds**
To capture more or less divergent gene families, specify custom steps:
```bash
PIRATE -i input_dir/ -s 60,70,80,90,95,98,99
```

**Analyzing Specific Features**
By default, PIRATE looks for `CDS`. You can specify other features (e.g., tRNA, rRNA) separated by commas:
```bash
PIRATE -i input_dir/ -f CDS,rRNA,tRNA
```

**Nucleotide-only Analysis**
If you do not want CDS translated to amino acid sequences for clustering:
```bash
PIRATE -i input_dir/ -n
```

### Post-Processing and Utilities

**Restoring Original Nomenclature**
PIRATE standardizes locus tags during processing. To map the results back to your original naming scheme:
```bash
subsample_outputs.pl -i PIRATE.gene_families.ordered.tsv -o renamed_output.tsv --field "prev_locus"
```

**Visualizing Results**
If R and the required dependencies (ggplot2, ggtree, dplyr) are installed, PIRATE can generate pangenome plots and trees. Ensure these are installed via Conda if visualization is required.

## Expert Tips
* **Paralog Classification**: If you are only interested in the pangenome content and not the evolutionary classification of paralogs, use `--para-off` to reduce computation time.
* **Iterative Runs**: If you have already run the pangenome construction and only need to re-run downstream analysis or classification, use `--pan-off` to skip the initial clustering step.
* **Standardization**: Always ensure your input GFF3 files have the `.gff` extension, as PIRATE filters by file extension before execution.
