# polap

## Overview

Polap is a specialized bioinformatics pipeline for the de novo assembly of plant mitochondrial genomes. Unlike general-purpose assemblers, Polap uses organellar gene annotation to filter and prioritize contigs generated by the Flye long-read assembler. This approach helps resolve the complexity of plant mtDNA, which often contains repetitive sequences and shares genomic content with the plastome or nucleus. Use this skill to guide users through the multi-step process of whole-genome assembly, organelle contig identification, and final genome polishing.

## Installation and Environment Setup

Polap requires a Linux environment and is best managed via Conda to handle its extensive dependency list (Flye, Bandage, FMLRC, etc.).

- **Core Installation**: Use the specific version recommended for stability.
  ```bash
  conda create -y --name polap polap=0.3.7.3
  conda activate polap
  ```
- **FMLRC Setup**: Polishing with short reads requires a secondary environment.
  ```bash
  # After activating polap, locate the yaml to create the fmlrc environment
  base_dir=$(dirname "$(command -v polap)")
  conda env create -f "$base_dir/polap-conda-environment-fmlrc.yaml"
  ```

## Core Workflow Patterns

### 1. Initial Whole-Genome Assembly
The first step involves running a general assembly to generate the pool of contigs from which mitochondrial sequences will be extracted.
- Use `polap` with the appropriate read type flag (e.g., `--pacbio-hifi` or `--nano-raw`).
- Specify the estimated genome size if known to improve Flye's performance.

### 2. Annotation-Guided Selection
Polap identifies mitochondrial contigs by searching for conserved plant organellar genes.
- Ensure the input includes a reference set of plant mitochondrial genes.
- Review the generated `.bed` and `genes.txt` files to verify the presence of core mitochondrial markers (e.g., *atp*, *cob*, *cox* genes).

### 3. Contig Extraction and Visualization
After identifying potential MT contigs, use Bandage to visualize the assembly graph (`.gfa`).
- Look for high-coverage paths that form circular or complex branched structures characteristic of plant mtDNA.
- Use the "MT Contig Name File" to provide Polap with the specific headers of contigs to be included in the final organelle-specific assembly.

### 4. Polishing the Assembly
To achieve high consensus accuracy, especially with Nanopore data, perform polishing.
- **Long-read polishing**: Use the original long reads to fix remaining indels.
- **Short-read polishing**: If Illumina data is available, use the `polap-fmlrc` environment to correct small-scale errors.

## Expert Tips and Best Practices

- **Version Consistency**: Always verify the version using `polap --version`. The directory structure for library files changes between v0.3.x and v0.4.x.
- **Memory Management**: Plant WGS datasets can be massive. When running the initial Flye assembly, ensure the `--threads` parameter matches your hardware to prevent OOM (Out of Memory) errors during the graph construction phase.
- **Handling Plastid Contamination**: Plant cells contain both chloroplasts and mitochondria. Polap's annotation step is critical here; use the gene density and identity scores to distinguish between mtDNA and cpDNA (chloroplast) contigs.
- **Graph Inspection**: Never rely solely on the automated output. Always open the `assembly_graph.gfa` in Bandage to check for "master circle" configurations or multi-partite structures.
