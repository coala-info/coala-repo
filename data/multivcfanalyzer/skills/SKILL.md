# multivcfanalyzer

## Overview
MultiVCFAnalyzer is a Java-based tool designed to aggregate and filter SNP calls from multiple VCF files. It is optimized for ancient DNA (aDNA) workflows where low coverage and damage patterns require stringent filtering. The tool produces SNP alignments for phylogenetic reconstruction and detailed tables for SNP effect and population genetic analyses. It is particularly effective for haploid microbial contexts but supports diploid calls through IUPAC uncertainty codes.

## Command Line Usage

### Basic Syntax
The tool requires a specific, strict order of parameters. Use `NA` for any optional files that are not being provided.

```bash
java -Xmx16G -jar MultiVCFAnalyzer.jar \
  <SnpEff_file|NA> \
  <Reference_fasta> \
  <Reference_gff|NA> \
  <Output_directory> \
  <Write_allele_freqs_T/F> \
  <Min_genotype_quality> \
  <Min_coverage> \
  <Min_homozygous_freq> \
  <Min_heterozygous_freq> \
  <Exclude_positions_gff|NA> \
  <VCF_file_1> <VCF_file_2> ... <VCF_file_N>
```

### Parameter Definitions
1. **SnpEff result**: `.txt` output from SnpEff for effect analysis.
2. **Reference genome**: FASTA file used during the original VCF construction.
3. **Reference annotation**: GFF file for gene-based filtering/analysis.
4. **Output directory**: Path where results will be written.
5. **Write allele frequencies**: 'T' or 'F'. If 'T', includes percentages in the SNP table (e.g., `A (70%)`).
6. **Minimal genotyping quality**: GATK quality score threshold (e.g., 30).
7. **Minimal coverage**: Minimum read depth required to make a call (e.g., 5).
8. **Minimal allele frequency (Homozygous)**: Fraction of reads required to call a base homozygous (e.g., 0.9).
9. **Minimal allele frequency (Heterozygous)**: Fraction threshold for calling a heterozygous position (e.g., 0.1).
10. **Exclude positions**: GFF file listing specific coordinates to ignore.
11. **Input VCFs**: One or more VCF files generated by GATK UnifiedGenotyper.

## Expert Tips and Best Practices

### Sample Naming Convention
MultiVCFAnalyzer derives sample names from the **directory name** containing the VCF file, not the filename itself. To ensure unique sample names in your output tables and alignments, place each VCF in a uniquely named folder:
*   `Sample_A/calls.vcf.gz` -> Sample name: `Sample_A`
*   `Sample_B/calls.vcf.gz` -> Sample name: `Sample_B`

### Memory Management
For large datasets or high-resolution reference genomes, increase the Java heap size using the `-Xmx` flag (e.g., `-Xmx16G` or `-Xmx32G`).

### Homozygous-Only Filtering
To restrict results to strictly homozygous SNPs (common in haploid microbial studies), set both the homozygous and heterozygous frequency thresholds to the same value (e.g., `0.9` and `0.9`).

### Input Requirements
*   **GATK Version**: Input VCFs must be generated by GATK UnifiedGenotyper (version 3.5 or earlier).
*   **Ploidy**: VCFs must be generated with `ploidy` set to 2 to ensure allele frequency values are present in the file, even for haploid organisms.
*   **Reference Consistency**: The reference FASTA must be identical to the one used for mapping and variant calling.

### Key Output Files
*   `snpTable.tsv`: The primary comparative table of all filtered SNP positions.
*   `snpAlignment.fasta`: A FASTA file containing only the SNP positions for all samples, ideal for RAxML or IQ-TREE.
*   `snpStatistics.tsv`: Summary metrics including coverage (fold and percent) and the number of discarded/filtered calls per sample.
