# fec

## Overview
`fec` (Fast Error Correction) is a tool designed to improve the quality of long-read sequencing data through a two-round overlapping and caching strategy. The first round quickly identifies overlaps to correct a subset of reads immediately, while the second round uses finely tuned parameters to find additional overlaps for the remaining reads. It is particularly effective for noisy datasets and supports both PacBio and Nanopore data types.

## Command Line Usage

The standard workflow for `fec` requires a pre-computed overlap file in PAF format, typically generated by `minimap2`.

### Basic Syntax
```bash
Fec [options] <overlaps.paf> <input_reads.fq> <output_corrected.fasta>
```

### Common Options
- `-t <INT>`: Number of threads (default: 1).
- `-r <FLOAT>`: Minimum accuracy of overlaps (default: 0.6).
- `-a <INT>`: Minimum overlap length.
- `-c <INT>`: Minimum coverage for correction.
- `-l <INT>`: Minimum read length to be corrected.
- `-x <INT>`: Mode selection (use `1` for Nanopore data).
- `-m <FLOAT>`: Filter out top fraction of repetitive minimizers (for large datasets).
- `-f <FLOAT>`: Filter out short overlaps in the second round.

## Recommended Workflows

### 1. Small PacBio Datasets
For smaller datasets, use standard `ava-pb` settings and filter for overlaps that cover at least 50% of the read length.
```bash
# Generate and filter overlaps
minimap2 -x ava-pb -w 20 -K 2g -t $THREADS reads.fq reads.fq | \
awk '{ if($4 - $3 >= 0.5 * $2 || $9 - $8 >= 0.5 * $7) print $0}' > ovlp.paf

# Run Fec
Fec -t $THREADS -r 0.6 -a 1000 -c 4 -l 2000 ovlp.paf reads.fq corrected.fasta
```

### 2. Large PacBio Datasets (e.g., Human)
For large genomes, use more aggressive minimizer filtering to improve speed and handle repetitive regions.
```bash
# Generate and filter overlaps with repetitive minimizer filtering (-f 0.005)
minimap2 -x ava-pb -w 20 -K 2g -f 0.005 -t $THREADS reads.fq reads.fq | \
awk '{ if($4 - $3 >= 0.2 * $2 || $9 - $8 >= 0.2 * $7) print $0}' > ovlp.paf

# Run Fec with matching repetitive filtering (-m 0.005)
Fec -t $THREADS -r 0.6 -a 1000 -c 4 -l 2000 -m 0.005 -f 0.2 ovlp.paf reads.fq corrected.fasta
```

### 3. Nanopore Datasets (Two-Round Correction)
Nanopore data requires a two-pass approach for optimal accuracy. Note the use of `-x 1` for Nanopore mode and `-R` for the second round.

**Round 1:**
```bash
minimap2 -x ava-ont -w 20 -K 2g -f 0.005 -t $THREADS reads.fq reads.fq | \
awk '{ if($4 - $3 >= 0.2 * $2 || $9 - $8 >= 0.2 * $7) print $0}' > ovlp.paf

Fec -x 1 -t $THREADS -r 0.6 -a 400 -c 0 -l 1000 -m 0.005 -f 0.2 ovlp.paf reads.fq corrected1.fasta
```

**Round 2:**
```bash
minimap2 -x ava-ont -w 20 -K 2g -f 0.005 -t $THREADS corrected1.fasta corrected1.fasta | \
awk '{ if($4 - $3 >= 0.2 * $2 || $9 - $8 >= 0.2 * $7) print $0}' > ovlp.paf

Fec -x 1 -R -t $THREADS -r 0.6 -a 1000 -c 4 -l 2000 -m 0.005 -f 0.2 ovlp.paf corrected1.fasta corrected2.fasta
```

## Expert Tips
- **Input Formats**: `fec` currently only supports uncompressed FASTA or FASTQ formats. If your data is gzipped, decompress it before processing.
- **Overlap Filtering**: The `awk` filtering step is critical. `minimap2` often produces many short, low-quality overlaps that can degrade the performance and accuracy of `fec`.
- **Memory Management**: When working with large datasets, the `-K` parameter in `minimap2` controls the batch size. Adjust this based on available RAM.
