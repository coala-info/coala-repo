# primalbedtools

## Overview
primalbedtools is a specialized suite designed for bioinformaticians working with primer schemes, such as those generated by PrimalScheme. It simplifies the management of primer coordinates when moving between different reference genomes or viral variants. Use this skill to automate the validation of primer pairings, sort BED files for downstream analysis, and generate trimmed amplicon sequences required for variant calling and genomic surveillance.

## Core CLI Operations

### Coordinate Remapping
To transfer primer coordinates from one reference to another (e.g., remapping SARS-CoV-2 primers from the ancestral strain to a new variant), use the `remap` command with a Multiple Sequence Alignment (MSA).
```bash
primalbedtools remap --bed primers.bed --msa alignment.fasta --from_id REF_ID --to_id VAR_ID
```
*   **Tip**: Ensure the IDs provided to `--from_id` and `--to_id` match the headers in your FASTA alignment exactly.

### Validation and Consistency
Before using a primer scheme in a pipeline, validate its internal structure and its fit against a reference genome.
*   **Internal Consistency**: Check for correct primer pairings and naming logic.
    ```bash
    primalbedtools validate_bedfile primers.bed
    ```
*   **Reference Validation**: Verify that primer sequences in the BED file actually exist within the target genome.
    ```bash
    primalbedtools validate primers.bed reference.fasta
    ```

### Amplicon Generation
Generate amplicon coordinate files from primer pairs. This is essential for tools that require the full insert or trimmed insert coordinates.
```bash
# Standard amplicon generation
primalbedtools amplicon primers.bed > amplicons.txt

# Generate primer-trimmed amplicon coordinates (useful for variant calling)
primalbedtools amplicon primers.bed --primertrim > trimmed_amplicons.txt
```

### Format Standardization
Standardize primer names to the v2 format (`prefix_number_DIRECTION_index`) or downgrade them for legacy tool compatibility.
*   **Upgrade to v2**: `primalbedtools update primers.v1.bed > primers.v2.bed`
*   **Downgrade to v1**: `primalbedtools downgrade primers.v2.bed > primers.v1.bed`
*   **Merge Alternatives**: When downgrading, use `--merge-alts` to remove `_alt` suffixes and consolidate primer entries.

## Expert Tips and Best Practices
*   **Sorting**: Always run `primalbedtools sort <bed_file>` before generating amplicons or merging primers. The tool sorts by chromosome, then amplicon number, then direction, ensuring deterministic outputs.
*   **Merging**: If your scheme contains redundant primer entries with identical properties (chromosome, amplicon, direction), use `primalbedtools merge` to clean the file.
*   **FASTA Extraction**: You can quickly generate a FASTA file of all primer sequences from a BED file using `primalbedtools fasta primers.bed`. This is useful for secondary structure analysis or off-target checking.
*   **Pipe Support**: Most commands output to stdout, allowing you to chain operations:
    `primalbedtools sort primers.bed | primalbedtools update - > primers.final.bed`
