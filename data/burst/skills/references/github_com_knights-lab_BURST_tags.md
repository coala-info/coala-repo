Feb 11, 2017 34ddb10 zip tar.gz Notes Downloads v0.99.1b Toggle v0.99.1b&#39;s commit message Added initial release (0.99.1b) Debuting "embalmer" - an optimal short-read DNA aligner espousing (by default) a BLAST-like ID scoring system within least-cost paths discovered by exhaustive dynamic-programming. It has perfect sensitivity and specificity, as it makes no heuristic trade-offs or approximations. This means if a qualifying alignment exists, it will be found, and if multiple exist, the best will always be selected. Alternatively, all ties can be reported, or all qualifying alignments period. It also can accept a taxonomy file and interpolate (intelligently assign) taxonomy (or other heirarchical labels) for each read using a least common ancestor approach. Alternatively, downstream ML approaches can tally all ties at each level and interpolate post-hoc ("speculative", without LCA). It is suitable for amplicon and shotgun data, but slower with larger databases. 500,000 126-bp reads align to the "IMG" microbial database (~20GB) in under 45 minutes on a 48-core Xeon E7-4850.
