# cageseq

## Overview
nf-core/cageseq is a bioinformatics analysis pipeline designed for CAGE-seq (Cap Analysis Gene Expression) sequencing data. It solves the problem of identifying and quantifying transcription start sites by processing raw reads, removing technical artifacts like linkers and ribosomal RNA, and clustering the resulting CAGE tags.

The pipeline accepts raw FASTQ files and a reference genome as primary inputs. It produces clustered tag counts and various quality control metrics, allowing researchers to verify the results through a centralized MultiQC report that aggregates data from FastQC, RSeQC, and alignment statistics.

## Data preparation
The pipeline requires raw demultiplexed FASTQ files and reference genome information. Input paths must be enclosed in quotes and include at least one wildcard character.

*   **FASTQ Files**: Specified via the `--input` parameter. For paired-end data, use the `{1,2}` notation.
*   **Reference Genome**: Use `--genome` to specify an iGenomes ID (e.g., `GRCh38`). Alternatively, provide paths to a FASTA file (`--fasta`) and a GTF file (`--gtf`).
*   **Trimming Sequences**: Default sequences for EcoP15 sites (`CAGCAG`) and linkers (`TCGTATGCCGTCTTC`) are used unless overridden by `--eco_site` or `--linker_seq`.
*   **rRNA Removal**: If using `--remove_ribo_rna`, the pipeline uses a default database manifest located in the assets folder unless a custom path is provided via `--ribo_database_manifest`.

Example input structure:
```bash
--input 'path/to/data/sample_*_{1,2}.fastq.gz'
```

## How to run
Run the pipeline using the `nextflow run` command. It is recommended to use a specific pipeline version with `-r` and a container profile like `docker` or `singularity`.

```bash
nextflow run nf-core/cageseq \
    -profile docker \
    --input '*_R1.fastq.gz' \
    --genome GRCh38 \
    --aligner star \
    --outdir ./results
```

Key parameters include:
*   `-profile`: Choose a configuration profile (e.g., `docker`, `singularity`, `conda`, or `test`).
*   `--aligner`: Specify the alignment tool, either `star` (default) or `bowtie1`.
*   `--bigwig`: Set to `true` to generate TSS-bigwig files in addition to BED files.
*   `--remove_ribo_rna`: Enable ribosomal RNA filtering using SortMeRNA.
*   `-resume`: Use this flag to restart a pipeline from the last successful step if it was interrupted.

## Outputs
Results are saved in the directory specified by `--outdir` (default is `./results`).

*   **MultiQC Report**: Located in the base of the output directory; this is the primary file to inspect for a summary of QC metrics.
*   **CTSS Files**: CAGE Transcription Start Site files in BED format, and optionally BigWig format if `--bigwig` was specified.
*   **Clustering Results**: Tag clusters generated by paraclu.
*   **QC Metrics**: Individual reports from FastQC (raw and trimmed reads), RSeQC (read distribution), and Samtools stats.
*   **Pipeline Info**: Nextflow logs and execution reports are stored in the `pipeline_info` subfolder.
