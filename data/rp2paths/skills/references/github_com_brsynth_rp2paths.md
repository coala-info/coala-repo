RetroPath2.0 is freely accessible on myExperiment.org at: https://www.myexperiment.org/workflows/4987.html . Installation From conda # installation in an already existing &lt;myenv&gt; environment (see prerequisites) conda activate &lt; myenv &gt; conda install -c brsynth -c conda-forge rp2paths Usage From CLI Once a scope has been produced by RetroPath2.0, a typical command line for extracting the pathways from the results is python -m rp2paths all &lt; retropath2_scope &gt; [--outdir &lt; outdir &gt; ] where: all specify that all the tasks needed for retrieving pathways will be executed at once. &lt;retropath2_scope&gt; is the metabolic space outputted by the RetroPath2.0 workflow. --outdir &lt;outdir&gt; specify the directory in which all files will be outputted. In the &lt;outdir&gt; folder, the complete set of pathways enumerated will be written in the out_paths.csv file. In addition, for each pathway there will be a .dot file (.dot representation of the graph) and a .svg file (.svg depiction of the pathway). Use cases python -m rp2paths all tests/data/input/rp2_pathways.csv --outdir output Available options Additional options are described in the embedded help: # List of possible modes python -m rp2paths -h # List of options for the all-in-one mode python -m rp2paths all -h Examples Precomputed results (outputted by RetroPath2.0) are provided in the examples folder for few compounds (carotene, naringenin, pinocembrin, violacein). Below are the command lines for generating pathways that lie in naringenin result file: source activate &lt; myenv &gt; python -m rp2paths all examples/naringenin/rp2-results.csv --outdir examples/naringenin/outdir No scope warnings A warning message WARNING: *** Scope Task: no scope matrix was produced, exit indicates that no reaction have been indified as being part of a pathway. This can happen when the retrosynthesis exploration performed by RetroPath2.0 does not reach a minimal set of sink compounds. For developers Development installation After a git clone: cd &lt; repository &gt; conda env create -f environment.yaml -n &lt; dev_env &gt; conda develop -n &lt; dev_env &gt; . conda activate &lt; dev_env &gt; Tests Test can be run with the following commands: Natively cd tests pytest -v CI/CD For further tests and development tools, a CI toolkit is provided in ci folder (see ci/README.md ).
