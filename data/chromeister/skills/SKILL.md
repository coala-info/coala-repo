# chromeister

## Overview
Chromeister is a high-performance tool designed for coarse-grained genomic analysis. It utilizes a heuristic approach to identify conserved signals between extremely large sequences, making it particularly effective for comparing full genomes or large chromosomes. Instead of producing fine-grained alignments, it generates visualization matrices (dotplots) and identifies structural events such as synteny blocks, transpositions, and inversions. Use this skill to quickly assess genomic similarity and structural variations across species or assemblies.

## Core Command Line Usage

### Simple 1-vs-1 Comparison
The standard workflow involves generating a comparison matrix and then using an R script to calculate the similarity score and generate the dotplot image.

```bash
# 1. Generate the comparison matrix
CHROMEISTER -query sequence_A.fasta -db sequence_B.fasta -out output.mat

# 2. Compute score and generate plot (default resolution 1000)
Rscript compute_score.R output.mat 1000
```

### Adjusting Resolution
For larger genomes or more detailed images, increase the dimension. Ensure the dimension value passed to the R script matches the `-dimension` parameter in the CHROMEISTER command.

```bash
CHROMEISTER -query seqA.fasta -db seqB.fasta -out output.mat -dimension 2000
Rscript compute_score.R output.mat 2000
```

### Event Detection
To classify Large-Scale Genome Rearrangements (LSGRs) such as inversions or transpositions:

```bash
# Requires the python environment to be active
python3 bin/detect_events.py output.mat.raw.txt
```
*Note: Adding `png` to the end of the command will generate a plot with overlapped detected events.*

## Batch and All-vs-All Comparisons

### Comparing a Single Folder (1/2 * n * (n+1))
Use this to compare every sequence in a directory against every other sequence without redundant A-vs-B and B-vs-A comparisons.

```bash
allVsAll.sh <folder> <extension> <matrix_size> <kmer_size> <inexactitude>
# Example:
allVsAll.sh ./genomes fasta 2000 32 4
```

### Comparing Two Folders (n * m)
Use this to compare all sequences in Folder 1 against all sequences in Folder 2 (e.g., comparing all chromosomes of Genome A to all chromosomes of Genome B).

```bash
allVsAll_incremental.sh <folder1> <folder2> <extension> <matrix_size> <kmer_size> <inexactitude>
```

## Expert Tips and Best Practices

- **Kmer Size**: A kmer size of **32** is generally considered optimal for most genomic comparisons.
- **Inexactitude Level**: A level of **4** is the recommended starting point for heuristic sensitivity.
- **Grid Visualization**: If comparing genomes with a high number of scaffolds, use `compute_score-nogrid.R` instead of the standard script to prevent the output image from being obscured by grid lines.
- **Parallelization**: Chromeister supports parallel execution by simply running multiple instances of the command; each instance will utilize a different core.
- **Fine-Grained Results**: Chromeister is a "coarse-grained" tool. If you require actual sequence alignments, use the signals generated by Chromeister to guide more intensive tools like GECKO.
- **Output Files**:
    - `.mat.filt.png`: The visual dotplot.
    - `.events.txt`: Text file containing coordinates and classifications of synteny blocks.
    - `.csv`: Contains coordinates for each sequence/chromosome within the query and reference.
