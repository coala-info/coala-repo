Computes base and base pair match probabilities from alignment partitition functions. USAGE: locarna_p [options] &lt;Input 1&gt; &lt;Input 2&gt; locarna_p computes sequence and structure match probabilities from partition functions of locarna pairwise alignments, i.e. from fast simultaneous folding and alignment based on two RNA sequences (or alignments). Input and Constraints Please see the documentation of locarna for inputs and the possible constraints specifications. Output Probabilities of base and base pair matches can be written to files. (The total partition function is reported to standard out.) OPTIONS -h , &ndash;help : Print this help. -V , &ndash;version : Print only version string. -v , &ndash;verbose : Be verbose. Prints input parameters, sequences and size information. -q , &ndash;quiet : Be quiet. Scoring parameters: -i , &ndash;indel= &lt;score&gt;( -150 ) : Indel score. Score contribution of each single base insertion or deletion. Indel opening score and indel score define the affine scoring of gaps. &ndash;indel-opening= &lt;score&gt;( -750 ) : Indel opening score. Score contribution of opening an insertion or deletion, i.e. score for a consecutive run of deletions or insertions. Indel opening score and indel score define the affine scoring of gaps. &ndash;ribosum-file= &lt;f&gt;(RIBOSUM85_60) : File specifying the Ribosum base and base-pair similarities. [default: use RIBOSUM85_60 without requiring a Ribosum file.] &ndash;use-ribosum= &lt;bool&gt;(true) : Use ribosum scores for scoring base matches and base pair matches; note that tau=0 suppresses any effect on the latter. -m , &ndash;match= &lt;score&gt;(50) : Set score contribution of a base match (unless ribosum scoring). -M , &ndash;mismatch= &lt;score&gt;(0) : Set score contribution of a base mismatch (unless ribosum scoring). -s , &ndash;struct-weight= &lt;score&gt;(200) : Maximal weight of 1/2 arc match. Balances structure vs. sequence score contributions. -e , &ndash;exp-prob= &lt;prob&gt; : Expected base pair probability. Used as background probability for base pair scoring [default: calculated from sequence length]. -t , &ndash;tau= &lt;factor&gt;(50) : Tau factor. Factor for contribution of sequence similarity in an arc match (in percent). tau=0 does not penalize any sequence information including compensatory mutations at arc matches, while tau=100 scores sequence similarity at ends of base matches (if a scoring matrix like ribosum is used, this adds the contributions for base pair match from the matrix). [default tau=0!] &ndash;temperature-alipf= &lt;int&gt;(300) : Temperature for the */sequence* alignment/ partition functions used by the probcons-like sequence-based match/trace probability computation (this temperature is different from the \'physical\' temperature of RNA folding!). Partition function representation: &ndash;pf-scale= &lt;scale&gt;(1.0) : Factor for scaling the partition functions. Use in order to avoid overflow. &ndash;extended-pf : Use extended precision for partition function values. This increases run-time and space (less than 2x), however enables handling significantly larger instances. &ndash;quad-pf : Use quad precision for partition function values. Even more precision than extended pf, but usually much slower (overrides extended-pf). Output: &ndash;write-arcmatch-probs= &lt;file&gt; : Write arcmatch probabilities &ndash;write-basematch-probs= &lt;file&gt; : Write basematch probabilities -a , &ndash;min-am-prob= &lt;amprob&gt;(0.001) : Minimal arc match probability. Write probabilities for only the arc matchs of at least this probability. -b , &ndash;min-bm-prob= &lt;bmprob&gt;(0.001) : Minimal base match probability. Write probabilities for only the base matchs of at least this probability. &ndash;include-am-in-bm : Include arc match cases in base match probabilities &ndash;stopwatch : Print run time informations. Heuristics for speed accuracy trade off: -p , &ndash;min-prob= &lt;prob&gt;(0.001) : Minimal probability. Only base pairs of at least this probability are taken into account. &ndash;max-bps-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #base pairs divided by sequence length. This serves as a second filter on the "significant" base pairs. [default: 0.0 = no effect]. -D , &ndash;max-diff-am= &lt;diff&gt;( -1 ) : Maximal difference for sizes of matched arcs. [-1=off] -d , &ndash;max-diff= &lt;diff&gt;( -1 ) : Maximal difference for positions of alignment traces (and aligned bases). [-1=off] &ndash;max-diff-at-am= &lt;diff&gt;( -1 ) : Maximal difference for positions of alignment traces at arc match ends. [-1=off] &ndash;max-diff-aln= &lt;aln file&gt;() : Maximal difference relative to given alignment (file in clustalw format) &ndash;max-diff-pw-aln= &lt;alignment&gt;() : Maximal difference relative to given alignment (string, delim=AMPERSAND) &ndash;max-diff-relax : Relax deviation constraints in multiple aligmnent &ndash;min-trace-probability= &lt;probability&gt;(1e-5) : Minimal sequence alignment probability of potential traces (probability-based sequence alignment envelope) [default=1e-4]. Computed probabilities: &ndash;fragment-match-probs= &lt;"i j k l"&gt;() : Requests probabilities for the match of fragments [i..j] and [k..l]. Accepts a \';\' separated list of ranges. Constraints: &ndash;maxBPspan= &lt;span&gt;( -1 ) : Limit maximum base pair span [default=off]. &ndash;relaxed-anchors : Use relaxed semantics of anchor constraints [default=strict semantics]. Input files: The tool is called with two input files &lt;Input 1&gt; and &lt;Input 2&gt;, which specify the two input sequences or input alignments. Different input formats (Fasta, Clustal, Stockholm, LocARNA PP, ViennaRNA postscript dotplots) are accepted and automatically recognized (by file content); the two input files can be in different formats. Extended variants of the Clustal and Stockholm formats enable specifying anchor and structure constraints. DISCLAIMER locarna_p is a low level tool; most often this tool is used indirectly, by calling mlocarna with **&ndash;probabilistic. Note that the performance of locarna_p (as well as basically all tools in the LocARNA package) is often significantly improved by the use of suitable application-specific options, deviating from the default settings.
