## Overview
nf-core/radseq is a bioinformatics pipeline designed for best-practice variant calling in Restriction site-associated DNA sequencing (RADseq) experiments. It addresses the challenge of identifying genetic markers in both model organisms with established genomes and non-model organisms where a reference must be constructed de novo.

The pipeline processes raw sequencing reads through quality control, trimming, and alignment. Depending on the availability of a reference, it either maps reads to a known genome or constructs a pseudoreference using clustering and assembly tools. The final outputs are high-quality variant calls in VCF format, accompanied by quality control metrics that summarize the sequencing and alignment performance.

## Data preparation
The pipeline requires a comma-separated (CSV) samplesheet provided via the `--input` parameter. This file must contain a header row and define the sample names and paths to the FASTQ files.

**Samplesheet columns:**
- `sample`: Unique sample identifier (no spaces).
- `fastq_1`: Path to the first-read FASTQ file (must end in `.fq.gz` or `.fastq.gz`).
- `fastq_2`: Path to the second-read FASTQ file for paired-end data (optional, can be empty).
- `umi_barcodes`: A boolean value (`TRUE` or `FALSE`) indicating if the reads contain UMI barcodes for deduplication.

**Example `samplesheet.csv`:**
```csv
sample,fastq_1,fastq_2,umi_barcodes
sample1,data/sample1_R1.fq.gz,data/sample1_R2.fq.gz,FALSE
sample2,data/sample2_R1.fq.gz,,FALSE
```

If using an existing reference genome, you can specify it using the `--genome` parameter (e.g., `--genome GRCh38`) to use iGenomes paths. For de novo analysis, the pipeline can construct a pseudoreference from the input reads.

## How to run
The pipeline is executed using Nextflow with a mandatory input samplesheet and a software profile (e.g., `docker`, `singularity`, or `conda`).

**Basic command:**
```bash
nextflow run nf-core/radseq \
    -profile <docker/singularity/conda> \
    --input samplesheet.csv \
    --outdir ./results
```

**Reference-based run:**
```bash
nextflow run nf-core/radseq \
    -profile docker \
    --input samplesheet.csv \
    --genome GRCh38 \
    --outdir ./results
```

**Common parameters:**
- `-r`: Specify a pipeline release version (e.g., `-r 1.0.0`).
- `-resume`: Restart a previous run from the last successful step.
- `--need_to_trim_fastq`: Enable this flag to trim reads prior to de novo clustering.
- `--save_bam_files`: Save the intermediate alignment BAM files to the output directory.

## Outputs
Results are saved in the directory specified by `--outdir` (default is `./results`).

**Key deliverables:**
- **VCF Files**: Final variant calls generated by FreeBayes, located in the variant calling subfolder.
- **MultiQC Report**: An HTML report (`multiqc_report.html`) summarizing quality control metrics from FastQC, FastP, and alignment steps.
- **Alignments**: If `--save_bam_files` is used, indexed BAM files and their associated metadata are provided.
- **Pipeline Info**: Execution reports, including runtime traces and resource usage.

For a detailed description of all output files and their formats, refer to the [official output documentation](https://nf-co.re/radseq/output).
