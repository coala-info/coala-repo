# v-dev

*(in progress)*

* Allow larger kmers (max 32), without the huge memory allocation overhead.
* Removed `--minkfreq` and expanded `--kmer` to dual coarse/fine kmer comparisons with `--coarse-kmer --fine-kmer`.
* Change default `--seqsim 0.80` and `--sizesim 0.85`.
* Fix python bindings with upgraded pyo3

# v2.0.0

*January 26, 2026*

* New command `kanpig trio` for joint genotyping trio SVs
* New command `kanpig mosaic` for joint genotyping low VAF SVs
* Symbolic deletions `<DEL>` can now be processed
* Optional GQ calibration (see `gqcalibration/README.md`)
* Optional `--rnames` output of `SVID \t read_name` supporting each SV (requires bam `--reads`, not plup)
* BAM/CRAM files can be read from remote URLs
* Removed FORMAT/NE. If PS tags aren't pulled from BAM/plup, the neighborhood id for short-range phase groups will be used which is equal to the 1-based start position of the upstream most SV in the neighborhood
* Comment line of kanpig version/command added to output VCF
* Fixed math to produce more informative GQ and tiny bit better genotypes
* Code cleaning for reusability and easier creation of sub-commands
* Python rust bindings to expose plup parsing, kmer vectorization for scripting
* Functional testing foundations in `repo_utils/`

### Patches:

* v2.0.1 - Minor patch of the dependencies so that bioconda can build/distribute.
* v2.0.2 - Minor patch to fix PS tags when haplotagged reads aren't provided.

# v1.1.0

*April 4, 2025*

* New `--ab` allele balance threshold lowers frequency of assigning erroneous compound heterozygous genotypes
* Experimental `--squish` changes gpenalty behavior to preferring simpler paths
* Fix score's gpenalty
* FORMAT/NE now uses neighborhood's start position for consistent labeling across samples

# v1.0.2

*January 19, 2025*

* UI tweaks, including cleaner `gt` parameters
* Slight htslib bam record and other minor efficiency improvement
* Fix bug arising from ploidy==1 regions sometimes.
* Fix minor bugs around PS and HP tag counting/record keeping

# v1.0.1

*December 10, 2024*

* New pileup index file format (`plup`) to optionally extract read information from a bam/cram. This allows smaller storage of reads and faster genotyping. Testing shows a 100x (113Gb) bam creates a 157Mb `plup.gz` file in 3.5 minutes. Genotyping on the plup takes 4 seconds compared to 2 minutes on the bam. (All tests performed with 16 cores)
* Subcommands of `kanpig gt` for the genotyper and `kanpig plup` for pileup index generation
* `kanpig gt --reads` (formerly `--bam`) can accept a .bam, .cram, or .plup.gz file
* Faster genotyping from bam by ≥2x, Faster genotyping from plup by up to 8x
* Switched read clustering from kmeans to kmedoid for ~1 percentage point (p.p) increase in genotype concordance.
* Leveraging haplotagged reads (Bam tag PS and HP) increases genotype concordance by 0.8p.p.
* FORMAT/NE for identifying neighborhood of variants evaluated together (local phasing)
* FORMAT/PS holds bam tag PS for between neighborhood phasing
* Ensures `--reference` index (`.fai`) exists
* `--fnmax` and `--pileupmax` parameters exposed for advanced use cases
* `--minkfreq` paramater now reflects ≥ instead of >
* SAMPLE columns cleared on reading to reduce memory usage a little bit (or a lot if there are many samples in input VCF)
* Removed experimental parameters `--spanoff` `--prune-graph`, and `--try-exact`.
* New experimental parameter `--one-to-one` will skip graph searching in favor of direct 1-to-1 pileup/variant comparison.
* New parameter `--maxnodes` will turn on `--one-to-one` when a graph would become too complex to reasonably search.
* Slightly faster runtime (upto 10% reduction) by only building variant graphs when there are alternate alleles present.
* Renamed `--chunksize` to `--neighdist` to better align with manuscript's description of kanpig algorithm
* Default `--gpenalty` increased to 0.02
* Fixed minor off-by-one errors for pileup generation
* `demo/` folder to help users perform small sanity testing

# v0.3.1

*June 25, 2024*

* New filtering of haplotypes without paths increases accuracy
* New path [scoring](https://github.com/ACEnglish/kanpig/wiki/Scoring-Function) improves accuracy and consistency
* ZS and SS FORMAT fields replaced by KS reporting the score
* Requiring reads to span the full variant graph window including `--chunksize` buffer increases accuracy
* Exhaustive search of partial haplotypes
* Slight runtime reduction from avoidance of redundant path searches

# v0.3.0

*June 11, 2024*

* ~8% speed increase from less work in the path-searching
* Partial haplotypes bug fix increases accuracy
* Fixed SQ and FT fields
* Phase group (PG) field is now canonical phase set (PS)
* Dedicated writing thread helps reduce memory usage by preventing a backlog of completed variants while reading
* Default `--out` is stdout to allow easier compression/indexing (e.g. `kanpig .. | bcftools sort -O z -o out.vcf.gz`)
* IUPAC codes are fixed by kanpig according to vcf specifications (Issue [#1](https://github.com/ACEnglish/kanpig/issues/1))
* Fixed filtering of symbolic alts and BNDs
* Argument validation

# v0.2.0

*May 21, 2024*

* Up to 40% reduction in runtime
* Hemizygous and sex chromosome aware genotyping with new `--ploidy-bed`
* Variants with alternate alleles of stars, monozygotic reference, and BNDs are filtered out
* PathScores now compared with average of size and sequence similarity for increased accuracy
* Cram files now readable

# v0.1.2

*May 5, 2024*

* New optional hompolymer filter doesn't kmerize long homopolymers
* Improved logging info
* Correcting GQ field
* Correcting kmer counting
* Small speed/memory/io improvements
  + Off-loaded annotation work from the single writer thread to the worker threads and using a large
    multiple of page size for the BufWriter capacity
  + Fewer bam file opens

# v0.1.1

*Apr 11, 2024*

* The `--no-prune` flag has been changed to `--prune` since not pruning is a better default.
* Partial haplotypes now only allow up to 3 false negatives for regions with fewer than 500 pileups. More than 500 do
  not attempt partials.
* Partial haplotypes now respect the `--kmer` option.

# v0.1.0

*Apr 9, 2024*

Initial version. Works well enough to freeze.
