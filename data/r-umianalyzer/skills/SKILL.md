# r-umianalyzer

name: r-umianalyzer
description: Specialized R package for analyzing sequencing data containing Unique Molecular Identifiers (UMIs) generated by UMIErrorCorrect. Use this skill when you need to import UMI data, create UMIexperiment objects, filter consensus tables, call variants, or visualize UMI-based sequencing results in R.

# r-umianalyzer

## Overview
The `umiAnalyzer` package provides tools for processing and analyzing UMI-tagged sequencing data. It allows users to merge multiple samples into a single `UMIexperiment` object, perform variant calling, filter data based on consensus depth, and generate graphical outputs. It is specifically designed to work with output from the `UMIErrorCorrect` pipeline.

## Installation
To install the stable version from CRAN:
```R
install.packages('umiAnalyzer')
```

To install the development version from GitHub:
```R
# Requires devtools
devtools::install_github('sfilges/umiAnalyzer')
```

## Core Workflow

### 1. Creating a UMIexperiment Object
The primary data structure is the `UMIexperiment` object. You initialize it by pointing to the directory containing `UMIErrorCorrect` output folders.

```R
library(umiAnalyzer)

# Path to directory containing sample folders
main_dir <- "path/to/umierrorcorrect_output"

# Create the experiment object
simsen <- createUmiExperiment(mainDir = main_dir)

# Optional: Parse BAM files for additional QC metrics
reads <- parseBamFiles(main_dir, consDepth = 10)
```

### 2. Data Filtering
You can create multiple filters within the same object. Filters are stored by name and can be retrieved for downstream analysis.

```R
# Apply default filters
simsen <- filterUmiObject(simsen, name = "my_filter")

# Retrieve filtered data
filtered_data <- getFilteredData(simsen, filterName = "my_filter")
```

### 3. Variant Calling
The package includes a built-in variant caller and filtering functions for significant variants.

```R
# Call variants
simsen <- callVariants(simsen)

# Filter variants based on p-value and depth
simsen <- filterVariants(
  object = simsen,
  p.adjust = 0.2,
  minDepth = 5
)
```

### 4. Metadata and Experimental Design
Import external metadata (e.g., clinical data or design matrices) to enrich the analysis.

```R
meta_file <- "metadata.txt"
simsen <- importDesign(object = simsen, file = meta_file)

# Retrieve design matrix
design <- getMetaData(simsen, attributeName = 'design')
```

### 5. Visualization
The package provides several plotting functions for QC and data exploration.

```R
# QC Plot
QCplot(simsen)

# Barcode family histogram (requires output from parseBamFiles)
BarcodeFamilyHistogram(reads)

# Amplicon coverage/distribution plot
AmpliconPlot(simsen)
```

### 6. Exporting Results
Generate VCF files for downstream bioinformatics pipelines.

```R
generateVCF(object = simsen, outFile = 'my_results', outDir = 'output_folder')
```

## Interactive Visualization
To launch the built-in Shiny application for graphical data exploration:
```R
umiAnalyzer::runUmiVisualizer()
```
