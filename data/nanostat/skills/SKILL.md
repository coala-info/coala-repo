# nanostat

---

## Overview
NanoStat is a specialized tool for the Oxford Nanopore Technologies (ONT) ecosystem that provides descriptive statistics for sequencing datasets. It allows researchers to quickly assess the quality and yield of a sequencing run by calculating metrics such as mean read length, median quality scores, and N50 values. It is a standard utility for quality control (QC) of long-read data, though for extremely large datasets, the developer suggests "Cramino" as a higher-performance alternative.

## Core Usage Patterns

### Basic Statistics from Reads
To generate a summary report from compressed FASTQ files and save it to a specific directory:
`NanoStat --fastq reads.fastq.gz --outdir QC_reports`

### Processing Alignment Files
NanoStat supports BAM, CRAM, and UBAM files. Use multi-threading to speed up the processing of large alignments:
`NanoStat --bam alignment.bam --threads 8`

### Analyzing Sequencing Summary Files
When using summary files generated by basecallers like Guppy or Albacore, you can specify the read type (1D, 2D, or 1D2):
`NanoStat --summary sequencing_summary.txt --readtype 1D`

### Tabular Output for Downstream Analysis
Use the `--tsv` flag to produce machine-readable output suitable for integration into spreadsheets or automated pipelines:
`NanoStat --fastq reads.fastq.gz --tsv > stats.tsv`

## Command Line Options

| Option | Description |
| :--- | :--- |
| `--fastq`, `--fasta` | Input one or more (compressed) sequence files. |
| `--bam` | Input one or more sorted BAM, CRAM, or UBAM files. |
| `--summary` | Input sequencing summary files from basecallers. |
| `-t, --threads` | Set the number of threads for parallel processing. |
| `-o, --outdir` | Specify the output directory. |
| `-n, --name` | Specify a custom filename for the output (default is stdout). |
| `--barcoded` | Use this flag to split summary file statistics by barcode. |

## Expert Tips
- **Performance**: Always utilize the `-t` flag when processing large BAM or CRAM files to leverage multi-threading.
- **Input Flexibility**: NanoStat natively handles gzipped files; do not waste time or disk space decompressing FASTQ or FASTA inputs.
- **Read Types**: When working with older ONT data or specific library preps, the `--readtype` flag is essential for correctly parsing sequencing summary files.
- **Legacy Note**: NanoStat is largely superseded by `cramino`. If you encounter performance bottlenecks on very large datasets, `cramino` is the recommended successor from the same developer.
