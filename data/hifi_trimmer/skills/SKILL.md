# hifi_trimmer

## Overview
hifi_trimmer is a command-line utility designed to clean HiFi read sets by filtering and trimming extraneous adapter hits. Unlike simple hard-clipping tools, it uses BLAST results to identify adapter locations and applies configurable logic to decide whether a read should be trimmed at the ends or discarded entirely if an adapter is found in the middle. The tool emphasizes reproducibility by generating a BED file as an intermediate manifest of excluded regions before performing the actual sequence filtering.

## Core Workflow

The standard hifi_trimmer pipeline consists of three distinct phases:

1.  **Adapter Identification**: Perform a BLAST search of your HiFi reads against a database of adapter sequences.
2.  **Coordinate Processing**: Use `process_blast` to convert BLAST hits into a BED file based on adapter-specific rules.
3.  **Sequence Filtering**: Use `filter_bam` to generate the final cleaned FASTA or FASTQ file using the BED file.

## Command Line Usage

### 1. Generating the BLAST Input
The tool requires a specific BLAST output format. Use the following parameters with `blastn`:
- `-outfmt "6 std qlen"` (The `qlen` column is mandatory unless using the `--bam` option in the next step).
- Recommended settings: `-reward 1 -penalty -5 -gapopen 3 -gapextend 3 -dust no -soft_masking true`.

### 2. Processing BLAST Results
Run `process_blast` to generate the exclusion BED file.

```bash
hifi_trimmer process_blast [OPTIONS] BLASTOUT ADAPTER_YAML
```

**Key Options:**
- `-p, --prefix`: Set the output basename (defaults to BLASTOUT basename).
- `-el, --end_length`: Define the window size (in bp) at read ends to be considered "ends" [default: 150].
- `-ml, --min_length_after_trimming`: Minimum read length required to keep a read after trimming [default: 300].
- `-hf, --hits`: Exports a TSV of identified hits with extra columns for debugging (read_length, discard, trim_l, trim_r).

### 3. Filtering the BAM File
Apply the BED coordinates to your original BAM to produce filtered sequences.

```bash
hifi_trimmer filter_bam [OPTIONS] BAM BED OUTFILE
```

**Key Options:**
- `-f, --fastq`: Output in FASTQ format instead of the default FASTA.
- `-t, --threads`: Number of threads for bgzip compression.

## Configuration Logic
The tool requires a configuration file in YAML format to define how each adapter is handled. For every adapter (identified by name or regular expression), you must specify:
- Whether to discard the read if the adapter is in the middle.
- Whether to discard or trim the read if the adapter is at the end.
- Minimum percent identity (`pident`) and match length for both middle and end hits.

## Expert Tips & Performance
- **Parallelization**: The tool uses a Polars backend for processing. If it does not automatically respect your scheduler's core count, set the environment variable `POLARS_MAX_THREADS=int` before execution.
- **Reproducibility**: Always keep the `.summary.json` file generated by `process_blast`. It contains raw counts of detected hits and the total length of sequences removed, which is essential for QC reports.
- **End Window Sensitivity**: If you have short adapters or specific library prep artifacts, adjust `--end_length`. Hits outside this window are automatically treated as "middle" hits and subject to `discard_middle` logic.
