# consensus-peak-calling-for-chip-seq-single-end-replicates

## Overview

This workflow identifies high-confidence consensus peaks from single-end ChIP-seq, ATAC-seq, or CUT&RUN replicates. It processes a collection of BAM files where PCR duplicates have been removed, requiring user-defined parameters for effective genome size, bin size, and the minimum number of replicates required for a peak to be considered "shared."

The analysis follows a two-part strategy to ensure statistical robustness. First, it calls peaks and computes normalized coverage for each replicate individually using [MACS2](https://github.com/macs3-project/MACS). It then generates an average normalized profile and identifies the intersection of peaks across replicates, filtering for those that meet the user-specified overlap threshold.

To control for differences in sequencing depth, the workflow subsamples all replicates to match the smallest library size. These normalized datasets are combined for a secondary round of peak calling. The final "shared" peak set is generated by retaining only the peaks from this combined pool whose summits overlap with the intersections identified in the individual replicates.

Key outputs include individual and merged narrowPeak files, an average bigWig profile for visualization, and a [MultiQC](https://multiqc.info/) report for quality assessment. This tool is released under the MIT license and is optimized for identifying consistent enrichment patterns across multiple experimental replicates.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | n rmDup BAMSR | data_collection_input | A collection with n replicates. BAM should not have duplicates |
| 1 | Minimum number of overlap | parameter_input | Minimum number of replicates into which the summit should be present. |
| 2 | effective_genome_size | parameter_input | Used by MACS2: H. sapiens: 2700000000, M. musculus: 1870000000, D. melanogaster: 120000000, C. elegans: 90000000 |
| 3 | bin_size | parameter_input | Bin size for average of normalized coverage |


Ensure your input BAM files are organized into a list of datasets (collection) with PCR duplicates already removed to ensure accurate peak calling across replicates. You must provide the effective genome size for MACS2 and a bin size for the averaged bigWig profile, while the overlap parameter determines the stringency of the final consensus peak set. For automated execution, you can initialize a job template using `planemo workflow_job_init` to generate a `job.yml` file. Refer to the `README.md` for detailed guidance on parameter selection and data preparation strategies.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run consensus-peaks-chip-sr.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run consensus-peaks-chip-sr.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run consensus-peaks-chip-sr.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init consensus-peaks-chip-sr.ga -o job.yml`
- Lint: `planemo workflow_lint consensus-peaks-chip-sr.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
