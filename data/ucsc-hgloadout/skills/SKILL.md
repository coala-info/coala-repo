# ucsc-hgloadout

## Overview
The `ucsc-hgloadout` utility is a specialized data-loading tool from the UCSC Kent Utilities suite. It is designed to parse the standard output format of RepeatMasker (typically `.out` files) and insert that data into a MySQL database table. This is a critical step for researchers building local mirrors of the UCSC Genome Browser or managing custom assembly databases where repeat annotation tracks are required.

## Usage Guidelines

### Prerequisites
*   **Database Configuration**: The tool requires a connection to a MySQL server. You must have an `.hg.conf` file in your home directory containing the database credentials (host, user, password).
*   **Permissions**: Ensure the binary has execution permissions (`chmod +x hgLoadOut`).
*   **RepeatMasker Data**: You must have the `.out` files generated by a RepeatMasker run.

### Basic Command Pattern
The tool is typically invoked as `hgLoadOut`. The standard syntax follows this pattern:

```bash
hgLoadOut [database] [table] [input.out]
```

*   **database**: The name of the target MySQL database (e.g., `hg38`).
*   **table**: The name of the table to create/populate (usually `rmsk`).
*   **input.out**: The path to the RepeatMasker output file.

### Common CLI Patterns
*   **Loading Multiple Files**: If RepeatMasker was run on individual chromosomes, you can often use shell globbing or a loop to load data, though `hgLoadOut` is typically used per-database/table.
*   **Joined Loading**: For specific workflows requiring joined output, the related utility `hgLoadOutJoined` may be used if the data has already been pre-processed into a joined format.

### Expert Tips
*   **Table Naming**: By convention, the repeat track in the UCSC Genome Browser is named `rmsk`. Using this name ensures compatibility with default track configurations.
*   **Public MySQL Access**: While this tool is for loading *into* a database, you can reference the UCSC public MySQL server (`genome-mysql.soe.ucsc.edu`) to inspect existing `rmsk` table schemas to ensure your local database structure matches.
*   **Binary Location**: If installed via Bioconda, the tool is available in your PATH. If downloaded manually from the UCSC server, it is located in the architecture-specific directory (e.g., `admin/exe/linux.x86_64/`).
