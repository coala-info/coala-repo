# proteomiqon-labelfreeproteinquantification

## Overview

The `proteomiqon-labelfreeproteinquantification` tool is the final aggregation component of the ProteomIQon label-free proteomics pipeline. It transforms quantified peptide ion data into protein-level abundances through a multi-step hierarchical aggregation process. The tool allows for fine-grained control over how different charge states and modifications are handled before the final roll-up into protein groups. It is essential for researchers needing to convert raw peptide intensities into biologically interpretable protein quantification values.

## Command Line Usage

### Basic Execution
To generate quantified protein output for a single run, use the following command structure:

```bash
proteomiqon -labelfreeproteinquantification \
  -i "path/to/your/run.quantAndProt" \
  -o "path/to/your/outDirectory" \
  -p "path/to/your/params.json"
```

### Batch Processing
To process multiple files simultaneously, list the input files followed by the output directory:

```bash
proteomiqon -labelfreeproteinquantification \
  -i "run1.quantAndProt" "run2.quantAndProt" "run3.quantAndProt" "outDirectory" \
  -p "params.json"
```

## Parameter Configuration

The tool requires a JSON parameter file to define the aggregation logic.

### Aggregation Hierarchy
The tool performs aggregation in three potential stages. If an optional step is skipped, the subsequent step inherits the results from the previous one:
1.  **Charge State Aggregation (Optional):** Combines peptides with the same sequence and modifications but different charge states.
2.  **Modification Aggregation (Optional):** Combines peptides with the same sequence but different modifications.
3.  **Protein Group Aggregation (Required):** Aggregates all associated peptides into the final protein group abundance.

### Key Parameter Fields
*   `ModificationFilter`: Determines which modified peptides are included (e.g., `All`, `No`, or `UseOnly`).
*   `Aggregation`: Defines the mathematical method for combining intensities:
    *   `Mean`
    *   `Median`
    *   `Sum`
*   `Transform`: Optional intensity transformations (e.g., Log2) applied during aggregation.
*   `SingleFilters` / `GroupFilters`: Used to apply intensity thresholds or quality filters to the data being aggregated.

## Expert Tips and Best Practices

*   **Input Requirements:** Ensure your input files are in the `.quantAndProt` format. This format is typically generated by the `JoinQuantPepIonsWithProteins` tool, which merges quantification data with protein inference results.
*   **Sequential Logic:** Remember that aggregation steps build upon each other. If you skip charge state aggregation but enable modification aggregation, the tool will aggregate modified peptides directly from the raw peptide ion list.
*   **Reproducibility:** Always maintain a version-controlled repository of your `params.json` files, as small changes in aggregation methods (e.g., switching from `Mean` to `Median`) can significantly impact downstream differential expression analysis.
*   **Help Command:** For a full list of available CLI arguments and flags, run:
    `proteomiqon -labelfreeproteinquantification --help`
