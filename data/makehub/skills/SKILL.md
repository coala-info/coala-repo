# makehub

## Overview

MakeHub is a Python-based pipeline that automates the complex process of creating UCSC track data hubs. It handles the conversion of various bioinformatics file formats into the specialized "Big" formats (BigBed, BigWig, 2bit) required by the UCSC Genome Browser. It is particularly useful for researchers who have completed genome annotation projects and need a shareable, interactive visualization of their results.

## Core Workflows

### Creating a New Assembly Hub
To initialize a hub, you must provide a genome FASTA file and basic metadata.

```bash
python3 make_hub.py --genome genome.fasta --email user@example.com --short_label MySpecies --long_label 'My Species Name (Common Name)'
```

### Adding Annotation Tracks
MakeHub supports gene models in GFF3 or GTF format. It is optimized for outputs from BRAKER and MAKER.

```bash
python3 make_hub.py --genome genome.fasta --gene_track predictions.gff3 --short_label MySpecies
```

### Visualizing RNA-Seq Data
Provide BAM files to generate coverage tracks (BigWig). MakeHub uses `samtools` and optionally `bam2wig` for this conversion.

```bash
python3 make_hub.py --genome genome.fasta --bam alignment1.bam alignment2.bam --short_label MySpecies
```

## Expert Tips and Best Practices

- **Dependency Management**: MakeHub attempts to download UCSC binary tools (like `bedToBigBed`) automatically if they are not in your PATH. For stable production environments, manually install the UCSC Kent utilities and `samtools` (version 1.8+).
- **GeMoMa Integration**: If using GeMoMa predictions, use the included helper script to convert GFF3 to the required AUGUSTUS-like GTF format before running the main pipeline:
  ```bash
  python3 gemoma_to_augustus_like_gtf.py -i gemoma.gff3 -o formatted_predictions.gtf
  ```
- **Softmasking**: If your genome FASTA is softmasked (repeats in lowercase), MakeHub will automatically detect this and create a "RepeatMasker" track.
- **Hub Hosting**: The output directory generated by MakeHub must be placed on a publicly accessible web server (or a password-protected one that the UCSC Genome Browser can reach via URL) to be viewed.
- **Updating Hubs**: You can add new tracks to an existing hub by running the command again with the same `--short_label` and the new data files.



## Subcommands

| Command | Description |
|---------|-------------|
| bedToBigBed | Convert bed file to bigBed. |
| faToTwoBit | Convert DNA from fasta to 2bit format |
| gtfToGenePred | convert a GTF file to a genePred |
| hgGcPercent | Calculate GC Percentage in 20kb windows |
| ixIxx | Create indices for simple line-oriented file of format <symbol> <free text> |
| make_hub.py | Generate UCSC assembly hub (e.g. from BRAKER or MAKER output). |
| makehub_bam2wig | Converts BAM files to WIG format for visualization. |
| python3 | Run a program in Python |
| samtools | Tools for alignments in the SAM format |
| twoBitInfo | get information about sequences in a .2bit file |
| wigToBigWig | Convert ascii format wig file (in fixedStep, variableStep or bedGraph format) to binary big wig format (bbi version: 4). |
