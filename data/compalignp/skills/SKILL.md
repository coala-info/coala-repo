# compalignp

## Overview
The `compalignp` tool is a specialized utility designed to quantify the accuracy of sequence alignments. It functions by comparing a test alignment (the output of an alignment tool) against a trusted reference alignment (often manually curated or structurally verified). By calculating the fractional identity, it provides a standardized metric to assess how well the test alignment preserves the positional relationships defined in the reference.

## Usage Guidelines

### Basic Command Structure
The tool typically requires two input files representing the alignments to be compared.
```bash
compalignp <trusted_alignment> <test_alignment>
```

### Input Requirements
- **Trusted Alignment**: This should be your reference or "gold standard" alignment file.
- **Test Alignment**: This is the alignment generated by the method you are currently evaluating.
- **Format**: Ensure both files are in a compatible alignment format (typically Clustal or FASTA, depending on the specific build environment).

### Interpreting Results
- The output is a fractional value representing the "identity" or "accuracy" of the test alignment relative to the trusted one.
- A value of **1.0** indicates a perfect match where all aligned positions in the test file correspond exactly to those in the reference.
- Lower values indicate misalignments or shifts in the sequences compared to the reference standard.

### Expert Tips
- **Consistency**: Ensure that the sequence headers/identifiers in both the trusted and test files are identical; otherwise, the tool may fail to map the sequences correctly.
- **Benchmarking**: Use this tool as part of a pipeline when testing new alignment parameters or software versions to objectively measure performance improvements.
- **Environment**: If the command is not found, ensure it is installed via bioconda using `conda install bioconda::compalignp`.
