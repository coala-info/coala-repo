# classpro

## Overview
ClassPro is a specialized bioinformatics tool designed to categorize every k-mer within a HiFi read into one of four functional types: Erroneous (E), Haploid (H), Diploid (D), or Repeat (R). It operates on k-mer count profiles generated by FASTK. This classification is essential for downstream genomic tasks like assembly filtering, variant calling, and understanding the repetitive landscape of a diploid genome. It is optimized for HiFi reads with sequencing coverage between 10x and 100x.

## Installation and Dependencies
ClassPro requires pre-computed k-mer profiles.
*   **Required**: [FASTK](https://github.com/thegenemyers/FASTK) must be installed to generate `.hist` and `.prof` files.
*   **Optional**: `DAZZ_DB` is required if using `.db` or `.dam` files as input.
*   **Conda**: `conda install bioconda::classpro`

## Core Workflow
The standard pipeline involves generating a k-mer profile with FASTK followed by classification with ClassPro.

### 1. Generate FASTK Profiles
FASTK must be run with specific flags (`-t1` and `-p`) for ClassPro compatibility.
```bash
# -k40: k-mer size 40
# -t1: keep all k-mer counts
# -p: generate profiles
FastK -v -k40 -t1 -p reads.fasta.gz
```

### 2. Run ClassPro Classification
Run ClassPro using the base name of the sequence file.
```bash
ClassPro -v reads.fasta.gz
```
This produces `reads.class`, a FASTQ-like file where the "quality" string contains the classification characters:
*   **E**: Erroneous k-mer
*   **H**: Haploid k-mer
*   **D**: Diploid k-mer
*   **R**: Repeat k-mer
*   **N**: Undefined (first k-1 bases)

## Command Line Options
*   `-T <int>`: Set thread count (default 4). Performance gains usually plateau at 16 threads due to I/O bottlenecks.
*   `-c <int>`: Manually set average diploid sequencing coverage. If omitted, ClassPro estimates this from the histogram.
*   `-r <int>`: Set average read length (default 20000). A rough estimate is sufficient.
*   `-P <dir>`: Specify a temporary directory for intermediate files.
*   `-s`: Enable preliminary feature to find alignment seeds based on classification.
*   `-N <name>`: Explicitly provide the FASTK output root name if it differs from the source filename.

## Evaluation and Utility Tools
The ClassPro suite includes auxiliary programs for validation:
*   `ClassGS`: Performs classification using global thresholds from GenomeScope2.0.
*   `class2acc`: Compares a classification file against a ground truth file to calculate accuracy.
*   `prof2class`: Generates a "ground truth" classification file if you have a relative profile against a known reference genome (useful for simulations).

## Best Practices and Limitations
*   **Ploidy**: ClassPro is strictly designed for diploid genomes. It does not currently support polyploid organisms.
*   **Read Type**: Optimized for HiFi reads. Using noisy reads (like standard ONT) may result in sub-optimal classification.
*   **Input Constraints**: Currently supports only a single sequence file per execution. BAM/CRAM formats are not natively supported; convert to FASTA/FASTQ first.
*   **FASTK Parameters**: Always ensure FASTK is run with `-t1`. Without this flag, ClassPro cannot access the full k-mer distribution needed for accurate E/H/D/R assignment.
