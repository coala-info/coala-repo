• 1 direct dependent &#10; • 1 total dependent 5 ++ 5 ++ / Devel::Assert Contents NAME SYNOPSIS USAGE DESCRIPTION CHANGES FROM 0.04 SEE ALSO AUTHOR COPYRIGHT LICENSE NAME Devel::Assert - assertions for Perl &gt;= 5.14 SYNOPSIS use Devel::Assert; # release mode, won&#39;t perform checks assert(3 &lt; 2); use Devel::Assert &#39;on&#39;; assert(3 &lt; 2); # oops, check your math USAGE use Devel::Assert; # import &#39;assert&#39; function, but doesn&#39;t enable it use Devel::Assert &#39;on&#39;; # import &#39;assert&#39; function and enable it use Devel::Assert &#39;global&#39;; # import &#39;assert&#39; function and enable it in all later &#39;use Devel::Assert&#39; imports use Devel::Assert::Global; # the same, can be used as perl -MDevel::Assert::Global your_program.pl use Devel::Assert &#39;off&#39;; # import &#39;assert&#39; function, but doesn&#39;t enable it, even when in &#39;global&#39; mode DESCRIPTION This module provides you with a C-like assert() function. And, like in C, it only is compiled in debug mode - it&#39;s completely removed together with it&#39;s arguments at compile time in release mode. That&#39;s useful to force functions&#39; contracts during development without sacrificing production performance. Think of an assert() being a test, but right inside your code: sub get_url { my ($self, $url, $cb) = @_; assert length $url; assert(ref($cb) eq &#39;CODE&#39;); ... } Failed assertions are reported by confess with failed test code embedded: Assertion &#39;ref $cb eq &#39;CODE&#39;&#39; failed at ... This module doesn&#39;t use source filters or any other form of source text parsing, but instead relies on the generated optree analysis and manipulation. CHANGES FROM 0.04 This version significantly differs from 0.04 release. The main reason behind such radical change was to get rid of the Devel::Declare dependency and to pass all the parsing work to Perl itself (instead of creating own statement parser).
