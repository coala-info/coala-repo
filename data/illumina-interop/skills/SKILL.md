# illumina-interop

## Overview
The `illumina-interop` skill enables the extraction and analysis of binary metric files generated by Illumina sequencing systems (e.g., NovaSeq, NextSeq, MiSeq). These files, typically stored in the `InterOp` folder of a run directory, contain critical metadata about run performance. This skill focuses on using the command-line interface (CLI) applications to convert these binary formats into human-readable tables or summary statistics for quality control and secondary analysis.

## Common CLI Applications
The InterOp package includes several standalone tools. The most frequently used commands include:

- `index-summary`: Reports indexing performance, including the number of reads assigned to each sample and the percentage of identified vs. unidentified indexes.
- `summary`: Provides a high-level overview of the run, including total yield, projected yield, and the percentage of bases >= Q30.
- `imaging-stats`: Extracts per-tile and per-cycle statistics for intensities, FWHM, and other imaging metrics.
- `dumptext`: A general-purpose tool to dump the contents of any InterOp binary file into a tab-delimited text format.

## Usage Patterns

### Generating a Run Summary
To get a quick overview of the run quality (equivalent to the "Summary" tab in Illumina SAV):
```bash
summary <run_directory_path>
```

### Extracting Indexing Results
To check demultiplexing performance and index hits:
```bash
index-summary <run_directory_path>
```

### Converting Binary Metrics to CSV
To export specific metrics (e.g., Quality Metrics) for custom analysis in R or Python:
```bash
dumptext <run_directory_path> --subset quality > quality_metrics.csv
```

### Filtering by Lane or Surface
Most tools support filtering to isolate specific hardware issues:
```bash
imaging-stats <run_directory_path> --lanes 1 --surfaces top
```

## Expert Tips
- **Run Directory Structure**: Ensure you point the tools at the root of the sequencing run directory (the folder containing `RunInfo.xml`), not the `InterOp` subfolder itself. The tools require `RunInfo.xml` to parse the binary data correctly.
- **Backwards Compatibility**: These tools support all modern Illumina platforms (NextSeq 1k/2k, NovaSeq, etc.) but do not support legacy GA (Genome Analyzer) systems.
- **Python Integration**: For complex data science workflows, the `interop` Python package (available via pip/conda) provides direct access to the C++ API for reading metrics into dataframes without intermediate text files.
