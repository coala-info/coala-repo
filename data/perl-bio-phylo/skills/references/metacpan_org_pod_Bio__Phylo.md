this module directly. This is # the base class for other modules. use Bio::Phylo; # sets global verbosity to &#39;error&#39; Bio::Phylo-&gt;VERBOSE( -level =&gt; Bio::Phylo::Util::Logger::ERROR ); # sets verbosity for forest ojects to &#39;debug&#39; Bio::Phylo-&gt;VERBOSE( -level =&gt; Bio::Phylo::Util::Logger::DEBUG, -class =&gt; &#39;Bio::Phylo::Forest&#39; ); # prints version, including SVN revision number print Bio::Phylo-&gt;VERSION; # prints suggested citation print Bio::Phylo-&gt;CITATION; DESCRIPTION This is the base class for the Bio::Phylo package for phylogenetic analysis using object-oriented perl5. In this file, methods are defined that are performed by other objects in the Bio::Phylo release that inherit from this base class (which you normally wouldn&#39;t use directly). For general information on how to use Bio::Phylo, consult the manual ( Bio::Phylo::Manual ). If you come here because you are trying to debug a problem you run into in using Bio::Phylo, you may be interested in the &quot;exceptions&quot; system as discussed in Bio::Phylo::Util::Exceptions . In addition, you may find the logging system in Bio::Phylo::Util::Logger of use to localize problems. METHODS CONSTRUCTOR new() The Bio::Phylo root constructor is rarely used directly. Rather, many other objects in Bio::Phylo internally go up the inheritance tree to this constructor. The arguments shown here can therefore also be passed to any of the child classes&#39; constructors, which will pass them on up the inheritance tree. Generally, constructors in Bio::Phylo subclasses can process as arguments all methods that have set_* in their names. The arguments are named for the methods, but &quot;set_&quot; has been replaced with a dash &quot;-&quot;, e.g. the method &quot;set_name&quot; becomes the argument &quot;-name&quot; in the constructor. Type : Constructor Title : new Usage : my $phylo = Bio::Phylo-&gt;new; Function: Instantiates Bio::Phylo object Returns : a Bio::Phylo object Args : Optional, any number of setters. For example, Bio::Phylo-&gt;new( -name =&gt; $name ) will call set_name( $name ) internally MUTATORS set_guid() Sets invocant GUID. Type : Mutator Title : set_guid Usage : $obj-&gt;set_guid($guid); Function: Assigns an object&#39;s GUID. Returns : Modified object. Args : A scalar Notes : This field can be used for storing an identifier that is unambiguous within a given content. For example, an LSID, a genbank accession number, etc. set_desc() Sets invocant description.
