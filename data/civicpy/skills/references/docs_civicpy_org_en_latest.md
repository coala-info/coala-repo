CIViCpy ¶ v5.1.1. CIViCpy is an analysis toolkit and SDK for extracting and analyzing knowledge from the CIViC knowledgebase . If you find CIViCpy useful for your work, please cite our manuscript ! Note The use of Python 3 is mandatory for CIViCpy. Python 2 has been sunset as of January 1, 2020 . Using CIViCpy : &gt;&gt;&gt; from civicpy import civic &gt;&gt;&gt; my_variant_ids = [ 12 , 306 , 79 ] &gt;&gt;&gt; my_variants = civic . get_variants_by_ids ( my_variant_ids ) &gt;&gt;&gt; my_variants [&lt;CIViC variant (gene_variant) 12&gt;, &lt;CIViC variant (gene_variant) 306&gt;, &lt;CIViC variant (gene_variant) 79&gt;] &gt;&gt;&gt; [( v . gene . name , v . name ) for v in my_variants ] [(&#39;BRAF&#39;, &#39;V600E&#39;), (&#39;KRAS&#39;, &#39;G12D&#39;), (&#39;ERBB2&#39;, &#39;AMPLIFICATION&#39;)] &gt;&gt;&gt; braf_id = my_variants [ 0 ] . feature_id # or my_variants[0].gene.id or my_variants[0].feature.id &gt;&gt;&gt; braf_variants = civic . get_gene_by_id ( braf_id ) . variants &gt;&gt;&gt; len ( braf_variants ) 95 &gt;&gt;&gt; set ( my_variants ) &amp; set ( braf_variants ) {&lt;CIViC variant (gene_variant) 12&gt;} CIViCpy lets you pull data from CIViC using the civic module and interact with records as dynamic objects. With the aid of caching, it is easy to explore relationships between CIViC records (e.g. assertions, genes, variants) without worrying about querying CIViC more than once for the same data.
