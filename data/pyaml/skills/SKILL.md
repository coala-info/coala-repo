# pyaml

## Overview

This skill provides procedural knowledge for using the `pyamlboot` utility to communicate with Amlogic System-on-Chips (SoCs) over USB. It is primarily used for low-level recovery, development, and booting of devices when standard storage media like SD cards or eMMC are empty, removed, or bypassed. The tool supports different protocols depending on the SoC generation (G12 vs. GXL/GXM/GXBB) and allows for pushing U-Boot binaries, Linux kernels, and initial RAM disks directly to the device's memory.

## Installation and Setup

Install the library and tools via pip:

```bash
pip3 install pyamlboot
```

Note: Most commands require `sudo` to access the USB device nodes unless udev rules are specifically configured for the Amlogic USB ID.

## CLI Usage Patterns

### G12 Series (S905X2, S905D2, S905X3, S922X, A311D)
The G12 protocol is more restrictive and is primarily used to chain-load U-Boot. It does not support loading a full Linux kernel and RootFS directly from memory via the tool.

**Basic Command:**
```bash
sudo boot-g12.py /path/to/u-boot.bin
```

**Target Boards:**
- Odroid-C4, N2, N2+
- Khadas VIM3, VIM3L

### GXL, GXM, and GXBB Series (S905X, S912, S905)
The older protocol supports more complex boot scenarios, including passing Device Tree Blobs (DTB) and initramfs.

**Basic Command Structure:**
```bash
sudo boot.py [board_type] --image [kernel] --fdt [dtb] --ramfs [initrd] --script [boot_scr]
```

**Supported Board Types:**
- `khadas-vim`
- `khadas-vim2`
- `libretech-cc` (Le Potato)
- `libretech-ac` (La Frite)
- `s400`
- `q200`

**Example for Libretech-CC:**
```bash
sudo boot.py --image arch/arm64/boot/Image --fdt arch/arm64/boot/dts/amlogic/meson-gxl-s905x-libretech-cc.dtb --ramfs rootfs.cpio.uboot --script boot.scr libretech-cc
```

## Hardware Triggering for USB Boot

To use these tools, the SoC must be in USB Boot mode. Common methods include:

1. **Media Removal:** Remove the eMMC module and SD card. The Boot ROM will fail over to USB mode.
2. **TST Mode (Khadas):** For VIM3, quickly press the Function key 3 times within 2 seconds to enter Upgrade/USB mode.
3. **SPI Erase (Libretech-AC):** If the SPI flash is populated, the first sectors may need to be erased to prevent it from booting, forcing a failover to USB.

## Expert Tips

- **U-Boot Binary:** The `u-boot.bin` used with `boot-g12.py` should be the final binary generated by Amlogic's build tools (often after the `mkboot` stage).
- **Boot Scripts:** If you modify `boot.cmd`, you must re-generate the `boot.scr` using `mkimage`:
  `mkimage -C none -A arm -T script -d boot.cmd boot.scr`
- **Serial Monitoring:** Always connect a Serial-to-USB adapter to the board's UART pins to monitor the boot log, as the USB tool may report success even if the loaded binary fails to execute.
- **G12 Limitation:** Since `boot-g12.py` cannot load a RootFS into memory, configure U-Boot to fetch the Linux kernel and RootFS via Ethernet (TFTP/NFS) or Fastboot once it has started.
