# ontime

## Overview

The `ontime` utility allows for the precise extraction of Nanopore reads by leveraging the `start_time` metadata found in read headers. It is designed to help researchers determine the optimal sequencing duration for their specific applications by allowing them to "slice" a single long sequencing run into various time-based subsets. It supports both relative durations (e.g., "the first 2 hours") and absolute UTC timestamps.

## Core Workflows

### 1. Inspecting File Timeframes
Before subsetting, always check the temporal range of your input file to understand the start and end points of the sequencing run.
```bash
ontime --show input.fastq
```

### 2. Extracting by Duration
Durations are the most common way to subset data. Units like `h` (hours), `m` (minutes), and `s` (seconds) can be combined.
- **First N hours**: `ontime --to 2h in.fq`
- **After the first hour**: `ontime --from 1h in.fq`
- **Specific window**: `ontime --from 1h30m --to 3h in.fq`

### 3. Relative Offsets from End-of-Run
Use negative durations to subset reads relative to the end of the sequencing session.
- **Exclude the last hour**: `ontime --to -1h in.fq`
- **The final 30 minutes**: `ontime --from -30m in.fq`

### 4. Absolute Timestamp Filtering
When you need to match sequencing data to a specific external event, use RFC3339-compliant timestamps (UTC).
```bash
ontime --from 2022-12-12T20:00:00Z --to 2022-12-12T21:00:00Z in.fq
```

## Expert Tips and Best Practices

- **Input/Output Formats**: `ontime` supports FASTQ, FASTA, BAM, and SAM. However, it does not perform format conversion. If your input is BAM, your output must be BAM. Use `samtools` if you need to convert formats after subsetting.
- **Compression**: Save disk space by outputting directly to compressed formats. `ontime` detects compression from the file extension or via the `-O` flag.
  - Example: `ontime --to 5h -o output.fq.gz in.fq`
- **UTC Requirement**: All timestamps provided to `--from` or `--to` must be in UTC. If your lab notes are in local time, convert them to UTC before running the command.
- **Metadata Dependency**: This tool relies on the `start_time=` (FASTQ/FASTA) or `st:Z:` (BAM/SAM) tags. If reads have been processed by tools that strip this metadata, `ontime` will not be able to filter them.
- **Dorado BAMs**: `ontime` is compatible with BAM files generated by the Dorado basecaller, which includes the necessary start time tags.
