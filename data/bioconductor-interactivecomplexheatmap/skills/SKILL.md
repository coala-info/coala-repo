# bioconductor-interactivecomplexheatmap

## Overview

The `InteractiveComplexHeatmap` package provides a bridge between the static, high-quality heatmaps produced by `ComplexHeatmap` and interactive web applications. It allows users to capture sub-heatmaps by clicking or brushing areas on a heatmap, making it ideal for exploring large-scale genomic datasets. It supports not only standard heatmaps but also specialized plots like OncoPrints, UpSet plots, and Enriched heatmaps.

## Core Workflows

### 1. Basic Interactivity
The simplest way to make a heatmap interactive is using `htShiny()`.

```r
library(ComplexHeatmap)
library(InteractiveComplexHeatmap)

# Create a standard heatmap
m = matrix(rnorm(100*100), nrow = 100)
ht = Heatmap(m)

# Recommended: draw it first to cache clustering
ht = draw(ht)

# Launch the Shiny app
htShiny(ht)
```

### 2. Interactivating Last Generated Heatmap
If a heatmap was generated by a package that doesn't return the object (e.g., `cola` or `pheatmap`), `htShiny()` can capture the last plot.

```r
library(InteractiveComplexHeatmap)
# Ensure the package is loaded before plotting
ComplexHeatmap::pheatmap(matrix(rnorm(100), 10))
htShiny()
```

### 3. Specialized Plots
Any object inheriting from `Heatmap` or `HeatmapList` can be interactivated.

*   **OncoPrint:** `htShiny(oncoPrint(mat))`
*   **UpSet Plot:** `htShiny(UpSet(make_comb_mat(list)))`
*   **Density Heatmap:** `htShiny(densityHeatmap(mat))`
*   **Enriched Heatmap:** `htShiny(EnrichedHeatmap(mat))`

### 4. Integration with DESeq2
The package provides a high-level function `interactivate()` for specific Bioconductor objects.

```r
# For a DESeqDataSet object 'dds'
interactivate(dds)
```

## Advanced Usage

### Customizing Layout and Response
You can control which events the heatmap responds to and how the UI is arranged.

```r
# Only respond to clicks, not brushing
htShiny(ht, response = "click")

# Float the output UI at the mouse position
htShiny(ht, output_ui_float = TRUE)

# Compact mode (removes sub-heatmap, floats output)
htShiny(ht, compact = TRUE)
```

### Adding Decorations
To include graphics added via `decorate_*()` functions in the interactive app, wrap them in a `post_fun`.

```r
post_fun = function(ht_list) {
    decorate_heatmap_body("matrix_name", {
        grid.text("Annotation", gp = gpar(fontsize = 20))
    })
}
ht = draw(ht, post_fun = post_fun)
htShiny(ht)
```

### Shiny Development (From Scratch)
For custom Shiny apps, use the UI/Server pair:

*   **UI:** `InteractiveComplexHeatmapOutput("id")`
*   **Server:** `makeInteractiveComplexHeatmap(input, output, session, ht, "id")`

To get raw indices from a click/brush without the default UI:
1. Use `htPositionsOnDevice(ht)` inside `renderPlot`.
2. Use `getPositionFromClick(input$click)` and `selectPosition()`.
3. Use `getPositionFromBrush(input$brush)` and `selectArea()`.

## Tips and Best Practices
*   **Pre-drawing:** Always call `draw(ht)` before `htShiny(ht)` if the heatmap involves randomness (like k-means) to ensure the interactive version matches the static one.
*   **Large Matrices:** For very large matrices, use `htShiny(ht, compact = TRUE)` to save screen space.
*   **Recursive Interactivity:** In the default `htShiny` app, you can click "Interactivate sub-heatmap" to open a new layer of interactivity for a selected region.
*   **Examples:** Run `htShinyExample()` to see a list of over 50 built-in examples, then run `htShinyExample(1.1)` to launch a specific one.
