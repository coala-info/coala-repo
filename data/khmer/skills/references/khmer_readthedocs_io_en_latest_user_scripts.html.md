Below is our documentation for these scripts. Note that all scripts can be given -h / --help which will print out a list of arguments taken by that script. Scripts that use k-mer counting tables or k-mer graphs take an -M parameter, which sets the maximum memory usage in bytes. This should generally be set as high as possible; see Setting khmer memory usage for more information. k-mer counting and abundance filtering Partitioning Digital normalization Read handling: interleaving, splitting, etc. Note Almost all scripts take in either FASTA and FASTQ format, and output the same.
