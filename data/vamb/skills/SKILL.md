# vamb

## Overview
Vamb (Variational Autoencoder for Metagenomic Binning) is a suite of tools designed to cluster metagenomic contigs into biological bins. It transforms sequence composition and abundance information into a latent space using deep learning, allowing for highly accurate binning. The package includes the standard Vamb binner, the semi-supervised TaxVamb (which incorporates taxonomic data), and Taxometer for refining taxonomic predictions. It is optimized for multi-sample datasets but remains effective for single-sample binning.

## Installation
Install Vamb via Bioconda:
```bash
conda install bioconda::vamb
```

## Core Workflow and CLI Patterns

### 1. Preprocessing Contigs
Vamb requires a specific naming convention for contigs in multi-sample projects: `S{sample}C{contig}`. Use the provided helper script to concatenate and rename your assemblies.
```bash
# Assuming assemblies are in asm_1, asm_2, etc.
python path/to/vamb/src/concatenate.py contigs.fna.gz asm_*/contigs.fasta
```

### 2. Abundance Estimation
While any mapper works, `strobealign` with the `--aemb` flag is recommended for speed and compatibility.
```bash
mkdir aemb
for sample in 1 2 3; do
    strobealign -t 8 --aemb contigs.fna.gz ${sample}.fw.fq.gz ${sample}.rv.fq.gz > aemb/${sample}.tsv
done

# Merge outputs into a single TSV
python path/to/vamb/src/merge_aemb.py aemb abundance.tsv
```

### 3. Running the Binner
The primary command for standard binning:
```bash
vamb bin default --outdir vambout --fasta contigs.fna.gz --abundance_tsv abundance.tsv
```

## Expert Tips and Best Practices

*   **Binner Selection**: 
    *   Use **TaxVamb** if you have taxonomic annotations (e.g., from MMseqs2). it consistently produces the highest quality results.
    *   Use **Vamb (default)** for a balance of speed and accuracy without requiring external annotations.
    *   **Avoid Avamb**: It is considered obsolete; TaxVamb is superior for high-compute scenarios.
*   **Contig Filtering**: Vamb automatically filters out contigs shorter than 2000bp by default. You can adjust this with `--minlength`, but shorter contigs often lack sufficient k-mer signal for reliable VAE embedding.
*   **GPU Acceleration**: Vamb is significantly faster on a GPU. If a CUDA-enabled device is available, Vamb will detect and use it automatically to speed up the VAE training process.
*   **Memory Management**: For very large datasets (hundreds of samples), abundance TSV files can become massive. Ensure your environment has sufficient RAM or consider using the `.npz` format for abundance data if generated by previous Vamb runs.
*   **Taxometer**: If your taxonomic annotations are noisy, run `taxometer` to refine them before passing them to TaxVamb.
