# nanofilt

## Overview
NanoFilt is a specialized tool designed for the refinement of long-read sequencing datasets. It processes FASTQ data to filter out reads that do not meet specific quality, length, or GC content criteria. Additionally, it can perform "cropping" to remove a fixed number of nucleotides from the start or end of reads, which is often necessary to eliminate adapter sequences or low-quality terminal regions. The tool is designed for high efficiency, primarily operating through Unix pipes (stdin/stdout) to minimize disk I/O.

## Command Line Usage and Patterns

### Basic Filtering
The most common use case involves filtering reads by a minimum average quality score (Phred) and a minimum length.
```bash
# Filter for reads with Q > 10 and length > 500bp
cat reads.fastq | NanoFilt -q 10 -l 500 > filtered_reads.fastq
```

### Handling Compressed Data
Since NanoFilt reads from stdin, it is best practice to stream gzipped FASTQ files directly into the tool.
```bash
# Stream gzipped input and compress the output
gunzip -c reads.fastq.gz | NanoFilt -q 12 -l 1000 | gzip > high_quality.fastq.gz
```

### Trimming and Cropping
Use cropping to remove potential adapter sequences or known low-quality regions at the beginning or end of reads.
```bash
# Trim 50bp from the start and 20bp from the end
NanoFilt --headcrop 50 --tailcrop 20 input.fastq > cropped.fastq
```

### Advanced Filtering (GC Content)
Filter reads based on GC content percentage (expressed as a float between 0.0 and 1.0).
```bash
# Keep reads with GC content between 40% and 60%
cat reads.fastq | NanoFilt --minGC 0.4 --maxGC 0.6 > gc_filtered.fastq
```

## Expert Tips and Best Practices

### Use Summary Files for Performance
When working with Oxford Nanopore data, using the `sequencing_summary.txt` file generated by basecallers (like Guppy or Albacore) is significantly faster than calculating quality scores from the FASTQ file itself.
```bash
# Use summary file for faster quality filtering
NanoFilt -s sequencing_summary.txt -q 10 reads.fastq > filtered.fastq
```

### Integration in Mapping Pipelines
To save time and storage, pipe NanoFilt output directly into an aligner like `minimap2`.
```bash
gunzip -c reads.fastq.gz | NanoFilt -q 10 -l 500 | minimap2 -ax map-ont genome.fa - | samtools sort -o aligned.bam
```

### Note on Deprecation
While NanoFilt is a standard tool in many legacy pipelines, the author recommends `chopper` for newer workflows as it provides similar functionality with significantly higher performance.
