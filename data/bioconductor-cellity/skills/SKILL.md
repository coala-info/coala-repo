# bioconductor-cellity

## Overview

The `cellity` package identifies low-quality cells in scRNA-seq data by extracting a suite of biological and technical features. It is particularly useful for detecting "deceptive" cells that might appear normal under a microscope but exhibit poor transcriptomic quality. The package supports both human and mouse data and offers two primary detection methods: PCA-based outlier detection and SVM-based classification.

## Core Workflow

### 1. Data Preparation and Normalization
`cellity` requires a gene expression matrix (Genes x Cells) and a read statistics matrix (Cells x Metrics). Counts should be normalized before feature extraction.

```r
library(cellity)
data(sample_counts)
data(sample_stats)

# Normalization for HTSeq counts (library size)
sample_counts_nm <- normalise_by_factor(sample_counts, colSums(sample_counts))

# For Cufflinks data, transform FPKM to TPM before proceeding
```

### 2. Feature Extraction
Extract features that distinguish high-quality from low-quality cells. This returns a list containing "all" features (cell-type specific) and "common" features (cell-type independent).

```r
# Default extraction (Mouse)
sample_features <- extract_features(sample_counts_nm, sample_stats)

# For Human data (requires org.Hs.eg.db)
data("extra_human_genes")
data("feature_info")
features_human <- extract_features(
  sample_counts_nm, 
  sample_stats, 
  common_features = feature_info[[2]],
  GO_terms = feature_info[[1]], 
  extra_genes = extra_human_genes, 
  organism = "human"
)
```

### 3. Quality Assessment Methods

#### Method A: PCA-based Outlier Detection
Best for exploratory analysis or when no training labels are available. It identifies outliers in the first two principal components.

```r
# Perform PCA assessment
sample_features_all <- sample_features[[1]]
sample_qual_pca <- assess_cell_quality_PCA(sample_features_all, file = "quality_plots.pdf")

# Output: Matrix where 0 = low quality, 1 = high quality
```

#### Method B: SVM Classification
Best when you have training data or want to use the pre-trained mES (mouse embryonic stem cell) models provided by the package.

```r
# Scenario: Using pre-trained mES model
data(param_mES_all) # Hyperparameters
data(training_mES_features)
data(training_mES_labels)

mES_quality_SVM <- assess_cell_quality_SVM(
  training_mES_features[[1]], 
  training_mES_labels[,2], 
  param_mES_all,
  sample_features[[1]]
)
```

### 4. Hybrid Approach
It is recommended to run both PCA and SVM methods. High agreement between the two (typically >85%) increases confidence in the filtering results.

## Usage Tips
- **Organism Support**: Currently supports Human and Mouse. Ensure the correct `org.Db` package is installed.
- **Feature Selection**: Use "all" features if your cells are similar to the training set (e.g., mES). Use "common" features for higher generalizability across different cell types.
- **Visual Inspection**: Always check the PDF output from `assess_cell_quality_PCA`. It highlights which features (e.g., mitochondrial proportion, intergenic reads) are driving the outlier classification.
- **Input Stats**: The `sample_stats` matrix is ideally generated by the `celloline` pipeline, but any matrix with equivalent read metrics (mapped reads, etc.) can be used.
