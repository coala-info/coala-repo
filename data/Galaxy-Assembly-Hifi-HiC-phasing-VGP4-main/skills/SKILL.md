# Genome Assembly from Hifi reads with HiC phasing - VGP4

## Overview

This Galaxy workflow generates a phased genome assembly using PacBio HiFi long reads and Hi-C data from the same individual. As part of the Vertebrate Genomes Project (VGP) suite, it is designed to be executed after the k-mer profiling workflow (VGP1). The pipeline utilizes [Hifiasm](https://github.com/chhylp123/hifiasm) to perform contigging, leveraging Hi-C integration to resolve haplotypes and produce high-quality, phased assembly contigs.

The workflow requires several inputs from the preceding VGP1 stage, including a Meryl k-mer database, GenomeScope model parameters, and estimated genome size. It also supports optional preprocessing of Hi-C reads, such as trimming 5' bases to reduce noise in contact maps for specific sequencing protocols like Arima.

The primary outputs include the Haplotype 1 and Haplotype 2 assemblies in both FASTA and GFA formats. Comprehensive quality control is integrated throughout the process, generating [Compleasm](https://github.com/huangnengCSU/compleasm) (BUSCO) reports, [Merqury](https://github.com/marbl/merqury) histograms and QV stats, [gfastats](https://github.com/VGGP/gfastats) assembly metrics, and visual plots for Nx and assembly size.

| 5 | Genomescope Summary | data_input | GenomeScope summary generated by K-mer profiling workflow |
| 6 | Meryl Database | data_input | Meryl database generated by K-mer profiling workflow |
| 7 | Database for Busco Lineage | parameter_input | Select the database to use for Busco lineages. |
| 8 | Lineage | parameter_input | Taxonomic lineage for the organism being assembled for Busco analysis |
| 9 | Name for Haplotype 1 | parameter_input | Name of the first haplotype without blank space. |
| 10 | Name for Haplotype 2 | parameter_input | Name of the second haplotype without blank space. |
| 11 | Bits for bloom filter | parameter_input | Defaults to 37 if not specified. For genomes much larger than human, applying -f38 or even -f39 is preferred to save memory on k-mer counting. |
| 12 | Homozygous Read Coverage | parameter_input | If empty, read coverage will be estimated from the Genomescope parameters. |
| 13 | Genomescope Model Parameters | data_input | GenomeScope model parameters generated by K-mer profiling workflow |


Ensure that PacBio HiFi reads and Hi-C paired-end reads are provided as data collections in fastq format to maintain proper sample association. This workflow requires specific outputs from the VGP1 k-mer profiling workflow, including the Meryl database, the GenomeScope summary text file, and the model parameters tabular file. Users should verify the taxonomic lineage for Compleasm and may need to adjust the bloom filter bits for memory management when processing large genomes. For a comprehensive guide on parameter settings and prerequisite steps, refer to the README.md file. You can use `planemo workflow_job_init` to generate a `job.yml` file for automated execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Assembly-Hifi-HiC-phasing-VGP4.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Assembly-Hifi-HiC-phasing-VGP4.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Assembly-Hifi-HiC-phasing-VGP4.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Assembly-Hifi-HiC-phasing-VGP4.ga -o job.yml`
- Lint: `planemo workflow_lint Assembly-Hifi-HiC-phasing-VGP4.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
