# ucsc-maftoaxt

## Overview
`mafToAxt` is a specialized utility from the UCSC Genome Browser "kent" toolset designed to bridge the gap between Multiple Alignment Format (MAF) and the axt format. While MAF is highly flexible for representing multiple sequences, many pairwise analysis tools and indexing workflows require the more structured, pairwise-focused axt format. This tool extracts pairwise alignments from a MAF file and formats them according to the axt specification, which includes alignment scores and specific coordinate metadata.

## Command Line Usage

The basic syntax for `mafToAxt` requires the input MAF file, the sequence data for the target and query (usually in .2bit format), and the destination path for the axt file.

### Basic Syntax
```bash
mafToAxt input.maf target.2bit query.2bit output.axt
```

### Common Parameters and Flags
- `target.2bit`: The sequence file for the "target" (reference) genome.
- `query.2bit`: The sequence file for the "query" genome.
- `-stripDb`: Use this flag to remove the database prefix from sequence names (e.g., changing `hg38.chr1` to `chr1`). This is often required to match naming conventions in downstream analysis.
- `-bed`: Output a BED file alongside the axt to track the coordinates of the alignments.

## Expert Tips and Best Practices

### Sequence Data Requirements
`mafToAxt` needs access to the actual genomic sequences to validate the alignment and correctly calculate coordinates. Using `.2bit` files is the most efficient method. If your sequences are in FASTA format, convert them using `faToTwoBit` before running the conversion.

### Handling Database Prefixes
MAF files generated by UCSC pipelines often prefix sequence names with the assembly version (e.g., `panTro6.chr10`). If your downstream tools expect standard chromosome names, always include the `-stripDb` flag to ensure compatibility.

### Coordinate Systems
Be aware that both MAF and axt formats use 0-based, half-open coordinates. If you are coming from a 1-based format (like GFF or SAM), ensure your mental model of the coordinates is adjusted accordingly when verifying the output.

### Validating Output
After conversion, you can use `axtCheck` (another UCSC utility) to verify that the resulting axt file is well-formed and matches the sequences provided in the 2bit files.
