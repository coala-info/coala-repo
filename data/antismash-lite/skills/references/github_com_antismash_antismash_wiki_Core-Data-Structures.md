# Core Data Structures

AntiSMASH 5 uses a variety of common data structures to cover both infrastructure needs (module results, command line arguments, etc) and ease of working with the data being analysed.

It isn't intended that this page list every attribute and method for these datastructures, only to point out where they are and what their intended use is. For an overview of each class, the python `help()` function can be used (e.g. `import antismash; print(help(antismash.config.args.ModuleArgs))`) or commandline tool `pydoc` (e.g. `pydoc antismash.config.args.ModuleArgs`).

## Infrastructure data structures

### antismash.config.args.ModuleArgs

This is used by every module to set up any command line options desired. An instance of this, or a class inheriting from this class, must be returned by every module's `get_arguments()` implementation.

### antismash.common.module\_results.ModuleResults

An instance of this class, or a class inheriting from this class, must be returned by every module's `run_on_record()` implementation.

### antismash.common.module\_results.DetectionResults

An instance of this class (which subclasses `ModuleResults`) is to be returned from all detection modules. This allows for an empty `add_to_record()` method as detection results are required to be present in the record for analysis modules to use, long before the final adding of results to the record.

## Input data structures

### antismash.common.secmet.Record

This class contains all the information usually kept in file formats such as Genbank or FASTA files with GFF annotations. It contains the contig sequence along with all annotated features. In most cases, attribute names used by BioPython's `SeqRecord` class are shared, with the notable exception of `.features` as features are gathered by type in `secmet.Record`.

Along with actual data, the `Record` class has many helper methods for finding specific sets of features. These methods range from simple fetching by type (e.g. `.get_pfam_domains()`) to more specific `get_cds_features_within_location(location)`.

### antismash.common.secmet.Feature

All features within a `Record` will be an instance of this class or a subclass of this class. Each feature class contains the relevant information that a feature type does within a Genbank file (e.g. all classes have a location and all CDS features have a translation).

Example `Feature` subclasses: `CDSFeature`, `PFAMDomain`, `Cluster`, `ClusterBorder`
