# nanopolish

## Overview

Nanopolish is a suite of tools designed to analyze the raw electrical signal data (FAST5 or SLOW5) produced by Oxford Nanopore sequencers. While basecallers convert signals into text sequences, nanopolish uses Hidden Markov Models (HMMs) to re-examine the raw signal in the context of a reference or draft genome. This allows for higher accuracy in consensus sequences and the detection of epigenetic modifications that are not visible in standard FASTQ files.

## Core Workflow: Data Indexing

The first step in any nanopolish workflow is linking your basecalled FASTQ reads to their corresponding raw signal files.

### Indexing FAST5 Data
To index FAST5 files efficiently, always provide the sequencing summary file generated by the basecaller (e.g., Guppy or Albacore).
```bash
nanopolish index -d /path/to/raw_fast5s/ -s sequencing_summary.txt basecalled_reads.fastq
```
*Tip: If you have multiple summary files, use `-f` with a text file containing the paths to all summaries.*

### Indexing SLOW5 Data
SLOW5/BLOW5 files are a more performant alternative to FAST5. Indexing them does not require a sequencing summary file.
```bash
nanopolish index basecalled_reads.fastq --slow5 signals.blow5
```

## Common Analysis Modules

### 1. Consensus Polishing
To improve a draft assembly, you must first align your reads to the draft (using `minimap2` or `bwa mem`) and then run the variants module in consensus mode.

**Step A: Alignment**
```bash
bwa index draft.fa
bwa mem -x ont2d -t 8 draft.fa reads.fastq | samtools sort -o reads.sorted.bam -
samtools index reads.sorted.bam
```

**Step B: Polishing (Parallelized)**
Polishing is computationally intensive and should be run in parallel across genomic windows.
```bash
python3 nanopolish_makerange.py draft.fa | parallel --results nanopolish.results -P 8 \
  nanopolish variants --consensus -o polished.{1}.vcf -w {1} -r reads.fastq -b reads.sorted.bam -g draft.fa
```

**Step C: Generate Polished Fasta**
```bash
nanopolish vcf2fasta -g draft.fa polished.*.vcf > polished_assembly.fa
```

### 2. Methylation Calling
Detect 5-methylcytosine (5mC) in a genomic context.
```bash
nanopolish call-methylation -t 8 -r reads.fastq -b reads.sorted.bam -g reference.fa > methylation_calls.tsv
```

### 3. Poly-A Tail Estimation
Estimate the length of poly-A tails for direct RNA sequencing.
```bash
nanopolish polya -t 8 -r reads.fastq -b reads.sorted.bam -g reference.fa > polya_results.tsv
```

## Expert Tips and Best Practices

- **R10.4 Support**: Nanopolish is primarily optimized for R9.4.1 data. For R10.4+ flow cells, signal-level analysis for variant calling is often unnecessary as modern basecallers (like Dorado) handle these accurately.
- **Memory Management**: If `nanopolish variants` crashes due to memory, reduce the window size (`-w`) or the number of parallel threads.
- **Signal Data Access**: Ensure the raw FAST5/SLOW5 files remain at the path specified during the `index` step, or nanopolish will fail to load the signal.
- **Filtering**: For methylation and variant calling, using high-quality, long reads typically yields better HMM convergence.



## Subcommands

| Command | Description |
|---------|-------------|
| nanopolish call-methylation | Classify nucleotides as methylated or not. |
| nanopolish detect-polyi | Detect presence of poly(I) tails and estimate length of tails in direct RNA reads |
| nanopolish eventalign | Align nanopore events to reference k-mers |
| nanopolish fast5-check | Check whether the fast5 files are indexed correctly and readable by nanopolish |
| nanopolish index | Build an index mapping from basecalled reads to the signals measured by the sequencer |
| nanopolish methyltrain | Train a methylation model |
| nanopolish phase-reads | Output a BAM file where each record shows the combination of alleles from variants.vcf that each read supports. variants.vcf can be any VCF file but only SNPs will be phased and variants that have a homozygous reference genotype (0/0) will be skipped. |
| nanopolish polya | Estimate the length of the poly-A tail on direct RNA reads |
| nanopolish scorereads | Score reads against an alignment, model |
| nanopolish variants | Find SNPs using a signal-level HMM |
| nanopolish vcf2fasta | Write a new genome sequence by introducing variants from the input files |
