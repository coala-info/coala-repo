# samestr

## Overview

SameStr is a specialized bioinformatic tool designed to detect and compare microbial strains across metagenomic datasets. Unlike tools that only consider majority variants, SameStr analyzes all possible variants within alignments to create high-resolution Single Nucleotide Variant (SNV) profiles. This allows for precise strain tracking between related sample pairs (e.g., fecal microbiota transplantation donors and recipients) and the construction of strain co-occurrence networks.

## Core Workflow

The standard SameStr pipeline follows a sequential progression from database preparation to final summary reporting.

### 1. Database Preparation
Before processing samples, you must have a reference database of clade-specific markers.
- **Precomputed**: Download from Zenodo (recommended for MetaPhlAn or mOTUs).
- **Custom**: Generate from MetaPhlAn marker files.
  ```bash
  samestr db --markers markers.pkl --output_dir ./samestr_db
  ```

### 2. Profile Generation
Convert alignments (SAM files) into SNV profiles (NumPy format).
- **From Metagenomes**: Convert SAM alignments generated by MetaPhlAn or mOTUs.
  ```bash
  samestr convert --input_file sample.sam --db_dir ./samestr_db --output_dir ./profiles
  ```
- **From Reference Genomes**: Extract profiles from FASTA files to use as a baseline.
  ```bash
  samestr extract --input_file genome.fasta --db_dir ./samestr_db --output_dir ./ref_profiles
  ```

### 3. Profile Management
Merge and filter profiles to ensure data quality before comparison.
- **Merge**: Combine profiles from multiple sources or samples.
  ```bash
  samestr merge --input_dir ./profiles --output_dir ./merged_profiles
  ```
- **Filter**: Remove low-quality or low-coverage profiles.
  ```bash
  samestr filter --input_dir ./merged_profiles --output_dir ./filtered_profiles
  ```

### 4. Comparison and Summarization
Determine the similarity between samples and identify shared strains.
- **Compare**: Calculate similarity and overlapping coverage between profile pairs.
  ```bash
  samestr compare --input_dir ./filtered_profiles --output_dir ./comparison_results
  ```
- **Summarize**: Generate final tables of shared strain calls and co-occurrence metrics.
  ```bash
  samestr summarize --input_dir ./comparison_results --output_dir ./summary_tables
  ```

## Expert Tips and Best Practices

- **Alignment Compatibility**: SameStr is highly sensitive to the alignment tool version. Ensure you use `samtools==0.1.19` for legacy compatibility if working with older MetaPhlAn 2.6.0 workflows.
- **Memory Management**: SNV profiles are stored in NumPy (`.npy`) format. When merging large cohorts, ensure sufficient RAM is available as SameStr loads profiles into memory for comparison.
- **Coverage Statistics**: Always run `samestr stats` on your generated profiles to check depth and breadth of coverage before performing comparisons. Low coverage can lead to false-negative shared strain calls.
  ```bash
  samestr stats --input_dir ./profiles --output_file coverage_stats.tsv
  ```
- **Database Integrity**: If using custom databases, run the built-in integrity check (available in versions 2024.1+) to ensure the marker sequences and taxonomy mappings are correctly formatted.



## Subcommands

| Command | Description |
|---------|-------------|
| samestr compare | Compare SNV profiles between samples. |
| samestr convert | Convert MetaPhlAn or mOTUs marker alignments to a standardized format. |
| samestr summarize | Summarize pairwise alignment comparisons to identify shared strains. |
| samestr_db | Database check arguments: |
| samestr_extract | Extracts marker sequences from input FASTA files based on a specified clade and database. |
| samestr_filter | Filter SNV profiles based on various criteria. |
| samestr_merge | Merge SNV profiles from multiple samples. |
| samestr_stats | Report statistics on SNV profiles. |
