# linkedsv

## Overview
LinkedSV is a specialized structural variant caller optimized for 10X Genomics linked-read data. It leverages the unique barcode information (BX tags) provided by linked-reads to identify genomic rearrangements with high sensitivity and precision. This skill provides guidance on executing LinkedSV for germline and somatic variant detection in both whole-genome (WGS) and targeted sequencing (WES) contexts.

## Installation and Setup
LinkedSV requires Python (2.7 or 3), SAMtools (>= 1.5), and BEDTools.

**Conda Installation (Recommended):**
```bash
conda install bioconda::linkedsv
```

**Manual Compilation:**
If installing from source, you must compile the C++ components:
```bash
git clone https://github.com/WGLab/LinkedSV.git
cd LinkedSV/
make
```

## Common CLI Patterns

### 1. Germline WGS Detection
The most common use case for identifying inherited structural variants from whole-genome data.
```bash
python linkedsv.py -i input.phased_possorted_bam.bam -d ./output_dir -r reference.fa -v hg38 -t 8 --germline_mode
```

### 2. Targeted Sequencing (WES)
When working with whole-exome or other capture-based data, you must specify the target regions.
```bash
python linkedsv.py -i input.bam -d ./output_dir -r reference.fa -v hg38 -t 8 --targeted --target_region capture_targets.bed --germline_mode
```

### 3. Somatic SV Detection
Used for identifying tumor-specific rearrangements.
```bash
python linkedsv.py -i tumor.phased_possorted_bam.bam -d ./somatic_output -r reference.fa -v hg38 -t 8 --somatic_mode
```

## Parameter Best Practices
*   **Reference Version (`-v`):** Always specify `hg19`, `b37`, or `hg38` if using human data. This allows LinkedSV to automatically apply internal blacklists and gap region filters.
*   **Resource Allocation:** Each thread (`-t`) typically requires ~4GB of RAM. Ensure your environment has sufficient memory (e.g., 32GB for 8 threads).
*   **Input Requirements:** The input BAM must be position-sorted and contain the `BX` tag. It is highly recommended to use the `phased_possorted_bam.bam` file generated by the 10X Genomics Longranger pipeline.
*   **Reference Indexing:** The reference FASTA must be indexed with `samtools faidx` prior to running LinkedSV.

## Output Interpretation
LinkedSV generates three primary BEDPE files in the output directory:
1.  `*.small_deletions.bedpe`: Deletions detected via short-read evidence (discordant pairs and local assembly).
2.  `*.large_cnv.bedpe`: Copy number variants identified through read-depth analysis.
3.  `*.filtered_large_svcalls.bedpe`: The primary output for large structural variants detected using barcode overlapping evidence.

## Expert Tips
*   **Custom Genomes:** If using a non-human reference or a custom build, you must manually provide `--gap_region_bed` and `--black_region_bed` files, as the `-v` auto-configuration only supports standard human builds.
*   **Visualization:** LinkedSV produces visualization plots for large SV calls. Check the output directory for PNG/PDF files to manually inspect high-confidence candidates.
*   **Temporary Files:** Use `--save_temp_files` only for debugging, as these can consume significant disk space during large WGS runs.
