# bioconductor-lrbase.dre.eg.db

## Overview

LRBase.Dre.eg.db is a specialized Bioconductor annotation package that serves two primary roles:
1. **Class Definition**: It defines the `LRBaseDb` class, which provides a unified interface for accessing ligand-receptor interaction data.
2. **Package Construction**: It provides the `makeLRBasePackage` function, allowing users to generate their own species-specific or project-specific ligand-receptor database packages (e.g., `FANTOM5.Hsa.eg.db`).

This package is essential for researchers performing cell-cell communication analysis, particularly those using the `scTensor` workflow.

## Core Workflows

### Loading the Package
```r
library(LRBase.Dre.eg.db)
```

### Creating a Custom LRBase Package
To create a new annotation package from your own ligand-receptor list:
1. Prepare a data frame with `GENEID_L` (Ligand NCBI Gene IDs) and `GENEID_R` (Receptor NCBI Gene IDs).
2. Prepare a meta information table.
3. Use `makeLRBasePackage`.

```r
# Example workflow (conceptual)
makeLRBasePackage(lrvalues, metavalues, destDir = ".", version = "0.99.0")

# After generation, install the resulting source package
install.packages("path/to/generated/package", repos = NULL, type = "source")
```

### Querying LRBase Objects
All packages generated by this framework (and the base package itself) use standard AnnotationDbi-style methods:

*   **columns(db)**: List the types of data available (e.g., GENEID_L, GENEID_R, SOURCEDB).
*   **keytypes(db)**: List the types of keys that can be used for filtering.
*   **keys(db, keytype)**: Retrieve all keys of a specific type.
*   **select(db, keys, columns, keytype)**: Perform the actual data retrieval.

```r
# Example: Querying a generated package (e.g., FANTOM5.Hsa.eg.db)
library(FANTOM5.Hsa.eg.db)

# Get all receptor IDs
receptors <- keys(FANTOM5.Hsa.eg.db, keytype = "GENEID_R")

# Find corresponding ligands for specific receptors
results <- select(FANTOM5.Hsa.eg.db, 
                  keys = receptors[1:5], 
                  columns = c("GENEID_L", "GENEID_R"), 
                  keytype = "GENEID_R")
```

### Metadata and Database Info
Use these functions to inspect the database properties:
*   `species(db)`: Common name of the organism.
*   `nomenclature(db)`: Scientific name.
*   `listDatabases(db)`: Sources of the LR interaction data.
*   `dbInfo(db)`: General package information.
*   `dbconn(db)`: Returns the underlying SQLite connection.

## Tips for Success
*   **Gene IDs**: Ensure your input IDs are NCBI (Entrez) Gene IDs, as the "eg" in the package name indicates Entrez Gene orientation.
*   **Integration**: This package is often used as a backend for `scTensor`. If you are performing tensor decomposition on single-cell data, ensure your LRBase object is correctly formatted to match your expression matrix row names.
*   **Interactive Help**: Use `example('makeLRBasePackage')` to see a live demonstration of package construction using FANTOM5 demo data.
