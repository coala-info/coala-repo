# reviewer

## Overview
REViewer (Repeat Expansion Viewer) is a specialized tool for the manual inspection and validation of tandem repeat genotypes. While standard genome browsers often struggle with the complex alignments found in repetitive regions, REViewer uses the graph-realigned reads produced by ExpansionHunter to create clear, haplotype-aware visualizations. Use this skill when you need to generate pileup images to distinguish between true repeat expansions, sequencing artifacts, or mis-genotyped loci.

## Usage Requirements
To use REViewer effectively, you must have the following inputs prepared:
- **BAMlet**: The specific BAM file containing graph-realigned reads generated by ExpansionHunter (v3.0.0 or higher).
- **VCF**: The variant call file produced by the same ExpansionHunter run.
- **Reference**: The FASTA file for the reference genome used during alignment.
- **Variant Catalog**: The JSON file defining the repeat regions.
- **Sorted/Indexed Reads**: The BAMlet file provided to the `--reads` parameter **must** be sorted and indexed (using `samtools sort` and `samtools index`).

## Command Line Interface
The primary command follows this structure:

```bash
REViewer \
  --reads <realigned_reads.bam> \
  --vcf <expansion_hunter_output.vcf> \
  --reference <ref_genome.fasta> \
  --catalog <variant_catalog.json> \
  --locus <locus_name> \
  --output-prefix <output_path_prefix>
```

### Key Parameters
- `--locus`: Specifies the specific repeat locus to analyze (e.g., "HTT" or "C9orf72").
- `--output-prefix`: Determines the naming convention for the resulting SVG/PNG visualization files and metrics.

## Expert Tips and Best Practices
- **Haplotype Resolution**: REViewer is unique because it attempts to resolve reads into specific haplotypes. If a visualization looks messy, check the phasing metrics in the output files to see if the tool successfully separated the alleles.
- **Input Validation**: If REViewer crashes, ensure your variant catalog matches the one used by ExpansionHunter. Using a catalog with features not supported by REViewer (such as certain complex variant locus features) is a common cause of failure.
- **Large Scale Review**: For projects involving hundreds of samples, use the companion tool **FlipBook** (an image server) to efficiently browse the pileups generated by REViewer.
- **Quality Metrics**: Beyond images, REViewer outputs quality metrics and phasing information (often in a `.tsv` or `.json` format). Always check the `diplotype_score` to assess the confidence of the visualized alignment.
- **Ambiguous Nucleotides**: The tool provides specific visual cues for alignments against ambiguous nucleotides; pay close attention to these when inspecting "noisy" repeat regions.
