# bioconductor-pd.rae230b

## Overview

The `pd.rae230b` package is a Bioconductor annotation data package specifically designed for use with the `oligo` package. It provides the necessary platform design information for the Affymetrix Rat Expression 230B array. This package is essential for low-level processing of .CEL files, including preprocessing, normalization, and feature-level analysis.

## Usage and Workflows

### Loading the Package
The package is typically loaded automatically by `oligo` when reading CEL files, but can be loaded manually:

```r
library(pd.rae230b)
```

### Integration with oligo
The primary use case is providing the platform design (pd) object to `read.celfiles`. The `oligo` package identifies the correct annotation package from the CEL file header.

```r
library(oligo)
# Read CEL files - pd.rae230b is used internally
affyData <- read.celfiles(filenames = list.celfiles())

# Perform RMA normalization
eset <- rma(affyData)
```

### Accessing Sequence Data
The package contains sequence information for Perfect Match (PM) probes. This is useful for researchers performing sequence-specific analysis or probe-level quality control.

```r
# Load the PM sequence data
data(pmSequence)

# View the structure (DataFrame with 'fid' and 'sequence' columns)
head(pmSequence)

# Access specific sequences
# 'fid' corresponds to the feature ID in the oligo object
```

### Key Data Objects
- **pmSequence**: A `DataFrame` containing the sequences for PM probes and their corresponding feature IDs (`fid`).
- **bgSequence**: Sequence data for background probes (if applicable).
- **mmSequence**: Sequence data for Mismatch (MM) probes (if applicable).

## Tips
- **Memory Management**: Annotation packages can be large. Ensure your R session has sufficient memory when processing large batches of Rat 230B arrays.
- **Platform Specificity**: This package is specifically for the "B" component of the Rat Expression 230 set. Ensure your samples were run on this specific array and not the "A" version (`pd.rae230a`).
- **Feature IDs**: Use the `fid` column to map sequences back to the expression data generated by `oligo`.
