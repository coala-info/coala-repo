# grid

## Overview
The Growth Rate Index (GRiD) tool provides a method for estimating the in situ replication rates of bacteria. By analyzing the ratio of DNA coverage between the origin of replication (ori) and the terminus (ter), GRiD determines how quickly specific bacteria are growing within a sample. This skill assists in configuring the two primary modules: `single` (for known reference genomes) and `multiplex` (for community-wide analysis without prior knowledge of composition).

## Core Workflows

### Single Genome Analysis
Use the `single` module when you have a specific reference genome in FASTA format and want to determine its growth rate across one or more samples.

```bash
grid single -r <reads_dir> -g <reference.fasta> -o <output_dir> [options]
```

*   **Input Formats**: Supports `.fq`, `.fastq`, `.fastq.gz`, and `.sam`.
*   **SAM Files**: Only valid in the `single` module. Ensure SAM files were generated by mapping reads to the specific reference genome provided.
*   **Key Outputs**: 
    *   A PDF plot showing coverage distribution across the genome.
    *   A text table containing the GRiD score, 95% confidence intervals, and quality metrics like `dnaA/ori` and `ter/dif` ratios (values close to 1.0 indicate higher accuracy).

### Multiplexed Community Analysis
Use the `multiplex` module for high-throughput growth analysis of all identified bacteria in a metagenomic sample.

```bash
grid multiplex -r <reads_dir> -d <database_dir> -o <output_dir> [options]
```

*   **Database Requirement**: You must provide a GRiD database. Use environment-specific databases (e.g., stool, skin) to reduce runtime compared to the comprehensive database.
*   **Coverage Cutoff**: Controlled by `-c`. The default is 1. If set below 1 (minimum 0.2), the tool only analyzes genomes with fragmentation levels below 90 fragments/Mbp.
*   **Read Reassignment**: Use the `-p` flag to enable Pathoscope2 for reassigning ambiguous reads that map to multiple genomes. Adjust the "theta prior" with `-t` to control the stringency of reassignment.

## Expert Tips and Best Practices

*   **Paired-End Data**: GRiD requires single-end format. If you have paired-end reads, either use only one of the mate pairs (R1 or R2) or concatenate them into a single file before processing.
*   **Output Directories**: Always specify a non-existing folder or an empty folder for the `-o` flag to prevent file conflicts.
*   **Sample Filtering**: If the input directory contains many files but you only want to process a few, use the `-l` flag to point to a text file listing the specific sample names.
*   **Species Heterogeneity**: Monitor the "Species Heterogeneity" metric in the output (0 to 1 scale). A value of 0 indicates no heterogeneity; higher values suggest that closely related strains may be confounding the growth prediction.
*   **Performance**: Always utilize the `-n` flag to specify multiple threads, as the alignment and reassignment steps are computationally intensive.
