# Overview

The DiMSum pipeline processes raw sequencing reads (in FASTQ format) or variant counts from deep mutational scanning (DMS) experiments to calculate estimates of variant fitness (and assocated error). These estimates are suitable for use in downstream analyses of epistasis, [fitting interpretable models](https://github.com/lehner-lab/MoCHI) and [protein structure determination](https://github.com/lehner-lab/DMS2structure).

The DiMSum pipeline consists of five stages grouped into two modules that can be run independently:

* ***WRAP*** (DiMSum stages 1-3) processes raw FASTQ files generating a table of variant counts
* ***STEAM*** (DiMSum stages 4-5) analyses variant counts generating variant fitness and error estimates

Further details of individual DiMSum pipeline stages can be found [here](/lehner-lab/DiMSum/blob/master/docs/PIPELINE.md).

# Installation

The easiest way to install DiMSum is by using the [bioconda package](http://bioconda.github.io/recipes/r-dimsum/README.html) to create a dedicated Conda environment:

```
conda create -n dimsum r-base=4.0 fastqc=0.11 r-dimsum
```

See the full [Installation Instructions](/lehner-lab/DiMSum/blob/master/docs/INSTALLATION.md) for further details and alternative installation options.

# Usage

In the example below, DiMSum will obtain variant sequences by aligning paired-end reads in the directory "FASTQ\_dir", count variant occurrences for all samples specified in the supplied [Experimental Design File](/lehner-lab/DiMSum/blob/master/docs/FILEFORMATS.md#experimental-design-file) ("experimentDesign.txt") and calculate fitness (and error) for all variants relative to the indicated wild-type sequence.

```
DiMSum --fastqFileDir FASTQ_dir --experimentDesignPath experimentDesign.txt --wildtypeSequence AGCTAGCT
```

By default, [output files](/lehner-lab/DiMSum/blob/master/docs/FILEFORMATS.md#output-files) are saved to the folder "DiMSum\_Project" in the current working directory.

See instructions regarding [Command-line Arguments](/lehner-lab/DiMSum/blob/master/docs/ARGUMENTS.md), [File Formats](/lehner-lab/DiMSum/blob/master/docs/FILEFORMATS.md) and the [Demo](/lehner-lab/DiMSum/blob/master/docs/DEMO.md) mode for full details and usage options.
