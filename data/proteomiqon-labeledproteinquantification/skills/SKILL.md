# proteomiqon-labeledproteinquantification

name: proteomiqon-labeledproteinquantification
description: >-
  Performs labeled protein quantification by aggregating peptide intensities from

  you need to combine and quantify peptides from light and heavy labeled samples
  to determine protein abundance.
```
## Overview

The `proteomiqon-labeledproteinquantification` tool is designed to aggregate peptide intensity data to the protein level, specifically for experiments involving labeled proteins (e.g., light and heavy isotopes). It takes the output from ProteomIQon's protein inference and peptide-based quantification steps and systematically combines peptide information. This process involves calculating ratios between labeled peptide intensities, with optional steps for aggregating by charge state and modifications, before finally aggregating all peptides belonging to a protein. The output is a tab-separated file detailing the quantified proteins.

## Usage

The `proteomiqon-labeledproteinquantification` tool is executed via the command line. It requires input files from previous ProteomIQon steps and allows for configuration through parameters.

### Basic Command Structure

The fundamental command to run the tool is:

```bash
proteomiqon - labeledproteinquantification -i <input_quant_file> -o <output_directory> -p <parameters_json_file>
```

### Key Parameters and Options

*   **`-i <input_quant_file>`**: Specifies the input file containing quantified peptide data. This file is typically generated by `PSMBasedQuantification` or `LabelFreeProteinQuantification`.
*   **`-o <output_directory>`**: Designates the directory where the output files will be saved.
*   **`-p <parameters_json_file>`**: (Optional) Path to a JSON file containing custom parameters for the quantification process. If not provided, default parameters will be used.

### Advanced Usage and Tips

*   **Processing Multiple Files**: The tool can process multiple input files simultaneously. Provide a space-separated list of files for the `-i` argument. The tool will match files based on their order in the list.

    ```bash
    proteomiqon -labeledproteinquantification -i run1.quant run2.quant run3.quant -o ./output -p params.json
    ```

*   **Aggregation Steps**: The tool performs several aggregation steps:
    1.  **Labeled Peptide Aggregation**: Aggregates peptides with the same sequence, modifications, and charge state, calculating the ratio between light and heavy intensities.
    2.  **Charge State Aggregation** (Optional): Aggregates peptides with the same sequence and modifications but different charge states.
    3.  **Modification Aggregation** (Optional): Aggregates peptides with the same sequence but different modifications.
    4.  **Protein Aggregation**: Aggregates all peptides belonging to a specific protein.

    The order and inclusion of optional steps are controlled by the parameters specified in the JSON configuration file.

*   **Parameter File Generation**: For detailed control over aggregation steps, charge state handling, and modification filtering, it is highly recommended to use a parameter file. Default parameter files can be downloaded from the ProteomIQon documentation, or custom parameters can be defined using F# scripts.

*   **Output Files**: The tool generates tab-separated files at each aggregation step, providing intermediate results and the final protein quantification.
