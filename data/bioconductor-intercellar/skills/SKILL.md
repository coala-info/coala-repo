# bioconductor-intercellar

## Overview
`InterCellar` is a Bioconductor package designed to analyze and visualize cell-cell communication (CCC) predicted from scRNA-seq data. It acts as a downstream analysis wrapper for results generated by tools such as CellPhoneDB, CellChat, ICELLNET, and SingleCellSignalR. The package organizes data into three "Universes" (Cluster, Gene, and Function) and provides data-driven modules to identify functional patterns in cellular interactions across different biological conditions.

## Installation and Setup
To install and load the package:

```r
if (!requireNamespace("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

BiocManager::install("InterCellar")
library(InterCellar)
```

## Launching the Interactive App
The primary interface for `InterCellar` is a Shiny application. It is recommended to run it in reproducible mode to ensure results can be saved and reloaded accurately.

```r
# Launch the interactive Shiny application
InterCellar::run_app(reproducible = TRUE)
```

## Workflow and Data Handling

### 1. Data Upload and Pre-processing
InterCellar requires pre-computed results from CCC inference tools.
- **Supported Tools:** CellPhoneDB (v2/v3), CellChat, ICELLNET, and SingleCellSignalR.
- **Custom Data:** Can be uploaded if it contains essential columns (Ligand, Receptor, Clusters, Scores).
- **Pre-processing:** The package automatically maps interaction pairs to genes, annotates molecular functions (L or R), and standardizes the directionality to Ligand-Receptor (L-R).

### 2. Exploration Universes
- **Cluster-verse:** Focuses on cell-type networks. Use this to filter by interaction score or p-value and visualize paracrine vs. autocrine signaling via network graphs and barplots.
- **Gene-verse:** Focuses on specific interaction pairs. Provides dot plots to visualize the expression of specific ligand-receptor pairs across clusters.
- **Function-verse:** Annotates interactions using Gene Ontology (via biomaRt) or pathway databases (via graphite). Use the Sunburst plot to connect specific functions to cluster pairs.

### 3. Int-Pair Modules
This analysis identifies groups of interaction pairs that share similar functional profiles.
- **Viewpoint Selection:** Choose a specific cluster and communication flow (e.g., outgoing L-R).
- **Clustering:** Uses UMAP and hierarchical clustering to define modules.
- **Significance:** Performs permutation tests to find functional terms significantly enriched in specific modules.

### 4. Multiple Conditions Comparison
Compare up to three conditions (e.g., Control vs. Disease):
- **Cluster-based:** Radar plots and back-to-back barplots compare total interaction counts.
- **Gene-based:** Identifies "couplets" (int-pair/cluster-pair) unique to specific conditions.
- **Function-based:** Identifies functional terms enriched by interactions unique to one condition.

## Tips for Effective Use
- **Output Folders:** Always specify a local output folder during the upload step; InterCellar saves all generated figures and tables there.
- **Viewpoints:** When analyzing complex datasets, use the "Viewpoint" feature in the Cluster-verse to simplify the network to interactions involving a single cell type of interest.
- **Flow Direction:** Distinguish between "Outgoing" (Viewpoint expresses Ligand) and "Incoming" (Viewpoint expresses Receptor) to understand the directionality of signaling.
