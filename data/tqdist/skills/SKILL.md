# tqdist

## Overview
The `tqdist` tool provides a high-performance implementation for measuring the topological distance between phylogenetic trees. It calculates the Triplet Distance for rooted trees (the number of three-leaf subsets that induce different topologies) and the Quartet Distance for unrooted trees (the number of four-leaf subsets that induce different topologies). These metrics are essential for quantifying the similarity or conflict between different evolutionary hypotheses or trees generated by different methods.

## Usage Guidelines

### Installation
The tool is available via Bioconda. Ensure your environment is configured with the bioconda channel:
```bash
conda install -c bioconda tqdist
```

### Core Commands
The package typically provides several binaries for different distance calculations:

- **triplet_dist**: Calculates the triplet distance between rooted trees.
- **quartet_dist**: Calculates the quartet distance between unrooted trees.
- **pairs_triplet_dist / pairs_quartet_dist**: Used for computing distances between pairs of trees in two separate files.
- **all_pairs_triplet_dist / all_pairs_quartet_dist**: Used for computing all-against-all distances within a single file or between two files.

### Common CLI Patterns
When using these tools, trees should be provided in Newick format.

1. **Distance between two single trees:**
   ```bash
   triplet_dist <tree1.newick> <tree2.newick>
   quartet_dist <tree1.newick> <tree2.newick>
   ```

2. **Comparing a list of trees (All-pairs):**
   If you have a file containing multiple Newick strings, you can compute a distance matrix:
   ```bash
   all_pairs_quartet_dist <trees.newick>
   ```

3. **Comparing two sets of trees (Pairs):**
   To compare trees in `file1.newick` to corresponding trees in `file2.newick` line-by-line:
   ```bash
   pairs_triplet_dist <file1.newick> <file2.newick>
   ```

### Expert Tips
- **Rooting Matters**: Ensure trees are consistently rooted when using `triplet_dist`. If trees are unrooted, `quartet_dist` is the mathematically appropriate metric.
- **Leaf Labels**: The trees must have identical sets of leaf labels. If labels do not match, the tool may produce an error or unexpected results depending on the specific implementation version.
- **Scaling**: Because `tqdist` uses an O(n log n) algorithm for triplets, it can handle trees with tens of thousands of leaves much faster than naive O(n^3) or O(n^4) implementations.
