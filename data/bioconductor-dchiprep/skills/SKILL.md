# bioconductor-dchiprep

name: bioconductor-dchiprep
description: Analysis of chromatin modification ChIP-Seq data with replication. Use this skill to perform differential analysis of histone modification profiles at base-pair resolution, including data import from count tables or BAM files, position-wise testing using DESeq2-based methodologies, and visualization of genomic profiles.

## Overview

DChIPRep implements a methodology to assess differences between chromatin modification profiles in replicated ChIP-Seq studies. It treats each position as a feature and uses a DESeq2-based framework to identify significant differences in enrichment between conditions. The package is specifically designed for histone modifications where profiles might change in shape or magnitude across a genomic region (e.g., around Transcription Start Sites).

## Typical Workflow

1. **Data Preparation**: Organize ChIP and Input (Control) data. DChIPRep requires a sample table describing the experimental design.
2. **Data Import**: Load data into a `DChIPRepResults` object using `importData` (from files), `importDataFromMatrices` (from R matrices), or `importData_soGGi` (directly from BAM files).
3. **Statistical Testing**: Run `runTesting` to perform position-wise differential analysis and FDR estimation.
4. **Visualization**: Use `plotProfiles` to see average enrichment and `plotSignificance` to highlight regions of statistical difference.

## Core Functions

### Data Import

The package supports three main ways to bring data into the DChIPRep environment:

- `importData(sampleTable, directory)`: Imports data from count tables generated by the package's associated Python scripts. The `sampleTable` must contain columns: `ChIP`, `Input`, `sampleID`, `upstream`, `downstream`, and `condition`.
- `importDataFromMatrices(inputData, chipData, sampleTable)`: Useful if you already have count matrices in R. `inputData` and `chipData` should have positions in columns and features in rows.
- `importData_soGGi(bam_paths, TSS, fragment_lengths, ...)`: Directly imports from BAM files using a `GRanges` object of interest (e.g., TSS locations).

### Analysis and Results

- `DChIPRepResults(object)`: Constructor to initialize the results object from a `DESeqDataSet`.
- `runTesting(object, lfcThreshold = 0.05)`: The main execution function. It performs the position-wise tests and calculates local FDR.
- `resultsDChIPRep(object)`: Extracts the data frame containing test results (log2 fold changes, p-values, etc.).
- `FDRresults(object)`: Accesses the FDR estimation details.

### Plotting

- `plotProfiles(object)`: Plots the position-wise mean of the log2 normalized counts for both conditions.
- `plotSignificance(object, lfdrThresh = 0.2)`: Similar to `plotProfiles`, but highlights positions that pass the local FDR threshold in black.

## Example Usage

```R
library(DChIPRep)

# 1. Load example data
data(exampleSampleTable)
data(exampleInputData)
data(exampleChipData)

# 2. Import into DChIPRep object
dcr <- importDataFromMatrices(inputData = exampleInputData,
                              chipData = exampleChipData,
                              sampleTable = exampleSampleTable)

# 3. Run differential testing
# This adds testing results and FDR calculations to the object
dcr <- runTesting(dcr)

# 4. Visualize results
plotProfiles(dcr)
plotSignificance(dcr, lfdrThresh = 0.1)

# 5. Extract result table
res_df <- resultsDChIPRep(dcr)
head(res_df)
```

## Tips and Best Practices

- **Normalization**: DChIPRep computes normalization factors from the Input (Control) counts. If your input counts are zero for certain positions, the package internally sets them to 1 to ensure valid normalization factors.
- **Robust Means**: By default, plotting functions use `robust_mean` (a Huber estimator) to summarize profiles across replicates, which helps mitigate the influence of outliers.
- **Local FDR**: The package uses `fdrtool` to estimate local False Discovery Rates. The default threshold in `plotSignificance` is 0.2, but this should be adjusted based on the stringency required for your biological analysis.
