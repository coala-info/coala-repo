# mpralib

## Overview
mpralib is a Python-based library and command-line toolset developed by the Kircher Lab for the systematic analysis of MPRA experiments. It serves as a critical quality control and processing layer, allowing researchers to validate their data structures, map barcodes to elements, and handle the complex count data generated by high-throughput reporter assays.

## Installation
The tool can be installed via PyPI or Bioconda:
- `pip install mpralib`
- `conda install -c bioconda mpralib`

## Command Line Interface (CLI)
The primary CLI utility is used for data validation. Ensuring your files match the expected schema is essential before running analysis pipelines.

### Validating Data Files
Use the `validate-file` command to check your input files against specific MPRA schemas.

**Syntax:**
`mpralib validate-file <schema> --input <input_file>`

**Supported Schemas:**
- `reporter-sequence-design`: For experimental design files.
- `reporter-barcode-to-element-mapping`: For linking barcodes to sequences.
- `reporter-experiment-barcode`: For barcode-level experimental data.
- `reporter-experiment`: For general experiment metadata/counts.
- `reporter-element`: For element-level data.
- `reporter-variant`: For variant-specific reporter data.
- `reporter-genomic-element`: For genomic coordinates of elements.
- `reporter-genomic-variant`: For genomic coordinates of variants.

**Example:**
`mpralib validate-file reporter-sequence-design --input data/design.tsv.gz`

## Best Practices and Expert Tips
- **Compression Support**: Always use gzipped files (e.g., `.tsv.gz` or `.bed.gz`) for input. mpralib natively handles compressed formats, which is standard for large-scale genomics data.
- **Early Validation**: Run the `validate-file` command immediately after generating your design or mapping files. Catching formatting errors early prevents failures in downstream analysis pipelines.
- **Schema Selection**: Choose the schema that precisely matches your assay type. For example, use `reporter-variant` specifically for saturation mutagenesis or SNP-based assays to ensure variant-level metadata is correctly structured.
- **Library Usage**: While the CLI is excellent for validation, use the Python API for custom filtering and plotting workflows. The library is designed to be imported into Jupyter notebooks for interactive data exploration.
