# longreadsum

## Overview

LongReadSum is a high-performance QC utility designed specifically for the unique characteristics of long-read sequencing. Unlike general-purpose QC tools, it handles raw signal formats (FAST5/POD5) and specialized metadata like base modification tags and ONT "Read Until" (RRMS) decisions. It produces interactive HTML reports and JSON summaries compatible with MultiQC, making it an essential step in long-read bioinformatics pipelines for both DNA and RNA applications.

## Installation and Setup

The recommended way to run LongReadSum is via Conda or Docker to ensure all dependencies (like htslib and VBZ compression plugins) are available.

```bash
# Conda installation
conda install -c wglab -c conda-forge -c bioconda longreadsum

# Docker usage (mapping local data to /mnt)
docker run -v /path/to/data:/mnt -it genomicslab/longreadsum bam -i /mnt/input.bam -o /mnt/output
```

## Common CLI Patterns

The tool uses a subcommand structure based on the input file type: `longreadsum <FILETYPE> [options]`.

### General Parameters
* `-i`: Single input file path.
* `-I`: Multiple comma-separated input paths.
* `-P`: Pattern matching (e.g., `"*.fastq"`). **Note:** Always enclose patterns in double quotes.
* `-t`: Number of threads (default is 1).
* `-o`: Output directory (default is `output_longreadsum`).
* `-Q`: Prefix for output files (default is `QC_`).

### Sequence and Alignment QC (BAM/FASTQ/FASTA)
For standard WGS or targeted sequencing:
```bash
# BAM alignment QC
longreadsum bam -i sample.bam -o bam_qc -t 8

# FASTQ/FASTA sequence QC
longreadsum fastq -i reads.fastq.gz -o fastq_qc
```

### Base Modification Analysis
To analyze 5mC or other modifications stored in BAM MM/ML tags:
```bash
longreadsum bam -i sample.bam -o mod_qc --mod --modprob 0.8 --ref reference.fasta
```
* `--mod`: Enables modification analysis.
* `--modprob`: Filtering threshold for modified bases (default 0.8).
* `--ref`: Providing the reference FASTA allows for CpG site identification.

### RNA-Seq Transcript Integrity (TIN)
To calculate Transcript Integrity Numbers (TIN) for long-read RNA-seq:
```bash
longreadsum bam -i rna_sample.bam -o tin_qc --genebed annotations.bed12
```
* `--genebed`: Required BED12 file containing gene/transcript coordinates.

### Signal-Level QC (ONT POD5/FAST5)
For raw data assessment before or during basecalling:
```bash
# Modern ONT POD5 format
longreadsum pod5 -i /path/to/pod5_dir/ -o signal_qc

# Legacy FAST5 format
longreadsum fast5 -i /path/to/fast5_dir/ -o signal_qc
```

### ONT Sequencing Summary
To quickly process the `sequencing_summary.txt` file generated by MinKNOW/Guppy/Dorado:
```bash
longreadsum seqtxt -i sequencing_summary.txt -o summary_qc
```

## Expert Tips

1. **MultiQC Integration**: LongReadSum generates a JSON summary file. To aggregate these into a MultiQC report, run:
   `multiqc . --module longreadsum`
2. **RRMS/Read Until**: If using ONT's adaptive sampling, use the `rrms` module with the associated CSV to visualize accepted vs. rejected (unblocked) reads:
   `longreadsum rrms -i sample.bam -c adaptive_sampling_decisions.csv`
3. **Thread Scaling**: While many modules are fast, the `bam` module (especially with `--mod` or `--genebed`) benefits significantly from increasing `-t`.
4. **Pattern Matching**: When using `-P`, if your shell expands the wildcard before the tool receives it, the command may fail. Always use quotes: `-P "data/*.fast5"`.
