# ucsc-fatrans

## Overview
The `ucsc-fatrans` skill provides guidance on using the `faTrans` utility from the UCSC Genome Browser "kent" source tree. This tool is designed for the rapid translation of DNA sequences stored in FASTA format into their corresponding amino acid sequences. It is particularly useful in genomic pipelines where you need to generate protein products from transcript sequences or open reading frames (ORFs).

## Usage Patterns

### Basic Translation
The primary usage of the tool requires an input DNA FASTA file and a destination path for the output peptide FASTA file.

```bash
faTrans input.fa output.fa
```

### Common CLI Options
While the specific help text is generated by running the binary without arguments, the following patterns are standard for this utility:

*   **Translate in a specific reading frame**: If the sequence is not frame-aligned, you may need to adjust the starting point.
*   **Handling Stop Codons**: By default, the tool translates the sequence based on the standard genetic code.
*   **Six-frame translation**: Often used to find potential ORFs in unannotated genomic DNA.

### Implementation Workflow
1.  **Verify Input**: Ensure the input file is a valid FASTA formatted DNA sequence.
2.  **Execution**: Run `faTrans` via the command line.
3.  **Permission Handling**: If using a direct binary download from UCSC (rather than a conda-installed version), ensure the binary is executable:
    ```bash
    chmod +x faTrans
    ./faTrans input.fa output.fa
    ```

## Expert Tips
*   **Standard Genetic Code**: `faTrans` typically uses the standard genetic code. If you are working with mitochondrial DNA or specific organisms with alternative codes, verify if the tool supports translation table overrides (though standard `faTrans` is usually fixed to the universal code).
*   **Pipe Integration**: Like many UCSC tools, `faTrans` can often be integrated into bash one-liners, though it primarily expects file paths for input and output.
*   **Memory Efficiency**: This tool is highly optimized for speed and can handle very large FASTA files (entire transcriptomes) with minimal memory overhead compared to heavy-weight Bio-language libraries.
