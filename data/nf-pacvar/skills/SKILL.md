# pacvar

## Overview
nf-core/pacvar is designed to handle the specific requirements of PacBio long-read data, offering specialized workflows for both whole-genome sequencing (WGS) and PureTarget expansion panels. It addresses the biological challenge of accurately identifying small variants, structural variants, and complex tandem repeats by leveraging PacBio-specific tools like pbmm2 and TRGT.

In practice, the pipeline transforms raw or unaligned reads into phased variant calls and detailed repeat motif plots. Users can choose between different SNP calling routes and toggle specific analysis stages such as demultiplexing or phasing depending on their experimental setup and data type.

## Data preparation
The pipeline requires a comma-separated samplesheet specified with the `--input` parameter. This file must contain a header and define the sample ID and the path to the unaligned BAM or CRAM file.

*   **Samplesheet columns:**
    *   `sample`: Unique sample identifier (no spaces).
    *   `bam`: Path to unaligned BAM or CRAM file.
    *   `pbi`: Optional path to the PacBio index file (.pbi, .bai, or .crai).

Example `samplesheet.csv`:
```csv
sample,bam,pbi
CONTROL,AEG588A1_S1_L002_R1_001.bam,AEG588A1_S1_L002_R1_001.pbi
```

Additional required files depending on the workflow:
*   **Reference Genome:** Specified via `--genome` (e.g., `GRCh38`) or `--fasta`.
*   **Barcodes:** A FASTA file provided via `--barcodes` if demultiplexing is required.
*   **Intervals:** A BED file provided via `--intervals` for targeted variant calling or repeat characterization.

## How to run
The pipeline is executed using `nextflow run`. You must specify the workflow type using `--workflow`, which defaults to `wgs`.

```bash
nextflow run nf-core/pacvar \
   -profile <docker/singularity/institute> \
   --input samplesheet.csv \
   --workflow wgs \
   --genome GRCh38 \
   --outdir ./results
```

Key execution parameters:
*   `--workflow`: Set to `wgs` for whole-genome analysis or `repeat` for tandem repeat characterization.
*   `--snv_caller`: Choose between `deepvariant` (default) or `gatk4`.
*   `--skip_demultiplexing`: Use if reads are already demultiplexed.
*   `--skip_snp`, `--skip_sv`, `--skip_phase`: Flags to bypass specific WGS analysis stages.
*   `-resume`: Use to restart a pipeline from the last successful process.

## Outputs
Results are saved in the directory specified by `--outdir`. The primary deliverables depend on the selected workflow:

*   **WGS Workflow:** Phased VCF files containing SNPs and structural variants (SVs), along with phased BAM alignments.
*   **Repeat Workflow:** Genotyped tandem repeat VCFs, spanning BAM files, and repeat motif plots generated by TRGT.
*   **Reports:** A MultiQC report aggregating statistics from alignment and processing tools.

For a comprehensive list of output files and directory structures, refer to the official [output documentation](https://nf-co.re/pacvar/output).
