# unmicst

## Overview
UnMicst (Universal Models for Identifying Cells and Segmenting Tissue) is a deep learning tool that addresses the challenge of nuclear segmentation in complex tissue morphologies. By employing a UNet architecture trained on diverse tissue types (including lung, prostate, and small intestine), it classifies pixels into three categories: nuclei foreground, nuclei contours, and background. The resulting probability maps are significantly more robust than traditional thresholding methods, especially when dealing with real-world artifacts like defocused planes or saturated pixels.

## Command Line Usage

The primary way to interact with UnMicst is through the `unmicstWrapper.py` script or the core `UnMicst.py` script.

### Basic Execution
To run UnMicst on a single image using the DNA channel:
```bash
python unmicstWrapper.py /path/to/image.ome.tif --tool unmicst-solo --channel 1 --outputPath /path/to/results
```

### Tool Variants
Select the appropriate model based on your available markers:
- `unmicst-solo`: (Default) Uses only the DNA channel (e.g., Hoechst or DAPI).
- `unmicst-duo`: Uses both the DNA channel and a nuclear envelope stain (e.g., Lamin B or Nucleoporin 98) for higher accuracy in crowded regions.
- `unmicst-legacy`: Deprecated mouse-specific model.

### Key Parameters
- `--channel`: Specify the 1-based index of the channel(s) to be used. For `unmicst-duo`, provide two indices.
- `--scalingFactor`: Adjusts for pixel size differences. The model is trained on **0.65 microns/pixel**. If your image is 0.325 microns/pixel, use `--scalingFactor 0.5`.
- `--mean` and `--std`: Use these if your image intensity distribution is significantly different from standard datasets to normalize the input.

## Docker Integration
For consistent environments, use the official Docker container.

```bash
# Pull the latest image
docker pull labsyspharm/unmicst:latest

# Run with GPU support (recommended)
docker run -it --runtime=nvidia -v /path/to/data:/data labsyspharm/unmicst:latest \
python /app/UnMicst.py /data/input.tif --outputPath /data/results
```

## Best Practices and Expert Tips

### Input Requirements
- **Format**: Prefers `.ome.tif` or `.tif`.
- **Pre-processing**: Images should ideally be flat-field corrected and in focus. While the model handles some defocus, extreme blur will degrade performance.
- **Resolution**: If your pixel size deviates significantly from 0.65 µm/px, always use the `--scalingFactor`.

### Interpreting Outputs
UnMicst produces two main files:
1. `[filename]_Probabilities.tif`: A 3-channel tiff stack.
   - Channel 1: Nuclei Foreground
   - Channel 2: Nuclei Contours
   - Channel 3: Background
2. `[filename]_Preview.tif`: A QC image concatenating the original DNA channel with the predicted nuclei contours. Use this to quickly verify if the segmentation is tracking correctly.

### Downstream Workflow
The probability maps generated by UnMicst are intended to be fed into binarization tools. The developers specifically recommend **s3segmenter** for converting these probabilities into final instance segmentation masks.
