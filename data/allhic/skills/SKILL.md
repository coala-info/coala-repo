# ALLHiC Scaffolding

ALLHiC is specialized for auto-polyploid or heterozygous diploid genomes. It uses a Genetic Algorithm (GA) to optimize the ordering and orientation of contigs.

## Standard Pipeline Workflow

The typical workflow follows a linear sequence of commands. For polyploid genomes, the `prune` step is critical.

### 1. Preprocessing (Extract)
Extracts inter-contig links, intra-contig links, and restriction sites.
```bash
allhic extract sample.bam draft_assembly.fasta
```
*   **Outputs**: `.clm` file (compact links), `.counts_GATC.txt`, and `.pairs.txt`.

### 2. Pruning (Polyploid/Heterozygous Only)
Removes allelic or cross-allelic links to prevent mis-clustering of homologous chromosomes.
```bash
allhic prune Allele.ctg.table sample.pairs.txt
```
*   **Note**: Requires an `Allele.ctg.table` defining which contigs are allelic.
*   **Output**: `sample.pairs.prune.txt`.

### 3. Partitioning
Clusters contigs into $k$ groups (usually the number of haploid chromosomes).
```bash
# Use pruned pairs for polyploids
allhic partition sample.counts_GATC.txt sample.pairs.prune.txt --k 8
```
*   **Parameter**: `--k` is the target number of partitions.

### 4. Optimization
Uses a Genetic Algorithm to find the best order and orientation for each partition.
```bash
allhic optimize sample.counts_GATC.group1.txt sample.clm
```
*   **Note**: This must be run for each group generated by the partition step.
*   **Output**: `.tour` files containing the ordering information.

### 5. Final Assembly (Build)
Generates the final FASTA and AGP files.
```bash
allhic build sample.counts_GATC.group?.tour draft_assembly.fasta final_assembly.fasta
```

## Visualization
To validate the assembly, generate a Hi-C contact heatmap:
```bash
allhic plot sample.bam sample.counts_GATC.group1.tour
```

## Expert Tips & Best Practices

- **Pruning Importance**: In polyploids, skipping `allhic prune` often results in "collapsed" assemblies where homologous chromosomes are clustered together rather than separated into subgenomes.
- **Restriction Sites**: Ensure the restriction site used in `extract` matches your Hi-C library prep (default is GATC for MboI/DpnII).
- **GA Convergence**: The `optimize` step is stochastic. For large chromosomes, you may need to run it multiple times or increase the number of generations if the tour score hasn't plateaued.
- **Input Quality**: Contigs should be pre-filtered for contaminants and ideally "purged" of primary-haplotig overlaps before running ALLHiC for best results.
