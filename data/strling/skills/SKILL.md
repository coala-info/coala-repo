# strling

description: |
  Detects large Short Tandem Repeat (STR) expansions from short-read sequencing data.
  Use when analyzing genomic data to identify novel or reference-annotated STR expansions,
  particularly when dealing with complex repeat regions that may be mis-mapped by standard aligners.
  This tool is useful for research in genetic disorders, population genetics, and evolutionary biology.
```
## Overview
STRling is a bioinformatics tool designed to identify significant expansions in Short Tandem Repeats (STRs) within short-read sequencing data. It excels at detecting both novel STR expansions (where no STR is present in the reference genome at a given locus) and expansions at known STR loci. STRling employs k-mer counting to recover reads that might be mis-mapped due to repetitive sequences and then uses soft-clipped reads to pinpoint the exact location and extent of these expansions.

## Usage Instructions

STRling is a command-line tool. The primary command involves specifying input files and output options.

### Core Command Structure

The general structure for running STRling is:

```bash
strling [options] <input_bam_or_cram> <reference_fasta> <output_prefix>
```

### Key Options and Best Practices

*   **Input Files**:
    *   `input_bam_or_cram`: Path to your aligned sequencing data in BAM or CRAM format. Ensure this file is indexed.
    *   `reference_fasta`: Path to the reference genome FASTA file used for alignment. This file must also be indexed (e.g., with `samtools faidx`).

*   **Output**:
    *   `output_prefix`: A prefix for all output files generated by STRling. STRling will create several files, including `.strling.tsv` (main results), `.strling.log`, and potentially others depending on options.

*   **Important Flags**:
    *   `--threads <int>`: Specify the number of threads to use for parallel processing. This can significantly speed up analysis on multi-core systems.
    *   `--min_len <int>`: Minimum length of an STR repeat unit to consider. Default is 1.
    *   `--max_len <int>`: Maximum length of an STR repeat unit to consider. Default is 100.
    *   `--min_expansion <int>`: Minimum number of repeat units for an expansion to be called. This is a crucial parameter for defining what constitutes a significant expansion.
    *   `--min_support <int>`: Minimum number of reads supporting an expansion call. Higher values increase confidence but may miss rarer events.
    *   `--novel_only`: If set, only report novel STR expansions and ignore those present in the reference.
    *   `--reference_only`: If set, only report STR expansions that are annotated in the reference genome.
    *   `--output_dir <path>`: Specify a directory to save all output files.

### Example Usage

**Basic analysis with default parameters:**

```bash
strling --threads 8 aligned_reads.bam reference.fasta output_results
```

**Analysis focusing on significant expansions and novel loci:**

```bash
strling --threads 16 --min_expansion 10 --novel_only aligned_reads.bam reference.fasta novel_expansions
```

**Specifying output directory and custom repeat unit length limits:**

```bash
strling --threads 4 --min_len 2 --max_len 20 --output_dir ./strling_output data/sample.cram genome/hg38.fasta sample_analysis
```

### Expert Tips

*   **Indexing**: Always ensure your input BAM/CRAM and reference FASTA files are properly indexed. For BAM/CRAM, use `samtools index`. For FASTA, use `samtools faidx`.
*   **Parameter Tuning**: The `--min_expansion` and `--min_support` parameters are critical for controlling sensitivity and specificity. Experiment with these values based on your research question and the expected characteristics of your data.
*   **Reference Genome**: Use a high-quality, appropriately indexed reference genome that matches the origin of your sequencing data.
*   **Log Files**: Always check the `.strling.log` file for any warnings or errors during execution.
*   **Output Interpretation**: The primary output file (`.strling.tsv`) contains detailed information about detected STR expansions, including locus, repeat unit, expansion size, and supporting read counts. Refer to the STRling documentation for a full explanation of each column.
