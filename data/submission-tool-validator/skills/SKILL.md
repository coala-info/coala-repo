# submission-tool-validator

## Overview

The submission-tool-validator is a Java-based command-line utility designed for client-side validation of proteomics data. It allows researchers to verify the integrity and format of their files—specifically mzIdentML, mzTab, and ProteomeXchange (PX) XML files—prior to uploading them to the PRIDE repository. By catching formatting errors or missing metadata locally, it streamlines the submission process and reduces the likelihood of rejection by the repository.

## Installation and Requirements

- **Environment**: Requires Java 1.8 (64-bit) or higher.
- **Resources**: At least 2 GB of RAM is recommended for complex mzIdentML files.
- **Conda**: `conda install bioconda::submission-tool-validator`
- **Manual**: Download the zip archive from the GitHub releases, extract it, and run the JAR file directly.

## Command Line Usage

The tool operates primarily through the `-v` (validate) flag followed by the specific file type flag.

### 1. Validating Submission Metadata (PX)
To validate the `submission.px` file which contains the project metadata (title, description, etc.):
```bash
java -jar submission-tool-validator.jar -v -px /path/to/data/submission.px
```

### 2. Validating mzTab Files
To check the syntax and consistency of an mzTab file:
```bash
java -jar submission-tool-validator.jar -v -mztab /path/to/data/file.mztab
```

### 3. Validating mzIdentML Files
To validate peptide and protein identification results:
```bash
java -jar submission-tool-validator.jar -v -mzid /path/to/data/file.mzid
```

### 4. Validating mzIdentML with Associated Spectra
For a more comprehensive check, you can validate the identification file against the peak list (spectra) files used.

**Single peak file:**
```bash
java -jar submission-tool-validator.jar -v -mzid /path/to/data/file.mzid -peak /path/to/data/file.mgf
```

**Multiple peak files:**
Use the `##` delimiter to separate multiple peak file paths.
```bash
java -jar submission-tool-validator.jar -v -mzid /path/to/data/file.mzid -peaks /path/to/data/file1.mgf##/path/to/data/file2.mgf
```

## Best Practices

- **Pre-validation**: Always run the validator on the `submission.px` file generated by the PRIDE submission tool to ensure all mandatory fields are populated.
- **Memory Management**: If the tool crashes on large mzIdentML files, increase the Java heap space using the `-Xmx` flag (e.g., `java -Xmx4G -jar ...`).
- **Path Handling**: Ensure absolute paths are used if the tool is being called from a different working directory than where the data resides.
- **Version Check**: Ensure you are using the latest version (currently 1.0.7) to support the most recent mzIdentML (1.2) and mzTab schemas.
