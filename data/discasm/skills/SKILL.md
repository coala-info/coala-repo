# discasm

## Overview

DISCASM (Discordant and Unmapped Read De novo Transcriptome Assembly) is a specialized utility within the Trinity Cancer Transcriptome Analysis Toolkit (CTAT). It bridges the gap between alignment-based and assembly-based RNA-Seq analysis by focusing specifically on "problematic" reads—those that map discordantly (suggesting fusions) or fail to map to the reference genome at all. By isolating these reads and performing a targeted de novo assembly, DISCASM allows for the reconstruction of transcripts that represent complex genomic rearrangements or novel insertions that standard pipelines might overlook.

## Installation

The most efficient way to install DISCASM is via Bioconda:

```bash
conda install -c bioconda discasm
```

## Core Workflow and Usage

DISCASM typically follows an initial alignment step performed by the STAR aligner (often via STAR-Fusion).

### Required Inputs
1. **Chimeric.out.junction**: Generated by STAR, containing discordantly-mapped reads.
2. **Aligned.sortedByCoord.out.bam**: The standard alignment file (required if including unmapped reads).
3. **Original FASTQ files**: The raw reads used for the initial alignment.

### Standard Execution Pattern

To assemble both discordant and unmapped reads using Trinity:

```bash
DISCASM --chimeric_junctions Chimeric.out.junction \
        --aligned_bam Aligned.sortedByCoord.out.bam \
        --left_fq reads_1.fq.gz --right_fq reads_2.fq.gz \
        --denovo_assembler Trinity \
        --out_dir DISCASM_outdir
```

### Parameter Best Practices

- **--denovo_assembler**: Choose between `Trinity`, `Oases`, or `OasesMultiK`.
  - Use `Trinity` for high-quality, comprehensive assemblies.
  - Use `OasesMultiK` to run Oases across multiple k-mer values (19, 23, 27, 31, and 35) and merge the results, which can increase sensitivity for certain transcript types.
- **Targeted Assembly**: To focus strictly on discordant reads (ignoring unmapped reads), omit the `--aligned_bam` parameter. This significantly reduces computational overhead if you are only interested in fusion candidates identified by chimeric junctions.
- **Output Utilization**: The resulting assembled transcripts are frequently used as input for downstream fusion discovery tools like **GMAP-fusion**.

## Expert Tips

- **STAR Configuration**: If running STAR manually (not via STAR-Fusion), ensure you use the parameters required to generate the `Chimeric.out.junction` file. Refer to STAR-Fusion documentation for the specific chimeric detection flags.
- **Resource Management**: De novo assembly is memory-intensive. Ensure the environment has sufficient RAM allocated, especially when using Trinity as the assembler.
- **Clean Up**: DISCASM generates intermediate files during the extraction and assembly process. Use a dedicated `--out_dir` for every sample to prevent file collisions and facilitate easy cleanup.
