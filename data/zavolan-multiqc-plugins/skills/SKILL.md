# zavolan-multiqc-plugins

## Overview
The `zavolan-multiqc-plugins` package is a collection of extensions for the MultiQC tool, developed by the Zavolan Lab at the University of Basel. These plugins allow MultiQC to recognize, parse, and visualize specific bioinformatics outputs that are not supported by the core MultiQC installation. Specifically, it adds support for ALFA (genomic feature distribution), TIN-scores (RNA-seq quality metrics), and ZPCA (sample clustering via PCA).

## Installation
The most reliable way to install the plugins is via Conda through the Bioconda channel. This ensures all dependencies, including MultiQC itself, are correctly managed.

```bash
conda install -c bioconda zavolan-multiqc-plugins
```

## Usage Patterns
Once installed, these plugins integrate seamlessly into the standard MultiQC workflow. You do not need to call the plugins individually; MultiQC will automatically detect and use them if the relevant log files are found in the search path.

### Standard Execution
Run MultiQC in the directory containing your analysis results:

```bash
multiqc .
```

### Targeted Search
If you have a large project and only want to generate a report for specific data types supported by these plugins:

```bash
multiqc path/to/results/ --include alfa --include tin_scores --include zpca
```

## Supported Modules and Best Practices

### ALFA (Distribution of Genomic Features)
*   **Purpose**: Visualizes the distribution of reads across different genomic categories (e.g., exons, introns, intergenic regions).
*   **Requirement**: Ensure the output files from the ALFA tool are present in the directory being scanned.

### TIN-scores (Transcript Integrity Number)
*   **Purpose**: Provides a metric for RNA-seq quality, specifically measuring the uniformity of coverage across transcripts to assess degradation.
*   **Requirement**: Typically parses output generated by RSeQC's `tin.py` script. Ensure the `.tin.xls` or summary files are available.

### ZPCA (Principal Component Analysis)
*   **Purpose**: Generates PCA plots to visualize sample-to-sample distances and clustering.
*   **Requirement**: Looks for specific PCA coordinate files or variance explained files generated by Zavolan Lab's ZPCA processing scripts.

## Troubleshooting
*   **Plugin Detection**: If the modules do not appear in the report, run `multiqc --list-modules` to verify that `alfa`, `tin_scores`, and `zpca` are listed as available modules.
*   **Version Conflicts**: Ensure that your version of MultiQC is compatible with the plugins (v1.3 of the plugins is the stable release).
*   **File Naming**: MultiQC relies on search patterns. If files are renamed significantly from their default tool outputs, the plugins may fail to identify them. Use the `-v` (verbose) flag in MultiQC to see which files are being skipped.
