# WF5_Data_Interpretation_Worklow

## Overview

This workflow provides a rigorous statistical framework for interpreting quantitative proteomic datasets generated by MaxQuant. By leveraging the [MSstatsTMT](https://toolshed.g2.bx.psu.edu/repos/galaxyp/msstatstmt/msstatstmt/2.0.0+galaxy1) toolset, it enables researchers to perform sophisticated differential abundance analysis on complex mass spectrometry data, such as Tandem Mass Tag (TMT) experiments.

The pipeline processes several primary inputs, including MaxQuant protein groups, evidence files, and quantified peptides, alongside a required annotation file and comparison matrix. It integrates [Unipept](https://toolshed.g2.bx.psu.edu/repos/galaxyp/unipept/unipept/4.5.1) for taxonomic and functional indexing, while utilizing filtering steps to refine the data before statistical modeling and group comparisons.

Developed following [GTN](https://training.galaxyproject.org/) standards and tagged for clinical proteomics (clinicalmp), this workflow ensures a reproducible approach to data interpretation. It is licensed under [CC-BY-4.0](https://creativecommons.org/licenses/by/4.0/), offering a standardized solution for transforming raw quantitative outputs into meaningful biological insights.

Ensure all MaxQuant outputs and the Quantified Peptides are uploaded as tabular files, while the Annotation.txt and Comparison Matrix must strictly follow MSstats formatting requirements to avoid processing errors. While individual datasets are used here, organizing replicates into collections can streamline the initial data upload and mapping process. Refer to the included README.md for specific column headers and experimental design requirements necessary for the MSstatsTMT tool. You can automate the configuration of these inputs by using `planemo workflow_job_init` to generate a `job.yml` file.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run wf5-data-interpretation-worklow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run wf5-data-interpretation-worklow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run wf5-data-interpretation-worklow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init wf5-data-interpretation-worklow.ga -o job.yml`
- Lint: `planemo workflow_lint wf5-data-interpretation-worklow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
