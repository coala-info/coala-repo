# Command Line Usage

[Jump to bottom](#wiki-pages-box)

Michael Barnett edited this page Aug 12, 2025
·
[11 revisions](/ibbis-bio/common-mechanism/wiki/Command-Line-Usage/_history)

This guide provides instructions on how to use `commec` for DNA sequence screening. `commec` provides four subcommands: [screen](#screen), [flag](#flag), [split](#split) and [setup](#setup).

```
# Run screening on a FASTA file
commec screen -d /path/to/databases input.fasta

# Parse screen files and generate flag CSVs
commec flag /path/to/directory/with/screen/files

# Split a multi-record FASTA file into individual files, one for each record
commec split input.fasta

# Interactively download databases needed for screening
commec setup
```

## screen

The `screen` command runs a sequence through the following screening steps:

1. **Biorisk search**: Fast HMM-based search against curated sequence profiles
2. **Taxonomy Search**: look for best matches to regulated pathogens using a two-step process:
   * **Protein search**: BLASTX/DIAMOND search against NCBI nr
   * **Nucleotide search**: BLASTN search against NCBI nt (optional)
3. **Low-concern search**: Clear earlier flags based on matches to house-keeping or low-concern sequences

### Required Arguments

You must provide a database path either through the `-d` argument of via a configuration YAML file.

| Argument | Type | Description |
| --- | --- | --- |
| fasta\_file | Required | FASTA file to screen. Should only contain a single query |
| -d DATABASE\_DIR, --databases DATABASE\_DIR | Required (unless -y specified) | Path to directory containing reference databases (e.g. taxonomy, protein, HMM) |
| -y CONFIG\_YAML, --config CONFIG\_YAML | Required (unless -d specified) | Configuration for screen run in YAML format, including custom database paths |
| **Screen run logic options** |  |  |
| -p {blastx,diamond}, --protein-search-tool {blastx,diamond} | Optional | Tool for protein homology search to identify regulated pathogens (default: blastx) |
| --skip-tx | Optional flag | Skip protein and nucleotide search (regulated pathogens will only be identified based on biorisk search) |
| --skip-nt | Optional flag | Skip nucleotide search (regulated pathogens will only be identified based on protein hits) |
| **Parallelisation options** |  |  |
| -t THREADS, --threads THREADS | Optional | Maximum number of CPU threads to use. Passed to search tools. (default: 1) |
| -j DIAMOND\_JOBS, --diamond-jobs DIAMOND\_JOBS | Optional, diamond-only | Number of runs to do in parallel on split Diamond database (to increase speed, DIAMOND databases have been split into several parts) |
| **Output options** |  |  |
| -v, --verbose | Optional flag | Output verbose (i.e. DEBUG-level) logs |
| -o OUTPUT\_PREFIX, --output OUTPUT\_PREFIX | Optional | Prefix for output files. Can be a string (interpreted as output basename) or a directory (files will be output there, names determined from input FASTA). Defaults to location of input FASTA. |
| -c, --cleanup | Optional flag | Delete intermediate output files for run |
| -F, --force | Optional flag | Overwrite any pre-existing output for run (cannot be used with --resume) |
| -R, --resume | Optional flag | Re-use any pre-existing output for run (cannot be used with --force) |

### Examples

See Tutorial for information on [Using YAML to configure your commec runs](https://github.com/ibbis-bio/common-mechanism/wiki/Tutorial#using-yaml-to-configure-commec-screen) (recommended).

```
# minimal setup example (only commec biorisk databases ~1.2 GB)
commec screen -d ~/databases --skip-tx input.fasta

# Multi-threaded full analysis with DIAMOND
commec screen -d ~/databases -t 8 -p diamond input.fasta

# Custom output location, skip nucleotide search
commec screen -d ~/databases -o results/my-screen --skip-nt input.fasta

# All configuration can be done through YAML (recommended)
commec screen -y config.yaml input.fasta
```

## flag

The `flag` command produces a CSV. Each line in each CSV corresponds to query from commec generated .output.json files.

| Argument | Type | Description |
| --- | --- | --- |
| directory | Required | Directory containing .output.json files to summarize |
| -o OUTPUT, --output OUTPUT | Optional | Output directory name (defaults to directory name if not provided) |
| -r, --recursive | Optional flag | Search directory recursively for json files |

### Example

```
commec flag -r ~/commec-results/2025-04/ -o ./flags-2025-04-03
```

See Tutorial for information the [Understanding commec flag outputs](https://github.com/ibbis-bio/common-mechanism/wiki/Tutorial#understanding-commec-flag-outputs).

## setup

This is an interactive helper script for downloading the databases required for running `screen`

| Argument | Type | Description |
| --- | --- | --- |
| -a, --auto | Optional flag | Don't ask for user input, and use default options for everything |

### Example

```
commec setup
```

## split

This is a small helper script to split a multi-record FASTA file into individual files, one for each record.

| Argument | Type | Description |
| --- | --- | --- |
| fasta\_file | Required | FASTA file to split |

### Example

```
commec split multi_record.fasta
```
