# Scaffolding with Hi-C data VGP8

## Overview

This Galaxy workflow performs genome assembly scaffolding using Hi-C data, specifically utilizing the [YAHS](https://github.com/c-zhou/yahs) (Yet Another Hi-C Scaffolder) tool. While designed as the eighth stage (VGP8) of the Vertebrate Genomes Project (VGP) analysis trajectory, it is versatile enough to be used on any genome assembly provided in GFA format alongside corresponding Hi-C reads.

The pipeline automates several critical steps, including Hi-C read alignment with BWA-MEM2, data filtering, and scaffolding. It includes optional preprocessing for Arima Hi-C data, such as trimming five bases from the start of reads to reduce noise. The workflow is highly integrated with quality control tools, generating Pretext maps to visualize the contact matrix both before and after scaffolding, as well as Nx and size plots to assess assembly continuity.

Comprehensive biological validation and technical statistics are provided in the final outputs. The workflow runs [compleasm](https://github.com/huangnengCSU/compleasm) to evaluate assembly completeness and uses MultiQC to aggregate Hi-C alignment and duplication statistics. Final scaffolds are delivered in both FASTA and GFA formats, with headers updated to include specific haplotype information for clear downstream analysis.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Species Name | parameter_input | For Workflow report. |
| 1 | Assembly Name | parameter_input | For Workflow report. |
| 2 | Assembly GFA | data_input | The input GFA must conform to gfa1.2 standards, i.e. should have P lines defined, and contain sequences. Output GFAs from assemblers can be run through a GFA-GFA conversion using gfastats to ensure this. |
| 3 | Haplotype | parameter_input | Select the haplotype being scaffolded. This will impact the suffix used when naming the scaffolds. |
| 4 | Hi-C reads | data_collection_input | Arima or Dovetail Hi-C data |
| 5 | Trim Hi-C Data? | parameter_input | Trim 5 bases at the beginning of each read. Use with Arima Hi-C data if the Hi-C map looks "noisy".  (Select No if you are using the trimmed data generated by a previous workflow) |
| 6 | Minimum Mapping Quality | parameter_input | Filter low mapping quality for scaffolding. Set to 0 if no filtering is wanted. |
| 7 | Database for Busco Lineage | parameter_input | Select the database to use for Busco lineage. |
| 8 | Lineage | parameter_input | Taxonomic lineage for the organism being assembled for Busco analysis. |
| 9 | Restriction enzymes | parameter_input | Restriction enzymes used in preparation of Hi-C libraries. |
| 10 | Estimated genome size - Parameter File | data_input | Estimated genome size from contiging workflow. |


Ensure your genome assembly is in GFA format, using the gfastats tool for conversion if starting from FASTA. Hi-C reads must be provided as a paired dataset collection, while the estimated genome size should be supplied as a simple text file containing a single integer. For VGP-standard analysis, use the Arima Hi-C 2.0 restriction enzyme setting and the vertebrata lineage for Compleasm. Refer to the README.md for comprehensive details on parameter settings and expected output formats. You can use `planemo workflow_job_init` to generate a `job.yml` file for streamlining execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Scaffolding-HiC-VGP8.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Scaffolding-HiC-VGP8.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Scaffolding-HiC-VGP8.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Scaffolding-HiC-VGP8.ga -o job.yml`
- Lint: `planemo workflow_lint Scaffolding-HiC-VGP8.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
