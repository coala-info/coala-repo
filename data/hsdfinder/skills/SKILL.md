# hsdfinder

## Overview

HSDFinder is a bioinformatics tool designed to identify gene duplicates with high sequence similarity in eukaryotic organisms. By integrating sequence alignment data (BLAST) with functional domain annotations (InterProScan/Pfam), it allows for the categorization and visualization of duplicated genes. This skill provides the necessary procedural knowledge to prepare input files, execute the prediction pipeline, and troubleshoot common data formatting issues.

## Installation and Setup

The most reliable way to install HSDFinder is via Bioconda:

```bash
conda install bioconda::hsdfinder
```

Ensure you are using a 64-bit Linux environment with Python 3. For small datasets, a minimum of 2 cores and 4GB of RAM is required, though more resources are recommended for large eukaryotic genomes.

## Input Preparation

HSDFinder requires two primary input files. Proper formatting is critical for the tool to function.

### 1. BLAST All-Against-All Results
The input must be a protein BLAST result in tabular format (`-outfmt 6`).

**Recommended BLAST Command:**
```bash
makeblastdb -in reference.fa -dbtype prot -out db_name
blastp -query species.fasta -db db_name -out BLASTP_results.txt -evalue 1e-5 -outfmt 6
```

**Critical Warning:** Do NOT use the `-max_target_seqs` parameter in your BLAST command. This often causes HSDFinder to fail because it results in missing gene length information required for the HSD prediction algorithm.

### 2. InterProScan Results
The second input is the functional annotation file generated by InterProScan (TSV format). It must contain the standard 13 columns, including:
- Protein accession
- Sequence length
- Signature accession (e.g., Pfam)
- Start/Stop locations
- E-value

## Command Line Usage

Run the prediction script using the following syntax:

```bash
hsdfinder -i <blast_output> -p <identity_threshold> -l <length_threshold> -f <interpro_file> -o <output_file>
```

### Parameter Guide:
- `-i` / `--input_file`: The 12-column BLAST output file.
- `-p` / `--percentage_identity`: The minimum identity percentage to qualify as an HSD (e.g., 90).
- `-l` / `--length`: The minimum alignment length.
- `-f` / `--file`: The InterProScan/Pfam result file.
- `-o` / `--output_file`: Desired name for the results spreadsheet.

## Expert Tips and Best Practices

- **Manual Curation:** HSDFinder provides a primary selection of HSDs. Always perform manual curation on the output to filter out partial genes and pseudogenes, as these can inflate duplication counts.
- **File Consistency:** Ensure that the gene/protein IDs in your BLAST file exactly match the IDs in your InterProScan file. Discrepancies will lead to `KeyError` exceptions during execution.
- **Memory Management:** If processing large genomes with many duplicates, monitor RAM usage. If the process is killed, try splitting the input or increasing available swap space.
- **Visualization:** The tool generates results in a spreadsheet format. For cross-species comparisons, use the output `.tsv` files to generate heatmaps or line graphs using external R or Python libraries.
