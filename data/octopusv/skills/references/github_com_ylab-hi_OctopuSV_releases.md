## [0.3.0] - 2025-02

**New Features**

* Added full support for Illumina DRAGEN CNV calling and field mapping.
* Enabled multi-sample VCF output in `svcf2vcf` with correct sample column preservation.

**Fixes & Improvements**

* Fixed ID truncation for source callers using colons (e.g., Manta).
* Resolved data loss issues during multi-sample merging.
* Refactored VCF parsing logic for better handling of missing SVTYPE fields.
* Updated documentation on SVCF intermediate coordinate handling for insertions.

[v0.2.4 - Significantly improve sample merging function](/ylab-hi/OctopuSV/releases/tag/v0.2.4)

This maintenance release focuses on improving the core stability of the merge module and enhancing compatibility with upstream callers.

### 🛠 Improvements & Bug Fixes

* **Fixed Critical Merge Deadlock**: Resolved a severe issue where `octopusv merge --mode sample` would hang indefinitely due to circular import locks. The module loading logic has been refactored to ensure smooth multi-sample processing.
* **Enhanced SVIM Support**: Fixed a format recognition issue for **SVIM** outputs. The parser now correctly identifies and processes SVIM-formatted VCFs.
