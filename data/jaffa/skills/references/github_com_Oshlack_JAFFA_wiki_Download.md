# Reference files

You will also need one of these supporting reference files:

* [JAFFA\_REFERENCE\_FILES\_HG38\_GENCODE49.big.tar.gz](https://doi.org/10.6084/m9.figshare.31247149) - Files for hg38 (JAFFA 2.5+ assumes this reference by default)
* [JAFFA\_REFERENCE\_FILES\_HG38\_GENCODE22.V2.tar.gz](https://figshare.com/articles/online_resource/JAFFA_REFERENCE_FILES_HG38_GENCODE22_V2_tar_gz/13198538) - Files for hg38 (JAFFA 1.07-2.4 assumes this reference by default)

T2T, hg19, mm39 and mm10 are also supported, but require you to follow the instructions [here](https://github.com/Oshlack/JAFFA/wiki/FAQandTroubleshooting#how-can-i-run-jaffa-with-hg19-or-mm10) in addition to downloading the reference below

* [JAFFA\_REFERENCE\_FILES\_HS1\_CATLIFTOFFGENESV1.big.tar.gz](https://doi.org/10.6084/m9.figshare.31259524) - Files for human T2T (JAFFA 2.5+ only)
* [JAFFA\_REFERENCE\_FILES\_HG19\_GENCODE19.tar.gz](https://figshare.com/articles/JAFFA_REFERENCE_FILES_HG19_GENCODE19_tar_gz/9210467) - Files for human hg19 (JAFFA v2.4 and below)
* [JAFFA\_REFERENCE\_FILES\_MM39\_GENCODEVM38.big](https://doi.org/10.6084/m9.figshare.31259608) - Files for mouse mm39 (All JAFFA versions)
* [JAFFA\_REFERENCE\_FILES\_MM10\_GENCODEVM4.tar.gz](https://figshare.com/articles/Untitled_Item/9211226) - Files for mouse mm10 (JAFFA 2.4 and below only)

Please see [FAQandTroubleshooting](https://github.com/Oshlack/JAFFA/wiki/FAQandTroubleshooting#how-can-i-generate-the-reference-files-for-a-non-supported-genome) if you would like to generate your own reference files for JAFFA (e.g. for an unsupported genome or annotations).

**If you are finding the downloads above too big**, from JAFFA v2.5 onwards we provide a script within JAFFA, prepare\_reference.sh, which will automatically fetches a smaller minimum set of files and generates all other required reference files for hg39, T2T or mm39. The script should be run **after** running install\_linux64.sh, or equivalently setting up all the required software and specifying their path in tools.groovy. The script will take a few hours to be run, but only needs to be performed once.

# Versions prior to 1.06 that are not hosted on github

[JAFFA\_1.05.tar.gz](https://drive.google.com/open?id=0B1FwZagazjpccldFNVQwb3l2MGs) - Fusion gene names are now ordered by fusion gene orientation (previously gene names were listed alphabetically)

[JAFFA\_1.04.tar.gz](https://drive.google.com/open?id=0B1FwZagazjpcdzlIRi1yMmpQeGs)  - This version fixes an issue that caused the direct mode to be slow on some large dataset.

[JAFFA\_1.03.tar.gz](https://drive.google.com/open?id=0B1FwZagazjpcTnNWSnFNWENYWVU)  - Major improvements over 1.0 such as:

* Reads that map to intronic, intergenic, and chrM regions are now first filtered out before running the pipeline. This improves the RAM and time requirements for the assembly and hybrid modes.
* For the direct mode we now calculate spanning pairs
* fusions classed as PotentialRegularTranscripts are no longer reported
* we now threshold on 2 spanning reads for LowConfidence fusions (previously this was 1), to help remove library prep false positives.

[JAFFA\_1.0.tar.gz](https://drive.google.com/open?id=0B1FwZagazjpccE42aVpjSGQzSFk)  -This version is identical to 0.91. Improvements of 0.9 are that it works out the read length itself and includes a few bug fixes and checks. Thanks to Aliaksei Holik for correcting the bug in the spanning pairs calculation.

[JAFFA\_0.9.tar.gz](https://drive.google.com/open?id=0B1FwZagazjpcLTFSYWY4THNHdlE)
