# virheat

## Overview
virheat is a specialized visualization tool for comparative genomics. It transforms variant call data (VCF) into intuitive heatmaps, allowing researchers to track how SNPs evolve across different samples or time points. By integrating gene annotations, it highlights which mutations fall within specific coding regions, making it a powerful tool for viral surveillance and evolutionary biology. It supports advanced features like zooming into specific genomic regions, filtering variants by frequency thresholds, and overlaying experimental scores (e.g., MAVE data).

## Installation
Install via Bioconda or pip:
```bash
conda install -c bioconda virheat
# OR
pip install virheat
```

## Core Usage Patterns

### Basic Heatmap Generation
To create a heatmap from a folder of VCF files, you must provide the input directory, an output directory, and either a GFF3 file or the genome length.
```bash
virheat <input_vcf_folder> <output_dir> -r <reference_id> -g <path_to_annotation.gff3>
```

### Filtering and Cleaning Data
*   **Frequency Threshold**: Use `-t` to only display mutations above a specific frequency (0.0 to 1.0).
    ```bash
    virheat input/ output/ -g ref.gff3 -t 0.2
    ```
*   **Occurrence Filtering**: Use `-n` to hide mutations that appear in $n$ or fewer samples.
*   **Divergence Cleanup**: By default, the tool removes mutations present in all samples if their maximum frequency divergence is $< 0.5$. Use `--no-delete` to disable this behavior.

### Genomic Zooming
Focus on a specific region of interest (e.g., a specific gene or domain) using the `-z` flag.
```bash
virheat input/ output/ -g ref.gff3 -z 21563 25384
```

### Visual Customization
*   **Gene Arrows**: Use `--gene-arrows` to render gene annotations as arrows instead of blocks.
*   **Sorting**: Use `--sort` to organize sample names alphanumerically on the Y-axis.
*   **Output Format**: Specify the file extension in `--name` to change the format (pdf, png, svg, jpg).
    ```bash
    virheat input/ output/ -g ref.gff3 --name my_plot.svg
    ```

## Advanced Integrations

### Coverage Masking
To display regions with insufficient coverage in grey, provide per-base coverage TSV files (generated by Qualimap) in the same folder as your VCFs.
*   **Requirement**: The TSV file must have the exact same base name as the corresponding VCF file.
*   **Command**: Use `--min-cov` to set the threshold.
    ```bash
    virheat input/ output/ -g ref.gff3 --min-cov 50
    ```

### Additional Score Tracks (MAVE)
You can map external scores (like binding affinity or antibody escape) to mutations using the `-s` flag.
*   **Format**: `-s <csv_file> <pos_column> <score_column> <label>`
*   **Example**:
    ```bash
    virheat input/ output/ -g ref.gff3 -s scores.csv AA_pos binding_score "Binding Affinity"
    ```

## Expert Tips
*   **Reference ID**: Ensure the `-r` (reference identifier) matches the sequence ID used in your VCF and GFF3 files exactly, or the tool may fail to map mutations.
*   **VCF Compatibility**: virheat includes a custom VCF parser designed to handle files from various callers like FreeBayes, including those with multiple variants per line.
*   **Memory Management**: For very large genomes or hundreds of samples, use the `-z` (zoom) feature to reduce the rendering load and improve plot readability.
