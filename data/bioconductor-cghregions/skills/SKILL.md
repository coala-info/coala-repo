# bioconductor-cghregions

name: bioconductor-cghregions
description: Dimension reduction for array Comparative Genomic Hybridization (aCGH) data. Use this skill when you need to reduce the complexity of segmented and called aCGH data by identifying common genomic regions across multiple samples with minimal information loss. It is specifically designed to process output from the CGHcall package.

## Overview

CGHregions is a Bioconductor package used to simplify array CGH data. In a typical aCGH workflow, data is first segmented and "called" (assigned states like loss, normal, or gain). Because breakpoints often vary slightly between samples due to noise or technical artifacts, CGHregions aligns these breakpoints across the entire dataset. This reduces the number of variables (features) from thousands of individual probes to a smaller set of consistent genomic regions, which facilitates downstream statistical analysis, clustering, and visualization.

## Workflow and Usage

### 1. Data Preparation
The input must be an object of class `cghCall` (typically generated by the `CGHcall` package). This object contains log2-ratios, segment values, and call probabilities.

```r
library(CGHregions)
# Example using built-in dataset
data(WiltingCalled)
```

### 2. Defining Regions
The primary function is `CGHregions()`. It uses an information loss threshold (`averror`) to determine how much detail can be sacrificed to achieve dimensionality reduction.

```r
# averror: maximum allowed information loss (default is 0.01)
regions <- CGHregions(WiltingCalled, averror = 0.01)
```

### 3. Inspecting Results
The function returns a `cghRegions` object. You can check the reduction in dimensionality by comparing the number of features in the input vs. the output.

```r
# View summary of the regions object
print(regions)

# Access the reduced data matrix
# The 'regions' element contains the calls for the defined regions
region_data <- assayData(regions)$regions
```

### 4. Visualization
The package provides a specialized plot method to visualize the frequency of gains and losses across the genome for the defined regions.

```r
# Plot the regions across chromosomes
plot(regions)
```
**Plot Interpretation:**
*   **X-axis:** Base pair position.
*   **Y-axis:** Chromosomes.
*   **Colors:** Red indicates loss, Green indicates gain.
*   **Intensity:** The segments are bi-colored; the height of the red/green bars corresponds to the proportion of samples sharing that aberration.
    *   1: < 10% of samples
    *   2: 10% – 30%
    *   3: 30% – 50%
    *   4: > 50%

## Tips and Best Practices
*   **Input Requirements:** Ensure your data has been processed through `CGHcall` or formatted as a `cghCall` object. Raw log2-ratios cannot be processed directly without segmentation and calling.
*   **Tuning Information Loss:** If the number of regions is still too high for your analysis, increase the `averror` parameter. If you are losing too much biological detail, decrease it.
*   **Downstream Analysis:** The output `cghRegions` object is highly efficient for testing associations between copy number aberrations and clinical phenotypes, as it significantly reduces the multiple testing burden.
