# nwkit sanitize

[Jump to bottom](#wiki-pages-box)

Kenji Fukushima edited this page Feb 5, 2026
·
[5 revisions](/kfuku52/nwkit/wiki/nwkit-sanitize/_history)

## Overview

`nwkit sanitize` eliminates non-standard Newick flavors and standardizes the format. It performs three operations:

* **Singleton removal** (`--remove_singleton yes`, default): Collapses singleton nodes (internal nodes with only one child). The branch lengths of the singleton and its child are summed.
* **Polytomy resolution** (`--resolve_polytomy yes`, default: no): Resolves multifurcation (polytomy) nodes into a series of dichotomies with zero-length branches.
* **Name quoting** (`--name_quote`): Adds quote characters around leaf names. Options: `single`, `double`, or `none` (default).

## Examples

### 1. Remove singleton and add single quotes

In this example, the singleton node between `(a,b)` and the root is collapsed, and single quotation marks are introduced to the tip labels.

#### Command

```
nwkit sanitize --name_quote single --infile input.nwk --outfile output.nwk
```

#### input.nwk

```
((((a:1,b:1):1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224152238-92662168-4a55-4c46-9333-1cf15d7f53f3.png)

#### output.nwk

```
(('c':1,('a':1,'b':1):2):1,(('d':1,'e':1):1,'f':1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224152238-92662168-4a55-4c46-9333-1cf15d7f53f3.png)

The singleton `((a:1,b:1):1):1` is collapsed and its branch lengths are summed (`1+1=2`), resulting in `('a':1,'b':1):2`.

### 2. Add double quotes without singleton removal

#### Command

```
nwkit sanitize --name_quote double --remove_singleton no --infile input.nwk --outfile output.nwk
```

This adds double quotes around leaf names while preserving singleton nodes.

### 3. Resolve polytomies (multifurcations)

Polytomy nodes (internal nodes with more than 2 children) can be resolved into a series of dichotomies with zero-length branches. This is useful for compatibility with programs that require fully bifurcating trees.

#### Command

```
nwkit sanitize --resolve_polytomy yes --remove_singleton no --infile input.nwk --outfile output.nwk
```

#### input.nwk

```
((A:1,B:1,C:1):1,(D:1,E:1,F:1):1);
```

#### output.nwk

```
(((B:1,C:1)0:0,A:1)1:1,((E:1,F:1)0:0,D:1)1:1)1:0;
```

Each 3-way polytomy is resolved into two dichotomies. Newly created internal branches have zero length. Original leaf branch lengths are preserved.

**Note**: The resolution is deterministic (same input always produces the same output), but the introduced bifurcations are arbitrary and do not reflect real phylogenetic relationships. This option should only be used for compatibility with programs that require fully bifurcating trees.
