It works by building a index of compulsory substrings from a collection of regular expressions, which it uses to quickly exclude those expressions which trivially do not match each input. Here is some example code that uses esmre: &gt;&gt;&gt; import esmre &gt;&gt;&gt; index = esmre.Index() &gt;&gt;&gt; index.enter( r " Major-General \W * $ " , " savoy opera " ) &gt;&gt;&gt; index.enter( r " \b way \W + haye ? \b " , " sea shanty " ) &gt;&gt;&gt; index.query( " I am the very model of a modern Major-General. " ) ['savoy opera'] &gt;&gt;&gt; index.query( " Way, hay up she rises, " ) ['sea shanty'] &gt;&gt;&gt; The esmre module builds on the simpler string matching facilities of the esm module, which wraps a C implementation some of the algorithms described in Aho's and Corasick's paper on efficient string matching [Aho, A.V, and Corasick, M. J. Efficient String Matching: An Aid to Bibliographic Search. Comm. ACM 18:6 (June 1975), 333-340]. Some minor modifications have been made to the algorithms in the paper and one algorithm is missing (for now), but there is enough to implement a quick string matching index. Here is some example code that uses esm directly: &gt;&gt;&gt; import esm &gt;&gt;&gt; index = esm.Index() &gt;&gt;&gt; index.enter( " he " ) &gt;&gt;&gt; index.enter( " she " ) &gt;&gt;&gt; index.enter( " his " ) &gt;&gt;&gt; index.enter( " hers " ) &gt;&gt;&gt; index.fix() &gt;&gt;&gt; index.query( " this here is history " ) [((1, 4), 'his'), ((5, 7), 'he'), ((13, 16), 'his')] &gt;&gt;&gt; index.query( " Those are his sheep! " ) [((10, 13), 'his'), ((14, 17), 'she'), ((15, 17), 'he')] &gt;&gt;&gt; You can see more usage examples in the tests.
