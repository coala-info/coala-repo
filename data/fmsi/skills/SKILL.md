# fmsi

## Overview

FMSI (From Masked Superstring to Indexing) is a specialized bioinformatics tool designed for space-efficient k-mer set representation. It implements the FMS-index, a simplified version of the FM-index, to allow membership queries on unconstrained k-mer sets. By utilizing masked superstrings—often generated by companion tools like KmerCamel—FMSI achieves significantly lower memory footprints than traditional hash-based methods, making it ideal for large-scale genomic analysis where RAM is a constraint.

## Core Workflow

### 1. Index Construction
Before querying, you must build an index from a masked superstring file (typically a FASTA file where k-mers are represented as superstrings with associated masks).

```bash
fmsi index -k <k_length> <masked_superstring.fa>
```
*   **Note**: This generates several component files with extensions like `.fmsi.ac_gt`, `.fmsi.mask`, and `.fmsi.misc`. Ensure all these files remain in the same directory for querying.

### 2. Membership Queries
To check if specific k-mers exist in the index:

```bash
fmsi query -O -q <query.fa> -k <k_length> <masked_superstring.fa>
```
*   **Output**: Returns an identifier followed by a bitstring (1 for present, 0 for absent).
*   **Optimization**: Use `-O` if the mask was optimized (e.g., via `kmercamel optimize`) to ensure correct results.

### 3. Lookup Queries
To retrieve unique integer identifiers for present k-mers:

```bash
fmsi lookup -q <query.fa> -k <k_length> <masked_superstring.fa>
```
*   **Output**: Returns a comma-separated list of integers. K-mers not found are marked as `-1`.

## Expert Tips and Best Practices

- **Streaming Performance**: If you are performing streaming queries and need maximum speed, add the `-S` flag. This enables speed enhancements at the cost of 1 additional bit per superstring character.
- **Mask Optimization**: Always ensure your input masked superstring maximizes the number of ones in the mask. If you are unsure if the mask is optimized, omit the `-O` flag during querying to avoid false negatives, though optimizing with `kmercamel` beforehand is preferred.
- **Invalid Characters**: FMSI automatically handles non-ACGT characters in query k-mers by marking them as absent (0 in membership queries).
- **Memory Efficiency**: FMSI is extremely lean; for example, it can index the human genome using approximately 2.41 bits per canonical 31-mer. Use it as an alternative to Bloom filters when exact membership or unique lookups are required without the false positive rate of probabilistic structures.
- **File Paths**: When running `query` or `lookup` commands, the path to the original masked superstring FASTA file must be the **last** argument.
