# r-quorts

## Overview
The `quorts` (Quality of RNA-Seq Toolset) package is a comprehensive toolkit designed for the analysis and quality control of RNA-Seq data. It provides functions to visualize sequencing metrics, identify technical biases, and prepare data for downstream differential expression or exon usage analysis. It is particularly effective at aggregating metrics across large cohorts to identify outlier samples.

## Installation
To install the package from CRAN:
install.packages("quorts")

## Core Workflow
The typical workflow involves processing BAM files with the QoRTs Java utility (external to R) and then using the R package to visualize the results.

1. **Load QC Data**: Import the results generated by the QoRTs tool for one or more samples.
2. **Build Plotter Object**: Create a centralized object that manages sample metadata and QC metrics.
3. **Generate Visualizations**: Produce standardized plots for library complexity, gene body coverage, error rates, and more.
4. **Export Reports**: Generate summary tables or multi-page PDF reports.

## Key Functions
- `read.qc.results.data()`: Loads the raw QC data for a single sample from a specified directory.
- `build.plotter.obj()`: Aggregates multiple samples into a single object for comparative plotting. Requires a sample list data frame.
- `makePlot.insert.size()`: Visualizes the distribution of insert sizes.
- `makePlot.genebody()`: Displays the 5' to 3' coverage bias across gene bodies.
- `makePlot.qual.score()`: Plots the distribution of quality scores across read positions.
- `makePlot.biotype.rates()`: Shows the proportion of reads mapping to different genomic features (e.g., rRNA, protein-coding, intronic).

## Usage Tips
- **Sample Lists**: When working with multiple samples, ensure your sample list data frame contains a `sample.ID` column that matches the directory names of your QC data.
- **Outlier Detection**: Use the `makePlot.cumul.dist()` function to identify samples with unusual mapping distributions or library complexity.
- **Batch Processing**: The package is designed to handle large-scale experiments; use the plotter object to apply consistent styling across all samples in a study.
