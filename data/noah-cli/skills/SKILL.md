# noah-cli

## Overview

Noah is a specialized project management tool designed to bring reproducibility and portability to bioinformatics workflows. It provides a structured way to initialize research projects, manage data dependencies through a unified interface, and ensure that collaborators can reconstruct the data environment with a single command. By integrating with Git and supporting various public data repositories, it simplifies the transition from data acquisition to analysis.

## Core Commands and Usage

### Project Initialization
Start every new bioinformatics project by creating a managed directory. This automatically initializes a Git repository to track configuration changes.

```bash
noah init <project_name>
cd <project_name>
```

### Managing Datasets
Noah allows you to link datasets to your project using either public database identifiers or direct remote paths.

**Adding Public Datasets:**
Supports identifiers from SRA (SRP), ENCODE (ENCSR), and GEO (GSE).
```bash
noah add SRP123456
noah add ENCSR123ABC
noah add GSE123456
```

**Adding Private or Custom Datasets:**
Use SSH or FTP protocols for datasets not hosted on public repositories.
```bash
noah add user@host:path/to/dataset
noah add ftp://ftp.example.com/data/file.tar.gz
```

**Removing Datasets:**
To stop tracking a specific dataset within the project:
```bash
noah uninstall <dataset_id_or_url>
```

### Collaboration and Portability
When a project is shared (e.g., via Git), the data files themselves are typically not included. Use the install command to fetch all defined dependencies.

```bash
# Run this after cloning a Noah project to retrieve all linked data
noah install
```

## Best Practices

- **Initialize Early**: Always run `noah init` at the very start of a project to ensure all data additions are tracked in the project's configuration from day one.
- **Use Public IDs**: Whenever possible, use official repository identifiers (SRP/GSE/ENCSR) rather than direct URLs. This improves metadata tracking and reproducibility.
- **Git Integration**: Do not manually edit the configuration files generated by Noah unless necessary; use the CLI commands to ensure the internal state remains consistent with the Git index.
- **Environment Parity**: Use `noah install` in CI/CD pipelines or on high-performance computing (HPC) clusters to ensure the execution environment matches the development environment exactly.
