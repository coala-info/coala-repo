# seq2squiggle

## Overview

seq2squiggle is a deep learning-based simulator that uses feed-forward transformers to generate realistic nanopore signals from digital DNA sequences. It allows for the creation of synthetic datasets that mimic specific sequencing chemistries (like R9.4.1 or R10.4.1) and provides fine-grained control over signal noise and read length distributions. By transforming sequences into raw "squiggles," it enables the development and validation of nanopore tools without the need for expensive physical sequencing runs.

## Installation and Setup

Install via bioconda for the most stable environment:

```bash
mamba install seq2squiggle -c bioconda
```

Pretrained model weights are required. If a model path is not provided via `--model`, the tool will attempt to download compatible weights automatically during the first run.

## Core Prediction Workflows

### Reference Mode (Default)
Use this mode to sample synthetic reads from a genome or transcriptome reference file.

*   **Generate a specific number of reads:**
    `seq2squiggle predict reference.fasta -o output.blow5 -n 10000`
*   **Simulate specific coverage and read length:**
    `seq2squiggle predict reference.fasta -o output.blow5 -c 30 -r 5000`
*   **Specify sequencing chemistry:**
    `seq2squiggle predict reference.fasta -o output.blow5 --profile dna_r9_min`

### Read Mode
Use this mode when you already have FASTA/FASTQ reads and want to generate signals for those specific sequences without further fragmentation.

*   **1:1 Mapping (One signal per input read):**
    `seq2squiggle predict reads.fastq -o output.blow5 --read-input`
*   **Subsampling from reads:**
    `seq2squiggle predict reads.fastq -o output.blow5 --read-input -n 1000`

## Signal Customization and Noise Control

Fine-tune the "realism" of the output by adjusting the noise and duration samplers:

*   **Increase amplitude noise:** Use `--noise-std 1.5` to simulate noisier pore conditions.
*   **Static Dwell Time:** Disable the duration sampler for perfectly uniform k-mer movement:
    `--duration-sampling False --dwell-mean 10.0 --dwell-std 0.0`
*   **Ideal Signals:** Disable both noise and duration sampling for "perfect" theoretical signals:
    `--noise-sampling False --duration-sampling False`

## Expert Tips and Best Practices

*   **Output Formats:** While POD5 is supported (`-o file.pod5`), BLOW5 is currently the more stable and preferred format for this tool.
*   **Hardware Acceleration:** Always run on a GPU for production-scale simulations. Inference on CPU is supported but significantly slower for high-coverage datasets.
*   **Read Length Distribution:** Use `--distr` to define specific length distributions (e.g., gamma or exponential) when simulating genomic libraries.
*   **Training Custom Models:** To train a model for a new chemistry, you must provide event-level alignments (eventalign tables) generated by tools like `uncalled4` or `f5c`.
