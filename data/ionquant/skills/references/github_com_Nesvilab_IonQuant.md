It enables label-free quantification with false discovery (FDR) controlled match-between-runs (MBR). It can also be used for quantification in labelling-based experiments such as those involving SILAC, dimethyl, or similar labelling strategies. IonQuant is available as a stand-alone tool or it can be run within FragPipe computational platform. Workflow of peak tracing and quantification Workflow of FDR-controlled MBR It is accurate and sensitive Quantification accuracy and precision evaluation using the PXD003881 (IonStar) dataset Quantification accuracy and precision evaluation using the PXD028735 dataset It is fast System requirements Java 11+. ext folder from the latest MSFragger . The latest FragPipe from here (optional). Note: Bruker's native library needs Visual C++ Redistributable for Visual Studio 2017 in Windows. If you see an error saying cannot find Bruker native library, please try to install the Visual C++ redistibutable. Usage: java -jar IonQuant.jar &lt; options &gt; --specdir &lt; one directory to the spectral files &gt; --psm &lt; path to psm.tsv file &gt; --psm &lt; path to psm.tsv file &gt; ... OR java -jar IonQuant.jar &lt; options &gt; --filelist &lt; path to filelist file &gt; Options: --specdir &lt; string &gt; # Directory containing the spectral files (d/mzml/mzxml/raw/quantindex). One --specdir indicates one spectral directory and can have multiple --specdir. --perform-ms1quant 0/1 # Perform MS1 quantification. 0 = no, 1 = yes. Default: 1 --perform-isoquant 0/1 # Perform isobaric labeling quantification. 0 = no, 1 = yes. Default: 0 --isotol &lt; float &gt; # MS2 tolerance in ppm. Default: 10 --isolevel &lt; int &gt; # Isobaric quantification level. 2 = MS2, 3 = MS3, 4 = ZOOM-HR. Default: 2 --isotype &lt; string &gt; # Isobaric quantification type. Case insensitive. Support iTRAQ-4, iTRAQ-8, TMT-0, TMT-2, TMT-6, TMT-10, TMT-11, TMT-16, TMT-18, TMT-35, sCLIP-6, iBT-16, DiLeu-12, DiLeu-1, DeAla-13, iodoTMT-6. Default: TMT-10 --annotation &lt; string &gt; # Annotation file info for the isobaric quantification. Format: &lt;path to psm.tsv&gt;=&lt;path to annotation file&gt;. One --annotation indicates one annotation file and can have multiple --annotation. Default: &lt;blank&gt; --site-reports 0/1 # Generate site reports. 0 = no, 1 = yes. The psm.tsv need to have the modification localization modification columns. Default: 1 --msstats 0/1 # Generate MSstats input files. 0 = no, 1 = yes. Default: 0 --threads &lt; integer &gt; # Number of threads. 0 = all logical cores. Default: 0 --mztol &lt; float &gt; # MS1 tolerance in PPM. Default: 10.0 --imtol &lt; float &gt; # 1/K0 tolerance. Default: 0.05 --rttol &lt; float &gt; # Retention time tolerance. Unit: min. Default: 0.4 --psm &lt; string &gt; # Path to Philosopher's psm.tsv. One --psm indicates one psm.tsv and can have multiple --psm. --multidir &lt; string &gt; # Output directory for the multi-experimental result. Optional. Default: &lt;blank&gt; --normalization 0/1 # Normalize the intensities across all runs. Default: 1 --minisotopes 1/2/3 # Minimum isotopes required in feature extraction. Default: 2 --minscans &lt; integer &gt; # Minimum MS1 scans required in feature extraction. Default: 3 --minions &lt; integer &gt; # Minimum ions required in quantifying proteins. Only for MaxLFQ intensity. Default: 1 --excludemods &lt; string &gt; # Excluded modifications in quantifying peptide sequences and proteins. Format: &lt;amino acid&gt;&lt;mass&gt;;... Default: &lt;blank&gt; --maxlfq 0/1 # Calculate MaxLFQ intensity. 0 = no, 1 = yes. Default: 1 --ibaq 0/1 # [experimental] Calculate iBAQ intensity. The iBAQ intensity is normalized by the protein length, not the number of theoretical peptides. 0 = no, 1 = yes. Default: 0 --minexps &lt; int &gt; # Minimum experiments in picking an ion for quantifying proteins. Only for intensity, not for MaxLFQ intensity. Default: 1 --minfreq &lt; float &gt; # Minimum required frequency of an ion being selected for protein quantification. Only for intensity, not for MaxLFQ intensity. Default: 0 --tp &lt; int &gt; # Number of ions used in quantifying each protein. If 0, using all ions. For intensity, not for MaxLFQ intensity. Default: 0 --mbr 0/1 # Perform match-between-runs. Default: 0 --mbrrttol &lt; float &gt; # Retention time tolerance used in match-between-runs. Unit: min. Default: 1.0 --mbrimtol &lt; float &gt; # 1/K0 tolerance used in match-between-runs. Default: 0.05 --mbrtoprun &lt; integer &gt; # Maximum number of donor runs for each acceptor run. Default: 10 --mbrmincorr &lt; float &gt; # Minimum correlation coefficient between a donor run and its acceptor run. Default: 0 --ionmobility 0/1 # The data has ion mobility information or not (for conventional LC-MS data). Default: 0 --ionfdr &lt; float &gt; # Transferred ion false discovery rate threshold. Default: 0.01 --peptidefdr &lt; float &gt; # Transferred peptide false discovery rate threshold. Default: 1 --proteinfdr &lt; float &gt; # Transferred protein false discovery rate threshold. Default: 1 --light &lt; string &gt; # Light labelling mass. Format: &lt;amino acids&gt;&lt;mass&gt;;&lt;amino acids&gt;&lt;mass&gt;;... Optional. Default: &lt;blank&gt; --medium &lt; string &gt; # Medium labelling mass. Format: &lt;amino acids&gt;&lt;mass&gt;;&lt;amino acids&gt;&lt;mass&gt;;... Optional. Default: &lt;blank&gt; --heavy &lt; string &gt; # Heavy labelling mass. Format: &lt;amino acids&gt;&lt;mass&gt;;&lt;amino acids&gt;&lt;mass&gt;;... Optional. Default: &lt;blank&gt; --formula &lt; string &gt; # List the formula of the modifications used in the data. Required if want to perform isotope-labeling quantification. Format: &lt;chemical composition&gt;;&lt;chemical composition&gt;... example: C(2)H(2)O;HO(3)P;2H(4)C(2).
