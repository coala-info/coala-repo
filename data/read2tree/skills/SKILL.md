# read2tree

## Overview
`read2tree` is a phylogenomic tool designed to bypass the traditional assembly-first pipeline. By mapping raw reads directly to a set of reference orthologous groups (marker genes), it produces alignment matrices and species trees with significantly reduced computational overhead. It supports both short-read and long-read data and integrates with standard bioinformatics tools like minimap2, MAFFT, and IQ-TREE to provide a seamless "reads-to-tree" experience.

## Core Workflows

### Single Species Mode
Use this for a quick inference of a single sample against a reference set.
```bash
read2tree --tree --standalone_path marker_genes/ --reads read_1.fastq read_2.fastq --output_path output --dna_reference dna_ref.fa --debug
```

### Multiple Species Mode (Distributed Workflow)
For large datasets, split the process into three steps to allow for parallelization of the mapping phase.

**Step 1: Initialize Markers**
Prepares the reference markers for mapping.
```bash
read2tree --step 1marker --standalone_path marker_genes --dna_reference dna_ref.fa --output_path output
```

**Step 2: Map Reads (Parallelizable)**
Run this command for each species/sample in your dataset. This is the most time-consuming step and should be distributed across cluster nodes if possible.
```bash
read2tree --step 2map --standalone_path marker_genes --dna_reference dna_ref.fa --reads sample_R1.fastq sample_R2.fastq --output_path output --debug
```

**Step 3: Combine and Infer Tree**
Aggregates the mapping results and performs the final tree inference.
```bash
read2tree --step 3combine --standalone_path marker_genes --dna_reference dna_ref.fa --output_path output --tree --debug
```

## Configuration and Optimization

### Handling Different Read Types
The `--read_type` argument passes options directly to the underlying `minimap2` aligner.
- **Short Reads:** Use `-ax sr` (default).
- **PacBio HiFi:** Use `-ax map-hifi`.
- **Oxford Nanopore:** Use `-ax map-ont`.
- **Example:** `read2tree ... --read_type "-ax map-ont" --threads 40`

### Downstream Analysis
- **Bootstrapping:** To add support values to the tree, run IQ-TREE manually on the output phylip file generated by `read2tree`:
  `iqtree -T AUTO -s output/concat_merge_aa.phy -bb 1000`
- **DNA-mode:** For closely related species where protein sequences might be too conserved, use the nucleotide alignment:
  `iqtree -T AUTO -s output/concat_merge_dna.phy`

## Expert Tips
- **Marker Selection:** For optimal results, use 200 to 400 marker genes obtained from the OMA browser.
- **Clean Starts:** `read2tree` attempts to resume previous runs by checking the output directory. If you want a fresh analysis or have changed parameters, you must delete or move the existing output directory.
- **Debugging:** Always include the `--debug` flag. It provides essential logs for troubleshooting mapping or alignment failures and helps identify which specific markers might be causing issues.
- **Species Naming:** Use 5-letter codes (e.g., HUMAN, MOUSE) for consistency with OMA standards and to avoid potential parsing issues with long filenames.
- **Resource Management:** MAFFT and IQ-TREE are configured to use multiple threads in version 2.0+. Ensure your `--threads` allocation matches your hardware to maximize performance during the alignment and tree-building phases.
