# kounta

## Overview

kounta is a bioinformatics tool designed to create a unified tab-separated matrix of k-mer counts across multiple genomic datasets. It leverages the efficiency of `kmc` for initial counting and standard Unix utilities for merging, ensuring that large-scale multi-sample data can be processed without requiring all data to be held in memory simultaneously. It is particularly useful for identifying unique or shared genomic signatures across a population of genomes.

## CLI Usage and Patterns

### Basic Matrix Generation

To generate a matrix from assembly files (contigs):
```bash
kounta --kmer 31 --out matrix.tsv *.fna
```

To generate a matrix from raw sequencing reads (FASTQ):
```bash
kounta --kmer 31 --threads 8 --ram 4 --out matrix.tsv *.fq.gz
```

### Key Parameters
- `--kmer`: Length of the k-mer (default is often small; 31 is common for bacterial genomics).
- `--out`: Path to the output TSV file.
- `--threads`: Number of CPU threads to use for parallel processing.
- `--ram`: Maximum memory (in GB) to allocate for `kmc` operations.
- `--minfreq`: Minimum k-mer frequency to report (essential when using reads to filter out sequencing errors).
- `--fofn`: Provide a "File Of File Names" if the input list is too long for the shell command line.

## Best Practices and Expert Tips

### Input Consistency
**Never mix reads and contigs** in a single kounta run. The k-mer frequencies between assemblies (usually 1x coverage) and raw reads (variable depth) are not directly comparable and will produce misleading results.

### Handling Sequencing Reads
- **Use R1 only**: When working with paired-end data, it is recommended to use only the forward reads (`R1`). The reverse reads (`R2`) are typically noisier and more error-prone, adding little information while significantly increasing computational overhead.
- **Set Frequency Thresholds**: Always use `--minfreq` (e.g., `--minfreq 2` or higher) when processing reads to prevent the matrix from being overwhelmed by k-mers generated by single-base sequencing errors.

### Performance Optimization
kounta's speed is primarily limited by disk I/O and the resources allocated to `kmc`.
- Provide as much `--threads` and `--ram` as the system can spare.
- Ensure the temporary directory (where `kmc` writes intermediate files) is on a fast disk (SSD/NVMe).

### Post-Processing the Matrix
The output is a standard TSV that can be manipulated with Unix tools:

- **Extract Core K-mers**: To find k-mers present in every sample (non-zero counts), filter out rows containing a `0`:
  ```bash
  grep -v -w 0 matrix.tsv > core_kmers.tsv
  ```
- **Binarize (Presence/Absence)**: To convert counts to 1 (present) or 0 (absent):
  ```bash
  sed -e '1 ! s/[1-9][0-9]*/1/g' matrix.tsv > presence_absence.tsv
  ```
  *Note: This specific sed command preserves the header but converts all positive integers to 1.*
