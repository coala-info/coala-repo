[SAVANA 1.3.4](/cortes-ciriano-lab/savana/releases/tag/1.3.4)

This release includes bug fixes for:

* Error when allele fraction (AF) is used in custom filtering JSON
* Allowing savana to run in case of empty or breakpoint free input ([@johanneskoester](https://github.com/johanneskoester), [#76](https://github.com/cortes-ciriano-lab/savana/pull/76))
* Alternative reference bases during CNA allele counting ([#80](https://github.com/cortes-ciriano-lab/savana/issues/80))

Additionally, it includes the following documentation updates:

* Clarifying phasing recommendations for optimal results ([#75](https://github.com/cortes-ciriano-lab/savana/issues/75))
* Updating a link to the HMF SV Panel of Normals ([#79](https://github.com/cortes-ciriano-lab/savana/issues/79))

Finally, it adds a template germline custom\_params that can be used as a template for germline calling (though we recommend users adapt this file to their specific needs).
