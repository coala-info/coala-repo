# ccmetagen

## Overview
CCMetagen is a microbiome classification pipeline designed for high-accuracy identification of organisms across all domains of life. It functions as a downstream processor for the KMA (ConClave) aligner. By leveraging the full NCBI nt database, it allows for the inclusion of understudied microbial eukaryotes alongside prokaryotes. The tool filters raw alignment data to produce publication-ready taxonomic summaries and interactive visualizations.

## Installation and Setup
The preferred method for installation is via Conda to ensure all dependencies (like KMA) are included.

```bash
conda create -n ccmetagen ccmetagen -c bioconda -c conda-forge
conda activate ccmetagen
```

## Core Workflow

### 1. Alignment with KMA
Before running CCMetagen, reads must be mapped to a reference database using KMA.

**Paired-end reads:**
```bash
kma -ipe $R1 $R2 -o sample_out -t_db $DB -t $THREADS -1t1 -mem_mode -and -apm f -ef
```

**Single-end reads:**
```bash
kma -i $READS -o sample_out -t_db $DB -t $THREADS -1t1 -mem_mode -and -ef
```
*Note: The `-ef` (extended features) flag is required if you intend to calculate RPM or fragment-based abundances.*

### 2. Taxonomic Classification
Process the `.res` file generated by KMA with CCMetagen.

```bash
CCMetagen.py -i sample_out.res -o results_prefix
```

## Advanced Usage and Parameters

### Abundance Estimation
Control how microbial presence is quantified using the `--depth_unit` flag:
- `default`: Template length correction (KMA standard).
- `nc`: Nucleotide count matching the reference.
- `rpm`: Reads Per Million (requires `.mapstats` file from KMA `-ef`).
- `fr`: Fragment count (requires `.mapstats` file from KMA `-ef`).

### Filtering and Stringency
Adjust these parameters to balance sensitivity and specificity:
- `--coverage`: Minimum coverage required for a taxonomic assignment.
- `--depth`: Minimum abundance/depth threshold.
- `--level`: Minimum taxonomic rank level for assignment.

### Merging Multiple Samples
To combine results from multiple CCMetagen runs into a single table:
```bash
python CCMetagen_merge.py -i sample1.csv sample2.csv -o merged_table.csv
```

## Best Practices
- **Database Selection**: Use the full NCBI nt database for comprehensive eukaryotic and prokaryotic detection. For faster, cleaner results, use the "no environmental/artificial" version of the nt database.
- **Custom Databases**: If building a custom KMA index, ensure taxids are included in the headers in the format `>taxid|description`.
- **Memory Management**: For large datasets (>1.5GB), always use the command-line version rather than the web service. Use KMA's `-mem_mode` to manage RAM usage during the alignment phase.
