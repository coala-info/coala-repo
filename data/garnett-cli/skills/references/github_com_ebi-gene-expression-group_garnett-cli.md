You can build cell type classification model using single-cell RNA-seq data and expressed marker genes. Alternatively, you can use pre-trained classifiers to determine cell types in your data. Note: Nextflow implementation of Garnett pipeline is also available here . Graphical representation of the general workflow: Installation Garnett-cli is installed through Conda. Miniconda is a good way of getting set up with a basic Conda installation. It will help if you have your Conda set up to use channels as per Bioconda: conda config --add channels defaults conda config --add channels bioconda conda config --add channels conda-forge We recommend you use a fresh environment to install the package: conda create --name &lt;garnett-env&gt; conda activate &lt;garnett-env&gt; conda install -c bioconda garnett-cli Testing To test Garnett-cli installation, run garnett-cli-post-install-tests.sh 'test' 'false' Commands Currently available Garnett functions are listed in the following section. Each script has usage instructions available via --help, consult function documentation in Garnett for more dertail. parse_expr_data(): Build CDS object input from raw expression matrix If you are using raw experiment data, you can build a CDS object using the following script: parse_expr_data.R --input-10x-dir &lt;10x-style expression data directory&gt;\ --output-cds &lt;output file path for query CDS object in .rds format&gt; Please refer here for description of the file structure. transform_marker_file(): Transform SCXA-style marker file into Garnett-compatible one Garnett has an author-defined marker file format. To work with marker files stored in SCXA (example), they need to be transformed in a specific way. Marker genes are filtered to leave only those with adjusted p-value below specified threshold. transform_marker_file.R --input-marker-file &lt;path to SCXA type marker file&gt;\ --marker-list &lt;path to serialised object containing marker genes per cell type&gt;\ --garnett-marker-file &lt;path to the output file in Garnett-compatible format&gt; garnett_check_markers(): Check marker file In order to verify that markers provide an accurate representation of corresponding cell types, run the following script: garnett_check_markers.R --cds-object &lt;path to CDS object in .rds format&gt;\ --marker-file-path &lt;path to marker file in .txt format&gt;\ --database &lt;name of gene database (e.g. org.Hs.eg.db for Homo Sapiens genes)&gt;\ --cds-gene-id-type &lt;Format of the gene IDs in your CDS object&gt;\ --marker-file-gene-id-type &lt;Format of the gene IDs in your marker file&gt;\ --marker-output-path &lt;output path for marker analysis file in .txt format&gt;\ --plot-output-path &lt;output path for the plot in .png format&gt; NB : before specifying the database, make sure you have it installed as package in your environment. For example, you can use conda or bioconductor to install org.Hs.eg.db. If the flag --plot-output-path is used, graphical representation of marker quality will be produced automatically. update_marker_file(): Remove suboptimal markers After the marker checking step, to avoid manual inspection of the reults, this scrip removes markers deemed suboptimal by Garnett. update_marker_file.R --marker-list-obj &lt;path to serialised object containing marker genes per cell type&gt;\ --marker-check-file &lt;path to marker analysis file in .txt format&gt;\ --updated-marker-file &lt;path to updated marker gene file in .txt format&gt; garnett_train_classifier(): Train the classifier Although a range of pre-trained classifiers are available for usage, you can train your own via the following command: garnett_train_classifier.R --cds-object &lt;path to CDS object in .rds format&gt;\ --marker-file-path &lt;path to marker file in .txt format&gt;\ --train-idf &lt;Path to the training data IDF file (optional)&gt;\ --database &lt;name of gene database (e.g. org.Hs.eg.db for Homo Sapiens genes)&gt;\ --output-path &lt;output path for the trained classifier in .rds format&gt; garnett_get_feature_genes(): Get genes used as features in the classification model In some cases, it might be of interest to investigate which genes are deemed important by the classification model and thus used as features for classification. garnett_get_feature_genes.R --classifier-object &lt;path to a classifier object (pre-trained or trained _de novo_) in .rds format&gt;\ --database &lt;name of gene database (e.g. org.Hs.eg.db for Homo Sapiens genes)&gt;\ --output-path &lt;path to output file in .txt format&gt; garnett_classify_cells(): Classify cell types garnett_classify_cells.R --cds-object &lt;path to CDS object in .rds format&gt; --classifier-object &lt;path to a classifier object (pre-trained or trained _de novo_) in .rds format&gt; --database &lt;name of gene database (e.g. org.Hs.eg.db for Homo Sapiens genes)&gt; Classification column will be added to the CDS object's metadata as an additional column. Note : to repeat classification on the same CDS object you will need first to delete the column with previous classification result. ### get standard-format output for downstream analysis garnett_get_std_output.R\ --input-object &lt;Path to the input CDS object in .rds format&gt;\ --cell-id-field &lt;Column name of the cell id annotations. If not supplied, it is assumed that cell ids are represented by index&gt;\ --predicted-cell-type-field &lt;Column name of the predicted cell type annotation&gt;\ --classifier &lt;Path to the classifier object in .rds format (Optional; required to add dataset of origin to output table)&gt;\ --output-file-path &lt;Path to the produced output file in .tsv format&gt; get_package_data(): Obtain test data In case you would like to extract raw data for the test CDS object and example marker file, run: get_package_data.R\ --marker-file &lt;output path for marker file in .txt format&gt;\ --expr-matrix &lt;output path for extracted expression matrix in .txt format&gt;\ --pheno
