# vamb

## Overview
Vamb (Variational Autoencoder for Metagenomic Binning) is a suite of tools designed to group metagenomic contigs into biologically meaningful bins. It leverages deep learning to combine sequence composition (k-mer frequencies) and co-abundance patterns across multiple samples. The toolset includes the original VAE binner, a semi-supervised taxonomic binner (TaxVamb), and a taxonomy refiner (Taxometer). It is particularly effective for multi-sample datasets where co-abundance provides strong signals for clustering.

## Common CLI Patterns

### Standard Binning Workflow
The most common workflow involves preparing a concatenated FASTA file and an abundance file.

```bash
# 1. Concatenate assemblies and rename contigs to S{sample}C{contig}
python src/concatenate.py contigs.fna.gz asm_1/contigs.fasta asm_2/contigs.fasta

# 2. Generate abundance (using strobealign --aemb is recommended)
strobealign -t 8 --aemb contigs.fna.gz sample1.fq.gz > aemb/sample1.tsv
python src/merge_aemb.py aemb abundance.tsv

# 3. Run the default VAE binner
vamb bin default --outdir vambout --fasta contigs.fna.gz --abundance_tsv abundance.tsv
```

### Using TaxVamb (Semi-supervised)
If taxonomic annotations (e.g., from MMseqs2) are available, TaxVamb provides superior accuracy.

```bash
vamb bin taxvamb --outdir taxout --fasta contigs.fna.gz --abundance_tsv abundance.tsv --taxonomy taxonomy.tsv
```

### Refining Taxonomy with Taxometer
To improve arbitrary taxonomic predictions using composition and co-abundance:

```bash
vamb taxometer --outdir taxo_refine --fasta contigs.fna.gz --abundance_tsv abundance.tsv --taxonomy raw_taxonomy.tsv
```

## Expert Tips and Best Practices

*   **Contig Naming**: Vamb requires a specific naming convention `S{sample}C{contig}` for binsplitting to work correctly. Use the provided `concatenate.py` script to ensure headers are formatted properly.
*   **Abundance Input**: While BAM files are supported via `--bamdir`, using TSV files generated by `strobealign --aemb` is significantly faster and more memory-efficient.
*   **Binsplitting**: By default, Vamb uses `-o C` to split bins by sample. To disable this and treat the entire assembly as a single sample, pass the `-o` flag without an argument.
*   **Contig Length**: Use the `-m` flag to set a minimum contig length (default is often 2000bp). Shorter contigs often have noisy k-mer profiles that can degrade clustering quality.
*   **GPU Acceleration**: Vamb supports PyTorch. If a GPU is available, it will significantly speed up the VAE training process, though clustering itself may still be CPU-bound depending on the version.
*   **Output Files**: 
    *   `vae_clusters_split.tsv`: The primary output containing contig-to-bin assignments.
    *   `composition.npz`: Contains TNFs and sequence metadata; keep this if you plan to re-run clustering without re-parsing the FASTA.



## Subcommands

| Command | Description |
|---------|-------------|
| vamb bin | Binning module of VAMB |
| vamb recluster | Use marker genes to re-cluster (DBScan) or refine (K-means) clusters. |
| vamb taxometer | Refine taxonomy using composition and abundance information. |
