# pathogentrack

## Overview

PathogenTrack is an unsupervised computational pipeline that identifies pathogenic microorganisms by analyzing the reads in scRNA-seq data that do not map to the host genome. It integrates tools like fastp, STAR, and Kraken2 to quantify pathogen expression across individual cells. This skill provides the necessary command-line patterns and configuration requirements to execute the pipeline, manage database dependencies, and correctly format barcode/UMI patterns for different scRNA-seq chemistries.

## Environment Setup

The tool is available via Bioconda. It is recommended to use a dedicated Conda environment due to specific dependency versions (Python 3.6, STAR 2.7.5a, Kraken2 2.1.1).

```bash
conda create -n pathogentrack -c bioconda pathogentrack
conda activate pathogentrack
```

## Required Inputs

Before running PathogenTrack, you must have the following files ready:
1.  **Raw FASTQ files**: Paired-end reads (e.g., `R1.fastq.gz` and `R2.fastq.gz`).
2.  **Valid Barcodes**: A `barcodes.tsv` file (typically obtained from the output of `cellranger count` or `alevin`).
3.  **STAR Index**: A host genome index (e.g., Human GRCh38) generated by STAR.
4.  **Kraken2 Database**: A standard or miniKraken database for microbial identification.

## Command Line Usage

The primary command is `PathogenTrack count`.

### Standard Execution Pattern

```bash
PathogenTrack count --project_id <output_name> \
    --pattern <CB_UMI_PATTERN> \
    --star_index /path/to/STAR_index/ \
    --kraken_db /path/to/kraken_db/ \
    --barcode barcodes.tsv \
    --read1 sample_R1.fastq.gz \
    --read2 sample_R2.fastq.gz \
    --min_reads 10 \
    --confidence 0.11
```

### Configuring the --pattern Argument

The `--pattern` argument is critical as it defines the structure of Read 1 (Cell Barcode and UMI). Incorrect patterns will result in zero quantification.

*   **10X Genomics v2 Chemistry**: 16bp Cell Barcode (C) + 10bp UMI (N)
    *   Pattern: `CCCCCCCCCCCCCCCNNNNNNNNNN`
*   **10X Genomics v3 Chemistry**: 16bp Cell Barcode (C) + 12bp UMI (N)
    *   Pattern: `CCCCCCCCCCCCCCCNNNNNNNNNNNN`

## Expert Tips and Best Practices

*   **Host Alignment**: Ensure the STAR index used for PathogenTrack matches the one used in your initial scRNA-seq processing (e.g., Cellranger) to maintain consistency in what is considered "unmapped."
*   **Confidence Threshold**: The `--confidence` parameter (default often 0.11) significantly impacts the precision of pathogen identification. Higher values reduce false positives from homologous sequences but may decrease sensitivity for low-abundance pathogens.
*   **Resource Allocation**: Processing a single sample typically takes 4–6 hours. Ensure the environment has sufficient RAM (at least 32GB-64GB) to load the STAR index and Kraken2 database simultaneously.
*   **Pre-processing**: Always run `cellranger count` or a similar tool first. PathogenTrack requires the `barcodes.tsv` from the filtered feature-barcode matrix to ensure it only quantifies pathogens within "real" cells.
