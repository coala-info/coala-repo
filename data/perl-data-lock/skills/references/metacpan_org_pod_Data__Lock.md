Unlike Readonly which implements immutability via tie , dlock makes use of the internal flag of perl SV so it imposes almost no penalty. Like Readonly , dlock locks not only the variable itself but also elements therein. As of verion 0.03, you can dlock objects as well. Below is an example constructor that returns an immutable object: sub new { my $pkg = shift; my $self = { @_ }; bless $self, $pkg; dlock($self); $self; } Or consider using Moose . EXPORT Like List::Util and Scalar::Util , functions are exported only explicitly. This module comes with dlock and dunlock . use Data::Lock; # nothing imported; use Data::Lock qw/dlock dunlock/; # imports dlock() and dunlock() FUNCTIONS dlock dlock($scalar); Locks $scalar and if $scalar is a reference, recursively locks referents. dunlock Does the opposite of dlock .
