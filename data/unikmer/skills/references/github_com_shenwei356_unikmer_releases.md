### [v0.20.0](https://github.com/shenwei356/unikmer/releases/tag/v0.20.0) - 2023-11-11 [![Github Releases (by Release)](https://camo.githubusercontent.com/ed5830f018821443c38be19bdaa6d44dfe84e53481db77424d7a2217a63e3ccd/68747470733a2f2f696d672e736869656c64732e696f2f6769746875622f646f776e6c6f6164732f7368656e7765693335362f756e696b6d65722f76302e32302e302f746f74616c2e737667)](https://github.com/shenwei356/unikmer/releases/tag/v0.20.0)

* `unikmer`:
  + update help messages
  + **rename subcommand `uniqs` to `map`**.
  + do not add the extension `.unik` if the value of flag `-o/--out-prefix` already has one.
* `unikmer sort`:
  + fix the bug of using both `-d/--repeated` and `-m/--chunk-size`, which seems to existed for 4 years.
* `unikmer merge`:
  + fix the bug of missing one record when taxid information is contained.
* `unikmer num`:
  + `-f/--force` also support sorted files.
* `unikmer map`:
  + fix a bug of missing some regions.
* `unikmer locate`:
  + fix seq ID error in output.
  + deduplicate output.
