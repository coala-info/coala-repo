# 06. Inspect or convert profiles with popdel view

[Jump to bottom](#wiki-pages-box)

Sebastian Niehus edited this page Mar 26, 2021
·
[5 revisions](/kehrlab/PopDel/wiki/06.-Inspect-or-convert-profiles-with-popdel-view/_history)

If you are interested in the contents of a profile you can use `popdel view` to display them in a human-readable format. The options `-e` and `-i` allow you to inspect the header and the sampled insert size histogram(s) of all read groups in the file. The output will be written to standard output and can be piped or redirected.

| Flag | Extended flag | Meaning |
| --- | --- | --- |
| `-e` | `--header` | Write the header |
| `-E` | `--onlyHeader` | Only write the header |
| `-i` | `--histograms` | Write insert size histograms |
| `-r` | `--region` | Limit view to this genomic region |
| `-x` | `--uncompressed-in` | Read uncompressed binary profile |

The output of a profile should look as follows:

```
$ popdel view myProfile.profile | less
chr21   5035777 230:-10,233:-34		4:32,85:68,103:-2,110:37
chr21   5036033 3:-37,11:0,214:15	23:110,28:-28,30:67,39:45,42:-37
chr21   5036289 4:-85,13:-96,13:-73,17:-6	89:54,93:63,106:-23,126:-43,130:12,147:12,191:106
```

The first two columns together describe the starting point of the window in the reference. The next column refers to the first read group of the sample, the second column to the second read group and so on. The value in front of the ':' denotes the positional offset of the respective read pair from the starting point of the window, while the value after the ':' denotes its deviation from the read group's median insert size.

**Explanation of one line of a profile**

![Sample profile](https://github.com/kehrlab/kehrlab.github.io/raw/master/assets/popdel/images/profile.jpg)

**Conversion and unzipping of profiles**

`popdel view` also offers options for the conversion or unzipping of profiles. Working with unzipped profiles reduces the I/O overhead when performing the calling on very big numbers of samples, albeit it increases the size of a profile by ~20% compared to the default profiles.

| Flag | Extended flag | Meaning | Default |
| --- | --- | --- | --- |
| `-o` | `--out` | Output path for the unzipped profile | NA |
| `-c` | `--chunk-size` | Partition the output profile into chunks of the given size in bp. The output file name (`-o`) will be extended by the generated regions | NA |
| `-p` | `--padding` | Add the given number of padding bases to the end of each chunk (`-c`), so that they overlap by that amount | 0 |
| `-s` | `--index-region-size` | Index region size for output profile. | Same as input profile |

**NOTE:** For performing the calling on an uncompressed profile, `popdel call` has to be run with the `-x` option. This holds for all scenarios mentioned below.

**Examples**

The `-o` option tells PopDel view to not print the profile contents in human readable format, but to save them in uncompressed binary format at specified location:

```
popdel view myProfile.profile -o myUncompressedProfile.profile
```

It is also possible to re-specify the index-region-size, i.e. the resolution of the built-in index via the `-s` option. Per default, `popdel view -o` keeps the index-region-size of the original profile, which is 10,0000 if not specified differently during the profile creation. A lower value than 10,000 allows for more efficient and faster navigation in the profile during the calling process, which might be desirable for very large cohorts, but will also slightly increase the size of the profiles, because more genomic positions have to be indexed:

```
popdel view myProfile.profile -o myUncompressedProfile.profile -s 5000
```

Please note, that all profiles must have the exact same index-region-size if you want to perform the calling with the `-e` option. Without the `-e` option the profiles can have individual index-region-sizes.

The unzipping of a profile can also be limited to a single chromosome or region, using the `-r` option giving the desired region in samtools-style:

```
popdel view myProfile.profile -o myUncompressedProfile.profile -r chr21
```

To only extract a profile of chr21, or

```
popdel view myProfile.profile -o myUncompressedProfile.profile -r chr21:6000000
```

to extract a profile of chr21, starting at the 6,000,000th base, or

```
popdel view myProfile.profile -o myUncompressedProfile.profile -r chr21:6000000-7000000
```

to extract a profile of chr21, starting at the 6,000,000th base and ending at the 7,000,000th base.

Finally, it is also possible to partition the profile into blocks of a fixed length with a specified overlap between them to allow for easier distribution of the calling between machines, e.g.:

```
popdel view myProfile.profile -o myPartitioned -c 10000000 -p 10000
```

will create a variable number of output profiles, depending on the number and lengths of the contigs/chromsomes in the reference genome used for mapping the sample the profile was created for. The filenames will follow the pattern "myPartioned.*CHROMOSOME*.*Start*.*End*.profile" and cover the whole genome in block of 10000000 + 10000 bp, with 10000 bp overlap of the blocks.
The partitioning can also be limited to a single contig/chromosome by using the `-r` option:

```
popdel view myProfile.profile -o myPartitioned -c 10000000 -p 10000 -r chr21
```

**Further Notes**

* When partitioning a profile, the sum of the sizes of the partitioned files will be bigger than then original profile, because each of the profiles contains the complete header and index of the profile. Geader+index typically amount to 1-2 MB depending on the genome and index-size-region-size.
* When partitioning a profile with a relatively small chunk size (below 10,000,000), several 'empty' profiles which only contain the header and index can be expected, especially for centromeric, telomeric or gap regions of the genome. `popdel call` will handle them adequately.
* When partitioning a profile and specifying a chromosome via `-r`, only the name of the chromosome will be considered. Further limitations of the region (e.g. chr21:6000000-7000000) will be ignored.
* When performing the variant calling with `popdel call` using unzipped profiles, the option `-x` must be provided. Failing to do so, will result in undefined behavior, most likely resulting in segmentation fault and crashes.
* When performing the variant calling with `popdel call` using unzipped profiles, ALL profiles of the cohort must be in unzipped format.
* When changing the index-region-size using the `-s` option, please be aware that all profiles must have the exact same index-region-size if you want to use the `-e` option of the calling for reduced memory consumption. Without the `-e` option, the profiles are allowed to have individual index-region-sizes.
