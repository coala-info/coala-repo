# pangene

## Overview
Pangene is a command-line utility designed to build pangenome gene graphs where nodes represent marker genes and edges represent genomic adjacency. By utilizing protein-to-genome alignments, it simplifies complex pangenomic data into a gene-level view, making it easier to identify orthologous groups, paralogs, and structural variations. It is particularly effective for comparative genomics where the goal is to understand how gene order and content differ across a population.

## Core Workflow

### 1. Prepare the Protein Set
Before building a graph, you must define the marker genes. It is recommended to use canonical isoforms and follow a specific naming convention: `GeneName:ProteinID` (e.g., `RGPD6:ENSP00000512633.1`).

```bash
# Extract protein sequences from GTF/GFF
k8 pangene.js getaa gene-anno.gtf protein-seq.faa > proteins.faa

# For GenCode, extract canonical isoforms only
k8 pangene.js getaa -c gene-anno.gtf protein-seq.faa > proteins.faa
```

### 2. Align Proteins to Genomes
Pangene requires alignments generated by `miniprot` in PAF format.

```bash
# Standard alignment for eukaryotes
miniprot --outs=0.97 --no-cs -Iut16 genome.fna proteins.faa > genome.paf

# For bacteria (disables splicing)
miniprot -S --outs=0.97 --no-cs -Iut16 genome.fna proteins.faa > genome.paf
```

### 3. Construct the Graph
Combine all PAF files into a single GFA graph.

```bash
# Basic construction
pangene *.paf > graph.gfa

# Filter out edges supported by only one genome (reduces clutter)
pangene -a2 *.paf > graph.gfa

# Retain rare genes (disables the default 5% frequency filter)
pangene -p0 *.paf > graph.gfa
```

## Analysis and Visualization

### Bubble Calling
Identify local gene-level variations (bubbles) within the graph:
```bash
k8 pangene.js call graph.gfa > bubbles.txt
```

### Presence/Absence Matrix
Generate a gene presence/absence matrix for downstream statistical analysis:
```bash
k8 pangene.js gfa2matrix graph.gfa > gene_presence_absence.Rtab
```

### Visualization
*   **BandageNG**: The GFA output is compatible with standard GFA viewers.
*   **gfa-server**: For interactive web-based visualization of local subgraphs, use the `gfa-server` utility provided in the pangene binary distribution.

## Expert Tips
*   **Gene Naming**: Always use human-readable gene names in the protein headers if you intend to visualize the graph; nodes in the GFA are named directly after the gene identifiers.
*   **Redundancy**: Pangene automatically attempts to reduce redundancy in input proteins while preserving distinct paralogs. If the graph is too complex, increase the alignment identity threshold in the `miniprot` step.
*   **W-lines vs L-lines**: Note that in the output GFA, arcs on Walk (W) lines may occasionally be absent from Link (L) lines.
