# wtforms-alchemy

---

## Overview

wtforms-alchemy is a specialized extension for WTForms that provides a seamless bridge to SQLAlchemy. Instead of manually defining form fields that mirror your database columns, this tool allows you to create a `ModelForm` that inspects your SQLAlchemy models to generate the appropriate fields and validators automatically. It handles complex mappings including unique constraints, length limits, and relationship-based select fields, ensuring your application's frontend validation remains consistent with your backend data model.

## Core Usage Pattern

To use wtforms-alchemy, your form classes should inherit from `wtforms_alchemy.ModelForm`. The configuration is handled via an inner `Meta` class.

```python
from wtforms_alchemy import ModelForm
from my_models import User

class UserForm(ModelForm):
    class Meta:
        model = User
        # Use 'only' to whitelist specific columns
        only = ['username', 'email', 'first_name']
        # Use 'exclude' to blacklist specific columns
        # exclude = ['password_hash']
```

## Advanced Configuration and Best Practices

### Handling Relationships
To automatically generate `QuerySelectField` or `QuerySelectMultipleField` for model relationships, use the `include_relationships` option.

- **Enable relationships**: Set `include_relationships = True` in the `Meta` class.
- **Manual overrides**: If a relationship field requires specific query logic, define it explicitly in the form class to override the auto-generated version.

### Field Customization via SQLAlchemy
You can influence how fields are generated by adding a `info` dictionary to your SQLAlchemy columns. wtforms-alchemy looks for specific keys here:

- **Labels**: `Column(String(50), info={'label': 'User Name'})`
- **Validators**: Add custom WTForms validators via the `validators` key in the `info` dict.
- **Widget tweaks**: Pass attributes to the field constructor using the `render_kw` or `widget` keys.

### Validation and Constraints
- **Unique Constraints**: The tool automatically adds a `Unique` validator for columns marked `unique=True` in SQLAlchemy. Note that this requires a session to be passed to the form or configured globally.
- **Length**: `String(length=X)` columns automatically receive a `Length(max=X)` validator.
- **Optional Fields**: Columns marked `nullable=True` are treated as optional; `nullable=False` columns typically receive a `DataRequired` or `InputRequired` validator.

### Expert Tips
- **SQLAlchemy 2.0 Support**: Ensure you are using version 0.19.0+ for full compatibility with SQLAlchemy 2.0 and WTForms 3.2.
- **JSON Columns**: The library supports `JSON` type columns, mapping them to text areas or custom fields depending on your configuration.
- **Type Coercion**: Be cautious with the `Unique` validator on fields where type coercion might fail (e.g., an Integer field receiving a string), as this can occasionally trigger 500 errors if not handled in the application logic.
- **Empty Lists in Meta**: When using `Meta.only`, ensure the list is not empty if you intend to restrict fields; older versions had issues where empty lists were ignored.
