# diffexpir

## Overview

DiffExpIR is a specialized bioinformatics tool designed to detect differential intron retention (DIR) events. It functions as a downstream analysis tool that consumes the output generated by TPMCalculator. By comparing two experimental groups, it calculates the Log2 TPM ratio, p-values, and R-values (the ratio of intron expression to the expression of neighboring exons). This allows researchers to pinpoint specific intronic regions that show significant changes in retention levels across different conditions or genotypes.

## Installation and Environment

Before running DiffExpIR, ensure that BAMTools is installed and the environment variables are correctly configured for compilation:

```bash
export BAMTOOLS_DIR=/path/to/bamtools
export CPPFLAGS="-I $BAMTOOLS_DIR/include"
export LDFLAGS="-L $BAMTOOLS_DIR/lib -Wl,-rpath,$BAMTOOLS_DIR/lib"
make
```

## Command Line Usage

The basic syntax for executing a differential expression analysis is:

```bash
./bin/diffexpIR -g <annotation.gtf> -o <output_prefix> -d <tpm_dir> -p <group1_prefix,group2_prefix>
```

### Required Arguments
- `-g`: Path to the GTF annotation file used in the upstream quantification.
- `-o`: Desired name for the output results file.
- `-d`: The directory containing the output files generated by TPMCalculator.
- `-p`: A comma-separated string defining the prefixes for the two groups of samples to be compared (e.g., `control,treated`).

### Statistical and Filtering Options
- `-s`: Statistical method. Choose `ttest` (default) or `wilcox` for the Wilcoxon rank-sum test.
- `-f`: Minimum fold change threshold for filtering results (default: 2.0).
- `-v`: Minimum P-Value threshold for filtering (default: 1.0E-6).
- `-r`: Minimum fold change between the intron and its neighboring exons (default: -1.0). This is useful for ensuring the retention is significant relative to the host gene's expression.

### Annotation Customization
- `-k`: Gene key to extract from the GTF file (default: `gene_id`).
- `-t`: Transcript key to extract from the GTF file (default: `transcript_id`).
- `-c`: Minimum allowed size for an intron. **Expert Tip**: It is highly recommended to set this value to your specific sequencing read length (default is 16).

## Output Interpretation

The output file provides a comprehensive table of intronic coordinates and expression metrics. Key columns include:
- **Log2TPMRatio**: The magnitude of change between group 1 and group 2.
- **PValue / minusLog10PValue**: Statistical significance of the differential retention.
- **RValue_1 / RValue_2**: The ratio `TPM_Intron_i / (TPM_Exon_i-1 + TPM_Exon_i+1)`. This metric normalizes the intron retention level against the expression of the flanking exons, helping to distinguish true retention from overall gene over-expression.

## Best Practices

1. **Input Consistency**: Ensure the GTF file used with DiffExpIR is identical to the one used during the TPMCalculator run to avoid coordinate mismatches.
2. **Sample Grouping**: The `-p` flag relies on the file prefixes in the TPMCalculator directory. Ensure your files are named consistently (e.g., `control_1.tpm`, `control_2.tpm`, `treated_1.tpm`) so the tool can correctly aggregate samples into groups.
3. **Read Length Alignment**: Always adjust the `-c` parameter to match your library's read length. Using the default (16) on 150bp data may lead to the inclusion of truncated or poorly supported intronic regions.
