# clipseq

## Overview
nf-core/clipseq is a bioinformatics pipeline designed for the analysis of cross-linking and immunoprecipitation (CLIP) sequencing data. It addresses the challenge of identifying precise RNA-protein interaction sites by processing raw reads through a standardized workflow of quality control, mapping, and specialized peak calling.

The pipeline accepts raw FastQ data and produces comprehensive results including aligned BAM files, crosslink site identifications, genomic coverage tracks, and motif enrichment analysis. It supports multiple peak calling algorithms to accommodate different CLIP-seq variants and experimental designs, providing a unified framework for studying RNA-protein binding dynamics.

## Data preparation
The pipeline requires raw sequencing data and reference genome information. Input FastQ files can be single-end or paired-end.

*   **Input Reads**: Specified via the `--input` parameter. The path must be enclosed in quotes and contain at least one `*` wildcard character. For paired-end data, use the `{1,2}` notation.
*   **Reference Genome**: You must provide a genome FASTA file (`--fasta`) or use an iGenomes identifier (`--genome`). A GTF annotation file (`--gtf`) is also required for regional distribution analysis.
*   **Optional References**: A small RNA FASTA (`--smrna_fasta`) can be provided for pre-mapping to rRNA/tRNA. If using the iCount peak caller, an iCount segment file (`--segment`) is required.
*   **UMIs**: If your data contains UMIs, you can configure their handling using `--move_umi`, `--deduplicate`, and `--umi_separator`.

Example input structure:
```bash
# Single-end data
--input 'path/to/data/*.fastq.gz'

# Paired-end data
--input 'path/to/data/sample_*_{1,2}.fastq.gz'
```

## How to run
The pipeline is executed using Nextflow with appropriate profiles for software management (Docker, Singularity, or Conda).

```bash
# Run with a minimal test dataset
nextflow run nf-core/clipseq -profile test,<docker/singularity/conda>

# Run with your own data
nextflow run nf-core/clipseq \
    -profile <docker/singularity/conda> \
    --input 'path/to/data/*_R1.fastq.gz' \
    --fasta 'path/to/reference.fasta' \
    --gtf 'path/to/annotation.gtf' \
    --peakcaller 'icount' \
    --outdir ./results
```

Key parameters include:
*   `-profile`: Specifies the software container or environment (e.g., `docker`, `singularity`).
*   `--peakcaller`: Choose one or more callers from `icount`, `paraclu`, `pureclip`, or `piranha`.
*   `--motif`: Set to `true` to perform motif detection using DREME.
*   `-resume`: Use this flag to restart a pipeline from the last successful step.
*   `-r`: Pin a specific version of the pipeline (e.g., `-r 1.0.0`).

## Outputs
Results are saved in the directory specified by the `--outdir` parameter (default is `./results`).

*   **MultiQC**: The `MultiQC/` directory contains the primary HTML report summarizing FastQC, library complexity (Preseq), mapping statistics (STAR), and peak calling comparisons.
*   **Alignments and Tracks**: BAM files from STAR and bedgraph coverage tracks generated by BEDTools.
*   **Analysis Results**: Crosslink sites identified by BEDTools, peaks called by the selected algorithm(s), and motif enrichment results if enabled.
*   **Pipeline Info**: Execution metrics and logs are stored in the `pipeline_info/` subfolder.

For a complete description of all output files, refer to the official documentation at `https://nf-co.re/clipseq/output`.
