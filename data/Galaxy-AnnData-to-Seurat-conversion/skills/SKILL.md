# anndata-to-seurat-conversion

## Overview

This workflow facilitates the conversion of single-cell RNA sequencing data from the AnnData format (commonly used in Python-based Scanpy pipelines) to the Seurat format (standard for R-based analysis). It automates the manual steps required to bridge these two ecosystems, taking a single AnnData file as input and restructuring its internal components for compatibility.

The process begins by using the [anndata_inspect](https://toolshed.g2.bx.psu.edu/repos/iuc/anndata_inspect/anndata_inspect/0.10.3+galaxy0) tool to extract the observation metadata (`obs`) and the expression matrix (`X`). The matrix is then processed through a [transpose](https://toolshed.g2.bx.psu.edu/repos/iuc/datamash_transpose/datamash_transpose/1.8+galaxy1) step and formatted via [DropletUtils](https://toolshed.g2.bx.psu.edu/repos/iuc/dropletutils/dropletutils/1.10.0+galaxy2) to prepare the data for ingestion into R.

The final output is a Seurat-compatible object generated by the [Seurat Read10x](https://toolshed.g2.bx.psu.edu/repos/ebi-gxa/seurat_read10x/seurat_read10x/3.2.3+galaxy0) tool, alongside the intermediate metadata and matrix files. This workflow is a [Galaxy Training Network (GTN)](https://training.galaxyproject.org/) resource and is provided under a [CC-BY-4.0](https://creativecommons.org/licenses/by/4.0/) license.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | AnnData | data_input | Input file in AnnData format |


Ensure your input is a valid AnnData file in `.h5ad` format, as the workflow specifically extracts the expression matrix and observation metadata for conversion into Seurat-compatible structures. While the workflow is designed for individual datasets, you can utilize dataset collections to efficiently process multiple single-cell experiments in parallel. Refer to the `README.md` for comprehensive details on specific tool parameters and required metadata formatting. You can also use `planemo workflow_job_init` to generate a `job.yml` for automated execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run anndata-to-seurat-conversion.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run anndata-to-seurat-conversion.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run anndata-to-seurat-conversion.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init anndata-to-seurat-conversion.ga -o job.yml`
- Lint: `planemo workflow_lint anndata-to-seurat-conversion.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.
