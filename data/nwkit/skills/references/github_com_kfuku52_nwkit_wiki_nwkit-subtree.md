# nwkit subtree

[Jump to bottom](#wiki-pages-box)

## Overview

`nwkit subtree` generates a subtree Newick file. The subtree is defined by the most recent common ancestor (MRCA) of the specified tips.

* `--left_leaf` and `--right_leaf` specify two tips whose MRCA defines the subtree
* `--leaves` specifies a comma-separated list of tips (alternative to `--left_leaf`/`--right_leaf`)
* `--orthogroup yes` extracts orthogroup(s) containing all seed genes, using duplication confidence scores to identify orthologous clades

For `--orthogroup yes`, species-aware parsing expects leaf labels of the form `GENUS_SPECIES` or `GENUS_SPECIES_SUFFIX` by default. Fields after the species epithet are ignored when assigning species identities to tips. If your labels use a different convention, provide `--species_regex`.

## Examples

### 1. Extract subtree by two tip names

#### Command

```
nwkit subtree --left_leaf a --right_leaf c --infile input.nwk --outfile output.nwk
```

#### input.nwk

```
(((a:1,b:1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

![image](https://user-images.githubusercontent.com/19270515/224156098-edd6af67-601b-433d-940d-ffe6f457cdf5.png)

#### output.nwk

```
((a:1,b:1):1,c:1):1;
```

![image](https://user-images.githubusercontent.com/19270515/224156057-22294e1f-f8e2-446a-90a7-8cf66bc3a1f3.png)

The subtree rooted at the MRCA of `a` and `c` is extracted, which includes `a`, `b`, and `c`.

### 2. Extract subtree by a list of tips

Instead of specifying two tips, you can provide a comma-separated list of tip names with `--leaves`.

#### Command

```
nwkit subtree --leaves a,b,c --infile input.nwk --outfile output.nwk
```

#### input.nwk

```
(((a:1,b:1):1,c:1):1,((d:1,e:1),f:1):1):0;
```

#### output.nwk

```
((a:1,b:1)1:1,c:1)1:1;
```

The MRCA of all specified tips (`a`, `b`, `c`) defines the subtree.

## Custom species parsing

```
nwkit subtree --leaves GENE1,GENE2 --orthogroup yes --infile input.nwk --outfile output.nwk \
  --species_regex '^([A-Za-z]+)\.([A-Za-z]+)\|'
```
