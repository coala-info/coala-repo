# bioconductor-peacoqc

## Overview

PeacoQC (Peak-based selection of high-quality cytometry data) is an R package designed for automated quality control of flow and mass cytometry data. It identifies unstable measurements by monitoring channel peaks over time and removes outliers based on Isolation Tree (IT) and Median Absolute Deviation (MAD) analyses. It is particularly effective at cleaning data before downstream analysis like clustering or dimensionality reduction.

## Core Workflow

### 1. Data Loading and Pre-processing
PeacoQC works with `flowFrame` objects from the `flowCore` package.

```r
library(PeacoQC)
library(flowCore)

# Load data
ff <- read.FCS("path/to/file.fcs")

# Define channels for QC (indices or names)
channels <- c(1, 3, 5:14)

# 1. Remove Margins (Perform BEFORE compensation/transformation)
ff <- RemoveMargins(ff = ff, channels = channels, output = "frame")

# 2. Compensate and Transform
ff <- compensate(ff, keyword(ff)$SPILL)
ff <- transform(ff, estimateLogicle(ff, colnames(keyword(ff)$SPILL)))
```

### 2. Running Quality Control
The `PeacoQC` function is the primary interface for cleaning data.

```r
peacoqc_res <- PeacoQC(
    ff = ff,
    channels = channels,
    determine_good_cells = "all",
    save_fcs = TRUE,
    plot = TRUE,
    output_directory = "QC_Results"
)

# Access the cleaned flowFrame
cleaned_ff <- peacoqc_res$FinalFF
```

### 3. Mass Cytometry (CyTOF) Specifics
For mass cytometry, adjust parameters to account for data sparsity and longer acquisition times.

```r
peacoqc_res <- PeacoQC(
    ff = ff,
    channels = channels,
    remove_zeros = TRUE,   # Important for CyTOF
    IT_limit = 0.6,        # Often ranges 0.6 - 0.65 for CyTOF
    time_units = 50000     # Adjust to avoid picket-fencing in plots
)
```

## Parameter Tuning

- `IT_limit`: Increasing this value makes the Isolation Tree analysis more strict (removes more cells). Default is 0.55.
- `MAD`: Controls the Median Absolute Deviation strictness. Higher values are less strict. Default is 6.
- `remove_zeros`: Set to `TRUE` for mass cytometry to ignore zero values in peak calculations.

## Visualization and Reporting

### Heatmap Overview
After running PeacoQC on multiple files, use the generated report to visualize the percentage of removed events across the experiment.

```r
# Path to the report generated by PeacoQC()
report_path <- "QC_Results/PeacoQC_report.txt"

PeacoQCHeatmap(
    report_location = report_path,
    show_values = TRUE,
    latest_tests = TRUE
)
```

### Plotting Without Cleaning
To inspect peaks or raw event rates without performing automated removal:

```r
PlotPeacoQC(
    ff, 
    channels, 
    display_peaks = TRUE, 
    prefix = "Inspection_"
)
```
