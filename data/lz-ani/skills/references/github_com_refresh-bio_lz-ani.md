The tool uses Lempel-Ziv parsing, achieving high sensitivity in discerning matched and mismatched nucleotides, thereby enabling accurate determination of ANI. Its efficiency stems from a simplified indel handling model, making LZ-ANI magnitudes faster than alignment-based tools (e.g., BLASTn, MegaBLAST) while maintaining accuracy comparable to the most sensitive BLASTn searches. LZ-ANI is a key component of Vclust , a tool for comparing and clustering virus genomes. Although optimized for virus genomes, LZ-ANI's parameters can be customized for longer genomes, such as those of bacteria and archaea. Sequence similarity measures LZ-ANI offers six similarity measures between two genomic sequences: ANI : The number of identical bases across local alignments divided by the total length of the alignments. Global ANI (gANI) : The number of identical bases across local alignments divided by the length of the query/reference genome. Total ANI (tANI) : The number of identical bases between query-reference and referece-query genomes divided by the sum length of both genomes. Coverage (alignment fraction) : The proportion of the query/reference sequence aligned with the reference/query sequence. Number of local alignments : The count of individual alignments found between the sequences. Ratio between query and reference genome lengths : A measure comparing the lengths of the two genomes. Installation Prebuilt binaries The easiest way to get started is by downloading the prebuilt binaries from the Releases page. Select your platform and download the tool. Building from source Alternatively, you can clone the repository and build LZ-ANI from source. You will need a modern C++17 compiler (e.g., g++10 or newer) and NASM, which can often be installed via a system package manager: sudo apt install nasm # Ubuntu Then, clone the repository and build the LZ-ANI binary: git clone --recurse-submodules https://github.com/refresh-bio/lz-ani cd lz-ani &amp;&amp; make -j Quick start Perform pairwise alignment among all genome pairs within a single multifasta file: ./lz-ani all2all --in-fasta example/multifasta.fna --out ani.tsv Perform pairwise alignment for all genome files in a directory: ./lz-ani all2all --in-dir example/fna --out ani.tsv Usage lz-ani &lt; mode &gt; [options] Modes Currently, LZ-ANI operates exclusively in the all2all mode, where sequence similarities are computed for all pairs of input sequences. It supports filtering the list of pairs, with future updates to introduce additional modes. Input files specification: --in-fasta &lt;file_name&gt; — FASTA file (for multisample-fasta mode) --in-txt &lt;file_name&gt; — text file with FASTA file names --in-dir &lt;path&gt; — directory with FASTA files --multisample-fasta &lt;bool&gt; — multi sample FASTA input (default: true) --flt-kmerdb &lt;fn&gt; &lt;float&gt; — filtering file (kmer-db output) and threshold Output files specification: -o, --out &lt;file_name&gt; — output file name --out-ids &lt;file_name&gt; — output file name for ids file (optional) --out-in-percent &lt;bool&gt; — output in percent (default: false) --out-type &lt;type&gt; — one of: tsv — two tsv files with: results defined by --out-format and sequence ids (default) single-txt — combined results in single txt file --out-format &lt;type&gt; — comma-separated list of values: id1 , id2 , idx1 , idx2 , len1 , len2 , tani , gani , ani , cov , len_ratio , nt_match , nt_mismatch , num_alns you can include also meta-names: complete=idx1,idx2,id1,id2,tani,gani,ani,cov,num_alns,len_ratio,len1,len2,nt_match,nt_mismatch lite=idx1,idx2,tani,gani,ani,cov,num_alns,len_ratio standard=idx1,idx2,id1,id2,tani,gani,ani,cov,num_alns,len_ratio (default: standard) --out-alignment &lt;file_name&gt; — output file name for alignments (optional) --out-filter &lt;par&gt; &lt;float&gt; — store only results with &lt;par&gt; (can be: tani , gani , ani , cov ) at least &lt;float&gt; ; can be used multiple times LZ-parsing options: -a, --mal &lt;int&gt; — min. anchor length (default: 11) -s, --msl &lt;int&gt; — min. seed length (default: 7) -r, --mrd &lt;int&gt; — max. dist. between approx. matches in reference (default: 40) -q, --mqd &lt;int&gt; — max. dist. between approx. matches in query (default: 40) -g, --reg &lt;int&gt; — min. considered region length (default: 35) --aw &lt;int&gt; — approx. window length (default: 15) --am &lt;int&gt; — max. no. of mismatches in approx. window (default: 7) --ar &lt;int&gt; — min. length of run ending approx. extension (default: 3) Other options: -t, --threads &lt;int&gt; — no of threads; 0 means auto-detect (default: 0) -V, --verbose &lt;int&gt; — verbosity level (default: 1) Input LZ-ANI accepts a single FASTA file containing genomic sequences ( example ) or a directory of FASTA files (one genome per file) ( example ).
