# kfilt

## Overview

kfilt is a high-performance bioinformatics tool designed to subset sequencing reads based on their similarity to a set of target k-mers. Unlike simple exact-match filters, kfilt utilizes a hybrid architecture featuring BK-trees (Burkhard-Keller trees), allowing it to identify reads containing k-mers that match the reference within a specific Hamming distance (mismatches). This makes it particularly effective for extracting reads belonging to specific alleles, pangenome variants, or filtering unmapped reads from BAM/CRAM files against a known database of unique k-mers.

## Core Workflows

### 1. Building an Index
Before filtering, you must convert a list of k-mers into a BK-tree index. The input should be a text file containing one k-mer per line (typically generated by tools like `meryl`).

```bash
kfilt build -k unique_kmers.txt -o unique.bktree.idx -K 31
```
- `-k`: Input k-mer text file.
- `-o`: Path for the generated index file.
- `-K`: K-mer size (default is 31).

### 2. Filtering Reads
The `filter` command processes sequencing data against the built index.

**Paired-End (Separate Files):**
```bash
kfilt filter -1 R1.fq.gz -2 R2.fq.gz -i unique.idx -o filtered.fq.gz -z -n 5 -m 1
```

**Interleaved or Single-End:**
```bash
kfilt filter -I interleaved.fa.gz -i unique.idx -o filtered.fa.gz -f fasta -z
```

## Expert Tips and CLI Patterns

### Matching Logic for Pairs
- **Default**: A read pair is kept if the *combined* number of k-mer matches across both R1 and R2 meets the `-n` threshold.
- **Strict Mode**: Use `--both-match` to require that *each* read in the pair independently meets the `-n` threshold.

### Tuning Sensitivity vs. Speed
- **Hamming Distance (`-m`)**: 
  - `-m 0`: Exact matches only. Fastest performance.
  - `-m 1`: Allows 1 mismatch. Good balance for sequencing errors.
  - `-m 2+`: Significantly increases computational overhead. Use only when high divergence is expected.
- **Minimum Matches (`-n`)**: Increase this value to reduce false positives from random k-mer hits. For a standard 150bp read, `-n 5` to `-n 10` is a common starting point.

### Performance Optimization
- **Threading**: Use `-t` to specify CPU cores. kfilt scales well across multiple cores.
- **Compression**: Use `-z` to output gzipped files directly. You can control the compression level (1-9) with `-Z`.
- **Format Detection**: kfilt automatically detects FASTA vs FASTQ for input, but use `-f` to explicitly set the output format if you wish to convert (e.g., FASTQ input to FASTA output).

### Debugging and Statistics
Use the `-v` (verbose) flag to generate a per-read TSV report. This is essential for troubleshooting why specific reads are being filtered or kept.
```bash
kfilt filter ... -v stats.tsv
```
The resulting file includes `total_kmers`, `matched_kmers`, and the `best_hamming_dist` found for every read.



## Subcommands

| Command | Description |
|---------|-------------|
| kfilt build | Build a fast hybrid index (Bloom filter + hash table + BK-tree) for efficient k-mer matching |
| kfilt filter | Filter FASTA/FASTQ reads using hybrid index with Bloom filter quick rejection. |
