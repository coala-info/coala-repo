## Overview

`nwkit image` retrieves representative media for species in a Newick tree and writes image files plus machine-readable metadata.

Leaf names are interpreted as `GENUS_SPECIES[_...]` by default. If your leaf labels do not follow that convention, either provide `--species_regex` or a TSV mapping with `--name_tsv`.

## Basic example

### Command

```
nwkit image -i tree.nwk --out_dir species_images
```

### Output files

* `images/`: downloaded image files
* `manifest.tsv`: one row per retained image
* `unmatched.tsv`: unresolved taxa and filtering/download failures
* `ATTRIBUTION.md`: human-readable attribution and license summary

## Providers

Supported providers are:

* `phylopic`
* `bioicons`
* `inaturalist`
* `wikimedia`
* `gbif`
* `eol`
* `idigbio`
* `openverse`
* `ncbi`

The default provider order depends on `--style`.

* `--style auto`: `phylopic,bioicons,inaturalist,wikimedia,gbif,eol,idigbio,openverse,ncbi`
* `--style photo`: `inaturalist,wikimedia,gbif,eol,idigbio,openverse,ncbi`
* `--style silhouette`: `phylopic,bioicons,wikimedia`

When `ncbi` appears after other providers, it is used as a lazy fallback because it may download the NCBI taxonomy image table.

## License and fallback controls

### Command

```
nwkit image -i tree.nwk --out_dir species_images \
  --license_max cc-by-sa \
  --allow_nd no \
  --fallback_rank genus
```

* `--license_max` sets the strongest restriction level allowed
* `--allow_nd` controls whether ND licenses are accepted
* `--fallback_rank none|genus|family` allows matching above species rank when exact species matches are unavailable

## Provider preference examples

Prefer silhouettes:

```
nwkit image -i tree.nwk --out_dir species_images --style silhouette
```

Prefer specific providers and include NCBI as the last fallback:

```
nwkit image -i tree.nwk --out_dir species_images \
  --source phylopic,bioicons,inaturalist,wikimedia,gbif,eol,idigbio,openverse,ncbi
```

## Post-processing

The following options require the optional image dependencies:

```
pip install "nwkit[image]"
```

* `--output_format original|png|jpg`: normalize the final file format
* `--max_edge INT`: downscale the longest edge to this maximum size
* `--canvas none|square`: pad to a square canvas after trimming and resizing
* `--background white|transparent`: background color used when `--canvas square` is requested
* `--trim off|white|transparent|semantic`: trim white margins, transparent margins, or a best-effort foreground subject
* `--trim_shape bbox|square`: keep the trimmed bounding box or center-crop the trimmed result to a square

Normalize mixed inputs into 1024 px PNG files with transparent square canvases:

```
nwkit image -i tree.nwk --out_dir species_images \
  --output_format png \
  --max_edge 1024 \
  --canvas square \
  --background transparent \
  --trim semantic
```

If you want a square crop with no padding, add `--trim_shape square`. This uses a centered square crop and may clip elongated subjects.

`--trim semantic` is a Pillow-only heuristic. It uses alpha when available, otherwise it estimates the main foreground region from the largest border-separated component. It works best when the subject is visually separated from the background.

## Name mapping

If tree labels do not contain recognizable species names even after `--species_regex`, prepare a TSV with `leaf_name` and `species_name` columns.

### names.tsv

```
leaf_name	species_name
sp1	Oryza sativa
sp2	Arabidopsis thaliana
```

### Command

```
nwkit image -i tree.nwk --out_dir species_images --name_tsv names.tsv
```

## Notes

* `--download_dir` can be used to share the ETE4 taxonomy cache and downloaded media across runs
* `manifest.tsv` records provider, match rank, license, attribution, source URL, and local output path
* `--fail_on_missing yes` turns unresolved taxa into a non-zero exit status

## Custom species parsing

For labels such as `Homo.sapiens|GENE1`, use:

```
nwkit image -i tree.nwk --out_dir species_images \
  --species_regex '^([A-Za-z]+)\.([A-Za-z]+)\|'
```

When the regex contains capture groups, non-empty captured groups are joined with underscores and then normalized to species names with spaces (for example, `Homo sapiens`).
