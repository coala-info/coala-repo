read_dir(&#39;/path/to/dir&#39;); DESCRIPTION This module provides subs that allow you to read or write entire files with one simple call. They are designed to be simple to use, have flexible ways to pass in or get the file contents and to be very efficient. There is also a sub to read in all the files in a directory. WARNING - PENDING DOOM Although you technically can , do NOT use this module to work on file handles, pipes, sockets, standard IO, or the DATA handle. These are features implemented long ago that just really shouldn&#39;t be abused here. Be warned: this activity will lead to inaccurate encoding/decoding of data. All further mentions of actions on the above have been removed from this documentation and that feature set will likely be deprecated in the future. In other words, if you don&#39;t have a filename to pass, consider using the standard do { local $/; &lt;$fh&gt; } , or Data::Section / Data::Section::Simple for working with __DATA__ . FUNCTIONS File::Slurp implements the following functions. append_file use File::Slurp qw(append_file write_file); my $res = append_file(&#39;/path/file&#39;, &quot;Some text&quot;); # same as my $res = write_file(&#39;/path/file&#39;, {append =&gt; 1}, &quot;Some text&quot;); The append_file function is simply a synonym for the &quot;write_file&quot; in File::Slurp function, but ensures that the append option is set. edit_file use File::Slurp qw(edit_file); # perl -0777 -pi -e &#39;s/foo/bar/g&#39; /path/file edit_file { s/foo/bar/g } &#39;/path/file&#39;; edit_file sub { s/foo/bar/g }, &#39;/path/file&#39;; sub replace_foo { s/foo/bar/g } edit_file \&amp;replace_foo, &#39;/path/file&#39;; The edit_file function reads in a file into $_ , executes a code block that should modify $_ , and then writes $_ back to the file. The edit_file function reads in the entire file and calls the code block one time. It is equivalent to the -pi command line options of Perl but you can call it from inside your program and not have to fork out a process. The first argument to edit_file is a code block or a code reference. The code block is not followed by a comma (as with grep and map ) but a code reference is followed by a comma. The next argument is the filename. The next argument(s) is either a hash reference or a flattened hash, key =&gt; value pairs. The options are passed through to the &quot;write_file&quot; in File::Slurp function. All options are described there. Only the binmode and err_mode options are supported. The call to &quot;write_file&quot; in File::Slurp has the atomic option set so you will always have a consistent file. edit_file_lines use File::Slurp qw(edit_file_lines); # perl -pi -e &#39;$_ = &quot;&quot; if /foo/&#39; /path/file edit_file_lines { $_ = &#39;&#39; if /foo/ } &#39;/path/file&#39;; edit_file_lines sub { $_ = &#39;&#39; if /foo/ }, &#39;/path/file&#39;; sub delete_foo { $_ = &#39;&#39; if /foo/ } edit_file \&amp;delete_foo, &#39;/path/file&#39;; The edit_file_lines function reads each line of a file into $_ , and executes a code block that should modify $_ . It will then write $_ back to the file. It is equivalent to the -pi command line options of Perl but you can call it from inside your program and not have to fork out a process. The first argument to edit_file_lines is a code block or a code reference. The code block is not followed by a comma (as with grep and map ) but a code reference is followed by a comma. The next argument is the filename. The next argument(s) is either a hash reference or a flattened hash, key =&gt; value pairs. The options are passed through to the &quot;write_file&quot; in File::Slurp function. All options are described there. Only the binmode and err_mode options are supported.
