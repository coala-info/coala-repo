In addition to URI expansion, it also supports URI extraction (used by Google Cloud Core and Google Cloud Client Library ). Note Due to the deprecation of implictly nullable parameter types in PHP 8.4 , we must introduce breaking change by adding explicit nullable types ( ?T ) which requires PHP 7.1+. As a result, version 0.4.0 and later will no longer support PHP versions below 8.1. Usage Expansion A very simple usage (string expansion). &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); $ uri -&gt; expand ( ' /{username}/profile ' , [ ' username ' =&gt; ' john ' ]); &gt;&gt; ' /john/profile ' Rize\UriTemplate supports all Expression Types and Levels specified by RFC6570. &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); $ uri -&gt; expand ( ' /search/{term:1}/{term}/{?q*,limit} ' , [ ' term ' =&gt; ' john ' , ' q ' =&gt; [ ' a ' , ' b ' ], ' limit ' =&gt; 10 , ]) &gt;&gt; ' /search/j/john/?q=a&amp;q=b&amp;limit=10 ' / Path segment expansion &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); $ uri -&gt; expand ( ' http://{host}{/segments*}/{file}{.extensions*} ' , [ ' host ' =&gt; ' www.host.com ' , ' segments ' =&gt; [ ' path ' , ' to ' , ' a ' ], ' file ' =&gt; ' file ' , ' extensions ' =&gt; [ ' x ' , ' y ' ], ]); &gt;&gt; ' http://www.host.com/path/to/a/file.x.y ' Rize\UriTemplate accepts base-uri as a 1st argument and default params as a 2nd argument. This is very useful when you're working with API endpoint. Take a look at real world example. &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate ( ' https://api.twitter.com/{version} ' , [ ' version ' =&gt; 1.1 ]); $ uri -&gt; expand ( ' /statuses/show/{id}.json ' , [ ' id ' =&gt; ' 210462857140252672 ' ]); &gt;&gt; https: //api.twitter.com/1.1/statuses/show/210462857140252672.json Extraction It also supports URI Extraction (extract all variables from URI). Let's take a look at the example. &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate ( ' https://api.twitter.com/{version} ' , [ ' version ' =&gt; 1.1 ]); $ params = $ uri -&gt; extract ( ' /search/{term:1}/{term}/{?q*,limit} ' , ' /search/j/john/?q=a&amp;q=b&amp;limit=10 ' ); &gt;&gt; print_r ( $ params ); ( [term: 1 ] =&gt; j [term] =&gt; john [q] =&gt; Array ( [ 0 ] =&gt; a [ 1 ] =&gt; b ) [limit] =&gt; 10 ) Note that in the example above, result returned by extract method has an extra keys named term:1 for prefix modifier. This key was added just for our convenience to access prefix data. strict mode &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); $ uri -&gt; extract ( $ template , $ uri , $ strict = false ) Normally extract method will try to extract vars from a uri even if it's partially matched. For example &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); $ params = $ uri -&gt; extract ( ' /{?a,b} ' , ' /?a=1 ' ) &gt;&gt; print_r ( $ params ); ( [a] =&gt; 1 [b] =&gt; null ) With strict mode , it will allow you to extract uri only when variables in template are fully matched with given uri. Which is useful when you want to determine whether the given uri is matched against your template or not (in case you want to use it as routing service). &lt;?php use Rize \ UriTemplate ; $ uri = new UriTemplate (); // Note that variable `b` is absent in uri $ params = $ uri -&gt; extract ( ' /{?a,b} ' , ' /?a=1 ' , true ); &gt;&gt;&gt; null // Now we give `b` some value $ params = $ uri -&gt; extract ( ' /{?a,b} ' , ' /?a=1&amp;b=2 ' , true ); &gt;&gt;&gt; print_r ( $ params ) ( [a] =&gt; 1 [b] =&gt; 2 ) Array modifier % By default, RFC 6570 only has 2 types of operators : and * . This % array operator was added to the library because current spec can't handle array style query e.g. list[]=a or key[user]=john . Example usage for % modifier &lt;?php $ uri -&gt;expand( ' {?list%,keys%} ' , [ ' list ' =&gt; [ ' a ' , ' b ' , ), ' keys ' =&gt; [ ' a ' =&gt; 1 , ' b ' =&gt; 2 , ), ]); // '?list[]=a&amp;list[]=b&amp;keys[a]=1&amp;keys[b]=2' &gt;&gt; ' ?list%5B%5D=a&amp;list%5B%5D=b&amp;keys%5Ba%5D=1&amp;keys%5Bb%5D=2 ' // [] get encoded to %5B%5D i.e. '?list[]=a&amp;list[]=b&amp;keys[a]=1&amp;keys[b]=2' $ params = $ uri -&gt; extract ( ' {?list%,keys%} ' , ' ?list%5B%5D=a&amp;list%5B%5D=b&amp;keys%5Ba%5D=1&amp;keys%5Bb%5D=2 ' , ) &gt;&gt; print_r ( $ params ); ( [ list ] =&gt; Array ( [ 0 ] =&gt; a [ 1 ] =&gt; b ) [keys] =&gt; Array ( [a] =&gt; 1 [b] =&gt; 2 ) ) Installation Using composer { "require": { "rize/uri-template": "~0.3" } } Changelogs 0.2.0 Add a new modifier % which allows user to use list[]=a&amp;list[]=b query pattern. 0.2.1 Add nested array support for % modifier 0.2.5 Add strict mode support for extract method 0.3.0 Improve code quality + RFC3986 support for extract method by @Maks3w 0.3.1 Improve extract method to parse two or more adjacent variables separated by dot by @corleonis 0.4.0 Fixes the deprecation of implicitly nullable parameter types introduced in PHP 8.4.
