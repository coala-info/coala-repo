# gmap-fusion

## Overview

GMAP-fusion is a specialized bioinformatic utility designed to discover gene fusions within transcript sequences generated by de novo assemblers like Trinity, Oases, or DISCASM. Unlike tools that find fusions directly from raw reads, GMAP-fusion leverages reconstructed transcript sequences to identify chimeric alignments via GMAP. It then validates these candidates by mapping the original raw reads back to the fusion junctions using Bowtie2 and applying rigorous filters through FusionFilter to minimize false positives.

## Core Workflow

To run a standard fusion discovery analysis, you must provide the assembled transcripts, the original paired-end reads, and a compatible genome resource library.

### Basic Execution

```bash
GMAP-fusion --transcripts Trinity.fasta \
            --genome_lib_dir /path/to/ctat_genome_lib_bundle \
            --left_fq reads_1.fastq \
            --right_fq reads_2.fastq \
            --CPU 8 \
            --output GMAP_fusion_outdir
```

### Key Parameters and Tuning

*   **Sensitivity Controls**:
    *   `--min_J`: Minimum number of junction-spanning fragments (default: 1). Increase this to reduce noise in high-depth datasets.
    *   `--min_sumJS`: Minimum combined count of junction and spanning fragments (default: 2).
    *   `--min_novel_junction_support`: Minimum junction reads required for non-reference (novel) exon-exon junctions (default: 3).
*   **Alignment Specifics**:
    *   `--min_chim_len`: Minimum length for a chimeric alignment segment (default: 30).
    *   `--split_breakpoint_extend_length`: Length to extend split sequences beyond the breakpoint for quality assessment (default: 25).
*   **Filtering**:
    *   `--max_promiscuity`: Maximum number of different partners allowed for a single gene before it is flagged as a likely artifact (default: 3).

## Output Interpretation

The primary result is located at `[output_dir]/GMAP-fusion.final`. This tab-delimited file includes:

| Column | Description |
| :--- | :--- |
| #fusion_name | The identified gene pair (e.g., RPS6KB1--SNF8). |
| J | Number of fragments aligning directly across the fusion junction. |
| S | Number of spanning fragments (pairs flanking the junction). |
| trans_acc | The specific transcript contig ID from the assembly. |
| junction_type | Indicates if the fusion occurs at reference splice sites (ONLY_REF_SPLICE). |

## Expert Tips

1.  **Genome Library**: GMAP-fusion requires a specific genome library directory. It is highly recommended to use the pre-built CTAT (Cloud-To-Analyse-Trinity) genome resource bundles provided by the Trinity project.
2.  **False Positive Mitigation**: The tool automatically runs FusionFilter. If you see many fusions involving genes with high sequence similarity (paralogs), they are likely being filtered. You can adjust the E-value threshold for homology searches using the `-E` flag.
3.  **Input Preparation**: Ensure your transcript FASTA headers do not contain complex characters that might break downstream Perl parsing. Trinity-style headers are natively supported.
4.  **Resource Allocation**: GMAP is the most computationally expensive part of the pipeline. Always scale the `--CPU` parameter to match your available hardware to speed up the initial chimeric alignment phase.
