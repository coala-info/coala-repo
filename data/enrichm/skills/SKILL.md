# enrichm

## Overview

EnrichM is a specialized toolbox designed for the functional analysis of large sets of population genomes. It bridges the gap between genome assembly and biological interpretation by providing a streamlined workflow to annotate sequences, classify metabolic potential based on reference pathways (like KEGG), and identify statistically significant functional differences between user-defined groups of genomes. It is particularly useful for researchers working with MAGs who need to understand the metabolic niches and functional diversity of microbial communities.

## Database Management

Before running any analysis, EnrichM requires a large backend database (~5.7GB) containing HMMs and reference sequences (Pfam, TIGRfam, UniRef100, KoFamKOALA).

- **Initialize/Update Database**: Run `enrichm data` to download or update the reference data.
- **Custom Location**: If you do not want the database in your home directory, set the environment variable:
  `export ENRICHM_DB=/path/to/database_folder/`
- **Uninstall**: Use `enrichm data --uninstall` to remove the local database.

## Core Workflow and CLI Patterns

### 1. Functional Annotation
The `annotate` subcommand is the primary entry point. It generates .gff files and frequency matrices.
- **Basic Usage**: `enrichm annotate --genomes <input_directory> --out <output_directory> --threads <int>`
- **Supported Annotations**: KO (KEGG Orthology), PFAM, TIGRFAM, and CAZY (via dbCAN).
- **Output**: Look for the `ko_frequency_table.tsv` or `pfam_frequency_table.tsv` in the output folder; these are required for downstream subcommands.

### 2. Metabolic Classification
The `classify` subcommand determines which KEGG modules are complete based on the annotations.
- **Basic Usage**: `enrichm classify --ko_matrix <ko_frequency_table.tsv> --out <output_directory>`
- **Utility**: This transforms raw counts into biological context by identifying entire pathways present in the genomes.

### 3. Enrichment Analysis
Use `enrichment` to find functional signatures that differentiate groups of genomes (e.g., genomes from different environments or different clades).
- **Requirement**: A metadata file (TSV) mapping genome IDs to groups.
- **Basic Usage**: `enrichm enrichment --ko_matrix <ko_table.tsv> --metadata <metadata.tsv> --out <output_directory>`
- **Analysis**: It performs statistical tests to identify modules or protein families significantly enriched within specific groups.

### 4. Network Reconstruction and Exploration
- **Pathway Mapping**: `enrichm pathway` generates Cytoscape-compatible files for metabolic networks.
  `enrichm pathway --ko_matrix <ko_table.tsv> --out <output_directory>`
- **Compound Exploration**: `enrichm explore` starts from a specific compound ID and identifies possible reactions based on the enzymes present in your genomes.

## Expert Tips and Best Practices

- **Input Suffixes**: Ensure your genome files have consistent extensions (e.g., .fasta, .fna). Recent updates require genomes to end in expected suffixes for proper directory processing.
- **Resource Allocation**: Annotation is computationally intensive. Always specify `--threads` to utilize multi-core environments, especially when processing hundreds of MAGs.
- **Matrix Inputs**: Most downstream commands (`classify`, `enrichment`, `pathway`) require the frequency matrices generated by the `annotate` step. Keep these matrices organized as they are the "source of truth" for functional comparisons.
- **Dependency Check**: EnrichM relies on several external tools (HMMER, Diamond, Prodigal/Pyrodigal). If running in a custom environment, ensure these are in your PATH. Using the Bioconda installation is the recommended way to manage these dependencies.
