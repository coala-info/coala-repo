# roary

## Overview
Roary is a high-performance standalone pipeline for calculating the pan-genome of prokaryotic organisms. It transforms annotated genomic data into a comprehensive map of shared and unique genes across multiple isolates. By clustering proteins and analyzing their distribution, Roary distinguishes between the "core genome" (genes present in nearly all isolates) and the "accessory genome" (genes present in only a subset). It is optimized for speed and memory efficiency, making it feasible to analyze thousands of samples on standard desktop hardware.

## Usage Guidelines

### Input Requirements
- **Format**: Roary requires GFF3 files as input.
- **Source**: While it accepts standard GFF3, it is specifically optimized for files generated by **Prokka**.
- **Content**: Ensure your GFF files contain the underlying FASTA sequence at the end (standard Prokka output).

### Common Command Patterns

**Basic Pan-genome Analysis**
Run with default settings (95% blastp identity, 99% core gene threshold):
```bash
roary -f ./output_dir *.gff
```

**Fast Core Gene Alignment**
To generate a multi-FASTA alignment of core genes (useful for building phylogenetic trees) using MAFFT for speed:
```bash
roary -e --mafft -p 8 *.gff
```

**High Sensitivity Analysis**
For more divergent samples, lower the percentage identity for blastp:
```bash
roary -i 90 -f sensitive_output *.gff
```

**Quality Control with Kraken**
Perform species contamination checks during the run (requires Kraken database):
```bash
roary -qc -k /path/to/kraken_db *.gff
```

### Key Options Reference
- `-p INT`: Number of threads (default: 1).
- `-e`: Create a multiFASTA alignment of core genes using PRANK.
- `-n`: Use MAFFT for core gene alignment (much faster than PRANK; use with `-e`).
- `-i INT`: Minimum percentage identity for blastp (default: 95).
- `-cd FLOAT`: Percentage of isolates a gene must be in to be considered "core" (default: 99).
- `-r`: Create R plots (requires R and ggplot2 installed).
- `-s`: Do not split paralogs.
- `-v`: Verbose output to STDOUT.

### Expert Tips
- **Parallelization**: Always use the `-p` flag to match your available CPU cores, as Roary is highly parallelizable.
- **Memory Management**: Roary is extremely memory efficient (e.g., ~1GB RAM for 128 samples), but very large datasets (thousands of samples) may still require monitoring.
- **Visualizations**: If you use the `-r` flag, Roary produces standard plots. For custom visualizations, the primary output is `gene_presence_absence.csv`, which can be loaded into R or Python for custom analysis.
- **Dependency Check**: Run `roary -a` to check if all required dependencies (blastp, mcl, bedtools, etc.) are correctly installed and visible in your PATH.
