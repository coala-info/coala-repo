# path2xnat-st-workflow-xenium-non-diseased-lung-test-data-ana

## Overview

This Galaxy workflow is designed for the analysis of Xenium spatial transcriptomics (ST) data, specifically utilizing a non-diseased lung test dataset. It facilitates the integration and exploration of high-resolution spatial molecular profiles within the PATH2XNAT framework, bridging the gap between complex spatial datasets and standardized imaging platforms.

The workflow processes several key inputs, including a minimal Xenium lung dataset in ZIP format, associated metadata, and pre-configured R objects (`.RDS` and `.RData`). These files provide the necessary spatial coordinates, transcript counts, and environment states required to initiate a comprehensive transcriptomic analysis.

The primary processing step utilizes an interactive RStudio environment, allowing researchers to perform custom data visualization, clustering, and statistical modeling directly within the Galaxy interface. For detailed setup instructions and execution steps, please refer to the [README.md](./README.md) located in the Files section.

Ensure all Xenium raw data is bundled in the `.zip` archive and that the `.RDS` and `.csv` files match the specific lung tissue version being analyzed. While individual datasets are used here, organizing large-scale spatial transcriptomics runs into Galaxy collections can streamline batch processing. The `.RData` and `.RHistory` files are essential for restoring the interactive RStudio session state and maintaining reproducibility across analysis stages. Refer to the `README.md` for comprehensive instructions on configuring the R environment and specific library dependencies. You can use `planemo workflow_job_init` to generate a `job.yml` for automated execution of this Xenium pipeline.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-PATH2XNAT_ST_workflow_Xenium_non-diseased_lung_test_data_analysis.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-PATH2XNAT_ST_workflow_Xenium_non-diseased_lung_test_data_analysis.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-PATH2XNAT_ST_workflow_Xenium_non-diseased_lung_test_data_analysis.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-PATH2XNAT_ST_workflow_Xenium_non-diseased_lung_test_data_analysis.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-PATH2XNAT_ST_workflow_Xenium_non-diseased_lung_test_data_analysis.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
