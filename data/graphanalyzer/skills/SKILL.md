# graphanalyzer

## Overview
The `graphanalyzer` tool automates the manual inspection of vConTACT2 outputs. In viral metagenomics, vConTACT2 clusters sequences into Viral Clusters (VCs) but does not automatically propagate taxonomy from reference genomes to user-provided viral operational taxonomic units (vOTUs). This skill provides the procedural knowledge to run the `graphanalyzer.py` script, which parses network files and cluster overviews to provide direct taxonomic associations and interactive visualizations.

## Command Line Usage

### Required Input Files
To run a successful analysis, you must have the following three files prepared:
1.  `--graph`: The `c1.ntw` file (the network graph generated by vConTACT2).
2.  `--csv`: The `genome_by_genome_overview.csv` file (cluster and confidence metrics from vConTACT2).
3.  `--metas`: The `data_excluding_refseq.tsv` file (taxonomic metadata from the INPHARED database).

### Basic Execution Pattern
```bash
graphanalyzer.py \
  --graph c1.ntw \
  --csv genome_by_genome_overview.csv \
  --metas data_excluding_refseq.tsv \
  --prefix vOTU \
  --output ./results_dir \
  --threads 8 \
  --view 2d
```

### Key Parameters and Best Practices
*   **Prefix Matching (`--prefix`)**: This is the most critical parameter for correct identification. It must match the starting string of your user-provided viral scaffolds (e.g., if your scaffolds are named `vOTU_1`, `vOTU_2`, use `--prefix vOTU`).
*   **Visualization (`--view`)**: Choose between `2d` or `3d` for the interactive Plotly graphs. 2D is generally more performant for large datasets.
*   **Performance**: Use the `--threads` flag to scale the analysis. The tool utilizes multiprocessing for graph traversal and data processing.
*   **Output Organization**: Use `--suffix` to distinguish between different runs or parameter sets within the same output directory.

## Interpreting Results
The tool generates several files that automate the logic suggested by vConTACT2 authors:
*   **Taxonomic Association**: If a vOTU is in the same VC subcluster as a reference, it is assigned that genus with high probability.
*   **Relationship Levels**: If in the same VC but a different subcluster, the relationship is likely at the genus-subfamily level.
*   **Interactive Subgraphs**: Use the generated HTML files to visually inspect the connectivity between your vOTUs and known INPHARED references.

## Expert Tips
*   **Database Versioning**: Ensure the `data_excluding_refseq.tsv` file matches the version of INPHARED used to generate the vConTACT2 input. Discrepancies in accessions will lead to failed taxonomic lookups.
*   **Environment Setup**: While available via Bioconda, the tool relies on `pygraphviz`. If manual installation fails, ensure the underlying `graphviz` system library is installed on your Linux environment.
*   **Memory Management**: For very large networks (e.g., "big-query" style datasets), ensure you have sufficient RAM as `networkx` loads the entire graph into memory for analysis.
