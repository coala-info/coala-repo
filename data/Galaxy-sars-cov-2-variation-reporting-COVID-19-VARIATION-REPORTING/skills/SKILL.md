# covid-19-variation-analysis-reporting

## Overview

This workflow is designed for the post-processing and reporting of SARS-CoV-2 genomic variations. It consumes VCF datasets generated by [SARS-CoV-2 variant-calling workflows](https://github.com/galaxyproject/iwc/tree/main/workflows/sars-cov-2-variant-calling) to provide a structured summary of identified mutations. By applying user-defined filters for allele frequency (AF) and sequencing depth (DP and DP_ALT), the pipeline ensures that only high-confidence variants are included in the final analysis.

The technical process utilizes SnpSift to filter and extract specific variant fields, followed by data manipulation using Datamash and various text-processing utilities. It integrates gene product translations to annotate variant effects and organizes the data into comprehensive tabular reports. These reports are categorized both by individual samples and by specific variants across the entire dataset, allowing for easy comparison of mutational profiles.

In addition to tabular data, the workflow generates a visual variant frequency plot that illustrates allele frequencies across samples and clusters. This visualization helps researchers identify common mutations and patterns within the viral population. For detailed instructions on input preparation and specific parameter settings, please refer to the [README.md](README.md) in the Files section.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Variation data to report | data_collection_input | Variation data in VCF format. Can be the output of any of the workflows in https://github.com/galaxyproject/iwc/tree/main/workflows/sars-cov-2-variant-calling |
| 1 | AF Filter | parameter_input | Allele Frequency Filter. This is the minimum allele frequency required for variants to be included in the reports. |
| 2 | DP Filter | parameter_input | Depth Filter. This is the minimum depth of all alignments at a variant site. |
| 3 | DP_ALT Filter | parameter_input | Depth Filter for variant allele. This is the minimum depth of alignments supporting a variant. |
| 4 | gene products translations | data_input | A custom tabular file mapping NCBI RefSeq Protein identifiers as used by snpEff version 4.5covid19 to their commonly used names. Can be obtained from https://doi.org/10.5281/zenodo.4555734 |
| 5 | Number of Clusters | parameter_input | Number of Clusters to use in Variant Frequency Plot. |


Provide the variation data as a VCF dataset collection, typically sourced from upstream SARS-CoV-2 variant-calling workflows. Ensure the gene products translations file is provided as a tabular dataset to correctly map variant effects during the reporting process. You should adjust the allele frequency and depth filters to suit your specific analysis stringency. Consult the README.md for exhaustive details on input preparation and parameter configuration. For automated runs, `planemo workflow_job_init` can be used to create a template `job.yml` file.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run variation-reporting.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run variation-reporting.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run variation-reporting.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init variation-reporting.ga -o job.yml`
- Lint: `planemo workflow_lint variation-reporting.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
