# bioconductor-bsseqdata

## Overview

The `bsseqData` package is a data-only experiment package providing example Whole Genome Bisulfite Sequencing (WGBS) data for chromosomes 21 and 22. The data is derived from a study of three patients with matched colon cancer and normal colon tissue. It is designed specifically to support the `bsseq` package by providing objects of class `BSseq` and `BSseqTstat`.

## Data Objects

The package contains four primary datasets that represent different stages of a methylation analysis pipeline:

- `BS.cancer.ex`: A `BSseq` object containing raw methylation evidence (M and Cov matrices) for 6 samples (3 cancer, 3 normal).
- `BS.cancer.ex.fit`: A `BSseq` object that includes the same data as `BS.cancer.ex` plus smoothed methylation values generated by the `BSmooth` algorithm.
- `BS.cancer.ex.tstat`: A `BSseqTstat` object containing t-statistics comparing cancer vs. normal, generated via `BSmooth.tstat`.
- `keepLoci.ex`: A logical vector indicating which loci were retained for the t-statistic analysis (filtering out low-coverage or non-informative sites).

## Typical Workflow

### 1. Loading the Data
To use the example data, load the package and use the `data()` function:

```r
library(bsseqData)
data(BS.cancer.ex)
data(BS.cancer.ex.fit)

# Inspect the object
BS.cancer.ex
```

### 2. Accessing Metadata
Since these are `BSseq` objects, you can inspect the sample information:

```r
pData(BS.cancer.ex)
```

### 3. Working with T-Statistics
To analyze differential methylation results provided by the package:

```r
data(BS.cancer.ex.tstat)
plot(BS.cancer.ex.tstat)
```

### 4. Exploring Raw Source Scripts
The package includes the original scripts used to process the raw alignment files (from the BSmooth alignment suite) into the R objects:

```r
# Locate the script
script_path <- system.file("scripts", "create_BS.cancer.R", package = "bsseqData")
# Read the processing logic
readLines(script_path)

# Locate the raw data directory (legacy 'umtab' format)
raw_data_dir <- system.file("umtab", package = "bsseqData")
```

## Usage Tips
- **Memory Management**: While these datasets are subsets (Chr 21 and 22), `BSseq` objects can be large. Use `BS.cancer.ex.fit` if you need to skip the computationally intensive smoothing step.
- **Integration**: This package is intended to be used alongside the `bsseq` package. Functions like `dmrFinder` or `plotRegion` should be called from `bsseq` using these data objects as input.
