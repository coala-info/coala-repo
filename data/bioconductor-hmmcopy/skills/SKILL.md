# bioconductor-hmmcopy

name: bioconductor-hmmcopy
description: Copy number estimation and segmentation for high-throughput sequencing (HTS) data. Use when analyzing tumour-normal pairs or single samples to correct for GC-content and mappability biases, and to perform Hidden Markov Model (HMM) based segmentation and classification of copy number states (HOMD, HETD, NEUT, GAIN, AMPL, HLAMP).

## Overview

HMMcopy provides a workflow for bias-free copy number estimation from whole-genome sequencing (WGS) data. It corrects read counts for GC-content and mappability biases, which are inherent in sequencing protocols. The package then applies a Hidden Markov Model to segment the genome into regions of constant copy number and classifies these segments into biological states.

## Core Workflow

### 1. Loading Data
The package expects bin-level data (typically 1kb windows) containing read counts, GC content, and mappability scores.

```R
library(HMMcopy)

# Load WIG files into a RangedData/data.table object
# Files are usually generated by the HMMcopy C++ suite
rfile <- "reads.wig"
gfile <- "gc.wig"
mfile <- "map.wig"
reads <- wigsToRangedData(rfile, gfile, mfile)
```

### 2. Bias Correction
Correct read counts for GC and mappability biases. This produces a `copy` column representing log2-transformed normalized values.

```R
# Correct biases
copy_profile <- correctReadcount(reads)

# For matched tumour-normal analysis:
# somatic_copy = tumour_copy$copy - normal_copy$copy
```

### 3. Segmentation and Classification
Use `HMMsegment` to identify copy number events.

```R
# Run segmentation with default parameters
segments <- HMMsegment(copy_profile)

# Access predicted states and segment boundaries
# segments$state contains the classification for each bin
# segments$segs contains the summarized segment table
```

### 4. Visualization
HMMcopy provides specialized plotting functions to inspect biases and results.

```R
# Visualize GC and Mappability bias
plotBias(copy_profile)

# Visualize the correction effects (Uncorrected vs GC vs Map)
plotCorrection(copy_profile)

# Plot segments with state colors
plotSegments(copy_profile, segments)
```

## Advanced Parameter Tuning

If the default segmentation is too noisy or fails to capture obvious events, adjust the HMM parameters.

### Reducing Over-segmentation
Increase `e` (probability of staying in a state) and `strength`.

```R
param <- HMMsegment(copy_profile, getparam = TRUE)
param$e <- 0.999999999999999
param$strength <- 1e30
segments <- HMMsegment(copy_profile, param)
```

### Adjusting State Ranges (mu)
If the medians of the states (e.g., Neutral vs Gain) are shifted, manually set the `mu` (initial means) and `m` (prior means) to force convergence to specific values.

```R
param$mu <- c(-0.5, -0.4, -0.15, 0.1, 0.4, 0.7) # HOMD, HETD, NEUT, GAIN, AMPL, HLAMP
param$m <- param$mu # Set prior means to match initial means to prevent drift
segments <- HMMsegment(copy_profile, param)
```

## Tips and Best Practices
- **Input Requirements**: Ensure input WIG files are pre-computed for the same bin size (e.g., 1000bp).
- **Matched Normal**: Always prefer matched tumour-normal subtraction (`tumour$copy - normal$copy`) to eliminate germline CNVs and reduce noise.
- **State Definitions**:
    - HOMD: Homozygous deletion (<= 0 copies)
    - HETD: Heterozygous deletion (1 copy)
    - NEUT: Neutral (2 copies)
    - GAIN: Gain (3 copies)
    - AMPL: Amplification (4 copies)
    - HLAMP: High-level amplification (>= 5 copies)
