# minnow

## Overview

Minnow is a principled framework designed to simulate droplet-based single-cell RNA-seq experiments at the sequence level. While most simulators operate at the gene-count level, Minnow generates raw reads by sampling from a reference transcriptome or a de-Bruijn graph (dBG). This allows for a realistic assessment of how different alignment and UMI resolution strategies impact downstream analysis. It is particularly useful for testing the robustness of pipelines against sequence-level characteristics and technical noise.

## Installation and Setup

Minnow can be installed via Conda or built from source to access the latest features.

- **Conda**: `conda install bioconda::minnow`
- **Source**: Use the `minnow-velocity` branch for the most recent updates.
  ```bash
  git clone --single-branch --branch minnow-velocity https://github.com/COMBINE-lab/minnow.git
  cd minnow && mkdir build && cd build
  cmake .. && make
  ```

## Core Workflow

### 1. Pre-requisite: Alevin Quantification
Minnow requires probability files generated by Alevin (part of the Salmon suite). You must run Alevin on a real dataset related to your organism of interest using the `--dumpBfh` flag.
```bash
salmon alevin -l ISR -i [index] -1 [R1.fq.gz] -2 [R2.fq.gz] -o alevin_out --tgMap [tx2gene.tsv] --chromium --dumpBfh
```

### 2. Indexing the Transcriptome
The indexing phase constructs the de-Bruijn graph and prepares the reference.
```bash
minnow index -r transcriptome.fa -k 101 -f 20 -p 10 -o minnow_index
```
- **Note on `-k`**: The k-mer length should typically be treated as the read length for the simulation.
- **Automatic Cleaning**: During indexing, Minnow automatically clips PolyA tails, replaces 'N's with random bases, and removes sequences shorter than the read length.

### 3. Simulation
While simulation parameters vary, the use of the `--dbg` flag is recommended to enable advanced features like sequence-level error modeling.

## CLI Best Practices and Tips

- **Header Cleaning**: If you require clean transcript names in your output, pre-process your FASTA headers:
  ```bash
  sed -e '/^>/ s/|.*//' input.fa > cleaned_input.fa
  ```
- **Memory Management**: Use the `--tmpdir` flag during indexing to specify a high-capacity directory for TwoPaCo temporary files if working with large transcriptomes.
- **Thread Optimization**: Use the `-p` flag to match your available CPU cores for faster dBG construction.
- **Validation**: Always ensure the BFH file from Alevin is present in the output directory before starting the simulation, as Minnow uses this to model realistic cell-level distributions.
