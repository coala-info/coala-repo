# curve-curator

## Overview

CurveCurator is a high-throughput analysis platform for dose-response data. It automates the process of fitting classical 4-parameter logistic equations to experimental observations to determine effect potency and statistical significance. By employing a unique 2D-thresholding strategy, it effectively separates biological hits from noise and false positives. The tool is particularly optimized for mass spectrometry-based proteomics (supporting MaxQuant, DIANN, Proteome Discoverer, and MSFragger) and general viability experiments.

## Installation and Environment

For optimal performance and dependency management, use a dedicated Python 3.12 environment:

```bash
conda create -n CurveCuratorEnv pip python=3.12
conda activate CurveCuratorEnv
pip install curve-curator
```

## Command Line Usage

The primary interface for CurveCurator is the CLI, which accepts either a single configuration file or a batch list.

### Basic Execution
Run the pipeline on a single dataset using a TOML configuration file:
```bash
CurveCurator path/to/config.toml
```

### Advanced Processing Options
*   **FDR Estimation**: Use the target-decoy approach to estimate False Discovery Rate. Note that this doubles the processing time.
    ```bash
    CurveCurator -f config.toml
    ```
*   **Outlier Detection**: Perform Median Absolute Deviation (MAD) analysis to identify and handle data outliers.
    ```bash
    CurveCurator -m config.toml
    ```
*   **Batch Processing**: Process multiple datasets sequentially by providing a text file containing paths to multiple TOML files.
    ```bash
    CurveCurator -b batch.txt
    ```
*   **Null Hypothesis Simulation**: Run the pipeline with N random values for H0 simulation.
    ```bash
    CurveCurator -r 1000 config.toml
    ```

## Data Preparation Best Practices

### Proteomics Search Engine Formatting
*   **MaxQuant**: Use `proteinGroups.txt` for protein-level analysis or `evidence.txt` for peptide-level.
*   **DIANN**: Manually rename raw file columns to `Raw 1..N` before processing.
*   **Proteome Discoverer (PD)**: Ensure the TOML file's dose correspondences match the `F1..N` labels generated by PD.
*   **MSFragger**: Name TMT channels or LFQ experiments as `Raw_1...N`. Use `combined_protein.tsv` or `combined_ion.tsv`.

### General Viability Data
*   Input must be a tab-separated `.txt` file.
*   Required columns: A `Name` column (sample identifier) and raw value columns (intensities or ratios).

## Expert Tips for Configuration

*   **TOML Syntax**: Ensure all strings are enclosed in quotation marks. Lists must be homogeneous; do not mix integers and floats (e.g., use `[1.0, 2.0]` instead of `[1, 2.0]`).
*   **2D-Thresholding**: Use the interactive dashboard (launched after the pipeline runs) to fine-tune significance and relevance thresholds. This is more effective than guessing parameters in the TOML file.
*   **Memory Management**: When running large-scale proteomics datasets with FDR estimation, ensure sufficient RAM is available as the tool loads significant portions of the data into memory for fitting.
