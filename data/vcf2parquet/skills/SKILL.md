# vcf2parquet

## Overview
The `vcf2parquet` tool is a high-performance Rust-based utility designed to bridge the gap between traditional bioinformatics file formats and modern data engineering ecosystems. It converts VCF files—including compressed versions (.gz, .bz2, .xz)—into Parquet files, which offer significantly faster query speeds and reduced storage footprints. This skill provides the necessary CLI patterns to execute single-file conversions, manage compression levels, and partition data for parallel processing.

## Installation
The tool is primarily distributed via Bioconda:
```bash
mamba install vcf2parquet
```

## Core Workflows

### Single File Conversion
To convert a standard or compressed VCF into a single Parquet file:
```bash
vcf2parquet -i input.vcf.gz convert -o output.parquet
```

### Partitioned Splitting
For extremely large datasets, use the `split` subcommand to create multiple Parquet files, each containing a specific number of records defined by the batch size:
```bash
vcf2parquet -i input.vcf.gz split -f output_part_{}.parquet
```
*Note: The `{}` in the filename acts as a placeholder for the partition index.*

### Performance Tuning
Adjust the batch size and compression algorithm to optimize for either conversion speed or disk space:
```bash
vcf2parquet -i input.vcf.gz -c lz4 -b 50000 convert -o output.parquet
```

## Command Reference

### Global Options
- `-i, --input <PATH>`: Path to the source VCF file (supports `.vcf`, `.gz`, `.bz2`, `.xz`).
- `-b, --batch-size <INT>`: Number of records per internal processing batch (default: 100,000).
- `-c, --compression <TYPE>`: Parquet compression codec.
    - Options: `uncompressed`, `snappy` (default), `gzip`, `lzo`, `brotli`, `lz4`.

#

## Expert Tips
- **Compression Choice**: Use `lz4` or `snappy` for the best balance of conversion speed and query performance. Use `gzip` or `brotli` only if storage space is the primary constraint.
- **Memory Management**: If encountering memory issues on systems with limited RAM, decrease the `--batch-size` from the default 100,000 to a lower value like 10,000.
- **Downstream Compatibility**: Parquet files generated by this tool are optimized for tools like `duckdb` and `polars`. After conversion, you can query the data immediately:
  ```python
  import duckdb
  duckdb.query("SELECT * FROM 'output.parquet' WHERE QUAL > 30").show()
  ```



## Subcommands

| Command | Description |
|---------|-------------|
| vcf2parquet convert | Convert VCF files to Parquet format |
| vcf2parquet split | Convert a vcf in multiple parquet file each file contains `batch_size` record |
