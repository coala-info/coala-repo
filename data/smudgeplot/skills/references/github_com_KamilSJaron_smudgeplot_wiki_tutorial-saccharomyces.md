### with hindsight

The default smudgeplot was misleading. There was a smudge on an unexpected place, and I thought it's because of too low cutoff (parameter L), but the problem was somewhere else. The sneaky part in this dataset was that there is no AB smudge and to figure that the "mysterious smudge" is just a problem of 1n coverage estimate and incorrect smudge annotation as a consequence. Once smudgeplot is told what is the expected monoploid coverage, everything starts to make sense.

### Intro to the funky yeast

There is [an issue](https://github.com/KamilSJaron/smudgeplot/issues/31#issuecomment-625979647) asking about a strange smudgeplot of supposedly tetraploid saccharomyces [[Knaus et al 2018]](https://www.frontiersin.org/articles/10.3389/fgene.2018.00123/full).

![strange_smudgeplot](https://user-images.githubusercontent.com/5849554/81441461-ad453480-9126-11ea-8b27-bc1bd70b9069.png)

Looking at the posted smudgeplot - yeah, it's really weird. Beside the AB smudge, there is also a one more ~ at the L \* cov line (the error line). I am suspecting that the L was too low, but I got to investigate a tiny bit if I am right.

### The analysis

Get the data

```
wget ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR326/001/SRR3265401/SRR3265401_[12].fastq.gz
```

Use `fastK` to get a kmer database (`kmer_db`)

```
FastK -v -t10 -k21 -M16 -T4 *_[12].fastq.gz -Nkmer_db
```

I suspected that the L is too low, hence we should look at the kmer histogram. I fit there a default diploid GenomeScope model, because I more care about the shape of the histogram than anything else at this point

```
Histex -G kmer_db > kmer_k21.hist
genomescope.R -i kmer_k21.hist -k 21 -p 2 -o . -n Scer_genomescope
```

![strange_genomescope](https://user-images.githubusercontent.com/8181573/82441921-a0193580-9a96-11ea-8719-a891a35ac230.png)

Alright. We see three peaks, ~15x, ~30x and ~60x. So the only question is if the smallest peak the 1n of the genome? I also expect that choosing L = 15 and high U (3000) would generate a smudgeplot similar to the one posted

```
smudgeplot hetmers -L 15 -t 4 -o kmerpairs_L15 --verbose kmer_db
smudgeplot all -o saccharomyces_L15 -t "yeast" kmerpairs_L15.smu
```

and indeed, it is somewhat similar. However, the updated version of smudgeplot actually deal much better with low-coverage datasets and even with the same settings the 1n coverage inference and the tetraploid structure of the genome is apparent.

![saccharomyces_L15_smudgeplot](https://private-user-images.githubusercontent.com/8181573/518079107-5b0878c8-4e78-4c55-974e-6eeb6a1ca0b0.png?jwt=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJpc3MiOiJnaXRodWIuY29tIiwiYXVkIjoicmF3LmdpdGh1YnVzZXJjb250ZW50LmNvbSIsImtleSI6ImtleTUiLCJleHAiOjE3NzQ2NjYyNTcsIm5iZiI6MTc3NDY2NTk1NywicGF0aCI6Ii84MTgxNTczLzUxODA3OTEwNy01YjA4NzhjOC00ZTc4LTRjNTUtOTc0ZS02ZWViNmExY2EwYjAucG5nP1gtQW16LUFsZ29yaXRobT1BV1M0LUhNQUMtU0hBMjU2JlgtQW16LUNyZWRlbnRpYWw9QUtJQVZDT0RZTFNBNTNQUUs0WkElMkYyMDI2MDMyOCUyRnVzLWVhc3QtMSUyRnMzJTJGYXdzNF9yZXF1ZXN0JlgtQW16LURhdGU9MjAyNjAzMjhUMDI0NTU3WiZYLUFtei1FeHBpcmVzPTMwMCZYLUFtei1TaWduYXR1cmU9MWMxNGVhZTJiODg2Zjg3NGUxNDZhMTFmZDM1MDkyNWIxYTY2OGZlZTZjMjI5MzM0YmIxMDNlODAwYmU0MjZlOCZYLUFtei1TaWduZWRIZWFkZXJzPWhvc3QifQ.gZGsmHUqb5NSSUFcJhTo88K5ot_oDWtN0k0XwBbBDS0)

How is that apparent? Well, only a tetraploid genome would predominantly feature tetraploid smudges (AAAB and AABB) while lacking any others. Nevertheless, if we chose lower L, we will likely see a much nicer plot as the smudges will be further from the coverage filter, say L = 5.

```
smudgeplot hetmers -L 5 -t 4 -o kmerpairs_L5 --verbose kmer_db
smudgeplot all -o saccharomyces_L5 -t "yeast" kmerpairs_L5.smu
```

And indeed we qualitatively see the same picture, but it looks cleaner

![saccharomyces_L5_smudgeplot](https://private-user-images.githubusercontent.com/8181573/518079425-4a419173-c5fd-4a1b-9187-660761fd3df8.png?jwt=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJpc3MiOiJnaXRodWIuY29tIiwiYXVkIjoicmF3LmdpdGh1YnVzZXJjb250ZW50LmNvbSIsImtleSI6ImtleTUiLCJleHAiOjE3NzQ2NjYyNTcsIm5iZiI6MTc3NDY2NTk1NywicGF0aCI6Ii84MTgxNTczLzUxODA3OTQyNS00YTQxOTE3My1jNWZkLTRhMWItOTE4Ny02NjA3NjFmZDNkZjgucG5nP1gtQW16LUFsZ29yaXRobT1BV1M0LUhNQUMtU0hBMjU2JlgtQW16LUNyZWRlbnRpYWw9QUtJQVZDT0RZTFNBNTNQUUs0WkElMkYyMDI2MDMyOCUyRnVzLWVhc3QtMSUyRnMzJTJGYXdzNF9yZXF1ZXN0JlgtQW16LURhdGU9MjAyNjAzMjhUMDI0NTU3WiZYLUFtei1FeHBpcmVzPTMwMCZYLUFtei1TaWduYXR1cmU9YTdhMzkzZDYyOTU3ZGQ1NzNkYjU2Y2YxODNhMzEzNmI1MWUzNDgzNDU4OTllYzBhYTYxZGM4NGMyMjFlZjdiNCZYLUFtei1TaWduZWRIZWFkZXJzPWhvc3QifQ.Lp7rQ4ErczkWt_1Ee8spaihdgRwE2phVR9UkHdON-YQ)

Now that we know we have a tetraploid and that we also have a good idea about the 1n coverage (around 16x), we can fit the genomescope model with the appropriate ploidy and coverage prior

```
genomescope.R -i kmer_k21.hist -k 21 -p 4 -o . -l 16 -n Scer_genomescope_p4
```

![strange_genomescope2](https://user-images.githubusercontent.com/8181573/82442622-ee7b0400-9a97-11ea-8a22-c1990e7348ce.png)

And Voilà, the genome size is approximately right (14Mbp vs expected 12Mbp) and the spacing of the individual peaks seems to be exactly matching the tetraploid model. GenomeScope and Smudgeplot converged on similar 1n coverages and both look very sane in the context of expected biology. Genome profiling well done.
