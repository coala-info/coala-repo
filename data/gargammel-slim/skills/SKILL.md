# gargammel-slim

## Overview

gargammel-slim is a specialized toolkit for modeling the unique characteristics of ancient DNA. It focuses on the "in vivo" to "in vitro" transition of genetic material, simulating how DNA degrades over time and how it is prepared for sequencing. The tool allows for the simultaneous simulation of endogenous ancient DNA, modern human contamination, and microbial background. By using its three core programs—fragSim, deamSim, and adptSim—users can create highly realistic datasets that mimic the fragment length distributions and biochemical damage (deamination) typical of ancient samples.

## Core Workflow and CLI Usage

The gargammel-slim suite is designed to be used as a pipeline where the output of one tool serves as the input for the next.

### 1. Fragment Simulation (fragSim)
This tool retrieves fragments at random from the provided genomes.

*   **Input Structure**: You must organize your input directory with three specific subfolders:
    *   `endo/`: Endogenous ancient human/target genome.
    *   `cont/`: Present-day human contaminant.
    *   `bact/`: Microbial contamination.
*   **Key Parameters**:
    *   `-n <int>`: Specify the exact number of fragments to generate.
    *   `-c <float>`: Specify the desired coverage (e.g., `-c 0.5` for 0.5X).
    *   `--comp <endo,cont,bact>`: Set the composition ratios (e.g., `--comp 0.90,0.05,0.05` for 90% endogenous).
    *   `-f <file>`: Provide a size frequency file (e.g., `src/sizefreq.size.gz`) to model realistic fragment lengths.

### 2. Deamination Simulation (deamSim)
This tool applies damage patterns to the fragments generated by fragSim.

*   **Briggs Model**: Use the `-damage` flag to specify the four parameters of the Briggs model:
    `deamSim -damage <v, l, d_s, d_d>`
    *   `v`: Probability of being single-stranded.
    *   `l`: Length of single-stranded overhang.
    *   `d_s`: Rate of deamination in single-stranded parts.
    *   `d_d`: Rate of deamination in double-stranded parts.
*   **Empirical Matrices**: Use `-matfile` to apply misincorporation rates measured from specific ancient individuals (e.g., `-matfile src/matrices/single-`).
*   **mapDamage Integration**: You can use a `misincorporation.txt` file produced by mapDamage to guide the simulation.

### 3. Adapter Simulation (adptSim)
This tool adds Illumina adapters to the damaged fragments to create raw reads.

*   **Usage**: Typically follows deamSim to prepare the fragments for downstream sequencing error simulators like ART.

## Expert Tips and Best Practices

*   **Genome Input**: Ensure the files in your `endo/`, `cont/`, and `bact/` folders represent full genomes. For the endogenous human component, use at least two genomes to represent a diploid individual.
*   **Log-Normal Distributions**: If you don't have a size frequency file, you can simulate fragment lengths using a log-normal distribution with `--loc` and `--scale` parameters.
*   **Single-End vs. Paired-End**: While gargammel-slim prepares the fragments, remember that the final read length and orientation (SE or PE) are usually defined in the final sequencing simulation step. If using the full gargammel wrapper, use `-se` for single-end.
*   **Microbial Databases**: For realistic microbial contamination, use a representative set of microbes relevant to the sample's depositional environment (e.g., soil bacteria for terrestrial remains).
