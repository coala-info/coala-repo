# eigensoft

## Overview
The EIGENSOFT package is a suite of tools designed for population genetics and bioinformatics. It is primarily used to identify and adjust for population structure, which is a common confounding factor in genetic association studies. By using Principal Component Analysis (PCA), it allows researchers to visualize genetic clusters and calculate ancestry-adjusted genotypes. The skill covers the core workflow of format conversion, PCA execution, and result interpretation using the package's native command-line interface.

## Core Programs
- **smartpca**: The primary program for performing PCA on large-scale genotype data.
- **convertf**: A utility for converting between various genetic data formats.
- **eigenstrat**: Used for stratification correction in association tests.
- **ploteig**: A script for plotting the principal components generated by smartpca.
- **pcaselection**: A tool to select the number of significant principal components.

## Common Workflows and CLI Patterns

### 1. Data Format Conversion
Before running analysis, use `convertf` to ensure your data is in the correct format (usually EIGENSTRAT or PACKEDANCESTRYMAP).

**Typical parameter file (`convert.par`):**
```text
genotypename:    input.ped
snpname:         input.map
indivname:       input.ped
outputformat:    EIGENSTRAT
genotypeoutname: output.geno
snpoutname:      output.snp
indivoutname:    output.ind
```
**Execution:**
```bash
convertf -p convert.par
```

### 2. Running PCA with smartpca
`smartpca` requires a parameter file to define input files and algorithmic settings.

**Key Parameters:**
- `fastmode: YES`: Implements a fast PCA approximation (recommended for very large datasets).
- `shrinkmode: YES`: Adjusts for "shrinkage" in PCA scores, useful when projecting new samples onto a reference.
- `numthreads: N`: Enables multi-threading for faster computation.

**Execution:**
```bash
smartpca -p smartpca.par
```

### 3. Handling Population Stratification
To perform EIGENSTRAT stratification correction, use the output from `smartpca` (the `.evec` file) as an input for the association test to adjust for ancestry.

## Expert Tips and Troubleshooting
- **Library Dependencies**: If you encounter "undefined reference to `LAPACKE_dlange`" during compilation, ensure you link the lapacke library using `make LDLIBS="-llapacke"`.
- **Memory Allocation**: For very large datasets, ensure your system has sufficient RAM. If `smartpca` fails with a memory error, check if `fastmode` is enabled to reduce the computational footprint.
- **SNP Selection**: PCA is sensitive to Long-Range Linkage Disequilibrium (LD). It is a best practice to prune SNPs for LD before running `smartpca` to avoid components being driven by single genomic regions (like the HLA or inversions).
- **Visualizing Results**: Use the `ploteig` script on the `.evec` output to generate plots. It typically requires `perl` and `gnuplot` to be installed on the system.
