The columns of this file are: sample metabolite derivative isotopologue isotopic_inchi area corrected_area isotopologue_fraction residuum mean_enrichment TraceGroomer transforms specific columns of the IsoCor output file, into separate files. As the total metabolite abundance column is not present in the input file, the total abundance per metabolite is the automatic result of the sum, per metabolite, of Isotopologues' absolute values (see AbundanceCorrected below). So here the correspondances: column in the IsoCor file TraceGroomer output filename corrected_area IsotopologuesAbsolute isotopologue_fraction IsotopologuesProportions mean_enrichment MeanEnrichment - AbundanceCorrected We provide an example which is downloadable from Zenodo (see here ). In the command line, the absolute path of this file must be provided with the -lm parameter. The "rule" format This format obeys to the rule that the isotopologues abundances in absolute values are always accompanied by a file that explains the variables, being the variables the isotopologues identifiers. An example is provided in the downloadable downloadable Zenodo material. The files must be available in tab delimited format. The names of the files are free, but require the .tsv extension: "dataMatrix": contains the isotopologues absolute values organized in a tabular format, having a first column with the isotopologue identifiers, whereas the sample names constitute the header of the rest of the columns. This means that isotopologues represent the rows, and samples represent the columns. The isotopologue identifiers must be unique. Example of a "dataMatrix" file: ID sampleA1 sampleA2 sampleB1 sampleB2 Peak1 0.54406 0.5299 0.6076 0.9352 Peak2 0.2715 0.3129 0.3139 0.0507 Peak3 0.1844 0.1571 0.0784 0.0142 .. .. .. .. .. In the command line, the absolute path to this file must be provided through the -lm parameter. Example of a "dataMatrix" file: "variableMetadata": the specifics of this file are explained in the subsection 2 Input files &gt; the metadata file in the sub-subsection Special case of the "rule" type of input . Example of a "variableMetadata" file: ID metabolite_name isotope_numeric Peak1 Acetyl_CoA 0 Peak2 Acetyl_CoA 1 Peak3 Acetyl_CoA 2 .. .. .. In the command line, the name of the "variableMetadata", and its columns , must be specified in the configuration file . Note : the samples metadata is not mentioned in this subsection because TraceGroomer requires it for all the supported input formats, and not only for the "rule" format. See 2 Input files &gt; the metadata file for a complete explanation of the metadata describing the samples. We provide an example which is downloadable from Zenodo (see here ). Custom or generic xlsx file We have created this option for those formats that do not correspond to the two scenarios above, so your data is expected to be in the form of a .xlsx file with sheets similar as in the provided in example-sheet present in the downloadable examples: this .xlsx file must NOT contain: formulas, symbols accompanying the numeric values, nor special characters. the header (first row, with the molecules IDs), and the first column (with the samples) can contain non numeric values. the isotopologues names, in the header, must follow the convention metaboliteID_labelX : the substring _label is compulsory and is located between the metabolite name (or identifier) metaboliteID and the number of marked carbon atoms X (e.g. glucose6phosphate_label0 ). Important : If you only have Isotopologue Absolute values, but not the other tables: put them as a single named sheet in your .xlsx file, and TraceGroomer automatically generate all the other types of tables for you ! Notes : sheets corresponding to isotopologue Proportions (when available) and isotopologue Absolute values (compulsory if the proportions not available) must have isotopologues as columns and samples as rows. sheets corresponding to abundance and mean enrichment (when available) must have metabolites as columns and samples as rows. the sheets names must be unambiguous. in a file, each sheet must corresponds to one type of quantification. Multiplets of a same type of quantification are not supported. We provide an example which is downloadable from Zenodo (see here ).
