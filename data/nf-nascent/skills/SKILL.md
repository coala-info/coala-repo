# nascent

## Overview
nf-core/nascent is a bioinformatics best-practice analysis pipeline for nascent transcript (NT) and Transcriptional Start Site (TSS) assays. It solves the problem of mapping and quantifying RNA that is actively being transcribed, providing a standardized workflow for various specialized sequencing protocols.

The pipeline accepts raw FASTQ files and produces aligned reads, normalized coverage tracks (bigWig/bedGraph), and identified transcript regions or TSS peaks. It supports a wide range of assay types including CoPRO, GRO-seq, PRO-seq, CAGE, and RAMPAGE, allowing researchers to move from raw data to quantified genomic elements and quality control summaries.

## Data preparation
The pipeline requires a comma-separated samplesheet and reference genome information.

### Samplesheet
The samplesheet must be a CSV file with a header and three columns:
- `sample`: Unique sample identifier (no spaces).
- `fastq_1`: Path to the first-read FASTQ file (must be `.fq.gz` or `.fastq.gz`).
- `fastq_2`: Path to the second-read FASTQ file for paired-end data (optional).

Example `samplesheet.csv`:
```csv
sample,fastq_1,fastq_2
CONTROL_REP1,AEG588A1_S1_L002_R1_001.fastq.gz,AEG588A1_S1_L002_R2_001.fastq.gz
TREATMENT_REP1,AEG588A1_S2_L002_R1_001.fastq.gz,
```

### Reference files
You must provide a reference genome using one of the following methods:
- **iGenomes**: Use `--genome <ID>` (e.g., `--genome GRCh38`).
- **Manual**: Provide `--fasta` and either `--gtf` or `--gff`.
- **Optional BED**: A `--gene_bed` file can be provided; otherwise, it is generated from the GTF.
- **Filtering**: Use `--filter_bed` for undesired regions (e.g., promoters) or `--intersect_bed` for desired regions (e.g., histone modifications).

## How to run
Run the pipeline using the `nextflow run` command. You must specify the `--assay_type` parameter to match your experimental protocol.

```bash
nextflow run nf-core/nascent \
  -profile <docker/singularity/conda> \
  --input samplesheet.csv \
  --outdir <RESULTS_DIR> \
  --assay_type PROseq \
  --genome GRCh38
```

Key parameters:
- `-profile`: Choose a configuration profile (e.g., `docker`, `singularity`, or an institutional profile).
- `--assay_type`: Required. Options include `PROseq`, `GROseq`, `CAGE`, `RAMPAGE`, `CoPRO`, `GROcap`, `PROcap`, `csRNAseq`, `STRIPEseq`, etc.
- `--aligner`: Choose between `bwa` (default), `bwamem2`, `dragmap`, `bowtie2`, `hisat2`, or `star`.
- `--with_umi`: Enable this if your data contains Unique Molecular Identifiers for deduplication.
- `-resume`: Use this to restart a run from the last successful step.

## Outputs
Results are saved in the directory specified by `--outdir`.

- **MultiQC**: The `multiqc/` folder contains the `multiqc_report.html` file, which should be the first file inspected for an overview of QC metrics from FastQC, fastp, alignment, and tool-specific statistics.
- **Alignments**: Sorted and indexed BAM files.
- **Coverage**: `bedGraph` and `bigWig` files for visualization in genome browsers.
- **Transcript Identification**: Results from HOMER, groHMM, or PINTS depending on the configuration.
- **Quantification**: Gene and transcript counts generated by featureCounts.

For a complete description of the directory structure and file formats, refer to the official [output documentation](https://nf-co.re/nascent/output).
