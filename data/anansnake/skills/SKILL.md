# anansnake

## Overview

anansnake is a Snakemake-based workflow designed to bridge the gap between processed sequencing data and advanced gene regulatory network analysis. It automates the execution of ANANSE, which predicts enhancer-gene interactions and calculates transcription factor influence by comparing different cellular states. By leveraging the outputs of seq2science (such as count tables and peak files), anansnake provides a reproducible path to determine which transcription factors are responsible for driving phenotypic changes between samples.

## Usage and Best Practices

### Environment Setup
Install anansnake using Mamba to ensure all dependencies, including the specific ANANSE versions and Snakemake, are correctly resolved.

```bash
mamba create -n anansnake -c bioconda anansnake
mamba activate anansnake
```

### Data Integration Logic
anansnake relies on a shared metadata column to link RNA-seq and ATAC-seq datasets.
- Ensure both your RNA-seq and ATAC-seq `samples.tsv` files contain a matching column (commonly named `anansnake`).
- This shared column allows the workflow to pair transcriptomic profiles with chromatin accessibility profiles for the same biological condition.
- Input files should include the counts tables generated by upstream processing (like seq2science) without manual modification.

### Execution Patterns
Since anansnake is built on Snakemake, use standard Snakemake flags to control the execution. It is recommended to run a dry-run first to verify the DAG (Directed Acyclic Graph).

```bash
# Dry-run to verify workflow
snakemake -n

# Execute the workflow with a specific number of cores
snakemake --cores 8
```

### Resource Management and Optimization
ANANSE is computationally demanding, particularly regarding memory (RAM).
- **Memory Constraints**: If the system freezes or processes are killed, reduce the number of concurrent jobs or threads.
- **Thread Allocation**: Use the `--resources` flag in Snakemake to cap memory usage if running on a cluster or a shared high-performance computing (HPC) node.
- **Large Datasets**: For analyses involving many contrasts, consider running specific targets rather than the entire workflow at once to manage disk space and intermediate file generation.

### Troubleshooting Common Issues
- **Sample Mismatches**: If the workflow fails to find RNA or ATAC samples, verify that the identifiers in the `anansnake` column of your sample tables are identical and that the file paths in your configuration point to the correct directories.
- **Version Compatibility**: Ensure you are using a compatible Miniconda/Mamba version, as some older environments may struggle with the specific dependency tree required for ANANSE 0.4+.
