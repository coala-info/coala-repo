# circtools

## Overview
`circtools` is a modular Python-based framework designed to unify circular RNA (circRNA) bioinformatics workflows. It acts as a "one-stop-shop" for researchers, providing a consistent command-line interface to handle everything from initial detection in RNA-seq data to the design of primers for laboratory verification. Use this skill to navigate the multi-stage process of identifying back-splice junctions, assessing mapping quality, and performing downstream functional or statistical analysis.

## Core Modules and CLI Usage

The tool follows a subcommand structure similar to `samtools` or `bedtools`.

### 1. Detection and Quality Control
*   **detect**: The primary entry point. It interfaces with DCC to identify circRNAs.
    *   *Usage*: `circtools detect [DCC_options]`
    *   *Tip*: All parameters are passed directly to DCC. Ensure your STAR mapping includes the `--chimOutType Junctions` flag, as DCC relies on chimeric junction files.
*   **quickcheck**: Generates PDF reports to assess the quality of a DCC run.
    *   *Usage*: `circtools quickcheck -l <STAR_log_dir> -d <DCC_results_dir>`
    *   *Output*: Visualizations of circRNA counts and mapping statistics across samples.

### 2. Structural Analysis
*   **reconstruct**: Interfaces with FUCHS to determine the internal structure of circRNAs.
    *   *Usage*: `circtools reconstruct [FUCHS_options]`
    *   *Requirement*: Requires the output directory generated by the `detect` module.
*   **exon**: Analyzes differential exon usage within circRNA candidates using the `ballgown` R package.
    *   *Workflow*: Combine DCC/CircTest data with StringTie or Cufflinks output.

### 3. Statistical and Functional Analysis
*   **circtest**: Performs statistical testing for differential expression of circRNAs.
    *   *Usage*: `circtools circtest [options]`
    *   *Benefit*: Automates the grouping of samples and execution of the CircTest R package without requiring manual R scripting.
*   **enrich**: Identifies circRNAs enriched for specific RNA Binding Proteins (RBP).
    *   *Input*: Requires DCC-identified circRNAs and processed eCLIP data (e.g., from ENCODE).

### 4. Experimental Validation
*   **primer**: Designs divergent primers for PCR validation of circRNA candidates.
    *   *Usage*: `circtools primer [options]`
    *   *Feature*: Provides visualization of the designed primers relative to the back-splice junction.

## Expert Tips and Best Practices
*   **Environment Management**: `circtools` requires a hybrid environment. While the main framework is Python 3, the `detect` and `reconstruct` modules require a Python 2.7 installation in the background. Using Conda is the recommended way to manage these dependencies.
*   **R Dependencies**: Modules like `circtest`, `exon`, and `primer` rely on R and BioConductor. Ensure R is in your path and that you have write permissions for R libraries if the tool needs to install missing packages during setup.
*   **Input Preparation**: For the `detect` module, ensure your genome annotations (GTF) and reference FASTA files match the ones used during the initial mapping (e.g., STAR) to avoid coordinate mismatches.
*   **Memory Considerations**: When running `detect` on large cohorts, DCC can be memory-intensive. Monitor your resource usage when processing many samples simultaneously.
