# mmult

## Overview
MMULT is a scalable suite of tools optimized for processing DNA methylation data from hundreds of samples. It provides three primary submodules to handle different statistical workflows: `bbglm` for group comparisons using beta-binomial models, `cpgcdifenrich` for consolidating differences via KL-divergence, and `vmcvmrnme` for identifying variability within a single biological group. Use this skill to execute these submodules with appropriate parameters for chromosome splitting, depth filtering, and statistical thresholds.

## Submodule Workflows

### 1. Differential Methylation with BBGLM
Use `bbglm` to resolve DNA methylation dynamics between multiple groups. It uses a beta-binomial generalized linear model and is highly efficient for whole-genome scales.

**Common Pattern:**
```bash
bbglm -m group1_rep1.bed,group1_rep2.bed -m group2_rep1.bed,group2_rep2.bed -o results.txt
```

**Key Parameters:**
- `-m | --methfile`: Specify methylation BED files (generated by MOABS `MCALL`). Replicates within a group are comma-separated; use multiple `-m` flags for different groups.
- `-c | --chrom`: Specify a chromosome (e.g., `chr1:248956422`). Including the size allows for batch splitting.
- `-l | --length`: Split length for coordinates (default: 20,000,000). Reduce this if memory is limited when handling many replicates.
- `--qval`: Q-value threshold for DMC (default: 0.05).
- `--nominaldiff`: Minimum methylation difference (default: 0.2).

### 2. Consolidating Differences with CpGCDIFEnrich
Use `cpgcdifenrich` to consolidate methylation differences from individual comparisons (generated by MOABS `MCOMP`) using KL-divergence.

**Common Pattern:**
```bash
cpgcdifenrich -c Comparison1.txt -c Comparison2.txt -o consolidated_output.txt
```

**Key Parameters:**
- `-c | --compfile`: Input comparison files.
- `--kldthr`: KL-divergence threshold for a DMC (default: 0.67957).
- `--cdifthr`: Credible methylation difference (CDIF) threshold (default: 0.2).
- `--maxzerocdif`: Maximum percentage of zero CDIFs allowed for a DMC (default: 0.05).

### 3. Variability Analysis with VMCVMRNME
Use `vmcvmrnme` to detect variable methylated CpGs (VMCs) and regions (VMRs) within a single biological group, useful for subtype detection.

**Common Pattern:**
```bash
vmcvmrnme -m r1.bed,r2.bed,r3.bed -o variability_results.txt
```

**Key Parameters:**
- `-v | --vmrmethod`: `0` for identifying VMCs first then VMRs (default); `1` for fixed-size window genome scanning.
- `-w | --window`: Window size for genome scan (default: 150).
- `-s | --sd`: Standard deviation threshold for VMC (default: 0.2).
- `-n | --nme`: Normalized entropy (NME) threshold for VMC (default: 0.25).
- `--minsample`: Minimum number of samples required for a CpG site (default: 5).

## Expert Tips
- **Memory Management**: For large-scale datasets (hundreds of samples), always use the `-c` flag with chromosome sizes and adjust `-l` to process the genome in smaller batches to prevent OOM (Out of Memory) errors.
- **Thread Optimization**: Balance `-t` (read threads) and `-b` (batch threads) based on your CPU core availability. For `bbglm`, the default is 10 read threads and 5 batch threads.
- **Input Requirements**: Ensure input BED files are formatted correctly by MOABS (`MCALL` for methylation files, `MCOMP` for comparison files) as MMULT is designed to work within that ecosystem.
