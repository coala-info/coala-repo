# 10x_bamtofastq

## Overview

The `10x_bamtofastq` utility is a specialized tool designed to reconstruct original sequencing data from BAM files generated by 10x Genomics pipelines. Unlike standard BAM-to-FASTQ converters, this tool specifically handles the extraction of 10x-specific metadata—such as 10x Barcodes (BX) and UMIs (RX)—stored in BAM tags, placing them back into the correct FASTQ headers or separate files. This ensures the resulting FASTQs are fully compatible with re-running 10x pipelines like Cell Ranger or Space Ranger.

## Usage Guidelines

### Basic Command Structure
The tool requires an input BAM file and a destination directory for the output FASTQs.
```bash
bamtofastq [options] <input_bam> <output_directory>
```

### Handling Legacy Data
For BAM files produced by older versions of 10x pipelines, you must specify the pipeline version to ensure correct tag parsing:
- **Cell Ranger 1.0 - 1.1**: Use the `--cr11` flag.
- **Long Ranger 2.0**: Use the `--lr20` flag.
- **GemCode (Long Ranger 1.0 - 1.3)**: Use the `--gemcode` flag.

### Filtering and Subsetting
To reduce processing time or focus on specific data subsets, use the following filters:
- **By Locus**: Extract only reads mapping to a specific region (e.g., `--locus=chr1:1000000-2000000`).
- **By Barcode**: Provide a text file containing a list of Barcodes (BX) to include only those specific cells or molecules (e.g., `--bx-list=barcodes.txt`). Note: This requires a BX-sorted and indexed BAM.

### Output Management
- **Read Groups**: If the BAM contains `@RG` headers, the tool automatically creates separate subdirectories for each read group, preserving the original library structure.
- **Chunking**: By default, the tool splits FASTQs into chunks of 50 million reads. Adjust this using `--reads-per-fastq=N` based on your filesystem's performance or downstream pipeline requirements.

## Expert Tips and Best Practices

### SRA Data Compatibility
If downloading 10x data from the Sequence Read Archive (SRA), you must download the **"Original format"** BAM (Type = TenX). Standard SRA-processed BAMs often strip the hidden tags (RX, QX, BC, etc.) required by `bamtofastq` to reconstruct the original sequences.

### Header Metadata
The tool relies on specific `@CO` (comment) lines in the BAM header that describe the sequence layout. If you are working with a custom-processed BAM and conversion fails, verify that the 10x-specific comment lines (e.g., `10x_bam_to_fastq:R1(...)`) are still present in the header.

### Performance
`bamtofastq` is written in Rust and is highly efficient. For maximum throughput, ensure you are using the pre-compiled release binary rather than a debug build, and point the output directory to a high-performance scratch filesystem (like Lustre or GPFS) to handle the high I/O volume of FASTQ generation.
