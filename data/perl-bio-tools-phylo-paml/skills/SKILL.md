# perl-bio-tools-phylo-paml

## Overview
This tool provides a BioPerl-based interface for interpreting the complex, text-heavy output generated by the PAML suite. It transforms raw PAML text into structured objects, allowing for the automated extraction of evolutionary parameters without the need for fragile custom regular expressions. It is particularly useful for high-throughput comparative genomics workflows where multiple PAML runs must be summarized.

## Usage Instructions

### Basic Parser Initialization
To process a PAML output file, initialize the `Bio::Tools::Phylo::PAML` object specifying the file and the specific PAML program that generated it.

```perl
use Bio::Tools::Phylo::PAML;

my $parser = Bio::Tools::Phylo::PAML->new(
    -file   => "codeml.out",
    -dir    => "./", # Optional: directory for auxiliary files
);
```

### Iterating Through Results
PAML files can contain multiple analysis results (e.g., from multiple tree searches or site models). Always use a loop to ensure all data is captured.

```perl
while (my $result = $parser->next_result) {
    # Access result-level data
    my $model = $result->get_modeldata;
    print "Likelihood: " . $result->get_loglikelihood . "\n";
}
```

### Extracting dN/dS (Omega) Values
For `codeml` or `yn00` runs, extracting the omega value is a primary task.

```perl
# For yn00 or simple codeml models
my $result = $parser->next_result;
my @statistics = $result->get_statistics; 
# Returns a hash or array depending on the specific PAML program
```

### Working with Phylogenetic Trees
The parser can extract the reconstructed trees (including branch lengths or dN/dS labels) as Bio::Tree::Tree objects.

```perl
my $tree = $result->next_tree;
print $tree->as_text('newick');
```

### Expert Tips
- **Memory Management**: For very large PAML files (e.g., site models with many categories), ensure you are processing results one by one using `next_result` rather than trying to load an entire multi-run file into an array.
- **Version Compatibility**: PAML output formats occasionally change between versions (e.g., 4.8 vs 4.9). If parsing fails, verify that the PAML version matches the BioPerl module's expected format.
- **Error Handling**: Wrap the parser in a `eval` block or use `Try::Tiny` to catch malformed PAML files which can occur if a PAML run was interrupted or crashed.
