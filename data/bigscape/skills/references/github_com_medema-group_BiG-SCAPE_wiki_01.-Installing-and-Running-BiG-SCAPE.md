## User instructions

There are a few different ways to install and run BiG-SCAPE 2, depending on your needs. We recommend following the conda/mamba installation option. If this is not possible for you, we also provide a alternative installation option using Docker.

---

### Prerequisites

Software:

* python 3.11 or up
* conda/mamba

Data:

* phmm([profile hidden Markov Models](https://www.ebi.ac.uk/training/online/courses/pfam-creating-protein-families/what-are-profile-hidden-markov-models-hmms/)) database, commonly [Pfam](http://pfam.xfam.org/), which can be downloaded [here](https://ftp.ebi.ac.uk/pub/databases/Pfam/current_release/Pfam-A.hmm.gz).

See more details [here](02.-BiG-SCAPE-Workflows#using-pfamphmm-databases-for-domain-detection).

---

### Option 1: Using Conda/Mamba

We assume that you already have a (mini)conda/(micro)mamba package manager installed. These steps use `mamba`, for conda/miniconda/micromamba simply replace `mamba` with your package manager of choice.

1. `mamba env create -n bigscape -c conda-forge -c bioconda bigscape`
2. `mamba activate bigscape`

You can now run BiG-SCAPE 2 anywhere:

`bigscape --help`

or

`bigscape [running mode] [options]`

specifically, for bigscape cluster:

`bigscape cluster -i /path_to_input_directory -o /path_to_output_directory -p /path_to_phmm_database_file [options]`

see the [tutorials](https://github.com/medema-group/BiG-SCAPE/wiki/12.-Tutorials), [workflows](https://github.com/medema-group/BiG-SCAPE/wiki/02.-BiG-SCAPE-Workflows), and [parameters](https://github.com/medema-group/BiG-SCAPE/wiki/13.-parameters) pages for detailed descriptions of all the BiG-SCAPE run options.

Note: Working on a macbook with a M1/M2 chip? follow these (or analog) instructions: <https://stackoverflow.com/questions/71515117/how-to-set-up-a-conda-osx-64-environment-on-arm-mac>

Alternatively, to install from source, clone this repository:

Note: cloning the most recent version of this repo will likely get you a working version, but we advise to always use a stable release.

1. `git clone https://github.com/medema-group/BiG-SCAPE`.
2. `cd BiG-SCAPE`
3. `mamba env create -f environment.yml`
4. `mamba activate bigscape`
5. `pip install .`

---

### Option 2: Using Docker

We assume you have a working version of Docker installed, and good knowledge of how Docker works.

Run BiG-SCAPE through docker using the docker run command:

```
docker run \
    --volume your_root_data_dir:/home/data \
    --detach=false \
    --rm \
    --user=$(id -u):$(id -g) \
    ghcr.io/medema-group/big-scape:2.0.0-beta.6 \
    # arguments from here are the same as using bigscape.py normally
    cluster \
    -i /home/data/your_input \
    -o /home/data/output_folder \
    -p /home/data/pfam_folder/Pfam-A.hmm
```

`your_root_data_dir` must be a parent folder of your input, your profile hmm database (commonly, Pfam) and where you want to put your output.

For example:

```
/home/example/data
  ├ /input
  |    ├ genome_1/
  ├ /output
  └ /pfam
      └ Pfam-A.hmm
```

Can use a command as such:

```
docker run \
    --volume /home/example/data:/home/data \
    --detach=false \
    --rm \
    --user=$(id -u):$(id -g) \
    ghcr.io/medema-group/big-scape:latest \
    cluster \
    -i /home/data/input/genome_1 \
    -o /home/data/output/genome_1\
    -p /home/data/pfam/Pfam-A.hmm
```

---

## Dev Instructions

We appreciate any contributions!
In order to setup a development environment, please follow the following steps:

### Using mamba

1. Install (micro)mamba
2. `git clone https://github.com/medema-group/BiG-SCAPE`.
3. `cd BiG-SCAPE`
4. `mamba env create -f environment.yml`
5. `mamba activate bigscape`
6. `pip install -e .[dev]`
7. `pre-commit install`

To run BiG-SCAPE:
`python bigscape.py --help`

You can use the `bigscape --help` runnable directly, but this must be re-installed each
time you make a change to the codebase.

To run tests:
`python -m pytest`

When developing, make sure to use the mamba environment you created (called bigscape).
