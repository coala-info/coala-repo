This page will guide you through the transition to version 1.0 of `commec`. It also contains the command-line reference for version 0.3.

## Transitioning to version 1.0

If you have been using commec already, you may be excited to see the new features offered in the 1.0 release! Here you will find some quick-start information to help orient those who are already familiar with using previous versions of commec.

#### New output formats:

commec now outputs a `.output.json`, an `.html` file, and a `.log`, instead of just a `.screen` text file.

* `.output.json` is a JSON format text file, which contains the core output data of commec screen, in a human readable, albeit dense, format. It is also handy to pass off to any other custom scripts you may use.
* `.screen.log` is the formatted log, also printed to console, typcially useful for in depth debugging (with --verbose) or just checking on the final output in a short-form table. This output is the most similar to the previous .screen file, however is more organised at a per-query level.
* `_summary.html` Provides a graphical output, highlighting which regions of your input queries were flagged, and/or subsequently cleared.

#### New Input formats:

As well as familarising yourself with commecs output formats, it is also important to consider potential changes to the input databases setup, as well as input query `.fasta` files.

* `.fasta` files support multiple queries in a single file. Each query will be treated independantly throughout the commec screen pipeline.
* The standard commec database folder layout has changed, and the download location of the latest commec biorisk and low-concern databases has switched to github. The databases are easily downloaded to a desired location using the `commec setup` command.
* The standard input .yaml config file setup has change, invalid keys will result in warnings in commec screen, but it is recommend you regenerate any .yaml config files you use to match the keys in the [defaults example](https://github.com/ibbis-bio/common-mechanism/blob/main/commec/screen-default-config.yaml).
* The default nucleotide database for commec is now `nt_core`. This database is smaller, and faster than nt, whilst still retaining the sensitivity requirements for commec screen.

#### Altered wordings

To better facilitate understanding, several terms in commec have changed. The Benign Step, has been renamed to the Low-concern Step. Similarly the benign databases are databases of Low-concern.

`--fast-mode` has been rebranded as `--skip-tx`, to better reflect that this is infact a truncated version of the commecs screen pipeline, which skips taxonomy searches.

#### commec flag

the `commec flag` command now parses the new `.output.json` files, instead of the previous `.screen` log files, and forms a single `.csv`. If you need to run flag on `.screen` and `.json` files, there is a helper script: dev\_scripts/flag-legacy.py, which can do this for you.

## v0 Command Line Usage

You will find the current [Command Line Usage](https://github.com/DNA-Screening/CommonMechanism/wiki/Command-Line-Usage) on another page.

### screen arguments

The arguments for `flag`, `setup`, and `split` were unchanged from v0 to v1.

| Argument | Type | Description |
| --- | --- | --- |
| fasta\_file | Required | FASTA file to screen. Should only contain a single query |
| -d DATABASE\_DIR, --databases DATABASE\_DIR | Required (unless -y specified) | Path to directory containing reference databases (e.g. taxonomy, protein, HMM) |
| -y CONFIG\_YAML, --config CONFIG\_YAML | Required (unless -d specified) | Configuration for screen run in YAML format, including custom database paths |
| **Screen run logic options** |  |  |
| -f, --fast | Optional flag | Run in fast mode and skip taxonomy homology search |
| -p {blastx,diamond}, --protein-search-tool {blastx,diamond} | Optional | Tool for protein homology search to identify regulated pathogens (default: blastx) |
| -n, --skip-nt | Optional flag | Skip nucleotide search (regulated pathogens will only be identified based on protein hits) |
| **Parallelisation options** |  |  |
| -t THREADS, --threads THREADS | Optional | Maximum number of CPU threads to use. Passed to search tools. (default: 1) |
| -j DIAMOND\_JOBS, --diamond-jobs DIAMOND\_JOBS | Optional, diamond-only | Number of runs to do in parallel on split Diamond database (to increase speed, DIAMOND databases have been split into several parts) |
| **Output options** |  |  |
| -v, --verbose | Optional flag | Output verbose (i.e. DEBUG-level) logs |
| -o OUTPUT\_PREFIX, --output OUTPUT\_PREFIX | Optional | Prefix for output files. Can be a string (interpreted as output basename) or a directory (files will be output there, names determined from input FASTA). Defaults to location of input FASTA. |
| -c, --cleanup | Optional flag | Delete intermediate output files for run |
| -F, --force | Optional flag | Overwrite any pre-existing output for run (cannot be used with --resume) |
| -R, --resume | Optional flag | Re-use any pre-existing output run (cannot be used with --force) |
