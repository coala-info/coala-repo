# gerp

## Overview
GERP is a computational tool designed to identify genomic regions that have been conserved throughout evolution. By comparing observed substitution rates in a multiple sequence alignment against neutral expectations (based on a provided phylogenetic tree), it calculates "rejected substitutions" (RS) scores. High RS scores indicate strong evolutionary constraint, suggesting that the region has a functional biological role.

## Usage Guidelines

### Core Workflow
The standard GERP++ implementation typically involves two main executable steps: `gerpcol` and `gerpelem`.

1.  **Generate Column-wise Scores (`gerpcol`)**:
    *   Input: A multiple sequence alignment (MSA) in .mfa or .fa format and a phylogenetic tree file (Newick format) with neutral branch lengths.
    *   Output: A `.rates` file containing the neutral rate, observed rate, and RS score for every column in the alignment.
    *   *Tip*: Ensure your neutral tree is representative of the species in your alignment to avoid inflated RS scores.

2.  **Identify Constrained Elements (`gerpelem`)**:
    *   Input: The `.rates` file generated by `gerpcol`.
    *   Output: A `.elems` file listing the coordinates of identified constrained elements, their total RS scores, and p-values.
    *   *Tip*: Use the `-f` flag to specify a false-discovery rate or the `-s` flag to set a minimum score threshold for defining an "element."

### Common CLI Patterns
*   **Basic Scoring**:
    `gerpcol -t neutral.tree -f alignment.mfa`
*   **Element Prediction with Custom Threshold**:
    `gerpelem -f alignment.rates -s 10.5`
*   **Excluding Specific Species**:
    Use the `-x` flag in `gerpcol` to exclude specific species from the calculation without regenerating the alignment.

### Best Practices
*   **Neutral Tree Calibration**: The accuracy of GERP scores is highly dependent on the neutral tree. Use four-fold degenerate sites or ancestral repeats from the same alignment to estimate neutral branch lengths.
*   **Alignment Quality**: Purifying selection signals can be masked by poor alignment. Mask repetitive elements or low-complexity regions before running GERP.
*   **Interpretation**: Positive RS scores indicate constraint (fewer substitutions than expected), while negative scores may indicate faster-than-neutral evolution (though GERP is optimized for detecting constraint).
