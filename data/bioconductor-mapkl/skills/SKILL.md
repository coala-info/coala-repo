# bioconductor-mapkl

name: bioconductor-mapkl
description: Hybrid gene selection method using multiple hypothesis testing and Affinity Propagation (AP) clustering. Use this skill to identify a small set of non-redundant gene exemplars from high-throughput biological data, build SVM classification models, and perform network/pathway analysis on selected genes.

## Overview
The `mAPKL` package implements a hybrid feature selection approach. It first filters genes using multiple hypothesis testing (maxT) to rank differential expression. It then uses the Krzanowski and Lai index to determine an optimal cluster number and applies Affinity Propagation (AP) clustering to identify "exemplars"—representative genes that capture the biological signal while reducing redundancy. This is particularly effective for creating robust classifiers from gene expression data.

## Core Workflow

### 1. Data Preparation and Sampling
The package works with `eSet` objects. Use the `sampling` function to split data into training and validation sets.

```R
library(mAPKL)
library(mAPKLData)
data(mAPKLData)

# Split data: 60% training, 40% validation
# classLabels refers to the column in phenoData defining groups
breast <- sampling(Data=mAPKLData, valPercent=40, classLabels="type", seed=135)
trainSet <- breast$trainData
testSet <- breast$testData
```

### 2. Preprocessing and Normalization
The `preprocess` function provides multiple normalization options (mean-centering, z-score, quantile, cyclic loess) with or without log2 transformation.

```R
normTrain <- preprocess(trainSet)
normTest <- preprocess(testSet)

# Select a specific method (e.g., cyclic loess + log2 transformation)
exprs(trainSet) <- normTrain$clL2.normdata
exprs(testSet) <- normTest$clL2.normdata
```

### 3. Gene Selection (mAPKL)
The core function identifies the exemplars. The `dataType` parameter specifies which normalization was used (refer to `attributes(normTrain)` for the index).

```R
# Perform the hybrid selection
out.clL2 <- mAPKL(trObj = trainSet, classLabels = "type", 
                  valObj = testSet, dataType = 7)

# Access identified exemplars
exemplars <- out.clL2@exemplars
```

### 4. Classification and Evaluation
Build and test an SVM classifier using the identified exemplars.

```R
# Build SVM and predict on test set
clasPred <- classification(out.clL2@exemplTrain, "type", out.clL2@exemplTest)

# Results include AUC, Accuracy, MCC, Specificity, and Sensitivity
```

## Advanced Analysis

### Annotation and Pathways
Convert probe IDs to gene symbols and perform pathway analysis using Reactome.

```R
# Annotate probe IDs
gene.info <- annotate(out.clL2@exemplars, "hgu133plus2.db")
print(gene.info@results)

# Identify relevant pathways
pathways <- probes2pathways(gene.info)
```

### Network Characteristics
Analyze the network properties of the selected genes, including centralities (degree, closeness, betweenness) and transitivity.

```R
# Compute network attributes
net.attr <- netwAttr(out.clL2)

# Identify hub genes (local degree > global mean + SD)
sdev <- sd(net.attr@degree$WdegreeL)
msd <- net.attr@degree$WdegreeG + sdev
hubs <- net.attr@degree$WdegreeL[which(net.attr@degree$WdegreeL > msd)]
```

### Reporting
Generate a comprehensive HTML report of the entire analysis.

```R
report(out.clL2, gene.info, clasPred)
```

## Tips
- **Memory/Time**: The Krzanowski and Lai cluster indexing step can be computationally intensive; ensure your environment has sufficient resources for large datasets.
- **Normalization**: Always inspect the density plots generated by `preprocess()` in your working directory to choose the most appropriate normalization method.
- **Exemplars**: Exemplars are the "cluster centers" in the gene expression space, making them more robust features than simple top-ranked lists.
