# poretools

## Overview

`poretools` is a specialized toolkit designed for the analysis and quality control of Nanopore sequencing data. It operates directly on the HDF5-based FAST5 files generated by Oxford Nanopore's MinKNOW software. Use this skill to convert raw basecalled data into standard bioinformatics formats and to perform diagnostic assessments of sequencing runs, such as monitoring yield over time or evaluating flowcell pore activity.

## Core CLI Usage

### Sequence Extraction
The most common use case is converting FAST5 files into FASTA or FASTQ formats for downstream alignment or assembly.

```bash
# Extract all reads in FASTQ format
poretools fastq /path/to/fast5/directory/

# Extract all reads in FASTA format
poretools fasta /path/to/fast5/directory/

# Extract only the "best" read (2D, or the longer of template/complement)
poretools fastq --type best /path/to/fast5/directory/
```

### Run Statistics and Diagnostics
Use these commands to get a high-level overview of the sequencing run's performance.

```bash
# Generate summary statistics (read counts, N50, total yield)
poretools stats /path/to/fast5/directory/

# Find the single longest read in a dataset
poretools winner /path/to/fast5/directory/

# Extract metadata and timing information for each read
poretools times /path/to/fast5/directory/
```

### Visualizations
`poretools` can generate plots to help visualize run quality. Note that these typically require a graphical environment or saving to a file.

```bash
# Plot a histogram of read lengths
poretools hist /path/to/fast5/directory/

# Plot cumulative yield over the duration of the run
poretools yield_plot /path/to/fast5/directory/ --saveas yield.png

# Visualize flowcell occupancy/activity
poretools occupancy /path/to/fast5/directory/ --saveas occupancy.png
```

## Expert Tips and Best Practices

*   **Recursive Processing**: If your FAST5 files are organized in subdirectories (common in newer MinKNOW versions), always use the directory path as the argument. `poretools` is designed to traverse directories to find `.fast5` files.
*   **Read Type Filtering**: Use the `--type` flag to control which strands are extracted. Options typically include `fwd`, `rev`, `2D`, and `best`. For most modern applications, `best` or `fwd` (template) is preferred.
*   **Filtering by Length**: You can filter reads during extraction to remove short fragments or outliers using `--min-length` and `--max-length`.
*   **Streaming Output**: `poretools` outputs sequences to `stdout` by default. Always redirect to a file (e.g., `> output.fastq`) or pipe into a compressor (e.g., `| gzip > output.fastq.gz`) to save disk space.
*   **Handling Large Datasets**: For very large runs, `poretools stats` can be memory-intensive. Ensure your environment has sufficient RAM or process data in batches if necessary.
