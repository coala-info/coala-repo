# cpstools

## Overview
cpstools is a specialized suite designed for the bioinformatic processing of chloroplast genomes. It streamlines common workflows such as verifying the accuracy of GenBank annotations, identifying the four-part structure of the chloroplast (LSC, SSC, and IR regions), and extracting shared genetic data for downstream evolutionary analysis. It is particularly useful for researchers performing comparative plastid genomics who need to standardize sequence starting points or calculate polymorphism metrics across multiple species.

## Command Line Usage and Best Practices

### Annotation Validation (`gbcheck`)
Use this to ensure GenBank files meet submission standards or to verify internal consistency.
- **Self-check**: `cpstools gbcheck -i genome.gb`
- **Reference-based check**: `cpstools gbcheck -i genome.gb -r reference.gb`
- **Expert Tip**: This tool specifically checks for valid start/stop codons and internal stop codons in CDS regions. Always run this before submitting to NCBI.

### Structural Analysis (`IR` and `Seq`)
Chloroplast genomes are typically circular but represented linearly. Use these tools to standardize your sequences.
1. **Identify Regions**: `cpstools IR -i genome.fasta` (Identifies LSC, SSC, and IR regions > 1000bp).
2. **Standardize Start Point**: Use the `Seq` subcommand to set the first base pair to the start of the LSC region.
   - `cpstools Seq -d ./input_dir -f info.txt -m LSC`
   - **Modes**: `LSC` (adjust start), `SSC` (adjust orientation), `RP` (reverse complement + LSC start).

### Comparative Genomics (`Pi` and `RSCU`)
- **Nucleotide Diversity**: `cpstools Pi -d ./work_dir`
  - Requires `mafft` in your PATH.
  - Automatically extracts shared genes and intergenic spacers (IGS) to calculate Pi values.
- **Codon Usage**: `cpstools RSCU -d ./work_dir -l 300`
  - The `-l` parameter filters out CDS shorter than the specified length (default 300bp) to avoid bias in codon usage statistics.

### Phylogeny Preparation (`phy`)
Quickly generate concatenated alignments for shared genes.
- **CDS Alignment**: `cpstools phy -d ./input_dir -m cds`
- **Protein Alignment**: `cpstools phy -d ./input_dir -m pro`
- This command identifies genes shared across all GenBank files in the directory, sorts them, and merges them into a single file ready for tree-building software.

### Format Conversion (`converse`)
Convert GenBank files to formats required by other popular tools:
- **mVISTA**: `cpstools converse -d ./input_dir -m mVISTA` (For comparative visualization).
- **NCBI Table**: `cpstools converse -d ./input_dir -m tbl` (For Sequin/Table2asn submission).

### Repeat Identification (`SSRs` and `LSRs`)
- **Simple Sequence Repeats**: `cpstools SSRs -i genome.gb -k 10,6,5,4,4,4`
  - The `-k` parameter defines the threshold for mono-, di-, tri-, tetra-, penta-, and hexa-nucleotide repeats.
- **Long Sequence Repeats**: `cpstools LSRs -i reputer_output.txt`
  - Used to process and locate results generated by the Reputer program.
