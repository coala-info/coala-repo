# Build Automation

Libraries and tools help with build automation.

* [1build - Command line tool to frictionlessly manage project-specific commands.](/build-automation/1build-gopinath-langote-github/)
* [air - Air - Live reload for Go apps.](/build-automation/air-cosmtrek-github/)
* [anko - Simple application watcher for multiple programming languages.](/build-automation/anko-guilhermecaruso-github/)
* [gaper - Builds and restarts a Go project when it crashes or some watched file changes.](/build-automation/gaper-maxclaus-github/)
* [gilbert - Build system and task runner for Go projects.](https://go-gilbert.github.io?utm_campaign=awesomego&utm_medium=referral&utm_source=awesomego)
* [gob - Gradle/Maven like build tool for Go projects.](/build-automation/gob-kcmvp-github/)
* [goyek - Create build pipelines in Go.](/build-automation/goyek-goyek-github/)
* [mage - Mage is a make/rake-like build tool using Go.](/build-automation/mage-magefile-github/)
* [mmake - Modern Make.](/build-automation/mmake-tj-github/)
* [realize - Go build a system with file watchers and live to reload. Run, build and watch file changes with custom paths.](/build-automation/realize-tockins-github/)
* [Task - simple "Make" alternative.](/build-automation/task-go-task-github/)
* [taskctl - Concurrent task runner.](/build-automation/taskctl-taskctl-github/)
* [xc - Task runner with README.md defined tasks, executable markdown.](/build-automation/xc-joerdav-github/)
