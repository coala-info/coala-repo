# fmsi

## Overview
FMSI (From Masked Superstring to Index) is a specialized bioinformatics tool designed for the space-efficient representation of k-mer sets. It utilizes masked superstrings and a simplified version of the FM-index (the FMS-index) to achieve high compressibility, often requiring only a few bits per k-mer. This makes it ideal for pangenomic applications and large-scale genomic searches where traditional indexing methods would exceed available RAM.

## Core Workflow

### 1. Pre-processing (Required)
FMSI does not build superstrings itself; it requires a masked superstring as input, typically generated by **KmerCamel**.

```bash
# Generate a masked superstring from a FASTA file
kmercamel compute -k 31 -o /dev/null -M ms.fa yourfile.fa
```

### 2. Indexing
Once you have the masked superstring (`.fa`), create the FMS index. This generates several component files (e.g., `.ac_gt`, `.mask`, `.misc`).

```bash
# Create the index
fmsi index -k 31 ms.fa
```

### 3. Querying
FMSI supports two primary query modes: membership (presence/absence) and lookup (unique identifiers).

**Membership Query:**
Outputs a bitstring for each query line (1 = present, 0 = absent).
```bash
fmsi query -O -q query.fa -k 31 ms.fa
```

**Lookup Query:**
Outputs unique integer IDs (0 to N-1) for present k-mers, or -1 for absent ones.
```bash
fmsi lookup -q query.fa -k 31 ms.fa
```

## Expert Tips and Best Practices

### Performance Optimization
*   **Optimization Flag (`-O`):** Use the `-O` flag during `query` if your masked superstring was optimized (e.g., via `kmercamel optimize`). This is recommended for the best results.
*   **Streaming Speedup (`-S`):** If you require high-speed streaming queries, use the `-S` flag. This increases speed but adds a memory overhead of 1 bit per superstring character.
*   **Memory Usage:** Expect approximately 2.4 to 3 bits per canonical k-mer for human or pangenome data.

### K-mer Set Operations (Experimental)
FMSI can perform set logic directly on indexed superstrings. Note that these operations use compaction, which is computationally expensive.

*   **Union:** `fmsi union -k 31 -p ms1.fa -p ms2.fa -r result.fa`
*   **Intersection:** `fmsi inter -k 31 -p ms1.fa -p ms2.fa -r result.fa`
*   **Difference:** `fmsi diff -k 31 -p ms1.fa -p ms2.fa -r result.fa`
*   **Symmetric Difference:** `fmsi symdiff -k 31 -p ms1.fa -p ms2.fa -r result.fa`

### Exporting Data
To convert an $f$-masked superstring back into a standard OR-masked superstring (the default format), use the export command:
```bash
fmsi export ms.fa
```
