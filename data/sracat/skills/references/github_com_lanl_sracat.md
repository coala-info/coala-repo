Unlike the fasterq-dump program included with the sra-toolkit, sracat does not output the reads in their origianlly submitted order, but rather outputs reads in the order in which they are stored in the SRA file. Usage The sracat program has the following command-line options: [-o &lt;output filename *prefix*&gt;] (by default, output is written to stdout) [--qual] (write both sequence and quality scores in fastq format. By default, only sequence data is written in fasta format) [-z] (zlib-based compression of file-based output; default is no compression) &lt;SRA accession or filename 1&gt; ... (any number of SRA accessions or filenames may be specified -- all output will be concatinated) sracat will read local SRA files that have a .sra file extension or download the read data from NCBI servers.
