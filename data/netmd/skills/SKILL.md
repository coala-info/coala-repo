# netmd

## Overview

NetMD is a computational framework designed to extract shared patterns from an ensemble of molecular dynamics simulations. It transforms complex protein conformations into graph-based embeddings (using the graph2vec algorithm) and applies Dynamic Time Warping to align trajectories. This process allows researchers to identify a "barycenter" or consensus behavior, making it easier to distinguish biologically relevant transitions from simulation artifacts or noise.

## Installation and Setup

NetMD is primarily distributed via Bioconda or Docker.

**Conda Installation:**
```bash
conda install -c conda-forge bioconda::netmd
```

**Docker Usage:**
```bash
docker pull quay.io/biocontainers/netmd:latest
```

## Command Line Usage

The core functionality is accessed via the `netmd` command. It requires residue-contact files as input, which should be pre-extracted from your trajectories using the `GetContacts` package.

### Basic Command Pattern
```bash
netmd -F <input_files> -o <output_directory> -e <entropy_threshold> -c <config_file>
```

### Key Arguments
- `-F`: A space-separated list of residue-residue contact files (.tsv).
- `-o`: The directory where results, including embeddings and alignment plots, will be saved.
- `-e`: Entropy filter for graph edges (0.0 to 1.0). 
    - **Lower values (e.g., 0.1):** Higher sensitivity; captures weaker interactions but may include more noise.
    - **Higher values (e.g., 0.5+):** Prioritizes only the most dynamic and strongest interactions.
- `-c`: Path to the graph2vec configuration file. This file defines embedding dimensions and model parameters.
- `--verbose`: Enables detailed logging of the embedding and alignment process.

## Best Practices and Workflow

### 1. Input Preparation
Before running NetMD, ensure your MD frames are converted into the expected format. NetMD does not read raw .xtc or .dcd files directly; it processes the contact frequencies generated by `GetContacts`.

### 2. Tuning the Entropy Filter
The entropy filter (`-e`) is the most critical parameter for structural resolution. If your consensus trajectory appears too erratic, increase the entropy threshold to filter out transient, low-information contacts. If the model fails to distinguish between states, lower the threshold.

### 3. Handling Outliers
NetMD identifies replicas that deviate significantly from the consensus. After an initial run, check the output directory for pruning logs. You may want to re-run the analysis after removing highly divergent replicas to achieve a cleaner consensus barycenter.

### 4. Docker Path Mapping
When using Docker, always map your local working directory to the container's internal path to ensure the tool can access your input files and write the output.
```bash
docker run --rm -v $(pwd):/data quay.io/biocontainers/netmd netmd -F /data/rep1.tsv /data/rep2.tsv -o /data/results -c /data/config.yml
```
