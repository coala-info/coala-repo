# bioconductor-tfarm

name: bioconductor-tfarm
description: Transcription Factor Association Rule Miner (TFARM). Use this skill to identify and extract relevant association rules between transcription factors (TFs) in genomic regions and compute the Importance Index of specific TFs. This is useful for finding direct or indirect interactions among regulatory factors and evaluating the contribution of specific TFs to co-regulation patterns with a target TF.

## Overview

TFARM (Transcription Factor Association Rule Miner) is designed to mine association rules from genomic data, specifically focusing on the presence/absence of transcription factors in regions of interest (e.g., promoters). It extends standard association rule mining (support, confidence, lift) by introducing an **Importance Index**, which quantifies how much the presence of a specific TF contributes to the prediction of a target TF.

## Core Workflow

### 1. Data Preparation
The input must be a `GRanges` object where metadata columns represent different transcription factors. Values are typically 0 (absent) or 1 (present).

```R
library(TFARM)
data("MCF7_chr1") # Example dataset
# Rows are genomic regions, columns are TFs
```

### 2. Generating Association Rules
Use `rulesGen` to extract rules where the Right-Hand Side (RHS) is a specific target TF.

```R
# target: "TF_NAME=1" (presence) or "TF_NAME=0" (absence)
# support: minimum support threshold (e.g., 0.01)
# confidence: minimum confidence threshold (e.g., 0.6)
# type: TRUE for LHS with only present TFs, FALSE for present/absent
rules <- rulesGen(MCF7_chr1, target = "TEAD4=1", support = 0.005, confidence = 0.62, type = TRUE)
```

### 3. Identifying Candidate Co-regulators
Extract TFs that appear in the Left-Hand Side (LHS) of the generated rules.

```R
tf_names <- names(mcols(MCF7_chr1))
candidates <- presAbs(tf_names, rules, type = TRUE)
present_tfs <- candidates$pres # TFs found in rules
```

### 4. Computing the Importance Index
The Importance Index measures the change in support and confidence when a TF is removed from a rule.

```R
# 1. Get rules containing a specific TF
r_tf <- rulesTF(TFi = "FOSL2=1", rules = rules)

# 2. Get corresponding rules where that TF is removed (set to 0)
r_no_tf <- rulesTF0(TFi = "FOSL2=1", rules_TF = r_tf, rules = rules, 
                    data = MCF7_chr1, target = "TEAD4=1")

# 3. Calculate Importance Index
imp_results <- IComp(TFi = "FOSL2=1", rules_TF = r_tf, rules_noTF = r_no_tf, figures = TRUE)
# imp_results$imp contains the index values for each rule
```

## Visualization and Analysis

### Distribution of Importance
Visualize the Importance Index across multiple TFs using boxplots.
```R
# IMP_Z is a list of importance distributions for various TFs
distribViz(IMP_Z, p_TFs)
```

### Heatmap of TF Pairs
Evaluate the importance of TF combinations (pairs).
```R
# i.heat is generated by passing TF names and a data frame of mean importances
i.heat <- heatI(p_TFs, importance_df)
```

### Principal Component Analysis (PCA)
Validate the Importance Index formula by checking if support and confidence variations contribute equally to variance.
```R
# DELTA is a matrix of support/confidence variations
# TF_Imp is a data frame with mean importance and rule counts
pca_results <- IPCA(DELTA, TF_Imp)
```

## Tips for Success
- **Thresholds:** Support and confidence thresholds are highly dependent on dataset size. Start with low support (e.g., 0.005) if rules are scarce.
- **Target Format:** Ensure the target string matches the metadata column name and state (e.g., `"CTCF=1"`).
- **Outliers:** Use `distribViz` to identify "outlier rules" where a TF is exceptionally important compared to its average contribution.
