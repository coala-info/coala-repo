# bioconductor-swath2stats

## Overview
The `SWATH2stats` package acts as a bridge between the OpenSWATH pipeline and downstream statistical analysis tools. It provides a comprehensive workflow for handling SWATH-MS data, including:
1. **Annotation**: Mapping experimental metadata (conditions, replicates) to raw results.
2. **Quality Control**: Visualizing correlations and coefficients of variation (CV) across runs.
3. **FDR Estimation**: Using a target-decoy strategy to estimate and control False Discovery Rates at the assay, peptide, and protein levels.
4. **Filtering**: Removing decoys, non-proteotypic peptides, or low-confidence identifications.
5. **Conversion**: Formatting data for MSstats, aLFQ, mapDIA, and PECA.

## Core Workflow

### 1. Loading and Annotation
Load the package and your OpenSWATH data. Use a study design data frame to annotate the runs.

```r
library(SWATH2stats)
library(data.table)

# Load data (OpenSWATH output)
data <- data.frame(fread('rawOpenSwathResults.tsv', sep='\t', header=TRUE))

# Define Study Design (Required columns: Filename, Condition, BioReplicate, Run)
# Filename must match the 'align_origfilename' column in the data
study_design <- data.frame(
  Filename = c("run1_file", "run2_file"),
  Condition = c("Control", "Treatment"),
  BioReplicate = c(1, 1),
  Run = c(1, 2)
)

# Annotate and reduce columns for efficiency
data.annotated <- sample_annotation(data, study_design, column_file = "align_origfilename")
data.reduced <- reduce_OpenSWATH_output(data.annotated)
```

### 2. Data Analysis and QC
Assess the consistency of the data across injections.

```r
# Count identifications per run
count_analytes(data.annotated)

# Plot intensity correlation
plot_correlation_between_samples(data.annotated, column.values = 'Intensity')

# Plot variation (CV) across replicates
variation <- plot_variation(data.annotated)
variation[[2]] # Summary table
```

### 3. FDR Estimation and Filtering
Estimate global FDR and identify m-score (q-value) cutoffs to reach target FDR thresholds.

```r
# Estimate overall FDR
fdr_table <- assess_fdr_overall(data.annotated, FFT = 0.7, output = 'Rconsole')

# Find m-score cutoff for 5% protein FDR
m_cutoff <- mscore4protfdr(data.annotated, FFT = 0.7, fdr_target = 0.05)

# Filter data
data.filtered <- filter_mscore_condition(data.annotated, m_cutoff, n_replica = 2)
data.filtered <- filter_proteotypic_peptides(data.filtered)
data.filtered <- filter_on_min_peptides(data.filtered, n_peptides = 2)
```

### 4. Conversion for Downstream Tools
Convert the filtered data into specific formats for statistical testing.

```r
# Most tools require transition-level (disaggregated) data
data.transition <- disaggregate(data.filtered)

# MSstats
msstats_input <- convert4MSstats(data.transition)

# mapDIA
mapdia_input <- convert4mapDIA(data.transition)

# aLFQ
alfq_input <- convert4aLFQ(data.transition)

# PECA (uses peptide-level data)
peca_input <- convert4PECA(data.filtered)
```

## Key Functions Reference
- `sample_annotation()`: Merges study design metadata with SWATH data.
- `assess_fdr_overall()`: Calculates global FDR for assays, peptides, and proteins.
- `mscore4protfdr()`: Determines the m-score threshold needed to achieve a specific protein FDR.
- `filter_mscore_condition()`: Filters for precursors passing an m-score threshold in a minimum number of replicates per condition.
- `disaggregate()`: Converts peptide-level rows into transition-level rows.
- `write_matrix_proteins()`: Generates a protein-level intensity matrix.
