# lollipop

## Overview

Lollipop is a specialized genomic tool designed for wastewater-based surveillance. It solves the problem of "deconvolving" mixed viral signals in wastewater—where multiple variants of concern often overlap—to provide a clear picture of variant prevalence. Unlike simple frequency counting, Lollipop leverages time-series data and kernel-based smoothing to account for the high noise and overdispersion typical of environmental samples. It is a core component of the V-pipe workflow but can be used as a standalone CLI tool for analyzing any viral NGS data where variants share mutations.

## Command-Line Usage

Lollipop operates through three primary subcommands that follow the standard wastewater genomics workflow: preparation, extraction, and analysis.

### 1. Generating Mutation Lists
Before deconvolution, you must define the "signatures" of the variants you are looking for.

```bash
lollipop generate-mutlist [VOC_YAMLs] --output mutlist.tsv --out-pangovars variants_pangolin.yaml
```
*   **Input**: One or more YAML files describing Variant of Concern (VOC) mutations (compatible with COJAC or PHE definitions).
*   **Key Options**:
    *   `-g, --genes [GFF]`: Annotates the list with gene information (useful for downstream visualization).
    *   `-p, --out-pangovars`: Generates a mapping file required for the deconvolution step.

### 2. Extracting Mutations from Samples
Extract mutation frequencies from per-position base counts (typically generated by V-pipe).

```bash
lollipop getmutations from-basecount [BASECOUNT_TSV] --outname sample_mutations.txt --location [NAME] --date [YYYY-MM-DD]
```
*   **Input**: A TSV table of base counts.
*   **Key Options**:
    *   `-m, --muttable`: Path to the mutation helper table generated in the previous step.
    *   `-l, --location` and `-d, --date`: Critical metadata for the time-series kernel to function correctly.
    *   `-a, --based [0|1]`: Specify if the input TSV uses 0-based or 1-based positioning.

### 3. Performing Deconvolution
The final step runs the kernel-based deconvolution on the aggregated mutation data.

```bash
lollipop deconvolute [TALLY_TSV] --variants-config [VAR_YAML] --deconv-config [DEC_YAML] --output results.csv
```
*   **Input**: A tally TSV containing the concatenated results of `getmutations`.
*   **Key Options**:
    *   `-n, --n-cores [INT]`: Enables parallel processing for multiple locations (highly recommended for large datasets).
    *   `-C, --fmt-columns`: Changes output format to one column per variant for easier spreadsheet analysis.
    *   `--out-json`: Generates a JSON file formatted for dashboard uploads (e.g., Cov-Spectrum).
    *   `-s, --seed [INT]`: Sets a random seed to ensure reproducible results across runs.

## Expert Tips and Best Practices

*   **Time-Series Integrity**: Lollipop's strength is its temporal kernel. Ensure that sampling dates are provided in a consistent format (`YYYY-MM-DD`). Missing dates or incorrect chronological order will significantly degrade the accuracy of the abundance curves.
*   **Handling Noise**: Wastewater data is notoriously overdispersed. If results look erratic, use the `--filters` option in the deconvolution step to remove problematic or highly homoplasic mutations that may be confusing the model.
*   **Parallelization**: When processing data from multiple Wastewater Treatment Plants (WWTPs), always use the `-n` (or `--n-cores`) flag. Lollipop re-initializes the random seed per location, ensuring that parallel runs produce results identical to sequential runs.
*   **Memory Management**: For very large timelines, limit the number of sub-threads spawned by linear algebra libraries (like BLAS) if you are already using high `-n` values to avoid thread oversubscription.
*   **Variant Signatures**: Use the `cojac sig-generate` command (from the COJAC toolset) to create the input YAMLs for `generate-mutlist` if you are pulling definitions from Cov-Spectrum or CoVariants.org.
