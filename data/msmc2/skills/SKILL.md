# msmc2

## Overview
msmc2 (Multiple Sequentially Markovian Coalescent 2) is a computational tool designed to reconstruct demographic history from genomic data. It analyzes the distribution of coalescence times between haplotypes to estimate how effective population sizes have changed over time. Unlike its predecessor, msmc2 is optimized for handling larger numbers of haplotypes (up to 16) and provides a more robust framework for estimating population divergence by calculating within-population and across-population coalescence rates separately.

## Installation
The most efficient way to install msmc2 is via Bioconda:
```bash
conda install bioconda::msmc2
```

## Common CLI Patterns

### 1. Estimating Population Size History
To estimate the effective population size over time for a single population, run msmc2 on all available haplotypes or a specific subset.

**Basic run on all haplotypes:**
```bash
msmc2 -o my_population_output input_chr1.txt input_chr2.txt
```

**Run on specific haplotype indices:**
If your input file contains many haplotypes but you only want to use a subset (e.g., indices 0, 1, 2, and 3):
```bash
msmc2 -I 0,1,2,3 -o subset_output input.txt
```

### 2. Cross-Population Analysis (Divergence)
Estimating population separation requires a three-step workflow to calculate the Relative Cross-Coalescence Rate (RCCR).

**Step 1: Estimate within-population rate for Population 1**
```bash
msmc2 -I 0,1,2,3 -o pop1_within input.txt
```

**Step 2: Estimate within-population rate for Population 2**
```bash
msmc2 -I 4,5,6,7 -o pop2_within input.txt
```

**Step 3: Estimate across-population rate**
Use the `-I` flag with pairs (hyphenated) to specify only cross-population comparisons. Use `-s` to skip sites with ambiguous phasing, which is critical for cross-population accuracy.
```bash
msmc2 -s -I 0-4,0-5,1-4,1-5 -o across_pops input.txt
```

*Note: After these runs, use `combineCrossCoal.py` from the `msmc-tools` repository to merge the `.final.txt` files.*

## Parameter Tuning and Best Practices

### Time Segment Patterns (`-p`)
The `-p` option defines the resolution of the hidden Markov model across different time scales.
- **Default**: `1*2+25*1+1*2+1*3` (Good for general human demography).
- **High Resolution**: Increase the number of segments (e.g., `10*1+15*2`) to capture more detail, though this requires more data to avoid overfitting.
- **MSMC1 Compatibility**: To replicate the original MSMC time intervals, use `--quantileBoundaries` combined with `-p 10*1+15*2`.

### Recombination and Mutation (`-r`)
- The `-r` (rho/mu) ratio defaults to 0.25. 
- Unlike the original MSMC, it is rarely necessary to fix the recombination rate with `-R` in msmc2. Let the tool estimate it unless you have a specific reason to fix it based on prior literature.

### Computational Efficiency
- **Threading**: Use `-t` to specify the number of CPU threads. By default, it uses all available CPUs.
- **Iterations**: The default `-i 20` (EM iterations) is usually sufficient for convergence. Check the log output to ensure the likelihood has stabilized.

## Expert Tips
- **Phasing Quality**: The accuracy of recent population size estimates is highly dependent on phasing quality. If using unphased data or poor quality phasing, focus your interpretation on more ancient time scales.
- **Haplotype Limits**: While msmc2 is mathematically accurate for many haplotypes, computational complexity increases significantly. 16 haplotypes is generally considered the practical upper limit for most high-performance computing environments.
- **Input Preparation**: msmc2 requires specific input formats generated by `msmc-tools` (specifically `generate_multi_msmc_input.py`). Ensure your input files are grouped by chromosome.
