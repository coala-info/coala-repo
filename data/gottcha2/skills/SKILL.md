## Overview

GOTTCHA2 is a powerful tool for determining the taxonomic origin of genomic sequences. It's particularly useful in metagenomics and microbial ecology for identifying the source of DNA, such as detecting contamination in samples or understanding the community composition of a microbiome. The tool analyzes sequence data against a database to assign taxonomic labels and infer origins.

## Usage Instructions

GOTTCHA2 is primarily used via its command-line interface. The core functionality involves providing input sequences and specifying output options.

### Core Command Structure

The general structure for running GOTTCHA2 is:

```bash
gottcha2 [options] -i <input_file> -o <output_directory>
```

### Key Options and Parameters

*   `-i <input_file>` or `--input <input_file>`: Specifies the input file containing genomic sequences. This can be in FASTA or FASTQ format.
*   `-o <output_directory>` or `--output <output_directory>`: Defines the directory where GOTTCHA2 will save its results.
*   `-t <threads>` or `--threads <threads>`: Sets the number of CPU threads to use for processing. This can significantly speed up analysis on multi-core systems.
*   `-db <database_path>` or `--database <database_path>`: (Optional) Specifies a custom path to the GOTTCHA2 database. If not provided, it will use the default database.
*   `-m <mode>` or `--mode <mode>`: (Optional) Sets the analysis mode. Common modes include `assign` (default) for taxonomic assignment and `filter` for filtering sequences based on taxonomic criteria.
*   `-p <prefix>` or `--prefix <prefix>`: (Optional) A prefix to add to all output files generated by GOTTCHA2.

### Common Workflow Patterns

1.  **Basic Taxonomic Assignment:**
    To perform a standard taxonomic assignment on a FASTA file:
    ```bash
    gottcha2 -i reads.fasta -o results/
    ```

2.  **Using Multiple Threads for Faster Processing:**
    If you have a multi-core processor, leverage it for faster analysis:
    ```bash
    gottcha2 -i reads.fastq -o results/ -t 8
    ```

3.  **Filtering Sequences by Taxonomic Origin:**
    To filter sequences that belong to a specific taxonomic group (e.g., bacteria):
    ```bash
    gottcha2 -i reads.fasta -o filtered_results/ -m filter --taxonomic_filter bacteria
    ```
    *(Note: The exact syntax for `--taxonomic_filter` might vary; consult the tool's documentation for specific filter criteria.)*

### Expert Tips

*   **Database Management:** Ensure you have the correct GOTTCHA2 database installed and accessible. If you encounter issues, try specifying the database path explicitly using the `-db` option.
*   **Input File Quality:** The accuracy of GOTTCHA2's results heavily depends on the quality of your input sequences. Ensure your FASTA/FASTQ files are properly formatted and contain relevant genomic data.
*   **Output Interpretation:** GOTTCHA2 typically generates several output files, including taxonomic assignments, confidence scores, and potentially filtered sequence files. Familiarize yourself with the output structure to effectively interpret the results. The primary output file for assignments is usually named `assignments.tsv` or similar within the specified output directory.
*   **Resource Allocation:** For large datasets, allocate sufficient threads (`-t`) and ensure your system has adequate memory.
