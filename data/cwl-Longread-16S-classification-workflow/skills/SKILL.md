# Longread 16S classification workflow

## Overview

This [Common Workflow Language (CWL)](https://workflowhub.eu/workflows/1952) workflow provides a standardized pipeline for the quality assessment and taxonomic classification of long-read amplicon sequences, such as those generated by Oxford Nanopore or PacBio platforms. The workflow accepts basecalled FASTQ files as primary input and is designed to handle full-length 16S rRNA sequences.

The processing pipeline consists of three primary stages:
*   **Quality Control:** Initial and post-filtering read quality is assessed using NanoPlot to provide a comprehensive overview of the data profile.
*   **Filtering:** Reads are processed via `fastplong` for quality and length filtering to remove suboptimal sequences before classification.
*   **Taxonomic Profiling:** Species-level abundance is determined using Emu, which is specifically optimized for the error rates and lengths associated with long-read 16S sequences.

To facilitate downstream data management, the workflow automatically organizes output files—including QC reports, filtered reads, and abundance profiles—into dedicated subfolders. This structure ensures that results are ready for immediate use in longitudinal studies or comparative analyses.

## Inputs

| ID | Name | Type | Description |
| --- | --- | --- | --- |
| identifier | identifier used | string | Identifier for this dataset used in this workflow |
| reads | Read file | File[] | Read file in FASTA or FASTQ format (can be gz) |
| reference_db | Reference database | Directory | Reference database used in FASTA format |
| readtype | Read type | enum | Type of read nanopore or pacbio. default nanopore |
| fastq_rich | Fastq rich (ONT) | boolean | Input fastq is generated by albacore, MinKNOW or guppy  with additional information concerning channel and time.  Used to creating more informative quality plots (default false)  |
| threads | Number of threads | int? | Number of threads to use for computational processes |
| skip_read_filter | Skip quality filtering | boolean | Skip quality reporting and filtering. (Default false) |
| disable_quality_filtering | Disable_quality_filtering | boolean? | "Quality filtering is enabled by default. If this option is specified, quality filtering is disabled.  Quality plots will still be generated when skip_read_filter is false. (Default false)"  |
| qualified_quality_phred | Qualified_quality_phred | int? | the quality value that a base is qualified. Default 8 means phred quality >=Q9 is qualified. |
| mean_qual | Mean quality | int? | if one read's mean_qual quality score < mean_qual, then this read is discarded. (Default 10) |
| minimum_length | Minimum length required | int? | Reads shorter will be discarded. (Default 1200) |
| length_limit | Maximum length limit | int? | Reads longer than length_limit will be discarded. (Default 1600) |
| trim_front | Trim_front | int? | Trimming how many bases in front for read. (Default not set, 0) |
| trim_tail | trim_tail | int? | Trimming how many bases in tail for read. (Default not set, 0) |
| cut_front | Cut front | boolean? | Move a sliding window from front (5') to tail, drop the bases in the window if its mean quality < threshold, stop otherwise. Default false |
| cut_tail | Cut tail | boolean? | Move a sliding window from tail (3') to front, drop the bases in the window if its mean quality < threshold, stop otherwise Default false. |
| cut_window_size | Cut window size | int? | The window size option shared by cut_front, cut_tail or cut_sliding. Range: 1~1000. Default 4 |
| cut_mean_quality | Cut mean quality | int? | The mean quality requirement option shared by cut_front, cut_tail or cut_sliding. Range: 1~36. Default 20 |
| start_adapter | start_adapter | string? | The adapter sequence at read start (5'). (Default auto-detect) |
| end_adapter | End adapter | string? | The adapter sequence at read end (3'). (Default auto-detect) |
| adapter_fasta | Adapter fasta | File? | Specify a FASTA file to trim both read ends by all the sequences in this FASTA file. (Default None) |
| disable_adapter_trimming | Disable adapter trimming | boolean | Adapter trimming is enabled by default. If this option is specified, adapter trimming is disabled. Default true |
| output_filtered_reads | Output filtered reads | boolean | Output filtered reads when filtering is applied. (Default false) |
| destination | Output Destination | string? | Optional Output destination used for cwl-prov reporting. |
