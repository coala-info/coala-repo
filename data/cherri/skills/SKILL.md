# cherri

## Overview
CheRRI (Computational Help Evaluating RNA-RNA interactions) is a specialized tool designed to identify functional RNA-RNA interaction sites by assessing their likelihood of occurring in nature. It serves as a high-fidelity filter for interaction sites generated by other prediction algorithms or experimental methods. The tool operates in two primary modes: **train** (for creating new models from interactome data) and **eval** (for scoring the biological relevance of interaction sites using an existing model).

## Installation
The recommended method for installing CheRRI is via Bioconda:
```bash
conda install bioconda::cherri
```

## Core Workflows

### 1. Model Generation (Train Mode)
Use this mode when you have a novel RNA-RNA interactome dataset and want to build a custom predictive model.
- **Data Preparation**: Provide an interactome dataset specifying RRI sites.
- **Replicate Handling**: CheRRI builds a "trusted" set of RRIs by identifying sites that appear across multiple replicates.
- **Overlap Threshold**: Ensure your data allows for overlap checking; the tool validates interactions by checking if they occur in all replicates within a specific threshold.
- **Feature Selection**: The tool automatically generates a feature set to select the most appropriate model during the training process.

### 2. RRI Evaluation (Eval Mode)
Use this mode to filter a set of candidate RRI sites and remove false positives.
- **Model Requirement**: You must specify a model path. You can use pre-trained models or a model generated via the `train` mode.
- **Input Formatting**: Interaction positions in the evaluation set are automatically reformatted to match the "trusted RRI" structure used during training.
- **Output**: The tool evaluates the biological relevance of the submitted sites based on the feature set.

## Expert Tips and Best Practices
- **Replicate Consistency**: For the most accurate "trusted RRI" set, provide as many biological replicates as possible. CheRRI's strength lies in its ability to filter noise by requiring cross-replicate validation.
- **Feature Importance**: After running an evaluation, check the generated feature importance plots (often found in the `plots/` directory or output) to understand which biochemical or structural features are driving the interaction scores.
- **Energy Normalization**: Be aware that CheRRI utilizes energy normalization features to evaluate interactions; ensure your input sequences are high quality to allow for accurate thermodynamic calculations.
- **Error Handling**: If using CheRRI in a pipeline with `IntaRNA`, the tool includes specific error catching for `IntaRNA` outputs to prevent pipeline crashes during feature generation.
