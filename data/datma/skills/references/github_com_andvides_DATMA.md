Installing DATMA NON distributed version (NON COMPSs support) Clone the DATMA directory: git clone https://github.com/andvides/DATMA.git Run the ./install_datma_UBUNTU.sh script: bash install_datma.sh Export the PATH: export PATH=&lt;install_path&gt;/DATMA/tools/bin/:$PATH Distributed version (COMPSs support) Intall COMPSs framework (see the COMPSs manual) Clone the DATMA directory: git clone https://github.com/andvides/DATMA.git Run the ./install_datma_UBUNTU.sh script: bash install_datma.sh Export the PATH: export PATH=&lt;install_path&gt;/DATMA/tools/bin/:$PATH Running DATMA Generate the 16S database index: see &lt;install_path&gt;/DATMA/16sDatabases/README Edit the configBmini.txt: nano &lt;install_path&gt;/DATMA/examples/configBmini.txt Run datma and select version (seq) or (compss): &lt;install_path&gt;/DATMA/runDATMA.sh &lt;install_path&gt;/datma/examples/configBmini.txt Output 16sSeq: Directory with the identified 16S sequences bins: Assembly and annotation of the bins clean: Quality control report round: CLAME executions readsForbin.fastq: Balance of reads without classification binsBlastn.html: Blastn annotation for the contigs in Krona format binsKaiju.html: Kaiju annotation for the contigs in Krona format report.html: Bins report resumenFile.html: Darma summarized report Authors Benavides A(1), Sanchez F (2), Alzate JF (3),(4) and Cabarcas F (1),(3) Grupo Sistemic, Departamento de Ingeniería Electrónica, Facultad de Ingenieria, Universidad de Antioquia.
