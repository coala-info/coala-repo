# plascope

## Overview
PlaScope is a specialized genomic tool designed to distinguish between chromosomal and plasmid DNA in bacterial genome assemblies. It functions as a wrapper around SPAdes (for assembly) and Centrifuge (for classification). By leveraging species-specific databases, it provides a high-confidence method for exploring the plasmidome, particularly in Enterobacteriaceae.

## Usage Modes

### Mode 1: Assembly and Classification
Use this mode when starting from raw paired-end reads. PlaScope will perform the assembly using SPAdes before classifying the contigs.

```bash
plaScope.sh \
  -1 forward_reads.fastq.gz \
  -2 reverse_reads.fastq.gz \
  --sample sample_name \
  --db_dir /path/to/database \
  --db_name database_prefix \
  -o output_directory \
  -t 8
```

### Mode 2: Classification Only
Use this mode if you already have a FASTA assembly generated by SPAdes or Unicycler.

```bash
plaScope.sh \
  --fasta assembly.fasta \
  --sample sample_name \
  --db_dir /path/to/database \
  --db_name database_prefix \
  -a [spades|unicycler] \
  -o output_directory
```

## Database Management
PlaScope requires a specific database structure consisting of three `.cf` files. 

- **E. coli**: Use the `chromosome_plasmid_db` prefix.
- **Klebsiella**: Use the `Klebsiella_PlaScope` prefix.
- **Custom**: You can build your own database using `centrifuge-build` with a custom taxonomy mapping (root, chromosome, plasmid).

## Best Practices
- **Thread Allocation**: Use the `-t` flag to match your environment's capabilities; the default is 8 threads.
- **Assembler Compatibility**: If using Mode 2, ensure the FASTA headers match the expected format of SPAdes (e.g., `>NODE_1_length_...`) or Unicycler.
- **Database Pathing**: The `--db_dir` must be the absolute or relative path to the folder containing the `.cf` files, while `--db_name` is just the filename prefix (do not include the `.1.cf` extension).
