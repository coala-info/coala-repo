# r-velocyto.r

## Overview
The `velocyto.r` package provides tools for estimating RNA velocity from single-cell RNA-seq data. It works by modeling the relationship between unspliced (pre-mRNA) and spliced (mature mRNA) abundance to predict the future state of individual cells. It is commonly used for trajectory inference and studying cellular differentiation.

## Installation
To install the package from GitHub:
```R
library(devtools)
install_github("velocyto-team/velocyto.R")
```

## Core Workflow

### 1. Data Loading
Load spliced and unspliced matrices, typically from a `.loom` file generated by the `velocyto` CLI or other pipelines like `dropEst`.

```R
library(velocyto.r)
# Load from loom file
ldat <- read.loom.matrices("path/to/data.loom")

# Extract matrices
spliced <- ldat$spliced
unspliced <- ldat$unspliced
```

### 2. Preprocessing and Filtering
Filter genes based on expression levels across cells to ensure robust velocity estimation.

```R
# Filter genes
spliced <- filter.genes.by.cluster.expression(spliced, clusters, min.max.cluster.average = 0.5)
unspliced <- unspliced[rownames(spliced), ]
```

### 3. Velocity Estimation
Calculate the velocity for each gene. This involves fitting a steady-state model to the spliced/unspliced ratio.

```R
# Estimate RNA velocity
rvel <- gene.relative.velocity.estimates(spliced, unspliced, deltaT = 1, kCells = 25, fit.quantile = 0.02)
```

### 4. Visualization on Embeddings
Project the high-dimensional velocity vectors onto a low-dimensional embedding (e.g., t-SNE, UMAP, or PCA).

```R
# Project velocity onto an existing embedding (emb)
show.velocity.on.embedding.cor(emb, rvel, n = 200, scale = 'grid', arrow.scale = 2, show.grid.flow = TRUE, min.grid.cell.count = 5)
```

## Key Functions
- `read.loom.matrices()`: Reads spliced, unspliced, and spanning matrices from a loom file.
- `gene.relative.velocity.estimates()`: The main function for calculating velocity using a relative gamma fit.
- `show.velocity.on.embedding.cor()`: Visualizes velocity as arrows or a grid flow on a cell embedding based on correlation.
- `pca.velocity.plot()`: Projects velocity onto PCA space.

## Tips for Success
- **Cell Clustering**: Velocity estimation often benefits from k-nearest neighbor (kNN) smoothing. Ensure you have a valid cell-cell distance matrix or kNN graph.
- **Normalization**: While `gene.relative.velocity.estimates` handles some scaling, ensure your input matrices are comparable (e.g., library size normalized if necessary).
- **Integration**: `velocyto.r` integrates well with `pagoda2` for clustering and embedding.
