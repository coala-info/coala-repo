# rascaf

## Overview

Rascaf (RNA-seq SCAFfolder) is a bioinformatics tool designed to improve draft genome assemblies by using the connectivity information inherent in paired-end RNA-seq data. It identifies candidate connections between contigs where transcript sequences bridge across assembly gaps. The process typically involves two main steps: generating connection candidates from individual RNA-seq libraries and then reconciling those connections to produce a final scaffolded assembly. It is particularly useful for refining gene-rich regions of a genome.

## Usage Patterns

### Core Workflow

The standard workflow consists of running `rascaf` on each BAM file, followed by `rascaf-join` to generate the final assembly.

**1. Generate connections for each library:**
```bash
rascaf -b alignment_1.bam -f assembly.fa -o library_1
rascaf -b alignment_2.bam -f assembly.fa -o library_2
```

**2. Join connections into a final scaffolded assembly:**
```bash
rascaf-join -r library_1.out -r library_2.out -o final_scaffold
```

### Simplified Execution
For a single dataset, the Perl wrapper can automate the sequence:
```bash
perl rascaf-wrapper.pl -b alignment.bam -f assembly.fa -o output_prefix
```

## Command Line Reference

### rascaf (Connection Identification)
- `-b <STRING>`: Path to the coordinate-sorted BAM file (Required).
- `-f <STRING>`: Path to the raw assembly FASTA file (Recommended).
- `-o <STRING>`: Prefix for the output `.out` file (Default: rascaf).
- `-ms <INT>`: Minimum number of reads supporting a connection (Default: 2).
- `-ml <INT>`: Minimum exonic length if no intron is detected (Default: 200).
- `-k <INT>`: K-mer size, maximum 32 (Default: 21).
- `-cs`: Optional flag to output genomic sequences involved in connections to `$prefix_cs.fa`.

### rascaf-join (Scaffold Construction)
- `-r <STRING>`: Path to the `.out` file generated by `rascaf`. Use multiple `-r` flags to merge multiple datasets.
- `-o <STRING>`: Prefix for the output FASTA and info files (Default: rascaf_scaffold).
- `-ms <INT>`: Minimum support alignments required to accept a connection during joining (Default: 2).
- `-ignoreGap`: Flag to ignore estimated gap sizes (number of Ns) between contigs.

## Expert Tips and Best Practices

- **BAM Preparation**: Ensure BAM files are coordinate-sorted. Rascaf relies on the spatial arrangement of reads to determine contig proximity.
- **Manual Filtering**: The `.out` file produced by the first step is a text file. If the assembly is particularly noisy, you can manually edit this file to remove weakly supported or biologically improbable connections before running `rascaf-join`.
- **Interpreting Results**: 
    - The `.fa` file contains the new scaffolded sequences.
    - The `.info` file provides a mapping of how the new scaffolds were constructed from the original contigs, including orientation (`+` or `-`).
- **Multiple Datasets**: When using multiple RNA-seq datasets (e.g., different tissues or conditions), run `rascaf` separately on each and provide all resulting `.out` files to `rascaf-join`. The join step is designed to resolve conflicts between different libraries.
- **Memory and Performance**: For large genomes, ensure sufficient RAM is available for the contig graph construction, especially when using a high number of input libraries.
