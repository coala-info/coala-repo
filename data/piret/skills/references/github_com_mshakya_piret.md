See here for instructions on how to install conda. Use following commands to create conda environments and then install corresponding packages. Also make sure that there is not an environment by the name of piret_env before attempting the installation. Delete the environment if its already present. 1.0 Testing Installation We have provided test data set to check if the installation was successful or not. fastq files can be found in tests/fastqs and corresponding reference fasta files are found in tests/data . To run the test, from within piret directory: For running tests on eukaryote datasets: $ cd piret $ source activate piret_env $LUIGI_CONFIG_PATH="/panfs/biopan01/scratch-311300/ecoli_usda/ecoli.cfg" bin/piret -c ecoli.cfg -d ecoli_piret -e exp_desn.txt $LUIGI_CONFIG_PATH="full_path_to/piret/tests/test_euk.cfg" bin/piret -c tests/test_euk.cfg -d tests/test_euk -e tests/test_euk.txt For running tests on prokarya datasets: $LUIGI_CONFIG_PATH="full_path_to/piret/tests/test_prok.cfg" bin/piret -c tests/test_prok.cfg -d tests/test_prok -e tests/test_prok.txt For running tests using both prokarya and eukarya datasets: $LUIGI_CONFIG_PATH="full_path_to/piret/tests/test_both.cfg" bin/piret -c tests/test_prok.cfg -d tests/test_prok -e tests/test_both.txt For getting KO ids for genes, PiReT uses emapper . The conda install of PiReT also includes emapper. However, its database need to be downloaed following instruction here . Briefly, 0.1 Dependencies PiReT requires following dependencies, all of which should be installed and in the PATH. 0.1.0 Programming/Scripting languages Python &gt;=v3.6.3 The pipeline is not compatible with Python v3.0 or higher. R &gt;=v3.3.1 Perl &gt;=v5.26.2 0.1.1 Installing dependencies conda v4.2.13 If conda is not installed, INSTALL.sh will download and install miniconda , a "mini" version of conda that only installs handful of packages compared to anaconda 0.1.2 Third party softwares/packages samtools (&gt;=v1.6) HiSat2 (&gt;=v2.1.0) featurecount (&gt;=v1.6.3) stringTie (&gt;=v1.3.4d) 0.1.3 R packages edgeR (&gt;=v3.14.0) DEseq2 (&gt;=v1.12.4) ballgown (&gt;=v2.8.0) 0.1.4 Python packages luigi (&gt;=v2.6.1) pandas (&gt;=v0.19.2) plumbum (&gt;=v1.6.3) Biopython (&gt;=v1.68) gffread (&gt;=v0.8.4rc1) 2.0 Running PiReT usage: piret [-h] -d WORKDIR -e EXPDSN -c CONFIG [-v] piret optional arguments: -h, --help show this help message and exit -v, --version show program's version number and exit required arguments: -d WORKDIR working directory where all output files will be processed and written (default: None) -e EXPDSN tab delimited experimental design file -c CONFIG, --config CONFIG luigi config file for setting parameters that control each step, see github repo for an example (default: None) Example runs: piret -d &lt;workdir&gt; -e &lt;design file&gt; -c &lt;config file&gt; 2.1 Experimental design file An experimental design file consist of sample name (SampleID), full path to fastq files (Files), and different groups of your samples (Group). We recommend that you use a text editor like BBedit or TextWrangler to generate the tab delimited experimental design file. Exporting a tab delimited file directly from Excel tend to cause formatting problem. If possible, please avoid any special characters in sample names and group names. For example: samp1, samp_1 : good name samp 1, samp.1: not a good name and will likely cause errors.
