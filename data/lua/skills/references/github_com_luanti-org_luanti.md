Prior to that we used 0.major.minor . Major is incremented when the release contains breaking changes, all other numbers are set to 0. Minor is incremented when the release contains new non-breaking features, patch is set to 0. Patch is incremented when the release only contains bugfixes and very minor/trivial features considered necessary. Since 5.0.0-dev and 0.4.17-dev, the dev notation refers to the next release, i.e.: 5.0.0-dev is the development version leading to 5.0.0. Prior to that we used previous_version-dev .
