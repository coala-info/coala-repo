# deepgoplus

## Overview
DeepGOPlus is a specialized bioinformatics tool that automates the prediction of protein functions by assigning Gene Ontology (GO) terms to amino acid sequences. It utilizes a hybrid approach, merging a deep convolutional neural network (CNN) with sequence similarity scores generated by DIAMOND. This skill should be used when you have protein sequences in FASTA format and need to determine their roles in biological processes, molecular functions, or cellular components.

## Installation and Environment
DeepGOPlus requires a specific environment to function correctly:
- **Python Version**: 3.7.9 is the recommended version.
- **Core Dependencies**: TensorFlow (requires CUDA 10.1 and cuDNN 7.6.5 for GPU acceleration) and the `diamond` alignment tool.
- **External Data**: You must download the model data (data.tar.gz) from the official repository and extract it into a local directory before running predictions.

## Command Line Usage

### Basic Prediction
The primary way to run DeepGOPlus is via the `deepgoplus` command. You must specify the root directory where the model data is stored and the input FASTA file.

```bash
deepgoplus --data-root <path_to_data_folder> --in-file <input_fasta_filename>
```

### Model Maintenance and Updates
To keep the model current with the latest Gene Ontology (GO) and SwissProt database releases, use the update script. This script will automatically download new releases and retrain the model if updates are available.

```bash
python update.py
```

### Performance Evaluation
If you are training or testing the model on custom datasets, use the evaluation scripts to calculate performance metrics such as Fmax, Smin, and AUPR.

```bash
bash new_evaluation.sh
```

## Tool-Specific Best Practices
- **DIAMOND Integration**: Ensure the `diamond` executable is available in your system's PATH, as DeepGOPlus relies on it for the sequence similarity component of its predictions.
- **Data Organization**: Keep your Gene Ontology files in OBO format within the data directory. The tool expects specific structures for `uni2pandas.py` and `deepgoplus_data.py` to process UniProt data into training sets.
- **Input Formatting**: Ensure your FASTA headers are clean and sequences contain standard amino acid characters to avoid parsing errors during the CNN inference phase.
- **GPU Utilization**: Given the TensorFlow dependency, running on a machine with a compatible NVIDIA GPU significantly reduces inference time for large-scale protein datasets.
