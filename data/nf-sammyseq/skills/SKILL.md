# sammyseq

## Overview
SAMMY-seq (Sequential Analysis of MacroMolecules accessibilitY sequencing) is a methodology used to analyze chromatin state and genome compartmentalization. The nf-core/sammyseq pipeline automates the processing of these sequencing libraries, providing a standardized workflow for mapping biochemical properties of chromatin domains.

The pipeline accepts raw FASTQ files (single-end or paired-end) and a reference genome to produce normalized coverage tracks and quality metrics. It supports various aligners and trimmers, and can optionally perform pairwise comparisons between different solubility fractions or identify differential solubility across experimental groups.

## Data preparation
The pipeline requires a comma-separated samplesheet provided via the `--input` parameter. This file defines the relationship between sequencing files, biological specimens, and solubility fractions.

`samplesheet.csv` structure:
- `sample`: Unique identifier for the library (e.g., `CTRL004_S2`).
- `fastq_1`: Path to the first FASTQ file (must be `.fq.gz` or `.fastq.gz`).
- `fastq_2`: Path to the second FASTQ file for paired-end data (optional).
- `experimentalID`: Identifier for the biological specimen (e.g., `CTRL004`).
- `fraction`: The solubility fraction, which must be one of: `S2`, `S2S`, `S2L`, `S3`, or `S4`.
- `sample_group`: Identifier used to group samples belonging to the same biological condition.

Example samplesheet:
```csv
sample,fastq_1,fastq_2,experimentalID,fraction,sample_group
CTRL004_S2,/path/to/S2_R1.fq.gz,,CTRL004,S2,CTRL
CTRL004_S3,/path/to/S3_R1.fq.gz,,CTRL004,S3,CTRL
```

A reference genome is required, provided either via a pre-configured iGenomes ID using `--genome` or a FASTA file using `--fasta`.

## How to run
Run the pipeline using the `nextflow run` command. It is recommended to use `-profile test` to verify the setup before processing full datasets.

Basic execution:
```bash
nextflow run nf-core/sammyseq \
   -profile <docker/singularity/conda> \
   --input samplesheet.csv \
   --fasta reference_genome.fa \
   --outdir ./results
```

To perform specific fraction comparisons (e.g., S2 vs S3) for each experimental ID:
```bash
nextflow run nf-core/sammyseq \
   -profile <profile> \
   --input samplesheet.csv \
   --fasta reference_genome.fa \
   --comparison S2vsS3 \
   --outdir ./results
```

Key parameters:
- `-profile`: Computing environment (e.g., `docker`, `singularity`).
- `--aligner`: Choice of `bwamem` (default), `bwaaln`, or `bowtie2`.
- `--trimmer`: Choice of `trimgalore` (default) or `trimmomatic`.
- `--differential_solubility`: Set to `true` to activate differential solubility analysis.
- `-resume`: Use this flag to restart a pipeline from the last cached step.

## Outputs
Results are saved in the directory specified by `--outdir`.

- **MultiQC**: An aggregate report in `multiqc/` containing quality control metrics from FastQC, alignment statistics, and duplicate marking.
- **Alignment**: Filtered BAM files and indices.
- **Profiles**: Single-track bigWig files generated by deepTools for visualizing genome coverage.
- **Comparisons**: (Optional) Pairwise comparison bigWig tracks generated by `spp` or `bigwigCompare`.
- **Differential Solubility**: (Optional) Regions of differentially enriched solubility.

For a complete description of output files, refer to the official [output documentation](https://nf-co.re/sammyseq/output).
