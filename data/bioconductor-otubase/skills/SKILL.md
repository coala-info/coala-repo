# bioconductor-otubase

## Overview
OTUbase provides a structured framework for analyzing OTU data and taxonomic classifications. It inherits from the `ShortRead` package, allowing for the integration of sequence reads, quality scores, and metadata. It is designed to bridge the gap between bioinformatics processing tools (like Mothur or RDP) and ecological analysis packages in R (like `vegan`).

## Core Workflows

### 1. Importing Data
OTUbase supports two primary data types: OTU-based (clustering sequences by similarity) and Taxonomy-based (classifying sequences into known taxa).

**Importing OTU-based data (Mothur format):**
```r
library(OTUbase)
# level: clustering threshold (e.g., "0.03")
# otufile: Mothur .list file
# samplefile: Mothur .groups file
soginOTU <- readOTUset(dirPath=dirPath, 
                       level="0.03", 
                       samplefile="sogin.groups", 
                       otufile="sogin.list", 
                       sampleADF="metadata.txt")
```

**Importing Taxonomy-based data (RDP format):**
```r
# taxfile: RDP 'fixed' format output
# namefile: Mothur .names file (for unique sequences)
soginTAX <- readTAXset(dirPath=dirPath, 
                       taxfile="sogin.tax", 
                       namefile="sogin.names", 
                       samplefile="sogin.groups")
```

### 2. Accessing Data
OTUbase uses standard accessors to retrieve components of the object:
- `id(obj)`: Sequence IDs.
- `sread(obj)`: DNA sequences (DNAStringSet).
- `quality(obj)`: Quality scores.
- `sampleID(obj)`: Sample identifiers for each read.
- `sData(obj)`: Sample metadata (AnnotatedDataFrame).
- `otuID(obj)`: OTU assignments (for `OTUset`).
- `tax(obj)`: Taxonomic classifications (for `TAXset`).

### 3. Data Analysis

**Generating Abundance Tables:**
The `abundance` function creates a matrix of counts per sample.
```r
# For OTUset
abundOTU <- abundance(soginOTU, weighted=FALSE, collab='Site')

# For TAXset (requires specifying the taxonomic rank)
abundTAX <- abundance(soginTAX, weighted=FALSE, taxCol='genus', collab='Site')
```

**Diversity Estimation:**
Use the `vegan` package on abundance tables generated by OTUbase.
```r
library(vegan)
richness <- apply(abundOTU, 2, estimateR)
```

**Sample Clustering:**
OTUbase provides a wrapper for `vegan`'s distance and clustering functions.
```r
clusterSamples(soginOTU, distmethod='jaccard', clustermethod='complete', collab='Site')
```

### 4. Subsetting
To extract specific samples or OTUs for downstream analysis:
```r
# Extract specific samples by ID
reducedSet <- subOTUset(soginOTU, samples=c("Sample1", "Sample2"))
```

## Tips for Usage
- **Memory Management:** Including `fastafile` and quality files in `readOTUset` or `readTAXset` significantly increases memory usage and slows down import. Only include them if sequence-level analysis is required.
- **Metadata:** Ensure your sample metadata file is formatted as an `AnnotatedDataFrame` or a tab-delimited text file that R can parse into one.
- **Mothur Integration:** OTUbase is highly compatible with Mothur outputs (.list, .groups, .names). Ensure these files are present in the `dirPath`.
