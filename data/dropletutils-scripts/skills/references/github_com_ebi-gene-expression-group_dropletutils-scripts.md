This package provides parameterised scripts providing direct command line access to the functions of that package, to facilitate inclusion in workflows. Install The recommended method for script installation is via a Bioconda recipe called dropletutils-scripts. With the Bioconda channels configured the latest release version of the package can be installed via the regular conda install command: conda install dropletutils-scripts Test installation There is a test script included: dropletutils-scripts-post-install-tests.sh This downloads test data and executes all of the scripts described below. Commands Currently available scripts are detailed below, each of which has usage instructions available via --help. dropletutils-read-10x-counts.R: call Read10xCounts() To read 10x data and create a SingleCellExperiment object comprising sparse matrices: dropletutils-read-10x-counts.R -s &lt;comma-separated list of 10x directories&gt; -c &lt;logical, should columns be named by barcode?&gt; -o &lt;file to store serialized SingleCellExperiment object&gt; dropletutils-downsample-matrix.R: call downsampleMatrix() dropletutils-downsample-matrix.R -i &lt;input SingleCellExperiment in .rds format&gt; -p &lt;single proportion or file with one value per line for each matrix column&gt; -c &lt;logical, should downsampling be done by column?&gt; -o &lt;file to store serialized SingleCellExperiment object&gt; dropletutils-barcoderanks.R: call barcodeRanks() dropletutils-barcoderanks.R -i &lt;input SingleCellExperiment in .rds format&gt; -l &lt;maximum total count below which all barcodes are assumed to be empty droplets&gt; -f &lt;fit bounds&gt; -o &lt;file to store serialized SingleCellExperiment object&gt; -p &lt;file to store barcode plot in&gt; dropletutils-empty-drops.R: call emptyDrops() and produce filtered object if desired dropletutils-empty-drops.R -i &lt;input SingleCellExperiment in .rds format&gt; -l &lt;maximum total count below which all barcodes are assumed to be empty droplets&gt; \ -n &lt;number of iterations&gt; -m &lt;logical, return barcodes with totals less than -l value?&gt; -g &lt;minimum total count required for a barcode to be considered as a potential cell&gt; \ -r &lt;retain&gt; -f &lt;logical, filter the output object?&gt; -d &lt;FDR value below which a droplet is said to contain a cell&gt; -o &lt;file to store serialized SingleCellExperiment object&gt; -t &lt;output text file&gt; About DropletUtils scripts to install as a Bioconda package for making workflows Resources Readme License Apache-2.0 license Uh oh!
