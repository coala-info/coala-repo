# crumble

## Overview

Crumble is a specialized tool for the controlled lossy compression of genomic alignment data. It operates by analyzing alignments using a heterozygous consensus algorithm to determine which base quality values are essential for downstream analysis and which can be discarded or simplified. When a base call is highly confident, Crumble replaces the original quality score with a fixed high or low value, significantly improving compression ratios in CRAM and BAM formats without substantially impacting the ability to recover biological variants.

## Common CLI Patterns

### Basic Compression
To compress a BAM file and output a new BAM file with reduced quality scores:
```bash
crumble input.bam -O bam > output.bam
```

### Multi-threaded Execution
For faster processing on large datasets, use the `nthreads` parameter within the output format option:
```bash
crumble input.bam -O bam,nthreads=8 > output.bam
```

### Preserving Specific Quality Values
If specific quality scores must be kept exactly as they are (e.g., for specific filtering pipelines), use the `-k` (keep) option with ranges or lists:
```bash
# Keep quality scores 10 through 20 and score 30
crumble -k 10-20,30 input.bam -O bam > output.bam
```

### Region-Based Preservation
To ensure quality scores are preserved in specific genomic regions (like known variant sites or clinical targets), provide a BED file:
```bash
crumble -R preserve.bed input.bam -O bam > output.bam
```

### Handling BQSR Tags
Crumble can quantize or remove specific tags like BD (Base Deletion) and BI (Base Insertion) which are often generated by GATK BQSR:
```bash
# Remove BD and BI tags to save space on Illumina data
crumble -T BD,BI input.bam -O bam > output.bam
```

## Expert Tips

*   **Consensus Logic**: Crumble uses the gap5 consensus algorithm. If the algorithm is highly confident in a call, it sets quality values to a fixed high/low state. If the call is ambiguous, it retains the original quality values to allow downstream callers to make their own determinations.
*   **PacBio HiFi Support**: For PacBio HiFi (CCS) data, Crumble includes specific configuration modes to handle the unique error profiles of long-read high-fidelity data.
*   **Output Formats**: While BAM is common, Crumble is particularly effective when the output is piped into a CRAM encoder, as the reduced entropy in the quality scores allows the CRAM format to achieve much higher compression.
*   **Metadata Control**: Use the `-z` flag if you wish to prevent Crumble from adding a `@PG` (Program) line to the BAM/CRAM header.
