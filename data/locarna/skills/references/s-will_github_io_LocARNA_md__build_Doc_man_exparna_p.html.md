USAGE: exparna_p [options] &lt;Input 1&gt; &lt;Input 2&gt; exparna_p performs pairwise matching and folding, i.e. it determines exactly matching local favourable sub-structures that can be simulataneously formed in the input RNAs. Input Input consists of two sequences or alignments, which are specified in fasta, clustal, stockholm, or LocARNA pp format. Optionally, one can specify structure and anchor constraints in these input files. Output The program enumerates exactly matching local substructures (exact pattern matches = EPMs) and optionally chains them. It returns lists of chained and unchained matches, visualizations of the results and anchor constraints for alignment. OPTIONS -h , &ndash;help : Help -V , &ndash;version : Version info -v , &ndash;verbose : Verbose -q , &ndash;quiet : Quiet Scoring parameters: &ndash;no-stacking : Do not use stacking terms (otherwise needs stacking probs by RNAfold -p2 ) &ndash;alpha_1= &lt;factor&gt;(1) : Multiplier for sequential score &ndash;alpha_2= &lt;factor&gt;(5) : Multiplier for structural score &ndash;alpha_3= &lt;factor&gt;(5) : Multiplier for stacking score, 0 means no stacking contribution &ndash;struct-mismatch-score= &lt;score&gt;( -10 ) : Score for a structural mismatch (nucleotide mismatch in an arcmatch) Heuristics for speed accuracy trade off: -p , &ndash;min-prob= &lt;prob&gt;(0.01) : Minimal probability &ndash;max-bps-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #base pairs divided by sequence length (default: no effect) &ndash;max-uil-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #unpaired bases in loops divided by sequence length (default: no effect) &ndash;max-bpil-length-ratio= &lt;factor&gt;(0.0) : Maximal ratio of #base pairs in loops divided by loop length (default: no effect) -D , &ndash;max-diff-am= &lt;diff&gt;(30) : Maximal difference for sizes of matched arcs -d , &ndash;max-diff= &lt;diff&gt;( -1 ) : Maximal difference for alignment traces &ndash;max-diff-at-am= &lt;diff&gt;( -1 ) : Maximal difference for alignment traces, only at arc match positions &ndash;prob_unpaired_in_loop_threshold= &lt;prob&gt;(0.01) : Threshold for prob_unpaired_in_\ loop &ndash;prob_basepair_in_loop_threshold= &lt;prob&gt;(0.01) : Threshold for prob_basepair_in_\ loop Output: -p , &ndash;out-min-prob= &lt;prob&gt;(0.0005) : Minimal probability for output (min-prob overrides if smaller) &ndash;output-ps : Output best EPM chain as colored postscript -a , &ndash;PS_fileA= &lt;file&gt;() : Postscript output file for sequence A -b , &ndash;PS_fileB= &lt;file&gt;() : Postscript output file for sequence B -o , &ndash;output-locarna= &lt;file&gt;() : Fasta file with anchor constraints for locarna &ndash;output-anchor-pp= &lt;fileroot&gt;() : PP files &lt;fileroot&gt;_A.pp and &lt;fileroot&gt;_B.pp, merging input PPs and anchor constraints from chaining &ndash;output-clustal= &lt;file&gt;() : Write file with chain as alignment in clustalw format &ndash;output-epm-list= &lt;file&gt;() : A list of all found epms &ndash;output-chained-epm-list= &lt;file&gt;() : A list of all EPMs that are present in the chain &ndash;inexact-struct-match : Allow inexact structure matches &ndash;add-filter : Apply an additional filter to enumerate only EPMs that are maximally extended (only inexact) &ndash;no-chaining : Do not use the chaining algorithm to find best overall chain Suboptimal traceback: &ndash;subopt : Use the suboptimal traceback &ndash;diff-to-opt-score= &lt;threshold&gt;( -1 ) : Threshold for suboptimal traceback &ndash;min-score= &lt;score&gt;(90) : Minimal score of a traced EPM &ndash;number-of-EPMs= &lt;threshold&gt;(100) : Maximal number of EPMs for the suboptimal traceback Constraints: &ndash;noLP : use &ndash;noLP option for folding &ndash;maxBPspan= &lt;span&gt;( -1 ) : Limit maximum base pair span (default=off) &ndash;relaxed-anchors : Relax anchor constraints (default=off) Miscellaneous: &ndash;stopwatch : Print run time information. Input files: The two input files &lt;Input 1&gt; and &lt;Input 2&gt; specify the input sequences in various, automatically detected formats. Accepted formats are: Fasta, Clustal, Stockholm LocARNA PP, ViennaRNA postscript dotplot. Unless in-loop probabilities are provided (only possible in LocARNA PP), base pair : probabilities are computed by partition function folding. Clustal, Stockholm, and PP input can contain constraints.
