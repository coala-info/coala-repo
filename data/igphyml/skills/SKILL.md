## Overview
IgPhyML is a specialized program designed for building phylogenetic trees and testing evolutionary hypotheses specifically within the context of B cell affinity maturation. It addresses the unique challenges of B cell biology, such as the context-sensitive nature of somatic hypermutation (SHM) and the analysis of large repertoires containing thousands of lineages. IgPhyML integrates with the Immcantation framework, enabling users to construct and visualize B cell lineage trees and investigate mutation and selection patterns.

## Usage Instructions

IgPhyML is typically used via its command-line interface. The core functionality revolves around building trees and estimating model parameters.

### Installation

Install IgPhyML via Conda:
```bash
conda install bioconda::igphyml
```

### Quick Start: Building Trees and Estimating Model Parameters

The most common use case is to build phylogenetic trees and estimate model parameters from AIRR-compliant datasets.

**Basic command structure:**

```bash
igphyml build-trees --input <input_file.tsv> --output <output_prefix> [options]
```

**Key Options:**

*   `--input <input_file.tsv>`: Path to the input dataset. This should be an AIRR-compliant TSV file.
*   `--output <output_prefix>`: A prefix for all output files generated by the command.
*   `--model <model_name>`: Specifies the substitution model to use. Common models include `hlp19` (for context-sensitive SHM).
*   `--nproc <number>`: Number of processors to use for parallel computation.
*   `--subsample <number>`: Subsample the input dataset to a specified number of clones for faster processing or analysis of specific subsets.
*   `--clean-intermediates`: Remove intermediate files after tree building.
*   `--remove-cdr3`: Remove the CDR3 region from sequences before analysis.

**Example:**

To build trees and estimate parameters using the `hlp19` model from `my_bcr_data.tsv`, saving output with the prefix `bcr_tree`:

```bash
igphyml build-trees --input my_bcr_data.tsv --output bcr_tree --model hlp19 --nproc 4 --remove-cdr3
```

### Analyzing Individual Lineages (Standalone Operation)

For analyzing a single lineage or a small set of lineages, IgPhyML can be used in standalone mode. This is useful for detailed examination of specific evolutionary paths.

**Command structure:**

```bash
igphyml analyze-lineage --input <input_file.tsv> --output <output_prefix> [options]
```

This mode allows for more granular control over the analysis of individual phylogenetic trees.

### Controlling Input and Data Preparation

*   **AIRR Datasets**: Ensure your input data is in the AIRR (Adaptive Immune Receptor Repertoire) Community standard format, typically a TSV file.
*   **Subsampling**: Use `--subsample` to manage large datasets. This is crucial for performance and can help focus analysis on representative subsets.
*   **Cleaning Intermediates**: Use `--clean-intermediates` to manage disk space, especially when processing many datasets.
*   **CDR3 Removal**: The CDR3 region can sometimes introduce noise or complexity. Use `--remove-cdr3` if it's not critical for your specific hypothesis testing.

### Evolutionary Hypothesis Testing

IgPhyML is designed to test hypotheses about mutation and selection. The `build-trees` command, when used with appropriate models like `hlp19`, inherently incorporates parameters that can be used to infer selection pressures and mutation rates. Further analysis of the generated tree files and associated statistics will be required to formally test hypotheses.

### Visualizing Results

While IgPhyML itself focuses on computation, its output is designed to be compatible with visualization tools within the Immcantation framework, such as Dowser. The output files (e.g., Newick format for trees) can be loaded into these tools.

## Expert Tips

*   **Model Selection**: For B cell affinity maturation, the `hlp19` model is generally recommended due to its ability to account for context-sensitive SHM.
*   **Parallel Processing**: Leverage multi-core processors with the `--nproc` option to significantly speed up tree building for large datasets.
*   **Resource Management**: For very large datasets, consider subsampling (`--subsample`) or running analyses on a cluster. Use `--clean-intermediates` to save disk space.
*   **Output Inspection**: Familiarize yourself with the output files generated by `build-trees` (e.g., `.nwk` for trees, `.log` for parameters) to understand the results of your analysis.
