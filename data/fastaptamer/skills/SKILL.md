# fastaptamer

## Overview
FASTAptamer is designed for bench scientists to process large-scale sequencing data from combinatorial libraries. It transforms raw sequence files into structured data by identifying unique species and tracking their behavior across multiple selection rounds. This skill provides the procedural knowledge to execute the toolkit's core functions: sequence counting, population comparison, edit-distance clustering, and motif searching.

## Core Workflows and CLI Patterns

### 1. Sequence Pre-processing and Counting
The first step in any FASTAptamer pipeline is converting raw FASTQ/FASTA data into a ranked list of unique sequences.
- **fastaptamer_count**: Scans the input file, counts occurrences of each unique sequence, and calculates RPM (Reads Per Million) for normalization.
- **Best Practice**: Always use the `-off` flag if you need to filter sequences by a specific length or pattern before counting to reduce noise in downstream clustering.

### 2. Population Comparison and Enrichment
To identify which sequences are "winning" the selection, compare different rounds or conditions.
- **fastaptamer_compare**: Takes two `.fastaptamer` files (e.g., Round 5 vs. Round 10) and calculates the fold-enrichment for every sequence present in both.
- **fastaptamer_enrich**: Used for multi-round analysis to track the trajectory of sequence abundance across an entire selection timeline.

### 3. Sequence Clustering
Group related sequences to identify families that likely share the same binding or catalytic motif.
- **fastaptamer_cluster**: Uses Levenshtein edit distance to group sequences. 
- **Expert Tip**: Start with a small distance (e.g., `-d 1` or `-d 2`) for high-stringency families. Increasing the distance significantly increases computational time on large datasets.
- **Input**: Requires a file generated by `fastaptamer_count`.

### 4. Motif Searching
Search for specific conserved functional domains within your population.
- **fastaptamer_search**: Allows for degenerate nucleotide searches (using IUPAC codes) across the entire population or specific clusters.

## Common Command Patterns
- **Normalization**: Ensure all files are processed with `fastaptamer_count` first; the toolkit relies on the normalized RPM values for all comparative statistics.
- **Sorting**: Output files are typically sorted by abundance (rank) by default, which is essential for identifying top candidates for synthesis and testing.



## Subcommands

| Command | Description |
|---------|-------------|
| fastaptamer_cluster | FASTAptamer-Cluster uses the Levenshtein algorithm to cluster together sequences based on a user-defined edit distance. The most abundant and unclustered sequence is used as the "seed sequence" for which edit distance is calculated from. |
| fastaptamer_compare | FASTAptamer-Compare facilitates statistical analysis of two populations by rapidly generating a tab-delimited output file that lists each unique sequence along with RPM (reads per million) in each population file (if available) and log(base 2) of the ratio of their RPM values in each population. RPM data for both populations can be utilized to generate an XY-scatter plot of sequence distribution across two populations. FASTAptamer-Compare also facilitates the generation of a histogram of the sequence distribution by creating 102 bins for the log(base2) values. This histogram can provide a quick visual comparison of the two populations: distributions centered around 0 indicate similar populations, while distributions shifted to the left or right indicate overall enrichment or depletion. Input for FASTAptamer-Compare MUST come from FASTAptamer-Count output files. |
| fastaptamer_count | FASTAptamer-Count serves as the gateway to the FASTAptamer toolkit. For a given .FASTQ input file (or FASTA input file if the -f flag is used), FASTAptamer-Count will determine the number of times each sequence was read, rank and sort sequences by decreasing total reads, and normalize sequence frequency to reads per million. |
| fastaptamer_enrich | FASTAptamer-Enrich rapidly calculates fold-enrichment values for each sequence across two or three input files. Output is provided as a tab-delimited plain text file and is formatted to include sequence composition, length, rank, reads, reads per million (RPM), and enrichment values for each sequence. If any files from FASTAptamer-Cluster are provided, output will include cluster information for that population. A threshold filter can be applied to exclude sequences with total reads per million (across all input populations) less than the number entered after the [-f] option. Default behavior is to include all sequences. Enrichment is calculated by dividing reads per million of y/x (and z/y and z/x, if a third input file is specified). |
| fastaptamer_search | FASTAptamer-Search allows users to search for specific patterns within one or more sequence files. |
