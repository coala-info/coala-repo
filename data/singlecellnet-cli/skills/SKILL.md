# singlecellnet-cli

## Overview

The `singlecellnet-cli` provides a command-line interface for the singleCellNet R package, enabling automated cell type classification workflows. It transforms the R-based singleCellNet methodology into executable scripts that handle the training of random forest classifiers and the subsequent prediction of cell identities in new datasets. This skill is particularly useful for bioinformaticians working with SingleCellExperiment (.rds) files who require a robust, reproducible way to transfer cell type labels from well-annotated references to unannotated query data, even across different sequencing technologies or species.

## Command Usage and Best Practices

### Training a Classifier (`scn-train-model.R`)

Use this command to build a model from a reference dataset. The input must be a SingleCellExperiment object saved in `.rds` format.

**Core Parameters:**
*   `--input-object`: Path to the reference SCE `.rds` file.
*   `--cell-type-col`: The metadata column name containing ground-truth cell type labels.
*   `--cell-barcode-col`: The metadata column name containing unique cell identifiers.
*   `--output-path`: Destination for the trained classifier (saved as an `.rds` file).

**Optimization Tips:**
*   **Feature Selection**: Adjust `--n-top-genes` and `--n-top-gene-pairs` to control the complexity of the classifier. Increasing these can improve accuracy but may lead to overfitting on platform-specific noise.
*   **Class Imbalance**: Set `--stratify TRUE` if your reference dataset has highly unequal numbers of cells across different types to ensure the random forest isn't biased toward dominant populations.
*   **Random Profiles**: The `--n-rand` parameter determines how many "random" cell profiles are generated to help the model identify "unknown" or "unassigned" cells.

### Predicting Cell Types (`scn-predict.R`)

Use this command to apply a trained model to a query dataset.

**Core Parameters:**
*   `--input-classifier-object`: Path to the `.rds` file generated by the training step.
*   `--query-expression-data`: Path to the query SCE `.rds` file.
*   `--prediction-output`: Path where the classification results will be saved.

**Expert Tips:**
*   **Gene Matching**: singleCellNet automatically handles missing genes in the query data using mean imputation. However, for best results, ensure that the gene naming convention (e.g., Gene Symbols vs. Ensembl IDs) matches between the training and query datasets.
*   **Output Format**: By default, the tool returns a processed table. Use `--return-raw-output TRUE` if you need the full probability matrix for downstream custom thresholding or visualization.
*   **Evaluation**: The `--n-rand-prof` parameter controls the evaluation process during prediction; higher values provide more robust assessment of classification confidence.

## Common CLI Patterns

**Standard Training Workflow:**
```bash
scn-train-model.R \
  --input-object reference_data.rds \
  --cell-type-col "cell_ontology_class" \
  --cell-barcode-col "barcode" \
  --n-top-genes 100 \
  --n-trees 1000 \
  --output-path my_classifier.rds
```

**Standard Prediction Workflow:**
```bash
scn-predict.R \
  --input-classifier-object my_classifier.rds \
  --query-expression-data query_data.rds \
  --prediction-output results.csv
```
