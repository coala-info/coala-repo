## Overview
nf-core/airrflow is a bioinformatics pipeline designed for the end-to-end analysis of B-cell receptor (BCR) and T-cell receptor (TCR) repertoires. It addresses the complexity of adaptive immune receptor sequencing (AIRR-seq) by providing standardized processing for both bulk and single-cell data, including sequence assembly, quality control, and advanced clonal analysis.

The pipeline transforms raw sequencing reads or pre-assembled sequences into AIRR-compliant databases and biological insights. It produces detailed repertoire reports, lineage trees, and clonal distributions, allowing researchers to characterize immune responses across different tissues, subjects, and time points.

## Data preparation
The pipeline requires a tab-separated (TSV) samplesheet specified with the `--input` parameter. The structure of the samplesheet depends on whether the input is raw FASTQ files or pre-assembled data.

Required columns for all samplesheets:
- `sample_id`: Unique identifier for the sample.
- `subject_id`: Identifier for the biological subject (used for grouping clones).
- `species`: `human` or `mouse`.
- `pcr_target_locus`: `IG` (for BCR) or `TR` (for TCR).
- `tissue`, `sex`, `age`, `biomaterial_provider`: Metadata for reporting.
- `single_cell`: `TRUE` or `FALSE`.

For raw FASTQ input (`--mode fastq`), include:
- `filename_R1`: Path to the first read FASTQ.
- `filename_R2`: Path to the second read FASTQ.
- `filename_I1`: (Optional) Path to the index read FASTQ for UMIs.

For assembled input (`--mode assembled`), include:
- `filename`: Path to the FASTA or AIRR-formatted TSV file.

Example samplesheet snippet:
```tsv
sample_id  subject_id  species  pcr_target_locus  tissue  sex     age  biomaterial_provider  single_cell  filename_R1      filename_R2
sample01   Subject01   human    IG                blood   female  45   clinic_a              FALSE        s1_R1.fastq.gz   s1_R2.fastq.gz
```

## How to run
The pipeline is executed using `nextflow run`. You must specify a processing mode and a profile (e.g., `docker` or `singularity`).

**Bulk FASTQ with UMIs:**
```bash
nextflow run nf-core/airrflow \
    -r 2.7.0 \
    -profile docker \
    --mode fastq \
    --input samplesheet.tsv \
    --library_generation_method specific_pcr_umi \
    --umi_length 12 \
    --outdir ./results
```

**Single-cell 10X Genomics:**
```bash
nextflow run nf-core/airrflow \
    -profile docker \
    --mode fastq \
    --input samplesheet.tsv \
    --library_generation_method sc_10x_genomics \
    --reference_10x path/to/cellranger_ref.tar.gz \
    --outdir ./results
```

**Assembled sequences:**
```bash
nextflow run nf-core/airrflow \
    -profile docker \
    --mode assembled \
    --input samplesheet.tsv \
    --outdir ./results
```

Commonly used flags include `-resume` to restart from a cached state and `-profile test` for running a minimal verification dataset.

## Outputs
Results are saved in the directory specified by `--outdir`. Key outputs include:

- **Repertoire Analysis:** A comprehensive HTML report generated by Alakazam/EnchantR containing clonal diversity and abundance metrics.
- **AIRR Databases:** Tabulated files in AIRR-compliant format containing V(D)J assignments and clonal information.
- **Lineage Trees:** Clonal lineage reconstructions (if `--lineage_trees` is enabled) in Newick format.
- **MultiQC:** An aggregate report summarizing quality control metrics from Fastp, pRESTO, and other tools.
- **Intermediate Files:** Processed FASTQ files, assembled contigs, and IgBlast alignment logs.

For a complete description of output files, refer to the [official output documentation](https://nf-co.re/airrflow/output).
