# pirs

## Overview
The pIRS (profile-based Illumina pair-end Reads Simulator) tool provides a framework for generating realistic synthetic sequencing data. It excels at creating diploid genomes from haploid references by introducing user-defined rates of SNPs, small indels, and large-scale structural variations. Once a genome is prepared, the tool simulates the sequencing process to produce paired-end FASTQ files that reflect the error profiles and biases typical of the Illumina platform.

## Core Workflows

### 1. Simulating a Diploid Genome
To simulate reads from a diploid organism, you must first generate a second "homolog" sequence containing variations.

```bash
pirs diploid [options] <reference.fa>
```

**Key Options:**
- `-s, --snp-rate`: Heterozygous SNP rate (default: 0.001).
- `-d, --indel-rate`: Heterozygous small indel rate (default: 0.0001).
- `-v, --sv-rate`: Large-scale structural variation rate (default: 0.000001).
- `-R`: Transition-to-transversion ratio (default: 2.0).
- `-o`: Output prefix for the generated FASTA and log files.

### 2. Simulating Illumina Reads
Once you have your reference (and optionally your diploid sequence), use the `simulate` command to generate reads.

```bash
pirs simulate [options] <reference.fa> [diploid_reference.fa]
```

**Usage Patterns:**
- **Haploid Simulation**: Provide only the original reference FASTA.
- **Diploid Simulation**: Provide both the original reference and the FASTA generated by `pirs diploid`.
- **Multi-chromosome Simulation**: You can provide multiple FASTA files if chromosomes are stored separately.

## Profile Generation
pIRS uses profiles to model GC-bias, base-calling errors, and InDel distributions. While it comes with default profiles, you can generate custom ones using the provided stator tools:
- **GC-Depth Profile**: Use `gc_coverage_bias` on depth files generated by SOAP or BWA.
- **Base-Calling Profile**: Use `error_matrix_calculator` on SAM/BAM files.
- **InDel Profile**: Use `indelstat_sam_bam` on samples with no polymorphism InDels (e.g., Coliphage control samples).

## Expert Tips and Best Practices
- **Reproducibility**: Always set a specific random seed using `-S <SEED>` if you need to regenerate the exact same set of variations or reads later.
- **Output Management**: Use the `-c gzip` option in `pirs diploid` to save disk space, especially when working with large mammalian genomes.
- **Validation**: After running `pirs diploid`, check the generated log files (unless disabled with `-n`) to verify the exact positions and types of variations introduced. These logs are essential "truth sets" for benchmarking.
- **Memory Efficiency**: If the reference genome is large, ensure you have sufficient RAM, as the simulator needs to load the sequences into memory to process them effectively.
