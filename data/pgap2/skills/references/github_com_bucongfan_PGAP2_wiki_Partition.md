**Main function of PGAP2 using to identify orthologs**

# Quick start

```
pgap2 main --indir your.input_file.dir --outdir .
```

# Parameters

```
usage: main.py main [-h] --indir INDIR [--outdir OUTDIR] [--dup_id DUP_ID] [--orth_id ORTH_ID] [--para_id PARA_ID]
                    [--exhaust_orth] [--sensitivity {soft,moderate,strict}] [--ins] [--accurate] [--retrieve]
                    [--threads THREADS] [--LD LD] [--AS AS] [--AL AL] [--evalue EVALUE] [--aligner {diamond,blastp}]
                    [--clust_method {cdhit,mmseqs2}] [--radius RADIUS] [--min_falen MIN_FALEN] [--gcode GCODE]
                    [--annot] [--verbose] [--debug] [--disable]

options:
  -h, --help            show this help message and exit
  --indir INDIR, -i INDIR
                        Input file contained, same prefix seems as the same strain. (default: None)
  --outdir OUTDIR, -o OUTDIR
                        Output directory (default: .)
  --dup_id DUP_ID       The maximum identity between the most recent duplication envent. (default: 0.99)
  --orth_id ORTH_ID     The maximum identity between the most similar panclusters. (default: 0.98)
  --para_id PARA_ID     Use this identity as the paralogous identity. (default: 0.7)
  --exhaust_orth, -e    Try to split every paralogs gene exhausted (default: False)
  --sensitivity {soft,moderate,strict}, -s {soft,moderate,strict}
                        The degree of connectedness between each node of clust. (default: strict)
  --ins, -n             Ignore the influence of insertion sequence. (default: False)
  --accurate, -a        Apply bidirection check for paralogous gene partition (useless if exhaust_orth asigned).
                        (default: False)
  --retrieve, -r        Retrieve gene that may lost with annotations (default: False)
  --threads THREADS, -t THREADS
                        threads used in parallel (default: 8)
  --LD LD               Minimum gene length difference proportion between two genes. (default: 0.6)
  --AS AS               Coverage for the shorter sequence. (default: 0.6)
  --AL AL               Coverage for the longer sequence. (default: 0.6)
  --evalue EVALUE       The evalue of aligner. (default: 1e-05)
  --aligner {diamond,blastp}
                        The aligner used to pairwise alignment. (default: diamond)
  --clust_method {cdhit,mmseqs2}
                        The method used to cluster the genes. (default: cdhit)
  --radius RADIUS       The radius of search region. (default: 3)
  --min_falen MIN_FALEN, -m MIN_FALEN
                        protein length of throw_away_sequences, at least 11 (default: 20)
  --gcode GCODE         The genetic code of your species. (default: 11)
  --annot               Discard original annotation, and re-annote the genome privately using prodigal (default:
                        False)
  --verbose, -v         Verbose output (default: False)
  --debug, -D           Debug mode. Note: very verbose (default: False)
  --disable             Disable progress bar (default: False)
```

# Techniques

1. PGAP2 by default identifies paralogous gene clusters, meaning duplicated genes within the cluster emerged after the divergence of orthologous genes. To disable this feature, similar to Roary or PanTa, and ensure that all genes are free from duplicated genes, you need to specify the parameter `--exhaust_orth` or `-e`.
2. To prevent insertion sequences from affecting the accuracy of orthologous gene identification, PGAP2 by default merges all highly similar genes. To disable this feature and treat all genes uniformly, like other software, you need to specify the parameter `--ins` or `-n`.
3. The `--sensitivity` parameter allows the use of "soft" clustering, which is better suited for clustering highly variable genes as orthologous genes, making it more appropriate for diverse sequences. On the other hand, "strict" clustering is more suitable for identifying conserved gene families.
4. Several factors influence the accuracy of gene identification. For instance, aligment method used by Diamond and Blast+, and the clustering methods used by CD-HIT and MMseqs2 can indirectly affect clustering outcomes. Additionally, post-alignment filtering thresholds, such as `--AS` , `--AL`, `--LD`, and `--evalue`, also play a critical role in determining the precision of the results.
5. The `--accurate` or `-a` parameter is used to determine whether duplicated genes meet the bidirectional best-hit criterion. If specified, the actual alignment values of the corresponding sequences will be used instead of the identity between representative sequences, which may slightly increase the computational time.
6. The `--retrieve` or `-r` parameter enables the search for missing genes in non-strict core clusters during the final step. The identified genes might include pseudogenes without coding functions or lost coding genes. Adding this parameter helps improve genome annotation by comparing pangenome results.
7. The `--radius` parameter defines the search radius for conserved gene neighbors. Increasing this value results in more conserved gene clusters, but larger values are not always better.
8. Due to feature analysis, not all genes that meet the sequence similarity and positional conservation criteria will be clustered. Instead, PGAP2 considers additional factors, such as the diversity and connectivity of the genes.
