# args_oap

## Overview

The `args_oap` tool is a specialized bioinformatics pipeline designed to streamline the detection of antibiotic resistance genes in metagenomic samples. It operates in two primary stages: an initial high-throughput screening and estimation of essential marker genes (16S rRNA and single-copy marker genes), followed by a detailed classification and normalization of ARG hits. This tool is essential for researchers looking to compare ARG abundance across different environments by providing normalized outputs (e.g., copies per cell or copies per 16S rRNA gene).

## Core Workflow

### 1. Stage One: Pre-processing and Estimation
This stage identifies ARG-like sequences and estimates the number of 16S rRNA genes and cells in the input samples.

```bash
args_oap stage_one -i <input_dir> -o <output_dir> -f <format> -t <threads>
```

*   **-i**: Directory containing input sequencing files.
*   **-o**: Directory for intermediate results (includes `metadata.txt`).
*   **-f**: Input file format (e.g., `fa`, `fq`, `fasta`, `fastq`).
*   **-t**: Number of CPU threads to utilize.
*   **-s** (Optional): Skip 16S/cell calculation if only ARG-like sequence extraction is needed.

### 2. Stage Two: Classification and Normalization
This stage performs the final ARG identification and produces normalized abundance tables.

```bash
args_oap stage_two -i <stage_one_output_dir> -t <threads>
```

*   **Input**: Must be the output directory generated by `stage_one`.
*   **Outputs**: 
    *   `normalized_16S.*`: ARG abundance normalized against 16S rRNA gene copies.
    *   `normalized_cell.*`: ARG abundance normalized against cell numbers (based on single-copy marker genes).
    *   `extracted.filtered.fa`: The actual sequences identified as ARGs.

## Expert Tips and Best Practices

### File Naming for Paired-End Data
To ensure `args_oap` correctly identifies paired-end reads as a single sample, use the following suffixes before the file extension:
*   `_1` and `_2`
*   `_R1` and `_R2`
*   `_fwd` and `_rev`

### De-contamination
Always perform host (e.g., human) and fungal DNA removal before running `stage_one`. Contaminant DNA can lead to inaccurate cell number estimations, as the pipeline relies on prokaryotic single-copy marker genes.

### Custom Databases
You can use `args_oap` for other gene categories (e.g., metal resistance) by providing custom FASTA and structure files:
1.  **Index the database**: `args_oap make_db -i custom_db.fasta`
2.  **Run Stage One**: Add `--database custom_db.fasta`
3.  **Run Stage Two**: Add `--database custom_db.fasta --structure1 custom_structure.txt`

### Interpreting Results
*   **Type vs. Subtype**: The SARG database is hierarchical. `normalized_cell.type` provides a broad overview (e.g., aminoglycoside), while `normalized_cell.subtype` provides specific resistance gene families.
*   **PPM**: If you require "Hits per million reads," look for the `ppm` output files.



## Subcommands

| Command | Description |
|---------|-------------|
| make_db | Create a database from a FASTA file (nucleotide or protein). |
| stage_one | Stage one of the ARGs-OAP pipeline for antibiotic resistance genes (ARGs) analysis. |
| stage_two | Stage two of the ARGs-OAP pipeline for antibiotic resistance gene analysis, performing sequence alignment and classification. |
