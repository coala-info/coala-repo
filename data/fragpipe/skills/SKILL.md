# fragpipe

---
## Overview
FragPipe is a powerful, GUI-driven platform for comprehensive shotgun proteomics data analysis. It integrates various tools like MSFragger for ultrafast peptide identification, Philosopher for downstream analysis (including FDR filtering and quantification), and specialized modules for PTM discovery, TMT/iTRAQ quantification, and DIA data analysis. FragPipe can be run via its graphical interface or through the command line, making it versatile for different analysis needs.

## Usage Instructions

FragPipe is primarily used for analyzing mass spectrometry-based proteomics data. The core of its functionality lies in its ability to orchestrate multiple bioinformatics tools into a cohesive pipeline.

### Core Components and Workflows:

FragPipe supports a wide range of analyses, including:

*   **Peptide Identification:** Using MSFragger for both conventional and "open" (wide precursor mass tolerance) searches.
*   **Post-processing and Quantification:** Leveraging the Philosopher toolkit for results analysis, including PeptideProphet, iProphet, ProteinProphet, FDR filtering, and label-based quantification.
*   **Label-Free Quantification (LFQ):** With IonQuant, including match-between-run (MBR) functionality.
*   **Isobaric Labeling Quantification:** Using TMT-Integrator for TMT/iTRAQ data.
*   **Post-Translational Modification (PTM) Discovery:** Aided by Crystal-C and PTM-Shepherd for interpreting open search results.
*   **Data Independent Acquisition (DIA) Analysis:** Through MSFragger-DIA and DIA-Umpire SE modules.
*   **Spectral Library Building:** Using EasyPQP.

### Command-Line Usage (General Principles):

While FragPipe has a GUI, command-line execution is crucial for batch processing and integration into automated workflows. The exact command structure can be complex due to the integrated nature of the tools.

**General Approach:**

1.  **Identify the specific analysis module:** Determine which part of FragPipe you need to run (e.g., MSFragger, Philosopher, IonQuant).
2.  **Prepare input files:** This typically includes raw spectral data (e.g., `.mzML`, `.raw`), protein sequence databases, and configuration files.
3.  **Construct the command:** The command will often involve specifying the main FragPipe executable or a specific tool's executable, followed by arguments for input files, parameters, and output directories.

**Example (Conceptual - actual commands may vary):**

To run MSFragger for peptide identification, a command might look something like this:

```bash
# This is a conceptual example. Refer to FragPipe documentation for exact syntax.
fragpipe --workflow msfragger --input_spectra /path/to/spectra/*.mzML --database /path/to/database.fasta --output_dir /path/to/output
```

For downstream analysis with Philosopher:

```bash
# This is a conceptual example. Refer to FragPipe documentation for exact syntax.
philosopher --input_dir /path/to/msfragger_output --output_dir /path/to/philosopher_output --params /path/to/philosopher.params
```

### Expert Tips:

*   **Configuration Files:** FragPipe relies heavily on configuration files (often `.params` or `.ini` files) to define search parameters, quantification settings, and workflow options. Familiarize yourself with these files to fine-tune your analyses.
*   **Parameter Optimization:** For optimal results, carefully adjust parameters related to mass tolerances, enzyme specificity, variable modifications, and FDR thresholds based on your experimental design and data type.
*   **Database Management:** Ensure you are using the correct and up-to-date protein sequence databases for your analyses. Consider including common contaminants.
*   **Output Interpretation:** Understand the various output files generated by FragPipe (e.g., `.pep.xml`, `.prot.xml`, `.tsv` reports) and their significance for downstream interpretation.
*   **Resource Management:** Proteomics data analysis can be computationally intensive. Ensure sufficient RAM and processing power are available, especially for large datasets or complex searches.
*   **Refer to Specific Tool Documentation:** While FragPipe integrates tools, understanding the individual documentation for MSFragger, Philosopher, IonQuant, etc., can provide deeper insights into specific parameters and advanced usage.
