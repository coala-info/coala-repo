# Configuration

Libraries for configuration parsing.

* [aconfig - Simple, useful and opinionated config loader.](/configuration/aconfig-cristalhq-github/)
* [argus - File watching and configuration management with MPSC ring buffer, adaptive batching strategies, and universal format parsing (JSON, YAML, TOML, INI, HCL, Properties).](/configuration/argus-agilira-github/)
* [azureappconfiguration - The configuration provider for consuming data in Azure App Configuration from Go applications.](/configuration/appconfiguration-goprovider-azure-github/)
* [bcl - BCL is a configuration language similar to HCL.](/configuration/bcl-wkhere-github/)
* [cleanenv - Minimalistic configuration reader (from files, ENV, and wherever you want).](/configuration/cleanenv-ilyakaznacheev-github/)
* [config - Cloud native application configuration. Bind ENV to structs in only two lines.](/configuration/config-jeremyloy-github/)
* [config - configure your app using file, environment variables, or flags in two lines of code.](/configuration/config-num30-github/)
* [configuration - Library for initializing configuration structs from env variables, files, flags and 'default' tag.](/configuration/configuration-borudar-github/)
* [configuro - opinionated configuration loading & validation framework from ENV and Files focused towards 12-Factor compliant applications.](/configuration/configuro-sherifabdlnaby-github/)
* [confiq - Structured data format to config struct decoder library for Go - supporting multiple data formats.](/configuration/confiq-greencoda-github/)
* [confita - Load configuration in cascade from multiple backends into a struct.](/configuration/confita-heetch-github/)
* [conflate - Library/tool to merge multiple JSON/YAML/TOML files from arbitrary URLs, validation against a JSON schema, and application of default values defined in the schema.](/configuration/conflate-the4thamigo-uk-github/)
* [enflag - Container-oriented, zero-dependency configuration library that unifies Env variable and Flag parsing. Uses generics for type safety, without reflection or struct tags.](/configuration/enflag-atelpis-github/)
* [env - Parse environment variables to Go structs (with defaults).](/configuration/env-caarlos0-github/)
* [env - A lightweight package for loading environment variables into structs.](/configuration/env-junk1tm-github/)
* [env - An environment utility package with support for unmarshaling into structs.](/configuration/env-syntaqx-github/)
* [envconfig - Read your configuration from environment variables.](/configuration/envconfig-vrischmann-github/)
* [envh - Helpers to manage environment variables.](/configuration/envh-antham-github/)
* [envyaml - Yaml with environment variables reader. it helps to have secrets as environment variable but load them configs as structured Yaml.](/configuration/envyaml-yuseferi-github/)
* [fig - Tiny library for reading configuration from a file and from environment variables (with validation & defaults).](/configuration/fig-kkyr-github/)
* [genv - Read environment variables easily with dotenv support.](/configuration/genv-sakirsensoy-github/)
* [go-array - A Go package that read or set data from map, slice or json.](/configuration/go-array-deatil-github/)
* [go-aws-ssm - Go package that fetches parameters from AWS System Manager - Parameter Store.](/configuration/go-aws-ssm-paddlehq-github/)
* [go-cfg - The library provides a unified way to read configuration data into a structure from various sources, such as env, flags, and configuration files (.json, .yaml, .toml, .env).](/configuration/go-cfg-dsbasko-github/)
* [go-conf - Simple library for application configuration based on annotated structs. It supports reading the configuration from environment variables, config files and command line parameters.](/configuration/go-conf-thomasobenaus-github/)
* [go-config - Simple and convenient library for working with app configurations.](/configuration/go-config-mordateam-github/)
* [go-ini - A Go package that marshals and unmarshals INI-files.](/configuration/go-ini-subpop-github/)
* [go-ssm-config - Go utility for loading configuration parameters from AWS SSM (Parameter Store).](/configuration/go-ssm-config-ianlopshire-github/)
* [go-up - A simple configuration library with recursive placeholders resolution and no magic.](/configuration/go-up-ufoscout-github/)
* [GoCfg - Config manager with Struct Tags based contracts, custom value providers, parsers, and documentation generation. Customizable yet simple.](/configuration/gocfg-jagerente-github/)
* [goconfig - Populate Go structs from flags, environment variables, config.json and defaults with deterministic precedence. No extra dependencies.](/configuration/goconfig-fulldump-github/)
* [godotenv - Go port of Ruby's dotenv library (Loads environment variables from .env).](/configuration/godotenv-joho-github/)
* [GoLobby/Config - GoLobby Config is a lightweight yet powerful configuration manager for the Go programming language.](/configuration/config-golobby-github/)
* [gone/jconf - Modular JSON configuration. Keep your config structs along with the code they configure and delegate parsing to submodules without sacrificing full config serialization.](/configuration/gone-one-com-github/)
* [gonfig - Tag-based configuration parser which loads values from different providers into typesafe struct.](/configuration/gonfig-milad-abbasi-github/)
* [gookit/config - application config manage(load,get,set). support JSON, YAML, TOML, INI, HCL. multi file load, data override merge.](/configuration/config-gookit-github/)
* [harvester - Harvester, a easy to use static and dynamic configuration package supporting seeding, env vars and Consul integration.](/configuration/harvester-beatlabs-github/)
* [hedzr/store - Extensible, high-performance configuration management library, optimized for hierarchical data.](/configuration/store-hedzr-github/)
* [hjson - Human JSON, a configuration file format for humans. Relaxed syntax, fewer mistakes, more comments.](/configuration/hjson-go-hjson-github/)
* [hocon - Configuration library for working with the HOCON(a human-friendly JSON superset) format, supports features like environment variables, referencing other values, comments and multiple files.](/configuration/hocon-gurkankaymak-github/)
* [ini - Go package to read and write INI files.](/configuration/ini-go-ini-github/)
* [ini - INI Parser & Write Library, Unmarshal to Struct, Marshal to Json, Write File, watch file.](/configuration/ini-wlevene-github/)
* [kelseyhightower/envconfig - Go library for managing configuration data from environment variables.](/configuration/envconfig-kelseyhightower-github/)
* [koanf - Light weight, extensible library for reading config in Go applications. Built in support for JSON, TOML, YAML, env, command line.](/configuration/koanf-knadh-github/)
* [konf - The simplest API for reading/watching config from file, env, flag and clouds (e.g. AWS, Azure, GCP).](/configuration/konf-nil-go-github/)
* [konfig - Composable, observable and performant config handling for Go for the distributed processing era.](/configuration/konfig-lalamove-github/)
* [kong - Command-line parser with support for arbitrarily complex command-line structures and additional sources of configuration such as YAML, JSON, TOML, etc (successor to kingpin).](/configuration/kong-alecthomas-github/)
* [nasermirzaei89/env - Simple useful package for read environment variables.](/configuration/env-nasermirzaei89-github/)
* [nfigure - Per-library struct-tag based configuration from command lines (Posix & Go-style); environment, JSON, YAML](/configuration/nfigure-muir-github/)
* [onion - Layer based configuration for Go, Supports JSON, TOML, YAML, properties, etcd, env, and encryption using PGP.](/configuration/onion-goraz-github/)
* [piper - Viper wrapper with config inheritance and key generation.](/configuration/piper-yiling-j-github/)
* [sonic - A blazingly fast JSON serializing & deserializing library.](/configuration/sonic-bytedance-github/)
* [swap - Instantiate/configure structs recursively, based on build environment. (YAML, TOML, JSON and env).](/configuration/swap-oblq-github/)
* [typenv - Minimalistic, zero dependency, typed environment variables library.](/configuration/typenv-diegomarangoni-github/)
* [uConfig - Lightweight, zero-dependency, and extendable configuration management.](/configuration/uconfig-omeid-github/)
* [viper - Go configuration with fangs.](/configuration/viper-spf13-github/)
* [xdg - Go implementation of the XDG Base Directory Specification and XDG user directories.](/configuration/xdg-adrg-github/)
* [yamagiconf - The "safe subset" of YAML for Go configs.](/configuration/yamagiconf-romshark-github/)
* [zerocfg - Zero-effort, concise configuration management that avoids boilerplate and repetitive code, supports multiple sources with priority overrides.](/configuration/zerocfg-chaindead-github/)
