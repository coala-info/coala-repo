# covid-19-variation-analysis-of-artic-ont-data

## Overview

This Galaxy workflow is designed for the variation analysis of SARS-CoV-2 ONT-sequenced data, specifically following the [ARTIC pipeline](https://artic.readthedocs.io/en/latest/) methodology. It replicates the core functionality of the ARTIC `minion` command by performing read mapping with **minimap2** and variant calling via **medaka**. To ensure data quality, the workflow incorporates **fastp** for initial processing and **ivar trim** for primer removal, mirroring the approach used in Illumina-based ARTIC analysis.

The pipeline processes ONT-sequenced reads against the NC_045512.2 FASTA reference genome using provided primer binding sites in BED format. Key steps include read filtering by length, alignment realignment with **BamLeftAlign**, and comprehensive quality control using **QualiMap BamQC**, **Samtools stats**, and **MultiQC**. Because ONT reads exhibit higher error rates, the workflow focuses on robust variant detection and does not attempt to resolve amplicons affected by potential primer-binding site mutations.

Final outputs include annotated variant calls generated by **SnpEff** and **bcftools**, alongside filtered consensus sequences and soft-filtered variant lists. This workflow is released under the **MIT** license and is part of the [covid19.galaxyproject.org](https://covid19.galaxyproject.org) initiative.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | ONT-sequenced reads | data_collection_input | ONT reads from ARTIC assay with fastqsanger encoding |
| 1 | Minimum read length | parameter_input | Discard reads that are shorter than this length |
| 2 | Maximum read length | parameter_input | Discard reads that are longer than this length |
| 3 | NC_045512.2 FASTA sequence of SARS-CoV-2 | data_input | Fasta sequence for Severe acute respiratory syndrome coronavirus 2 isolate Wuhan-Hu-1, complete genome |
| 4 | Primer binding sites info in BED format | data_input | BED file containing (ARTIC) primer positions. For ARTIC primer schemes the file can be obtained from https://usegalaxy.eu/u/wolfgang-maier/h/covid-19-resources. |


Organize your ONT-sequenced reads into a dataset collection of fastq files to ensure efficient batch processing through the medaka and minimap2 steps. The SARS-CoV-2 reference must be in FASTA format, while the primer binding sites require a BED file tailored to your specific ARTIC scheme. Carefully calibrate the minimum and maximum read length parameters to filter out noise and non-target fragments before alignment. Refer to the README.md for exhaustive instructions on preparing these inputs and managing primer scheme versions. You may use `planemo workflow_job_init` to create a `job.yml` for streamlined configuration of these inputs.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run ont-artic-variation.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run ont-artic-variation.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run ont-artic-variation.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init ont-artic-variation.ga -o job.yml`
- Lint: `planemo workflow_lint ont-artic-variation.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
