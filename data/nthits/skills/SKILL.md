# nthits

## Overview
ntHits is a specialized bioinformatics tool designed for the efficient counting and filtering of k-mers based on their frequency of occurrence. It bridges the gap between k-mer frequency estimation (using tools like ntCard) and downstream analysis by allowing users to extract specific subsets of k-mers—such as "solid" k-mers (likely non-erroneous) or highly repetitive sequences—into compact data structures. It supports standard Bloom filters, counting Bloom filters, and tabular outputs, making it a versatile component in genomic assembly and repeat-masking pipelines.

## Command Line Usage

### Core Syntax
The basic structure for running ntHits requires an output type, input files, a frequency histogram, and an output filename.

```bash
nthits [options] --frequencies <hist_file> --out-file <output_name> <out_type> <input_files...>
```

**Output Types (`out_type`):**
- `bf`: Standard Bloom filter (presence/absence).
- `cbf`: Counting Bloom filter (stores k-mer counts).
- `table`: Tabular format.

### Common CLI Patterns

**1. Filtering for Solid K-mers**
To automatically filter out erroneous k-mers (low frequency) and keep "solid" k-mers in a Bloom filter:
```bash
nthits --frequencies freq.hist --solid --kmer-length 64 --out-file solid_kmers.bf bf reads.fastq
```

**2. Extracting Specific Frequency Ranges**
To isolate k-mers that appear between 10 and 100 times:
```bash
nthits --frequencies freq.hist --min-count 10 --max-count 100 --out-file repeats.bf bf reads.fastq
```

**3. Processing Long Reads**
When working with PacBio or Oxford Nanopore data, use the optimized data reader:
```bash
nthits --frequencies freq.hist --long-mode --out-file long_reads.bf bf long_reads.fasta
```

**4. Using Spaced Seeds**
For increased sensitivity in certain alignments, use spaced seed patterns instead of contiguous k-mers:
```bash
nthits --frequencies freq.hist --seeds 11011,10101 --out-file spaced.bf bf reads.fastq
```

### Querying Results
If you generated a Bloom filter (`bf`) or Counting Bloom filter (`cbf`), use the companion tool `nthits-bfq` to query specific k-mers.

**Query a Counting Bloom Filter:**
```bash
nthits-bfq --cbf input.cbf
```

## Expert Tips and Best Practices

- **Frequency Histogram Requirement**: ntHits requires a frequency histogram file (typically generated by `ntCard`). Ensure this file is accurate, as the `--solid` mode and count filtering rely entirely on these statistics.
- **Memory Optimization**: The target Bloom filter error rate (`--error-rate`, default 0.0001) directly impacts memory usage. Increasing the error rate slightly can reduce the memory footprint for extremely large datasets.
- **Thread Scaling**: Use the `--threads` (or `-t`) flag to match your CPU cores. ntHits is highly parallelized and scales well with additional threads.
- **K-mer Length Limits**: The default k-mer length is 64. If your analysis requires longer k-mers, ensure your input frequency histogram was also generated with the same `k` value.
- **API Integration**: For developers, the output Bloom filters are compatible with the `btllib` C++ and Python libraries, allowing for programmatic querying without using the CLI query tool.
