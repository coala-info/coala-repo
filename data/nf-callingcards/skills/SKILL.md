# callingcards

## Overview
The nf-core/callingcards pipeline is designed for the automated processing of Calling Cards experimental data, a method used to track transcription factor binding or transposon insertions. It handles the complexities of demultiplexing yeast libraries or processing mammalian data to identify and count "hops" across the genome.

Users provide raw sequencing data and metadata describing the barcodes associated with specific transcription factors. The pipeline produces aligned reads, quantified insertion sites, and comprehensive quality control reports to validate the experimental run.

## Data preparation
Input requires a CSV samplesheet and a barcode details JSON file. For yeast, the FASTQ files are typically multiplexed, while the mammalian workflow currently expects single-end R1 reads.

**Samplesheet columns:**
- `sample`: Unique sample identifier (no spaces).
- `fastq_1`: Path to Read 1 FASTQ file (`.fastq.gz` or `.fq.gz`).
- `fastq_2`: Path to Read 2 FASTQ file (optional for mammals).
- `barcode_details`: Path to a JSON file defining the barcodes for the library.

Example `samplesheet.csv`:
```csv
sample,fastq_1,fastq_2,barcode_details
run_6177,run_6177_sample_R1.fastq.gz,run_6177_sample_R2.fastq.gz,run_6177_barcode_details.json
```

Reference requirements include a FASTA genome and a GTF annotation. If not using iGenomes, you must provide `--fasta`. You can also provide a `regions_mask` BED file to hard-mask specific genomic regions.

## How to run
Execute the pipeline using the `nextflow run` command. It is recommended to use the provided convenience profiles for yeast or mammalian data to set appropriate defaults.

```bash
nextflow run nf-core/callingcards \
  -profile docker,default_yeast \
  --input samplesheet.csv \
  --datatype yeast \
  --outdir ./results
```

For mammalian data using iGenomes:
```bash
nextflow run nf-core/callingcards \
  -profile singularity,default_mammals \
  --input samplesheet.csv \
  --datatype mammals \
  --genome GRCh38 \
  --outdir ./results
```

Key parameters include `--aligner` (defaults to `bwamem2`, but also supports `bwa`, `bowtie`, or `bowtie2`) and `--min_mapq` (defaults to 10) to control which transpositions are counted. Use `-resume` to restart a run from the last successful step.

## Outputs
Results are saved in the directory specified by `--outdir`. Key deliverables include:

- **MultiQC**: A summary report (`multiqc_report.html`) found in the main output folder, aggregating QC metrics from FastQC, alignment, and hop counting.
- **Hops**: Quantified calling card insertions (transpositions) generated by `callingCardsTools`.
- **Alignment**: Sorted and indexed BAM files; intermediate files like un-indexed BAMs can be saved using `--save_alignment_intermediate`.
- **QC Metrics**: Detailed reports from RSeQC (e.g., read distribution) and Picard.

For more details about the output files and reports, refer to the [official output documentation](https://nf-co.re/callingcards/output).
