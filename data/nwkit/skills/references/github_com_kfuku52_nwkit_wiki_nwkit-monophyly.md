# nwkit monophyly

## Overview

`nwkit monophyly` tests whether species-defined or trait-defined groups are monophyletic in a tree.

The output is a tab-separated table containing one row per group.

## Species-based grouping

### Command

```
nwkit monophyly -i tree.nwk -o monophyly.tsv
```

When no trait table is provided, groups are inferred from the configured species parser.

## Trait-based grouping

### Command

```
nwkit monophyly -i tree.nwk -o monophyly.tsv \
  --trait traits.tsv \
  --group_by habitat
```

The trait table must contain a `leaf_name` column and the selected grouping column.

### Example input (`tree.nwk`)

```
(((A:1,B:1):1,C:1):1,(D:1,E:1):1);
```

### Example input (`traits.tsv`)

| leaf\_name | group |
| --- | --- |
| A | x |
| B | x |
| C | y |
| D | z |
| E | y |

### Example output (`monophyly.tsv`)

| group | status | is\_monophyletic | num\_target\_leaves | num\_mrca\_leaves | num\_intruder\_leaves | target\_leaves | intruder\_leaves |
| --- | --- | --- | --- | --- | --- | --- | --- |
| x | monophyletic | True | 2 | 2 | 0 | A,B |  |
| y | polyphyletic | False | 2 | 5 | 3 | C,E | A,B,D |
| z | monophyletic | True | 1 | 1 | 0 | D |  |

## Unrooted mode

### Command

```
nwkit monophyly -i tree.nwk -o monophyly.tsv \
  --trait traits.tsv \
  --group_by habitat \
  --unrooted yes
```

## Other options

* `--fail_on_non_monophyly yes|no`: exit with a non-zero status when any group is not monophyletic
