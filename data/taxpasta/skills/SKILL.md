## Overview
Taxpasta is a powerful command-line tool designed to standardize and aggregate taxonomic profiles generated by a wide array of bioinformatics tools. It addresses the challenge of disparate output formats from different profilers by converting them into a unified, comparable table. This allows for seamless merging of profiles across samples and facilitates downstream analysis by providing consistent taxonomy identifiers and their corresponding counts.

## Usage Instructions

Taxpasta's primary interface is its command-line interface (CLI). You can explore all available commands and their options using the help flags.

### Core Commands

Taxpasta offers two main commands for its core functionalities: `standardise` and `merge`.

#### 1. Standardise

This command normalizes a single taxonomic profile file into a consistent two-column format: `taxonomy_id` and `count`. You must specify the profiler that generated the input file.

**Basic Usage:**

```bash
taxpasta standardise -p <profiler_name> -o <output_file.tsv> <input_profile.txt>
```

*   `-p <profiler_name>`: Specifies the taxonomic profiler used (e.g., `metaphlan`, `kraken2`, `bracken`). Refer to the taxpasta documentation for a full list of supported profilers.
*   `-o <output_file.tsv>`: The name of the output file where the standardized data will be saved.
*   `<input_profile.txt>`: The path to the input taxonomic profile file.

**Example:** Standardizing a MetaPhlAn profile:

```bash
taxpasta standardise -p metaphlan -o standardised_profile.tsv input_metaphlan.txt
```

#### 2. Merge

This command takes multiple taxonomic profile files (which can be raw or already standardized) and merges them into a single table. The output table will include the `taxonomy_id` and a separate column for the counts from each input profile.

**Basic Usage:**

```bash
taxpasta merge -p <profiler_name> -o <merged_output.tsv> <input_profile_1.txt> <input_profile_2.txt> ...
```

*   `-p <profiler_name>`: Specifies the taxonomic profiler used for all input files. This is crucial for correct parsing.
*   `-o <merged_output.tsv>`: The name of the output file for the merged table.
*   `<input_profile_1.txt> <input_profile_2.txt> ...`: One or more input profile files to be merged. Wildcards can often be used (e.g., `*.txt`).

**Example:** Merging multiple MetaPhlAn profiles:

```bash
taxpasta merge -p metaphlan -o merged_profiles.tsv sample1.txt sample2.txt sample3.txt
```

Or using a wildcard:

```bash
taxpasta merge -p metaphlan -o merged_profiles.tsv MOCK_*.metaphlan3_profile.txt
```

### Optional Dependencies and Output Formats

Taxpasta supports additional output formats through optional dependencies. You can install these using pip with the `extras` syntax.

**Installing with extras:**

```bash
pip install 'taxpasta[rich,biom,arrow,parquet,ods,xlsx]'
```

*   `rich`: For rich-formatted command-line output and logging.
*   `biom`: To output tables in BIOM format.
*   `arrow`: To output tables in Apache Arrow format.
*   `parquet`: To output tables in Apache Parquet format.
*   `ods`: To output tables in ODS format.
*   `xlsx`: To output tables in Microsoft Excel format.

When using the `standardise` or `merge` commands, you can specify the output format using the `-O` or `--output-format` flag, provided the corresponding optional dependency is installed.

**Example:** Merging and outputting to Excel format:

```bash
taxpasta merge -p metaphlan -o merged_profiles.xlsx --output-format xlsx sample1.txt sample2.txt
```

## Expert Tips

*   **Profiler Consistency:** Always ensure you specify the correct profiler name with the `-p` flag. Mismatches can lead to incorrect parsing and erroneous results.
*   **Input File Types:** While the examples show `.txt` files, taxpasta can handle various tabular formats. Consult the documentation for specific profiler input requirements.
*   **Output File Naming:** Use descriptive filenames for your output to easily identify the standardization or merge operation performed.
*   **Downstream Analysis:** The standardized output (typically `taxonomy_id` and `count`) is ideal for direct input into other bioinformatics analysis tools or for creating summary reports.
*   **Error Handling:** If you encounter issues, check the input file format and ensure the correct profiler is specified. The `--verbose` flag can provide more detailed output for debugging.
