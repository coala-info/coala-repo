# bioconductor-drivernet

## Overview

DriverNet is an R package designed to identify driver mutations by quantifying their "influence" on gene expression networks. It operates on the principle that functional driver mutations will be associated with a large number of outlying gene expression events in genes they are known to interact with (via a pathway-derived influence graph). The package uses a greedy optimization algorithm to find the minimum set of mutated genes that explains the maximum number of expression outliers across a patient cohort.

## Typical Workflow

### 1. Data Preparation
DriverNet requires four primary inputs:
*   **Mutation Matrix**: A binary matrix (patients x genes) where 1 indicates a mutation or CNV.
*   **Outlier Matrix**: A binary matrix (patients x genes) where 1 indicates the gene is an expression outlier in that patient.
*   **Influence Graph**: An adjacency matrix representing gene-gene interactions (e.g., from Reactome or other pathway databases).
*   **Gene Name List**: A character vector of all genes considered in the analysis.

```r
library(DriverNet)

# Load example GBM data
data(samplePatientMutationMatrix)
data(samplePatientOutlierMatrix)
data(sampleInfluenceGraph)
data(sampleGeneNames)
```

### 2. Ranking Driver Genes
The `computeDrivers` function executes the greedy algorithm to rank mutated genes based on their coverage of expression outliers.

```r
# Compute the ranked list of drivers
driversList = computeDrivers(samplePatientMutationMatrix, 
                             samplePatientOutlierMatrix, 
                             sampleInfluenceGraph, 
                             outputFolder=NULL, 
                             printToConsole=FALSE)

# View top 10 predicted drivers
topDrivers = drivers(driversList)[1:10]
print(topDrivers)
```

### 3. Statistical Significance (Null Model)
To assess if the coverage of a gene is higher than expected by chance, use `computeRandomizedResult`. This performs a label-permutation test.

```r
randomDriversResult = computeRandomizedResult(
    patMutMatrix=samplePatientMutationMatrix,
    patOutMatrix=samplePatientOutlierMatrix,
    influenceGraph=sampleInfluenceGraph,
    geneNameList=sampleGeneNames, 
    numberOfRandomTests=100, # Higher values increase precision
    purturbData=TRUE
)
```

### 4. Summarizing Results
The `resultSummary` function combines the driver rankings with the p-values and provides detailed metadata, including which patients were "covered" by which driver.

```r
res = resultSummary(driversList, 
                    randomDriversResult, 
                    samplePatientMutationMatrix, 
                    sampleInfluenceGraph)

# View results: includes rank, p-value, total_events, and covered_events
head(res)
```

## Key Functions
*   `computeDrivers()`: Main greedy optimization engine.
*   `drivers()`: Accessor to extract the ranked gene list from the result object.
*   `computeRandomizedResult()`: Generates a background distribution to calculate p-values.
*   `resultSummary()`: Formats the final output into a readable dataframe with clinical/patient associations.

## Tips for Success
*   **Outlier Definition**: The `samplePatientOutlierMatrix` is typically generated by identifying genes with expression values > 2 standard deviations from the mean of a reference (normal) population.
*   **Influence Graph**: The performance of DriverNet is highly dependent on the quality of the influence graph. Ensure the gene symbols in your mutation/expression data match the symbols used in the graph.
*   **Computational Cost**: `computeRandomizedResult` is the most time-consuming step. For initial exploratory runs, use a low `numberOfRandomTests` (e.g., 20), but increase this to 500-1000 for publication-quality p-values.
