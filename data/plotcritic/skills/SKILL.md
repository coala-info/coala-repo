# plotcritic

## Overview
PlotCritic is a deployment tool that transforms a directory of images into a functional, local HTML/JavaScript website for rapid curation. It allows users to define custom questions and keyboard-mapped answers, enabling efficient human-in-the-loop verification of visual data. The tool is particularly optimized for genomic workflows but can be used for any image-scoring task.

## Installation
Install PlotCritic via Bioconda:
```bash
conda install -c bioconda plotcritic
```

## Core CLI Usage
The primary command is `plotcritic`. It generates a project directory containing an `index.html` file and all necessary assets.

### Basic Project Creation
To set up a standard curation project, provide a unique project name, the image source, and the curation parameters:
```bash
plotcritic \
  -p project_name \
  -i path/to/images/ \
  -q "Does this plot show a valid signal?" \
  -A "y":"Yes" "n":"No" "u":"Unsure"
```
*   `-p`: Unique project name (creates the output directory).
*   `-i`: Directory containing images (and optional metadata .json files).
*   `-q`: The question displayed to the curator.
*   `-A`: Answer options formatted as `'key':'value'`.

### Genomic SV Curation (Samplot Integration)
If using images generated by Samplot, use the `-s` flag to automatically include genomic metadata fields (chrom, start, end, sv_type) in the reports:
```bash
plotcritic -p sv_curation -i ./samplot_plots -s
```

### Custom Metadata Reporting
To include specific metadata from sidecar `.json` files (which must match the image filenames), use the `-R` and `-S` flags:
```bash
plotcritic \
  -p custom_project \
  -i ./data \
  -R field1 field2 field3 \
  -S field1 field2
```
*   `-R`: List of fields from the JSON metadata to include in the final report.
*   `-S`: Subset of report fields to display in the web-based summary table.

## Curation Workflow
1.  **Generate Website**: Run the `plotcritic` command to create the project folder.
2.  **Launch**: Open the `index.html` file within the project folder in any modern web browser.
3.  **Score**: Navigate to "Score Images," set a username, and use the defined hotkeys or buttons to score images.
4.  **Save Progress**: Click "Download Report" to save a JSON/CSV of the scores. **Warning**: Closing or refreshing the tab before downloading will result in data loss.
5.  **Aggregate**: To combine results from multiple curators, go to "View Report" and upload all downloaded scoring files.

## Expert Tips
*   **Metadata Matching**: For custom fields to work, each image (e.g., `plot1.png`) must have a corresponding JSON file (e.g., `plot1.json`) in the same directory.
*   **Team Distribution**: For collaborative curation, compress the generated project directory and send it to team members. They can score locally and return the downloaded report files for central aggregation.
*   **Security**: Since the website runs locally or can be hosted on a private server, it is suitable for sensitive scientific data that cannot be uploaded to public curation platforms.
