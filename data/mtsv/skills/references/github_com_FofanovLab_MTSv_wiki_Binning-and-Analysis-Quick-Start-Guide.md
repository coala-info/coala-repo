# Overview

The main MTSv pipeline is run with the `mtsv` command. Which provides a list of the subcommands.

```
$ mtsv
Usage: mtsv [OPTIONS] COMMAND [ARGS]...

Options:
  --version  Show the version and exit.
  --help     Show this message and exit.

Commands:
  init               Initialize new config file with default parameters.
  cluster-init       Initialize new cluster config file.
  pipeline           Run MTSv pipeline from readprep to analyze.
  readprep           QC reads, generate and deduplicate query kmers.
  binning            Alignment-based metagenomic binning.
  summary            Summarize taxa hits.
  analyze            Perform statistical analysis on candidate taxa.
  report             Generate report.
  extract            Extract queries that hit a given taxid.
  extract-unaligned  Extract queries that did not align to any taxa.
  wgfast             SNP typing for strain-level resolution.
  concoct            Alignment-free binning of unaligned queries.
  datastore          Manage datastore of expected value calculations.
```

The main MTSv pipeline is running a Snakemake file and most of the commands accept Snakemake options that will be passed to Snakemake. Therefore, it is very helpful to be familiar with the [Snakemake](https://snakemake.readthedocs.io/en/stable/) options and documentation. Also see our [Snakemake Tips](https://github.com/FofanovLab/MTSv/wiki/Snakemake-Tips) for additional help.

## Init

Typically, a new pipeline run should start with `mtsv init`. This command generates a new config file, pre-filled with default values. **A config file is required to run most of the other commands.**

```
$ mtsv init --help
Usage: mtsv init [OPTIONS] DB_PATH

  Initialize new config file with default parameters. Requires path to
  sequence database config file.

Options:
  -c, --configfile FILENAME  Specify path to write config file.  [default:
                             mtsv.cfg]
  --help                     Show this message and exit.
```

To run the main pipeline you should have already completed the [database download and setup](https://github.com/FofanovLab/MTSv/wiki/Sequence-Download-and-Setup-Quick-Start-Guide). In the download and setup directory, a database config file was generated in the `artifacts` subdirectory. The `init` command requires you to pass the path to the database config file (`DB_PATH`) that is located in the `artifacts` directory (e.g. `/path/to/database/artifacts/genbank.json`) so that it can pre-fill this value in the config.

By default the config will be written in the current working directory as `mtsv.cfg` but a different location can be provided using the `--configfile` option.

See [sample config](https://github.com/FofanovLab/MTSv/wiki/Sample-Config).

## Cluster Init

Cluster init is only required if you want to run the pipeline using Snakemake's cluster mode. This is recommended if you want to add the jobs to a scheduler on a cluster.

```
$ mtsv cluster-init --help
Usage: mtsv cluster-init [OPTIONS]

  Initialize new cluster config file. Sections for each rule are provided
  with defaults but values should be modified depending on the size of the
  dataset.

Options:
  -c, --configfile FILENAME  Specify path to write cluster config file.
                             [default: cluster.cfg]
  --help                     Show this message and exit.
```

Cluster init creates a yaml config file (default: `cluster.cfg`) with sections for each of the pipeline rules. In this file, resources can be defined for each rule, otherwise the values under `__default__` will be used.

By default, the config will be written in the current working directory as `cluster.cfg` but a different location can be provided using the `--configfile` option.

```
$ head cluster.cfg

  __default__:
    cpus: '{threads}'
    mem: 5000
    log: '{log}.cluster'
    jobname: '{rule}'
    time: "30:00"

  fastp:
    jobname: "fastp"

  readprep:
    jobname: "readprep"
```

The values in curly braces `{}` are pulled from the rules in the Snakefile. So by default,
the maximum number of threads defined for each rule will be the number of cpus requested.

See sample [cluster config](https://github.com/FofanovLab/MTSv/wiki/Sample-Cluster-Config) for complete file.

To run the pipeline in cluster mode (SLURM Example):

```
$ mtsv pipeline --cluster-config cluster.cfg --cluster "sbatch --mem {cluster.mem} --output {cluster.log} --job-name {cluster.jobname} --time {cluster.time} --cpus-per-task {cluster.cpus}"
```

The VALUES in `{cluster.VALUES}` are pulled from the cluster config for each rule, using the default value if it is not provided for the rule.

The cluster config can be modified to include scheduler specific variables. For example, `log: '{log}.cluster_%A'` where the `%A` is the Slurm job id.

See [Snakemake's Cluster Configuration Guide](https://snakemake.readthedocs.io/en/stable/snakefiles/configuration.html#cluster-configuration) for more information.

## Pipeline

The pipeline command runs the complete pipeline from the Readprep module to the Analyze module (see below for module details).
Running any of the main pipeline modules on their own (e.g. `readprep`, `binning`, `summary`, or `analyze`) will simply stop the pipeline at that step (by passing `--until MODULE` to Snakemake).

```
$ mtsv pipeline --help
Usage: mtsv pipeline [OPTIONS] [SNAKEMAKE_ARGS]...

  Run MTSv pipeline from readprep to analyze.

  Additional Snakemake parameters should be passed at the command line.

  Runs:

  'snakemake --snakefile SNAKEPATH --configfile CONFIGFILE'

Options:
  -c, --configfile FILENAME  Specify path to write config file.  [required]
  --ignore-changes           Do not check for changes to config parameters.
  --help                     Show this message and exit.
```

**NOTE:** Like most of the main module commands, the pipeline command requires a config file.
Any changes to the config file will force any previously completed steps to be
rerun if they are affected by the parameter changes. This is because MTSv adds
the targets produced by running Snakemake's `--list-params-changes` to each run.
This is important because the analysis step must simulate data using the same
parameters as the binning step and, if these parameters are changed without
rerunning the binning, the analysis would be incorrect. This can be overridden using the `--ignore-changes` option.

## Readprep

The Readprep module performs quality control on reads using
[fastp](https://github.com/OpenGene/fastp). After QC, it generates equal-sized, non-overlapping kmer queries from the reads.
These queries are deduplicated (while maintaining a record of the number of copies) to reduce the number of redundant alignments.
The final queries are written to a fasta file `queries/{sample}.fasta`.
**Note: The pipeline does not currently accept gzipped fastq files.**

### Usage

```
$ mtsv readprep --help
Usage: mtsv readprep [OPTIONS] [SNAKEMAKE_ARGS]...

  QC reads, generate and deduplicate query kmers.

  Additional Snakemake parameters should be passed at the command line.

  Runs:

  'snakemake --snakefile SNAKEPATH --configfile CONFIGFILE --until readprep'

Options:
  -c, --configfile FILENAME  Specify path to write config file.  [required]
  --ignore-changes           Do not check for changes to config parameters.
  --help                     Show this message and exit.
```

### Parameters

---

* `fastp_params`: Additional parameters to be passed to fastp. `[default: ""]`

Default parameters are used but these are modified by passing
parameters as a string. Run `fastp --help` for options (avoid changing input/output and
thread parameters as these
