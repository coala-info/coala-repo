The relevant data used in this tutorial is accessible at: [![DOI](https://camo.githubusercontent.com/75764d327f9f7ba30b45feb7f0ce0c367b8fe65ab42803e38babfed1874aa430/68747470733a2f2f7a656e6f646f2e6f72672f62616467652f444f492f31302e353238312f7a656e6f646f2e31373033363333382e737667)](https://doi.org/10.5281/zenodo.17036338)

Warning

Make sure to download the relevant index file for each `.bam` (`.bam.bai` index), `.cram` (`.cram.crai` index), or `.gz` (`.tbi` index).

---

* `BH01.hg19.frag.bed.gz`: [DOWNLOAD](https://zenodo.org/records/14284133/files/BH01.hg19.frag.bed.gz?download=1)
* `BH01.hg19.frag.bed.gz.tbi`: [DOWNLOAD](https://zenodo.org/records/14284133/files/BH01.hg19.frag.bed.gz.tbi?download=1)
* `hg19.chrom.sizes`: [DOWNLOAD](https://zenodo.org/records/14284133/files/hg19.chrom.sizes?download=1)
* `b37.common_chr.10kb_interval.autosome.bed`: [DOWNLOAD](https://github.com/epifluidlab/FinaleToolkit/blob/main/tests/data/ftk-tutorial/b37.common_chr.10kb_interval.autosome.bed)

---

For 1, 2, 4, 8, 16 cores using the -w option, run the following command:

```
finaletoolkit wps \
    BH01.hg19.frag.bed.gz \
    b37.common_chr.10kb_interval.autosome.bed \
    -c hg19.chrom.sizes \
    -o BH01.hg19.$CORES.raw_wps.bw \
    -w $CORES \
    -v
```

Determine the max RSS (Resident Set Size) for the job and retrieve the time taken. We looked at the SLURM scheduler logs for this.

Plot the figure.

```
import pandas as pd
import numpy as np
import matplotlib as mpl
import matplotlib.pyplot as plt

data = pd.read_csv("wps_bh01_benchmark.tsv", sep="\s+")
def parse_time(hms):
    hours, minutes, seconds = hms.split(':')
    return 3600 * int(hours) + 60 * int(minutes) + int(seconds)

data['Elapsed'] = data['Elapsed'].apply(parse_time)

plt.figure(figsize=(4,3))
bar = plt.bar([str(core) for core in np.insert(data.loc[8:3:-1,"cores"].values, 0, [1])], np.insert(data.loc[8:3:-1,"Elapsed"].values, 0, [222447])/60/60, color=['C1', 'C0','C0','C0','C0','C0',])
plt.xlabel("Number of Processes")
plt.ylabel("Elapsed Time (hours)")
for i, rect in enumerate(bar):
    height = rect.get_height()
    width = rect.get_x() + rect.get_width() / 2
    plt.text(width, height+0.3, f"{height:2.2f}", ha='center')
# legend
colors = {'Snyder et al (2016)':'C1', 'FinaleToolkit':'C0'}
labels = list(colors.keys())
handles = [plt.Rectangle((0,0),1,1, color=colors[label]) for label in labels]
plt.legend(handles, labels)

plt.ylim(top=70)
plt.subplots_adjust(left=0.15, bottom=0.15)
plt.savefig("wps_elapsed_time_bar.svg", format="svg",)

import matplotlib.pyplot as plt

# Data
processes = [1, 2, 4, 8, 16]
time = [8.66, 9.33, 10.85, 13.69, 48.80]

# Plotting
plt.figure(figsize=(4, 4), dpi=300)
plt.plot(processes, time, marker='o', color='skyblue', linewidth=2, markersize=10)

# Titles and labels
plt.title("WPS", fontsize=20, pad=10)
plt.xlabel("Number of Processes", fontsize=14)
plt.ylabel("Max RSS - Physical Memory (GB)", fontsize=14)

# Specify ticks explicitly
plt.xticks(ticks=processes, fontsize=12)
plt.yticks(fontsize=12)

# Spines and grid
plt.grid(False)
# Final touch
plt.tight_layout()
plt.show()
```
