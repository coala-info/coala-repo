# Running Eukfinder with long read data (PACBIO or ONT) or assembled contigs

**Notes**

* long read data is assumed to have already gone through basecalling, adapter trimming and quality control before using Eukfinder and should be in fastq format (.fastq or .fq) you can not use gzipped files
* assembled contigs are expected to be in fasta format (.fasta or .fa) you can not use gzipped files

## Usage

```
 Eukfinder long_seqs [-h] -l LONG_SEQS -o OUT_NAME --mhlen MHLEN
                            -n NUMBER_OF_THREADS -z NUMBER_OF_CHUNKS
                            -t TAXONOMY_UPDATE -e E_VALUE --pid PID --cov COV
```

## Required arguments:

**Description**

   -l, --long-seqs   LONG\_SEQS

* file containing input sequences (assembled contigs or long read sequencing data)

   -o, --out\_name   OUT\_NAME

* prefix for output results
* **Important Note:**
  The `-o` or `--out_name` parameter should be a simple filename prefix only.
  It must not include any path (e.g., /home/user/output\_prefix will cause an error).
  Eukfinder saves all outputs to the current directory, and assumes it is writable.
  Please make sure to run it inside a folder where you have permission to write files.

  --mhlen, --min-hit-length   MHLEN

* minimum hit length for Centrifuge searches it controls the minimum sequence length required for Centrifuge classification matches. Purpose is to filter out spurious short matches. A value of 50 works well for most datasets; lower values (e.g., 30-40) are more sensitive, catch shorter reads, but higher false-positive rate; higher values (e.g., 60-100) are more stringent, fewer false positives, but may miss real eukaryotic sequences

   -n, --number-of-threads   NUMBER\_OF\_THREADS

* number of threads, controls parallelization. Set based on available CPU cores. Higher values speed up processing but increase memory usage.

   -z, --number-of-chunks   NUMBER\_OF\_CHUNKS

* number of chunks the input files is split into for processing with PLAST. Higher values reduce memory per chunk put increase overhead. Recommendation is to use 2-4 for most datasets, for very large samples (>100M reads) try 4-8.

   -t, --taxonomy-update   TAXONOMY\_UPDATE

* set to True the first time you run otherwise set to False

   -e, --e-value   E\_VALUE

* e-value threshold for PLAST searches; lower values = more strigent

   --pid, --percent\_id   PID

* percent identity to be used for PLAST searches; higher values = more stringent

   --cov, --coverage   COV

* percent coverage to be used for PLAST searches; higher value = more stringent

## optional arguments:

-h, --help

* show this help message and exit

 --cdb, --centrifuge-database   CDB

* path to downloaded (or created) **Basename** of Centrifuge index files
* **Important Note:**
  Make sure to define the Centrifuge database using the common filename prefix
  (before .1.cf, .2.cf, etc.), not the full path to a single .cf file.

   -p, --plast-database   PLAST\_DATABASE

* path to downloaded (or created) PLAST database

   -m, --plast-id-map   PLAST\_ID\_MAP

* path to downloaded (or created) PLAST database taxonomy map

## Example command to run Eukfinder\_long

```
     conda activate Eukfinder

     Eukfinder long_seqs -l "long_read.fq or contig.fa file" -o output_prefix  \
                                    -n 48 -z 2 -t False  -e 0.01 --pid 60 --cov 30 --mhlen 100
```

## Expected output from Eukfinder long\_seqs

* log file (Long\_seqs\_DATE.log) - details of the script running
* **Eukfinder\_results** folder - location of main results
* **Intermediate\_data** folder - location of intermediate results

**Eukfinder\_results** folder:

* this will contain the classified long read or contig data by classification type
  + There can be up to 6 different classification files: Euk, Unk, EUnk (combined Euk and Unk), Bact, Arch, and Misc
* final result format will vary depending on input sequence type
  + long read fastq files will result in fastq files with naming format of: output\_prefix.Classification\_type.un.fq
  + assembled read files will result in fasta files with naming format of: output\_prefix.Classification\_type.fasta

**Intermediate\_data** folder:

* contains intermediate results
  + classification results from Centrifuge
  + intermediate files from PLAST classification
