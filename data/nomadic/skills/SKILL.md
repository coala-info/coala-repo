# nomadic

## Overview

Nomadic is a specialized pipeline and dashboard designed to process nanopore sequencing data as it is generated. It transforms raw data into actionable insights by performing read mapping with Minimap2 and variant calling (using bcftools or the internal "delve" caller) while the sequencer is still running. The results are visualized in an interactive graphical dashboard, making it ideal for field-based genomics and rapid sample evaluation.

## Core Workflows

### 1. Workspace Initialization
Before starting an analysis, you must set up a workspace tailored to your specific reference genome or amplicon panel.

```bash
# Initialize a workspace for P. falciparum
nomadic start pfalciparum
```

### 2. Real-time Analysis
Once the workspace is created, navigate into it and launch the real-time monitoring service. This command watches for new FASTQ files generated by MinKNOW.

```bash
cd nomadic
# Start the real-time pipeline for a specific experiment
nomadic realtime <experiment_name>
```

### 3. Simulation and Testing
To verify the pipeline setup without an active sequencing run, use the included simulation script to move example data into the monitored directory.

```bash
# Run the simulation script from the nomadic repository
python scripts/simulate_sequencing.py
```

## Expert Tips and Best Practices

- **Metadata Preparation**: A metadata file is required for real-time analysis. Ensure your sample IDs in the metadata match the barcodes or file prefixes expected by the pipeline.
- **Resource Management**: For high-throughput runs, specify the number of threads to optimize mapping and variant calling speed:
  ```bash
  nomadic realtime <experiment_name> --threads 8
  ```
- **Directory Structure**: Always run `nomadic realtime` from within the directory created by `nomadic start`. The tool expects a specific internal folder structure (e.g., `data/`, `results/`) to function correctly.
- **Dashboard Access**: Once `realtime` is active, the dashboard is typically served locally. Use it to monitor amplicon coverage depth across samples to determine if a run has reached sufficient depth to stop sequencing early.
- **Variant Caller Selection**: While `bcftools` is the standard, `delve` is optimized for specific nomadic workflows. Treat preliminary real-time calls with caution and validate them after the run completes.



## Subcommands

| Command | Description |
|---------|-------------|
| nomadic backup | Backup entire nomadic workspace and associated minknow data to a different folder e.g. on a local hard disk drive. |
| nomadic configure | Configure different nomadics functionality. This mostly sets standard   options in '.config.yaml' that can be overwritten from the command line. |
| nomadic dashboard | Launch the dashboard without performing real-time analysis, used to view results of a previous experiment. |
| nomadic download | Download target reference genome by specifying a `reference_name`, or download all available genomes with the `--all` flag |
| nomadic process | (Re)Process data that was produced by MinKNOW |
| nomadic realtime | Analyse data being produced by MinKNOW while sequencing is ongoing |
| nomadic share | Share summary nomadic workspace and associated minknow data to another folder e.g. a cloud synchronised folder for sharing. |
| nomadic start | Get started with nomadic. |
