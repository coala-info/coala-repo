# translate-gard

## Overview
The `translate-gard` utility is a specialized bioinformatics tool designed to bridge the gap between legacy HyPhy GARD outputs and modern data formats. GARD is a widely used method for detecting recombination breakpoints in nucleotide sequences, but its native output from version 2.3.2 requires transformation to be compatible with web-based visualizers (such as those in the HyPhy Vision suite) or JSON-based pipelines. This tool automates that conversion, ensuring that recombination site data, model fits, and topological information are preserved in a machine-readable schema.

## Installation and Setup
The tool is available via Bioconda and Node.js environments.

*   **Conda**: `conda install -c bioconda translate-gard`
*   **NPM**: `npm install -g translate-gard`

## Command Line Usage
The tool operates as a standard CLI utility. It typically reads a GARD output file and produces JSON.

### Basic Conversion
To convert a GARD output file to JSON, use the following pattern:
`translate-gard <input_file> > <output_file>.json`

### Common CLI Patterns
*   **Help**: Access the help menu to see available flags and version information.
    `translate-gard --help`
*   **Input Handling**: The tool is designed specifically for HyPhy 2.3.2 GARD results. Ensure the input file is the primary output generated by the GARD processor.

## Best Practices and Expert Tips
*   **Version Specificity**: This tool is specifically optimized for HyPhy version 2.3.2. If you are using a significantly newer or older version of HyPhy, verify the JSON output structure, as GARD output formats have evolved.
*   **Rate Variation**: The tool includes specific logic to handle GARD runs where rate variation was enabled. If your analysis involved complex site-to-site rate variation, `translate-gard` is the preferred tool to ensure the baseline scores are calculated correctly in the resulting JSON.
*   **Downstream Visualization**: The JSON produced by this tool is the standard input format for the [HyPhy Vision](https://vision.hyphy.org/GARD) web application. After conversion, the JSON file can be uploaded directly to visualize breakpoints and phylogenetic trees for each segment.
*   **Error Handling**: If the tool fails, ensure the input file is not truncated. GARD runs can be computationally intensive and occasionally terminate prematurely; `translate-gard` requires a complete output file to parse the final model results.
