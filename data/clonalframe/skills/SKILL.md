# clonalframe

## Overview

ClonalFrameML is a maximum likelihood implementation of the ClonalFrame model designed for large-scale bacterial genomic datasets. It identifies genomic regions introduced via horizontal gene transfer (recombination) and adjusts phylogenetic branch lengths to reflect the true clonal relationship between isolates. Use this skill when you need to distinguish between mutation and recombination, or when you require a phylogeny that is not distorted by the high density of substitutions typically found in recombinant regions.

## Input Requirements

- **Starting Tree**: A phylogeny in Newick format (e.g., generated by RAxML, PhyML, or FastTree).
- **Sequence Alignment**: An alignment in FASTA or Extended Multi-Fasta (XMFA) format.
- **Consistency**: The number of leaves in the tree must match the number of sequences in the alignment, and the names must be identical.

## Common CLI Patterns

### Standard Analysis
The most common usage involves estimating global recombination parameters across the entire tree.

```bash
ClonalFrameML starting_tree.nwk alignment.fasta output_prefix -kappa 4.5 -emsim 100
```

- `-kappa`: The transition/transversion ratio (usually obtained from the phylogenetic software used to build the starting tree).
- `-emsim`: Number of bootstrap replicates to estimate parameter uncertainty.

### Handling Missing or Non-Core Data
To prevent uncalled sites or gaps from creating artificial recombination signals, use the `-ignore_user_sites` option with a file containing positions to exclude.

```bash
ClonalFrameML tree.nwk alignment.fasta output_prefix -ignore_user_sites non_core_sites.txt
```

### Per-Branch Recombination Model
For datasets where recombination rates vary significantly across the tree, use the per-branch model. It is recommended to run a standard analysis first to obtain initial values.

```bash
ClonalFrameML tree.nwk alignment.fasta output_prefix -embranch true -initial_values "0.14 0.002 0.015"
```

## Output Interpretation

- **`output_prefix.importation_status.txt`**: A tab-delimited list of every detected recombination event, including the branch and genomic coordinates.
- **`output_prefix.labelled_tree.newick`**: The phylogeny with internal nodes labeled; essential for mapping events in the importation status file.
- **`output_prefix.em.txt`**: Point estimates for:
    - **R/theta**: Relative rate of recombination vs mutation.
    - **1/delta**: Inverse of the mean DNA import length.
    - **nu**: Mean divergence of imported DNA.
- **`output_prefix.ML_sequence.fasta`**: Reconstructed ancestral sequences for all internal nodes.

## Expert Tips

1. **Kappa Estimation**: Always provide a `-kappa` value. If unknown, estimate it using a tool like RAxML or PhyML on your alignment before running ClonalFrameML.
2. **Prior Adjustments**: If the tool fails to converge or produces unrealistic branch lengths, adjust the priors using `-prior_mean` and `-prior_sd`. The default order is R/theta, 1/delta, nu, and mean branch length.
3. **Visualization**: Use the bundled R script to generate a visual map of recombination:
   ```bash
   Rscript cfml_results.R output_prefix
   ```
4. **Memory/Time**: While much faster than the original Bayesian ClonalFrame, large datasets (hundreds of genomes) may still take several hours. Use the `-imputation_only true` flag if you only need ancestral state reconstruction without re-estimating parameters.
