# spliced_bam2gff

## Overview
`spliced_bam2gff` is a specialized utility designed to transform BAM alignment files into GFF2 (General Feature Format version 2). Unlike generic converters, it is optimized for spliced alignments—common in RNA-seq or long-read cDNA/direct RNA sequencing—where it interprets CIGAR strings to correctly define exon and intron boundaries. It is particularly useful for Oxford Nanopore Technologies (ONT) data processing pipelines.

## Installation
The recommended installation method is via Bioconda:
```bash
conda install -c bioconda spliced_bam2gff
```

## Common CLI Patterns

### Basic Conversion (Minimap2)
When using alignments generated by `minimap2 -x splice`, use the `-M` flag to ensure compatibility:
```bash
spliced_bam2gff -M input.bam > output.gff
```

### Handling Deletions as Introns
By default, the tool uses the `N` CIGAR feature to identify introns. If your aligner represents large gaps as deletions (`D`) rather than skipped regions (`N`), use the `-d` flag to set a size threshold:
```bash
# Classify any deletion larger than 20bp as an intron
spliced_bam2gff -d 20 -M input.bam > output.gff
```

### Determining Feature Orientation
Strand information is critical for GFF2. You can control how the tool determines orientation:
- **Read Strand (`-s`)**: Uses the BAM flag (SAM bitwise flag).
- **Strand Tag (`-g`)**: Uses the `XS` strand tag (often provided by aligners like GMAP or Minimap2) and falls back to the read strand if the tag is missing.

```bash
# Example using GMAP alignments with strand tags
spliced_bam2gff -g input_gmap.bam > output.gff
```

### Parallelization and Loci Splitting
For large datasets, you can partition the output into "loci" (regions separated by zero coverage). This enables easier parallel processing for downstream tools.
```bash
# Split output into a directory of loci
spliced_bam2gff -M -L ./output_loci_dir/ input.bam

# Bundle loci together (e.g., batches of at least 2000 features)
spliced_bam2gff -M -L ./output_bundles/ -b 2000 input.bam
```

## Expert Tips
- **Secondary Alignments**: By default, the tool discards secondary and supplementary alignments. To include them in your GFF2 output, use the `-S` flag.
- **Performance**: The tool defaults to using 8 cores (`-t 8`). Adjust this based on your environment to speed up the conversion of large BAM files.
- **Downstream Conversion**: GFF2 is a legacy format. If you require GFF3 or GTF for specific tools (like StringTie or featureCounts), pipe the output or use `gffread` to convert the resulting file:
  ```bash
  gffread output.gff -T -o output.gtf
  ```
- **Visualization**: The generated GFF2 files are natively supported by the Integrative Genomics Viewer (IGV) for inspecting splice junctions and transcript structures.
