# Understanding nanomonsv result filtering

nanomonsv `get` command performs several filters and records the result in the `Is_Filter` column. SVs that pass all filters are labeled `PASS`. Filtered SVs are labeled with one or more filter tags joined by `;`.

## Canonical SV filters

These filters are applied to canonical SVs (result.txt). Implemented in `post_proc.py`.

### Duplicate\_with\_close\_SV

Two SVs share the same chromosomes and directions, and the breakpoint positions are close together (within 30 bp by default). These likely represent the same SV detected from different supporting read clusters (e.g., soft-clipping vs CIGAR deletion). The duplicate with fewer supporting reads is removed (not shown in output).

### Duplicate\_with\_insertion

A non-insertion SV is part of a larger insertion-type SV. This occurs when the inserted sequence comes from another genomic region (e.g., LINE1 transduction). Around long insertions, alignment tools generate both soft-clipping supporting reads (which become translocation-type SVs) and insertion ('I' in CIGAR) supporting reads. The non-insertion SV is removed (not shown in output).

### Duplicate\_with\_close\_insertion

Two insertion-type SVs have breakpoints in close proximity (within 60 bp). These likely represent the same insertion split by the aligner. The insertion with the shorter insert sequence is removed (not shown in output).

### Too\_small\_size

Deletion or insertion whose size is below the threshold (50 bp by default). nanomonsv focuses on SVs of 50 bp or larger. Smaller SVs are labeled but not removed.

### Too\_low\_VAF

Variant allele frequency (VAF) is below the threshold (0.05 by default). VAF is calculated as the number of variant supporting reads divided by the total reads (variant + reference) around the breakpoint.

### SV\_with\_decoy

One or both breakpoints are located in decoy sequences (chromosome name ending with "decoy"). These SVs are typically unreliable.

### Simple\_repeat

Insertion or deletion confined within a simple repeat region. Only applied when `--simple_repeat_bed` is provided. The SV must be an indel type (same chromosome, +/- orientation) and both breakpoints must fall within a simple repeat region (with 30 bp margin).

## Single breakend SV filters

These filters are applied to single breakend SVs (sbnd.result.txt). Implemented in `post_proc.py`.

### Simple\_repeat (sbnd)

The breakpoint is located within a simple repeat region (with 50 bp margin). Only applied when `--simple_repeat_bed` is provided.

### Canonical\_SV\_overlap

The breakpoint of the single breakend SV overlaps with a breakpoint of a canonical SV (within 50 bp). Since the canonical SV already captures this breakpoint, the single breakend SV is redundant.

## Extracting PASS-only results

To extract only SVs that passed all filters:

```
# Canonical SVs
head -n 1 sample.nanomonsv.result.txt > sample.nanomonsv.result.pass.txt
tail -n +2 sample.nanomonsv.result.txt | grep -P '\tPASS$' >> sample.nanomonsv.result.pass.txt

# Single breakend SVs
head -n 1 sample.nanomonsv.sbnd.result.txt > sample.nanomonsv.sbnd.result.pass.txt
tail -n +2 sample.nanomonsv.sbnd.result.txt | grep -P '\tPASS$' >> sample.nanomonsv.sbnd.result.pass.txt
```

Note: Use `grep -P '\tPASS$'` (tab + PASS + end of line) to avoid matching partial strings like "PASS" within other columns.
