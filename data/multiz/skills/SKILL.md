## Overview
The `multiz` skill is designed for performing multiple sequence alignments of DNA sequences. It takes a set of related DNA sequences as input and produces an alignment that highlights conserved regions and evolutionary divergence. This is crucial for tasks such as understanding gene evolution, identifying functional elements, and constructing phylogenetic trees.

## Usage Instructions

The `multiz` tool is typically used via its command-line interface. The core functionality involves providing input alignment files and specifying parameters for the alignment process.

### Basic Alignment Workflow

1.  **Input Files**: `multiz` operates on pre-aligned pairwise alignments, often in formats like MAF (Multiple Alignment Format) or LAV (Local Alignment Viewer format). These pairwise alignments are typically generated by tools like `LASTZ` or `BLASTZ`.

2.  **Core Command Structure**:
    ```bash
    multiz <input_maf_or_lav_files> <species_tree_file> <output_maf_file> [options]
    ```

    *   `<input_maf_or_lav_files>`: One or more input alignment files. These can be specified individually or as a wildcard.
    *   `<species_tree_file>`: A file defining the evolutionary relationships (phylogenetic tree) between the species being aligned. This is crucial for `multiz` to correctly infer ancestral sequences and evolutionary events. The tree should be in a format compatible with `multiz` (e.g., Newick format).
    *   `<output_maf_file>`: The desired name for the output multiple alignment file.

### Key Options and Best Practices

*   **Specifying Input Alignments**:
    *   When providing multiple pairwise alignment files, ensure they cover the same genomic regions across the species defined in your tree.
    *   Wildcards can be used to specify multiple input files (e.g., `*.maf`).

*   **Species Tree (`<species_tree_file>`)**:
    *   The species names in the tree file **must exactly match** the species names used in the input alignment files.
    *   The tree structure dictates how `multiz` builds the multiple alignment. A well-defined tree is essential for accurate results.

*   **Output Format**:
    *   The default output format is MAF. This format is widely used in comparative genomics and is compatible with visualization tools like the UCSC Genome Browser.

*   **Controlling Alignment Behavior**:
    *   `--chain`: This option is often used to chain together multiple pairwise alignments into a single, contiguous multiple alignment. It's essential when dealing with large genomic regions that might be broken into smaller alignment blocks.
    *   `--fill`: This option can be used to fill in gaps in the alignment based on the provided species tree, attempting to infer ancestral sequences.
    *   `--gaps`: Controls how gaps are represented and handled in the alignment.

*   **Example Command Snippet**:
    ```bash
    # Assuming pairwise alignments are in pairwise_alignments/ and species tree is in species.tree
    multiz pairwise_alignments/*.maf species.tree output.maf --chain --fill
    ```

### Expert Tips

*   **Pre-alignment is Key**: `multiz` relies heavily on the quality of the input pairwise alignments. Ensure you use robust alignment tools (like `LASTZ`) with appropriate parameters for your specific sequences and species.
*   **Tree Accuracy**: The accuracy of your phylogenetic tree directly impacts the quality of the `multiz` output. Use established phylogenetic methods to construct your species tree.
*   **MAF Format**: Familiarize yourself with the MAF format for understanding the output and for compatibility with other bioinformatics tools. The `lav_format.html` document provides details on related alignment formats.
*   **Iterative Refinement**: For complex genomic regions or highly divergent species, you may need to iteratively refine your pairwise alignments and species tree to achieve optimal multiple alignment results.
