Mapper): java -jar x-mapper.jar [--out-mutations &lt;out.txt&gt;] [--out-vcf &lt;out.vcf&gt;] [--out-sam &lt;out.sam&gt;] [--out-refs-map-count &lt;counts.txt&gt;] [--out-unaligned &lt;unaligned.fastq&gt;] --reference &lt;ref.fasta&gt; --queries &lt;queries.fastq&gt; [options] java -jar x-mapper.jar [--out-mutations &lt;out.txt&gt;] [--out-vcf &lt;out.vcf&gt;] [--out-sam &lt;out.sam&gt;] [--out-refs-map-count &lt;counts.txt&gt;] [--out-unaligned &lt;unaligned.fastq&gt;] --reference &lt;ref.fasta&gt; --paired-queries &lt;queries.fastq&gt; [--spacing ] [options] Aligns genomic sequences quickly and accurately using relatively high amounts of memory INPUT: --reference &lt;file&gt; the reference to use. Should be in .fasta/.fa/.fna or .fastq/.fq format or a .gz of one of those formats. May be specified multiple times for multiple reference genomes --queries &lt;file&gt; the reads to align to the reference. Should be in .fastq or .fasta format. May be specified multiple times for multiple query files --paired-queries &lt;file1&gt; &lt;file2&gt; [--spacing &lt;mean&gt; &lt;distancePerPenalty&gt;] Illumina-style paired-end reads to align to the reference. For each read &lt;r1&gt; in &lt;file1&gt; and the corresponding &lt;r2&gt; in &lt;file2&gt;, the alignment position of &lt;r1&gt; is expected to be slightly before the alignment position of the reverse complement of &lt;r2&gt;. Each file should be in .fastq/.fasta format May be specified multiple times for multiple query files --spacing &lt;expected&gt; &lt;distancePerPenalty&gt; (default: 100 50) Any query alignment whose inner distance deviates from &lt;expected&gt; has an additional penalty added. That additional penalty equals (the difference between the actual distance and &lt;expected&gt;) divided by &lt;distancePerPenalty&gt;, unless the two query sequence alignments would overlap, in which case the additional penalty is 0. --infer-ancestors Requests that X-Mapper look for parts of the genome that likely shared a common ancestor in the past, and will lower the penalty of an alignment that mismatches the given reference but matches the inferred common ancestor. --no-infer-ancestors Disables ancestor inference. --split-queries-past-size &lt;size&gt; Any queries longer than &lt;size&gt; will be split into smaller queries. THIS OPTION IS A TEMPORARY EXPERIMENT FOR LONG READS TO DETECT REARRANGEMENTS AND IMPROVE PERFORMANCE. --allow-duplicate-contig-names if multiple contigs have the same name, continue instead of throwing an error. This can be confusing but shouldn't cause any incorrect results.
