## Overview

HiFiCNV is a specialized tool for identifying copy number variations (CNVs) in genomic data, particularly optimized for PacBio HiFi reads. It excels at segmenting and calling CNVs, automatically correcting for GC bias, and generating visualization tracks (bigwig files) for depth and allele frequency that can be inspected in genome browsers like IGV. The output is provided in standard formats like BEDGraph and VCF.

## Usage Instructions

HiFiCNV is a command-line tool. The primary workflow involves providing input BAM files and specifying output locations.

### Core Functionality

The main command structure is:

```bash
hificnv <input_bam_file> <output_prefix> [options]
```

### Key Options and Parameters

*   **Input BAM File**: The primary input is a BAM file containing PacBio HiFi reads.
*   **Output Prefix**: A prefix for all output files generated by the tool.
*   **`--threads`**: Number of threads to use for multi-threaded analysis.
*   **`--min_depth`**: Minimum read depth required for analysis.
*   **`--max_depth`**: Maximum read depth to consider.
*   **`--min_len`**: Minimum length of segments to call.
*   **`--max_len`**: Maximum length of segments to call.
*   **`--gc_correction`**: Enable or disable GC-bias correction (defaults to enabled).
*   **`--output_format`**: Specify output format for CNV calls (e.g., `bedgraph`, `vcf`).

### Example Workflow

A typical analysis might look like this:

1.  **Run HiFiCNV for CNV calling and depth visualization:**

    ```bash
    hificnv my_hifi_reads.bam output_cnv --threads 8 --min_len 1000 --max_len 1000000 --output_format vcf
    ```

    This command will:
    *   Process `my_hifi_reads.bam`.
    *   Generate output files prefixed with `output_cnv`.
    *   Use 8 threads for faster processing.
    *   Consider segments between 1kb and 1Mb for CNV calling.
    *   Output CNV calls in VCF format.

    This will produce files such as:
    *   `output_cnv.cnv.bedgraph`: Depth track in BEDGraph format.
    *   `output_cnv.allele.bedgraph`: Allele frequency track in BEDGraph format.
    *   `output_cnv.cnv.vcf`: CNV calls in VCF format.

2.  **Visualize results in IGV:**
    Load the generated `.bedgraph` files and the `.vcf` file into IGV to visualize the depth, allele frequency, and called CNVs.

### Expert Tips

*   **GC Bias Correction**: Always ensure GC-bias correction is enabled (`--gc_correction`) unless you have a specific reason not to. This is crucial for accurate CNV calling with HiFi data.
*   **Thread Usage**: Maximize the `--threads` option based on your system's available cores to speed up analysis.
*   **Parameter Tuning**: Experiment with `--min_len` and `--max_len` to fine-tune the sensitivity and specificity of CNV detection for your specific research question and data. Shorter minimum lengths might detect smaller CNVs but could increase false positives.
*   **Output Formats**: Choose the output format (`--output_format`) that best suits your downstream analysis. VCF is standard for variant calls, while BEDGraph is excellent for visualization.
