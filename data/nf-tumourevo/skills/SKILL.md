## Overview
nf-core/tumourevo is a bioinformatics pipeline designed to model tumour evolution from whole-genome sequencing (WGS) data. It performs downstream analysis of variant and copy-number calls from tumour-normal matched assays to reconstruct the evolutionary processes leading to the observed tumour genome.

The pipeline supports single-sample experiments as well as multi-region or longitudinal assays where multiple samples from the same patient are available. It integrates state-of-the-art tools for variant annotation, quality control of copy-number calls, subclonal deconvolution, clone tree inference, and mutational signature analysis.

## Data preparation
The pipeline requires a comma-separated samplesheet provided via the `--input` parameter. This file defines the relationships between datasets, patients, and samples, and points to the required genomic files.

Required columns in `samplesheet.csv`:
- `dataset`: Dataset identifier.
- `patient`: Patient identifier.
- `tumour_sample`: Tumour sample identifier.
- `normal_sample`: Normal sample identifier.
- `vcf`: Path to somatic VCF file (must be `.vcf` or `.vcf.gz`).
- `tbi`: Path to VCF index file (must be `.tbi`).
- `cna_segments`: Path to copy-number segments file (pattern: `Seg`, `segments`, or `subclones`).
- `cna_extra`: Path to purity/ploidy metadata (pattern: `confints_CP`, `purityploidy`, or `rho_and_psi`).
- `cna_caller`: The tool used for copy-number calling (supported: `sequenza`, `ASCAT`, `Battenberg`).
- `cancer_type`: The type of cancer (defaults to `PANCANCER`).

Minimal `samplesheet.csv` example:
```csv
dataset,patient,tumour_sample,normal_sample,vcf,tbi,cna_segments,cna_extra,cna_caller,cancer_type
dataset1,patient1,sample1,N1,p1_s1.vcf.gz,p1_s1.vcf.gz.tbi,seg.txt,purity.txt,sequenza,PANCANCER
dataset1,patient1,sample2,N1,p1_s2.vcf.gz,p1_s2.vcf.gz.tbi,seg.txt,purity.txt,sequenza,PANCANCER
```

## How to run
Run the pipeline using the `nextflow run` command. You must specify a profile (e.g., `docker` or `singularity`), the input samplesheet, and an output directory.

```bash
nextflow run nf-core/tumourevo \
   -profile docker \
   --input samplesheet.csv \
   --outdir ./results \
   --genome GRCh38
```

Key parameters:
- `-profile`: Computing environment (e.g., `docker`, `singularity`, `conda`).
- `--genome`: Reference genome name (e.g., `GRCh38`, `GRCh37`).
- `--fasta`: Path to the reference FASTA file.
- `--tools`: Comma-separated list of tools to run (default: `mobster,viber,pyclone-vi,sparsesignatures,sigprofiler`).
- `-resume`: Restart the pipeline from the last successful step.

## Outputs
Results are saved in the directory specified by `--outdir`. The pipeline produces several categories of results:

- **Variant Annotation**: Annotated VCF files generated by VEP.
- **Quality Control**: Reports from CNAqc and TINC assessing the quality of copy-number and variant calls.
- **Subclonal Deconvolution**: Clonal and subclonal population estimates from PyClone-VI, MOBSTER, or VIBER.
- **Clone Tree Inference**: Reconstructed evolutionary trees from ctree.
- **Signature Deconvolution**: Mutational signature profiles from SparseSignatures or SigProfiler.

For a detailed description of all output files and how to interpret them, refer to the [official output documentation](https://nf-co.re/tumourevo/output).
