YES >>&nbsp; &nbsp;&nbsp;&nbsp;&nbsp; NO >> Reproducible footprinting Requirements R version '2.15.' or higher R libraries &lsquo;mixtools&rsquo;, &lsquo;gtools&rsquo; and &lsquo;GenomicRanges&rsquo; Perl SAMtools version 1.1 BEDTools version 2.17.0 Download Linux users can download the pipeline from here . Installation Extract the contents of the downloaded archive: tar -xvzf footprint-pipeline-1.0.1.tar.gz Change into the pipeline's directory and configure: cd footprint-pipeline-1.0.1 ./configure --prefix=/tmp/my-test Note: If the necessary executables are not in the path, or their versions are incompatible with the pipeline (see the requirements section above), the full paths of the correct executables can be specified for the configuration. See below an example containing all BEDTools executables used by the pipeline, alongside SAMtools: ./configure COVERAGEBED=/path/to/coverageBed FASTAFROMBED=/path/to/fastaFromBed BAMTOBED=/path/to/bamToBed BEDTOBAM=/path/to/bedToBam BEDTOOLS=/path/to/bedtools SAMTOOLS=/path/to/samtools --prefix=/tmp/my-test Make, run the test and install: make make check make install Note: This will install the pipeline to /tmp/my-test directory. This can be changed by supplying the desired directory to the --prefix argument in the configuration step. Usage The main executable find_footprints.sh can be found in /bin, under the specified installation directory. Running "bash find_footprints.sh" will list the expected inputs. To run the pipeline, find_footprints.sh is executed in the following way: bash find_footprints.sh bam_file chrom_sizes motif_coords genome_fasta factor_name bias_file peak_file no_of_components background fixed_bg The arguments are explained below: &nbsp;&nbsp;&nbsp; bam_file: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The bam file from the ATAC-seq or DNase-seq experiment. &nbsp;&nbsp;&nbsp; chrom_sizes: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A tab delimited file with 2 columns, where the first column is &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; the chromosome name and the second column is the chromosome &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; length for the appropriate organism and genome build. &nbsp;&nbsp;&nbsp; motif_coords: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A 6-column file with the coordinates of motif matches (eg &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; resulting from scanning the genome with a PWM) for the &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; transcription factor of interest. The 6 columns should contain &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; chromosome, start coordinate, end coordinate, name, score and &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; strand information in this order. There should not be any additional &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; columns.The coordinates should be closed (1-based). &nbsp;&nbsp;&nbsp; genome_fasta: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A multi-fasta format file that contains the whole genome &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; sequence for the appropriate organism and genome build. This &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; file should be indexed (eg by using samtools faidx) and placed &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; in the same directory. &nbsp;&nbsp;&nbsp; factor_name: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The name of the transcription factor of interest supplied by &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; the user. This is used in the naming of the output files. &nbsp;&nbsp;&nbsp; bias_file: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A file listing the cleavage/transposition bias of the &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; different protocols, for all 6-mers. Provided options: ATAC, &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; DNase double hit or DNase single hit protocols. &nbsp;&nbsp;&nbsp; peak_file: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A file with the coordinates of the ChIP-seq peaks for the &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; transcription factor of interest. The format is flexible as &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; long as the first 3 columns (chromosome, start coordinate, end &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; coordinate) are present. &nbsp;&nbsp;&nbsp; no_of_components: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Total number of footprint and background components that &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; should be learned from the data. Options are 2 (1 fp and 1 bg) &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; and 3 (2 fp and 1 bg) components. &nbsp;&nbsp;&nbsp; background: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The mode of initialization for the background &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; component. Options are "Flat" or "Seq". Choosing "Flat" &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; initializes this component as a uniform distribution. Choosing &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; "Seq" initializes it as the signal profile that would be &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; expected solely due to the protocol bias (given by the &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; bias_file parameter). &nbsp;&nbsp;&nbsp;&nbsp; fixed_bg: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Whether the background component should be kept fixed. Options &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; are TRUE/T or FALSE/F. Setting "TRUE" keeps this component &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; fixed, whereas setting "FALSE" lets it be reestimated during &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; training. In general, if the background is estimated from bias &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; (option "Seq"), it is recommended to keep it fixed. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp; For instance to run the pipeline on the test data with 2 components, estimating the background from the protocol bias and keeping it fixed, execute the following command: bash find_footprints.sh test_data/ATAC_HEK293_hg19_chr1.bam test_data/hg19.chr1.chrom.size test_data/CTCF_motifs_hg19_chr1.bed test_data/hg19_chr1.fa CTCF SeqBias_ATAC.txt test_data/CTCF_HEK293_chip_hg19_chr1.bed 2 Seq T Important note: Please make sure that the "bam_file" and the "motif_coords" inputs are the only files in their respective directories. This is recommended to avoid overwriting any existing files, as the pipeline generates temporary files in these directories. The pipeline outputs 4 files (in the same directory as the input "bam_file"): &nbsp;&nbsp;&nbsp; File with the ".RESULTS" extension: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Gives the results of the footprinting analysis. The first 6 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; columns harbor the motif information (identical to the motif_coords &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; input file). The 7th column has the footprint score (log-odds of &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; footprint versus background) for each motif instance. The following &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; columns show the probabilities for the individual footprint and &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; background components. &nbsp;&nbsp;&nbsp; File with the ".PARAM" extension: &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Gives the trained parameters for the footprint and background &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; components. It includes as many lines as components (eg the first &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; line has the parameters for the first component). &nbsp; &nbsp; &nbsp;&nbsp;&nbsp; Png file named "plot1": &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A plot with two panels, showing the initial components above &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; and the final trained components below. The plotted values for the &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; final components are given in the .PARAM output file explained above. &nbsp;&nbsp;&nbsp; Png file named "plot2": &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A plot only with the final trained components. In a model where 2 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; components are used, this plot is identical to the bottom panel in &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; plot1.
