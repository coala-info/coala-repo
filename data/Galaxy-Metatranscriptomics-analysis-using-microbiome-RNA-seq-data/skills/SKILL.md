# metatranscriptomics-analysis-using-microbiome-rna-seq-data

## Overview

This workflow provides a comprehensive pipeline for metatranscriptomics analysis of microbiome RNA-seq data, following best practices established by the [Galaxy Training Network (GTN)](https://training.galaxyproject.org/). It begins with rigorous quality control using [FastQC](https://www.bioinformatics.babraham.ac.uk/projects/fastqc/) and [MultiQC](https://multiqc.info/), followed by adapter trimming with [Cutadapt](https://cutadapt.readthedocs.io/). To isolate messenger RNA for functional analysis, the pipeline utilizes [SortMeRNA](https://github.com/biocore/sortmerna) to filter out ribosomal RNA sequences before interlacing paired-end reads.

Taxonomic characterization is performed using [MetaPhlAn](https://huttenhower.sph.harvard.edu/metaphlan/), which identifies the composition of the microbial community. These results are visualized through interactive [Krona](https://github.com/marbl/Krona/wiki) pie charts and high-quality phylogenetic trees generated by [GraPhlAn](https://huttenhower.sph.harvard.edu/graphlan). This allows researchers to identify "who is there" within the complex microbiome sample.

Functional profiling is conducted via [HUMAnN](https://huttenhower.sph.harvard.edu/humann/), which quantifies gene families and reconstructs metabolic pathways. The workflow includes extensive post-processing steps to renormalize abundances, regroup gene families into Gene Ontology (GO) terms, and split stratified tables. By combining MetaPhlAn and HUMAnN outputs, the pipeline enables a "who is doing what" analysis, linking specific taxa to their functional contributions.

This technical skill is released under the MIT license and is designed for researchers performing microbiome studies requiring integrated taxonomic and functional insights from transcriptomic data.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Reverse raw reads | data_input | Metatranscriptomics reverse raw reads |
| 1 | Forward raw reads | data_input | Metatranscriptomics forward raw reads |


For this metatranscriptomics workflow, ensure your forward and reverse raw reads are in fastqsanger or fastqsanger.gz format to maintain compatibility with FastQC and Cutadapt. While the workflow accepts individual datasets, organizing large cohorts into paired-end dataset collections is recommended for efficient batch processing through SortMeRNA and HUMAnN. Consult the README.md for specific database requirements and parameter tuning related to taxonomic and functional profiling. You can use `planemo workflow_job_init` to generate a `job.yml` file for automated execution and reproducible input mapping.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run main-workflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run main-workflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run main-workflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init main-workflow.ga -o job.yml`
- Lint: `planemo workflow_lint main-workflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
