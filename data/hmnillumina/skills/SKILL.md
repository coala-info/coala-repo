# hmnillumina

## Overview

hmnillumina is a specialized C++ tool designed to process the binary InterOp files generated by Illumina sequencers. It consolidates metadata from XML files and binary metrics into a single JSON output, making it easier to programmatically assess run quality—including error rates, indexing results, and Q-scores—without needing the full Illumina InterOp library suite.

## Installation

The tool is primarily distributed via Bioconda:

```bash
conda install -c bioconda hmnillumina
```

## Command Line Usage

The tool requires an input directory (the Illumina run folder) and an output path for the generated JSON file.

### Basic Syntax
```bash
HmnIllumina --input <RUN_FOLDER> --output <OUTPUT_FILE.json>
```

### Arguments
- `-i, --input`: Path to the Illumina run directory (e.g., `20200101_M0000_...`).
- `-o, --output`: Path to the destination JSON file.

## Input Requirements

For the tool to function correctly, the input directory must follow the standard Illumina output structure and contain the following files:

1.  **RunInfo.xml**: Contains basic run metadata.
2.  **RunParameters.xml**: Contains instrument-specific settings.
3.  **InterOp/ Directory**: Must contain at least these binary files:
    - `ErrorMetricsOut.bin`
    - `IndexMetricsOut.bin`
    - `QMetricsOut.bin`
    - `TileMetricsOut.bin`

## Best Practices and Tips

- **Directory Naming**: While the tool parses the contents, it is best practice to point it at the root of the run folder (the directory named with the date and instrument ID).
- **Handling Missing Data**: In the resulting JSON output, undefined or missing values are represented as `""` (empty string) or `"NA"`. Ensure your downstream parsing logic accounts for these strings rather than null values.
- **QC Automation**: Use this tool as a lightweight alternative to the full Illumina Savvy or InterOp C++ library when you only need a summary JSON for automated reporting or database ingestion.
- **Validation**: If the tool fails, verify that the binary files in the `InterOp` folder are not corrupted and that the XML files are well-formed.
