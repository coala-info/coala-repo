# unifire

## Overview

UniFIRE (UniProt Functional annotation Inference Rule Engine) is the official toolset used by UniProt to automate protein functional annotation. It integrates two primary systems: UniRule (manually curated rules) and ARBA (Association Rule Based Annotator, which uses machine learning-derived rules). This skill enables the execution of these complex annotation workflows, transforming raw sequence data or InterProScan outputs into structured functional predictions. It is particularly valuable for annotating large-scale genomic or proteomic datasets where consistency with UniProtKB standards is required.

## Installation and Environment

The most reliable way to run UniFIRE is via the provided Docker image, which bundles all heavy dependencies like InterProScan and HMMER.

- **Conda Installation**: `conda install bioconda::unifire`
- **Hardware Requirements**: Minimum 4 GB RAM (8 GB+ recommended) and ~100 GB of disk space for the Docker image and intermediate files.
- **Docker Image**: The image is approximately 25 GB. Ensure your environment allows large downloads and has sufficient storage before execution.

## Command Line Usage Patterns

The primary interface is the `run_unifire_docker.sh` script located in the `docker/bin/` directory of the repository.

### Basic Annotation from FASTA
To run the full workflow (including InterProScan) on a protein sequence file:
```bash
./docker/bin/run_unifire_docker.sh -i sequences.fasta -o ./output_dir
```

### Annotating from Existing InterProScan Results
If you have already run InterProScan, you can skip the heavy computation step by providing the XML output:
```bash
./docker/bin/run_unifire_docker.sh -i results.xml -t iprscanxml -o ./output_dir
```

### Resource Management
For large datasets, manage temporary files and memory effectively:
```bash
./docker/bin/run_unifire_docker.sh -i input.fasta -o ./output -w ./tmp_work -c
```
- `-w`: Specifies a custom working directory (useful if your `/tmp` partition is small).
- `-c`: Automatically cleans up temporary files after completion.

## Data Preparation Best Practices

### FASTA Header Requirements
UniFIRE is strict about FASTA headers. They must follow UniProtKB conventions for the engine to parse taxonomy and metadata correctly.

**Minimal Header Format**:
`>{id}|{name} OX={TaxonomyID}`

**Example**:
`>P12345|ALBU_HUMAN OX=9606`

**Mandatory Flags**:
- `OX=`: The NCBI Taxonomy ID is required for rule application.

**Optional but Recommended Flags**:
- `OS=`: Organism name.
- `GN=`: Gene name.
- `(Fragment)`: Include in the name string if the sequence is not full-length.

## Output Interpretation

The tool generates three primary output files in TSV format within the specified output folder:
1. `predictions_unirule.out`: Functional predictions derived from manually curated UniRules.
2. `predictions_unirule-pirsr.out`: Predictions specifically from PIRSR rules.
3. `predictions_arba.out`: Predictions generated by the Association Rule Based Annotator.

## Expert Tips

- **First Run Latency**: The initial execution will download the ~25 GB Docker image. Plan for several hours of setup time depending on network speed.
- **Container Runtimes**: While Docker is the default, the script supports other runtimes via the `-s` flag (e.g., `-s singularity` or `-s podman`) for HPC environments where rootless execution is required.
- **Input Validation**: If UniFIRE returns no predictions, verify that your FASTA headers contain the `OX=` flag and that the IDs are unique. The engine will ignore sequences with malformed headers.
