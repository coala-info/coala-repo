# genetic-variation-analysis-reporting

## Overview

This Galaxy workflow provides comprehensive reporting for genetic variation data, specifically designed to process VCF datasets generated by [SARS-CoV-2 variant calling workflows](https://github.com/galaxyproject/iwc/tree/main/workflows/sars-cov-2-variant-calling). It transforms raw variation data into structured, human-readable formats suitable for downstream epidemiological analysis and visualization.

The pipeline accepts a collection of VCF files and applies user-defined filters for Allele Frequency (AF), Depth (DP), and Alternate Depth (DP_ALT). Using tools like SnpSift and Datamash, it extracts key fields, recalculates frequencies, and collapses variant effects to identify high-impact mutations. The workflow is tagged for **mpvx** and **generic** use cases, reflecting its flexibility across different viral datasets.

The final outputs include detailed tabular reports organized both by sample and by variant, as well as a processed dataset optimized for generating overview plots of variant distribution and allele frequencies. These reports provide a clear summary of the mutational landscape across a given dataset, facilitating the identification of significant genetic changes.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Variation data to report | data_collection_input | Variation data in VCF format. Can be the output of any of the workflows in https://github.com/galaxyproject/iwc/tree/main/workflows/sars-cov-2-variant-calling |
| 1 | AF Filter | parameter_input | Allele Frequency Filter. This is the minimum allele frequency required for variants to be included in the reports. |
| 2 | DP Filter | parameter_input | Depth Filter. This is the minimum depth of all alignments at a variant site. |
| 3 | DP_ALT Filter | parameter_input | Depth Filter for variant allele. This is the minimum depth of alignments supporting a variant. |


Ensure your input variation data is provided as a VCF dataset collection, typically generated from upstream variant calling pipelines. You should adjust the numerical filters for allele frequency (AF), depth (DP), and alternate depth (DP_ALT) to match your specific study's stringency requirements. For large-scale batch processing, you can use `planemo workflow_job_init` to generate a `job.yml` template for these parameters. Refer to the `README.md` for comprehensive details on how specific SnpSift filters and field extractions are applied to the tabular outputs. Always verify that your VCF tags match the expected format to ensure accurate recalculation of allele frequencies and variant effects.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Genetic_variation_analysis_reporting.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Genetic_variation_analysis_reporting.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Genetic_variation_analysis_reporting.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Genetic_variation_analysis_reporting.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Genetic_variation_analysis_reporting.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
