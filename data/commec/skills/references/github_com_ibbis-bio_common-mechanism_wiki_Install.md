# Install

This guide will walk you through the process of installing `commec` and its dependencies.

## Quick Start

For most users, this is the fastest way to get started:

```
# Set up conda channels
conda config --add channels bioconda
conda config --add channels conda-forge
conda config --set channel_priority strict

# Create and activate new environment
conda create -n commec-env commec
conda activate commec-env

# Download core databases using commec setup (~2GB)
commec setup
```

You should then create a config YAML file pointing to your databases. We recommend running with 4 threads:

```
cat <<EOF > config.yaml
base_paths:
  default: /PATH/TO/YOUR/DATABASES/DIR/
threads: 4
EOF
```

Within your conda environment, you can now run screen with `--skip-tx` on an input FASTA of your choice with:

```
commec screen --skip-tx input.fasta -y config.yaml
```

## System Requirements

* **Operating system**: `commec` has only been extensively tested on Linux systems; it can also be run through a Docker biocontainer
* **Software**: Python >=3.10 and Conda (we recommend either [Miniconda](https://docs.conda.io/en/latest/miniconda.html) or [Miniforge](https://github.com/conda-forge/miniforge))
* **Disk space**: depends on use case. Note that additional space will be required to store the output of runs.
  + using `–-skip-tx`: 2 GB custom databases
  + full search: ~275 - 650 GB, depending on protein/nt databases used
* **RAM**: while most jobs run with 20GB RAM or less, unless running in `--skip-tx` mode it is best to allow 100GB for RAM-intensive queries.
* **CPU**: at least 4 threads recommended to improve protein homology search speed.

## Installing the `commec` package

The `commec` package has both Python and command-line dependencies, as the pipeline makes use of bioinformatics tools such as `BLAST` and `hmmer`.

As such, we recommend creating a `conda` environment and installing the package into it. You will need to install conda, then make sure [your channels, including bioconda and conda-forge, are configured correctly](http://bioconda.github.io/).

### Install from Bioconda (recommended)

We recommend installing `commec` as a [bioconda package](https://anaconda.org/bioconda/commec/). First, follow the [bioconda setup instructions](https://bioconda.github.io/). Then run:

```
# Configure channels
conda config --add channels bioconda
conda config --add channels conda-forge
conda config --set channel_priority strict

# Create and activate new environment
conda create -n commec-env commec
conda activate commec-env

# Verify installation
commec --help
```

To update the bioconda install, run:

```
conda activate commec-env
conda update commec
```

### Install using Docker Biocontainer

If using a Windows system, you will need to run commec through a Docker [biocontainer](https://biocontainers-edu.readthedocs.io/en/latest/what_is_biocontainers.html). First, you will need to ensure docker is set up on your system.

Then, in the command prompt (via terminal or Docker Desktop) run:

```
docker pull quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1
docker run quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1 commec -help
```

To run commec with Docker, you will need to set up your databases and then mount them. So, for example, if you’ve downloaded the databases for `–-skip-tx` and placed them and the FASTA you wanted to screen in the folder “Documents\Commec”, you will run:

```
docker run -it -v "Documents\Commec:/mnt/files" quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1
```

You can have databases (and FASTA files) in multiple locations, you’ll just need to mount the locations separately:

```
docker run -it -v "your\path\to\folder:/mnt/files" -v "your\other\path\to\folder:/mnt/files2" quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1
```

Within the container (if using Docker Desktop), you should then be able to write a command like:

```
commec screen -d /mnt/files/databases/ /mnt/files/sequence.fasta --skip-tx
```

You can also concatenate these commands if running from a command line:

```
docker run -it -v "Documents\Commec:/mnt/files" quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1 commec screen -d /mnt/files/databases/ /mnt/files/sequence.fasta --skip-tx
```

You can also write a dockerfile to directly pull the databases needed from (thanks to [mcrone for the suggestion](https://github.com/ibbis-bio/common-mechanism/issues/28)):

```
FROM quay.io/biocontainers/commec:0.3.2--pyhdfd78af_1

RUN wget -O commec-dbs.zip https://f005.backblazeb2.com/file/common-mechanism-dbs/common-mechanism-dbs.zip \
&& unzip 'commec-dbs.zip' -d /commec-dbs && rm \
commec-dbs.zip || true
```

### Install from source

You can install from source, which may be useful for development. The `main` branch should contain release-ready code, while feature development should begin from the `develop` branch.

```
# Clone repository
git clone https://github.com/ibbis-screening/common-mechanism.git
cd common-mechanism

# Create conda environment from environment.yaml
conda env create -f environment.yaml
conda activate commec-dev
```

To update the locally-installed package, use the following commands:

```
conda activate commec-dev
conda env update -f environment.yaml --prune [-n commec-dev]
```

You can give the environment a custom name (other than commec-dev) by adding --name custom-name to both the env create and env update commands.

### Troubleshooting

If you see one of the following errors:

```
 BLAST Database error: Error: Not a valid version 4 database.
 Error: Database was built with a different version of Diamond and is incompatible.
```

It is likely because `defaults` conda channel contains outdated versions of dependencies for both Diamond and BLAST, and so if you don't set a strict channel priority that includes `conda-forge`​ at the top, you end up stuck with really old versions of Diamond and BLAST.

This can be solved by running the following commands in order (which is part of [Bioconda setup](https://bioconda.github.io/#usage)):

```
conda config --add channels defaults
conda config --add channels bioconda
conda config --add channels conda-forge
conda config --set channel_priority strict
```

For further assistance, please open an issue on our [GitHub repository](https://github.com/ibbis-screening/common-mechanism/issues).

## Installing the `commec` databases

Before running the Common Mechanism, you must install the databases referenced for sequence screening. `commec` requires different databases depending on your usage.

### Install with `commec setup` (recommended)

You can run `commec setup` to interactively download the databases you need.

### Required: Core databases (~2 Gb)

These custom databases are hosted in Github Large File Storage, and can be downloaded from the [commec-databases](https://github.com/ibbis-bio/commec-databases) repository

Pick the latest release, and extract the commec-dbs.zip into your chosen database directory.

### Taxonomy databases (Required for Full Analysis)

Unless you are only planning to screen in `--skip-tx` mode, you will need databases to perform the taxonomy search. These include:

#### NCBI nr (~375 Gb)

BLAST database files are available through the [NCBI BLAST FTP](https://ftp.ncbi.nlm.nih.gov/blast/db/) site, but since your `commec` conda environment should contain the `blast+` command-line tools, we recommend installing using the following commands:

```
mkdir [databases_path]/blast_nr
cd [databases_path]/blast_nr
update_blastdb.pl --passive --decompress nr
```
