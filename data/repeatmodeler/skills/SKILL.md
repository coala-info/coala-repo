# repeatmodeler

## Overview

RepeatModeler is a de novo transposable element (TE) family identification and modeling package. It integrates several repeat-finding programs—including RECON, RepeatScout, and LTRHarvest/LTR_retriever—to identify repeat boundaries and family relationships directly from genomic sequence data. 

Use this skill to:
1. Prepare genomic FASTA files for repeat analysis.
2. Run the automated discovery pipeline to generate consensus sequences for TE families.
3. Perform structural LTR (Long Terminal Repeat) discovery.
4. Produce a classified repeat library suitable for genome masking and submission to databases like Dfam.

## Core Workflow

### 1. Prepare the Database
Before running the discovery pipeline, you must format your genomic sequences into a RepeatModeler-compatible database using `BuildDatabase`.

```bash
# Create a database from a single FASTA file
BuildDatabase -name "my_genome" genome.fasta

# Create a database from a directory of FASTA files
BuildDatabase -name "my_genome" -dir ./path/to/fasta_files

# Create a database using a batch file (list of file paths)
BuildDatabase -name "my_genome" -batch manifest.txt
```

**Note**: Sequence identifiers are not preserved in the database; they are assigned new GIs. Refer to the `.translation` file generated by the tool to map these back to original headers.

### 2. Run RepeatModeler
The main pipeline executes the discovery algorithms, clusters redundant results, and refines the models.

```bash
# Basic execution
RepeatModeler -database my_genome

# Execution with parallel processing (recommended)
RepeatModeler -database my_genome -pa 16

# Execution including the LTR structural discovery pipeline
RepeatModeler -database my_genome -pa 16 -LTRStruct
```

### 3. Structural LTR Discovery (Optional)
If you need to run the LTR pipeline independently or specifically polish LTR results, use `LTRPipeline`.

```bash
LTRPipeline -threads 8 input_sequence.fa
```

## Expert Tips and Best Practices

- **Parallelization**: Use the `-pa` (parallel) flag to specify the number of shared memory processors. This significantly reduces the time required for the computationally intensive clustering and refinement steps.
- **LTR Discovery**: Always include the `-LTRStruct` flag when working with eukaryotic genomes where LTR retrotransposons are prevalent. This enables an additional pipeline (LTRHarvest and LTR_retriever) that improves the sensitivity and boundary detection of LTR elements.
- **Engine Selection**: Modern versions of RepeatModeler are optimized for `rmblast`. While older versions supported `wublast` or `abblast`, these are largely deprecated.
- **Output Files**:
    - `*-families.fa`: The final classified consensus sequences.
    - `*-families.stk`: Seed alignments for the families in Dfam Stockhom format.
    - `RM_*/`: The working directory containing intermediate results for each round of discovery.
