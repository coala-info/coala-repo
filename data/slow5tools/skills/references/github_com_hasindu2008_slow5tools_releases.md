## What's Changed

* `slow5tools skim` supports the new auxiliary field `open_pore_level` introduced in latest ONT pod5
* `slow5tools degrade` has new profiles added (by [@sashajenner](https://github.com/sashajenner) and [@hiruna72](https://github.com/hiruna72)) and are documented at <https://hasindu2008.github.io/slow5tools/bits-lossy.html>
* Documentation improvements, including a new guide for archiving lossy data (by [@sashajenner](https://github.com/sashajenner)) in <https://hasindu2008.github.io/slow5tools/archive-lossy.html>
* Option `-m, --demux-missing` added to slow5tools split by [@sashajenner](https://github.com/sashajenner)
* Usability improvements and minor bug fixes (mainly for `slow5tools degrade` and `slow5tools split` when demultiplexing)

[slow5tools-v1.3.0](/hasindu2008/slow5tools/releases/tag/v1.3.0)

## What's Changed

* new `ex-zd` signal compression method. See the [bioxiv preprint](https://www.biorxiv.org/content/10.1101/2024.10.02.616377v1)
* new subtool called `degrade` for lossy compression through bit elimination. Irreversible lossy compression, so be careful! See the [bioxiv preprint](https://www.biorxiv.org/content/10.1101/2024.10.02.616377v1)
* new options to `split` subtool to enable demultiplexing a S/BLOW5 based on barcodes (or custom categories) from a tsv file (`--demux`, `--demux-code`, `--demux-rid` and `--demux-uniq`)
* documentation update (updates to [faq](https://hasindu2008.github.io/slow5tools/faq.html), new datasets added to [datasets](https://hasindu2008.github.io/slow5tools/datasets.html) and updates to [commands](https://hasindu2008.github.io/slow5tools/commands.html)) and [workflows](https://hasindu2008.github.io/slow5tools/workflows.html)
* using $(MAKE) by [@sashajenner](https://github.com/sashajenner) in [#126](https://github.com/hasindu2008/slow5tools/pull/126)
* ./configure now supports --prefix (solves [#124](https://github.com/hasindu2008/slow5tools/issues/124))

Many contributions by [@sashajenner](https://github.com/sashajenner).

**Full Changelog**: [v1.2.0...v1.3.0](https://github.com/hasindu2008/slow5tools/compare/v1.2.0...v1.3.0)

## changes/improvements

* minor s2f bug fix [#96](https://github.com/hasindu2008/slow5tools/issues/96)
* documentation improvements (more bash oneliners, faq, workflows, datasets, software etc)
* new `--cite` option added to slow5tools
