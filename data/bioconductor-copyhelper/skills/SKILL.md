# bioconductor-copyhelper

## Overview

CopyhelpeR is a data-only Bioconductor package that provides essential annotation files for the `CopywriteR` package. It contains pre-calculated 1kb bin data for GC-content and mappability, along with CNV blacklists. These files are required by `CopywriteR` to perform copy number analysis using off-target reads from targeted sequencing data.

## Usage

The primary function of this package is to locate the directory containing the helper files for a specific reference genome.

### Locate Helper Files

Use `getPathHelperFiles()` to retrieve the full system path to the folder containing the annotation data.

```r
library(CopyhelpeR)

# Get path for human genome hg19
helper_path <- getPathHelperFiles("hg19")
print(helper_path)
```

### Supported Reference Genomes

The package supports the following genome builds:
- **Human:** `hg18`, `hg19`, `hg38`
- **Mouse:** `mm9`, `mm10`

### Data Content

The directory returned by `getPathHelperFiles()` contains files stored as `GRanges` objects (from the `GenomicRanges` package). These include:
- **GC-content:** 1kb bin GC-content data obtained via `bedtools nuc`.
- **Mappability:** Average mappability of 51bp fragments within each 1kb region, aligned using BWA.
- **Blacklist:** Regions of known copy number variation or problematic alignment, derived from ENCODE (DacMapabilityConsensusExcludable and DukeMapabilityRegionsExcludable).

## Integration with CopywriteR

This package is typically used as a prerequisite for the `CopywriteR()` function. You do not usually need to load the `GRanges` objects manually; instead, pass the path generated by `getPathHelperFiles()` to the `CopywriteR` analysis pipeline.

```r
# Example of how the path is passed to CopywriteR
# CopywriteR(sample.control.df, destination.folder, helper.files.folder = helper_path)
```
