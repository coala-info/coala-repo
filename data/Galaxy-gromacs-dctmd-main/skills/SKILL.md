# dctmd-calculations-with-gromacs

## Overview

This Galaxy workflow performs Dissipative Corrected Targeted Molecular Dynamics (dcTMD) simulations to calculate free energy and friction profiles for protein-ligand dissociation pathways. It automates the complex setup required for nonequilibrium simulations, utilizing [GROMACS](https://www.gromacs.org/) to execute an ensemble of pulling simulations based on user-defined parameters such as pulling rate, temperature, and force field selection.

The process begins by parameterizing the input protein (PDB) and ligand (SDF) files through a dedicated subworkflow. After system solvation, ion addition, and energy minimization, the workflow executes multiple parallel TMD simulations using the GROMACS PULL code. These simulations are configured via a provided `tmd.mdp` file to drive the ligand along a specified dissociation path.

In the final stages, the workflow applies a friction correction to the resulting trajectories using the method described by [Wolf and Stock (2018)](https://doi.org/10.1021/acs.jctc.8b00835). The primary outputs include XVG and XTC trajectory files, along with calculated friction data and free energy profiles. Users are responsible for verifying the ensemble's pathway separation to ensure the physical relevance of the dissociation results.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Ligand SDF | data_input | Ligand SDF |
| 1 | pH to protonate ligand | parameter_input | Set to -1.0 to skip protonation |
| 2 | Protein PDB | data_input | Protein PDB |
| 3 | Salt concentration | parameter_input | NaCl concentration (mol/dm^3) |
| 4 | Water model | parameter_input | Water model |
| 5 | Force field | parameter_input | Force field |
| 6 | Number of simulations | parameter_input | Number of simulations in the TMD ensemble |
| 7 | Temperature | parameter_input | Temperature |
| 8 | Number of equilibration steps | parameter_input | Number of equilibration steps |
| 10 | Pulling rate | parameter_input | Rate to pull groups apart (nm/ps) |
| 11 | Step length (ps) | parameter_input | MD step length |
| 12 | Protein pull group | parameter_input | List of atom indices (separated with spaces) which make up the protein pull group |
| 13 | Number of steps | parameter_input | Number of steps for the production simulation |
| 14 | Pull group pbcatom | parameter_input | pull-group1-pbcatom - the reference atom index for the treatment of periodic boundary conditions inside the group |


Ensure your input protein is in PDB format and the ligand in SDF format to allow for proper parameterization via the integrated subworkflow. You should verify that the pull group names and simulation parameters like salt concentration and water model match your specific molecular system requirements. For large-scale ensembles, consider organizing your simulation parameters into a `job.yml` file for use with `planemo workflow_job_init` to automate execution. Consult the `README.md` for comprehensive details on the required `tmd.mdp` configuration and guidance on interpreting the resulting friction and free energy profiles. Remember that you are responsible for manually validating the dissociation pathways generated by the ensemble.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run gromacs-dctmd.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run gromacs-dctmd.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run gromacs-dctmd.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init gromacs-dctmd.ga -o job.yml`
- Lint: `planemo workflow_lint gromacs-dctmd.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
