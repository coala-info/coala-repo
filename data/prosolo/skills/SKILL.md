# prosolo

## Overview

ProSolo is a specialized genomic variant caller designed to handle the unique noise profile of Multiple Displacement Amplification (MDA) in single-cell sequencing. Unlike standard callers, it uses a joint model that compares a single-cell sample against a bulk population sample. This allows the tool to distinguish between true biological variants and artifacts like allele dropouts or amplification errors. It is currently optimized for SNV calling; indel calling is not yet fully supported in the primary workflow.

## Usage Guidelines

### Core Workflow: single-cell-bulk

The primary command for variant calling is `single-cell-bulk`. This requires four specific inputs: single-cell BAM, bulk BAM, reference FASTA, and a candidate VCF/BCF.

**Standard CLI Pattern:**
```bash
prosolo single-cell-bulk \
    --omit-indels \
    --candidates candidates.bcf \
    --output output.bcf \
    single-cell.bam \
    bulk.bam \
    ref.fa
```

**Best Practices:**
- **Indel Handling**: Always use the `--omit-indels` flag for production runs, as indel calling is still under development.
- **Candidate Generation**: ProSolo does not discover variants from scratch. You must provide a candidate file (VCF/BCF) generated by tools like `freebayes` or `samtools mpileup`.
- **Parallelization**: To speed up processing, split the `--candidates` file by genomic regions (e.g., by chromosome) and run multiple ProSolo instances in parallel.

### False Discovery Rate (FDR) Control

After calling, use the `control-fdr` subcommand to filter results based on specific biological events. ProSolo outputs probabilities for various states (e.g., HET, HOM_ALT, ADO_TO_REF).

**FDR Control Pattern:**
```bash
prosolo control-fdr \
    input.bcf \
    --events ADO_TO_REF,ADO_TO_ALT \
    --var SNV \
    --fdr 0.05 \
    --output filtered_ado.bcf
```

**Expert Tips for Event Selection:**
- **Allele Dropout**: To filter for sites specifically affected by dropout, use `--events ADO_TO_REF,ADO_TO_ALT`.
- **Variant Presence**: To control the FDR for the presence of any alternative allele, combine all relevant events: `--events HET,HOM_ALT,ADO_TO_ALT,ADO_TO_REF,ERR_REF`.
- **Event Naming**: When specifying events, omit the `PROB_` prefix found in the BCF INFO field.

### Troubleshooting and Help
- Access global help: `prosolo --help`
- Access subcommand specific options: `prosolo single-cell-bulk --help` or `prosolo control-fdr --help`



## Subcommands

| Command | Description |
|---------|-------------|
| control-fdr | Filter calls for controlling the false discovery rate (FDR) at given level. |
| prosolo | Prosolo variants tool |
| prosolo | ProSolo is a tool for processing and analyzing single-cell RNA sequencing data. |
| prosolo estimate-mutation-rate | Estimate the effective mutation rate of a tumor sample from a VCF/BCF with candidate variants from STDIN. |
| prosolo single-cell-bulk | Call somatic and germline variants from a single cell-bulk sample pair and a VCF/BCF with candidate variants. |
