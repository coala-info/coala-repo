# bioconductor-tpp2d

## Overview

The `TPP2D` package implements a functional analysis approach to detect ligand-protein interactions from 2D-TPP experiments. Unlike conventional TPP which uses a single dose across a temperature gradient, 2D-TPP uses multiple ligand concentrations at each temperature. This skill guides the workflow from data import and normalization to null model bootstrapping and FDR-controlled hit identification.

## Typical Workflow

### 1. Data Import and Preprocessing
The workflow begins with a configuration table and raw data (or a list of data frames).

```r
library(TPP2D)
library(dplyr)

# Import dataset
import_df <- import2dDataset(
    configTable = config_tab,
    data = raw_dat_list, # Optional if paths are in configTable
    idVar = "protein_id",
    intensityStr = "signal_sum_",
    fcStr = "rel_fc_",
    nonZeroCols = "qusm",
    geneNameVar = "gene_name",
    medianNormalizeFC = TRUE,
    filterContaminants = TRUE
)

# Recompute signal intensities from ratios and resolve naming
recomp_sig_df <- recomputeSignalFromRatios(import_df)
preproc_df <- resolveAmbiguousProteinNames(recomp_sig_df)
```

### 2. Model Fitting and F-Statistics
`TPP2D` compares two nested models for each protein:
- **Null Model (H0):** Protein profile is unaffected by treatment.
- **Alternative Model (H1):** Constrained sigmoidal dose-response model across temperatures.

```r
# Fit models and compute F-statistics
model_params_df <- getModelParamsDf(df = preproc_df)
fstat_df <- computeFStatFromParams(model_params_df)
```

### 3. FDR Estimation via Bootstrapping
To determine significance, a null distribution of F-statistics is generated by bootstrapping.

```r
# Recommended: B >= 20 for reliable results
set.seed(123)
null_model <- bootstrapNullAlternativeModel(
    df = preproc_df,
    params_df = model_params_df,
    B = 20
)

# Calculate FDR and identify hits
fdr_tab <- getFDR(df_out = fstat_df, df_null = null_model)
hits <- findHits(fdr_df = fdr_tab, alpha = 0.1)
```

### 4. Visualization
The package provides several functions to visualize the results of the 2D-TPP analysis.

```r
# Volcano plot of results
plot2dTppVolcano(fdr_df = fdr_tab, hits_df = hits)

# Plot specific protein fits (H0 vs H1)
plot2dTppFit(recomp_sig_df, "ProteinName", model_type = "H1")

# Heatmap of fold changes
plot2dTppFcHeatmap(recomp_sig_df, "ProteinName", drug_name = "CompoundX")
```

## Key Functions

- `import2dDataset`: Primary entry point for loading TMT reporter ion intensities.
- `recomputeSignalFromRatios`: Converts relative fold changes back to absolute intensity space for modeling.
- `getModelParamsDf`: Fits the dose-response surfaces.
- `bootstrapNullAlternativeModel`: Generates the null distribution required for FDR calculation.
- `findHits`: Filters the results based on a specified FDR threshold ($\alpha$).

## Tips for Success
- **Replicates:** Ensure the `experiment` column in your config table correctly identifies different MS runs.
- **Normalization:** `medianNormalizeFC = TRUE` is generally recommended to account for systematic bias across TMT channels.
- **Bootstrapping:** While `B = 2` is fast for testing, use at least `B = 20` (or higher) for publication-quality FDR estimation.
- **Protein IDs:** Use `resolveAmbiguousProteinNames` to handle cases where multiple protein identifiers map to the same gene or cluster.
