# Hi-C_fastqToCool_hicup_cooler

## Overview

This workflow processes paired-end FASTQ reads from Hi-C or Hi-ChIP experiments to generate normalized contact matrices. It utilizes the [HiCUP](https://www.bioinformatics.babraham.ac.uk/projects/hicup/) pipeline to truncate reads at religation sites, map mates independently using Bowtie2, and filter for valid, uniquely mapped pairs. The workflow specifically uses the middle of the restriction fragments as coordinates to ensure high-resolution accuracy during the pairing and deduplication stages.

Following initial processing, valid pairs are converted into a sorted, Tabix-indexed format using `cooler csort`. These pairs are then binned into a matrix at a user-defined resolution and balanced using the Iterative Correction and Eigenvector decomposition (ICE) method via [cooler](https://cooler.readthedocs.io/). While designed for Hi-C, the workflow is compatible with Hi-ChIP data by utilizing the raw matrix outputs instead of the balanced values.

The final outputs include a filtered and sorted valid pairs dataset and a visualization of the contact matrix for a specified genomic region generated by [pyGenomeTracks](https://github.com/deeptools/pyGenomeTracks). For detailed instructions on input parameters—such as restriction enzyme motifs, genome indices, and bin sizes—refer to the [README.md](README.md) in the Files section.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | PE fastq input | data_collection_input | Should be a paired collection with Hi-C fastqs |
| 1 | genome name | parameter_input | only use genome ids which have bowtie2 indexes |
| 2 | Restriction enzyme | parameter_input | Restriction enzyme used e.g. A^GATCT,BglII. Some Hi-C protocols may use several enzymes. To specify several enzymes, use the ':' to separate them e.g. A^GATCT,BglII:A^AGCTT,HindIII:^GATC,DpnII. HiCUP accomodates N in restriction enzyme: e.g. :A^ANCTT |
| 3 | No fill-in | parameter_input | Hi-C protocol did NOT include a fill-in of sticky ends prior to re-ligation and therefore reads shall be truncated at the restriction site sequence |
| 4 | minimum MAPQ | parameter_input | can be set to 0 for no filtering |
| 5 | Bin size in bp | parameter_input | For example 10000 for 10kb |
| 6 | Interactions to consider to calculate weights in normalization step | parameter_input | Nothing means genome-wide, '--cis-only' means only Cis interactions, '--trans-only' means only Trans interactions |
| 7 | region for matrix plotting | parameter_input | You can use your region of interest or a region known for high level of insulation for example HoxD locus: hg19: chr2:175,689,620-178,604,461 hg38: chr2:174,692,032-177,585,317 mm10: chr2:73,779,626-75,669,724 mm39: |


Ensure your input is organized as a paired-end fastqsanger collection to match the workflow's requirement for automated processing of multiple samples. Key parameters include the specific restriction enzyme cut site (e.g., A^GATCT) and the genome name corresponding to available bowtie2 indices. For downstream analysis at different resolutions, the "valid pairs filtered and sorted" output (juicer_medium_tabix.gz) can be reused in the `hic_tabix_to_cool_cooler` subworkflow. Refer to the README.md for detailed guidance on biotin fill-in settings and specific coordinate filtering for capture Hi-C. You can use `planemo workflow_job_init` to generate a `job.yml` for reproducible execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run hic-fastq-to-cool-hicup-cooler.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run hic-fastq-to-cool-hicup-cooler.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run hic-fastq-to-cool-hicup-cooler.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init hic-fastq-to-cool-hicup-cooler.ga -o job.yml`
- Lint: `planemo workflow_lint hic-fastq-to-cool-hicup-cooler.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
