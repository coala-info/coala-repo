## Checklist

### preparation

* [ ]  freeze the master-branch (forbid everyone from making changes)
* [ ]  make sure that SEQAN\_VERSION macros are correct (`include/seqan/version.h`)
* [ ]  make sure that app versions are greater than the last packages on <http://packages.seqan.de> (set as SEQAN\_APP\_VERSION in each app's CMakeLists.txt)
* [ ]  write the Changelog is up to date (@cpockrandt)

### build packages / tarballs

* [ ]  Prepare library source tarballs
* [ ]  Prepare binary tarballs for each app -- Linux (@marehr)
* [ ]  Prepare binary tarballs for each app -- OS X (@rrahn)
* [ ]  Prepare binary tarballs for each app -- FreeBSD (@h-2)
* [ ]  Prepare binary tarballs for each app -- Win32/Win64 (@cpockrandt)

### tag the release

* [ ]  Tag the release
* [ ]  publish the release on github

### update resource

* [ ]  upload tarballs to packages.seqan.de and update
* [ ]  build dox for docs.seqan.de
* [ ]  build manual for the seqan.readthedocs.org
* [ ]  Tag KNIME builds on Jenkins for permanent versioning (@temehi)

### post-release version changes

* [ ]  bump SEQAN\_VERSION macros so the reflect the next release (`include/seqan/version.h`)
* [ ]  bump app versions so that they reflect the next release / being in development
* [ ]  update seqan and app versions in the resource.txt file on the server in Tuebingen
* [ ]  unfreeze the master-branch

### announcements

* [ ]  write a release announcement for <http://www.seqan.de>
* [ ]  send announcement to seqan-dev at lists.fu-berlin.de
* [ ]  announce via seqan\_lib twitter account

### Update upstream packages

* [ ]  macports
* [ ]  homebrew
* [ ]  debian
* [ ]  fedora
* [ ]  gentoo
* [ ]  ubuntu
* [ ]  openBSD
* [ ]  freeBSD

## Release tarballs

Please see the manual for instructions:
<http://seqan.readthedocs.org/en/master/Infrastructure/Manage/Releases.html>

You have to create the library package once (best do it on Linux).
Then you have to create one package for every app on all supported platforms (linux/64, mac/64, freebsd/64, windows/64 and windows/32). On Linux, make sure that you have cmake-3.1 to get `.tar.xz` instead of `.tar.bz2`.

A simple loop for making all the apps on unix could look like this:

```
# regular build
rm -rf CMake*
for APP in $(cd ../../seqan/apps/ && ls -d */ | sed -e 's#\/##g'); do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP:$APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1; make -j 4; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64; done

# optimized build
rm -rf CMake*
for APP in $(cd ../../seqan/apps/ && ls -d */ | sed -e 's#\/##g'); do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP:$APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1 -DSEQAN_ARCH_SSE4=1 ; make -j 4; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64; done

#32bit build
rm -rf CMake*
for APP in $(cd ../../seqan/apps/ && ls -d */ | sed -e 's#\/##g'); do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP:$APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1 -DCMAKE_CXX_FLAGS=-m32 ; make -j 4; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64; done
```

## Tag release

Locally do:

```
git tag -a seqan-v2.x.y
```

As text add

```
Release 2.x.y of the SeqAn library. See
https://github.com/seqan/seqan/blob/master/CHANGELOG.rst
for a full list of changes.
```

Then eplicity push the tag to upstream:

```
git push upstream seqan-v2.x.y
```

## Publish the release on GitHub

Go to <https://github.com/seqan/seqan/tags> and click "Add release notes" beside the new tag.

As release title enter `seqan-v2.x.y`

As text enter

```
See https://github.com/seqan/seqan/blob/master/CHANGELOG.rst for a summary of changes.
See http://packages.seqan.de for binary downloads of the applications.
```

Select the `library source tarballs` previously built, i.e. the zip/tar files that are less then 10MB big, and add them to the release (unfittingly called "Attach binaries" but that doesn't matter).

Finally click "Publish release".

## Updating <http://packages.seqan.de>

First copy the package files that you created to `/web/packages.seqan.de/htdocs` (on any `imp.fu-berlin.de`). Assuming you have all the zipballs and tarballs in `/tmp/packages`, you can:

```
cd /web/packages.seqan.de/htdocs
for APP in $(ls -1 | sort -r | sed -e 's#\/##g'); do mv /tmp/packages/${APP}* ${APP}/; done
```

Next run the script that generates the static packages site. It can be found in every seqan-git-checkout:

```
git clone https://github.com/seqan/packages.seqan.de.git
python /your/seqan-git-checkout/release_page.py -d /web/packages.seqan.de/htdocs/ -o /web/packages.seqan.de/htdocs/
```

## Updating <http://docs.seqan.de>

```
cd /web/docs.seqan.de/htdocs
```

Edit the index.html by adding a new link with `2.x.y`.

```
mkdir -p /web/docs.seqan.de/htdocs/seqan/2.x.y
```

Copy the dox files from the directory where you built the `SEQAN_LIBRARY_RELEASE`.

```
cp -r /path/to/seqan-2.x.y-package/dox/html/* /web/docs.seqan.de/htdocs/seqan/2.x.y/
```

2. select SeqAn project
3. select Admin
4. select Versions
5. find the new tag and set the "active"-checkbox

## Bumping app version

This code snippet, run from inside the apps folder increases the last digit of each app's version string:

```
for f in $(find ./ -name CMakeLists.txt); do gawk -i inplace -F \" '$0 ~ "^set\\ \\(SEQAN_APP_VERSION" { split($2, num, "."); printf $1 "\"" num[1] "." num[2] "." num[3]+1 "\"" $3 "\n"; next };  { print $0 }' $f; done
```

You can easily move the `+1` to `num[2]` if you instead want to increase the second digit.

## Collecting internal app version in a resource.txt file

Paste code snippet into a executable, run from inside the seqan root folder to collect versions in a new file called resources.txt:

```
#!/bin/sh

if [ "$#" -ne 1 ]; then
    echo "[INPUT ERROR] Please specify the current library release (e.g. 2.3.0) as the 2nd command line argument!"
    return
fi

# pass current library release as command line argument
CURRENT_RELEASE=$1
echo seqan'\t'$CURRENT_RELEASE > resources.txt

# get app version from CMakeList.txt in each app folder
for APP_DIR in $(ls -d apps/*/)
do
  VERSION=$(awk 'match($0, /SEQAN_APP_VERSION "(.*)"/, v) {print v[1]}' $APP_DIR/CMakeLists.txt)

  for APP in $(awk 'match($0, /add_executable\s*\(([^ ]+)/, v) {print v[1]}' $APP_DIR/CMakeLists.txt)
  do
    echo $APP'\t'$VERSION >> resources.txt
  done
done
```
