# Using refasplit to split fasta files

[Jump to bottom](#wiki-pages-box)

Jose Manuel Martí edited this page Jan 16, 2023
·
[1 revision](/khyox/recentrifuge/wiki/Using-refasplit-to-split-fasta-files/_history)

## Overview

The script `refasplit` is useful to split a large FASTA file in several files so that the records are "symmetrically" distributed and hence they have a similar weight regarding the number of nucleotides.

In cases of metagenomic databases contained in FASTA files, the length of the different entries (in nucleotides) may differ in orders of magnitude, and a simple partition may yield files with very different sizes resulting in downstream bottlenecks —for instance, when processed in parallel. The use of `refasplit` avoids these cases by optimizing the distribution of the records to keep similar sizes.

## Command layout

The layout of the Rextract (`rextract`) command (ver. 1.12.0) is:

```
usage: refasplit [-h] [-d] [-n NUMBER] [-i FILE] [-o PATH] [-m NUMBER] [-c] [-V]
```

### General arguments

```
  -h, --help            show this help message and exit
  -d, --debug           increase output verbosity and perform additional checks
  -n NUMBER, --num NUMBER
                        number of output FASTA files to generate
  -i FILE, --input FILE
                        single FASTA file (no paired-ends), which may be gzipped
  -o PATH, --outprefix PATH
                        path with prefix for the series of FASTA output files
  -m NUMBER, --maxreads NUMBER
                        maximum number of FASTA reads to process; default: no maximum
  -c, --compress        the resulting FASTA files will be gzipped
  -V, --version         show program's version number and exit
```

## Example

For example, if you:

* want to symmetrically split a decontaminated nt database `nt_decon.fa` in FASTA format,
* in 256 FASTA files for using that number of threads in a downstream processing,
* with the `nt_decon_` prefix, and
* have cloned the repo in `~/recentrifuge`,
  then you may run:

```
~/recentrifuge/refasplit -i nt_decon.fa -o nt_decon_ -n 256
```

If you use Recentrifuge in your research, please consider citing the paper. Thanks!

Martí JM (2019) **Recentrifuge: Robust comparative analysis and contamination removal for metagenomics**. *PLOS Computational Biology* 15(4): e1006967. <https://doi.org/10.1371/journal.pcbi.1006967>
