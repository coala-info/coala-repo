# bioconductor-spikeinsubset

name: bioconductor-spikeinsubset
description: Access and utilize subsetted data from Affymetrix HGU95 and HGU133 spike-in experiments. Use this skill when a user needs to perform benchmark testing of microarray analysis methods, evaluate cross-hybridization, or practice with gold-standard spike-in datasets in R.

## Overview

The `SpikeInSubset` package provides a curated subset of the Affymetrix Spike-In Experiment data. It includes probe-level intensities (PM/MM) and pre-processed expression values (RMA and MAS5) for both HGU95 and HGU133 platforms. This dataset is primarily used for validating preprocessing algorithms and differential expression methods because the concentrations of specific "spiked-in" genes are known.

## Loading the Data

The package contains several datasets that must be loaded using the `data()` function after loading the library.

```r
library(SpikeInSubset)

# Load HGU95 data
data(spikein95)  # Probe-level data (ProbeSet object)
data(rma95)      # RMA pre-processed data
data(mas95)      # MAS5 pre-processed data

# Load HGU133 data
data(spikein133) # Probe-level data (ProbeSet object)
data(rma133)     # RMA pre-processed data
data(mas133)     # MAS5 pre-processed data
```

## Working with Spike-In Data

### Accessing Intensities and Concentrations
The `spikein95` and `spikein133` objects are of class `ProbeSet`. You can examine the experimental design and known concentrations using `pData`.

```r
# View sample information and spike-in concentrations
pData(spikein95)

# Access Perfect Match (PM) intensities
pm(spikein95)

# Access Mismatch (MM) intensities
mm(spikein95)
```

### Pre-processed Data and Calls
The package includes pre-calculated expression sets to save time during benchmarking.

- **RMA**: `rma95` and `rma133` contain expression values processed via the Robust Multi-array Average algorithm.
- **MAS5**: `mas95` and `mas133` contain expression values processed via the MAS 5.0 algorithm.
- **Detection Calls**: `pacalls95` and `pacalls133` contain the Present/Absent calls generated by MAS5.

## Cross-Hybridization Analysis

The package includes a specific list of probe sets likely to cross-hybridize with the spiked-in probes in the HGU133A experiment.

```r
data(hgu133a.spikein.xhyb)

# This is a list with three components based on BLAST thresholds:
# 100bp, 150bp, and 200bp
names(hgu133a.spikein.xhyb)

# Access probe sets likely to cross-hybridize at the 100bp threshold
hgu133a.spikein.xhyb[["100"]]
```

## Typical Workflow: Benchmarking

1. **Load Data**: Load the probe-level `spikein` data.
2. **Apply Method**: Apply a new normalization or summarization method to the probe-level data.
3. **Compare**: Compare the resulting expression estimates against the known concentrations found in `pData(spikein95)` or `pData(spikein133)`.
4. **Baseline**: Use the provided `rma` or `mas` objects as a performance baseline.
