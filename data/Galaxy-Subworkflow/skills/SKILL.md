# subworkflow

## Overview

This Galaxy workflow demonstrates the modular application of subworkflows to streamline complex data processing pipelines. It is designed to take an **Input dataset collection** and process it through multiple nested layers, showcasing the reusability of established analytical routines within the [Galaxy-interface](https://galaxyproject.org/).

The core logic involves executing two instances of a **Tutorial workflow** as sub-components. This hierarchical structure, often featured in [GTN (Galaxy Training Network)](https://training.galaxyproject.org/) materials, allows users to encapsulate specific multi-step tasks into single nodes, making the overall pipeline easier to manage and scale.

In the final stage, the workflow utilizes the **Paste** tool to aggregate or manipulate the results generated by the preceding subworkflows. The process concludes by generating a single output file (`out_file1`), providing a clear example of how to orchestrate modular bioinformatics workflows using the Galaxy workflow editor.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run subworkflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run subworkflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run subworkflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init subworkflow.ga -o job.yml`
- Lint: `planemo workflow_lint subworkflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
