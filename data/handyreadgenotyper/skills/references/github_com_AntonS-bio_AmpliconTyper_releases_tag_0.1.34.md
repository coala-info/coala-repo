[0.1.34](/AntonS-bio/AmpliconTyper/tree/0.1.34)

[`252ff44`](/AntonS-bio/AmpliconTyper/commit/252ff447c273c80b78403324d0fd32361bd87140)

* Amended the definition of "valid amplicon" for purposes of "Support for target organism column". Previously it had to be chromosomal non-AMR amplicon, now it is simply chromosomal amplicon i.e. AMRs are permitted. The rest of the definition remains: >50 reads, <=2 unknown SNPs.
* Fixed reporting of multiple genotypes in Summary Table. Previously only one genotype was reported even if two from separate phylogeny branches were present. E.g. previously if a sample had genotypes 2.3.2 and 4.3.1.1 only one of these would be reported, now both would be. However, if the genotypes are direct ancestor-descendants (e.g. 4.3.1 and 4.3.1.1.P1) , only lowest one (4.3.1.1.P1) would be reported.
