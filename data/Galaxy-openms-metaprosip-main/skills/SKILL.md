# MetaProSIP OpenMS 2.8

## Overview

MetaProSIP is a Galaxy workflow designed for the automated inference of stable isotope incorporation rates in proteins, facilitating functional metaproteomics studies. By analyzing centroided LC-MS datasets against a provided FASTA database, the pipeline quantifies elemental fluxes within microbial communities. It is specifically optimized for data generated by high-resolution instruments like Orbitrap.

The workflow integrates several [OpenMS](https://www.openms.de/) tools to process mass spectrometry data. It begins by generating a decoy database for FDR estimation and identifying peptides via the MSGFPlusAdapter. Simultaneously, it detects elution profiles of unlabeled peptides using FeatureFinderMultiplex. These identification and quantification results are then mapped together to provide a foundation for isotope analysis.

In the final stages, the MetaProSIP tool calculates protein-SIP features to determine the rate of labeled element incorporation. The workflow outputs comprehensive peptide-centric results and feature fitting data, allowing for functional grouping and protein inference. This automated approach streamlines the complex task of tracking metabolic activity across different taxa in environmental or clinical samples.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Centroided LC-MS datasets | data_collection_input | mgf, ms2, mzml or mzxml datasets |
| 1 | Fasta Database | data_input | A protein sequence database in FASTA format |
| 2 | Precursor monoisotopic mass tolerance (ppm) | parameter_input | passed to  - MSGF+ -precursor_mass_tolerance - MetaProSIP -mz_tolerance_ppm |
| 3 | Fixed modifications | parameter_input | used for the MSGF+ search |
| 4 | Variable modifications | parameter_input | used for the MSGF+ search |
| 5 | Labeled element | parameter_input | Used for MetaProSIP (single letter code) |


Ensure your LC-MS data is in centroided mzML format and organized into a data collection to facilitate batch processing through the FeatureFinder and MetaProSIP steps. The protein database must be provided as a FASTA file, which the workflow automatically augments with decoy sequences for FDR estimation. Verify that your precursor mass tolerance and modification parameters align with your instrument settings and experimental design. For comprehensive configuration details and parameter descriptions, refer to the project's README.md. You can use `planemo workflow_job_init` to generate a `job.yml` for automated execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run metaprosip.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run metaprosip.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run metaprosip.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init metaprosip.ga -o job.yml`
- Lint: `planemo workflow_lint metaprosip.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
