# gangstr

## Overview

GangSTR is a genomic tool designed to estimate the length of tandem repeats by analyzing short-read alignments (BAM files). It utilizes a maximum likelihood model that incorporates multiple types of read information—including spanning reads, flanking reads, and abbreviated fragments—to accurately genotype STRs even when the repeat sequence is longer than the read length. The tool outputs results in VCF format, providing both the estimated allele lengths and quality scores for each locus.

## Installation

Install GangSTR via bioconda for the most stable environment:

```bash
conda install -c bioconda -c conda-forge gangstr
```

## Basic Usage

To perform a standard genome-wide STR profiling, use the following command structure:

```bash
GangSTR --bam input.bam --ref reference.fa --regions target_regions.bed --out output_prefix
```

### Required Inputs
- `--bam`: A comma-separated list of aligned BAM files.
- `--ref`: The reference genome in FASTA format.
- `--regions`: A BED file defining the target TR loci.
- `--out`: The prefix for the output VCF and log files.

## Common CLI Patterns

### Targeted or Disease-Locus Analysis
When analyzing specific disease-associated loci rather than the whole genome, use the `--targeted` flag to optimize the model for high-depth, specific regions.

```bash
GangSTR --bam input.bam --ref ref.fa --regions disease_loci.bed --targeted --out targeted_results
```

### Exome Sequencing (Non-uniform Coverage)
For Whole Exome Sequencing (WES) or other targeted capture data where coverage is not uniform across the genome, you must use the `--nonuniform` flag and manually specify coverage.

```bash
GangSTR --bam input.bam --ref ref.fa --regions regions.bed --nonuniform --coverage 50.0 --out exome_results
```

### Sex-Aware Genotyping
To correctly genotype STRs on sex chromosomes, provide sample sex information.

```bash
GangSTR --bam sample1.bam --ref ref.fa --regions regions.bed --bam-samps sample1 --samp-sex M --out sex_aware_results
```
*Note: Use 'M' for male and 'F' for female.*

### Filtering by Motif Length
If only interested in specific repeat types (e.g., trinucleotide repeats), filter by the period length.

```bash
GangSTR --bam input.bam --ref ref.fa --regions regions.bed --period 3 --out trinucleotide_only
```

## Expert Tips and Best Practices

- **Coverage Specification**: For targeted sequencing, GangSTR cannot accurately estimate global coverage. Always provide the expected average coverage using `--coverage` to ensure the likelihood model functions correctly.
- **Read Length**: While GangSTR can extract read length from the BAM, explicitly setting it with `--readlength` can prevent errors in datasets with variable or poorly flagged read lengths.
- **Memory Management**: For large-scale runs, use the `--chrom` option to parallelize the workload by chromosome across different compute nodes.
- **Stutter Calibration**: If you have known stutter characteristics for your sequencing protocol, adjust `--stutterup`, `--stutterdown`, and `--stutterprob`. The defaults (0.05, 0.05, 0.90) are generally robust for standard Illumina PCR-plus preps.
- **Quality Control**: Always look at the `Q` score in the resulting VCF. If scores are consistently low, consider increasing the number of bootstrap samples with `--numbstrap` (default is 100) for better confidence interval estimation.
- **Post-Processing**: GangSTR output is best filtered using `dumpSTR` (part of the TR-toolkit), which is designed to handle the specific quality metrics generated by this tool.
