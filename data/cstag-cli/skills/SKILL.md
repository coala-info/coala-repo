# cstag-cli

## Overview
`cstag-cli` is a specialized command-line utility designed to generate and append "cs" tags to genomic alignment data. The CS tag is a compact string format used by minimap2 to describe mismatches, insertions, and deletions relative to a reference. This tool is particularly valuable when your alignment files (SAM/BAM) were generated by aligners that do not produce CS tags by default, or when you need to convert between short and long CS tag formats.

## Core Usage Patterns

### Appending CS Tags
The primary command is `cstag append`. By default, it generates the short format CS tag.

```bash
# Basic usage with a BAM file
cstag append input.bam > output_with_cs.sam

# Generating long format CS tags
cstag append input.bam --long > output_long_cs.sam
```

### Working with Standard Input/Output
`cstag-cli` follows Unix philosophy and can be integrated into command pipelines. If the file path is omitted, it reads from stdin.

```bash
# Piping from another tool
cat input.sam | cstag append > output.sam

# Processing BAM files through samtools
samtools view -h input.bam | cstag append > output.sam
```

## Critical Prerequisites and Best Practices

### The MD Tag Requirement
`cstag-cli` requires the input SAM/BAM file to contain the **MD tag** (String for mismatching positions). If your file lacks MD tags, you must generate them first using `samtools calmd`.

```bash
# Example: Adding MD tags before appending CS tags
samtools calmd -b input.bam reference.fa | cstag append > output_with_cs.sam
```

### Format Selection
*   **Short Format (Default):** Best for saving disk space and general compatibility.
*   **Long Format (`-l` / `--long`):** Includes the reference sequences within the tag. Use this when downstream tools require the explicit reference base information for every operation.

### Performance Tip
When processing large datasets, use BAM files as input to reduce I/O overhead, but remember that `cstag-cli` outputs SAM format by default. You may want to pipe the output back into `samtools view -b` to save space.

```bash
cstag append large_input.bam | samtools view -b > final_output.bam
```

## Troubleshooting and Environment Notes

### Apple Silicon (M1/M2/M3) Users
Bioconda does not natively support the `osx-64` architecture for this package on ARM64. If installing via Conda on Apple Silicon, you must force the `osx-64` platform using Rosetta:

```bash
CONDA_SUBDIR=osx-64 conda create -n env-cstag -c conda-forge -c bioconda python=3.10 cstag-cli -y
conda activate env-cstag
conda config --env --set subdir osx-64
```

### Python Dependencies
If you encounter issues with `pysam` during installation via PyPI, ensure you have the necessary development headers for `htslib` installed on your system, or prefer the Bioconda installation which manages these dependencies automatically.
