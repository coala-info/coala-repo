# ucsc-psltobigpsl

## Overview
The `pslToBigPsl` utility is part of the UCSC Genome Browser "kent" source tool suite. It transforms standard PSL alignment files—typically generated by tools like BLAT—into a specialized BED format known as bigPsl. This format is essential for creating bigPsl tracks, which allow for the efficient display of alignments (including protein-to-DNA and cross-species alignments) with features like codon coloring and base-level details in the UCSC Genome Browser.

## Usage Patterns

### Basic Conversion
The tool takes a PSL file as input and produces a bigPsl file (which is essentially a BED file with specific extra columns).

```bash
pslToBigPsl input.psl output.bigPsl
```

### Standard Workflow for BigPsl Tracks
To create a functional bigPsl track for the UCSC Genome Browser, you must follow a specific sequence: conversion, sorting, and indexing.

1.  **Convert PSL to bigPsl:**
    ```bash
    pslToBigPsl alignments.psl alignments.bigPsl
    ```

2.  **Sort the output:**
    BigBed indexing requires the input to be sorted by chromosome and then by start position.
    ```bash
    bedSort alignments.bigPsl alignments.sorted.bigPsl
    ```

3.  **Create the BigBed file:**
    Use `bedToBigBed` with the `-type=bed12+12` and the appropriate `.as` (AutoSql) file.
    ```bash
    bedToBigBed -type=bed12+12 -tab -as=bigPsl.as alignments.sorted.bigPsl chrom.sizes alignments.bb
    ```

## Expert Tips

### Required AutoSql Schema
When converting the resulting bigPsl file to a bigBed file using `bedToBigBed`, you must use the `bigPsl.as` schema definition. This file defines the 24 columns (12 standard BED columns + 12 PSL-specific columns) that make up a bigPsl record. Without this schema, the Genome Browser will not recognize the alignment-specific fields.

### Handling Protein Alignments
If your PSL file contains protein-to-DNA alignments (often generated by `blat -t=dnax -q=prot`), `pslToBigPsl` preserves the coordinate mapping necessary for the Genome Browser to correctly display the translated alignment.

### Chromosome Sizes
Always ensure you have a `chrom.sizes` file for the target assembly. You can generate this using `fetchChromSizes` or by extracting it from a 2bit file using `twoBitInfo`.
