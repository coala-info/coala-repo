# cegroomer

## Overview
TraceGroomer (cegroomer) is a specialized preprocessing tool designed to bridge the gap between metabolomics quantification outputs and downstream statistical analysis. It automates the transformation of "corrected" or "labeled" metabolomics data into a consistent, tab-delimited structure. Beyond simple reformatting, it provides essential data grooming capabilities, including normalization by tissue weight or cell number, internal standard correction, and automated calculation of derived metrics like mean enrichment and isotopologue proportions.

## Command Line Usage

The tool is typically executed as a Python module. The basic syntax requires specifying the measurement file, the input format type, and a configuration file.

### Core Execution
```bash
python3 -m tracegroomer -lm [PATH_TO_MEASUREMENTS] -tf [INPUT_TYPE] -cf [PATH_TO_CONFIG]
```

### Supported Input Types (-tf)
You must specify one of the following four formats based on your source data:
- `IsoCor_out_tsv`: For outputs generated by the IsoCor tool.
- `rule_tsv`: For datasets where a data matrix is accompanied by a variable metadata file.
- `VIBMEC_xlsx`: For data originating from the VIB Metabolomics Expertise Center.
- `generic_xlsx`: For custom Excel spreadsheets following the TraceGroomer naming convention (e.g., using the `_labelX` suffix).

### Normalization Options
TraceGroomer performs normalization in a specific order (material amount first, then internal standards) to ensure data integrity.

- **Material Amount**: Use `--amountMaterial_path [FILE]` to provide a tab-delimited file with sample names and weights/cell counts.
  - By default, it performs `(quotient * average_amount)` to maintain abundance units.
  - Use `--no-alternative_div_amount_material` to return raw quotients only.
- **Internal Standard**: Use `--use_internal_standard [METABOLITE_NAME]` to divide total abundances by a specific reference compound.
- **VIB-Specific**: For VIB data, use `--subtract_blankavg` and `--under_detection_limit_set_nan` (both active by default) to handle background noise and LOD thresholds.

### Data Filtering and Quality Control
- **Metabolite Exclusion**: Use `--remove_these_metabolites [FILE]` to provide a list of compartment-specific metabolites to drop.
- **Stomping**: The tool automatically stomps fractional contributions and isotopologue proportions to a range of [0.0, 1.0] by default.
- **Cutoffs**: Use `--isosprop_min_admitted [FLOAT]` to remove metabolites with proportions below a specific threshold (default is -0.5).

## Best Practices
- **Metadata Preparation**: Ensure your metadata file contains the six mandatory columns: `name_to_plot`, `condition`, `timepoint`, `timenum`, `compartment`, and `original_name`.
- **File Extensions**: While the tool generates tab-delimited content, you can customize the output extension (e.g., .csv, .tsv, .txt) using the `-ox` parameter.
- **Folder Organization**: Maintain a project structure where `data/` (for measurements) and `groom_files/` (for configs and normalization files) are separate. This facilitates seamless integration with the DIMet analysis suite.
- **Isotopologue Naming**: For generic Excel inputs, ensure isotopologues follow the `metaboliteID_labelX` convention (e.g., `citrate_label3`).
