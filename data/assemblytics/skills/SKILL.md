# assemblytics

## Overview
Assemblytics is a specialized tool for identifying six classes of genomic variants based on alignment signatures between a de novo assembly and a reference. It uses a unique "anchor filtering" approach to handle repetitive elements, making it robust for comparing aberrant genomes (like cancer) or related species. The tool bridges the gap between raw assembly alignments and actionable variant calls.

## Installation
Install via Bioconda for the most stable environment:
```bash
conda install -c bioconda assemblytics
```

## Core Workflow

### 1. Generate Input with MUMmer
Assemblytics requires a `.delta` file generated by `nucmer`. Use the following parameters to ensure the alignment is compatible:
```bash
nucmer -maxmatch -l 100 -c 500 REFERENCE.fa ASSEMBLY.fa -prefix OUT
```
*   `-maxmatch`: Includes all matches regardless of uniqueness (Assemblytics handles filtering later).
*   `-l 100`: Minimum length of a match.
*   `-c 500`: Minimum length of a cluster of matches.

### 2. Run Assemblytics CLI
Execute the analysis by providing the delta file and defining the variant size boundaries:
```bash
Assemblytics OUT.delta OUT_prefix UNIQUE_ANCHOR_LENGTH MIN_SIZE MAX_SIZE
```
*   **UNIQUE_ANCHOR_LENGTH**: (e.g., 10000) The minimum unique sequence length required on either side of a variant to "anchor" the call.
*   **MIN_SIZE**: (e.g., 50) Minimum variant size to report.
*   **MAX_SIZE**: (e.g., 100000) Maximum variant size to report.

## Expert Tips & Best Practices
*   **Contigs vs. Scaffolds**: Always use **contigs** rather than scaffolds as input. Using scaffolds with "N" gaps can lead to false-positive variant calls if the gap size in the assembly doesn't perfectly match the reference distance.
*   **Anchor Selection**: For highly repetitive genomes, increase the `UNIQUE_ANCHOR_LENGTH` to reduce false positives. For very fragmented assemblies, you may need to decrease it, though this reduces confidence in the calls.
*   **Performance**: If working with large genomes, gzip your delta file (`gzip OUT.delta`) before processing; Assemblytics can handle compressed inputs to save I/O time.
*   **Variant Classes**: The tool identifies six specific types:
    1. Insertions
    2. Deletions
    3. Tandem expansions
    4. Tandem contractions
    5. Repeat expansions
    6. Repeat contractions
