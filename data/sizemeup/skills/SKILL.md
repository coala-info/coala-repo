# sizemeup

## Overview
sizemeup is a specialized utility that provides the genome size of organisms by querying a curated database. It aggregates data from NCBI Assembly Reports, the AllTheBacteria (ATB) project, and manually curated user entries. It is designed to eliminate the need for manual lookups when processing large batches of genomic data, ensuring consistency in metrics like depth of coverage.

## Installation and Setup
Install sizemeup via Bioconda to ensure all dependencies (like the NCBI Datasets API) are handled:

```bash
conda create -n sizemeup -c conda-forge -c bioconda sizemeup
conda activate sizemeup
```

## Command Line Usage

### Basic Queries
You can query the database using either a formal species name or a numerical NCBI TaxID.

**Query by Species Name:**
```bash
sizemeup --query "Staphylococcus aureus" --sizes path/to/sizes.tsv
```

**Query by TaxID:**
```bash
sizemeup --query 1280 --sizes path/to/sizes.tsv
```

### Output Management
By default, the tool prints a formatted table to STDOUT and writes a TSV file to the current directory.

*   **Change Output Directory:** Use `--outdir` or `-o` to specify where the result file should be saved.
*   **Custom Filenames:** Use `--prefix` or `-p` to change the default "sizemeup" prefix for the output file.
*   **Scripting Mode:** Use `--silent` to suppress the STDOUT table and only receive critical error messages, making it easier to parse in automated pipelines.

### Database Maintenance
If you need to update the underlying genome size database with the latest NCBI reports:

```bash
sizemeup-build [OPTIONS]
```
This helper tool downloads the latest NCBI Assembly Reports and merges them with existing data to produce a fresh TSV file for use with the `--sizes` flag.

## Best Practices
*   **Mandatory Sizes File:** The `--sizes` flag is required. Always ensure you are pointing to the correct TSV database file provided by the installation or generated by `sizemeup-build`.
*   **Verify Methods:** Review the `method` column in the output. Sizes marked as `manually-set` or `ncbi` are generally more reliable than those derived from automated assembly aggregations.
*   **Handling Unknowns:** If a species is not found, sizemeup returns a result with `UNKNOWN_SPECIES` and a size of `0`. Always check the output file or STDOUT for this state before proceeding with coverage calculations.
*   **TaxID Precision:** When possible, use TaxIDs instead of species names to avoid issues with synonyms or spelling variations in the database.
