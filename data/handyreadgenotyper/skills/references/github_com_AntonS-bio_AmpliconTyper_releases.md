[Latest](/AntonS-bio/AmpliconTyper/releases/latest)

* Amended the definition of "valid amplicon" for purposes of "Support for target organism column". Previously it had to be chromosomal non-AMR amplicon, now it is simply chromosomal amplicon i.e. AMRs are permitted. The rest of the definition remains: >50 reads, <=2 unknown SNPs.
* Fixed reporting of multiple genotypes in Summary Table. Previously only one genotype was reported even if two from separate phylogeny branches were present. E.g. previously if a sample had genotypes 2.3.2 and 4.3.1.1 only one of these would be reported, now both would be. However, if the genotypes are direct ancestor-descendants (e.g. 4.3.1 and 4.3.1.1.P1) , only lowest one (4.3.1.1.P1) would be reported.

[Fix to 2.3.1 misreporting and added quality check](/AntonS-bio/AmpliconTyper/releases/tag/0.1.32)

* The Deletion specific to genotype 2.3.1 (amplicon 2.3.1\_v12) was previously incorrectly reported because it is located in homopolymer region and Oxford Nanopore sequencing has high error in homopolymer regions. To correct this, the genotype 2.3.1 will only be reported if the deletion is present in >20% of reads. The default for other mutations remains 5%. **This requires model typhi\_v11.pkl.**
* A new column "Support for Target Organism" has been added to Summary table. This column reports one of three values: "Strong", "Weak" and "None" based on number of valid amplicons being 4+, 2-3 and 0-1 respectively. Amplicon is valid if it is a) not AMR related, b) has >50 reads and c) has <=2 unknown SNPs. This is based on discussions among the project members.
* Program now creates two, instead of one, TSV tables. In addition to table with detailed per amplicon results, it now creates at table called YOURFILENAME\_summary.tsv which contains the values from the Summary table.
* A new warning has been added to Summary table. For most runs, there will be a message above the Summary table saying:
  "Expected error rate based on Q-scores: XYZ%. This is within the expected range."
  Occasionally, the message will be in red and say:
  "Expected error rate based on Q-scores: RTS%. This is much higher than expected range (~3.00%) and will make genotype and AMR calls very unreliable. This is a sequencing, not analysis problem. Please check MinKNOW to identify possible causes."
  This indicates that something went wrong with the sequencing run.
* The column "Amplicons by # of dominant unknown SNPs" has been moved and is now the last column in Summary table.
* I've made several improvements to the code to improve speed.

If you encounter any issues, please raise them on GitHub.
