# scater-scripts

## Overview

The `scater-scripts` package provides a suite of R-based command-line wrappers for the Bioconductor `Scater` package. It is designed to allow researchers to execute complex single-cell analysis steps—such as QC, normalization, and visualization—directly from the terminal or within automated pipelines without needing to write R code. The scripts primarily use the `SingleCellExperiment` (.rds) format for intermediate data storage, ensuring compatibility between different steps of the analysis.

## Command Line Usage and Best Practices

### Core Workflow Steps

1.  **Data Ingestion**
    Convert 10X Genomics output directories into the standard `.rds` format used by subsequent scripts.
    `scater-read-10x-results.R -d <data_dir> -o <output.rds>`

2.  **Quality Control Calculation**
    Generate comprehensive QC metrics for cells and features.
    `scater-calculate-qc-metrics.R -i <input.rds> -o <output_with_qc.rds>`
    *Tip: Use `-f` to specify feature controls (e.g., mitochondrial genes) to calculate specific percentages.*

3.  **Filtering**
    Apply thresholds to remove low-quality cells or lowly-expressed features.
    `scater-filter.R -i <input.rds> -s "total_counts,total_features_by_counts" -l "500,200" -o <filtered.rds>`
    *Note: Metrics provided to `-s` must exist in the object's colData (generated by the QC script).*

4.  **Normalization**
    Convert raw counts to normalized expression values.
    `scater-normalize.R -i <input.rds> -o <normalized.rds>`

5.  **Dimensionality Reduction**
    Compute PCA or t-SNE coordinates for visualization and downstream clustering.
    `scater-run-pca.R -i <input.rds> -o <pca_results.rds>`
    `scater-run-tsne.R -i <input.rds> -o <tsne_results.rds>`

### Advanced Patterns and Tips

*   **Metric Extraction**: Use `scater-extract-qc-metric.R` to pull specific metrics into a CSV for external analysis or for use with the outlier detection script.
*   **Outlier Detection**: Instead of hard filtering, use `scater-is-outlier.R` to statistically identify outliers based on Median Absolute Deviation (MAD).
    `scater-is-outlier.R -m <metrics.csv> -n 3 -t "lower" -o <outliers.csv>`
*   **Help Access**: Every script supports the `--help` flag. Use it to discover specific parameters for R functions like `calculateCPM()` or `normalize()`.
*   **Intermediate Files**: Since the scripts output `.rds` files, you can inspect the state of your object at any time in an R session using `readRDS()`.
