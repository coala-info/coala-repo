# concoord

## Overview

CONCOORD is a computationally efficient alternative to Molecular Dynamics for sampling protein conformations. It operates in two distinct stages: first, it analyzes a starting structure to define a set of interatomic distance bounds (covalent bonds, hydrogen bonds, and non-bonded interactions); second, it generates new structures by iteratively adjusting random coordinates until they satisfy all defined geometric constraints. This method is ideal for researchers needing to predict protein flexibility or generate diverse structural ensembles for docking and refinement.

## Environment Setup

Before running the tools, ensure the environment is initialized to locate necessary library files:

```bash
source /path/to/concoord/CONCOORDRC
```

## Core Workflow

The process requires running two separate programs in sequence.

### 1. Generating Distance Bounds (dist)
The `dist` program analyzes the input PDB and creates a bounds file.

```bash
dist -p protein.pdb -o protein.dat
```

**Key Options:**
- `-p <file>`: Input PDB file.
- `-o <file>`: Output bounds file (default is `dist.dat`).
- `-h`: Display full usage and interaction strength categories.

### 2. Generating Conformations (disco)
The `disco` program uses the bounds file to produce new structural models.

```bash
disco -d protein.dat -n 100 -o ensemble.pdb
```

**Key Options:**
- `-d <file>`: Input bounds file generated by `dist`.
- `-n <int>`: Number of structures to generate.
- `-o <file>`: Output filename for the generated structures.
- `-on`: Write output in NMR-style PDB format (multiple models in one file).
- `-dyn`: Enable dynamical adaptation of tolerances for more robust generation.
- `-reg`: Apply structure regularization to the generated models.

## Expert Tips and Patterns

### Parallel Processing
For large ensembles on multi-core machines, use the parallel operation feature to speed up structure generation:
```bash
disco -d protein.dat -n 500 -p <number_of_cpus>
```

### Handling Convergence Issues
If `disco` struggles to fulfill all bounds (common in highly constrained or crowded systems), try the following:
- Use the `-dyn` flag to allow the program to adapt tolerances dynamically.
- Adjust the de-bumping parameters using `-rc` and `-l` for more flexible control over atom clashes.
- Increase the maximum number of iterations or adjust convergence checks using the `-con` option.

### Neighbor Searching
In version 2.1 and later, CONCOORD uses grid-based neighbor searching. This significantly enhances efficiency for large proteins. Ensure you are using the latest binary (v2.1.2) to benefit from these inner-loop optimizations.



## Subcommands

| Command | Description |
|---------|-------------|
| disco | disco generates (macro)molecular structures based on a set of geometric constraints (usually generated by the program "dist", but also additional restraints may be specified). |
| dist | dist defines a set of geometric constraints from which structures can be generated by the program disco. |
