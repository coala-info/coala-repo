# methphaser

## Overview

MethPhaser is a bioinformatics tool designed for methylation-based haplotype phasing of human genomes. It serves as a secondary phasing step that utilizes methylation signals from Oxford Nanopore Technologies (ONT) reads to connect and extend haplotype blocks initially generated by SNV-based methods like HapCUT2 or WhatsHap. 

This skill should be used when working with high-coverage (30X+) human autosome data where traditional phasing has left significant gaps or unphased regions. It is specifically designed to handle the unique challenges of methylation data to provide a more contiguous chromosomal-scale phasing.

## Installation and Requirements

Install MethPhaser via Bioconda:

```bash
mamba install -c bioconda methphaser
# OR
conda install -c bioconda methphaser
```

### Critical Constraints
*   **Autosomes Only**: Do not use on sex chromosomes (X/Y) due to random X-inactivation and non-diploid nature.
*   **Coverage**: Recommended for ONT reads with at least 30X coverage.
*   **BAM Format**: Input BAM files must contain methylation annotations (MM/ML tags) and **only primary alignments**. Secondary and supplementary alignments must be removed to avoid `pysam` errors.

## Core Workflow

The MethPhaser workflow consists of two primary steps: block relationship determination and post-processing.

### Step 1: Determine Block Relationships
Use `meth_phaser_parallel` to identify relationships between existing phased blocks.

```bash
meth_phaser_parallel \
  -b input_methylation_annotated.bam \
  -r reference_genome.fna \
  -g whatshap_visualization.gtf \
  -vc hapcut2_phased.vcf.gz \
  -t 16 \
  -o output_directory
```

**Key Parameters:**
*   `-ml`: Maximum homozygous region length. 
    *   Default (`-1`): Ignores the largest unphased region (typically the centromere).
    *   Use `-2`: If the input data does not contain large poorly mapped regions and you want to phase across all gaps.
*   `-c`: Minimum percentage of "votes" required to determine a read's haplotype.
*   `-k`: Maximum iterations (use `-1` for unlimited).

### Step 2: Post-Processing
Generate the final modified VCF and haplotagged BAM files.

```bash
meth_phaser_post_processing \
  -ib input_methylation_annotated.bam \
  -if output_directory/ \
  -ov final_output.vcf \
  -ob final_output_bam_prefix \
  -vc hapcut2_phased.vcf.gz
```

## Expert Tips and Best Practices

### BAM Pre-processing
Before running MethPhaser, ensure your BAM file is filtered to contain only primary alignments. Use `samtools` to clean the file:
```bash
samtools view -bF 2304 -o primary_only.bam input.bam
```

### Handling Centromeres
If your goal is to phase across the centromere and you have sufficient spanning reads, ensure you set `-ml -2`. By default, MethPhaser assumes the largest gap is the centromere and will not attempt to bridge it to avoid false joins in low-complexity regions.

### Input Preparation
MethPhaser requires a GTF file typically generated from WhatsHap visualization and a phased VCF from a tool like HapCUT2. Ensure these upstream files are generated using the same reference genome version used in the `-r` flag.
