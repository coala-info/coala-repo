***Note:* Scoring of VCF files with CADD v1.7 is still rather slow if many new
variants need to be calculated from scratch (e.g., if many insertion/deletion or multi-nucleotide substitutions
are included). If possible use the pre-scored whole genome and pre-calculated indel files directly where possible.**

# How to contact us?

If you have technical problems using CADD, please check the information
provided [here](/info). If this does not resolve your issues, please
contact us at cadd-support [at] uw.edu.

CADD is currently developed by [Martin Kircher](https://www.bihealth.org/en/research/research-group/kircher-lab-computational-genome-biology/), [Max
Schubach](https://kircherlab.bihealth.org/people/max.html), [Thorben Maaß](https://orcid.org/0000-0003-4764-2740), and [Lusine Nazaretyan](https://orcid.org/0000-0001-5820-4143).

Former developers are [Philipp Rentzsch](https://orcid.org/0000-0002-0413-7974),  [Sebastian Röner](https://orcid.org/0000-0002-8578-1269), [Daniela M. Witten](https://www.biostat.washington.edu/people/daniela-witten), [Gregory M. Cooper](https://hudsonalpha.org/faculty/greg-cooper/), and [Jay Shendure](http://krishna.gs.washington.edu/people.html).

If you are planning on using CADD in a commercial application, you can obtain a license through the [UW CoMotion Express Licensing System](https://els2.comotion.uw.edu/product/cadd-scores). If in doubt about
whether you need a license for your application, please contact
[Jay Shendure](http://krishna.gs.washington.edu/people.html) and
[Gregory M. Cooper](https://hudsonalpha.org/faculty/greg-cooper/).
