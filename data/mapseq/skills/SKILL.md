# mapseq

## Overview

MAPseq is a highly efficient k-mer based classifier designed for ribosomal RNA (rRNA) sequence analysis. It transforms raw metagenomic FASTA data into taxonomic assignments by comparing reads against a reference database of full-length sequences. It is the preferred tool when you require fast processing speeds without sacrificing the ability to see confidence scores at every taxonomic level, from Kingdom down to Species.

## Installation

The most reliable way to install MAPseq is via Bioconda:

```bash
conda install bioconda::mapseq
```

*Note: If installing from a binary package manually, do not move the binary file out of its original folder, as it relies on relative paths to locate its internal data files.*

## Common CLI Patterns

### Basic Classification
To classify a demultiplexed and quality-filtered FASTA file using the default reference database:

```bash
mapseq rawseqs.fa > output.mseq
```

### Performance Optimization
Use the `-nthreads` flag to speed up processing on multi-core systems:

```bash
mapseq -nthreads 8 rawseqs.fa > output.mseq
```

### Using Custom Reference Databases
To use a specific reference set, provide the FASTA file and the corresponding taxonomy file(s):

```bash
mapseq rawseqs.fa custom_ref.fasta custom_ref.tax > output.mseq
```

### Generating Count Summaries
After classification, you can generate a summary of counts for a single sample:

```bash
mapseq -otucounts sample.mseq > sample_counts.txt
```

### Creating an OTU Table for Multiple Samples
To compare multiple samples, use the `-otutable` command. You can specify the taxonomy index (`-ti`) and level (`-tl`):

```bash
mapseq -otutable sample1.mseq sample2.mseq sample3.mseq > combined_table.tsv
```

## Expert Tips and Best Practices

- **Input Preparation**: MAPseq expects FASTA input. Ensure your data is already demultiplexed and quality-filtered. If you have FASTQ files, convert them to FASTA first.
- **Output Formats**: Use `--outfmt confidences` (default in newer versions) to get detailed confidence values for each taxonomic level. This is critical for filtering out low-confidence assignments in downstream analysis.
- **Custom Taxonomy Headers**: When creating a custom `.tax` file, the header must include identity cutoff parameters (e.g., `#cutoff: 0.00:0.08 ...`) and level names (e.g., `#levels: Kingdom Phylum ...`) for the tool to parse the assignments correctly.
- **R Integration**: The OTU tables generated by MAPseq are standard TSV files. They can be loaded directly into R for ecological analysis using:
  `my_data <- read.table("combined_table.tsv", sep="\t", header=TRUE)`
