# panacus

## Overview

Panacus is a high-performance tool designed for the statistical analysis of pangenome graphs. It operates on GFA files (supporting both P and W lines) to provide insights into how a pangenome expands as more haplotypes are added. By calculating metrics such as pangenome growth and core size estimation, it helps researchers quantify the genomic diversity within a graph. The tool is particularly effective for generating comprehensive HTML reports that visualize node distributions, path similarities, and allele frequencies.

## Tool-Specific Best Practices

### Graph Requirements
* **Blunt Graphs Only**: Ensure the input GFA is "blunt." Panacus requires that nodes do not overlap; every link (L) must point from the end of one segment (S) to the start of another.
* **Line Support**: The tool natively supports GFA files containing Path (P) and Walk (W) lines for haplotype representation.

### Core Analysis Concepts
* **Count Types**: When running analyses, specify the feature type to be counted. Valid options include:
    * `Bp`: Base pairs (length-based analysis).
    * `Node`: Individual graph segments.
    * `Edge`: Connections between segments.
    * `All`: A combination of all features.
* **Pangenome Growth**: Use this to estimate how many new features are expected on average when building a graph from $1$ to $n$ haplotypes.
* **Quorum vs. Coverage**:
    * **Coverage**: A preprocessing filter. It defines how many times a feature must appear in the paths to be considered valid (useful for filtering sequencing errors).
    * **Quorum**: A calculation filter. It defines the fraction of paths a feature must appear in to be included in a specific category (e.g., a quorum of `1.0` identifies the "core" genome, while `0.0` includes the entire pangenome).

### Common CLI Operations
* **Generating Reports**: The primary workflow involves the `report` subcommand to produce an interactive HTML file.
    * Command: `panacus report <input_file> > output.html`
* **Visualization**: Use the `panacus-visualize` script (requires Python libraries like matplotlib and pandas) to generate static plots from the raw data generated by the main tool.
* **Path Similarity**: Use this analysis to compare how similar different haplotypes are based on the graph features they share.

### Expert Tips
* **Reference-Based Analysis**: When using the `!CoverageLine` analysis, always specify a `reference` path (e.g., "GRCh38") to plot features that are non-reference against their frequencies.
* **Grouping**: If your GFA contains many paths, use the grouping feature to aggregate statistics by sample or species, which simplifies the resulting bar charts and histograms.
* **Custom Integrations**: Panacus can incorporate external tool outputs (PNG, SVG, PDF, CSV, or Vega-Lite JSON) into its final HTML report using the `!Custom` analysis type.



## Subcommands

| Command | Description |
|---------|-------------|
| panacus | For more information, try '--help'. |
| panacus | For more information, try '--help'. |
| panacus | Panacus is a tool for analyzing and visualizing genomic data. |
| panacus hist | Calculate coverage histogram |
| panacus node-distribution | Return the list of bins with there coverages, log10-lengths and log10-sizes. Due to this being the values for the centers of the hexagons shown in the html plot and not real values, some values might be negative. |
| panacus ordered-histgrowth | Calculate growth curve based on group file order (if order is unspecified, use path order in GFA) |
| panacus render | Render an html report from one or more JSON result files |

| panacus similarity | Compute coverage table for count type |
| panacus table | Compute coverage table for count type |
| panacus_growth | Calculate growth curve from coverage histogram |
| panacus_info | Return general graph and paths info |
