[v4.1.0](/CompOmics/ms2pip/releases/tag/v4.1.0)

### Added

* ✨ Support for Thermo raw (requiring dotnet runtime) and gzipped spectrum files, through mobiusklein/mzdata ([#226](https://github.com/CompOmics/ms2pip/pull/226), by [@paretje](https://github.com/paretje))
* ✨ New Python API usage mode `correlate-single` to correlate to a single `ObservedSpectrum` object with predictions ([#232](https://github.com/CompOmics/ms2pip/pull/232))
* ✨ Added support for Python 3.12 and 3.13, Numpy v2, and newer Pandas and XGBoost versions ([#228](https://github.com/CompOmics/ms2pip/pull/228))
* ✅ CI: Add integration test for `predict-single` ([#228](https://github.com/CompOmics/ms2pip/pull/228))

### Changed

* ⚠️ Ion mobility is now returned instead of collisional cross section when IM2Deep is used through the `add_ion_mobility` option ([#236](https://github.com/CompOmics/ms2pip/pull/236), by rodvrees)
* 🏗️ Model files are now downloaded from Zenodo instead of Genesis ([#225](https://github.com/CompOmics/ms2pip/pull/225), by paretje, fixes [#229](https://github.com/CompOmics/ms2pip/issues/229))
* 📝 Docs: Updated README to include correlate-single; moved webserver API docs from old Wiki ([#239](https://github.com/CompOmics/ms2pip/pull/239))

### Removed

* 🐍 Removed support for Python 3.8 (EOL) and for musllinux distributions ([#228](https://github.com/CompOmics/ms2pip/pull/228), [#237](https://github.com/CompOmics/ms2pip/pull/237))
* 🗑️ Removed old conversion scripts, mostly implemented in psm\_utils ([#239](https://github.com/CompOmics/ms2pip/pull/239))
* 🗑️ Removed old train script, to be replaced with page in docs ([#239](https://github.com/CompOmics/ms2pip/pull/239))
