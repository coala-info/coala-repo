# bmtagger

## Overview
BMTagger (Best Match Tagger) is a bioinformatics tool designed to rapidly screen Next-Generation Sequencing (NGS) reads against a reference host genome. It works by using a combination of a bitmask (for fast lookup) and a sensitive alignment engine (SRPRISM) to tag reads that match the host. This skill provides the necessary command-line patterns to configure the tool, specify input FASTQ files, and generate a list of contaminated sequences for removal.

## Usage Guidelines

### Input Requirements
BMTagger requires several pre-computed index files for the host genome:
- **Bitmask file**: Usually generated by `bmtool`, used for initial rapid screening.
- **SRPRISM index**: A prefix for the SRPRISM genomic index files.
- **Config file**: A text file defining the `BMTAGGER_DB` path.

### Command Execution
The primary execution script is `bmtagger.sh`. For paired-end data, use the following pattern:

```bash
bmtagger.sh -C <config_file> \
            -b <host.bitmask> \
            -x <host.srprism_prefix> \
            -q1 \
            -1 <forward.fastq> \
            -2 <reverse.fastq> \
            -o <output_list_file>
```

### Parameter Reference
- `-C`: Path to the configuration file.
- `-b`: Path to the bitmask file (e.g., `Mouse.bitmask`).
- `-x`: Prefix for the SRPRISM index (e.g., `Mouse.srprism`).
- `-q1`: Indicates that the input files are in FASTQ format.
- `-1`: Path to the forward reads (R1).
- `-2`: Path to the reverse reads (R2).
- `-o`: The name of the output file which will contain a list of the tagged host sequence IDs.

### Best Practices
- **Path Management**: Ensure all index files (bitmask and srprism) are in the same directory or correctly referenced relative to the execution path.
- **Memory Allocation**: BMTagger can be memory-intensive depending on the size of the host genome; ensure the environment has sufficient RAM to load the bitmask and index.
- **Downstream Filtering**: The output is a LIST of read IDs. Use tools like `seqtk` or custom scripts to filter these IDs out of your original FASTQ files to obtain "clean" microbial data.
