# pispino

## Overview
PISPINO (PIpits SPIN-Off tools) provides a streamlined workflow for the preliminary processing of raw sequencing data. It is particularly effective for handling demultiplexed Illumina reads, allowing users to generate sample manifests and execute sequence preparation tasks like merging and quality filtering through a simple command-line interface. Originally developed as part of the PIPITS pipeline, these tools are now independent and optimized for generic NGS data preparation on Linux and macOS.

## Installation and Environment
PISPINO requires a Python 3 environment and is distributed via Bioconda. It is recommended to run these tools within a dedicated Conda "sandbox" to avoid dependency conflicts.

```bash
# Setup required channels
conda config --add channels defaults
conda config --add channels conda-forge
conda config --add channels bioconda

# Create and activate environment
conda create -n pispino_env python=3.6 pispino
source activate pispino_env
```

## Core Workflow

### 1. Generate Read Pairs List
The first step in the PISPINO workflow is mapping sample names to their respective forward and reverse FASTQ files. The tool `pispino_createreadpairslist` automates this by scanning an input directory.

```bash
pispino_createreadpairslist -i <raw_data_dir> -o readpairslist.txt
```
- **Input (-i)**: The directory containing your raw FASTQ sequences. The tool supports uncompressed, `.gz`, and `.bzip2` formats.
- **Output (-o)**: A tab-delimited text file containing the sample mapping.

### 2. Sequence Preparation
After verifying the generated list, use `pispino_seqprep` to perform quality filtering, reindexing, and merging of the read pairs.

```bash
pispino_seqprep -i <raw_data_dir> -o <output_dir> -l readpairslist.txt
```
- **List (-l)**: The manifest file created in the previous step.
- **Output (-o)**: The destination directory for the processed NGS data.

## Expert Tips and Best Practices
- **Manifest Verification**: Always manually inspect the `readpairslist.txt` before proceeding to the `seqprep` step. Ensure that the sample names and file pairings accurately reflect your experimental setup.
- **Input Requirements**: PISPINO is designed for demultiplexed FASTQ files where the sequencer has already split reads into separate files based on barcodes.
- **Compatibility**: The output generated by `pispino_seqprep` is specifically formatted for compatibility with downstream pipelines such as PIPITS and QIIME.
- **Extended Options**: For advanced configurations regarding quality thresholds or merging parameters, use the help flag: `pispino_seqprep -h`.
