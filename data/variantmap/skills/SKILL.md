# variantmap

## Overview
The `variantmap` tool provides an interactive, browser-based interface for exploring structural variant data across sample cohorts. Built on the Dash framework, it transforms complex genomic data into a heatmap where rows represent samples and columns represent SV breakends. This skill enables the deployment and utilization of the tool to identify variant patterns, filter by gene annotations, and extract specific variant metrics like size, score, and read coverage.

## Installation and Setup
The tool can be installed via Conda or Pip. Conda is generally preferred for bioinformatics environments to manage dependencies like `pytables`.

```bash
# Installation via Bioconda
conda install -c bioconda variantmap

# Installation via Pip
pip install variantmap
```

## Core Usage
The tool is primarily a web application launched from the command line.

### Launching the Application
To start the interactive server, execute the application script. By default, this launches a local Dash server.

```bash
variantmap_app.py
```

Once running, access the interface through your web browser (typically at `http://127.0.0.1:8050`).

### Data Input Requirements
`variantmap` is not a standalone SV caller; it is a visualization layer for specific upstream pipelines.
- **Required Format**: HDF5 (.h5) files.
- **Upstream Tool**: Input must be generated by **VariantBreak**.
- **Compatibility**: VariantBreak currently processes VCFs from **NanoVar** (long-read sequencing data).

## Expert Tips and Best Practices
- **Browser Selection**: For large HDF5 datasets (exceeding 32MB), use **Mozilla Firefox**. Google Chrome may experience performance issues or crashes with large memory-intensive heatmaps.
- **Interactive Exploration**: Use the "Customize" tab within the web UI to adjust heatmap components and color schemes based on SV classes.
- **Filtering**: Leverage the built-in filtering options to narrow down variants by gene name, feature, or repetitive elements, provided the input HDF5 was properly annotated during the VariantBreak step.
- **Data Snapshots**: Use the one-click snapshot feature in the UI to capture high-quality images of the heatmap for publications or reports.
- **Dependency Management**: If the app fails to launch, ensure `pandas (>=1.1.4)`, `dash (>=1.17.0)`, and `tables (>=3.6.1)` are correctly installed in your environment.
