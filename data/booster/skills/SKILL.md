# booster

## Overview
BOOSTER provides a modern alternative to traditional Felsenstein bootstrap methods, which often yield low support values in very large trees due to minor taxa movements. This skill enables the calculation of Transfer Bootstrap Expectation (TBE), a metric that measures the distance between branches rather than requiring exact matches. Use this tool to analyze a reference tree against a set of bootstrap replicates generated by external phylogenetic software like RAxML, PhyML, or IQ-TREE.

## CLI Usage and Best Practices

### Core Command Structure
The basic execution requires a reference tree and a single file containing all bootstrap replicates:
```bash
booster -i <reference_tree.nwk> -b <bootstrap_trees.nwk> -o <output_tree.nwk>
```

### Algorithm Selection
*   **TBE (Default):** Use `-a tbe` for large phylogenies where traditional bootstrap values are consistently low. It is more robust to "rogue" taxa.
*   **FBP:** Use `-a fbp` if you require the traditional Felsenstein Bootstrap Proportion for comparison or for smaller datasets.

### Performance Optimization
*   **Multi-threading:** Use the `-@` flag to specify the number of CPU cores. This is highly recommended for large trees.
    ```bash
    booster -i ref.nwk -b boot.nwk -@ 8 -o supported_tree.nwk
    ```

### Advanced Analysis and Diagnostics
*   **Statistical Logging:** Use `-S` to output a statistics file containing average transfer distances per branch and transfer indices per taxa.
*   **Identifying Unstable Taxa:** Combine `-S` with `-c` (and `-a tbe`) to generate a matrix of individual taxa moves. This helps identify which specific species are responsible for lowering support on a given branch.
    ```bash
    booster -i ref.nwk -b boot.nwk -a tbe -S stats.log -c -o output.nwk
    ```
*   **Distance Cutoff:** The `-d` parameter (default 0.3) controls the sensitivity of the TBE calculation. Adjust this only if you need to change the threshold for considering a branch as "moving."

### Integration with Upstream Tools
BOOSTER does not infer trees itself; it processes trees generated by other tools:
*   **RAxML:** Provide the `RAxML_bestTree` as `-i` and `RAxML_bootstrap` as `-b`.
*   **IQ-TREE:** Provide the `.treefile` as `-i` and `.boottrees` as `-b`.
*   **FastTree:** Concatenate individual bootstrap trees into a single file to use as the `-b` input.
