# Requests Made from the Client[ÿ](#requests-made-from-the-client "Link to this heading")

The Auspice client will make a number of requests to various endpoints, which this document aims to provide an overview of. Some of these are dependent on the [build-time customisations](api.html) in use, and we plan to allow more flexibility here using this mechanism.

## Requests to the Auspice Server[ÿ](#requests-to-the-auspice-server "Link to this heading")

For the purposes of this section we are assuming you are running locally on ãlocalhost:4000ã, however the idea is the same if Auspice is deployed to a server.

### Auspice Client JavaScript Code[ÿ](#auspice-client-javascript-code "Link to this heading")

The (bundled) client code (which is generated by `auspice build`) is broken up into a number of ãchunksã which are delivered to the client as needed. These originate from the following block in the `index.html` file:

```
<script async src="/dist/auspice.bundle.js"></script>
```

### ãCharonã GET requests[ÿ](#charon-get-requests "Link to this heading")

When a dataset, narrative, or listing of available datasets is to be displayed in Auspice, a selection of the following requests are made to localhost:4000.

* `/charon/getAvailable` ã return a list of available datasets and narratives
* `/charon/getDataset` ã return the requested dataset
* `/charon/getNarrative` ã return the requested narrative

See [the server API documentation](../server/api.html) for more details about the requests, or the [client customisation API](api.html) for the ability to change ã/charonã to a different value.

### Image Requests[ÿ](#image-requests "Link to this heading")

The initial Auspice page (i.e. the one which displays a listing of available datasets) requests some images for the footer.

## External Requests[ÿ](#external-requests "Link to this heading")

### Leaflet[ÿ](#leaflet "Link to this heading")

We use [Leaflet](https://leafletjs.com/) to display the map tiles.

If a map is displayed by Auspice the individual tiles are requested from api.mapbox.com. Panning and zooming of the map result in futher requests for tiles. These are requested using our hardcoded access key, however this may change to a build-time-customisation in the future.

### Web Analytics (Optional)[ÿ](#web-analytics-optional "Link to this heading")

Auspice has the potential to include either Plausible Analytics or Google Analytics in a limited fashion. See [the client customisation API documentation](api.html#client-api-available-customisations) for how to set this. Note that these requests are only made if you set up build-time customisation of the client. (For nextstrain.org we use [Plausible analytics](https://github.com/nextstrain/nextstrain.org/blob/2685e6b363cf70808fa11ee38ac520dfb3a9cdc4/auspice-client/customisations/config.json#L14).)
