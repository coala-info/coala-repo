# parebrick

## Overview
PaReBrick (PArallel REarrangements and BReakpoints Identification Toolkit) is a bioinformatics tool designed to detect and rank parallel genomic rearrangement events in bacterial populations. It identifies events such as inversions and deletions that appear independently in phylogenetically distant strains, suggesting potential parallel adaptation or horizontal gene transfer. The tool automates the labor-intensive process of verifying phyletic pattern consistency by testing rearrangements against a provided phylogenetic tree.

## Installation and Environment
PaReBrick requires Python 3.8 or lower due to specific library dependencies.

```bash
# Recommended: Create a dedicated Conda environment
conda create -n py38 python=3.8
conda activate py38

# Install via pip
pip install PaReBrick

# Or install via Bioconda
conda install bioconda::parebrick
```

## Core Usage
The tool requires two primary inputs: a phylogenetic tree and a folder containing synteny blocks.

### Basic Command Pattern
```bash
PaReBrick --tree path/to/tree.nwk --blocks_folder path/to/blocks/
```

### Required Arguments
- `--tree` / `-t`: Path to a phylogenetic tree in Newick format (compatible with the ete3 library).
- `--blocks_folder` / `-b`: Path to a folder containing synteny blocks. These are typically generated by tools like Sibelia, SibeliaZ, or maf2synteny.

### Optional Arguments
- `--labels` / `-l`: Path to a CSV file for visualization. Must contain two columns: `strain` and `label`.
- `--output` / `-o`: Path to the output directory (default: `./parebrick_output`).

## Data Preparation Best Practices
- **Identifier Matching**: Ensure that the strain identifiers used in the Newick tree exactly match the identifiers/filenames within the synteny blocks folder.
- **Synteny Blocks**: If starting from raw sequences, use SibeliaZ to generate the initial alignment and blocks. PaReBrick expects oriented synteny blocks.
- **Tree Format**: Use a standard Newick format. If the tool fails to parse the tree, ensure it is compatible with the ete3 library's standard formats.

## Interpreting Results
The tool categorizes output into three main directories:

1. **preprocessed_data**: Contains synteny blocks converted into common formats (infercars, GRIMM, CSV) and genome length statistics.
2. **balanced_rearrangements_output**: Contains statistics and visualizations for rearrangements that do not change the copy number of blocks (e.g., inversions).
3. **unbalanced_rearrangements_output**: Contains data for rearrangements that involve gain or loss of genetic material (e.g., deletions, insertions).

### Key Output Files
- `stats.csv`: Found in both rearrangement folders. Use this to identify events with high parallelism scores.
- `tree_colorings/`: PDF renderings of the phylogenetic tree with nodes colored by the presence/absence of specific characters, useful for visual verification of parallel events.
