At the end you will see a short summary: Sharing data The results folder for an experiment ( results/&lt;expt_name&gt; ) contains all the outputs from Nomadic . Only the CSV files starting summary and the metadata folder are required to relaunch the dashboard with the nomadic dashboard command (see Basic Usage ). Nomadic allows easy sharing of your workspace by copying key summary Nomadic and MinKNOW files to a new location, e.g. a cloud synchronised folder. nomadic share -t &lt;path/to/your/shared/location&gt; Once shared, Nomadic can be run by collaborators and other members of the group as needed. Configuring default values The above settings can be saved per workspace so that you don't need to enter the details each time. To save the configuration do the following: nomadic configure backup -t &lt;path/to/your/backup/location&gt; nomadic configure share -t &lt;path/to/your/shared/location&gt; Once configured you can omit the -t option e.g. nomadic backup nomadic share Made with Material for MkDocs
