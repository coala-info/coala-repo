# jbrowse2

## Overview
JBrowse 2 is a highly extensible, next-generation genome browser capable of visualizing complex structural variants, synteny, and 3D protein structures. This skill provides the procedural knowledge required to initialize JBrowse 2 environments and populate them with biological data using the command-line interface. It focuses on transforming raw genomic files into interactive visualizations through proper indexing and configuration.

## Core CLI Workflows

### Initialization
To start a new JBrowse 2 web instance in the current directory:
```bash
npx @jbrowse/cli create .
```

### Adding Reference Genomes
Genomes must be indexed before they can be used.
```bash
# Add a fasta reference
npx @jbrowse/cli add-assembly reference.fa --name hg38 --load copy
```

### Adding Data Tracks
JBrowse 2 supports various formats. Use the `add-track` command to register files in the `config.json`.

- **Alignments (BAM/CRAM):**
  ```bash
  npx @jbrowse/cli add-track alignments.bam --load copy
  ```
- **Variants (VCF):**
  ```bash
  npx @jbrowse/cli add-track variants.vcf.gz --load copy
  ```
- **Features (GFF3/BED):**
  ```bash
  npx @jbrowse/cli add-track features.gff3.gz --load copy
  ```
- **Quantitative Data (BigWig):**
  ```bash
  npx @jbrowse/cli add-track coverage.bw --load copy
  ```

### Text Searching
To enable searching for gene names or IDs, you must generate a text index:
```bash
npx @jbrowse/cli text-index
```

## Best Practices
- **Indexing:** Always ensure tabix-indexed files (.tbi) or CSI indexes are present in the same directory as your VCF or GFF files.
- **Compression:** Use bgzip for text-based formats (VCF, GFF, BED) to ensure compatibility with remote fetching and indexing.
- **URI Access:** For web deployments, you can point tracks to URLs instead of local files by using the `--location` flag with the specific URL.
- **Configuration Manual Edits:** While the CLI is preferred, complex views (like dotplots or circular views) may require manual adjustments to the `config.json` generated by the tool.
