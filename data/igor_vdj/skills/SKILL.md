# igor_vdj

## Overview
IGoR (Inference and Generation Of Repertoires) is a C++ framework designed to provide a detailed statistical characterization of the V(D)J recombination process. Unlike tools that simply annotate sequences, IGoR uses a probabilistic approach to infer the underlying distributions of V, D, and J gene usage, deletions, and non-templated insertions. It is particularly useful for calculating the generation probability ($P_{gen}$) of specific TCR/BCR sequences and for generating synthetic repertoires that follow realistic biological distributions.

## Installation and Setup
The most reliable way to install IGoR is via Conda:
```bash
conda install -c bioconda igor_vdj
```
To verify the installation and locate the default model data directory, use:
```bash
igor -datadir
```

## Common CLI Patterns and Workflows

### Chain Specification
IGoR supports both T-cell and B-cell chains. When running commands, you can specify the chain using the following identifiers:
- **T-cell:** `TRA` (or `alpha`), `TRB` (or `beta`)
- **B-cell:** `IGL` (or `lambda`), `IGK` (or `kappa`)

### Calculating Generation Probabilities (Pgen)
One of the most common tasks is calculating the probability that a specific sequence would be generated by the V(D)J recombination process. IGoR provides a dedicated script for this:
```bash
igor-compute_pgen --sequence <SEQUENCE> --chain <CHAIN_TYPE>
```

### Inference and Generation
IGoR's core functionality is split between inferring models from existing data and generating new sequences from those models.
- **Inference:** Used to learn the recombination parameters (probabilities of gene usage, junctional deletions, etc.) from a set of experimental sequences.
- **Generation:** Used to produce a synthetic repertoire based on a previously inferred or provided model.

### Working with Models
IGoR relies on model files that define the recombination process. 
- Use the `-datadir` flag to ensure the software can find the genomic templates and model parameters.
- If working with human TCR beta chains, ensure the model parameters file is clean of superfluous IMGT information to avoid parsing errors.

## Expert Tips and Troubleshooting
- **Memory and Performance:** IGoR is heavily object-oriented and designed for parallel architectures. When processing large datasets, ensure your environment supports multi-threading.
- **Segmentation Faults:** If you encounter a segfault during inference, verify that your genomic reference files match the chain type and species of your input data. Mismatched reference files are a common cause of crashes.
- **Non-productive Sequences:** When analyzing BCRs, be aware that IGoR can process both productive and non-productive sequences. High frequencies of non-productive sequences in your output may indicate specific biological selection pressures or sequencing artifacts.
- **Compiler Compatibility:** If building from source, IGoR has known issues with GCC versions greater than 7. It is recommended to use the Conda distribution to avoid these toolchain conflicts.
