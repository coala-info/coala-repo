# Partial ref-guided workflow - bwa mem only

## Overview

This workflow performs reference-guided alignment for RAD-seq data, specifically designed as a modular component of the [Stacks](http://catchenlab.life.illinois.edu/stacks/) pipeline. It takes demultiplexed FASTQ reads (provided as a dataset collection) and a reference genome in FASTA format to generate high-quality alignments.

The core processing is handled by [BWA-MEM2](https://github.com/bwa-mem2/bwa-mem2), which maps the reads to the provided reference. The resulting alignments are then processed through [Samtools view](http://www.htslib.org/doc/samtools-view.html) for filtering and [Samtools stats](http://www.htslib.org/doc/samtools-stats.html) to generate comprehensive mapping metrics.

The final outputs include filtered BAM files ready for downstream analysis in `gstacks`, along with visual quality control reports generated by [MultiQC](https://multiqc.info/). This workflow is a subset of the larger [reference-guided stacks workflow](https://workflowhub.eu/workflows/347) and is optimized for use on platforms like [Galaxy Australia](https://usegalaxy.org.au/).

Ensure your reference genome is in FASTA format and your demultiplexed reads are organized into a paired-end or single-end dataset collection of FASTQ files to enable efficient batch processing. Using collections instead of individual datasets is essential for this workflow to correctly map multiple samples and aggregate quality metrics through MultiQC. For automated execution, you can initialize your configuration using `planemo workflow_job_init` to generate a `job.yml` file. Refer to the README.md for comprehensive details on data preparation and specific parameter requirements for RAD-seq analysis.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Partial_ref-guided_workflow_bwa_mem_only.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Partial_ref-guided_workflow_bwa_mem_only.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Partial_ref-guided_workflow_bwa_mem_only.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Partial_ref-guided_workflow_bwa_mem_only.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Partial_ref-guided_workflow_bwa_mem_only.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
