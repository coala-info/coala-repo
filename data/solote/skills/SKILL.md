# solote

## Overview

SoloTE is a computational pipeline designed to quantify transposable element expression at single-cell resolution. It is particularly useful for researchers who have already processed scRNA-seq data (e.g., via STARsolo) and wish to extend their analysis beyond standard gene features to include TEs. The tool handles the re-annotation of alignments and produces count matrices that can be integrated into downstream analysis frameworks like Seurat or Scanpy.

## Installation

The most reliable way to install SoloTE and its dependencies (Samtools, BEDTools, R, Pysam, and Pandas) is via Bioconda:

```bash
conda install bioconda::solote
```

## Preparing TE Annotations

SoloTE requires a specific BED format for TE annotations. You can generate this using the included utility script.

### Using UCSC RepeatMasker Files
To download and convert annotations for standard genomes (e.g., hg38, mm10):

1. **List available genomes**:
   ```bash
   python SoloTE_RepeatMasker_to_BED.py -l
   ```
2. **Download and convert**:
   ```bash
   python SoloTE_RepeatMasker_to_BED.py -g hg38
   ```

### Custom Annotation Format
If using a custom TE library, the 4th column (ID) must follow this exact pipe-delimited and colon-delimited structure:
`sequenceName|startPosition|endPosition|TE_Subfamily:TE_Family:TE_Class|strand`

**Example entry:**
`chr1 11505 11675 chr1|11505|11675|L1MC5a:L1:LINE|25.1|- 25.1 -`

## Running the Pipeline

The main pipeline processes a BAM file (containing Cell Barcode and UMI tags) against the TE annotation BED file.

### Command Pattern
```bash
python SoloTE_pipeline.py \
    --threads <number_of_threads> \
    --bam <input_bam_file> \
    --teannotation <formatted_te_bed> \
    --outputprefix <prefix_string> \
    --outputdir <output_directory>
```

### Key Parameters
- `--bam`: Input BAM file. Ensure this file contains the `CB` (Cell Barcode) and `UB` (UMI) tags, typically generated by STARsolo or CellRanger.
- `--teannotation`: The BED file generated in the preparation step.
- `--outputdir`: The directory where results will be stored; the tool creates this if it does not exist.

## Expert Tips and Best Practices

- **BAM Compatibility**: SoloTE relies on specific tags. If your BAM lacks `CB`/`UB` tags, the pipeline will fail to assign reads to cells. Always verify your upstream aligner settings.
- **Memory Management**: For large datasets, ensure you allocate sufficient threads, but monitor memory usage as `pysam` and `pandas` operations on large BAMs can be resource-intensive.
- **Output Interpretation**: SoloTE generates two types of matrices:
    - **LocusTEs**: Quantifies TEs at specific genomic coordinates.
    - **LegacyTEs**: Aggregates counts at the subfamily/family level.
- **Downstream Integration**: The output `.mtx` files are standard Matrix Market format. Use `Read10X()` in Seurat or `scanpy.read_10x_mtx()` in Scanpy to load the results.
