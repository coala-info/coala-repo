# mlstdb

## Overview

`mlstdb` is a specialized utility designed to bridge the gap between the `mlst` command-line tool and the evolving online MLST repositories (PubMLST and BIGSdb Pasteur). It automates the OAuth2 authentication required by these APIs and provides a structured two-step workflow: fetching scheme metadata and then downloading/indexing the actual sequences. This skill ensures that sequence typing is performed against the most current allelic profiles and locus definitions.

## Core Workflow

### 1. Fetching Schemes
The first step identifies available schemes and generates a configuration file.

*   **Basic Fetch**: `mlstdb fetch -d pubmlst` (or `pasteur`)
*   **Targeted Filtering**: To avoid downloading unnecessary data (like cgMLST) or non-bacterial schemes, use filters:
    ```bash
    # Fetch only Staphylococcus schemes
    mlstdb fetch -d pubmlst --filter "Staphylococcus*"
    
    # Exclude core genome MLST to keep the database lean
    mlstdb fetch -d pubmlst --exclude "cgMLST"
    ```
*   **Authentication**: On the first run, the tool will prompt for client registration. Credentials are saved to `~/.config/mlstdb`.

### 2. Updating the Database
Once you have the `mlst_schemes_<db>.txt` file, use the `update` command to download the sequences and build the BLAST index.

*   **Standard Update**:
    ```bash
    mlstdb update -i mlst_schemes_pubmlst.txt -d ./mlst_data -b ./blast_db
    ```
*   **Customization**: You can manually edit the `.txt` or `.tab` file generated by the `fetch` step to include only specific rows before running `update`.

## Best Practices

*   **Backup First**: Always backup your existing `mlst` data directory before running an update, as `mlstdb` may overwrite existing schemes.
*   **Species Validation**: `mlst` is designed for bacterial species. Ensure your fetch filters or manual edits remove any viral or eukaryotic schemes downloaded from the APIs.
*   **Resuming Downloads**: If a large fetch operation is interrupted, use the `-r` or `--resume` flag to continue from the last processed scheme.
*   **Verification**: After updating, verify the new database by pointing `mlst` to the new directories:
    ```bash
    mlst --blastdb ./blast_db/mlst.fa --datadir ./mlst_data [fasta_files]
    ```

## Troubleshooting

*   **Dependency Errors**: If installing via Conda and encountering `rauth` version issues, ensure the `conda-forge` channel is enabled: `conda install -c conda-forge -c bioconda mlstdb`.
*   **Scheme Naming**: If the tool cannot determine a scheme name automatically, it will prompt for user input. Choosing "auto-generate" typically creates a name based on the URL (e.g., `borrelia`).
