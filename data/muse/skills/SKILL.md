# muse

## Overview
MuSE (Mutation Somatic Evolution) is a statistical tool designed to identify somatic point mutations by modeling the molecular evolution of the tumor and normal tissue. It utilizes a Markov substitution model to account for tumor heterogeneity, which improves the balance between sensitivity and specificity. The tool is optimized for multi-core environments and follows a two-step command-line process to move from raw alignments to a filtered VCF of somatic variants.

## CLI Usage and Patterns

### 1. MuSE call
The first step performs the initial mutation calling and generates a raw output file.

**Command Syntax:**
`./MuSE call -f <reference.fa> -O <output_prefix> -n <cores> <tumor.bam> <normal.bam>`

**Key Parameters:**
- `-f`: Path to the indexed reference genome FASTA file.
- `-O`: Prefix for the output file (creates `<prefix>.MuSE.txt`).
- `-n`: Number of parallel threads/cores.
- `<tumor.bam> <normal.bam>`: The paired alignment files.

### 2. MuSE sump
The second step applies a sample-specific error model to the output of the `call` step to produce the final VCF. You must choose between the Exome (`-E`) or Genome (`-G`) model.

**For Whole Exome Sequencing (WES):**
`./MuSE sump -I <prefix>.MuSE.txt -O <output.vcf> -E -n <cores> -D <dbsnp.vcf.gz>`

**For Whole Genome Sequencing (WGS):**
`./MuSE sump -I <prefix>.MuSE.txt -O <output.vcf> -G -n <cores> -D <dbsnp.vcf.gz>`

**Key Parameters:**
- `-I`: The input text file generated by the `MuSE call` step.
- `-O`: The name of the final output VCF file.
- `-E` or `-G`: Selects the appropriate error model for WES or WGS data.
- `-D`: Path to a bgzipped and tabix-indexed dbSNP VCF file.

## Expert Tips and Best Practices

- **Pre-processing Requirements**: BAM files must be aligned using BWA and should follow GATK Best Practices, specifically:
    - Marking duplicates.
    - Jointly realigning the tumor-normal pair.
    - Recalibrating base quality scores (BQSR).
- **Performance**: MuSE 2.0 is significantly faster than version 1.0. For WGS data, it is recommended to use a high core count (e.g., 60-80 cores) to complete the analysis in under an hour.
- **VCF Indexing**: Ensure the dbSNP VCF provided to the `sump` step is compressed with `bgzip` and indexed with `tabix`. The reference genome version must match across all inputs.
- **Output Interpretation**: The final VCF contains the somatic variants that passed the sample-specific error model filters.
