# padel

## Overview

The padel skill provides a framework for applying computer vision to padel game recordings. It transforms raw video into actionable data by tracking four players and the ball, projecting movements onto a 2D court plane, and classifying specific player actions. This is particularly useful for generating performance heatmaps, calculating player error rates, and providing data-driven insights for continuous improvement or live broadcast enhancements.

## Environment Setup

To use the padel analytics tool, ensure the environment is correctly configured:

1.  **Environment Creation**: Use Python 3.12.
    ```bash
    conda create -n padel_analytics python=3.12
    conda activate padel_analytics
    pip install -r requirements.txt
    ```
2.  **Dependencies**: Install PyTorch following the official guide for your specific hardware (CUDA support is highly recommended).
3.  **Model Weights**: Download the required model checkpoints and place them in the paths defined in your configuration.

## Configuration and Inference

The tool relies on a centralized configuration file for model paths and performance tuning.

### 1. Configure Model Paths
Before running inference, edit `config.py` to point to your local model weights:
*   Update player tracking model paths.
*   Update ball detection and pose estimation checkpoints.

### 2. Manage VRAM
The default settings require at least **8GB of VRAM**. If you encounter Out-Of-Memory (OOM) errors:
*   Open `config.py`.
*   Reduce the `batch_size` parameters for the trackers and classifiers.

### 3. Run Analytics
Execute the main processing script from the root directory:
```bash
python main.py
```

## Court Calibration (Keypoints)

Because the tool assumes a fixed camera setup, you must calibrate the court for homographic computations (mapping video pixels to 2D court coordinates).

*   **Selection UI**: A UI will pop up during execution asking for 12 unique court keypoints.
*   **Numeration**: You must follow the specific numbering sequence (1-12) defined in the `main.py` source code (lines 24-38).
*   **Accuracy**: Precise selection of these points is critical for accurate velocity and position data.

## Expert Tips

*   **Fixed Camera**: Ensure the camera does not move during the recording. Any camera shake or panning will invalidate the homography matrix generated by the initial keypoint selection.
*   **Stroke Classification**: The tool classifies 13 degrees of freedom for player poses. It is most effective at identifying distinct padel strokes like the *bandeja*, *backhand volley*, and *topspin smash*.
*   **Data Retrieval**: Use the `main.py` script to access the raw data collector if you need to export metrics to an external database for long-term player tracking.
