# nanoplot

## Overview

NanoPlot is a command-line tool designed to provide comprehensive visualization and statistical reporting for long-read sequencing datasets. It transforms raw sequencing data or alignments into an interactive HTML report and high-quality static images. Use this skill to assess the success of a sequencing run, identify potential data issues, and calculate key metrics like N50. It supports a wide variety of input formats and allows for extensive filtering and customization of the resulting plots.

## Core CLI Patterns

### Basic Data Processing
The most common use case is processing raw reads. Multiple files of the same type can be passed simultaneously.

*   **FASTQ files:**
    `NanoPlot --fastq reads.fastq.gz -o output_dir`
*   **BAM files (sorted):**
    `NanoPlot --bam alignment.bam -o output_dir`
*   **FASTA files:**
    `NanoPlot --fasta sequences.fasta -o output_dir`

### High-Speed Processing with Summary Files
For Oxford Nanopore data, using the `sequencing_summary.txt` file generated by basecallers (like Guppy or MinKNOW) is significantly faster than processing FASTQ files directly.

`NanoPlot --summary sequencing_summary.txt --loglength -o summary_plots`

### Filtering and Data Transformation
Refine the output by excluding outliers or focusing on specific data ranges.

*   **Length and Quality filters:**
    `NanoPlot --fastq reads.fastq.gz --maxlength 50000 --minqual 10`
*   **Logarithmic scaling:** Essential for long-read data where lengths span several orders of magnitude.
    `NanoPlot --fastq reads.fastq.gz --loglength`
*   **Downsampling:** Use for extremely large datasets to speed up plot generation.
    `NanoPlot --fastq reads.fastq.gz --downsample 10000`

### Customizing Visualizations
NanoPlot allows you to specify the types of bivariate plots and their aesthetics.

*   **Specify plot types:** Choose from `kde` (Kernel Density Estimate), `hex` (Hexagonal binning), or `dot`.
    `NanoPlot --fastq reads.fastq.gz --plots hex dot`
*   **Color and Format:**
    `NanoPlot --fastq reads.fastq.gz --color green --format pdf svg`

## Expert Tips

*   **Performance:** Always use the `-t` or `--threads` option to utilize multiple CPU cores, especially when processing large BAM or FASTQ files.
*   **N50 Visualization:** Use the `--N50` flag to explicitly show the N50 mark in the read length histogram, which is a critical metric for assembly quality.
*   **Memory Management:** If dealing with "huge" files that might exceed memory, use the `--huge` flag to trigger more memory-efficient processing.
*   **Persistence:** Use the `--store` flag to save extracted data into a pickle file. This allows you to re-generate plots later without re-parsing the original large data files.
*   **Alignment Metrics:** When using BAM files, use `--alength` to plot aligned read lengths instead of the total sequenced length, which helps in assessing the quality of the mapping.
