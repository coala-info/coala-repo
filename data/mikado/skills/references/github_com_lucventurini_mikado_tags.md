## [v2.0.1](/lucventurini/mikado/releases/tag/v2.0.1)

Toggle v2.0.1's commit message

```
Fix configure (EI-CoreBioinformatics#372)

* Automate version increase with bumpversion
* Fixed the configuration errors found through testing. Added manual testing of Mikado in the Snakefile.
* Fix mm metadata deprecation warnings
* Fix config arg parsing, small formatting change
* Now checking properly about output directories being strings of length at least 1
* Solved deprecation warning re: np.int
* Solved deprecation warning re: np.bool
* Updated Singularity/Docker images
* Updated the CHANGELOG
* Fixed the seed test
* Made the leniency parameter (in pick/blast_params) required for the configuration files.
* Now Mikado configure will be able to handle correctly external configuration files that are partial (e.g. because they are from an older version of Mikado).
* Code reorganisation to prevent any circular import
* Added the publishing workflow (triggered by new tag)
* Code cleanup (including changing some function signatures removing unused parameters), making the configuration objects checking straight away their validity with __post_init__, now Locus will not modify the configuration in-place but rather modify copies of the attributes, now configuration objects will also look for the scoring in the same directory of the configuration file, fixed the compiling of the requirements expressions, fixed dependent tests.
* Added sourcetrail files for easier navigation of the source code base

Co-authored-by: ljyanesm <yanes.luis@gmail.com>
```

Release Candidate 2.0rc5

* Now Mikado pick will use lightweight SQLite databases for inteprocess data exchange (EI-CoreBioinformatics#218). It could still be improved by allowing to remove more fragments.
