# nwkit table2nwk

[Jump to bottom](#wiki-pages-box)

Kenji Fukushima edited this page Mar 25, 2026
·
[3 revisions](/kfuku52/nwkit/wiki/nwkit-table2nwk/_history)

## Overview

`nwkit table2nwk` converts a tab-separated parent-child table back into a Newick tree.

The required columns are `branch_id` and `parent`.

## Basic example

### Command

```
nwkit table2nwk -i tree.tsv -o tree.nwk
```

If the table also contains `name`, `dist`, and `support`, those values are transferred to the output tree.

### Example input (`tree.tsv`)

| branch\_id | parent | name | dist | support |
| --- | --- | --- | --- | --- |
| 0 | -1 |  |  |  |
| 1 | 0 |  | 3 | 80 |
| 2 | 0 | C | 4 |  |
| 3 | 1 | A | 1 |  |
| 4 | 1 | B | 2 |  |

### Example output (`tree.nwk`)

```
((A:1,B:2)80:3,C:4);
```

This table describes the same tree as the `nwk2table` example: branch `1` is the internal branch joining `A` and `B`, and its `support` value becomes the internal support label in the output Newick.

## Output format selection

### Command

```
nwkit table2nwk -i tree.tsv -o tree.nwk \
  --outformat 1
```

When `--outformat auto` is used, NWKIT chooses a suitable Newick format based on whether the table defines internal support values or internal node names.
