# racon

## Overview
Racon is a standalone consensus module designed to correct raw contigs generated by rapid assembly methods. It functions as a polishing tool that takes raw reads and their alignments to a target (contigs or other reads) to generate a high-quality consensus sequence. It is highly optimized for speed and supports data from both Pacific Biosciences and Oxford Nanopore Technologies. Beyond polishing, Racon can also be utilized for read error correction.

## Installation
Racon is commonly available via Bioconda:
```bash
conda install bioconda::racon
```

## Core Usage Pattern
The basic command structure requires three positional arguments:
```bash
racon [options] <sequences> <overlaps> <target_sequences> > polished_assembly.fa
```
*   **`<sequences>`**: The raw reads used for correction (FASTA/FASTQ, can be gzipped).
*   **`<overlaps>`**: Alignments between the reads and the target (MHAP, PAF, or SAM format).
*   **`<target_sequences>`**: The sequences to be corrected, typically contigs (FASTA/FASTQ).

### Common CLI Options
*   `-t, --threads <int>`: Set the number of threads (default: 1).
*   `-u, --include-unpolished`: Output target sequences even if they were not polished.
*   `-f, --fragment-correction`: Perform read error correction instead of contig polishing. In this mode, the overlaps file should contain dual/self-overlaps.
*   `-w, --window-length <int>`: Size of the window for Partial Order Alignment (POA) (default: 500).
*   `-q, --quality-threshold <float>`: Threshold for average base quality of windows used in POA (default: 10.0).
*   `-e, --error-threshold <float>`: Maximum allowed error rate for filtering overlaps (default: 0.3).

### Scoring Parameters
Adjust these if working with specific sequencing chemistries or non-standard error profiles:
*   `-m, --match <int>`: Match score (default: 3).
*   `-x, --mismatch <int>`: Mismatch score (default: -5).
*   `-g, --gap <int>`: Gap penalty (default: -4, must be negative).

## Expert Tips and Best Practices
*   **Input Detection**: Racon automatically detects the type of input data (PacBio vs. Nanopore).
*   **Paired-end Reads**: Racon expects single-end reads. If using paired-end data, rename reads with unique names (up to the first whitespace) and join them into a single file before mapping.
*   **Memory Management**: For large datasets, use the `racon_wrapper` script. It allows you to use the `--split <int>` flag to process target sequences in smaller chunks (size in bytes) to reduce memory consumption.
*   **Subsampling**: If execution time is a bottleneck, `racon_wrapper` provides a `--subsample <ref_length> <coverage>` flag to reduce the sequence data to a specific coverage level.
*   **CUDA Acceleration**: If the binary was built with CUDA support, use `-c, --cudapoa-batches <int>` to enable GPU-accelerated polishing. Note that CUDA support for short-read polishing is generally considered experimental compared to long-read support.
*   **Trimming**: By default, Racon trims consensus at window ends. Use `--no-trimming` if you need to preserve the full length of the target sequences, though this may impact quality at the edges.
