[Release v1.1.0 - mlstdb purge and cleanup fixes](/MDU-PHL/mlstdb/releases/tag/v1.1.0)

Pre-release

Pre-release

The headline for this release is `mlstdb purge`, a new command that lets you remove schemes, STs, or individual alleles from your local database. This feature is designed for cleaning up contaminated sequences or correcting errors without having to wipe everything and start over. You can target specific entries, and the tool will handle orphaned alleles and rebuild the BLAST database for you. You can use `--config` to batch-purge across multiple schemes from a YAML file.

**Added**

* `mlstdb purge`: remove a whole scheme, a specific ST, or a single allele, with BLAST rebuild
* Batch purge via YAML config (`mlstdb purge -c purge_config.yaml`)

**Fixed**

* Incomplete scheme directories are removed before BLAST build when using `--no-auth` or after auth failures ([#29](https://github.com/MDU-PHL/mlstdb/issues/29))
* `{scheme_name}_info.json` now correctly writes `"locii"` instead of `"locus"` ([#28](https://github.com/MDU-PHL/mlstdb/issues/28)) Both changelogs are updated. The v1.1.0 entry now leads with `mlstdb purge` as the headline feature, followed by the two bug fixes.

## Release v1.0.0

### Summary

Major release introducing the streamlined `connect` + `update` workflow, replacing the previous `fetch`-first approach. The `update` command now works out of the box with a built-in curated scheme list of ~300 MLST schemes from both PubMLST and Pasteur.

### What's changed

**New features**

* `mlstdb connect` command for one-time OAuth credential setup
* Built-in curated scheme list: no `fetch` step needed for standard usage ([#10](https://github.com/MDU-PHL/mlstdb/issues/10))
* `--no-auth` flag for unauthenticated access to public APIs
* `--resume` flag to recover from interrupted downloads
* `--threads` for parallel downloads (both `fetch` and `update`)
* Session reuse and connection pooling for faster API calls
* Restrictive permissions (`0600`) on stored credential files

**Bug fixes**

* Fetch looping error at 76% ([#19](https://github.com/MDU-PHL/mlstdb/issues/19))
* Missing scheme URI resolution errors ([#18](https://github.com/MDU-PHL/mlstdb/issues/18))
* 401 errors on unregistered databases no longer crash the process, skipped schemes are reported at the end ([#17](https://github.com/MDU-PHL/mlstdb/issues/17))

**Breaking changes**

* `fetch` is now an advanced command; the recommended workflow is `connect` → `update` ([#25](https://github.com/MDU-PHL/mlstdb/issues/25))
* `update --input` is now optional (defaults to built-in scheme list)

**Documentation**

* Complete README rewrite focused on the two-command workflow
* New MkDocs documentation site with getting started guide, detailed command references, and troubleshooting

### How to test

```
pip install mlstdb==1.0.0
mlstdb connect --db pubmlst
mlstdb connect --db pasteur
mlstdb update
mlst --blastdb blast/mlst.fa --datadir pubmlst test.fasta
```

**Full Changelog**: [v0.2.0...v1.0.0](https://github.com/MDU-PHL/mlstdb/compare/v0.2.0...v1.0.0)

## What's Changed

### New Features

* **Add scheme metadata JSON file for each downloaded scheme**
  + Each scheme now includes a `<scheme>_info.json` file with name, allele count, last updated date, source database, and API URL
  + Improves traceability and provenance information for downloaded schemes

### Bug Fixes

* **Add newline to database\_version.txt for Unix tool compatibility**
  + Fixes issue where missing newline made file difficult to manipulate with Unix tools

### Documentation

* **Update installation instructions with recommended method and conda-forge channel**
  + Recommends creating conda environment with mlst first, then pip installing mlstdb
  + Adds conda-forge channel flag to resolve rauth dependency issues
  + Includes troubleshooting note for common installation errors
