# r-grimport

name: r-grimport
description: Importing and manipulating external vector graphics (PostScript) in R. Use this skill when you need to bring external images into R as vector objects that can be scaled, transformed, and integrated into the grid graphics system.

## Overview
The `grImport` package provides tools for importing external vector graphics into R. It primarily targets PostScript files, converting them into an intermediate XML format that R can represent as "Picture" objects. This allows for high-quality rendering of external logos, diagrams, or illustrations within R plots.

## Installation
To install the package from CRAN:
```R
install.packages("grImport")
```
Note: This package requires Ghostscript to be installed on your system to handle the conversion of PostScript files.

## Core Workflow
The standard process for importing an image involves three main steps:

1.  **Trace**: Use `PostScriptTrace()` to convert a PostScript (.ps) file into an XML file.
2.  **Read**: Use `readPicture()` to load that XML file into R as a `Picture` object.
3.  **Draw**: Use `grid.picture()` to render the image.

### Key Functions
- `PostScriptTrace(file, out)`: Invokes Ghostscript to parse a PostScript file and write an XML representation of its paths and colors to the `out` file.
- `readPicture(file)`: Reads the XML file generated by the trace and creates a `Picture` object in R.
- `grid.picture(picture, x, y, width, height)`: Draws the imported picture at the specified location and size using the `grid` graphics system.
- `pictureGrob(picture, ...)`: Creates a grid graphical object (grob) from the picture, which is useful for including the image in `ggplot2` or complex `grid` layouts.

## Usage Example
```R
library(grImport)

# 1. Trace the PostScript file (creates 'image.ps.xml')
PostScriptTrace("image.ps", "image.ps.xml")

# 2. Read the XML into R
img <- readPicture("image.ps.xml")

# 3. Draw the image
grid.newpage()
grid.picture(img)
```

## Tips and Best Practices
- **Ghostscript Path**: If `PostScriptTrace` fails, ensure Ghostscript is installed and its executable is in your system's PATH.
- **Format Conversion**: If your source image is not PostScript (e.g., it is a PDF or SVG), use the `grConvert` package or external tools to convert it to PostScript before using `grImport`.
- **SVG Alternative**: For modern SVG files (specifically those generated by Cairo), the `grImport2` package is often a more direct solution.
- **Scaling**: Because the imported objects are vector-based, they can be resized using the `width` and `height` arguments in `grid.picture` without any pixelation or loss of quality.
