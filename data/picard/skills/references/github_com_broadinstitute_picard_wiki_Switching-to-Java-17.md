## Java 17

Picard now requires Java 17 to both build and run, so you’ll need to install a version 17 JDK. I’d recommend using “homebrew” to install JDKs on Macs, and [jenv](https://www.jenv.be/) to manage Java versions on the command line. jenv is especially handy if you need to switch between projects that use different versions of Java.

We have tested Picard with the [Adoptium](https://adoptium.net/) JDK known as [temurin](https://projects.eclipse.org/projects/adoptium.temurin). To install temurin with brew, do:

`brew tap homebrew/cask-versions`

`brew install --cask temurin17`

## IntelliJ:

There are several settings in IntelliJ where you may need to make changes.

### 1) Project structure:

Go to File/Project Structure. Make sure the "SDKs" section includes your version 17 JDK:

![Screen Shot 2023-02-14 at 2 21 59 PM](https://user-images.githubusercontent.com/10062863/218836702-e627c950-97ef-4442-b9af-9ec9b4e7e961.png)

Then choose “Project” under “Project Settings”. You’ll need to update both “SDK” (point to your local Java 17 JDK), and “Language Level” (choose "17 - Sealed types, always strict floating point semantics"):

![Screen Shot 2023-02-14 at 2 20 29 PM](https://user-images.githubusercontent.com/10062863/218836110-a2b50b8c-e328-4c1c-b460-329a3be7819b.png)

### 2) Java plugin:

Go to Preferences/Build, Execution, Deployment/Compiler/Java Compiler:

![Screen Shot 2023-02-14 at 2 14 13 PM](https://user-images.githubusercontent.com/10062863/218834860-6e6826cf-159c-422e-9fa3-f875232bdf52.png)

Set the "project bytecode version" to 17.

### 3) Gradle plugin:

Go to Preferences/Build, Execution, Deployment/Build Tools/Gradle. Update the “Gradle JVM” to use a Java 17 JVM. If you don’t do this, you may get errors when refreshing build.gradle.

![Screen Shot 2023-02-14 at 2 15 51 PM](https://user-images.githubusercontent.com/10062863/218835212-3e1f6163-a609-4da1-b4aa-5dc12cbbc105.png)

Make sure to refresh gradle in the gradle tab after making these changes.
