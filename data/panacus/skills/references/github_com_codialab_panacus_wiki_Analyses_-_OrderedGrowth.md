# OrderedGrowth section

This section contains a plot showing pangenome growth for a single specific order of the paths. For calculating the exact pangenome growth (over all orders) see [!Growth](./Analyses%3A-%21Growth) instead.

In `report.yaml`:

```
- !OrderedGrowth
  count_type: Bp
  order: ./order.txt
  coverage: 1,1,2
  quorum: 0,0.9,0
```

where `count_type` can be `Bp`, `Node`, `Edge`. Due to the fact that the ordered calculation requires more memory than the exact one, it is currently not possible to directly use `All` for `count_type` (unlike in all other cases).

`order` names the text file that contains the order of the paths, one path per line.

For `coverage` and `quorum` see the description in [!Growth](./Analyses%3A-%21Growth).
