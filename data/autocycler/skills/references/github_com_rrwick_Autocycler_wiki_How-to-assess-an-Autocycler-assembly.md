# How to assess an Autocycler assembly

[Jump to bottom](#wiki-pages-box)

Ryan Wick edited this page Dec 20, 2024
·
[10 revisions](/rrwick/Autocycler/wiki/How-to-assess-an-Autocycler-assembly/_history)

How do you know if your Autocycler assembly went well or not? This page offers a few approaches.

## Autocycler combine output

The [Autocycler combine](/rrwick/Autocycler/wiki/Autocycler-combine) command produces a final assembly by combining all of the clusters. Hopefully, each cluster [resolved](/rrwick/Autocycler/wiki/Autocycler-resolve) to a single consentig, in which case it will print this at the end of its stderr output:

```
Consensus assembly is fully resolved 😄
```

If, however, one or more clusters did not resolve to a single sequence, you'll see this message instead:

```
One or more clusters failed to fully resolve 😟
```

## View assembly graph in Bandage

|  |  |
| --- | --- |
| If you load the `consensus_assembly.gfa` file in [Bandage](https://github.com/rrwick/Bandage), you will hopefully see something like this. Note that each component of the graph has only one sequence, and each sequence is coloured blue, indicating that it's an Autocycler consentig. | However, if things didn't go as well, you might see something like this. Note that one component of the graph has multiple sequences where non-consentigs are coloured orange. |
| ![Bandage assessment - good](https://github.com/rrwick/Autocycler/wiki/images/assessment/good.png) | ![Bandage assessment - bad](https://github.com/rrwick/Autocycler/wiki/images/assessment/bad.png) |

## Check metrics

The [Metrics](/rrwick/Autocycler/wiki/Metrics) page describes all of the assembly metrics generated by Autocycler, but for assessment purposes, the most useful is likely the `consensus_assembly_fully_resolved` metric, which can be `true` or `false`. You can find this metric in the `consensus_assembly.yaml` file made by [Autocycler combine](/rrwick/Autocycler/wiki/Autocycler-combine) in your Autocycler output directory.

Alternatively, you can use [Autocycler table](/rrwick/Autocycler/wiki/Autocycler-table) to access the metrics (including `consensus_assembly_fully_resolved`). If you only have one Autocycler assembly to assess, these commands will build a simple one-row TSV with some key metrics:

```
autocycler table > metrics.tsv
autocycler table -a autocycler_out >> metrics.tsv
```
