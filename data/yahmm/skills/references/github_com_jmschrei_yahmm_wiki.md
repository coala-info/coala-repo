Its major features include an easy to use object model, simple to use silent states, and the ability to specify any distribution type for each individual state, instead of being forced to use the same distribution type for every state. YAHMM is on pip, which makes installing it a breeze. Simply type in pip install yahmm and you're good to go! Lets dive in with the Wikipedia HMM example describing what Bob does on rainy and sunny days. The proposed graph structure has two hidden states, Rainy and Sunny, and three emissions, Walk, Shop, and Clean. &gt;&gt;&gt; from yahmm import * &gt;&gt;&gt; # Initialize the model object &gt;&gt;&gt; model = Model( name="Rainy-Sunny" ) &gt;&gt;&gt; # Initialize the two hidden states, with an appropriate discrete distribution &gt;&gt;&gt; rainy = State( DiscreteDistribution({ 'walk': 0.1, 'shop': 0.4, 'clean': 0.5 }), name='Rainy' ) &gt;&gt;&gt; sunny = State( DiscreteDistribution({ 'walk': 0.6, 'shop': 0.3, 'clean': 0.1 }), name='Sunny' ) &gt;&gt;&gt; # Add the states to the model &gt;&gt;&gt; model.add_state( rainy ) &gt;&gt;&gt; model.add_state( sunny ) &gt;&gt;&gt; # Now add the two transitions from the start of the model to the hidden states &gt;&gt;&gt; model.add_transition( model.start, rainy, 0.6 ) &gt;&gt;&gt; model.add_transition( model.start, sunny, 0.4 ) &gt;&gt;&gt; # Add the transitions from the hidden states to each other &gt;&gt;&gt; model.add_transition( rainy, rainy, 0.7 ) &gt;&gt;&gt; model.add_transition( rainy, sunny, 0.3 ) &gt;&gt;&gt; model.add_transition( sunny, rainy, 0.4 ) &gt;&gt;&gt; model.add_transition( sunny, sunny, 0.6 ) &gt;&gt;&gt; # Finalize the model structure &gt;&gt;&gt; model.bake( verbose=True ) Creating all models looks similar to this, in that you first initialize the model, then add the states to the model, and then add the transitions. In an infinite HMM such as this, there do not need to be transitions to an explicit end state, but in a finite HMM as we'll see next, you can add transitions to the end. Lastly, you must bake the model, which converts the internal structure from a networkx graph to the sparse matrices that allow for constant time access. Some useful things the model can do now are calculate the forward matrix, backward matrix, forward-backward emission and transition matrices, calculate the viterbi or maximum a posteriori path, or train on sequences. &gt;&gt;&gt; Now lets try using a small, continuous distribution HMM which models ionic current readings from a source which bifurcates between two levels. We could just model the ionic current means, but lets also model the duration using an independent multivariate distribution. &gt;&gt;&gt; from yahmm import * &gt;&gt;&gt; model = Model( "stuff" ) &gt;&gt;&gt; high_d = MultivariateDistribution( [ NormalDistribution( 25, 0.5 ), ExponentialDistribution( 2 ) ] ) &gt;&gt;&gt; low_d = MultivariateDistribution( [ NormalDistribution( 10, 0.5 ), ExponentialDistribution( 0.1 ) ] ) &gt;&gt;&gt; s1 = State( high_d, name="High" ) &gt;&gt;&gt; s2 = State( low_d, name="Low" ) &gt;&gt;&gt; model.add_state( s1 ) &gt;&gt;&gt; model.add_state( s2 ) &gt;&gt;&gt; model.add_transition( model.start, s1, 0.9 ) &gt;&gt;&gt; model.add_transition( model.start, s2, 0.1 ) &gt;&gt;&gt; model.add_transition( s1, s1, 0.3 ) &gt;&gt;&gt; model.add_transition( s1, s2, 0.7 ) &gt;&gt;&gt; model.add_transition( s2, s1, 0.2 ) &gt;&gt;&gt; model.add_transition( s2, s2, 0.8 ) &gt;&gt;&gt; &gt;&gt;&gt; seq = model.sample() &gt;&gt;&gt; print seq [[25.59142420969153, 0.08755865379162744], [10.885243661269064, 13.814628095045833], [10.377273315125588, 10.705500898626891], [10.90843360756541, 28.384614691170952], [24.173988910212547, 0.31915020852750725], [25.396986380950644, 1.2089825964484309]] &gt;&gt;&gt; print model.log_probability( seq ) -26.8297631449 &gt;&gt;&gt; model.train( [seq], algorithm='baum-welch' ) Training improvement: 5.97434667321 Training improvement: -7.1054273576e-15 Total Training Improvement: 5.97434667321 &gt;&gt;&gt; print model.log_probability( seq ) -20.8554164717 Here we perform Baum-Welch training on a randomly generated sequence, and see an improvement in the model fitting the sequence.
