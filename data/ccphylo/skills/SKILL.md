# ccphylo

## Overview
CCPhylo is a specialized tool designed for rapid phylogenetic inference. It processes nucleotide overlap data (typically output from KMA) to calculate genetic distances and reconstruct evolutionary relationships. It is optimized for large-scale genomic epidemiology, allowing for the efficient creation of phylogenetic trees from raw sequence comparisons without requiring full multiple sequence alignments.

## Usage Guidelines

### Input Requirements
- **Primary Input**: CCPhylo accepts `.mat` files or overlap files generated by KMA.
- **Argument Order**: Provide input files as non-option arguments at the end of the command line string.

### Common Command Patterns
- **Distance Matrix Generation**: Use ccphylo to compute distance matrices from KMA results.
- **Tree Reconstruction**: Generate Newick-formatted trees using various algorithms (e.g., Neighbor-Joining or UPGMA).

### Best Practices
- **Integration with KMA**: Ensure that KMA has been run with the appropriate settings to output the overlap data required by ccphylo.
- **Memory Management**: For very large datasets, monitor system resources as distance matrix calculations scale with the number of samples.
- **Output Handling**: Redirect standard output to a file (e.g., `> tree.nwk`) when generating phylogenetic trees to save the results for downstream visualization.
