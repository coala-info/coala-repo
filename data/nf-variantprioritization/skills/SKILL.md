## Overview
nf-core/variantprioritization automates the clinical interpretation of genomic variants in cancer research. It bridges the gap between raw variant calls and actionable clinical insights by integrating somatic and germline annotation frameworks to identify clinically relevant mutations, mutational signatures, and tumor mutational burden.

The pipeline transforms VCF files and copy number segment data into structured reports. It is specifically designed to handle outputs from upstream callers such as Mutect2, Strelka, DeepVariant, and HaplotypeCaller, providing a standardized path from variant calling to precision medicine reporting.

## Data preparation
The pipeline requires a comma-separated samplesheet provided via the `--input` parameter. This file defines the relationship between patients, samples, and their respective variant files.

The samplesheet must contain the following columns:
- `patient`: Unique identifier for the patient/subject.
- `status`: Integer indicating sample type (0 for Normal, 1 for Tumor).
- `sample`: Unique identifier for the specific sample.
- `vcf`: Full path to the SNV/InDel VCF file (.vcf or .vcf.gz).
- `cna`: Full path to the copy number segment file (.cns or .txt); required for every entry if `--cna_analysis true` is set.

Example `samplesheet.csv`:
```csv
patient,status,sample,vcf,cna
HCC1395,1,HCC1395T,HCC1395T_vs_HCC1395N.mutect2.vcf.gz,HCC1395T.segments.txt
HCC1395,0,HCC1395N,HCC1395N.deepvariant.vcf.gz,
```

Reference data for PCGR and CPSR is required. By default, the pipeline will attempt to download the reference bundle (`--pcgr_download true`), but users can provide a local path using `--pcgr_database_dir`.

## How to run
Run the pipeline using the `nextflow run` command. Specify a profile (e.g., `docker` or `singularity`) and provide the input samplesheet and output directory.

```bash
nextflow run nf-core/variantprioritization \
   -profile docker \
   -r 1.0.0 \
   --input samplesheet.csv \
   --outdir ./results
```

Key parameters include:
- `--genome`: The reference genome assembly (e.g., `GRCh38`).
- `--tumor_only`: Set to `true` for somatic interpretation without a matched normal sample.
- `--cna_analysis`: Enable copy-number alteration reporting.
- `--panel_id`: Specify CPSR virtual panel identifiers (e.g., `0` for exploratory, or specific PanelApp IDs).
- `-resume`: Use this to restart a pipeline from the last cached step if it was interrupted.

Note: The `-profile conda` option is currently not supported for the PCGR and CPSR modules.

## Outputs
Results are saved in the directory specified by `--outdir`. The primary deliverables include:

- **PCGR Reports**: Clinical interpretation reports for somatic variants in HTML, JSON, and Tab-separated formats.
- **CPSR Reports**: Germline cancer predisposition reports in HTML and JSON formats.
- **Annotated Variants**: VCF and MAF files containing functional annotations from VEP and other databases.
- **MultiQC**: A summary report aggregating statistics and quality control metrics across all samples.

For a detailed description of all output files, refer to the [official output documentation](https://nf-co.re/variantprioritization/output).
