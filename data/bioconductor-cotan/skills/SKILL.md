# bioconductor-cotan

## Overview

COTAN (COexpression Tables ANalysis) is a statistical tool for analyzing single-cell RNA-seq data. Unlike methods that rely on data normalization, COTAN uses a contingency table approach based on the presence or absence of gene transcripts. It models the zero-expression probability to identify gene-gene interactions (COEX) and uses the Global Differentiation Index (GDI) to identify specifically expressed genes. It is particularly effective for identifying cell states and ensuring cluster uniformity.

## Core Workflow

### 1. Initialization and Metadata
Initialize the COTAN object from a raw count matrix (UMI counts).

```r
library(COTAN)
library(zeallot) # For multi-assignment %<-%

# Create object
obj <- COTAN(raw = raw_counts)
obj <- initializeMetaDataset(obj, 
                             GEO = "GEO_ID", 
                             sequencingMethod = "Drop_seq", 
                             sampleCondition = "condition_name")

# Set logging for transparency
setLoggingLevel(1L)
```

### 2. Data Cleaning Pipeline
COTAN provides specific plots to identify low-quality cells, doublets, and mitochondrial contamination.

*   **Visual Inspection:** Use `cellSizePlot(obj)`, `genesSizePlot(obj)`, and `mitochondrialPercentagePlot(obj)`.
*   **Filtering:** Use `dropGenesCells()` to remove outliers.
*   **Parameter Estimation:** Run `clean(obj)` after every filtering step to re-estimate parameters.

```r
# Example: Drop cells with high mitochondrial percentage
c(mitPlot, mitSizes) %<-% mitochondrialPercentagePlot(obj, genePrefix = "^Mt")
cellsToRem <- rownames(mitSizes)[mitSizes[["mit.percentage"]] > 1.0]
obj <- dropGenesCells(obj, cells = cellsToRem)

# Re-run cleaning to estimate nu (efficiency) and other parameters
obj <- clean(obj)
```

### 3. COEX Analysis
Calculate the co-expression matrix. This is the most computationally intensive step.

```r
obj <- proceedToCoex(obj, 
                     calcCoex = TRUE, 
                     optimizeForSpeed = TRUE, 
                     cores = 6L, 
                     saveObj = FALSE)
```

### 4. Gene Differentiation and Heatmaps
Identify marker genes using GDI and visualize relationships.

*   **GDI:** Values > 1.43 typically indicate specifically expressed genes (potential markers).
*   **Heatmaps:** Visualize COEX between gene sets.

```r
# Calculate GDI
gdiDF <- calculateGDI(obj)
obj <- storeGDI(obj, genesGDI = gdiDF)

# Plot GDI with specific markers highlighted
GDIPlot(obj, genes = list("Markers" = c("Satb2", "Bcl11b")))

# COEX Heatmap
genesHeatmapPlot(obj, primaryMarkers = c("Satb2", "Vim"), symmetric = TRUE)
```

### 5. Uniform Clustering
COTAN defines clusters based on "uniformity"—where gene expression within a cluster satisfies the null hypothesis (independence).

*   **`cellsUniformClustering`:** Splits data into uniform clusters.
*   **`mergeUniformCellsClusters`:** Merges clusters that remain uniform when combined.
*   **`DEAOnClusters`:** Performs Differential Expression Analysis to get COEX values for clusters.

```r
# Perform DEA on existing clusters
cluster_coex <- DEAOnClusters(obj, clusters = my_clusters)
obj <- addClusterization(obj, clName = "my_cl", clusters = my_clusters, coexDF = cluster_coex)

# Summary of clusters (cell counts, efficiency, expressed genes)
c(summaryData, summaryPlot) %<-% clustersSummaryPlot(obj, clName = "my_cl")
```

## Tips for Success
*   **Torch Support:** COTAN can use the `torch` library for GPU acceleration. Check availability with `COTAN::canUseTorch(TRUE, "cuda")`.
*   **Efficiency (nu):** The `nu` parameter represents cell efficiency. In the PCA plot generated by `cleanPlots(obj)`, `nu` should not correlate strongly with the main principal components.
*   **B-Group Cells:** During cleaning, COTAN often identifies a "B group" in PCA (often enriched for hemoglobin genes). These are typically removed to improve signal.
