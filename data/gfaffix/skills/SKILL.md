# gfaffix

## Overview
gfaffix is a specialized tool for pangenomics designed to simplify variation graphs. It identifies nodes that share identical sequences at their beginnings or ends (affixes) and collapses them into a single representation. Unlike simple graph simplification, gfaffix is "walk-preserving," meaning it ensures that the actual genomic sequences (paths or walks) represented in the graph remain identical after the transformation. This process reduces graph complexity and redundancy, making the data more manageable for downstream alignment and analysis.

## Installation
The most reliable way to install gfaffix is via Bioconda:
```bash
mamba install -c conda-forge -c bioconda gfaffix
```

## Common CLI Patterns

### Basic Graph Refinement
To collapse a graph and output the refined version to a new file:
```bash
gfaffix input.gfa -o refined.gfa
```

### Refinement with Verification
It is highly recommended to use the check flag to verify that the transformed graph spells out the exact same sequences as the original:
```bash
gfaffix -c input.gfa -o refined.gfa
```

### Tracking Node Transformations
When collapsing nodes, you often need to know how the original nodes map to the new structure. Use the `-t` option to generate a transformation report:
```bash
gfaffix input.gfa -o refined.gfa -t transformation.tsv
```

### Excluding Specific Paths from Collapsing
If you have specific reference paths or walks (e.g., primary chromosomes) that should remain untouched, use a regular expression with the `-x` flag:
```bash
# Prevents collapsing nodes on paths starting with "chr"
gfaffix input.gfa -x "^chr" -o refined.gfa
```

### High-Performance Execution
For large pangenome graphs, utilize multiple threads:
```bash
gfaffix -p 8 input.gfa.gz -o refined.gfa.gz
```

## Expert Tips and Best Practices

*   **Compressed I/O**: gfaffix natively supports Gzipped GFA files. Always provide `.gz` extensions for both input and output to save disk space and reduce I/O overhead.
*   **The Transformation File**: The file generated by `-t` is critical if you intend to lift over coordinates or annotations from the original graph to the refined graph. It maps original nodes to their corresponding walks in the new structure.
*   **Affix Reporting**: If you are researching graph topology or redundancy patterns, use `-a affixes.txt` to get a detailed report of every shared affix identified before they are collapsed.
*   **Memory Management**: While gfaffix is efficient, very large graphs with high redundancy can be memory-intensive. Ensure your environment has sufficient RAM relative to the number of nodes and edges in the GFA.
*   **Validation**: Always run with `-c` at least once on a new dataset to ensure the tool's logic holds for your specific graph topology.
