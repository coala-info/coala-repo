# dastk

## Overview
DAStk (Differential ATAC-seq Toolkit) is a command-line suite designed to identify transcription factor (TF) activity shifts between biological states. By analyzing the spatial distribution of TF motifs within open chromatin regions, it calculates a "Motif Displacement score" (MD-score). This allows researchers to pinpoint which TFs are most affected by perturbations—such as drug treatments or genetic knockouts—without requiring factor-specific ChIP-seq data. While optimized for ATAC-seq, it is compatible with any BedGraph-formatted data representing regions of interest, including PRO-seq or ChIP-seq peaks.

## Installation
The toolkit is available via Bioconda or PyPI:
```bash
conda install bioconda::dastk
# OR
pip install DAStk
```

## Standard Workflow

### 1. Calculate MD-Scores
Process each biological condition separately to generate the initial motif displacement statistics.
```bash
process_atac -e <PEAKS_FILE> -m <MOTIF_SITES_DIR> -g <GENOME_CODE> -o <OUTPUT_DIR>
```
*   **-e**: Input peaks in BedGraph or tab-delimited format.
*   **-m**: Directory containing putative binding sites (one file per TF motif).
*   **-g**: Genome abbreviation (e.g., `hg38`, `mm10`).

### 2. Differential Analysis
Compare two processed conditions to identify statistically significant changes in TF activity.
```bash
differential_md_score -1 <MD_FILE_1> -2 <MD_FILE_2> -p 0.0000001 -m "Control" -n "Treated" -b -o <OUTPUT_DIR>
```
*   **-1 / -2**: The `*_md_score.txt` files generated in step 1.
*   **-p**: P-value threshold for significance.
*   **-b**: Flag to automatically generate barcode plots.

### 3. Downstream TF Relations
Analyze the relationships and commonalities among the significant TFs identified.
```bash
tf_result_explanations -p 0.0000001 -d <DIFF_OUTPUT_FILE> -o <EXPLANATION_FILE>
```

## Tool-Specific Best Practices

### Critical File Sorting
All input files—both the ATAC-seq peaks and the motif site files—**must** be sorted using the same criteria. Inconsistent lexical sorting (e.g., one file ordering `chr10` after `chr1` and another ordering `chr2` after `chr1`) will cause the tool to produce incorrect results or fail.

### Motif Site Requirements
*   Motif files should be in BedGraph-like format (tab-delimited).
*   Expected extensions: `.bed`, `.BedGraph`, or `.txt`.
*   Files are typically sourced from databases like HOCOMOCO.

### Visualization Tips
*   **Label Limits**: When using `ma_plot` or `barcode_plot`, labels are truncated at 19 characters to maintain plot scaling. Use concise names for the `-m` and `-n` arguments.
*   **Customizing Plots**: If the default plots generated by `differential_md_score` are insufficient, use the standalone plotting modules (`ma_plot`, `barcode_plot`, `tf_intersect`) to adjust p-value thresholds or labels without re-running the full analysis.
*   **Comparing Multiple Conditions**: Use `tf_intersect` to generate Venn diagrams (for 3 or fewer conditions) or Upset plots (for more than 3) to find common significant motifs across multiple experiments.

### MD-Score Interpretation
*   **Raw MD-score**: Values > 0.1 generally indicate motif enrichment relative to background.
*   **Differential MD-score**: Represents the difference in displacement between two conditions. Ensure you do not mix raw MD-score files and differential stats files when using the `tf_intersect` tool.
