# matchtigs

## Overview
The `matchtigs` tool implements algorithms to reduce the redundancy of k-mer sets by finding efficient paths through a de Bruijn graph. It is primarily used to compress genomic data into "matchtigs" (which allow k-mer repetitions for better compression) or "Eulertigs" (which represent each k-mer exactly once). This skill helps you select the right algorithm based on your memory constraints and compression needs, and provides the correct CLI syntax for processing unitig files.

## Core Workflows

### Input Preparation
Matchtigs requires unitigs as input. For maximum performance, use unitigs generated by **GGCAT** (with the `-e` flag) or **BCALM2**.
- **Fast Loading**: Use `--bcalm-in` instead of `--fa-in` when your input comes from GGCAT or BCALM2 to leverage existing annotations.
- **Formats**: Supports FASTA (`.fa`, `.fasta`) and GFA (`.gfa`). Gzip compression (`.gz`) is automatically handled.

### Algorithm Selection
1.  **Greedy Matchtigs (Recommended)**: Near-optimal compression with significantly lower resource requirements. Use for most practical applications.
2.  **Eulertigs**: Use when you need a representation without any k-mer repetitions (unique k-mers only).
3.  **Optimal Matchtigs**: Requires significant memory ($O(|V|^2)$) and the Blossom V library. Generally not feasible for large datasets.
4.  **Pathtigs**: A heuristic approach for Eulertigs, similar to ProphAsm.

### Common CLI Patterns

**Compute Greedy Matchtigs from FASTA**
```bash
matchtigs --fa-in unitigs.fa --greedytigs-gfa-out greedy-matchtigs.gfa
```

**Compute Eulertigs from BCALM2/GGCAT Output**
```bash
matchtigs --bcalm-in unitigs.fa --eulertigs-fa-out eulertigs.fa --eulertigs-gfa-out eulertigs.gfa
```

**Multi-threaded Execution**
Matchtigs supports multi-threading for greedy matchtig and matchtig computation. Use the `--threads` flag to specify the count.

## Expert Tips
- **Memory Management**: If the tool crashes or runs out of memory while attempting `--matchtigs-out`, switch to `--greedytigs-out`. The compression difference is usually negligible, but the memory savings are massive.
- **GFA vs FASTA**: Use GFA output if you need to preserve the graph structure (though matchtigs often outputs GFA without full topology). Use FASTA for simple sequence analysis.
- **Duplicate Handling**: If your application requires unique k-mers but you are using a format that might contain duplicates, matchtigs can output (ASCII-) bitvectors to track duplicate k-mers.
