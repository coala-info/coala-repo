# SARS-CoV-2 Illumina Amplicon pipeline - iVar based

## Overview

This workflow provides an [iVar](https://github.com/andersen-lab/ivar) based pipeline for analyzing SARS-CoV-2 Illumina amplicon sequencing data. It is designed for rapid variant calling, annotation, and lineage assignment, following established protocols used by organizations like [COG UK](https://www.cogconsortium.uk/) and [CanCOGeN](https://www.genomecanada.ca/en/cancogen). The pipeline is particularly suited for identifying majority variants and classifying samples using standard nomenclature.

The analysis begins with raw paired-end reads, which are preprocessed using `fastp` and mapped to a reference genome with `BWA-MEM`. It utilizes `ivar trim` for primer removal based on a provided BED file, followed by variant calling and consensus genome generation. Variants are further annotated using `SnpEff` to provide biological context for the identified mutations.

Final outputs include lineage and clade assignments generated by [Pangolin](https://github.com/cov-lineages/pangolin) and [Nextclade](https://github.com/nextstrain/nextclade). The workflow also produces comprehensive quality control reports via `MultiQC` and `QualiMap`, alongside a combined multi-FASTA file of all consensus genomes for downstream phylogenetic analysis.

For detailed information on input requirements and configuration, please refer to the [README.md](README.md) in the Files section. This workflow is released under the MIT license and is tagged for COVID-19 and ARTIC-based research.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Paired read collection for samples | data_collection_input | FASTQ format Illumina Reads (Amplicon Protocol) |
| 1 | Reference FASTA | data_input | SARS-CoV-2 reference genome (typically MN908947.3) |
| 2 | Primer BED | data_input | Primer BED file (from ARTIC project or similar) |
| 3 | Read fraction to call variant | parameter_input | Specify the proportion of reads that need to agree with each other to call a variant. This is a floating point value between 0 and 1. |
| 4 | Minimum quality score to call base | parameter_input | Minimum base quality score to count a base towards the sequence consensus. |
| 5 | Version of pangolin-data to use | parameter_input | You can select any pangolin-data version set up on this Galaxy server that should be used at the pangolin lineage assignment step. If you don't see any options to select from, this means the server doesn't offer any pangolin-data versions. In this case or if you don't want to bother with version selection, leave this input blank and the workflow will use pangolin-data version 1.37 bundled with pangolin. |


To run this pipeline, organize your paired-end Illumina reads into a paired dataset collection and ensure your reference genome is in FASTA format and primer scheme in BED format. You should verify that the primer BED file matches the specific ARTIC version used during library preparation to ensure accurate iVar trimming. For large-scale batch processing, you can automate the setup using `planemo workflow_job_init` to generate a `job.yml` file. Consult the README.md for comprehensive details on parameter settings for variant calling and lineage assignment. Refer to the Galaxy Training Network for additional guidance on preparing SARS-CoV-2 ampliconic data.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run pe-wgs-ivar-analysis.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run pe-wgs-ivar-analysis.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run pe-wgs-ivar-analysis.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init pe-wgs-ivar-analysis.ga -o job.yml`
- Lint: `planemo workflow_lint pe-wgs-ivar-analysis.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
