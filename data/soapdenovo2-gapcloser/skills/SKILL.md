# soapdenovo2-gapcloser

## Overview
GapCloser is a post-processing utility specifically designed to reduce the number of unknown bases (Ns) in genomic scaffolds. It works by mapping short reads back to the scaffold sequences and using the local assembly of those reads to replace gap regions with actual nucleotide sequences. While originally part of the SOAPdenovo2 suite, it can be applied to scaffolds generated by various assembly tools.

## Configuration File Setup
GapCloser requires a configuration file (usually named `config.txt`) to define the input read libraries. This is the same format used by SOAPdenovo2.

- **Library Section**: Define `[LIB]` for each library.
- **Read Paths**: Use `q1=` and `q2=` for paired-end fastq files, or `f1=` and `f2=` for fasta.
- **Insert Size**: Specify `avg_ins=` (average insert size).
- **Reverse Sequence**: Use `reverse_seq=0` (standard) or `1` (if reads need RC).
- **Rank**: Use `rank=` to determine the order in which libraries are used for gap filling.

## Command Line Usage
The basic execution pattern is:
`GapCloser -b config.txt -a scaffolds.fasta -o output_filled.fasta -t thread_count`

### Key Parameters
- `-b`: Path to the library configuration file.
- `-a`: Input scaffold file (FASTA format).
- `-o`: Output file name for the gap-filled scaffolds.
- `-t`: Number of CPU threads (GapCloser is multi-threaded).
- `-p`: Overlap parameter (default is 25, usually sufficient for short reads).

## Best Practices
- **Read Quality**: Always perform adapter trimming and quality filtering on reads before running GapCloser to prevent mis-assemblies within gaps.
- **Memory Management**: GapCloser loads the read index into RAM. Ensure the available memory is roughly 1.5x to 2x the size of the input read set.
- **Library Selection**: Use short-insert libraries (200-500bp) for the most effective gap filling, as they provide higher coverage and better local assembly resolution.
- **Iterative Filling**: While GapCloser is typically run once, if significant gaps remain, ensure the `avg_ins` in the config file accurately reflects the library prep, as incorrect insert sizes will lead to failed mapping across gaps.
- **Validation**: After running GapCloser, check the output log to see how many gaps were closed versus how many remain. A successful run should significantly decrease the "N" count in the FASTA file.
