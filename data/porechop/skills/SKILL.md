# porechop

## Overview
Porechop is a specialized tool designed for the "cleaning" phase of Oxford Nanopore Technologies (ONT) data processing. It performs three primary functions: trimming adapters from the ends of reads, identifying and splitting chimeric reads where an adapter appears in the middle of a sequence, and demultiplexing reads based on barcodes. It is particularly effective because it performs thorough alignments to find adapters even at low sequence identity, which is common in raw Nanopore data.

## Installation
The most reliable way to install Porechop is via Bioconda:
```bash
conda install -c bioconda porechop
```

## Common CLI Patterns

### Basic Adapter Trimming
To remove adapters from a single FASTQ or FASTA file:
```bash
porechop -i input_reads.fastq.gz -o trimmed_reads.fastq.gz
```

### Barcode Demultiplexing
To sort reads into separate folders based on their barcodes:
```bash
porechop -i input_reads.fastq.gz -b output_directory
```
*Note: Porechop will create subdirectories within the output directory for each detected barcode (e.g., BC01, BC02).*

### High-Performance Processing
For large datasets, always specify the number of CPU threads to speed up the alignment process:
```bash
porechop -i input.fastq.gz -o output.fastq.gz --threads 16
```

### Processing Albacore Output
Porechop can process an entire directory of FASTQ files generated by the Albacore basecaller:
```bash
porechop -i albacore_output_dir -b demux_dir
```

## Expert Tips and Best Practices

### Handling Chimeras
Porechop splits reads when it finds an adapter in the middle. If you are working with very long reads and find that Porechop is being too aggressive (splitting reads that aren't actually chimeric), increase the middle threshold:
- Default: `--middle_threshold 85`
- More conservative: `--middle_threshold 90`
- More aggressive: `--middle_threshold 75`

### Sensitivity vs. Specificity
- **End Trimming**: The default `--end_threshold` is 75. This is intentionally low because losing a few bases at the end of a long Nanopore read is usually preferable to leaving adapter sequence behind.
- **Internal Adapters**: The default `--middle_threshold` is 85. This is higher than the end threshold because splitting a read incorrectly is more disruptive to downstream assembly than failing to trim a few bases.

### Read Length Filtering
When splitting chimeric reads, Porechop may create small fragments. Use `--min_split_read_size` to discard fragments that are too short for your specific analysis (default is 1000 bp).

### Verification
Use `--verbosity 2` to get detailed information about which adapters were found and how many reads were trimmed or split. This is useful for troubleshooting samples with low recovery or unexpected adapter contamination.
