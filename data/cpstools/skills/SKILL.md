# cpstools

## Overview

CPStools is a bioinformatics suite designed specifically for the comparative analysis and quality control of chloroplast genomes. It streamlines the processing of GenBank and FASTA files to extract structural information, verify annotation accuracy, and perform population-level statistics. Use this skill to automate the extraction of shared coding regions, calculate codon usage bias, or standardize the starting position of chloroplast sequences for downstream comparative genomics.

## Common CLI Patterns

### Annotation and Statistics
*   **Validate GenBank files**: Check gene names, product labels, and codon start/stop/internal integrity.
    ```bash
    cpstools gbcheck -i input.gb
    ```
*   **Comparative check**: Compare a test file against a reference GenBank file.
    ```bash
    cpstools gbcheck -i test.gb -r reference.gb
    ```
*   **Gene statistics**: Get a summary of gene types and counts.
    ```bash
    cpstools info -i input.gb
    ```

### Structural Analysis
*   **Identify IR regions**: Locate the four main regions of the chloroplast (LSC, SSC, and two IRs). IRs are identified if they exceed 1,000 bp.
    ```bash
    cpstools IR -i input.gb
    ```
*   **Adjust sequence orientation**: Standardize sequences to start at the 1st bp of the LSC region or adjust SSC direction. This requires an info file generated by the `IR` command.
    ```bash
    # Modes: LSC (start at LSC), SSC (forward SSC), RP (reverse complement + LSC start)
    cpstools Seq -d work_dir -f info.txt -m LSC
    ```

### Comparative Genomics
*   **Nucleotide Diversity (Pi)**: Extract shared gene/intergenic regions and calculate Pi values. Requires `mafft` for alignment.
    ```bash
    cpstools Pi -d input_directory
    ```
*   **Codon Usage (RSCU)**: Calculate Relative Synonymous Codon Usage for shared coding regions.
    ```bash
    cpstools RSCU -d input_directory -l 300
    ```
*   **Phylogenetic Preparation**: Extract and merge shared CDS or protein sequences from multiple GenBank files.
    ```bash
    cpstools phy -d input_directory -m cds
    ```

### Repeat Identification
*   **SSRs**: Identify Simple Sequence Repeats with custom k-mer lengths.
    ```bash
    cpstools SSRs -i input.gb -k 10,5,4,3,3,3
    ```
*   **LSRs**: Sort and locate Long Sequence Repeats based on output from the `Reputer` tool.
    ```bash
    cpstools LSRs -i reputer_output.txt
    ```

### Format Conversion
*   **Batch convert**: Transform GenBank files into other formats for visualization or submission.
    ```bash
    # Modes: fasta, mVISTA, tbl
    cpstools converse -d input_dir -m mVISTA
    ```

## Expert Tips and Best Practices

*   **Environment Setup**: Ensure `mafft` is in your system PATH if running the `Pi` subcommand. If it is not, you must specify the path using the `-m` flag.
*   **Sequence Standardization**: Always run the `IR` command before `Seq`. The `info.txt` output from `IR` is a mandatory input for the `Seq` tool to correctly identify region boundaries.
*   **Data Preparation**: When using subcommands that take a directory (`-d`), ensure the directory contains only the relevant GenBank (.gb) or FASTA files to avoid processing errors.
*   **Annotation Quality**: Use `gbcheck` early in your workflow. Many downstream tools (like `Pi` and `phy`) rely on consistent gene naming across all input files to identify "shared" regions.
