# tandem

## Overview
TANDEM is a real-time monocular tracking and dense mapping framework developed by TUM-Vision. It bridges the gap between classical visual odometry and deep learning-based multi-view stereo. The system performs pose estimation using photometric bundle adjustment on a sliding window of keyframes, while a global model is built incrementally from dense depth predictions. These depth maps are generated by the CVA-MVSNet architecture and fused into a consistent global map represented as a Truncated Signed Distance Function (TSDF) voxel grid.

## Usage and Best Practices

### Environment Configuration
Before running TANDEM components, ensure the data directory is correctly exported. This variable is used by the system to locate datasets and model weights.
```bash
export TANDEM_DATA_DIR=/path/to/your/data
```

### Tracking and SLAM
The core tracking functionality is implemented in C++. For standard benchmarks like EuroC, use the provided entry-point scripts.
- **EuroC Dataset**: Execute the tracking pipeline using `bash tracking_euroc.sh`.
- **Custom Data**: To run on custom sequences, you must generate `scale.txt` and `tuples_dso_optimization_windows.txt` files to define the optimization windows and scale initialization.

### Dense Depth Prediction (CVA-MVSNet)
The depth prediction component is located in the `cva_mvsnet/` directory.
- **Model Export**: Use the provided utility to prepare models for inference:
  ```bash
  python cva_mvsnet/export_model.py
  ```
- **Training Data**: The framework utilizes Replica training data for the MVSNet component.

### Deployment via Docker
For environments with complex dependency requirements (C++, CUDA, Python), utilize the official TANDEM Docker image to ensure a consistent runtime environment for both the tracking front-end and the MVSNet back-end.

### Troubleshooting Common Issues
- **Scale Handling**: Monocular SLAM is scale-ambiguous. Ensure proper initialization or provide a `scale.txt` if absolute scale is required for mapping.
- **Fusion Options**: When working with scenes of varying depth ranges, adjust the TSDF fusion options to prevent artifacts in the global voxel grid.
- **Compilation**: The project uses CMake. If encountering "undefined reference" errors during build (e.g., `pthread`), ensure the linker flags are correctly set in the environment.
