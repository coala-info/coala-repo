# Changelog

## 0.26.0 (2026-01-04)

* [:issue:`135`](#id2): Fix GRCh38 coordinate of a CYP17A1 variant (thanks [@NeiH4207](https://github.com/NeiH4207)).
* Add the `fileformat` field to the VCF header when writing VCF files.
* [:issue:`150`](#id4): Fix bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id6) method when overlapping samples exist between the input VCF and reference panel, but the window contains only a single position (thanks [@toddknutson](https://github.com/toddknutson)).
* Add new gene MT-RNR1.

## 0.25.0 (2024-06-16)

* Add 26 new genes: ACYP2, ADRA2A, ADRB2, ANKK1, APOE, ATM, BCHE, BDNF, DBH, DRD2, F2, GRIK1, GRIK4, GRIN2B, HTR1A, HTR2A, IFNL4, ITGB3, ITPA, OPRK1, OPRM1, RARG, SLC6A4, SLC28A3, SLC47A2, UGT1A6.

## 0.24.0 (2024-03-31)

* Change G6PD haplotype name from `*MALE` to `MALE`, as the gene does not utilize star allele nomenclature.
* Update G6PD haplotype nomenclature in accordance with the latest CPIC version (2024-01-07).
* [:issue:`119`](#id10): Add G6PD phenotype data in accordance with the latest CPIC version (2024-01-07).
* Update CYP4F2 haplotype nomenclature in accordance with the latest PharmVar version (2024-01-27).

## 0.23.0 (2023-12-24)

* [:issue:`115`](#id13): Add new genes COMT and MTHFR (thanks [@nbiesot](https://github.com/nbiesot)).
* [:issue:`117`](#id15): Fix major bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id17) method. The bug was introduced in version 0.22.0 ([:issue:`113`](#id19)) and carries significant implications, especially if the user's input VCF contains sample names that overlap with the reference panel or if there are differences in chromosome annotation compared to the reference panel (e.g. 'chr22' vs. '22').

## 0.22.0 (2023-12-11)

* [:issue:`100`](#id22): Add new method [:meth:`sdk.utils.get\_bundle\_path`](#id24) to enable customization of the `pypgx-bundle` directory's location instead of the user's home directory.
* [:issue:`114`](#id26): Fix bug in [:meth:`api.core.get\_recommendation`](#id28) method where string `'None'` was treated as missing value by `pandas.read_csv` version 2.0 or higher.
* [:issue:`113`](#id30): Fix bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id32) method where Beagle's expectation-maximization algorithm estimated a parameter value that was outside the permitted range.

## 0.21.0 (2023-08-25)

* [:issue:`96`](#id35): Fix bug in [:command:`compute-copy-number`](#id37) command where it threw an error "Different sample sets found" when the sample name consists of only numbers with one or more leading zeros.
* [:issue:`97`](#id39): Fix bug in IFNL3 genotyping where certain variants were not considered because they were located outside of the target region.
* [:issue:`99`](#id41): Refresh the CPIC table for [:meth:`api.core.load\_cpic\_table`](#id43) method. The table's latest update was on February 6, 2022.
* [:issue:`63`](#id45): Fix bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id47) method where Beagle throws an error "Window has only one position" even when multiple overlapping variants exist.

## 0.20.0 (2023-01-12)

* [:issue:`73`](#id50): Fix bug in [:command:`run-ngs-pipeline`](#id52) command where empty VCF (i.e. no variants were found in the target gene) was causing error when plotting allele fraction. From now on, a warning will be produced telling users to specify `--do-not-plot-allele-fraction` to suppress this warning.
* [:issue:`78`](#id54): Fix bug in [:command:`compute-copy-number`](#id56) command where it threw an error "Different sample sets found" when the sample name consists of only numbers.

## 0.19.0 (2022-09-13)

* Add new method [:meth:`api.core.has\_sv`](#id59).
* Update [:meth:`api.core.sort\_alleles`](#id61) method to handle `'Indeterminate'` haplotype call when `by='name'`.
* Update [:meth:`api.utils.estimate\_phase\_beagle`](#id63) method to handle situations where there are overlapping samples between input VCF and reference panel -- i.e. users are no longer required to change sample names. Before this update, the Beagle program would throw an error if there were overlapping samples (e.g. 1KGP samples). From now on, offending samples will be temporarily renamed before statistical phasing.
* Add new methods [:meth:`api.core.load\_recommendation\_table`](#id65) and [:meth:`api.core.get\_recommendation`](#id67).

## 0.18.0 (2022-08-12)

* PyPGx now has a citation! Please refer to the publication "[ClinPharmSeq: A targeted sequencing panel for clinical pharmacogenetics implementation](https://doi.org/10.1371/journal.pone.0272129)" by Lee et al., 2022 (Steven is the first author). Fore more details, see the Citation section in README.
* Update phenotype data and star allele nomenclature for SLCO1B1 in accordance with the latest PharmVar version (v5.2.1). Note that SLCO1B1 was first formally added to PharmVar v5.1 on October 14, 2021. For more details, please refer to the publication "[PharmVar GeneFocus: SLCO1B1](https://doi.org/10.1002/cpt.2705)" by Ramsey et al., 2022 (Steven is a co-author of this paper by the way) and the change log in [the PharmVar SLCO1B1 page](https://www.pharmvar.org/gene/SLCO1B1). The PharmVar-developed SLCO1B1 nomenclature has been incorporated by CPIC 2022 guideline on statin-associated musculoskeletal symptoms.
* Remove duplicate DYPD entry from `phenotype-table.csv` file (i.e. Poor Metabolizer).
* Fix major bug in [:command:`run-chip-pipeline`](#id70) command where `--impute` argument is essentially ignored.
* [:issue:`68`](#id72): Fix bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id74) method when there are no overlapping variants between input VCF and reference panel.
* [:issue:`68`](#id76): Update [:meth:`api.utils.estimate\_phase\_beagle`](#id78) method to warn when statistical phasing is skipped.
* [:issue:`68`](#id80): Upgrade Beagle version from v5.2 (beagle.28Jun21.220.jar) to v5.4 (beagle.22Jul22.46e.jar) due to a bug in v5.2.
* [:issue:`68`](#id82): Update [:meth:`api.utils.estimate\_phase\_beagle`](#id84) method to filter out variants with improper allele ('I', 'D', 'N', '.'). Note that this issue is specific to chip data.
* [:issue:`68`](#id86): Update [:meth:`api.utils.import\_variants`](#id88) method to handle input VCF with duplicate variants. Basically, it will warn the user about it and and only keep the first record. This issue seems to occur frequently with chip data.

## 0.17.0 (2022-07-12)

* [:issue:`63`](#id91): Fix bug in [:meth:`api.utils.estimate\_phase\_beagle`](#id93) method when there is only one variant in input VCF and Beagle throws an error.
* Update [:command:`compare-genotypes`](#id95) command to print the entire discordant calls when `--verbose` is used.
* Update [:command:`compute-copy-number`](#id97) command to ensure that the samples in CovFrame[ReadDepth] and SampleTable[Statistics] are in the same order.
* [:issue:`64`](#id99): Update [:meth:`api.utils.import\_variants`](#id101) method to 'diploidize' the input VCF when the target gene is G6PD. This is because some variant callers output haploid genotypes for males for the X chromosome, interfering with downstream analyses.
* Remove unnecessary optional argument `assembly` from [:meth:`api.core.get\_ref\_allele`](#id103).

## 0.16.0 (2022-06-08)

* Add new optional argument `--comparison-table` to [:command:`train-cnv-caller`](#id106) and [:command:`test-cnv-caller`](#id108) commands.
* Update [:meth:`sdk.utils.add\_cn\_samples`](#id110) and [:meth:`sdk.utils.simulate\_copy\_number`](#id112) methods to check input files more rigorously.
* Update [:meth:`api.utils.test\_cnv\_caller`](#id114) and [:meth:`api.utils.train\_cnv\_caller`](#id1
