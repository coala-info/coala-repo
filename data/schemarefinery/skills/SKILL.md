# schemarefinery

## Overview
SchemaRefinery is a specialized toolkit designed to improve the quality and reliability of genomic schemas used in multilocus sequence typing (MLST). Unlike basic schema generation tools, it focuses on the "refinement" phase—ensuring that the loci included in a schema are biologically relevant, non-redundant, and properly annotated. It is particularly useful for bioinformaticians developing core-genome (cgMLST) or whole-genome (wgMLST) schemas who need to filter out problematic sequences that could lead to inaccurate subtyping.

## Installation and Setup
The tool is primarily distributed via Bioconda. It is recommended to use a dedicated Conda environment to manage dependencies.

```bash
# Install via Bioconda
conda install bioconda::schemarefinery

# Verify installation
schemarefinery --version
```

## Core Modules and CLI Usage
SchemaRefinery operates through a modular command-line interface. Each module handles a specific stage of the schema refinement workflow.

### 1. Assembly Acquisition
Use the `DownloadAssemblies` module to gather the genomic data required for schema development.
*   **Key Parameter**: `assembly_source` (specifies where to fetch the genomes).
*   **Usage Pattern**: `schemarefinery DownloadAssemblies [options]`

### 2. Quality Control and Loci Filtering
The primary strength of the toolkit lies in identifying loci that should be excluded from a high-quality schema.
*   **IdentifyParalogousLoci**: Detects loci that appear in multiple copies within a genome, which can cause ambiguity in allelic calling.
*   **Spurious Loci**: Identifies and flags loci that do not meet the stability or conservation requirements for a standardized schema.

### 3. Schema Annotation
Use the `AnnotateSchema` module to add functional metadata to the identified loci, facilitating better biological interpretation of the typing results.

### 4. Finalizing Schema Structure
Once recommendations for removal or modification are generated, they must be applied to the physical schema files.
*   **ApplyRecommendations**: (Formerly `CreateSchemaStructure` in versions < 0.5.0). This module takes the output from the identification modules and generates the final, refined schema directory.
*   **AdaptLoci**: Used to transform sets of loci into standardized formats compatible with common MLST software.

## Expert Tips and Best Practices
*   **Version Awareness**: If you are using version 0.5.0 or later, ensure you use the `ApplyRecommendations` command. Older documentation may refer to this as `CreateSchemaStructure`.
*   **Workflow Sequencing**: Always run `IdentifyParalogousLoci` before `ApplyRecommendations`. The latter relies on the recommendations file generated by the identification modules.
*   **Paralog Filtering**: When building cgMLST schemas, be aggressive with paralog filtering. Including paralogous loci often leads to "missing data" or false recombinational signals in downstream phylogenetic analysis.
*   **Assembly Selection**: Use the `assembly_source` options carefully to ensure your training set for the schema is representative of the genetic diversity of the target species.
