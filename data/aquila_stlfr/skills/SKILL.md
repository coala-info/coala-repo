# aquila_stlfr

## Overview
Aquila_stLFR is a specialized genomic tool designed for the reconstruction of diploid human genomes using stLFR technology. It transforms linked-read data into haplotype-resolved assemblies by leveraging barcode information to phase variants and perform targeted local assemblies. This skill should be used when you need to generate high-quality, phased contigs (haplotype 1 and haplotype 2) and detect assembly-based variants from stLFR sequencing runs.

## Installation and Setup
The most efficient way to install Aquila_stLFR is via Bioconda:
```bash
conda install bioconda::aquila_stlfr
```

### Required External Resources
Before running the pipeline, you must obtain the human reference genome and the uniqueness map:
- **Reference (hg38):** `wget https://zenodo.org/record/7689958/files/source.tar.gz`
- **Uniqueness Map (hg38):** `wget https://zenodo.org/record/7689958/files/Uniqness_map_hg38.tar.gz`

## Core Workflow

### 1. Preprocessing
Input FASTQ files must contain the `BX:Z:barcode` tag in the header. If your data is not formatted correctly, use the preprocessing script:
```bash
Aquila_stLFR_fastq_preprocess.py --fastq_file input.fastq --out_dir processed_data
```
You also need a BAM file (aligned via BWA-MEM) and a VCF file (generated by a caller like FreeBayes) as starting materials for the phasing step.

### 2. Step 1: Phasing and Fragment Generation
This step performs the initial phasing and generates fragment files for each chromosome.
```bash
Aquila_stLFR_step1.py \
  --fastq_file S12878.fastq \
  --bam_file S12878.bam \
  --vcf_file S12878_freebayes.vcf \
  --sample_name S12878 \
  --uniq_map_dir ./Uniqness_map_hg38 \
  --out_dir Assembly_results_S12878
```
**Expert Tips for Step 1:**
- **Memory Management:** For a 90X coverage human sample, Chromosome 1 requires ~350GB of RAM. Ensure you are running on a high-memory node.
- **Parallelization:** Use `--chr_start` and `--chr_end` to process specific chromosomes on different nodes to save time. For example, `--chr_start 21 --chr_end 21` targets only chromosome 21.

### 3. Step 2: Local Assembly
This step uses SPAdes to perform local assemblies of the phased chunks.
```bash
Aquila_stLFR_step2.py \
  --out_dir Assembly_results_S12878 \
  --reference ./source/ref.fa \
  --num_threads 30 \
  --num_threads_spades 5
```
**Expert Tips for Step 2:**
- **Thread Tuning:** `--num_threads` controls how many chunks are assembled simultaneously, while `--num_threads_spades` controls the threads used by the SPAdes assembler for each chunk.
- **Output:** The final contigs are found in `Assembly_Contigs_files/Aquila_contig.fasta`.

## Common CLI Patterns

### Hybrid Assembly
If working with hybrid linked-reads, use the hybrid-specific entry point:
```bash
Aquila_step1_hybrid --fastq_file <file> --bam_file <file> --vcf_file <file> [options]
```

### Cleaning Up
To remove intermediate files and save disk space after a successful run:
```bash
Aquila_stLFR_clean --out_dir Assembly_results_S12878
```

## Troubleshooting and Best Practices
- **Barcode Format:** Ensure barcodes are in the `BX:Z` format. If the tool fails to recognize fragments, the barcode header is the most likely culprit.
- **Reference Consistency:** Always ensure the reference fasta used in Step 2 matches the one used to generate the BAM and VCF files in Step 1.
- **Thresholds:** If assembly continuity is poor, consider adjusting `--block_threshold` (default 200kb) or `--block_len_use` (default 100kb) in Step 1 to change how phase blocks are handled.
