# bioconductor-dewseq

## Overview

DEWSeq is a Bioconductor package designed to identify RBP binding sites from iCLIP and eCLIP data. It employs a sliding window approach to analyze truncation counts (crosslink sites) at single-nucleotide resolution. By extending the DESeq2 framework, it performs one-sided significance testing to find regions significantly enriched in IP samples over SMI controls, while accounting for the dependencies between overlapping windows through a specialized Bonferroni correction.

## Typical Workflow

### 1. Data Import
DEWSeq requires a count matrix (windows vs. samples) and an annotation data frame. These are typically generated by the `htseq-clip` python pipeline.

```r
library(DEWSeq)
library(data.table)

# Load counts and annotation
countData <- fread("counts.txt.gz")
annotationData <- fread("annotation.txt.gz")

# Define sample metadata (IP vs SMI)
colData <- data.frame(
  row.names = colnames(countData)[-1],
  type = factor(c("IP", "IP", "SMI"), levels = c("IP", "SMI"))
)

# Create DESeqDataSet
ddw <- DESeqDataSetFromSlidingWindows(countData = countData,
                                      colData = colData,
                                      annotObj = annotationData,
                                      tidy = TRUE,
                                      design = ~type)
```

### 2. Normalization and Filtering
Normalization is critical. It is often recommended to estimate size factors based on protein-coding genes or non-significant windows to avoid bias from highly enriched RBP targets.

```r
# Standard normalization
ddw <- estimateSizeFactors(ddw)

# Optional: Filter low-count windows
keep <- rowSums(counts(ddw)) >= 10
ddw <- ddw[keep,]

# Optional: Filter based on max count in any single position (requires maxCountFile)
ddw <- filterCounts(ddw, maxCountFile = "max_counts.txt.gz", countThresh = 5, nsamples = 2)
```

### 3. Dispersion Estimation
Choosing the correct fit (parametric vs. local) is important for CLIP data.

```r
# Compare fits
parametric_ddw <- estimateDispersions(ddw, fitType = "parametric")
local_ddw <- estimateDispersions(ddw, fitType = "local")

# Use residuals to decide the better fit
# Usually, CLIP data benefits from fitType = "local"
ddw <- local_ddw 
```

### 4. Significance Testing
The `resultsDEWSeq` function performs the Wald test and applies a correction for overlapping windows.

```r
# Run Wald test
ddw <- nbinomWaldTest(ddw)

# Get results with sliding window correction
resultWindows <- resultsDEWSeq(ddw,
                               contrast = c("type", "IP", "SMI"),
                               tidy = TRUE)

# Optional: Use IHW for p-value adjustment
library(IHW)
resultWindows$p_adj_IHW <- adj_pvalues(ihw(pSlidingWindows ~ baseMean, 
                                           data = resultWindows, 
                                           alpha = 0.05))
```

### 5. Extracting Binding Regions
Combine significant overlapping windows into contiguous binding regions.

```r
resultRegions <- extractRegions(windowRes = resultWindows,
                                padjCol = "p_adj_IHW",
                                padjThresh = 0.01,
                                log2FoldChangeThresh = 0.5)

# Export to BED for visualization
toBED(windowRes = resultWindows,
      regionRes = resultRegions,
      fileName  = "binding_sites.bed")
```

## Key Functions

- `DESeqDataSetFromSlidingWindows()`: Constructor for the DEWSeq object.
- `filterCounts()`: Filters windows based on the maximum crosslink count at any single position within the window.
- `resultsDEWSeq()`: Performs the differential analysis and applies the specific p-value correction for overlapping windows.
- `extractRegions()`: Merges significant windows into genomic regions.
- `toBED()`: Helper to export results for genome browsers.

## Tips for Success

- **Input Controls**: Always use Size-Matched Input (SMI) controls. DEWSeq is optimized to find enrichment over this specific background.
- **One-sided Testing**: DEWSeq defaults to a right-tailed test (IP > SMI).
- **Window Parameters**: Typical settings are window size 50nt and step size 20nt, but these should be adjusted based on the RBP's expected binding footprint.
- **Normalization**: If the IP is highly enriched, standard `estimateSizeFactors` may over-normalize. Consider using a subset of "stable" features (e.g., mRNAs or non-significant windows) to calculate size factors.
