# shiba2sashimi

## Overview
The `shiba2sashimi` utility transforms the output of Shiba (a computational method for identifying differential RNA splicing) into Sashimi plots. These plots are essential for validating and presenting RNA-seq data, as they provide a clear visual representation of isoform-specific expression and splice junction usage. Use this skill when you need to visualize specific splicing events (like skipped exons) across different experimental groups or samples using data processed through the Shiba pipeline.

## Core Usage Patterns

### Basic Plotting by Event ID
The most common way to generate a plot is by using the unique positional ID generated by Shiba.
```bash
shiba2sashimi -e experiment_table.tsv -s /path/to/shiba_workdir/ -o output_plot.png --id "SE@chr2@157561213-157561293@157560260-157561542"
```

### Plotting by Genomic Coordinates
If you want to visualize a specific region rather than a predefined Shiba event, use the coordinate flag.
```bash
shiba2sashimi -e experiment_table.tsv -s /path/to/shiba_workdir/ -o output_plot.png -c "chr2:157560000-157562000"
```

### Grouping and Comparison
To compare specific experimental conditions, use the `--groups` and `--colors` flags. This aggregates samples based on the groups defined in your experiment table.
```bash
shiba2sashimi -e experiment_table.tsv -s /path/to/shiba_workdir/ -o comparison.png --groups "Control,Treatment" --colors "blue,red"
```

## Expert Tips and Best Practices

- **Smoothing Coverage**: For noisy RNA-seq data, increase the `--smoothing_window_size` (default is 21). A higher value results in a smoother coverage profile, which is often better for publication.
- **Managing Junction Clutter**: If a plot is overwhelmed by low-frequency junction arcs, use `--minimum_junc_reads` to filter out junctions with few supporting reads.
- **Visual Refinement**:
    - Use `--width` to adjust the aspect ratio of the figure (default is 8).
    - Use `--extend_up` and `--extend_down` (default 500bp) to add genomic context around an event when plotting by ID.
    - Use `--nolabel` and `--nojunc` for minimalist plots where only coverage is required.
- **Environment Setup**: Ensure `pysam` and `matplotlib` are available in the environment. If using Conda, the `bioconda` and `conda-forge` channels are required.
- **Input Verification**: The `-s` (shiba) argument must point to the Shiba **working directory**, not a specific file. The tool expects to find the standard Shiba output structure within that directory.
