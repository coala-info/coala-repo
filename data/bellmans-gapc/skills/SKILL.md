# bellmans-gapc

## Overview
Bellman's GAP (GAPc) provides a high-level declarative language to describe dynamic programming algorithms by separating the search space (grammar) from the scoring schemes (algebras). This skill assists in writing `.gap` files, compiling them into efficient C++ code, and executing the resulting binaries for complex sequence analysis tasks.

## Core Workflow
1.  **Define the Grammar**: Create a `.gap` file defining the recursive structure of the problem.
2.  **Define Algebras**: Specify how to evaluate or score the candidates generated by the grammar.
3.  **Compile**: Use `gapc` to translate the high-level description into optimized C++.
4.  **Execute**: Compile the generated C++ code with a standard compiler (like `g++`) to create the final tool.

## CLI Usage Patterns
- **Basic Compilation**:
  `gapc program.gap`
  This generates `program.cpp` and `program.hh`.

- **Specifying Output**:
  `gapc -o my_algorithm program.gap`

- **Optimization Levels**:
  Use the `-O` flag to enable compiler optimizations for the generated code:
  `gapc -O program.gap`

- **Generating C++ Code**:
  After running `gapc`, compile the resulting source:
  `g++ -O3 program.cpp -o program_executable`

## Expert Tips
- **Separation of Concerns**: Always keep your `grammar` (the "what") distinct from your `algebra` (the "how much"). This allows you to swap scoring functions (e.g., switching from "minimum energy" to "counting solutions") without changing the underlying logic.
- **Signature Matching**: Ensure that the signature defined in your `.gap` file matches the functions implemented in your algebras; `gapc` is strict about type and argument consistency.
- **Bioconda Environment**: Since `bellmans-gapc` is distributed via Bioconda, ensure your environment is active to avoid path issues with the compiler's standard libraries.
