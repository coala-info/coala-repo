# perl-exporter-tidy

## Overview

Exporter::Tidy provides a lightweight and flexible alternative to the standard Perl Exporter. Rather than using inheritance and package globals, it works by exporting a custom `import` method into your module's namespace. This allows for a more declarative style of defining exports and gives consumers of your module fine-grained control over how symbols are imported, including the ability to prefix imported names to avoid collisions.

## Usage Patterns

### Defining Exports in Your Module

To set up exports, pass a hash-like list to `Exporter::Tidy`. Keys represent tags (groups of symbols), and values are array references containing the symbol names.

```perl
package MyModule;
use Exporter::Tidy 
    default => [ qw(func1 func2) ],
    utils   => [ qw(debug log) ],
    vars    => [ qw($VERSION %CONFIG) ];
```

### Special Tags and Keys

- **default**: Symbols exported when the user calls `use MyModule;` without arguments.
- **all**: Automatically generated by the module to include every exportable symbol unless you define it manually.
- **_map**: A hash reference used to rename symbols or define them on the spot.
- **_prefix**: Used by the *importer* to prepend a string to imported symbols.

### Advanced Mapping and On-the-fly Definitions

Use the `_map` key to alias internal names to different export names or to define subroutines directly within the exporter configuration.

```perl
use Exporter::Tidy 
    _map => {
        alias_name => 'internal_name',
        get_time   => sub { time() },
        $app_state => \$internal_state
    };
```

## Best Practices for Importers

### Avoiding Name Clashes with Prefixes

One of the most powerful features of Exporter::Tidy is the `_prefix` option available during import. This prevents namespace pollution in the calling package.

```perl
# Imports func1 as my_func1 and func2 as my_func2
use MyModule _prefix => 'my_', qw(func1 func2);
```

### Selective Importing

When a module uses Exporter::Tidy, specifying any import list disables the `:default` tag. If you want both specific symbols and the defaults, you must request `:default` explicitly.

```perl
use MyModule qw(:default extra_func);
```

### Symbol Types

- **Subroutines**: Do not use the ampersand (`&`) sigil in the export list.
- **Variables**: Always include the appropriate sigil (`$`, `@`, `%`).
- **Typeglobs**: Use the `*` sigil.

## Expert Tips

- **Performance**: Exporter::Tidy is significantly faster than the standard Exporter when using tags, as it avoids the overhead of `Exporter::Heavy`.
- **Lexical Scoping**: Because Exporter::Tidy doesn't require package globals, you can keep your internal logic strictly scoped or even use lexically scoped subroutines if mapped correctly via `_map`.
- **Multiple Prefixes**: You can switch prefixes mid-stream in a single `use` statement:
  ```perl
  use MyModule _prefix => 'api_', qw(get post), _prefix => 'internal_', qw(debug);
  ```
