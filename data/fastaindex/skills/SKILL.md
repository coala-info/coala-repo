# fastaindex

## Overview

fastaindex is a Python-based utility designed to manage FASTA indices (.fai) while maintaining full compatibility with the standard samtools faidx format. It distinguishes itself by extending the index with four additional columns that store the counts of A, C, G, and T bases for every sequence. Beyond indexing, it provides high-performance sequence retrieval and assembly statistics, making it a versatile tool for genomic data processing and quality control.

## Command Line Usage

### Basic Indexing
To generate a samtools-compatible index with extended base counts:
```bash
FastaIndex -i sequences.fa
```

### Sequence Retrieval
Extract specific regions using the standard `contig:start-end` notation.
*   **Standard extraction**: `FastaIndex -i genome.fa -r 'scaffold_1:100-200'`
*   **Reverse complement**: Simply swap the start and end coordinates to retrieve the reverse complement of a region:
    `FastaIndex -i genome.fa -r 'scaffold_1:200-100'`

### Assembly Statistics
The tool includes built-in functions for rapid assembly assessment:
*   **Calculate N50**: `FastaIndex -i contigs.fa -N50`
*   **Calculate L50**: `FastaIndex -i contigs.fa -L50`
*   **Full Statistics**: Generate a comprehensive report of the FASTA file:
    `FastaIndex -i contigs.fa -S`
    or use the dedicated script:
    `fasta_stats -i contigs.fa`

## Expert Tips and Best Practices

*   **Automatic Updates**: fastaindex automatically checks the modification times of your files. If the FASTA file is newer than the existing `.fai` index, or if the index is corrupted, the tool will automatically regenerate it.
*   **Symlink Support**: If your FASTA file is a symbolic link, the tool is intelligent enough to create a symbolic link for the `.fai` file as well, provided the source index exists and is valid.
*   **Extended Index Format**: Remember that the `.fai` generated by this tool contains 9 columns instead of the standard 5. The extra columns (6-9) represent the counts for A, C, G, and T respectively. This is useful for calculating GC content without re-reading the entire sequence.
*   **Coordinate System**: Retrieval uses 1-based inclusive coordinates for the region string, matching standard bioinformatics conventions.
*   **Compressed Files**: Note that current versions of this specific implementation do not support compressed (.gz or .bz2) FASTA files directly; files should be decompressed before indexing.
