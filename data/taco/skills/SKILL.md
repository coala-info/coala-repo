# taco

## Overview
TACO is a specialized meta-assembly tool designed to aggregate transcriptome data from multiple RNA-Seq samples. It processes GTF files generated by initial assemblers (like StringTie or Cufflinks) to produce a comprehensive, unified transcriptome. This skill provides guidance on the core workflow of TACO, including input preparation, expression-based filtering, and assembly comparison.

## Installation
The recommended method for installing TACO is via Bioconda:
```bash
conda install bioconda::taco
```

## Core Workflow and CLI Patterns

### 1. Input Preparation
TACO requires a text file containing the paths to the GTF files you wish to merge.
- Create a file (e.g., `gtf_files.txt`) with one absolute or relative path per line.
- Ensure all input GTF files are valid and follow standard formatting to avoid "GTFError" or "Transcript feature" sequence issues.

### 2. Basic Assembly
To run the standard multi-sample assembly:
```bash
taco --num-processes 8 -o output_directory gtf_files.txt
```
- `-o`: Specifies the output directory where `assembly.gtf` will be created.
- `--num-processes`: Sets the number of CPU cores for parallel processing.

### 3. Expression and Quality Filtering
TACO allows for fine-grained filtering of transcripts based on expression metrics.
- **TPM Filtering**: Use to remove low-abundance transcripts.
- **Coverage Filtering**: Use to filter based on the depth of reads supporting a transcript.
- **Splice Junctions**: TACO can be configured to permit additional splice junction motifs during filtering if your data contains non-canonical sites.

### 4. Reference Comparison (refcomp)
TACO includes a utility called `taco_refcomp` to compare your assembled transcriptome against a known reference (e.g., Gencode or Ensembl).
- Use this to categorize transcripts as known, novel isoforms, or intergenic.
- Note: Ensure the strand information in your GTF matches the reference to avoid intragenic overlap errors.

## Expert Tips
- **GTF Validation**: TACO is sensitive to the order of features in GTF files. If you encounter a "Transcript feature appeared prior to transcript" error, ensure your GTF is sorted correctly (Transcript line followed by its Exon lines).
- **Memory Management**: For very large multi-sample cohorts, monitor memory usage as the graph-based assembly can be resource-intensive.
- **CPAT Integration**: TACO has historically supported CPAT (Coding-Potential Assessment Tool) options for identifying coding vs. non-coding transcripts; ensure the CPAT executables are in your PATH if using these flags.
