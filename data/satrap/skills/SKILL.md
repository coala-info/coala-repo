# satrap

## Overview
Satrap (SOLiD Assembly TRAnslation Program) is a utility designed to bridge the gap between the unique color-space output of ABI SOLiD sequencers and standard bioinformatics tools that require base-space (A, C, G, T) input. It is particularly useful for processing de novo assemblies or read sets where the original data was captured in the di-base encoding format.

## Usage Guidelines

### Core Functionality
The primary workflow involves taking color-space sequences (represented by digits 0-3) and converting them into DNA sequences. Because color-space encoding relies on the transition between bases, a single error can lead to a "frame shift" in the translation; satrap is designed to handle these transitions during the assembly translation process.

### Common CLI Patterns
While specific flags vary by version, the general execution follows this pattern:

- **Translation**: Inputting a color-space FASTA file and a quality file (if available) to produce a base-space FASTA.
- **Assembly Processing**: Converting contigs generated by color-space assemblers (like Velvet in color-space mode or DenovoGear) into standard formats.

### Best Practices
- **Verify Primer Base**: Ensure the first base (the "T" or "A" primer) is correctly identified in your input files, as the entire translation sequence depends on this starting point.
- **Quality Scores**: Always provide the corresponding `.qual` or `.csqual` files when possible. Satrap uses these to make more informed decisions when a color-space call is ambiguous.
- **Validation**: After translation, check for an over-representation of "N" characters, which may indicate a mismatch between the expected color-space encoding and the input data format.
