# perl-bio-rna-barriers

## Overview
The `perl-bio-rna-barriers` suite provides a programmatic interface and command-line utilities to handle the complex output generated by the `barriers` tool (part of the ViennaRNA package). It is designed to simplify the post-processing of RNA folding landscapes by allowing users to filter energy barriers, identify specific secondary structures within basins, and convert barrier data into more accessible formats for downstream analysis.

## Core Usage Patterns

### Parsing Barriers Output
The primary utility is designed to read the standard output from the `barriers` program. This output typically contains a list of local minima, their energies, and the saddle points connecting them.

- **Filtering by Energy**: Use the tool to extract only those basins that fall within a specific energy range above the global minimum.
- **Basin Identification**: Quickly map a specific RNA secondary structure (in dot-bracket notation) to its corresponding basin ID in the barriers tree.
- **Saddle Point Analysis**: Retrieve the energy height of the barrier between two specific local minima to calculate transition rates.

### Common CLI Operations
While often used as a Perl library (`Bio::RNA::Barriers`), the package provides scripts to perform common tasks:

- **Summarizing Landscapes**: Generate a condensed report of the number of basins found at different energy cutoffs.
- **Structure Extraction**: Extract the representative structure for each local minimum found in the barriers file.
- **Path Finding**: Trace the path from a high-energy state down to its corresponding local minimum through the gradient descent defined in the barriers output.

## Expert Tips
- **Input Consistency**: Ensure that the barriers output was generated using the same sequence and thermodynamic parameters as your initial landscape calculation to avoid coordinate mismatches.
- **Memory Management**: For very large landscapes (e.g., long RNA sequences with many states), use the streaming parser options if available to avoid loading the entire landscape into RAM.
- **Integration**: Use this tool as a bridge between `barriers` and kinetic simulators like `treekin`. It is particularly useful for re-formatting the tree structure into the input format required for custom Markov Chain models.
