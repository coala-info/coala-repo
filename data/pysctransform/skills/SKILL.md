# pysctransform

## Overview

pysctransform is a Python-native implementation of the SCTransform algorithm, providing a statistical framework for the normalization and variance stabilization of single-cell transcriptomic data. It models the technical noise in UMI counts using regularized negative binomial regression, effectively removing the influence of sequencing depth while preserving biological heterogeneity. This skill enables the seamless integration of SCTransform into Scanpy workflows, allowing users to generate Pearson residuals that are better suited for dimensionality reduction than traditional scaling methods.

## Installation

The recommended method for installation is via conda through the bioconda channel:

```bash
conda install bioconda::pysctransform
```

## Core Workflow with Scanpy

The primary interface for Scanpy users is the `SCTransform` function. This function calculates Pearson residuals and identifies highly variable genes in a single step.

### Standard Pipeline

```python
import scanpy as sc
from pysctransform import SCTransform

# Load data (AnnData object)
adata = sc.read_h5ad("data.h5ad")

# Calculate Pearson residuals for the top 3000 highly variable genes
# This returns a sparse matrix of residuals
residuals = SCTransform(adata, var_features_n=3000)

# Store residuals in obsm for downstream use
adata.obsm["pearson_residuals"] = residuals

# Perform PCA specifically on the Pearson residuals
adata.obsm["X_pca"] = sc.pp.pca(adata.obsm["pearson_residuals"])

# Proceed with standard Scanpy clustering/visualization using the new PCA
sc.pp.neighbors(adata, use_rep="X_pca")
sc.tl.umap(adata)
sc.tl.louvain(adata)
```

## Advanced Variance Stabilization (vst)

For more granular control or when working outside of the `AnnData` structure, use the `vst` module. This is useful for inspecting the model fit and plotting residual variance.

### Manual VST and Plotting

```python
from pysctransform import vst
from pysctransform.plotting import plot_residual_var

# Perform variance stabilization on a UMI matrix (genes x cells)
# method="fix-slope" is a common regularization approach
vst_out = vst(
    umi=adata.X.T, 
    gene_names=adata.var_names.tolist(), 
    cell_names=adata.obs_names.tolist(), 
    method="fix-slope", 
    exclude_poisson=True
)

# Visualize the relationship between gene mean and residual variance
plot_residual_var(vst_out)
```

## Expert Tips and Best Practices

- **Feature Selection**: By default, `SCTransform` selects 3000 highly variable genes. To calculate residuals for all genes in the dataset, set `var_features_n=None`.
- **PCA Integration**: Always ensure that `sc.pp.pca` is called on the residuals stored in `obsm` rather than the raw or log-normalized `X` matrix. The residuals are already centered and scaled by the model.
- **Batch Effects**: Note that `batch_var` is currently not supported in the Python implementation. If your data has significant batch effects, consider using other integration tools (like Harmony or Scanorama) on the Pearson residuals generated by this tool.
- **Memory Efficiency**: The output of `SCTransform` is a sparse matrix. Keep it in sparse format as long as possible to minimize memory overhead, especially with large-scale datasets.
- **Regularization**: The `method="fix-slope"` parameter in the `vst` function is often more stable for datasets with lower sequencing depth or fewer cells.
