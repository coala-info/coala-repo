# coolpuppy

## Overview

coolpuppy is a specialized tool for performing pile-up analysis on Hi-C data stored in the `.cool` format. It allows users to average multiple 2D regions (snippets) of a contact matrix to identify focal enrichments, such as loop strength or TAD boundary intensity. Unlike other tools that store all snippets in memory, coolpuppy is designed for high performance and low memory usage, making it suitable for processing millions of potential interactions. It supports both intra-chromosomal (cis) and inter-chromosomal (trans) pileups and provides built-in normalization methods to account for the decay of contact probability with distance.

## Installation

Install via bioconda:
```bash
conda install bioconda::coolpuppy
```

## Core CLI Usage

The primary command is `coolpup.py`. The basic syntax requires a `.cool` file and a `.bed` (or BEDPE) file defining the regions of interest.

### Basic Pile-up
To perform a standard pile-up of regions defined in a BED file:
```bash
coolpup.py data.cool regions.bed --out output.coolpup.json
```

### Normalization Strategies
Normalization is critical for comparing different datasets or genomic distances.

1.  **By-distance expected**: Use a file containing average interaction values at different distances (typically generated by `cooltools expected-cis`).
    ```bash
    coolpup.py --expected expected.tsv data.cool regions.bed
    ```
2.  **Random shifts**: Normalize by dividing the pileup over randomly shifted control regions.
    ```bash
    coolpup.py --nshifts 10 data.cool regions.bed
    ```
3.  **No normalization**: Equivalent to traditional APA (Rao et al., 2014).
    ```bash
    coolpup.py --nshifts 0 data.cool regions.bed
    ```

### Common CLI Options
- `--nproc [INT]`: Number of processes to use for parallel computation.
- `--rescale`: Rescale all snippets to the same size (useful for TAD analysis).
- `--rescale_flank [FLOAT]`: Fraction of the window size to use as padding when rescaling.
- `--local`: Perform pile-up of regions on the diagonal (e.g., TADs).
- `--trans`: Enable inter-chromosomal pile-up analysis.

## Visualization

Use `plotpup.py` to visualize the results generated by `coolpup.py`.

```bash
plotpup.py --output pileup_plot.png output.coolpup.json
```

### Plotting Tips
- **Log scale**: Use `--log` to plot the color scale in log10.
- **Color limits**: Set `--vmin` and `--vmax` to standardize the color scale across multiple plots.
- **Multiple files**: You can pass multiple `.json` files to `plotpup.py` to create a grid of plots for comparison.

## Expert Tips

- **Memory Efficiency**: coolpuppy loads whole chromosomes one by one. If you have limited RAM, avoid using too many processes (`--nproc`) on very high-resolution files.
- **Large Snippet Counts**: Because coolpuppy does not store the 3D stack of snippets in memory, it is the preferred tool for "huge" numbers of interactions where `cooltools pileup` might crash.
- **Trans Interactions**: Use coolpuppy for trans-pileups (interactions between different chromosomes), as this feature is not supported in many other Hi-C suites.
- **Input Conversion**: If your data is in `.hic` format, use `hic2cool` to convert it to `.cool` before using coolpuppy.
