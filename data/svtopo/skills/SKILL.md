## Overview
SVTopo is a specialized tool designed for visualizing complex structural variants (SVs) identified in PacBio HiFi sequencing data. It takes VCF files and aligned BAM files as input to determine the connectivity between SV breakends. The primary output is a JSON file detailing these connections, which can then be used by the `SVTopoVz` Python utility to generate high-quality visualizations of complex structural rearrangements. This tool is particularly useful when paired with variant callers like sawfish to interpret intricate genomic alterations.

## Usage Instructions

SVTopo primarily operates via its command-line interface. The core functionality involves processing VCF and BAM files to output a JSON file representing connected breakends. The `SVTopoVz` utility is then used to convert this JSON into visual plots.

### Core Command Structure

The general command structure for SVTopo involves specifying input files and output destinations.

```bash
svtopo <input.vcf> <input.bam> --output <output.json>
```

### Key Parameters and Options

*   **`input.vcf`**: Path to the input Variant Call Format (VCF) file containing structural variant calls.
*   **`input.bam`**: Path to the input aligned Binary Alignment Map (BAM) file. This file should be indexed.
*   **`--output <output.json>`**: Specifies the output file path for the JSON containing connected breakends.

### Visualization with `SVTopoVz`

After generating the JSON output from `svtopo`, use `SVTopoVz` for visualization.

```bash
SVTopoVz <input.json> --output <output.png>
```

*   **`input.json`**: Path to the JSON file generated by `svtopo`.
*   **`--output <output.png>`**: Specifies the output file path for the generated visualization image (e.g., PNG).

### Best Practices and Expert Tips

*   **BAM Indexing**: Ensure your BAM file is indexed (`.bai` file) for efficient access. If not, you can create one using `samtools index <input.bam>`.
*   **VCF and BAM Alignment**: For optimal results, ensure the VCF and BAM files are derived from the same sequencing data and align to the same reference genome.
*   **Sawfish Integration**: SVTopo is designed to work well with variant calls from `sawfish`. If you are using `sawfish`, ensure its output VCF is used as input.
*   **Gene Annotations**: SVTopo can optionally include gene annotations in its visualizations. Refer to the tool's documentation for specific flags or configuration if this feature is needed.
*   **Interpreting Output**: The visualizations represent connected breakends as blocks, indicating their order and orientation relative to the reference genome. Block weight often signifies alignment support strength, and block sizes are encoded in the legend. Chained plots highlight deletion, duplication, and inversion effects.
*   **Pre-release Software**: Note that SVTopo is pre-release software intended for research use. While efforts are made for quality, it's not for diagnostic procedures. Report issues via GitHub.
