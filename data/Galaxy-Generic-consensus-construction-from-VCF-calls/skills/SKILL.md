# generic-consensus-construction-from-vcf-calls

## Overview

This Galaxy workflow generates a consensus sequence from VCF calls by incorporating high-confidence variants while masking problematic genomic regions. It processes aligned reads and variant calls to identify FILTER PASS variants that meet a user-defined allele-frequency (AF) threshold. The pipeline is designed to handle intrasample variation by distinguishing between consensus-level variants and failed variants that should be excluded or masked.

The workflow utilizes [bedtools](https://toolshed.g2.bx.psu.edu/repos/iuc/bedtools/bedtools_genomecoveragebed/2.29.2) to calculate genome coverage and identify low-depth regions based on a configurable threshold. It then employs [SnpSift](https://toolshed.g2.bx.psu.edu/repos/iuc/snpsift/snpSift_filter/4.3+t.galaxy1) and various text manipulation tools to isolate specific variant sites. A critical feature of this process is the subtraction of valid consensus variants from the masking set, ensuring that high-quality variants are preserved even if they fall within otherwise low-coverage areas.

The final consensus sequence is constructed using [bcftools consensus](https://toolshed.g2.bx.psu.edu/repos/iuc/bcftools_consensus/bcftools_consensus/1.10+galaxy1), which applies the filtered variants to the reference genome. Regions identified as having low coverage or containing failed variants are hard-masked to maintain the integrity of the output. The workflow concludes by collapsing results into a multisample FASTA file, making it suitable for downstream phylodynamic or comparative genomic analyses.

This tool is licensed under the **MIT** license and is tagged for **mlxv** and **generic** use cases.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | aligned reads data for depth calculation | data_collection_input | Fully processed BAMs as generated by upstream workflows for variation analysis. Note: for ARTIC data, these BAMs should NOT have undergone processing with ivar removereads. |
| 1 | Depth-threshold for masking | parameter_input | Sites in the viral genome covered by less than this number of reads are considered questionable and will be masked (with Ns) in the consensus sequence independent of whether a variant has been called at them or not. |
| 2 | Variant calls | data_collection_input | Collection of VCFs produced by upstream workflows for variation analysis |
| 3 | min-AF for consensus variant | parameter_input | Only variant calls with an allele-frequency greater this value will be considered consensus variants. |
| 4 | min-AF for failed variants | parameter_input | Variant calls with an allele frequency higher than this value, but lower than the AF threshold for consensus variants will be considered questionable and the respective sites be masked (with Ns) in the consensus sequence. |
| 5 | Reference genome | data_input | The reference genome |


Ensure your input BAM files and VCF calls are organized into paired data collections to maintain sample consistency throughout the consensus building process. The reference genome must be provided in FASTA format, while the VCF collection should contain FILTER PASS variants with appropriate allele-frequency metadata for the thresholding steps. For automated execution, you can use `planemo workflow_job_init` to generate a `job.yml` template for these parameters. Refer to the README.md for comprehensive details on configuring the specific depth and AF masking thresholds. Always verify that your chromosome identifiers match exactly between the reference FASTA and the VCF inputs to avoid tool failures during the bcftools consensus step.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run Galaxy-Workflow-Generic_consensus_construction_from_VCF_calls.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run Galaxy-Workflow-Generic_consensus_construction_from_VCF_calls.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run Galaxy-Workflow-Generic_consensus_construction_from_VCF_calls.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init Galaxy-Workflow-Generic_consensus_construction_from_VCF_calls.ga -o job.yml`
- Lint: `planemo workflow_lint Galaxy-Workflow-Generic_consensus_construction_from_VCF_calls.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.
