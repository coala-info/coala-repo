## Overview
nf-core/denovotranscript addresses the challenge of reconstructing transcriptomes in the absence of a reference genome. It processes raw paired-end RNA-seq reads to generate a non-redundant set of transcripts and their corresponding expression levels, facilitating research in non-model organisms or complex biological systems.

The pipeline takes a samplesheet and FASTQ files as primary inputs. It produces a final transcriptome assembly in FASTA format, a gene-to-transcript mapping, and abundance estimates, allowing researchers to move directly from raw sequencing data to a quantified transcriptome ready for downstream analysis.

## Data preparation
The pipeline requires a comma-separated samplesheet provided via the `--input` parameter. This file must contain a header and three columns: `sample`, `fastq_1`, and `fastq_2`.

Example `samplesheet.csv`:
```csv
sample,fastq_1,fastq_2
CONTROL_REP1,AEG588A1_S1_L002_R1_001.fastq.gz,AEG588A1_S1_L002_R2_001.fastq.gz
```

**Constraints and Optional Files:**
* **FASTQ files:** Must be gzipped and use extensions `.fastq.gz` or `.fq.gz`.
* **Reference files:** If using rnaQUAST for evaluation, a reference genome (`--fasta`) and gene coordinates (`--gtf`) can be provided.
* **Ribosomal RNA:** To remove rRNA, provide a text file with paths to FASTA databases via `--ribo_database_manifest`.
* **Existing Assemblies:** If skipping assembly with `--skip_assembly`, a transcriptome FASTA must be provided via `--transcript_fasta`.

## How to run
The pipeline is executed using Nextflow with the `-profile` flag to specify the container engine (e.g., Docker, Singularity). It is recommended to use `-r` to pin a specific version and `-resume` for incremental runs.

```bash
nextflow run nf-core/denovotranscript \
   -profile <docker/singularity/conda> \
   --input samplesheet.csv \
   --outdir <OUTDIR> \
   -r 1.1.0
```

Key parameters include:
* `--assemblers`: Choose between `trinity`, `trinity_no_norm`, and `rnaspades` (default is both).
* `--ss`: Set strand-specificity for rnaSPAdes (`rf` or `fr`).
* `--qc_only`: Run only quality control steps without assembly.

## Outputs
Results are saved in the directory specified by `--outdir`. The primary deliverables include:

* **Assembly:** The final non-redundant transcriptome assembly in FASTA format.
* **Quantification:** Transcript abundance estimates in TSV format generated by Salmon.
* **Quality Control:** A MultiQC HTML report aggregating metrics from FastQC, fastp, BUSCO, and Salmon.
* **Evaluation:** Detailed assembly metrics from rnaQUAST and TransRate (note: TransRate is not available with `conda` or `mamba` profiles).

For a complete description of the directory structure and file contents, refer to the official [output documentation](https://nf-co.re/denovotranscript/output).
