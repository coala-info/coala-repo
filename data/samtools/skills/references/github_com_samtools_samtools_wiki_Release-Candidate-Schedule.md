# Release Candidate Schedule

[Jump to bottom](#wiki-pages-box)
 [Edit](/samtools/samtools/wiki/Release-Candidate-Schedule/_edit)
 [New page](/samtools/samtools/wiki/_new)

Martin Pollard edited this page Nov 25, 2013
·
[3 revisions](/samtools/samtools/wiki/Release-Candidate-Schedule/_history)

* Release candidate builds will be made each Monday morning, aiming to have builds ready and announcement made by 12 noon UK time, so that it is (just) possible to discuss issues arising at Monday afternoon samtools meetings.

  So code to go into a RC build needs to be committed to the respective **develop** branches at samtools GitHub by 1pm UK time each Friday.
* In general, bug fixes that have been completed and tested during a week must be made available in **develop** for the following Monday's RC. Larger new features and other particular items are scheduled to appear in RC builds as follows.

##### 18 November

* Martin's fixmate PR
* Fix for [samtools #83](https://github.com/samtools/samtools/issues/83)

##### 25 November

* trio calling update
* misc bugfixes

##### 2 December

* iterator update to support CRAM
* Martin's merge updates
* Tview support reenabled
* Fix for [samtools #96](https://github.com/samtools/samtools/issues/96)
