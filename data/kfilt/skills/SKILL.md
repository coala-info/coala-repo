# kfilt

## Overview
kfilt is a specialized bioinformatics tool designed to subset sequencing data by matching read k-mers against a pre-defined index. Unlike simple exact-match filters, kfilt employs Burkhard-Keller trees (BK-trees), which enable efficient searching within a specific Hamming distance. This makes it highly effective for capturing reads that contain sequencing errors or minor variations relative to the target k-mer set. The tool follows a two-step logic: first, constructing a searchable index from a list of k-mers, and second, filtering single-end, paired-end, or interleaved read files against that index.

## Core Workflows

### 1. Index Construction
Before filtering, you must provide a list of k-mers (typically generated by tools like `meryl`).

```bash
# Build a BK-tree index from a text file of k-mers
kfilt build -k unique_kmers.txt -o target_index.bktree.idx -K 31
```
*   **Tip**: Ensure the `-K` (k-mer size) matches the size used during the k-mer counting/extraction phase.

### 2. Filtering Reads
kfilt supports various input configurations and provides flexible matching logic.

**Single-end reads (Exact match):**
```bash
kfilt filter -1 reads.fq.gz -i target_index.bktree.idx -o filtered.fq.gz -z -n 1 -m 0
```

**Paired-end reads (Approximate match):**
```bash
kfilt filter -1 R1.fq.gz -2 R2.fq.gz -i target_index.bktree.idx -o filtered.fa.gz -f fasta -z -n 5 -m 1
```

**Interleaved input (Mixed paired/single):**
```bash
kfilt filter -I interleaved.fq.gz -i target_index.bktree.idx -o filtered.fq.gz -z
```

## Command Options and Best Practices

### Matching Logic
*   **`-n, --min-matches`**: The minimum number of k-mers in a read that must match the index for the read to be kept. Increase this to improve specificity.
*   **`-m, --hamming-dist`**: Maximum allowed mismatches per k-mer. 
    *   `0`: Exact matching (fastest).
    *   `1`: Allows one mismatch (good balance for Illumina data).
    *   `2+`: Significantly increases search space and slows down processing.
*   **`-B, --both-match`**: For paired-end data, by default, a pair is kept if the *combined* matches meet the threshold. Use `-B` to require that *each* read in the pair independently satisfies the `-n` requirement.

### Performance and Output
*   **Threading**: Use `-t` to specify CPU cores. kfilt scales well across multiple threads.
*   **Compression**: Use `-z` to output gzip-compressed files. You can control the level with `-Z` (1-9).
*   **Format Conversion**: You can input FASTQ and output FASTA by specifying `-f fasta`.
*   **Statistics**: Always use `-v stats.tsv` during pilot runs. This produces a per-read report showing `matched_kmers` and `best_hamming_dist`, which is invaluable for fine-tuning the `-n` and `-m` parameters.

## Expert Tips
*   **Upstream K-mer Prep**: kfilt expects a simple text list of k-mers (one per line). If using `meryl`, use `meryl print` to generate the compatible input.
*   **Memory Management**: While BK-trees are fast for searching, building an index from tens of millions of k-mers can be memory-intensive. Filter your k-mer list (e.g., removing low-frequency k-mers) before building the index if memory is a constraint.
*   **Unmapped Read Extraction**: A common use case is filtering unmapped reads from a BAM/CRAM file to find specific alleles. Use `samtools` to pipe unmapped, interleaved reads directly into `kfilt filter -I -`.
