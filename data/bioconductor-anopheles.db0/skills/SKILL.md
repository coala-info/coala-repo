# bioconductor-anopheles.db0

## Overview

The `anopheles.db0` package is a Bioconductor annotation BASE package. Unlike standard annotation packages (like `org.Ag.eg.db`), it is not intended for direct data querying by end-users. Instead, its primary purpose is to serve as a central repository for the latest raw annotation databases. These databases are used by the `AnnotationDbi` package's `sqlForge` functionality to construct more user-friendly, schema-stable annotation packages.

## Usage Guidelines

### Primary Role
The package acts as a data source for building other packages. You should generally not call the databases inside this package directly because the internal schema is unstable and subject to change without notice.

### Installation
To ensure you have the latest base databases for building Anopheles-related annotation packages, install via `BiocManager`:

```r
if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

BiocManager::install("anopheles.db0")
```

### Workflow: Building Annotation Packages
The typical workflow involves using `AnnotationDbi` functions to wrap the data found in `anopheles.db0` into a standard SQLite-based annotation package. While the documentation specifically mentions functions for human, mouse, and fly, the general principle applies to generating custom organism or chip packages where Anopheles data is the source.

1. **Load the library**:
   ```r
   library(anopheles.db0)
   ```

2. **Identify the Database Location**:
   If you must find the raw database file for a custom `sqlForge` task:
   ```r
   system.file("extdata", package="anopheles.db0")
   ```

### Recommendations
- **Avoid Direct Queries**: Do not write SQL queries against the internal tables of `anopheles.db0` for production scripts.
- **Use Derived Packages**: For genomic analysis, GO enrichment, or identifier mapping, use the resulting "org" or "chip" packages generated by `sqlForge` (e.g., `org.Ag.eg.db` for Anopheles gambiae) rather than this base package.
- **Stay Updated**: Regularly update this package to ensure that any new annotation packages you build are using the most recent source data.
