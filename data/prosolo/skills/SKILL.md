# prosolo

## Overview
ProSolo is a Bayesian variant caller designed to handle the unique noise profiles of single-cell DNA sequencing, particularly data generated via Multiple Displacement Amplification (MDA). By comparing a single-cell sample against a background bulk sample, it distinguishes true biological variants from amplification artifacts. It provides probabilistic scoring for various single-cell events, allowing for precise False Discovery Rate (FDR) control.

## Installation and Setup
The tool is primarily distributed via Bioconda.
```bash
conda create -n prosolo prosolo
conda activate prosolo
```

## Core Workflow

### 1. Variant Calling
The `single-cell-bulk` subcommand is the primary entry point. It requires a candidate VCF/BCF file (generated by tools like `freebayes` or `samtools mpileup`) to define the sites for evaluation.

**Standard Pattern:**
```bash
prosolo single-cell-bulk \
    --omit-indels \
    --candidates candidates.bcf \
    --output output.bcf \
    single-cell.bam \
    bulk.bam \
    reference.fa
```
*Note: Currently, `--omit-indels` is recommended as indel calling is not yet fully optimized.*

### 2. Controlling False Discovery Rate (FDR)
ProSolo outputs probabilities for specific events (e.g., HET, HOM_ALT, ADO_TO_REF). Use `control-fdr` to filter results based on a target FDR threshold.

**To filter for any evidence of an alternative allele:**
```bash
prosolo control-fdr input.bcf \
    --events HET,HOM_ALT,ADO_TO_ALT,ADO_TO_REF,ERR_REF \
    --var SNV \
    --fdr 0.05 \
    --output filtered_variants.bcf
```

**To filter specifically for Allele Dropout (ADO) events:**
```bash
prosolo control-fdr input.bcf \
    --events ADO_TO_REF,ADO_TO_ALT \
    --var SNV \
    --fdr 0.01 \
    --output ado_events.bcf
```

## Expert Tips
- **Parallelization**: ProSolo does not have built-in multi-threading for a single file. To speed up processing, split your `--candidates` BCF file by genomic regions (e.g., by chromosome) and run multiple instances of `prosolo single-cell-bulk` in parallel.
- **Input Preparation**: Ensure both the single-cell and bulk BAM files are position-sorted and indexed. The reference FASTA must match the one used for alignment.
- **Event Selection**: When using `control-fdr`, you can combine any events found in the `PROB_` fields of the BCF header. Omitting the `PROB_` prefix in the CLI command allows you to jointly control the FDR for those specific biological or technical states.
