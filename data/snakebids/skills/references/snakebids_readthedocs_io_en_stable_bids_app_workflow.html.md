# Workflows[¶](#workflows "Link to this heading")

Snakebids workflows are constructed the same way as any other [Snakemake workflows](https://snakemake.readthedocs.io/en/stable/snakefiles/rules.html "Snakemake 9.13.7"), but with a few additions that make it easier to work with BIDS datasets.

To get access to these additions, the base Snakefile for a snakebids workflow should begin with the following boilerplate:

```
 1import snakebids
 2from snakebids import bids
 3
 4configfile: 'config/snakebids.yml'
 5
 6# Get input wildcards
 7inputs = snakebids.generate_inputs(
 8    bids_dir=config["bids_dir"],
 9    pybids_inputs=config["pybids_inputs"],
10    pybidsdb_dir=config.get("pybidsdb_dir"),
11    pybidsdb_reset=config.get("pybidsdb_reset"),
12    derivatives=config.get("derivatives"),
13    participant_label=config.get("participant_label"),
14    exclude_participant_label=config.get("exclude_participant_label"),
15)
16
```

## Snakebids workflow features[¶](#snakebids-workflow-features "Link to this heading")

The [`snakebids.bids`](../api/paths.html#snakebids.bids "snakebids.bids") function generates a properly-formatted BIDS filename with the specified entities, as documented in more detail elsewhere in this documentation.

[`snakebids.generate_inputs`](../api/creation.html#snakebids.generate_inputs "snakebids.generate_inputs") returns an instance of [`snakebids.BidsDataset`](../api/structures.html#snakebids.BidsDataset "snakebids.BidsDataset"), a special [`dict`](https://docs.python.org/3/library/stdtypes.html#dict "Python 3.14") with keys mapping to the [`BidsComponents`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") defined in [the config file](config.html). Each [`BidsComponent`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") contains a number of attributes to assist processing a BIDS dataset with snakemake. [`generate_inputs()`](../api/creation.html#snakebids.generate_inputs "snakebids.generate_inputs") should be called at the beginning of the workflow and assigned to a variable called `inputs`.

The [`path`](../api/structures.html#snakebids.BidsComponent.path "snakebids.BidsComponent.path") member of [`BidsComponent`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") is generated by snakebids and contains a list of matched files for every input type. Often, the first rule to be invoked will use one or more entries in `inputs.path` as the input file specification.

The [`expand()`](../api/structures.html#snakebids.BidsComponent.expand "snakebids.BidsComponent.expand") method of [`BidsComponent`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") is used to expand over files using only the entity-values found in your dataset. A usage pattern is as follows:

```
inputs["bold"].expand(
    bids(
        root="results",
        datatype="func",
        suffix="func.shape.gii",
        hemi="{hemi}",
        **inputs.wildcards["bold"]
    ),
    hemi=config["hemi"],
)
```

Extra entities provided to [`BidsComponent.expand()`](../api/structures.html#snakebids.BidsComponent.expand "snakebids.BidsComponent.expand") will expand the path across every possible combination of values, just like in the snakemake [`expand()`](https://snakemake.readthedocs.io/en/stable/snakefiles/rules.html#snakefiles-expand "Snakemake 9.13.7").

By default, [`BidsComponent.expand()`](../api/structures.html#snakebids.BidsComponent.expand "snakebids.BidsComponent.expand") prevents partial expansion over paths, consistent with the default snakemake behaviour. To allow the presence of extra wildcards in the path, set the `allow_missing` argument in [`BidsComponent.expand()`](../api/structures.html#snakebids.BidsComponent.expand "snakebids.BidsComponent.expand") to `True`.

The [`wildcards`](../api/structures.html#snakebids.BidsComponent.wildcards "snakebids.BidsComponent.wildcards") member of [`BidsComponent`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") is generated by snakebids and contains a dictionary mapping the wildcards for each input type to snakemake-formatted wildcards, for convenient use in the `bids` function.

## Accessing the underlying *pybids* dataset[¶](#accessing-the-underlying-pybids-dataset "Link to this heading")

In addition to mapping all of the [`BidsComponents`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent") to their names, [`BidsDataset`](../api/structures.html#snakebids.BidsDataset "snakebids.BidsDataset") also has a [`layout`](../api/structures.html#snakebids.BidsDataset.layout "snakebids.BidsDataset.layout") member which gives access to the underlying [`BIDSLayout`](https://bids-standard.github.io/pybids/generated/bids.layout.BIDSLayout.html#bids.layout.BIDSLayout "(in PyBIDS v0.16.4)"). This can be used to access advanced pybids features not covered by `snakebids`. Note that if `custom_paths` are specified for every [`BidsComponent`](../api/structures.html#snakebids.BidsComponent "snakebids.BidsComponent"), pybids indexing will be skipped and [`layout`](../api/structures.html#snakebids.BidsDataset.layout "snakebids.BidsDataset.layout") will be set to `None`. If your workflow relies on accessing this [`layout`](../api/structures.html#snakebids.BidsDataset.layout "snakebids.BidsDataset.layout"), you must ensure your users do not provide a `custom_path` for every single component, either in the config file or [via the CLI](../running_snakebids/overview.html) (`--path_{component}`).
