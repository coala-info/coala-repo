# mars

## Overview
MARS (Multiple Alignment-based Refinement of SVs) is a bioinformatics tool designed to refine structural variant calls by leveraging data across multiple samples. It processes diploid assembly contigs (typically generated by assemblers like Aquila) and uses multiple sequence alignment to improve the precision of SV detection. The tool operates in a two-step workflow: first calling variants from individual assemblies, then merging and refining them across the population. It also supports specialized modes for identifying ancestral states (HARP mode) and linking variants to gnomAD SNPs.

## Installation and Environment
MARS is available via Bioconda. Ensure that core dependencies are available in your PATH.

```bash
# Installation
conda install -c bioconda mars

# Core Dependencies
# Ensure samtools, minimap2, and paftools are installed and executable
```

## Core Workflow

### Step 1: Assembly-based SV Calling
Use `MARS_step1.py` to perform initial SV calling from diploid assembled contigs.

```bash
MARS_step1.py \
    --assembly_dir <input_contig_folder> \
    --ref_file <reference.fasta> \
    --sample_list 'Sample1','Sample2','Sample3' \
    --chr_start 1 --chr_end 22 \
    --SV_len 20 \
    --out_dir ./MARS_step1_results
```

**Key Parameters:**
- `--assembly_dir`: Path to the folder containing diploid contig files (e.g., Aquila output).
- `--sample_list`: Comma-separated list of sample names (must match contig file prefixes).
- `--num_threads`: Sets the number of samples to process in parallel.
- `--SV_len`: Minimum SV size to detect (default is 20bp).

### Step 2: Population Merging and Refinement
Use `MARS_step2.py` to merge results from Step 1 and generate refined multiple-sample alignment files.

```bash
MARS_step2.py \
    --in_dir ./MARS_step1_results \
    --assembly_dir <input_contig_folder> \
    --ref_file <reference.fasta> \
    --sample_list 'Sample1','Sample2','Sample3' \
    --chr_start 1 --chr_end 22 \
    --out_dir ./MARS_step2_results
```

**Key Parameters:**
- `--num_threads_bychr`: Sets the number of chromosomes to process in parallel.
- `--in_dir`: The output directory from Step 1.

## Advanced Analysis Modes

### HARP Mode (Ancestral State Analysis)
To identify derived deletions and insertions relative to ancestral genomes, enable the HARP flag.

```bash
MARS_step2.py [standard_params] \
    --HARP_flag 1 \
    --Ape_ref_list "Gorilla.fasta,Chimpanzee.fasta,Orangutan.fasta"
```

### gnomAD Integration
To output linked SNPs from gnomAD for each detected SV, use the gnomAD flag.

```bash
MARS_step2.py [standard_params] \
    -gnomad_flag 1 \
    --gnomad_dir <path_to_gnomad_vcf_folder>
```
*Note: gnomAD VCF files must be downloaded locally and organized by chromosome.*

## Best Practices and Tips
- **Resource Management**: In Step 1, memory usage scales with the number of samples processed in parallel (`--num_threads`). In Step 2, it scales with the number of chromosomes (`--num_threads_bychr`).
- **Chromosome Targeting**: For large datasets, process chromosomes individually or in small batches using `--chr_start` and `--chr_end` to manage compute time and avoid job timeouts.
- **Input Formatting**: Ensure the `sample_list` names exactly match the prefixes of the `.fasta` contig files in your assembly directory.
- **Output Interpretation**: The primary results are found in `MARS_step2_results/Final_tables/`. Look for `SV_msa_table_chr*.txt` for the refined population-scale SV calls.
