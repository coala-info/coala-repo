# postmaster

## Overview
Postmaster is a specialized bioinformatics tool designed to bridge the gap between Salmon's high-speed transcript quantification and alignment-based downstream tools. It processes the mapping files generated by Salmon and assigns a posterior probability to each alignment based on the abundance estimates found in the Salmon `quant.sf` file. This probability is stored in the `ZW` tag, allowing Salmon-generated alignments to be used in pipelines originally designed for RSEM.

## Installation
The recommended way to install postmaster is via Bioconda:
```bash
conda install bioconda::postmaster
```

## Core Workflow
To use postmaster effectively, you must follow a specific two-step sequence:

1.  **Generate Mappings**: Run `salmon quant` with the `--writeMappings` flag.
    ```bash
    salmon quant -i <index> -l <libType> -1 <reads1> -2 <reads2> --writeMappings=alignments.bam -o salmon_output
    ```
2.  **Annotate Alignments**: Run `postmaster` using the resulting BAM and the `quant.sf` file.
    ```bash
    postmaster --quant salmon_output/quant.sf --alignments alignments.bam --output annotated_alignments.bam
    ```

## CLI Usage and Patterns

### Basic Command Structure
```bash
postmaster -q <quant.sf> -a <input.bam> -o <output.bam>
```

### Streaming with Pipes
Postmaster supports stdin and stdout, which is useful for saving disk space when handling large alignment files.
*   **Input**: If `-a` is omitted, postmaster reads from `stdin`.
*   **Output**: If `-o` is omitted, postmaster writes to `stdout` in **BAM** format.

Example of a piped workflow:
```bash
samtools view -h alignments.bam | postmaster -q salmon_output/quant.sf > annotated.bam
```

### Thread Management
Use the `-n` or `--num-threads` option to speed up processing. The default is 2 threads.
```bash
postmaster -q salmon_output/quant.sf -a alignments.bam -o annotated.bam -n 8
```

## Expert Tips and Best Practices
*   **Strict Matching**: Ensure the `quant.sf` file and the alignment file come from the exact same Salmon run. Using a quantification file from a different sample or a different index will result in incorrect or null probability assignments.
*   **Output Format Detection**: Postmaster determines the output format based on the file extension provided to the `-o` flag. Use `.sam` for text output and `.bam` for compressed binary output.
*   **Salmon Compatibility**: Postmaster is specifically designed to work with the output of Salmon. It expects the alignment file to contain the mappings that Salmon used for its quantification estimate.
*   **The ZW Tag**: After processing, each alignment record in the output file will contain a `ZW:f:<value>` tag, where `<value>` is a floating-point number between 0 and 1 representing the posterior probability of that alignment.
