## Foldseek Release 10

Foldseek introduces GPU support for monomer and multimer search, improved profile search and ProstT5 integration, new databases, several performance improvements and bug fixes.

## Major Features

* **GPU Support for Search and Multimers**
  Both `easy-search` and `easy-multimersearch` now support accelerated searches on the GPU execution. Use `--gpu 1` to enable GPU mode and `CUDA_VISIBLE_DEVICES` to control the number of GPUs. ([#391](https://github.com/steineggerlab/foldseek/pull/391), [#411](https://github.com/steineggerlab/foldseek/pull/411)). GPU-enabled binaries require glibc >= 2.17, NVIDIA driver >= 525.60.13, and a Turing or newer GPU. On a single 4090 GPU, searches are 4x faster, and on eight GPUs, they are up to 37x faster than a 128-core CPU using the k-mer prefilter. For more details, see our [preprint](https://www.biorxiv.org/content/10.1101/2024.11.13.623350v2).
* **Improved Structural Profile Search**
  Alignment results can now be converted into position-specific scoring profiles with `result2profile`, enabling the creation of structural protein family representations. ([#411](https://github.com/steineggerlab/foldseek/pull/411))
* **Enhanced ProstT5 Integration**
  Multi-GPU and Apple Metal support added, Improved handling of large input sequence through splitting and switched backend to llama.cpp for better compatibility and performance. ([#391](https://github.com/steineggerlab/foldseek/pull/391))
* **New Databases**
  Introduced BFVD as a new virus-specific Foldseek database. ([#344](https://github.com/steineggerlab/foldseek/pull/344)).
  For more details about the database, check out the BFVD [paper](https://academic.oup.com/nar/article/53/D1/D340/7906834).
* **Improved Multimer Search Workflows**
  Optimized multimer workflows for improved speed and reliability, with contributions by [@Woosub-Kim](https://github.com/Woosub-Kim). For more details, see our Foldseek-Multimer [preprint](https://www.biorxiv.org/content/10.1101/2024.04.14.589414v4).
* **Clustering Multimers (First Version)**
  Introduced experimental multimer clustering (easy-multimercluster) by [@sooyoung-cha](https://github.com/sooyoung-cha) and [@rachelse](https://github.com/rachelse), supporting clustering by interface LDDT, chain TM-score, and complex coverage. See `filtercomplex` for more details.

## Breaking Changes

* Results may differ as masking of letters repeated six or more times is now enabled by default `--mask-n-repeat`. Disable this option to reproduce previous results.

## Other Features

* Improved Compatibility with MMseqs2 Modules: `createsubdb`, `makepaddedseqdb`, and `result2profile` now work seamlessly with Foldseek databases.
* Taxonomy Reports in `easy-search`: Added options to generate taxonomy reports directly within `easy-search`. ([#389](https://github.com/steineggerlab/foldseek/pull/389))
* Residue Mapping Rework: Residue mapping has been reworked to combine most gemmi amino acids with previous Foldseek amino acids. ([#387](https://github.com/steineggerlab/foldseek/pull/387))

## Bug Fixes

* Fix order dependent `--format-output` issue of `qtmaln,ttmaln,lddt,u,t` ([b40729c](https://github.com/steineggerlab/foldseek/commit/b40729c18b631cc781af4dc9c8e8ff8b67078193))
* Fix clustering of structures without Cα information ([0d8d966](https://github.com/steineggerlab/foldseek/commit/0d8d966cfa50b07c5ee83aa9060d795f5ee186a4))
