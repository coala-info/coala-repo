# telseq

## Overview
telseq is a C++ tool designed to calculate telomere length from BAM files generated by whole genome sequencing. It works by scanning for reads containing telomeric repeat sequences and normalizing these counts against the genome's GC content to provide a length estimate. This skill provides the necessary command-line patterns for analyzing single samples, batch processing multiple BAMs, and interpreting the resulting tabular data.

## Installation and Setup
The most reliable way to use telseq is via Bioconda or Docker to avoid complex dependency management with `bamtools`.

- **Conda**: `conda install -c bioconda telseq`
- **Docker**: `docker run -v /path/to/data:/data telseq-docker /data/sample.bam`

## Common CLI Patterns

### Basic Analysis
Analyze one or more BAM files and output results to the terminal:
```bash
telseq sample1.bam sample2.bam
```

### Batch Processing with a List
For large cohorts, provide a text file containing one BAM path per line:
```bash
telseq -f bam_list.txt -o cohort_results.tsv
```

### Merging Read Groups
By default, telseq outputs results per read group. To get a single weighted mean estimate for the entire sample:
```bash
telseq -m sample.bam > sample_estimate.tsv
```

### Header Management
When concatenating results from multiple runs, use `-H` to suppress the header line:
```bash
telseq -H next_sample.bam >> total_results.tsv
```

To extract only the header for a new file:
```bash
telseq -h > header.tsv
```

## Output Interpretation
The output is a tab-delimited format with the following key columns:
- **LENGTH_ESTIMATE**: The calculated telomere length.
- **TEL0 - TEL16**: Read counts containing specific numbers of telomeric repeats.
- **GC0 - GC9**: Read counts within specific GC content bins (used for normalization).
- **Mapped/Duplicates**: Quality control metrics to ensure the BAM file was processed correctly.

## Expert Tips
- **Inter-lane Variation**: It is often beneficial to run telseq *without* the `-m` flag initially. This allows you to inspect variation between different sequencing lanes/read groups before merging them into a final sample estimate.
- **Piping**: telseq supports piped input for the BAM list. You can filter your file list dynamically:
  ```bash
  find ./bams -name "*.bam" | telseq -o results.tsv
  ```
- **Memory and Performance**: telseq is generally efficient, but ensure your BAM files are indexed. While telseq reads the BAM sequentially, valid headers and mapping information are critical for the normalization step.
