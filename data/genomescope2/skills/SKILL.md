# genomescope2

## Overview
GenomeScope 2.0 is a mathematical modeling tool that infers global genome properties by analyzing k-mer frequencies derived from raw sequencing reads. Unlike the original version, GenomeScope 2.0 uses a polyploid-aware mixture model to accurately profile complex genomes. It requires a k-mer count histogram as input (typically generated by KMC or Jellyfish) and provides a detailed report and visualizations of the inferred genomic landscape.

## Pre-processing: Generating k-mer Histograms
Before running GenomeScope, you must generate a k-mer frequency histogram.

### Using KMC (Recommended)
1. Count k-mers:
   `kmc -k21 -t10 -m64 -ci1 -cs10000 @files_list.txt reads_out tmp/`
2. Transform to histogram:
   `kmc_tools transform reads_out histogram reads.histo -cx10000`

### Using Jellyfish
1. Count k-mers (always use `-C` for canonical k-mers):
   `jellyfish count -C -m 21 -s 1G -t 10 *.fastq -o reads.jf`
2. Generate histogram:
   `jellyfish histo -t 10 reads.jf > reads.histo`

## Running GenomeScope 2.0
Execute the analysis using the `genomescope.R` script.

### Basic Command
`genomescope.R -i reads.histo -o output_dir -k 21`

### Parameters
- `-i`: Path to the input histogram file (Required).
- `-o`: Path to the output directory (Required).
- `-k`: k-mer length used during counting (Required).
- `-p`: Ploidy level (Optional; default is 2).
- `-l`: Initial guess for average k-mer coverage (lambda).
- `-m`: Max k-mer coverage; excludes high-frequency k-mers (repeats/organelles) from the model fit.
- `-n`: Name prefix for output files.

## Best Practices and Expert Tips
- **k-mer Length Selection**: Use `k=21` for most standard genomes. For extremely large (>10GB) or highly repetitive genomes, increase k to improve uniqueness.
- **Coverage Requirements**: Ensure at least 15x haploid genome coverage. Lower coverage often leads to model convergence failures.
- **Sequencing Quality**: Use only low-error rate data (e.g., Illumina). Raw long-reads (Nanopore/PacBio) with 5-15% error rates are not supported because errors disrupt k-mer matching at standard lengths.
- **Model Validation**: Always inspect the generated linear and log-scale plots. A "good" fit will show the model peaks (red lines) overlapping closely with the observed histogram peaks (blue bars).
- **Ploidy Adjustment**: If the default diploid model (p=2) fails to capture the peak structure, provide an explicit ploidy estimate based on known cytogenetics of the organism.
