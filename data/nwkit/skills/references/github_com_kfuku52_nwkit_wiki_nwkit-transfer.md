# nwkit transfer

## Overview

`nwkit transfer` transfers information (node names, support values, or branch lengths) from one tree (`--infile2`) to another (`--infile`). The two trees must have the same set of leaf names, but their topologies may differ. Nodes are matched by comparing the sets of descendant leaf names.

* `--name yes` transfers node names
* `--support yes` transfers support values
* `--length yes` transfers branch lengths
* `--target` specifies which nodes to update: `all`, `root`, `leaf`, or `intnode`
* `--fill` specifies a fill value for nodes that have no corresponding match in `--infile2`

## Examples

### Transfer node names

#### Command

```
nwkit transfer --infile input1.nwk --infile2 input2.nwk --name yes --target intnode --outfile output.nwk
```

#### input1.nwk

```
((A:1,B:2):3,(C:4,D:5):6);
```

#### input2.nwk

```
((A:0.1,B:0.2)clade_AB:0.3,(C:0.4,D:0.5)clade_CD:0.6)root;
```

#### output.nwk

```
((A:1,B:2)clade_AB:3,(C:4,D:5)clade_CD:6)root:0;
```

Internal node names (`clade_AB`, `clade_CD`, `root`) are transferred from `input2.nwk` to `input1.nwk`. Leaf names and branch lengths from `input1.nwk` are preserved.

### Transfer branch lengths with --fill

When the two trees have different topologies, some nodes in `--infile` may not have a corresponding node in `--infile2`. The `--fill` option specifies a value to use for those unmatched nodes.

#### Command

```
nwkit transfer --infile input1.nwk --infile2 input2.nwk --length yes --fill 0.0 --outfile output.nwk
```

#### input1.nwk

```
(((A:1,B:1)n1:1,C:1)n2:1,(D:1,E:1)n3:1)root;
```

#### input2.nwk

```
(((A:2,C:3)x:4,B:5)y:6,(D:7,E:8)z:9)root;
```

#### output.nwk

```
(((A:2,B:5)n1:0,C:3)n2:7.5,(D:7,E:8)n3:7.5)root:0;
```

Branch lengths from `input2.nwk` are transferred where possible. The clade `(A,B)` in `input1.nwk` has no matching clade in `input2.nwk` (which groups `(A,C)` instead), so its branch length is filled with `0.0`.
