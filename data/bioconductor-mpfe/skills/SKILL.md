# bioconductor-mpfe

## Overview

The `MPFE` (Methylation Patterns Frequency Estimation) package provides a statistical framework to estimate the true distribution of DNA methylation patterns. It corrects for two primary sources of error in bisulfite sequencing:
1. **Incomplete conversion**: Unmethylated cytosines that fail to convert to uracil.
2. **Sequencing errors**: Misreads during the sequencing process.

The package uses a maximum log-likelihood approach (via `constrOptim`) to distinguish between true biological patterns and spurious patterns generated by these errors.

## Core Workflow

### 1. Data Preparation
Input data must be a data frame where:
- The first column contains the methylation patterns (e.g., "m00010" where 0 is unmethylated and 1 is methylated).
- Subsequent columns contain the counts for each pattern across different samples.

```r
library(MPFE)
data(patternsExample)
# patternsExample contains 'mPattern', 'k1', 'k2'
```

### 2. Estimating Patterns
Use `estimatePatterns()` to calculate the distribution.

**Key Parameters:**
- `epsilon`: The non-conversion rate (e.g., 0.01 for 99% efficiency).
- `eta`: Sequencing error rate. Can be a single value or a vector with length equal to the number of CpG sites.
- `column`: The index of the count column to analyze (defaults to all count columns).
- `fast`: Boolean. If `TRUE` (default), it ignores patterns with zero observed counts to speed up computation.

```r
# Single sample analysis
result <- estimatePatterns(patternsExample, epsilon=0.01, eta=0.005, column=2)

# Multi-site dependent error rates
site_errors <- c(0.008, 0.01, 0.01, 0.01, 0.008)
results_list <- estimatePatterns(patternsExample, epsilon=0.01, eta=site_errors)
```

### 3. Interpreting Results
The function returns a list of data frames (one per sample). Each data frame includes:
- `pattern`: The CpG methylation state.
- `observedDistribution`: The raw frequency in the data.
- `estimatedDistribution`: The corrected frequency.
- `spurious`: A boolean indicating if the pattern is likely an artifact of error.

### 4. Visualization
The package provides two main plotting functions:

**Comparison Plot:**
`plotPatterns()` compares observed vs. estimated frequencies. It produces two panes: a full-scale plot and an expanded view of lower-frequency patterns.
```r
# Plot results for the first sample in the list
plotPatterns(results_list[[1]])
```

**Pattern Map:**
`patternMap()` creates a heatmap-style visualization of the patterns and their proportions.
```r
# Plot estimated frequencies with a frequency cutoff
patternMap(results_list[[1]], 
           estimatedDistribution = TRUE, 
           maxFreq = 0.5, 
           main = "Estimated Frequencies")
```

## Tips and Constraints
- **Pattern Format**: Ensure patterns in the input data frame are strings of 0s and 1s. The package often expects a leading 'm' or similar prefix which it strips during processing.
- **Convergence**: If the estimation fails to converge, you can adjust `steps` and `reltol` parameters, which are passed to the underlying optimizer.
- **Spurious Patterns**: Patterns marked as `TRUE` in the `spurious` column have an estimated true frequency of zero (or near-zero), suggesting they are entirely products of non-conversion or sequencing error.
