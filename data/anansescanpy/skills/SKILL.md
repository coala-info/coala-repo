# anansescanpy

## Overview
anansescanpy is a Python implementation of the scANANSE workflow, specifically designed to integrate with the Scanpy ecosystem. It allows researchers to infer cell-type-specific gene regulatory networks by combining single-cell expression data and chromatin accessibility data. This skill provides the necessary commands and procedural knowledge to bridge Scanpy objects with the ANANSE analysis pipeline, typically executed via the anansnake Snakemake workflow.

## Installation and Setup
Install the package via Bioconda or PyPI. Bioconda is recommended for managing bioinformatics dependencies.

```bash
# Via Conda (recommended)
conda config --add channels defaults
conda config --add channels bioconda
conda config --add channels conda-forge
conda install anansescanpy

# Via PyPI
pip install anansescanpy
```

## Core Workflow
The typical workflow involves preparing Scanpy AnnData objects (.h5ad), exporting them for analysis, running the GRN inference, and importing the results.

### 1. Data Preparation
Ensure you have separate AnnData objects for RNA and ATAC data. These should be clustered and annotated.
- **RNA data**: `rna_PBMC.h5ad`
- **ATAC data**: `atac_PBMC.h5ad`

### 2. Running the GRN Analysis
The heavy computational work is handled by `anansnake`. Use the following Snakemake pattern to execute the analysis using the configuration generated by the tool:

```bash
snakemake --use-conda --conda-frontend mamba \
  --configfile scANANSE/analysis/config.yaml \
  --snakefile scANANSE/anansnake/Snakefile \
  --resources mem_mb=48000 --cores 12
```

### 3. Python Integration
Within a Python/Jupyter environment, use the package to handle the transition between Scanpy and ANANSE. The package includes modules for:
- `anansescanpy.export`: Preparing files for the Snakemake pipeline.
- `anansescanpy.import`: Loading GRN scores and transcription factor activities back into `adata.uns` or `adata.obs`.
- `anansescanpy.motif`: Handling motif-specific analysis.

## Expert Tips and Best Practices
- **Layer Selection**: Be aware that by default, the tool may take data from `adata.X`. If your normalized/scaled data is in `adata.X` but you require raw counts for specific steps, ensure you are pointing to the correct layer. (Note: Issue #28 in the repository indicates a known behavior where `adata.X` is prioritized over `adata.raw.X`).
- **Resource Management**: GRN inference is memory-intensive. When running the Snakemake pipeline, allocate at least 48GB of RAM (`mem_mb=48000`) for standard single-cell datasets.
- **Cluster Consistency**: Ensure that the cluster names/identifiers are identical between your RNA and ATAC AnnData objects to allow proper integration during the GRN inference.
- **Environment**: Always use `--use-conda` with Snakemake to ensure that the specific versions of ANANSE and its dependencies are correctly managed.
