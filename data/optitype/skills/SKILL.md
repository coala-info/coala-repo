# optitype

## Overview

OptiType is a specialized bioinformatic tool designed for high-accuracy HLA typing. It works by formulating the allele selection problem as an optimization task, allowing it to simultaneously consider all major and minor HLA Class I alleles. It is particularly useful for researchers needing 4-digit resolution from standard sequencing reads (FASTQ or BAM) and supports both single-end and paired-end data.

## Usage Guidelines

### Core Command Structure

The primary entry point is `OptiTypePipeline.py`. You must specify the input files, the sequencing type (DNA or RNA), and an output directory.

**DNA Sequencing (Exome/WGS):**
```bash
python OptiTypePipeline.py -i sample_1.fastq sample_2.fastq --dna -o ./output_dir/
```

**RNA Sequencing:**
```bash
python OptiTypePipeline.py -i sample_1.fastq sample_2.fastq --rna -o ./output_dir/
```

### Input Formats
- **FASTQ**: Raw or "fished" (filtered) sequencing reads.
- **BAM**: You can provide intermediate BAM files generated by a previous OptiType run to skip the mapping step and re-run the ILP solver with different parameters.

### Key Parameters
- `--beta B`: Controls the threshold for homozygosity detection. The default is 0.009. Adjust this with caution if the sample is expected to have high homozygosity or if allele calls seem merged.
- `--enumerate N`: Outputs the optimal solution plus the top N-1 suboptimal solutions. This is useful for assessing the confidence of the top hit.
- `--prefix`: Allows setting a custom string for output filenames instead of the default timestamp.

## Expert Tips and Best Practices

### The "Fishing" Pre-processing Step
For large datasets (multi-gigabyte FASTQ files), it is highly recommended to filter for HLA-relevant reads first. This reduces the processing time from hours to minutes.

1. **Map to HLA reference**: Use RazerS3 to "fish" reads.
   ```bash
   razers3 -i 95 -m 1 -dr 0 -o fished_1.bam data/hla_reference_dna.fasta sample_1.fastq
   ```
2. **Convert to FASTQ**:
   ```bash
   samtools bam2fq fished_1.bam > sample_1_fished.fastq
   ```
3. **Important**: If using paired-end data, filter the two ends **individually**. Do not use the read mapper's paired-end capabilities during the fishing step.

### Docker Execution
If the environment lacks the complex dependencies (HDF5, GLPK/CBC, Pyomo), use the official Docker image. Ensure you mount your local data directory to the container.

```bash
docker run -v /path/to/data:/data -t fred2/optitype -i /data/input_1.fq /data/input_2.fq --dna -o /data/output/
```

### Interpreting Outputs
- **CSV File**: Contains the predicted HLA-A, HLA-B, and HLA-C alleles.
- **PDF Coverage Plot**: Always inspect this file. It provides a visual representation of read coverage across the predicted alleles, which is essential for diagnosing false positives or low-confidence calls.
