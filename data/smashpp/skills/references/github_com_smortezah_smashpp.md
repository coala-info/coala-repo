Installation To install Smash++, you'll need CMake version 3.5 or higher, along with a compiler that supports C++14. Docker # Pull the image docker pull smortezah/smashpp ; # Run the container docker run -it smortezah/smashpp Conda Begin by installing Miniconda . Once installed, execute the following commands: conda install -c bioconda -y smashpp Ubuntu # Install Git, CMake and g++ apt update &amp;&amp; apt install -y git cmake g++ ; # Clone the repository and install it git clone --depth 1 https://github.com/smortezah/smashpp.git ; cd smashpp ; bash install.sh macOS # Install Homebrew, Git and CMake /bin/bash -c " $( curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh ) " ; brew install git cmake ; # Clone the repository and install it git clone --depth 1 https://github.com/smortezah/smashpp.git ; cd smashpp ; bash install.sh Windows First, set up WSL (Windows Subsystem for Linux). After that, clone the Smash++ repository and proceed with the installation as you would on Ubuntu: git clone --depth 1 https://github.com/smortezah/smashpp.git ; cd smashpp ; ./install.sh Note If you encounter permission issues on any operating system, consider using the sudo command. Usage ./smashpp [OPTIONS] -r &lt; REF_FILE &gt; -t &lt; TAR_FILE &gt; For example: ./smashpp -r ref -t tar For optimal results, it's advisable to select short names for both reference and target sequences. Options To explore the available options for Smash++, execute the following command: ./smashpp which will yield the following: SYNOPSIS ./smashpp [OPTIONS] -r &lt;REF_FILE&gt; -t &lt;TAR_FILE&gt; SAMPLE ./smashpp -r ref -t tar -l 0 -m 1000 ./smashpp \ --reference ref \ --target tar \ --format json \ --verbose OPTIONS Required: -r, --reference &lt;FILE&gt; reference file (Seq/FASTA/FASTQ) -t, --target &lt;FILE&gt; target file (Seq/FASTA/FASTQ) Optional: -l, --level &lt;INT&gt; level of compression: 0 to 6. Default: 3 -m, --min-segment-size &lt;INT&gt; minimum segment size: 1 to 4294967295. Default: 50 -fmt, --format &lt;STRING&gt; format of the output (position) file: {pos, json}. Default: pos -e, --entropy-N &lt;FLOAT&gt; entropy of 'N's: 0.0 to 100.0. Default: 2.0 -n, --num-threads &lt;INT&gt; number of threads: 1 to 255. Default: 4 -f, --filter-size &lt;INT&gt; filter size: 1 to 4294967295. Default: 100 -ft, --filter-type &lt;INT/STRING&gt; filter type (windowing function): {0/rectangular, 1/hamming, 2/hann, 3/blackman, 4/triangular, 5/welch, 6/sine, 7/nuttall}. Default: hann -fs, --filter-scale &lt;STRING&gt; filter scale: {S/small, M/medium, L/large} -d, --sampling-step &lt;INT&gt; sampling step. Default: 1 -th, --threshold &lt;FLOAT&gt; threshold: 0.0 to 20.0. Default: 1.5 -rb, --reference-begin-guard &lt;INT&gt; reference begin guard: -32768 to 32767. Default: 0 -re, --reference-end-guard &lt;INT&gt; reference ending guard: -32768 to 32767. Default: 0 -tb, --target-begin-guard &lt;INT&gt; target begin guard: -32768 to 32767. Default: 0 -te, --target-end-guard &lt;INT&gt; target ending guard: -32768 to 32767. Default: 0 -ar, --asymmetric-regions consider asymmetric regions. Default: not used -nr, --no-self-complexity do not compute self complexity. Default: not used -sb, --save-sequence save sequence (input: FASTA/FASTQ). Default: not used -sp, --save-profile save profile (*.prf). Default: not used -sf, --save-filtered save filtered file (*.fil). Default: not used -ss, --save-segmented save segmented files (*.s[i]). Default: not used -sa, --save-profile-filtered-segmented save profile, filetered and segmented files. Default: not used -rm, --reference-model k,[w,d,]ir,a,g/t,ir,a,g:... -tm, --target-model k,[w,d,]ir,a,g/t,ir,a,g:... parameters of models &lt;INT&gt; k: context size &lt;INT&gt; w: width of sketch in log2 form, e.g., set 10 for w=2^10=1024 &lt;INT&gt; d: depth of sketch &lt;INT&gt; ir: inverted repeat: {0, 1, 2} 0: regular (not inverted) 1: inverted, solely 2: both regular and inverted &lt;FLOAT&gt; a: estimator &lt;FLOAT&gt; g: forgetting factor: 0.0 to 1.0 &lt;INT&gt; t: threshold (number of substitutions) -ll, --list-levels list of compression levels -h, --help usage guide -v, --verbose more information -V, --version show version To explore the options available for the Smash++ Visualizer, enter the following command: ./smashpp viz or ./smashpp -viz which will yield the following: SYNOPSIS ./smashpp viz|-viz [OPTIONS] -o &lt;SVG_FILE&gt; &lt;POS_FILE&gt; SAMPLE ./smashpp -viz -o simil.svg ref.tar.pos ./smashpp viz \ --output similarity.svg \ --vertical-view \ ref.tar.json OPTIONS Required: &lt;POS_FILE&gt; position file generated by Smash++ (*.pos/*.json) Optional: -o, --output &lt;SVG_FILE&gt; output image name (*.svg). Default: map.svg -rn, --reference-name &lt;STRING&gt; reference name shown on output. If it has spaces, use double quotes, e.g. "Seq label". Default: the name in the header of position file -tn, --target-name &lt;STRING&gt; target name shown on output -l, --link &lt;INT&gt; type of the link between maps: 1 to 6. Default: 1 -c, --color &lt;INT&gt; color mode: {0, 1}. Default: 0 -p, --opacity &lt;FLOAT&gt; opacity: 0.0 to 1.0. Default: 0.9 -w, --width &lt;INT&gt; width of the sequence: 8 to 100. Default: 10 -s, --space &lt;INT&gt; space between sequences: 5 to 200. Default: 40 -tc, --total-colors &lt;INT&gt; total number of colors: 1 to 255 -rt, --reference-tick &lt;INT&gt; reference tick: 1 to 4294967295 -tt, --target-tick &lt;INT&gt; target tick: 1 to 4294967295 -th, --tick-human-readable &lt;INT&gt; tick human readable: {0: false, 1: true}. Default: 1 -m, --min-block-size &lt;INT&gt; minimum block size: 1 to 4294967295. Default: 1 -vv, --vertical-view vertical view. Default: not used -nrr, --no-relative-redundancy do not show relative redundancy (relative complexity). Default: not used -nr, --no-redundancy do not show redundancy. Default: not used -ni, --no-inverted do not show inverse maps. Default: not used -ng, --no-regular do not show regular maps. Default: not used -n, --show-N show 'N' bases. Default: not used -stat, --statistics save statistics (*.csv).
