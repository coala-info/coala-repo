# best

[Jump to bottom](#wiki-pages-box)

Nikos Pappas edited this page Dec 13, 2021
·
[1 revision](/MGXlab/cirtap/wiki/best/_history)

This module reads stats from the `genome_summary` file for all genomes, collapses taxids based on the NCBI taxonomy, applies some filtering rules and outputs a best genome per taxonomic rank.

## Usage

```
cirtap best -h

Usage: cirtap best [OPTIONS] OUTPUT_PATH

  Select best genomes based on stats retrieved from genome_summary

  Results are written in the provided output_path and include tables
  containing genome ids with their stats and lineages attached

Options:
  -i, --index-path PATH  Path to the index file  [required]
  -d, --db-dir PATH      Path to the local mirror. Must contain a
                         `RELEASE_NOTES` directory and a `genomes` directory
                         [required]
  --thresh INTEGER       Integer threshold for including a genome based on
                         completeness and contamination stats. This is useed
                         as completeness - 5*contamination > thresh  [default:
                         70]
  --ncbi-db PATH         Path to the taxa.sqlite created by ete3  [default:
                         /linuxhome/mgx/.etetoolkit/taxa.sqlite]
  --loglevel TEXT        Define loglevel  [default: INFO]
  --logfile TEXT         Write logging information in this file
  -h, --help             Show this message and exit.
```

## Description

The taxonomy information is loaded as a taxonomy tree with the help of ete3's [NCBITaxa()](http://etetoolkit.org/docs/latest/tutorial/tutorial_ncbitaxonomy.html).
You can use your own copy of the taxa.sqlite created by ete3 if you want it frozen to a particular "version". Otherwise, ete3 will initialize a database under `~/.etetoolkit` the first time it is run and search for this database in subsequent invocations.

All taxids are grouped based on the available taxonomic tree and the original `genome_summary` table is iteratively filtered to produce subsets of genomes that have the highest scores based on completeness and contamination.
For a genome to be considered it needs to satisfy the rule `completeness - 5*contamination > threshold`, where threshold is a configurable integer value that defaults to 70.

## Output

The output directory will contain 8 tables, 1 with the best genome per taxonomy id and 7 for the official taxonomic ranks (superkingdom, phylum, class, order, family, genus, species). Files are named accordingly and contain the stats from the `genome_summary` concatenated with the information found in the `genome_lineage` file.
