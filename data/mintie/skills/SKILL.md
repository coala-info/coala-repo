## Overview
MINTIE is a powerful bioinformatics tool designed to uncover novel and rare transcriptional variants and isoforms within RNA-seq data, particularly in the context of cancer and rare disease research. It excels at identifying various types of genetic alterations, including fusions (both standard exon-exon and those involving non-exonic regions), transcribed structural variants (like insertions, deletions, and intergenic rearrangements), and novel splice variants that deviate from the reference genome. MINTIE can also characterize complex variants by annotating all novel transcriptional alterations on a given assembled contig.

## Usage Instructions

MINTIE is a command-line tool. The primary executable is `MINTIE.groovy`. Configuration and parameters are typically managed through a `params.txt` file.

### Core Functionality and Parameters

The main `MINTIE.groovy` script orchestrates the analysis. Key parameters often include:

*   **Input Files**:
    *   `--reads`: Path to the input RNA-seq reads (e.g., FASTQ files). Can be single-end or paired-end.
    *   `--ref`: Path to the reference genome FASTA file.
    *   `--gtf`: Path to the reference gene annotation GTF file.
*   **Output**:
    *   `--outdir`: Directory to save the output files.
*   **Configuration**:
    *   `--params`: Path to a `params.txt` file containing specific analysis parameters. This is the recommended way to manage complex settings.

### Running MINTIE

A typical command structure involves specifying the input files, reference, annotation, and output directory, often with a parameter file:

```bash
groovy MINTIE.groovy \
  --reads path/to/your/reads.fq \
  --ref path/to/your/reference.fa \
  --gtf path/to/your/annotation.gtf \
  --outdir path/to/output/directory \
  --params path/to/your/params.txt
```

### Parameter File (`params.txt`)

The `params.txt` file allows for fine-grained control over MINTIE's behavior. Common parameters you might find or need to set include:

*   **Variant Calling Parameters**:
    *   `min_novel_splice_variant_score`: Minimum score for a novel splice variant to be reported.
    *   `min_fusion_score`: Minimum score for a fusion event to be reported.
    *   `min_sv_length`: Minimum length for structural variants.
*   **Reference and Annotation Specifics**:
    *   `assembly_fasta`: Alias for the reference FASTA.
    *   `annotation_gtf`: Alias for the annotation GTF.
*   **Computational Resources**:
    *   `threads`: Number of threads to use for parallel processing.

**Example `params.txt` snippet:**

```
min_novel_splice_variant_score = 0.8
min_fusion_score = 0.9
min_sv_length = 50
threads = 8
```

### Expert Tips

*   **Use `params.txt`**: Always manage your parameters via a `params.txt` file for reproducibility and clarity. This is crucial for complex analyses.
*   **Reference Genome and Annotation**: Ensure your reference genome and GTF annotation files are compatible and up-to-date for the organism you are studying.
*   **Read Type**: MINTIE can handle both single-end and paired-end reads. Ensure your `--reads` argument is formatted correctly for your data type.
*   **Output Interpretation**: Familiarize yourself with the output files generated by MINTIE, particularly the VCF files and summary reports, to interpret the detected variants. The wiki provides details on output formats.
*   **Dependencies**: MINTIE may have external dependencies (e.g., Bpipe, specific bioinformatics tools). Ensure these are installed and accessible in your environment. The `install_linux64.sh` script might be relevant for setting up dependencies.
