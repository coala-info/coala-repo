## [Preprocessing Reads](#id1)

Debarcer requires that reads be reheadered before any analysis. Reheadering `fastq` files before alignment and indexing can be achieved using the `preprocess` command. Example:

```
$ python debarcer.py preprocess -o /path/to/output_dir -r1 /path/to/read1.fastq
   -r2 /path/to/read2.fastq -prepname "prepname"
   -prepfile /path/to/library_prep_types.ini -c /path/to/config.ini
```

Arguments:
:   * **-o/--output\_path:** the path where the reheadered fastq files will be written. **(Required)**
    * **-r1/--read1:** the path to your first fastq file (r1 in a paired-end library). **(Required)**
    * **-r2/--read2:** the path to your second fastq file, if applicable.
    * **-r3/--read3:** the path to your third fastq file, if applicable.
    * **-p/--prepname:** the name of the library prep used (must be defined in prepfile!). **(Required)**
    * **-pf/--prepfile:** path to your library\_prep\_types.ini file. If this is not provided, Debarcer will look for a path provided in your config.
    * **-c/--config:** path to your config file, if used.

For more information on the library\_prep\_types.ini file and how to create a new library definition, see [Defining Library Types](/oicr-gsi/debarcer/wiki/3.-Defining-Library-Types).

## [UMI Grouping & Error Correction](#id2)

After aligning and indexing reheadered `fastq` files into `bam` (and `bam.bai`) files, you can perform some UMI grouping and error correction on them before collapse. This is achieved with the `group` command. Example:

```
$ python debarcer.py group -o /path/to/output_dir -r chrN:posA-posB
  -b /path/to/bam_file.bam -c /path/to/config.ini
```

Arguments:
:   * **-o/--output\_path:** the path where the output files will be written. **(Required)**
    * **-r/--region:** region to find UMIs in (string of the form chrX:posA-posB). **(Required)**
    * **-b/--bam\_file:** path to your BAM file. If this is not provided, Debarcer will look for a path provided in your config.
    * **-c/--config:** path to your config file, if used.

This command uses the clustering method created by UMI-tools (<https://github.com/CGATOxford/UMI-tools>) to build a network of UMIs and group those that have a high likelihood of being sequencing errors with their correct UMI parent. It then further groups UMIs into positional families so that, for instance, the UMI `AACG` at `chrX:1000` is considered to be a distinct family from the UMI `AACG` at `chrX:100000`. The exact distance threshold can be changed by modifying `umi_edit_distance_threshold` in your config file (10bp by default).

This process will generate a `.umis` binary file that will be used in the subsequent command. If you do not wish to perform any error correction, simply skip this step and do not provide a `.umis` file.

## [Generating Consensus](#id3)

Base collapsing is handled by the `collapse` command. Example:

```
$ python debarcer.py collapse -o /path/to/output_dir -r chrN:posA-posB
  -b /path/to/bam_file.bam -u /path/to/umi_file.umis
  -c /path/to/config.ini
```

Arguments:
:   * **-o/--output\_path:** the path where the output files will be written. **(Required)**
    * **-r/--region:** region to find UMIs in (string of the form chrX:posA-posB). **(Required)**
    * **-b/--bam\_file:** path to your BAM file. If this is not provided, Debarcer will look for a path provided in your config.
    * **-u/--umi\_file:** path to your .umis file. This is created by the `group` command. Leave unspecified if grouping is not desired.
    * **-c/--config:** path to your config file, if used.

This process will generate a `.cons` file, a plain-text file of sequential base position information. You can scroll through this file to manually look for areas of interest, or you can use the `call` command to further parse this file into the standard VCF format.

## [Variant Calling](#id4)

After collapsing, VCF files can be generated from the given `.cons` file. Example:

```
$ python debarcer.py call -o /path/to/output_dir -r chrN:posA-posB
  -cf /path/to/cons_file.cons -f 1,2,5 -c path/to/config.ini
```

Arguments:
:   * **-o/--output\_path:** the path where the output files will be written. **(Required)**
    * **-r/--region:** region to find UMIs in (string of the form chrX:posA-posB). **(Required)**
    * **-cf/--cons\_file:** path to your .cons file. This is generated by the `collapse` command. **(Required)**
    * **-f/--f\_sizes:** a comma-separated list of family sizes to generate VCF files for. **(Required)**
    * **-c/--config:** path to your config file, if used.

A VCF file for each specified family size will be created in your output directory. Note that these family sizes must correspond with sizes used with the `collapse` command. By default this is 0 (uncollapsed), 1, 2, and 5, but these can be changed by modifying `min_family_sizes` in your config file.

## [Run UMI Grouping, Collapse and Variant Calling Across Multiple Intervals](#id5)

Provided a BED file with multiple regions and an aligned, index & sorted BAM file, this module will run UMI grouping, collapse and variant calling across all intervals, in parallel. UMI files & merged UMI data file, CONS files & merged CONS file, and VCF files will be generated. Example:

```
$ python debarcer.py run-o /path/to/output_dir -b /path/to/bam_file.bam
  -be /path/to/bed_file.bed -c path/to/config.ini
```

Arguments:
:   * **-o/--output\_path:** the path where the output .umi, .cons & .vcf files will be written. **(Required)**
    * **-b/--bam\_file:** path to your .bam file. If this is not provided, Debarcer will look for a path provided in your config. **(Required)**
    * **-be/--bed\_file:** path to your .bed file. **(Required)**
    * **-id/--run\_id:** Unique Run ID. **(Required)**
    * **-c/--config:** path to your config file, if used.

## [Plot](#id6)

After generating UMI data file(s) and CONS file(s) plots can be generated for each data file. Example:

```
$ python debarcer.py plot -o /path/to/output_dir -c /path/to/cons_file.cons
  -u /path/to/umi_datafile.vcf -uf umi_flag -cf cons_flag
```

Arguments:
:   * **-o/--output\_path:** the path where the reheadered fastq files will be written. **(Required)**
    * **-c/--cons\_file:** path to .cons file.
    * **-u/--umi\_datafile:** path to your .vcf umi data file.
    * **-uf/--umi\_flag:** pass 'all' or 'rs' as a parameter, to generate all or region-specific umi plots, respectively.
    * **-cf/--cons\_flag:** pass 'all' as a parameter, to generate all cons plots.
