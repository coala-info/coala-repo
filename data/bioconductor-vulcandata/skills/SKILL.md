# bioconductor-vulcandata

## Overview
The `vulcandata` package is an experiment data package designed to support the `vulcan` analysis pipeline. It contains a subset of ChIP-Seq data from MCF7 cell lines treated with estradiol at different time points (0, 45, and 90 minutes). The data is restricted to chromosome 22 to ensure computational feasibility for examples. It includes BAM, BED, and BAI files, a dummy regulatory network (regulon object), and utility functions to generate sample sheets compatible with `DiffBind`.

## Installation
To install the package, use the BiocManager:

```r
if (!requireNamespace("BiocManager", quietly = TRUE))
    install.packages("BiocManager")
BiocManager::install("vulcandata")
```

## Typical Workflow

### 1. Loading the Package
```r
library(vulcandata)
```

### 2. Generating a Sample Sheet
The `vulcansheet` function creates a CSV file that acts as a sample sheet for the internal dataset. This sheet follows the `DiffBind` format, which is required for the initial steps of a VULCAN analysis.

```r
# Define the output file name
vfile <- "mysamplesheet.csv"

# Generate the system-specific sample sheet
vulcansheet(vfile)

# Read and inspect the generated sheet
samples <- read.csv(vfile)
head(samples)

# Clean up when finished
unlink(vfile)
```

### 3. Accessing the Example Regulatory Network
The package provides a dummy `regulon` object (compatible with the `viper` package) representing a regulatory network. This is used by VULCAN to project differential binding signatures onto transcription factor activities.

```r
library(viper)
# Load the network object from the package's external data
load(system.file('extdata', 'network.rda', package = 'vulcandata', mustWork = TRUE))

# Inspect the network object
print(network)
```

### 4. Locating Raw Data Files
If you need to access the raw BAM or BED files for testing `DiffBind` or `csaw` functions manually, they are located in the `extdata` directory of the package.

```r
# List all files in the extdata directory
list.files(system.file("extdata", package = "vulcandata"))
```

## Usage Tips
- **VULCAN Pipeline**: This package is intended to be used alongside the `vulcan` package. The sample sheet generated by `vulcansheet` points to the absolute paths of the BAM files on your local system where the package is installed.
- **Data Scope**: Remember that the provided data only contains peaks for Chromosome 22. Any analysis attempting to look for peaks on other chromosomes using this specific dataset will return empty results.
- **Regulon Format**: The `network` object is a list where each element represents a Transcription Factor and its targets, formatted for use with the `viper` integration in VULCAN.
