# bioconductor-cytopipelinegui

## Overview

`CytoPipelineGUI` is the companion visualization package for `CytoPipeline`. It provides interactive Shiny applications and programmatic plotting functions to inspect, compare, and refine flow cytometry pre-processing pipelines. Its primary purpose is to allow users to visualize the state of data (flowFrames) at any given step in a pipeline and to manually tune scale transformations when automated methods are insufficient.

## Core Workflows

### 1. Interactive Pipeline Inspection
The `CytoPipelineCheckApp` allows for the comparison of flow frames resulting from different `CytoProcessingSteps` within the same or different experiments.

```r
library(CytoPipelineGUI)

# Launch the Shiny app pointing to the CytoPipeline cache directory
CytoPipelineGUI::CytoPipelineCheckApp(dir = "path/to/workDir")
```

### 2. Interactive Scale Transformation
The `ScaleTransformApp` is used to visualize and manually adjust scale transformation parameters (e.g., logicle, linear). Adjusted parameters can be saved as RDS objects for reuse in future pipelines.

```r
# Launch the Shiny app for scale adjustment
CytoPipelineGUI::ScaleTransformApp(dir = "path/to/workDir")
```

### 3. Programmatic Visualization
You can generate plots without the Shiny interface using exported functions. This is useful for reports or comparing specific pipeline outputs.

**Plotting a Workflow:**
```r
CytoPipelineGUI::plotSelectedWorkflow(
    experimentName = "MyExperiment",
    whichQueue = "pre-processing",
    sampleFile = "sample_1.fcs",
    path = "path/to/workDir"
)
```

**Comparing Flow Frames:**
Use `plotSelectedFlowFrame` to view a specific state and `plotDiffFlowFrame` to see the difference between two states (e.g., comparing flowAI vs PeacoQC results).

```r
# Plot state from Experiment A
p1 <- plotSelectedFlowFrame(experimentName = "ExpA", flowFrameName = "QC_obj", path = workDir)

# Plot state from Experiment B
p2 <- plotSelectedFlowFrame(experimentName = "ExpB", flowFrameName = "QC_obj", path = workDir)

# Plot the difference
p3 <- plotDiffFlowFrame(
    path = workDir,
    experimentNameFrom = "ExpA", flowFrameNameFrom = "QC_obj",
    experimentNameTo = "ExpB", flowFrameNameTo = "QC_obj"
)

library(patchwork)
p1 + p2 + p3
```

## Key Functions

| Function | Purpose |
| :--- | :--- |
| `CytoPipelineCheckApp()` | Launches Shiny app for flowFrame comparison and pipeline step inspection. |
| `ScaleTransformApp()` | Launches Shiny app for interactive scale transformation tuning. |
| `plotSelectedWorkflow()` | Visualizes the sequence of steps in a specific pipeline queue. |
| `plotSelectedFlowFrame()` | Generates a 2D plot of a flowFrame at a specific pipeline step. |
| `plotDiffFlowFrame()` | Generates a comparison plot between two flowFrames. |
| `plotScaleTransformedChannel()` | Programmatically plots a channel with specific transformation parameters applied. |

## Usage Tips
- **Cache Dependency:** `CytoPipelineGUI` relies on the cache generated by `CytoPipeline`. Ensure your pipeline has been executed with `execute(..., rmCache = FALSE)` before attempting to visualize results.
- **Channel Labels:** When using programmatic plotting functions, ensure `xChannelLabel` and `yChannelLabel` match the names in the flowFrame (e.g., "Time", "FSC-A").
- **Manual Scaling:** If automated scaling fails to produce a "pretty" plot, use `ScaleTransformApp` to find the optimal `t`, `m`, `w`, and `a` parameters for logicle transformations.
