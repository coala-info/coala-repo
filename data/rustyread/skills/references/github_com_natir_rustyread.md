Rustyread is very heavily inspired by badread , it reuses the same error and quality model file. But Rustyreads is multi-threaded and benefits from other optimizations. Usage Installation Minimum supported Rust version Difference with badread WARNING : Rustyread has not yet been evaluated or even compared to any other long read generators Rustyread is tested only on Linux Rustyread is still in developpement many thing can change or be break Usage If previously you called badread like this: badread simulate --reference {reference path} --quantity {quantity} &gt; {reads}.fastq you can now replace badread by rustyread: rustyread simulate --reference {reference path} --quantity {quantity} &gt; {reads}.fastq But by default rustyread use all avaible core you can control it with option threads : rustyread --theads {number of thread} simulate --reference {reference path} --quantity {quantity} &gt; {reads}.fastq If you have badread installed in your python sys.path rustyread can found error and quality model automatically, but you can still use --error_model and --qscore_model option. Control memory usage Rustyread memory usage could be estimated with formula: 2 * reference base + 2 * targeted base + epsilon , to limit memory impact of Rustyread you can use parameter number_base_store it's take an absolute value or a relative depth, if this option is set memory usage became 2 * reference base + 2 number_base_store + epsilon . Full usage rustyread 0.4.1 Machamp Pierre Marijon &lt;pierre@marijon.fr&gt; A long read simulator based on badread idea and model USAGE: rustyread [OPTIONS] &lt;SUBCOMMAND&gt; OPTIONS: -h, --help Print help information -t, --threads &lt;THREADS&gt; Number of thread use by rustyread, 0 use all avaible core, default value 0 -v, --verbosity Verbosity level also control by environment variable RUSTYREAD_LOG if flag is set RUSTYREAD_LOG value is ignored -V, --version Print version information SUBCOMMANDS: help Print this message or the help of the given subcommand(s) simulate Generate fake long read rustyread-simulate Generate fake long read USAGE: rustyread simulate [FLAGS] [OPTIONS] --reference &lt;reference-path&gt; --quantity &lt;quantity&gt; FLAGS: -h, --help Prints help information --small_plasmid_bias If set, then small circular plasmids are lost when the fragment length is too high (default: small plasmids are included regardless of fragment length) -V, --version Prints version information OPTIONS: --chimera &lt;chimera&gt; Percentage at which separate fragments join together [default: 1] --end_adapter &lt;end-adapter&gt; Adapter parameters for read ends (rate and amount) [default: 50,20] --end_adapter_seq &lt;end-adapter-seq&gt; Adapter parameters for read ends [default: GCAATACGTAACTGAACGAAGT] --error_model &lt;error-model&gt; Path to an error model file [default: nanopore2020] --glitches &lt;glitches&gt; Read glitch parameters (rate, size and skip) [default: 10000,25,25] --identity &lt;identity&gt; Sequencing identity distribution (mean, max and stdev) [default: 85,95,5] --junk_reads &lt;junk&gt; This percentage of reads wil be low complexity junk [default: 1] --length &lt;length&gt; Fragment length distribution (mean and stdev) [default: 15000,13000] --number_base_store &lt;nb-base-store&gt; Number of base, rustyread can store in ram before write in output in absolute value (e.g. 250M) or a relative depth (e.g. 25x) --output &lt;output-path&gt; Where read is write --qscore_model &lt;qscore-model&gt; Path to an quality score model file [default: nanopore2020] --quantity &lt;quantity&gt; Either an absolute value (e.g. 250M) or a relative depth (e.g. 25x) --random_reads &lt;random&gt; This percentage of reads wil be random sequence [default: 1] --reference &lt;reference-path&gt; Reference fasta (can be gzipped, bzip2ped, xzped) --seed &lt;seed&gt; Random number generator seed for deterministic output (default: different output each time) --start_adapter &lt;start-adapter&gt; Adapter parameters for read starts (rate and amount) [default: 90,60] --start_adapter_seq &lt;start-adapter-seq&gt; Adapter parameters for read starts [default: AATGTACTTCGTTCAGTTACGTATTGCT] Installation Bioconda If you haven't bioconda setup follow this instruction conda|mamba install rustyread With rust environment If you haven't a rust environment you can use rustup or your package manager.
