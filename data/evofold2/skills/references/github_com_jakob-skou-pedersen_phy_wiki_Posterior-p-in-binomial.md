### Specification in phy(currently only available on ObsParam branch)

If we always observe N and X but they are sample specific we can get the posterior distribution on p. Maybe we want to use this fraction as a parameter in other factors, but want to include the uncertainty with which it is observed. This can be obtained in the phy framework using factors that subscribe to variables.
The variables that are subscribed should be put in a separate file following the same format as VarData(TODO: reference to VarData page). In this example this could look like

```
#test2SubVarData.txt
NAME:   N1      X1
id_1    50      5
id_2    45      3
id_3    20      4
id_4    90      10
id_5    84      9
id_6    10      1
```

In the factorPotentials.txt file we specify that we have a factor that "subscribes" to these two variables.

```
#factorPotentials.txt
NAME:           beta
TYPE:           discCont
BINS:           200
MIN:            0
MAX:            1
STATES:         1
DIST:           BETA
N:              N1
X:              X1
```

In the variables file we specify a single variable H1, that is the hidden variable p. Even if the factor is really a three-way factor, we only state it as a one-way factor, as we assume N and X is always observed and we are not interested in modelling them. Thus the factor graph specification looks like this:

```
#factorGraph.txt
NAME:           POSTH1
NB1:            H1
POT:            beta
```

The full example is the repository and you can run it and find for instance the most probable state of H1 running `dfgEval`.

```
cd src/test/dfgTrainBetaEx/
../../dfgEval test2VarData.tab -m - --subVarFile test2SubVarData.tab -s dfgSpec2/
```
