# alevin-fry

`alevin-fry` is a suite of tools for rapid, memory-frugal processing of single-cell sequencing data. It operates on RAD files to produce quantified count matrices.

## Core Workflow

The standard pipeline consists of three sequential steps:

1. **generate-permit-list**: Filters and identifies valid cell barcodes.
2. **collate**: Sorts and groups the RAD file records by the corrected cell barcodes.
3. **quant**: Performs UMI resolution and generates the final quantification matrix.

## Command Reference

### 1. Generate Permit List
Identifies which barcodes are "real" cells.

```bash
# Using a forced number of cells
alevin-fry generate-permit-list --input <RAD_DIR> --output-dir <OUTPUT_DIR> --force-cells <NUM_CELLS>

# Using an unfiltered permit list (e.g., from 10x Genomics)
alevin-fry generate-permit-list --input <RAD_DIR> --output-dir <OUTPUT_DIR> --unfiltered-pl <BARCODE_FILE>
```

### 2. Collate
Groups records by barcode to prepare for quantification. This step is required before `quant`.

```bash
alevin-fry collate --input-dir <PERMIT_LIST_DIR> --rad-dir <RAD_DIR> --threads <NUM_THREADS>
```

### 3. Quantify
Generates the gene-by-cell matrix.

```bash
# Standard quantification
alevin-fry quant --input-dir <COLLATED_DIR> --output-dir <QUANT_DIR> --tg-map <TXP_TO_GENE_MAP> --resolution <RESOLUTION_STRATEGY> --threads <NUM_THREADS>
```

**Resolution Strategies:**
- `cr-like`: CellRanger-like resolution.
- `em`: Expectation-Maximization based resolution.
- `cr-like-em`: Hybrid approach.

## USA Mode (Spliced/Unspliced/Ambiguous)
Crucial for single-nucleus RNA-seq (snRNA-seq) or velocity analysis. It quantifies spliced, unspliced, and ambiguous UMIs separately.

- **Requirement**: Must be enabled during both mapping (salmon/piscem) and quantification.
- **Command**: Add `--usa` flag to the `quant` command.
- **Output**: Produces a matrix with 3x the number of genes (suffixing gene IDs with S, U, or A).

## Expert Tips & Best Practices

- **Mapper Choice**: Prefer `piscem` over `salmon` for RAD file generation; it is faster and uses significantly less memory.
- **Memory Efficiency**: `alevin-fry` is designed to be extremely memory-frugal. If memory is an issue, ensure you are using the latest version and check if your index is sparse.
- **Downstream Integration**:
    - Use the `fishpond` R package to read USA-mode output into `SingleCellExperiment` objects.
    - Use `alevinqc` for quality control reports.
- **Reference Construction**: Use `pyroe` to build the "splici" (spliced + intronic) reference required for USA mode.
- **Empty Drops**: For high-sensitivity cell calling, use `--expect-cells` in `generate-permit-list` to provide a heuristic for the expected number of cells.
