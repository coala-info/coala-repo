GMAP-fusion works as follows: Given a de novo transcriptome assembly (from running Trinity , Oases , or DISCASM , GMAP-fusion aligns the transcripts using GMAP to identify chimeric (split) alignments. Candidate fusion transcripts identified by GMAP are then further examined to determine the original reads that support the junction in the fusion transcript, identifying the number of spanning paired-end RNA-Seq fragments that align as proper-pairs on both sides of the fusion junction, and those reads that align directly across the candidate fusion junction. The original reads are aligned to the candidate fusion transcripts using Bowtie2 . To eliminate likely false positives, extended segments of each fusion transcript region are realigned separately to the genome using GMAP, assessing the precision of the putative fusion breakpoint. Likely false positives are removed. (Note, this step is based on earlier observations and a related implementation in JAFFA , which leverages BLAT). Likely false positive fusions involving partners that contain sequence similarity are further excluded. This filtering process leverages FusionFilter . GMAP-fusion Installation Requirements The following tools and data resources are required for GMAP-fusion: Tools: GMAP v2017-11-15 Bowtie2 Note, GMAP-fusion comes bundled with FusionFilter and DISCASM Data: Data resources are provided for fusion-finding with human transcriptomes. Visit the FusionFilter website for links to data, or instructions on how to set up your own custom data resources for use with GMAP-fusion. 3 ways to get started: git clone https://github.com/GMAP-fusion/GMAP-fusion.git Conda installation Galaxy toolshed installation of the tool Run this tool via our Trinity CTAT Galaxy Portal . Running GMAP-fusion GMAP-fusion can be invoked like so: GMAP-fusion --transcripts /path/to/Transcripts.fasta \ --genome_lib_dir /path/to/FusionFilter/genome_lib_dir \ --left_fq reads_1.fq --right_fq reads_2.fq However, GMAP-fusion has additional options available, and full usage info is below: ############################################################################################################ # # Required: # # --transcripts|T &lt;string&gt; :transcript fasta file # # --genome_lib_dir &lt;string&gt; directory containing genome lib (see http://FusionFilter.github.io for details) # # --left_fq &lt;string&gt; :Illumina paired-end reads /1 # # --right_fq &lt;string&gt; :Illumina paired-end reads /2 # # Optional: # # --CPU &lt;int&gt; :number threads for GMAP (default 4) # # --min_chim_len|L &lt;int&gt; :minimum length for a chimeric alignment (default: 30) # # --output|o &lt;string&gt; :output directory name (default: GMAP_Fusion) # # # --min_J|J &lt;int&gt; :minimum number of junction frags (default: 1) # # --min_sumJS|S &lt;int&gt; :minimum sum (junction + spanning) frags (default: 2) # # --min_novel_junction_support &lt;int&gt; :minimum number of junction reads required for novel (non-reference) exon-exon junction support. # (default: 3) # --split_breakpoint_extend_length &lt;int&gt; :in assessing breakpoint quality, the length to extend each split sequence beyond # the proposed breakpoint.
