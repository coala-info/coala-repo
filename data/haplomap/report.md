### Original Help Text
```text
Convert VCF to NIEHS compact format

usage: convert [options] <in.vcf> 

Required arguments:
    in.vcf                Input sorted VCF file or stdin
    -o, --output          Output file name

Optional arguments:
    -s,  --samples         New sample order file. One name per line.
    -t,  --type            Select variant type: [snp|indel|sv]. Default: snp
    -q,  --qual            QUAL field of VCF file. Only keep variant > qual. Default 50. 
    -p,  --plink           Output tped, tfam file for plink. Default: false.

SNP only arguments (INDEL and SV have not effects!):
    -d,  --pl-diff         Phred-scaled genotype likelihood (PL) difference. Default 20.
                           GT's PL must at least pl-diff unit lower than any other PL value. 
                           The greater, the more '?' in the output. 
    -a,  --allelic-depth   Min allelic depth (AD) of samples. Default 3.
    -m,  --mapping-quality Min average mapping quality. Default 20. 
    -b,  --strand-bias     Max Phred-scaled pvalue for strand bias (the lower, the better). Default 50. 
    -r,  --ratio           Min ratio of (%MAX(AD) / %MAX(DP)). Default 0.1.

    -v,  --verbose
    -h,  --help
```

### Original Help Text
```text
Usage: eblocks [options]

Required arguments:
    -s, --strains            <name of strains file>   
                                  The same input file as (haplomap ghmap -p)
    -a, --alleles            <name of alleles file>   
                                  The file generated by (haplomap convert) 
    -g, --genes              <name of SNPs-to-genes annotation file> 
                                  The file generated by (haplomap annotate)
    -o, --output             <output haploblocks file>   
                                  Input file for (haplomap ghmap -b) 

Optional arguments:
    -p, --block-variant      <output variants coordinates>   
                                  Used for viewing variants in a certain haploblocks
                                  NOTE:: If set (-r) correctly, ref: 0, alternate: 1-3, unkwnon: ?. 
                                         If not, ref could be any of 0-3.
    -r, --reference          <name of reference genome name> 
                                  Input one of the strain names from the header of (-a) input. Default: C57BL/6J. 
                                  Only affect the pattern output of (-p). 
    -m, --minblock-variant        Minimum variants per block. Default 3.
    -n, --non-overlapping         Run non-overlapping haploblock finding algorthim
    -v, --verbose
    -h, --help
```


## haplomap_ghmap

Output gene-summaried results by default.

### Original Help Text
```text
usage: ghmap [options]

Required arguments:
    -p, --phenotypes       <phenotype file> 
                                The same input file of (eblocks -b)
    -b, --blocks           <haploblocks file >
                                The output file from (eblocks -o)
    -o, --output           <output file name>
                                Output gene-summaried results by default.

Optional arguments:
    -n, --name             <name of phenotype dataset> 
                                NOTE:: Add suffix `_SNP`|`_INDEL`|`_SV`(e.g. MPD123_SNP)
                                can help select correct CodonFlag in the output.
    -c, --categorical           phenotype (-p) is categorical
    -r, --relation         <genetic relation file> 
                                <.rel> file for population structure analysis.
                                n x n matrix with header line (startswith '#') contain sample names.
    -e, --expression       <name of file>
    -f, --filter_coding         Filter out non-coding blocks
    -g, --gene                  Output gene-summaried results. Default.
                                NOTE:: Only write the overlapped halpoblock with best pvalue/Fstat, representing all overlapped blocks. 
                                     The CodonFlag is an aggregated indicator showing that a gene has blocks with coding change. 
                                     The best block itself might not contain any coding changes. 
                                     Run the ghmap with -a/k/m tag will give you all overlapped blocks with correct CodonFlag. 
    -a, --gene_all_blocks       Output gene-oriented results of all blocks that overalp a gene. 
    -m, --gene_block            Output gene-oriented results block by block. almost the same to -a
    -k, --haploblocks           Output block-oriented results.
    -l, --pvalue_cutoff    <float>   Only write results with pvalue < cutoff. Default: 0.05
    -v, --verbose
    -h, --help

Required arg missing: blocks file name (-b)
```
