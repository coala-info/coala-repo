```text
INFO:    Environment variable SINGULARITY_CACHEDIR is set, but APPTAINER_CACHEDIR is preferred
INFO:    Using cached SIF image
Usage: dist -g <file> (or -p <file>) [-dssp <name>]
            [-r] [-nb] [-c value] [-m value] [-damp value]
            [-op <file>] [-og <file>] [-od <file>]  [-noe <file>] [-q] [-h] [-v] [-V]
 
dist defines a set of geometric constraints from which
structures can be generated by the program disco.
 
Command line options:
-g <file>    - name of input GROMOS87 coordinate file
-p <file>    - name of input PDB coordinate file (either
               -g or -p is mandatory, from these
               coordinates the distances are calculated)
-dssp <file> - DSSP can be used for determining
               secondary structure-related pairs
               <file> is the name of the dssp executable
-r           - retain hydrogen atoms (by default they will be removed)
-nb          - try to find alternatives for non-bonded interactions
               (by default the native contacts will be preserved). Warning: EXPERIMENTAL!
-c value     - cut-off radius (Angstroms) for
               non-bonded interacting pairs (default 4.0)
               (the cut-off distances are additional to the sum of VDW radii)
-m value     - minimum nr of distances to be defined for
               each atom (default 50, or 1 with -noe)
-damp value  - multiply each distance margin by value (default 1.0)
-op <file>   - output structure in PDB file format (default dist.pdb)
-og <file>   - output GROMOS87 coordinate file (default dist.gro)
-od <file>   - output file containing distances (default dist.dat)
-noe <file>  - input NOE restrictions
-q           - interpret zero occupancy as atom to keep fixed (this used to be the default)
-h           - displays this message
-v           - be verbose
-V           - write version information and exit
```

```text
INFO:    Environment variable SINGULARITY_CACHEDIR is set, but APPTAINER_CACHEDIR is preferred
INFO:    Using cached SIF image
Usage: disco [-d <file>] [-p <file>] [-n value] [-i value] [-r <file>]
             -op <file> (or -ox <file> or -ot <file>) [-or <file>] [-of <file>] [-on <file>]
             [-c <value>] [-bs <value>] [-s <value>] [-t <value>] [-l <value>] [-rc <value>]
             [-viol <value>] [-f] [-con <n>] [-damp <value>] [-dyn <value>] 
             [-is <value>] [-ref] [-v] [-V] [-h]
 
disco generates (macro)molecular structures based
on a set of geometric constraints (usually
generated by the program "dist", but also additional
restraints may be specified).
 
Command line options:
Required (but defaults present, except for -op, -ot, -on and -ox):
-d <file>    - file containing input distances. This file
               is usually the output of the program dist. Default: dist.dat
-p <file>    - file containing input structure in
               PDB file format. This file is usually the
               output of the program dist. Default: dist.pdb
-n value     - nr. of structures to be generated. Default: 500
-i value     - maximum nr. of iterations per structure. Default: 2500
-op <file>   - basic filename for output PDB structures.
               (a number plus extension .pdb will be added)
-on <file>   - output trajectory in NMR-PDB format
-ox <file>   - output trajectory in xtc format
-ot <file>   - output trajectory in formatted GROMOS87 format
               (-on, -op, -ox or -ot must be specified)
-or <file>   - file with RMSD values of output structures to initial 
               structure. Default : disco.rms
-of <file>   - file with mean square atomic fluctuations
               expressed as B-factors (default disco_Bfac.pdb)
 
Optional:
-r <file>    - file containing additional distance 
               restraints. Default: none
-c <value>   - chirality check. 0 : no chirality checks; 1: only check 
               afterwards 2: check on the fly (default: 2)
-bs <value>  - number of rounds of triangular bound smoothing (default 0)
               (if >= 6, tetragonal BS is activated)
-s <value>   - initial random seed (default 741265)
-viol <value>- maximal acceptable sum of violations  (default 1.0 nm)
-f           - do not fit generated structures to reference
               (default : do fit)
-con <n>     - consider convergence failed after n non-productive iterations.
               (default: 500 steps)
-t <value>   - maximum nr. of trials per run. Default: 25
-damp <value>- damping factor for distance corrections. Default: 1
               (try 2 in case of convergence problems)
-dyn <value> - nr of rounds to dynamically set tolerances.  Default: 0
-bump        - do extra bump check. (default: no check)
-l <value>   - pairlist update frequency (only valid together with -bump;
               default: 10 steps)
-rc <value>  - cut-off radius for pairlist (only valid together with -bump;
               default: 0.5 nm)
-ref         - use input coordinates instead of random starting coordinates
-is <value>  - pre-scale coordinates with this factor.  Default: 1
-v           - be verbose
-h           - displays this message
-V           - write version information and exit
```
