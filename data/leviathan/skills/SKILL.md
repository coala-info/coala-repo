# leviathan

## Overview
LEVIATHAN is a structural variant calling method specifically designed for Linked-Reads data. It utilizes barcode indexing to identify long-range genomic interactions that suggest structural rearrangements. The tool operates in two primary phases: first, it indexes barcode occurrences using the LRez utility; second, it identifies SV candidates by analyzing shared barcodes between distant regions and refines these candidates using short-read alignment evidence to output high-confidence variants in VCF format.

## Barcode Indexing with LRez
Before running the variant caller, you must generate a barcode index (.bci) from your BAM file.

**Requirements:**
- The input BAM must have barcodes extracted and appended to the read headers (e.g., via Long Ranger basic).
- A BAM index (.bai) must exist in the same directory.

**Command:**
```bash
./LRez/bin/LRez index bam -p -b input.bam -o barcodeIndex.bci
```
- `-p`: Enables progress bar.
- `-b`: Path to the input BAM file.
- `-o`: Path for the output barcode index file.

## Structural Variant Calling
Once the index is built, execute the main LEVIATHAN binary to detect variants.

**Basic Usage:**
```bash
./bin/LEVIATHAN -b input.bam -i barcodeIndex.bci -g genome.fasta -o output.vcf
```

**Key Parameters:**
- `-b`: Input BAM file.
- `-i`: The .bci index generated by LRez.
- `-g`: Reference genome in FASTA format.
- `-o`: Output VCF file path.
- `-t`: Number of threads (default: 8).

## Optimization and Tuning
Adjust these parameters based on the specific library prep and expected variant sizes:

- **Region Sensitivity**: Use `-r` (region size) and `-v` (min variant size) to control the resolution. Default is 1000bp.
- **Variant Size Thresholds**: 
  - `-M`: Minimum size for "medium" variants (default: 2000).
  - `-L`: Minimum size for "large" variants (default: 10000).
- **Filtering Candidates**:
  - `-c`: Minimum number of shared barcodes required to consider a candidate (default: 1).
  - `-n`: Max links threshold to filter out repetitive or noisy regions (default: 1000).
- **Translocations**: Use `-s` to skip translocation detection if only intrachromosomal variants are of interest.

## Best Practices
- **Memory Management**: LEVIATHAN is highly efficient; even large datasets typically use very low RAM (often < 100 KB for small examples, scaling linearly with binning).
- **Thread Scaling**: Use the `-t` flag to match your CPU core count for faster processing during the shared barcode computation phase.
- **Candidate Inspection**: If you need to debug the calling process, use `-C <filename>.bedpe` to save the raw SV candidates before the refinement step.
