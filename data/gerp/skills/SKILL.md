# gerp

## Overview
GERP is a tool designed to identify genomic regions that are under evolutionary constraint. By comparing observed substitution rates to expected neutral rates across a multiple sequence alignment, it quantifies "rejected substitutions" (RS scores). High RS scores indicate intense purifying selection, suggesting the region has a functional biological role. This skill provides the necessary command-line patterns to generate these scores and define constrained elements.

## Command Line Usage

### 1. Generate RS Scores (gerpcol)
The primary command for calculating site-by-site conservation scores.
- **Input**: A multiple sequence alignment (typically in .mfa or .maf format) and a phylogenetic tree with branch lengths representing neutral evolution.
- **Basic Pattern**: `gerpcol -t <tree_file> -f <alignment_file> -e <reference_id>`
- **Key Arguments**:
    - `-t`: Neutral evolutionary tree (Newick format).
    - `-f`: Multiple sequence alignment file.
    - `-e`: Specify the reference sequence ID to project scores onto.
    - `-j`: Use Jukes-Cantor distance correction.

### 2. Identify Constrained Elements (gerpelem)
Used to group individual site scores into statistically significant constrained blocks.
- **Input**: The `.rates` file generated by `gerpcol`.
- **Basic Pattern**: `gerpelem -f <rates_file> -s <threshold>`
- **Key Arguments**:
    - `-f`: The output file from the scoring step.
    - `-s`: RS score threshold for defining an element (e.g., 2.0).
    - `-p`: P-value cutoff for element significance.

## Best Practices
- **Neutral Tree Calibration**: Ensure the neutral tree branch lengths are accurately estimated (e.g., using 4-fold degenerate sites) to avoid overestimating conservation.
- **Alignment Quality**: GERP is sensitive to alignment errors. Pre-filter alignments to remove poorly aligned or gap-heavy regions.
- **Score Interpretation**: Positive RS scores indicate constraint; negative scores suggest faster-than-neutral evolution (though these are often treated as noise).
- **Resolution**: While `gerpcol` provides base-pair resolution, `gerpelem` is better for identifying functional domains or regulatory motifs.
