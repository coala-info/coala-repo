# pisad

## Overview
PISAD (Phsaed Intraspecies Sample Anomalies Detection) is a specialized bioinformatics tool designed for cohort quality control. Unlike traditional methods that rely on a reference genome, PISAD uses k-mer counting to identify intraspecies anomalies. The workflow involves selecting a high-quality "target" sample to generate a variant sketch, which is then used as a baseline to compare against other samples in the cohort. This is particularly useful for non-model organisms or when reference-based alignment is computationally expensive or unavailable.

## Workflow and CLI Usage

### Stage 1: SNP Calling and Sketch Construction
The first stage identifies variants in a high-quality representative sample and packages them into a searchable sketch.

**1. Reference-free SNP Calling**
Use `run.sh` to perform initial SNP calling on a target FASTQ file.
```bash
run.sh -i <input.fastq.gz> -m <0|1> -t <threads> -o <prefix>
```
*   `-m`: Set to `0` if heterozygosity is < 1.2%, otherwise use `1`.
*   `-k`: K-mer size (default: 21). Ensure this matches across all subsequent steps.

**2. Construct Variant Sketch**
Convert the generated `.snp` file into a variant sketch format.
```bash
create -i <input.snp> -k 21 -l 21 -o <output_prefix>
```
*   `-l`: Filtering threshold (default: 21).

### Stage 2: Cohort Comparison and Evaluation
The second stage compares the rest of the cohort against the sketch created in Stage 1.

**1. K-mer Counting**
Compare other cohort samples against the variant sketch.
```bash
pisadCount -s <sketch.fa> -t <threads> <sample.fastq.gz>
```
*   **Multiple Sketches**: You can input multiple sketches separated by commas: `-s sketch1.fa,sketch2.fa`.
*   **Early Termination**: For high-coverage samples, use `-m <threshold>` (e.g., `-m 5`) to stop reading once sufficient coverage is reached, saving time.

**2. Sample Evaluation**
Calculate relationships and detect swaps based on the statistics generated by `pisadCount`.
```bash
pisadEval <stats_file.txt> > summary.tsv
```

## Expert Tips and Best Practices
*   **Target Selection**: Always choose the sequencing dataset with the lowest error rate and highest quality as the target for Stage 1. The accuracy of the entire cohort detection depends on this initial variant sketch.
*   **K-mer Consistency**: Ensure the `-k` parameter is identical across `run.sh`, `create`, and `pisadCount`. The default is 21, which is generally optimal for most intraspecies tasks.
*   **Performance Tuning**: `pisadCount` defaults to 6 threads per sequencing file. If processing a large cohort on a high-core machine, manually adjust `-t` to balance the load.
*   **Memory Management**: PISAD utilizes `gperftools` and `hdf5`. Ensure your environment has sufficient memory for k-mer counting, especially when working with large genomes or high k-mer sizes.
