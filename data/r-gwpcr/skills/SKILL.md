# r-gwpcr

## Overview
The `gwpcr` package (associated with the TRUmiCount project) provides an R implementation of a mechanistic model for PCR amplification. It is designed to improve the accuracy of quantitative NGS experiments (like RNA-Seq) by correcting for two major sources of error:
1. **Molecules lost** during the sequencing process (under-estimation).
2. **Phantom UMIs** created by PCR chimeras and amplification artifacts (over-estimation).

The package computes "loss-corrected" and "phantom-filtered" molecule counts, which are more accurate than simply counting distinct UMI sequences.

## Installation
To install the package from CRAN:
```R
install.packages("gwpcr")
```

## Core Workflow
The package typically operates on UMI frequency data (the number of times each unique UMI was observed for a specific gene/feature).

### 1. Preparing Data
Data should be organized as a vector of observation counts for each UMI.

### 3. Handling PCR Chimeras
The model distinguishes between "real" UMIs and those likely generated by artifacts.
- **Thresholding**: Low-count UMIs are often evaluated against the model to determine if they are sequencing errors or chimeras.
- **Efficiency**: The model accounts for the amplification rate to predict how many UMIs should have been observed given the starting material.

## Tips for Accuracy
- **UMI Length**: Ensure your UMI length provides enough complexity for the expected number of molecules to avoid saturation.
- **Sequencing Depth**: Higher depth allows the TRUmiCount model to better estimate the underlying PCR efficiency parameters.
- **Genewise Analysis**: For best results, apply the correction genewise, especially when amplification bias varies across different sequences.
