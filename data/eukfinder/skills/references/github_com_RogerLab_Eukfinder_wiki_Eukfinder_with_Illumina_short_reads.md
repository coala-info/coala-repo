# Running Eukfinder with short read Illumina data requires two steps

1. **Eukfinder read\_prep** to remove adapters, low quality data, and host sequences
2. **Eukfinder short\_seqs** to classify short reads as bacterial, eukaryote, archaea, misc or unk, assemble those sequences indentified as Eukaryote or unknown using metaSdes, and then reclassify contigs

## 1. Preparing samples using Eukfinder read\_prep

### Usage

```
        Eukfinder read_prep [-h] --r1 R1 --r2 R2 -n THREADS -i ILLUMINA_CLIP \
                                 --hcrop HCROP -l LEADING_TRIM -t TRAIL_TRIM --wsize WSIZE \
                                 --qscore QSCORE --mlen MLEN --mhlen MHLEN \
                                 --hg HG -o OUT_NAME  [--cdb Centrifuge_DB_common_filename_prefix]
```

### Required arguments:

**Description**

--r1, --reads-r1   R1

* left fastq file, must have .fastq or .fq ending it does not take gzipped files

--r2, --reads-r2   R2

* right fastq file, must have .fastq or .fq endingit does not take gzipped files

-n, --threads   THREADS

* number of threads

-i, --illumina-clip   ILLUMINA\_CLIP

* location of file containing adaptor sequences

--hcrop, --head-crop   HCROP

* specified number of bases to crop from the start of the read

-l, --leading-trim   LEADING\_TRIM

* quality value threshold required for bases at the start of a read to be retained

-t, --trail-trim   TRAIL\_TRIM

* quality value threshold required for bases at the end of a read to be retained

--wsize, --window-size   WSIZE

* number of bases to average across for accessing quality

--qscore, --quality-score   QSCORE

* quality score for trimming

--mlen, --min-length   MLEN

* minimum length of read after trimming to be kept by trimmomatic

--mhlen, --min-hit-length   MHLEN

* minimum hit length for Centrifuge searches it controls the minimum sequence length required for Centrifuge classification matches. Purpose is to filter out spurious short matches. A value of 50 works well for most datasets; lower values (e.g., 30-40) are more sensitive, catch shorter reads, but higher false-positive rate; higher values (e.g., 60-100) are more stringent, fewer false positives, but may miss real eukaryotic sequences

--hg, --host-genome   HG

* path to host genome sequence, used for removal of host sequences (NOTE you must specify a host genome)

-o, --out\_name   OUT\_NAME

* prefix to be used for output files
* **Important Note:**
  The `-o` or `--out_name` parameter should be a simple filename prefix only.
  It must not include any path (e.g., /home/user/output\_prefix will cause an error).
  Eukfinder saves all outputs to the current directory, and assumes it is writable.
  Please make sure to run it inside a folder where you have permission to write files.

**optional arguments:**

-h, --help show this help message and exit

--cdb, --centrifuge-database   CDB

* path to downloaded (or created) **Basename** of Centrifuge index files
* **Important Note:**
  Make sure to define the Centrifuge database using the common filename prefix
  (before .1.cf, .2.cf, etc.), not the full path to a single .cf file.

--qenc, --quality-encoding (default value phred64)

* allows you to change the fastq quality coding to PHRED+33 using value phred33

### Example command line to run Eukfinder read\_prep

```
     conda activate Eukfinder

     Eukfinder read_prep  --r1 "R1.fastq" --r2 "R2.fastq"  \
                          -n 10 --hcrop 10 -l 15 -t 15 --wsize 40 --qscore 25 --mlen 40  \
                          -i "adaptor_file" --hg "host_genome_file" \
                          -o "output_prefix"  --mhlen40 [--cdb Centrifuge_DB_common_filename_prefix]
```

### Expected output from Eukfinder read\_prep

log file with run information from the script (Read\_prep\_date.log)
files needed as input for Eukfinder short\_seqs

* paired and unpaired fastq files after removal of adapters, low quality and host reads
  + output\_prefix\_p.1.fastq (--r1 input Eukfinder short\_seqs)
  + output\_prefix\_p.2.fastq (--r2 input Eukfinder short\_seqs)
  + output\_prefix\_un.fastq (--un input Eukfinder short\_seqs)
* paired and unpaired centrifuge results
  + output\_prefix\_centrifuge\_P (--plcass input Eukfinder short\_seqs)
  + output\_prefix\_centrifuge\_UP (--ulcass input Eukfinder short\_seqs)

### Note:

We have created a special case of read\_prep called **read\_prep\_env** that can be used when samples do not contain host reads that need to be removed like in environmental samples. This special case does not require you to specify a host genome. Other than --hg all other parameters are the same and can be used

### Example command line to run Eukfinder read\_prep\_env

```
     conda activate Eukfinder

     Eukfinder read_prep_env  --r1 Raw_R1.fastq --r2 Raw_R2.fastq -n 10 --hcrop 10  \
                              -l 15 -t 15 --wsize 40 --qscore 25 --mlen 40  \
                              -i Adaptor.fa  -o OUTPUT_PREIFX  --mhlen 40 \
                              [--cdb Centrifuge_DB_common_filename_prefix]
```

## 2. Running Eukfinder short\_seqs

### Usage

```
     Eukfinder short_seqs [-h] --r1 R1 --r2 R2 --un UN -o OUT_NAME -n NUMBER_OF_THREADS \
                                 -z NUMBER_OF_CHUNKS -t TAXONOMY_UPDATE  \
                                 -e E_VALUE --pid PID --cov COV --max_m MAX_M  \
                                 --mhlen MHLEN --pclass PCLASS --uclass UCLASS  \
                                 [-p PLAST_DB -m PLAST_MAP --cdb Centrifuge_DB_common_filename_prefix]
```

### Required arguments

**Description**

  --r1, --reads-r1   R1

* left fastq file from Eukfinder read\_prep (output\_prefix\_p.1.fastq from read\_prep)

  --r2, --reads-r2   R2

* right fastq file from Eukfinder read\_prep (output\_prefix\_p.2.fastq from read\_prep)

  --un, --un-pair-reads   UN

* unpaired fastq file from Eukfinder read\_prep (output\_prefix\_un.fastq from read\_prep)

   -o, --out\_name   OUT\_NAME

* prefix for output results
* **Important Note:**
  The `-o` or `--out_name` parameter should be a simple filename prefix only.
  It must not include any path (e.g., /home/user/output\_prefix will cause an error).
  Eukfinder saves all outputs to the current directory, and assumes it is writable.
  Please make sure to run it inside a folder where you have permission to write files.

   -n, --number-of-threads   NUMBER\_OF\_THREADS

* number of threads, controls parallelization. Set based on available CPU cores. Higher values speed up processing but increase memory usage.

   -z, --number-of-chunks   NUMBER\_OF\_CHUNKS

* number of chunks the input files is split into for processing with PLAST. Higher values reduce memory per chunk put increase overhead. Recommendation is to use 2-4 for most datasets, for very large samples (>100M reads) try 4-8.

   -t, --taxonomy-update   TAXONOMY\_UPDATE

* set to True the first time you run otherwise set to False

   -e, --e-value   E\_VALUE

* e-value threshold for PLAST searches; lower values = more strigent

  --pid, --percent\_id   PID

* percent identity to be used for PLAST searches; higher values = more stringent

  --cov, --coverage   COV

* percent coverage to be used for PLAST searches; higher value = more stringent

  --mhlen, --min-hit-length   MHLEN

* minimum hit length for Centrifuge searches it controls the minimum sequence length required for Centrifuge classification matches. Purpose is to filter out spurious short matches. A value of 50 works well for most datasets; lower values (e.g., 30-40) are more sensitive, catch shorter reads, but higher false-positive rate; higher values (e.g., 60-100) are more stringent, fewer false positives, but may miss real eukaryotic sequences

  --pclass, --p-reads-class   PCLASS

* file name for classification results of paired reads from Eukfinder read\_prep (output\_prefix\_centrifuge\_P)

  --uclass, --u-reads-class   UCLASS

* file name for for classification results of un-paired reads from Euk
