# Genome-wide alternative splicing analysis: human

## Overview

This training workflow provides a comprehensive pipeline for analyzing genome-wide alternative splicing and isoform switching in human transcriptomics data. It begins with raw RNA-seq data processing using [fastp](https://github.com/OpenGene/fastp) and [FastQC](https://www.bioinformatics.babraham.ac.uk/projects/fastqc/) for quality control, followed by high-accuracy alignment to a reference genome using [RNA STAR](https://github.com/alexdobin/STAR) in a two-pass mapping approach.

Following alignment, the workflow performs extensive quality control via the [RSeQC](http://rseqc.sourceforge.net/) suite to evaluate read distribution, junction saturation, and gene body coverage. Transcripts are then assembled and quantified using [StringTie](https://ccb.jhu.edu/software/stringtie/), with [rnaQUAST](https://github.com/ablab/rnaquast) providing detailed assembly metrics. This stage ensures a robust reconstruction of the transcriptome, including the identification of novel isoforms.

The core analysis utilizes [IsoformSwitchAnalyzeR](https://bioconductor.org/packages/release/bioc/html/IsoformSwitchAnalyzeR.html) to identify differential isoform usage between experimental conditions. The pipeline integrates external functional annotation tools, including [CPAT](https://cpat.readthedocs.io/) for assessing coding potential and [PfamScan](https://www.ebi.ac.uk/Tools/pfa/pfamscan/) for protein domain identification, to predict the functional consequences of observed splicing events.

Licensed under CC-BY-SA-4.0, this workflow is tagged for `transcriptomics` and `alternative_splicing`. It ultimately produces refined datasets of protein-coding and long non-coding RNA (lncRNA) sequences, facilitating a deep understanding of the functional diversity generated by alternative splicing.

Ensure your RNA-seq data is organized into a paired-end dataset collection to facilitate automated processing through the STAR and StringTie steps. Key inputs include genome annotations in GTF format and reference sequences in FASTA, alongside specialized Pfam-A HMM and CPAT files for functional isoform characterization. You must provide a specific list of control IDs as a text dataset to correctly define experimental groups for IsoformSwitchAnalyzeR. For comprehensive guidance on parameter settings and data formatting, refer to the README.md. You can use `planemo workflow_job_init` to generate a `job.yml` for streamlined execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run main_workflow.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run main_workflow.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run main_workflow.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init main_workflow.ga -o job.yml`
- Lint: `planemo workflow_lint main_workflow.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
