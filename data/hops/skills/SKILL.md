# hops

## Overview

HOPS (Heuristic Operations for Pathogen Screening) is a Java-based pipeline that streamlines the identification and authentication of pathogen DNA in metagenomic data. It acts as a wrapper to run MALT and MaltExtract in unison, reducing the manual overhead of managing intermediate files and complex command-line arguments for each tool. It is the standard tool for researchers looking to detect specific target species within ancient samples while applying aDNA-specific authentication criteria (like damage patterns).

## Installation and Setup

The most reliable way to install HOPS and its dependencies (MALT, MaltExtract, and AMPS post-processing) is via Bioconda:

```bash
conda install hops -c bioconda
```

### External Dependencies
HOPS requires an indexed MALT database, which can be several hundred gigabytes. You must build this separately using `malt-build` or obtain a pre-built index. Ensure the index is compatible with MALT version 0.38 or higher.

## Command Line Usage

### Core Modes (`-m` / `--mode`)
- `full`: Runs the entire pipeline (MALT -> MaltExtract -> Post-processing).
- `malt`: Executes only the MALT alignment step.
- `maltex`: Executes only the MaltExtract analysis on existing RMA6 files.
- `me_po`: Runs MaltExtract and post-processing on MALT output.
- `post`: Runs only the post-processing scripts.

### Basic Execution
To run a full screening on gzipped FASTQ files:

```bash
hops -Xmx600G -input /path/to/reads/*.fastq.gz -output /path/to/output -m full
```

### Using a Configuration File
For complex analyses or to save environment-specific paths, use a config file. Parameters in the config file are overwritten by direct CLI arguments.

```bash
hops -c my_config.txt -i /path/to/input -o /path/to/output -m full
```

## Expert Tips and Best Practices

### Memory Management
HOPS launches MALT and MaltExtract as subprocesses. You must provide enough heap space via the `-Xmx` flag to accommodate the MALT database loading. If the job hangs or crashes immediately, increase the heap space (e.g., `-Xmx500G` to `-Xmx800G` depending on database size).

### Configuration File Keys
If HOPS cannot find its dependencies, define them explicitly in your `configFile.txt`:
- `pathToMalt`: Path to the `malt-run` shell script.
- `index`: Path to the indexed MALT database.
- `pathToMaltExtract`: Path to the `MaltExtract.jar` (v1.2 or higher).
- `pathToPostProcessing`: Path to the `postprocessing.AMPS.r` script.
- `pathToList`: Path to the text file containing the list of target taxa nodes.

### Input Handling
- For `full` or `malt` modes: Input can be a directory or a list of `.fa`, `.fasta`, `.fastq`, or `.fq` files (optionally gzipped).
- For `maltex` mode: Input must be `.rma6` files generated by MALT.
- For `post` mode: Input must be the directory containing MaltExtract output.

### Troubleshooting
Always check the generated log file. HOPS logs the exact command sent to the system (or Slurm), which is the first place to look for pathing errors or Java environment conflicts.
