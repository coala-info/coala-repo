# rdp-classifier

## Overview

The RDP Classifier is a specialized tool for the rapid taxonomic placement of ribosomal RNA sequences. It utilizes a Naive Bayesian algorithm to provide assignments across all major ranks (Domain to Genus) and calculates bootstrap confidence values for each assignment. This skill enables the execution of classification workflows, cross-validation, and library comparisons, while also supporting advanced features like gene copy number normalization and integration with BIOM-formatted data for downstream ecological analysis.

## Core CLI Usage

The tool is executed as a Java JAR file. The primary subcommand is `classify`.

### Basic Classification
To classify a FASTA file with a specific confidence cutoff (e.g., 0.5) and generate both a detail file and a hierarchical count file:

```bash
java -Xmx1g -jar /path/to/classifier.jar classify \
  -c 0.5 \
  -o output_details.txt \
  -h output_hierarchical.txt \
  input_sequences.fasta
```

### Gene Selection
By default, the classifier assumes 16S rRNA. For other markers, use the `-g` flag:
- `16srrna` (Default)
- `fungallsu`
- `fungalits_warcup`
- `fungalits_unite`

```bash
java -jar classifier.jar classify -g fungallsu -o fungal_output.txt input.fasta
```

### Output Formats (`-f`)
- `allrank`: (Default) Full results for every rank.
- `fixrank`: Results for fixed ranks (Domain to Genus) in a consistent order.
- `filterbyconf`: Only outputs ranks meeting the confidence cutoff; lower ranks are binned into "unclassified" nodes.
- `biom`: Produces a rich dense BIOM file (requires `-m` for the input BIOM file).

## Advanced Workflows

### Speed Optimization via Dereplication
For large datasets with many identical sequences, dereplicate first using RDP Clustering tools to create a unique sequence file and an ID mapping file. The classifier can process these together to expand counts automatically:

```bash
java -Xmx1g -jar classifier.jar classify \
  -c 0.8 \
  -o results.txt \
  -h counts.txt \
  unique_seqs.fasta,id_mapping.ids
```

### BIOM Format Integration
To update an existing BIOM file with RDP taxonomic assignments:

```bash
java -jar classifier.jar classify \
  -f biom \
  -m input_cluster.biom \
  -o output_rich.biom \
  representative_seqs.fasta
```

### Gene Copy Number Adjustment
The classifier provides adjusted assignment counts in the hierarchical output files (`-h`). This is particularly useful for 16S data to better estimate organism abundance. The precompiled version uses rrnDB data.

## Subcommand Reference

- `classify`: Main classification routine.
- `libcompare`: Statistical comparison of two samples to find significantly different taxa.
- `merge-detail`: Combines multiple detail files into a single taxon count file.
- `train`: Retrains the classifier on a custom taxonomy/sequence set (requires a property file via `-t`).
- `taxa-sim`: Calculates and plots similarities within taxa.
- `rm-dupseq` / `rm-partialseq`: Pre-processing utilities to clean input data.

## Expert Tips

- **Memory Allocation**: Always use the `-Xmx` flag (e.g., `-Xmx2g`) to ensure Java has enough heap space for large training sets or high-throughput sequence files.
- **Confidence Cutoffs**: While the default is 0.8, a cutoff of 0.5 is often used for shorter reads (e.g., partial Illumina or 454 reads) to avoid excessive "unclassified" assignments at the genus level.
- **Hierarchical Files**: The `.txt` file generated by `-h` is tab-delimited and can be imported directly into R or Excel for visualization and community composition analysis.
