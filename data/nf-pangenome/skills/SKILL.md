## Overview
The pipeline addresses the challenge of representing multiple genomic sequences in a single graph structure that maintains local linearity. By rendering a collection of sequences into a pangenome graph, it enables interpretation, visualization, and reuse of genomic data while preserving both small and large-scale variations.

The workflow processes input sequences through alignment, induction, and smoothing to produce a graph that is optimized for downstream comparative genomics. Primary deliverables include the pangenome graph in GFA format, graph statistics, qualitative visualizations, and a MultiQC report summarizing the diagnostic information.

## Data preparation
The pipeline requires a single BGZIPPED FASTA file containing all sequences (chromosomes, contigs, or long reads) to be included in the graph.

*   **Input FASTA**: Must be compressed with `bgzip` and indexed with `samtools faidx`.
*   **Naming Convention**: Sequence identifiers should follow the [PanSN-spec](https://github.com/pangenome/PanSN-spec) (e.g., `Sample#Haplotype#Contig`) for maximum compatibility.
*   **Haplotype Count**: You must specify the total number of haplotypes present in the input file via `--n_haplotypes`.

Minimal preparation example:
```bash
# Compress and index the input
bgzip <SEQUENCES.fa> @<THREADS> > <SEQUENCES.fa.gz>
samtools faidx <SEQUENCES.fa.gz>
```

## How to run
Run the pipeline by providing the path to the compressed FASTA and the number of haplotypes. Use `-profile` to specify the container engine (e.g., `docker` or `singularity`).

```bash
nextflow run nf-core/pangenome \
    -r 1.1.0 \
    --input sequences.fa.gz \
    --n_haplotypes 2 \
    --outdir ./results \
    -profile docker
```

To distribute the computationally heavy alignment step across a cluster, use the `--wfmash_chunks` parameter to split the work into equal-sized problems. For example, on a 10-node cluster, set `--wfmash_chunks 10`. Use `-resume` to restart a run from the last successful step if interrupted.

## Outputs
Results are saved in the directory specified by `--outdir`.

*   **MultiQC Report**: Found in `multiqc/`, this is the primary diagnostic report to open first.
*   **Pangenome Graph**: The final normalized graph is provided in GFA format.
*   **Visualizations**: Qualitative graph visualizations generated by `ODGI`.
*   **Statistics**: Summary statistics regarding graph topology and sequence content.

For a detailed description of all output files, refer to the [official output documentation](https://nf-co.re/pangenome/output).
