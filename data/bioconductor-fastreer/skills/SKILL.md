# bioconductor-fastreer

## Overview
The `fastreeR` package provides an efficient interface for genomic distance calculation and tree construction. It utilizes a Java backend (via `rJava`) to minimize memory usage, keeping only the distance matrix in RAM. It is particularly useful for large VCF or FASTA datasets where traditional R-based distance calculations might exceed memory limits.

## Preparation
Before loading the library, you must allocate memory to the Java Virtual Machine (JVM). This must be done **before** the package is initialized.

```R
# Allocate 1GB of RAM to the JVM
options(java.parameters = "-Xmx1G")
library(fastreeR)
library(ape) # For tree handling and plotting
```

## Workflows for VCF Files

### 1. Calculate Distances and Statistics
Generate sample-level statistics or a distance matrix directly from a VCF.
```R
# Get individual statistics (e.g., heterozygosity)
stats <- vcf2istats(inputFile = "samples.vcf.gz")

# Calculate distance matrix
dist_matrix <- vcf2dist(inputFile = "samples.vcf.gz", threads = 1)
```

### 2. Phylogenetic Tree Construction
You can generate a tree from a distance matrix or directly from the VCF. Trees generated by `fastreeR` are ultrametric.
```R
# Direct VCF to Newick string
newick_tree <- vcf2tree(inputFile = "samples.vcf.gz", threads = 1)

# Convert Newick string to ape phylo object for plotting
tree_obj <- ape::read.tree(text = newick_tree)
plot(tree_obj, direction = "down")
```

### 3. Bootstrapping
To assess branch support, use the `bootstrap` parameter.
```R
# Run 100 bootstrap replicates
boot_tree_str <- vcf2tree(inputFile = "samples.vcf.gz", bootstrap = 100)
boot_tree <- ape::read.tree(text = boot_tree_str)

# Support values are stored in node.label
plot(boot_tree)
ape::nodelabels(boot_tree$node.label, cex = 0.7, frame = "none")
```

## Workflows for FASTA Files
For FASTA files, distances are calculated based on k-mer compositions.
```R
# Calculate distances using 6-mers
fasta_dist <- fasta2dist(inputFile = "sequences.fasta.gz", kmer = 6)

# Generate tree from distances
fasta_tree <- dist2tree(inputDist = fasta_dist)
```

## Hierarchical Clustering
Identify sample clusters by cutting the tree at a specific height.
```R
# Cluster samples based on a distance threshold
clusters <- vcf2clusters(inputFile = "samples.vcf.gz", cutHeight = 0.067)

# Access cluster assignments
# clusters[[1]] contains the tree, clusters[[2]] contains the group assignments
print(clusters[[2]])
```

## Troubleshooting & Tips
- **Memory:** If you encounter "Out of Memory" errors, increase the `-Xmx` value in `options(java.parameters)`.
- **Threads:** Use the `threads` parameter in `vcf2dist` or `vcf2tree` to speed up calculations on multi-core systems.
- **Tree Format:** The package returns trees as Newick-formatted strings. Always use `ape::read.tree(text = ...)` to manipulate them in R.
- **Java Version:** Ensure Java 11 or higher is installed and configured for use with R.
