# r-rnexml

name: r-rnexml
description: Expert guidance for the R package 'RNeXML' to read, write, and manipulate NeXML (Phylogenetic XML) files. Use this skill when you need to convert between 'ape' phylo objects and NeXML, extract character data/traits from NeXML files, manage semantically rich metadata (RDFa), or validate NeXML documents against the official schema.

## Overview
RNeXML provides a semantically rich interface for the NeXML format, a standard for representing phylogenetic trees, character matrices, and associated metadata. It bridges the gap between the 'ape' package (S3 objects) and the more complex, hierarchical NeXML (S4 objects), allowing for the preservation of taxonomic identifiers and custom annotations.

## Installation
To install the package from CRAN:
```r
install.packages("RNeXML")
```

## Core Workflows

### Reading and Writing NeXML
The primary functions for I/O handle the conversion between standard R phylogenetic objects and NeXML files.

```r
library(RNeXML)

# Read a NeXML file
nex <- nexml_read("path/to/file.xml")

# Write an ape 'phylo' object to NeXML
data(bird.orders)
nexml_write(bird.orders, file = "birds.xml", creator = "Your Name")

# Validate a NeXML file against the schema
nexml_validate("birds.xml")
```

### Extracting Data
RNeXML provides "get" functions to extract specific components into data frames or 'ape' objects.

*   **Trees**: `get_trees(nex)` returns a list of `phylo` objects.
*   **Characters**: `get_characters(nex)` returns a data frame of traits/characters.
*   **Taxa**: `get_taxa(nex)` returns the OTU (Operational Taxonomic Unit) labels and IDs.
*   **Metadata**: `get_metadata(nex)` returns document-level metadata.

### Adding Metadata and Annotations
NeXML supports RDFa metadata. You can add basic Dublin Core metadata or custom URI-based annotations.

```r
# Add basic metadata
nex <- add_basic_meta(title = "My Study", creator = "Author Name", nexml = nex)

# Add custom metadata using the meta() constructor
modified <- meta(property = "prism:modificationDate", content = "2023-10-01")
nex <- add_meta(modified, nex)

# Add taxonomic identifiers (NCBI, ITIS, etc.) using taxize_nexml
# Requires internet connection for lookups
nex <- taxize_nexml(nex, db = "NCBI")
```

### Working with the S4 Object
The `nexml` object is an S4 object. While "get" functions are preferred, direct access uses the `@` operator.

```r
# Inspect slots
slotNames(nex)

# Access namespaces
nex@namespaces

# Access a specific OTU label (deeply nested)
nex@otus[[1]]@otu[[1]]@label
```

## Tips and Best Practices
*   **Piping**: When using `magrittr` or base R pipes, note that `add_` functions often take the `nexml` object as the *second* argument. Use `add_trees(tr) %>% add_meta(m, nexml = .)` or `add_characters(dat, nex)`.
*   **Character Matrices**: `get_characters()` merges all character blocks. If you have distinct continuous and discrete blocks, use `get_characters_list()` to keep them separate.
*   **Validation**: Always run `nexml_validate()` on files generated by other software before parsing to ensure they follow the standard.
*   **Namespaces**: If adding custom metadata properties, ensure you provide the namespace URL in `add_meta(..., namespaces = c(prefix = "URL"))`.
