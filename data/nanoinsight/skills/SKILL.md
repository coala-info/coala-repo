# nanoinsight

## Overview
NanoINSight is a specialized bioinformatics tool designed to characterize the content of genomic insertions. While NanoVar identifies structural variants, NanoINSight specifically focuses on the "what" of an insertion by generating consensus sequences and running repeat annotation. It is essential for identifying Transposable Element (TE) insertions, such as LINEs, SINEs, and Alus, which are often the biological drivers behind large insertion events in long-read sequencing data.

## Execution Patterns

### Standalone Annotation
Use this pattern when you already have a VCF file containing insertions (typically from NanoVar) and want to perform repeat classification.

```bash
nanoinsight -t [threads] -s [species] [input.vcf] [output_dir]
```

**Key Arguments:**
- `-s`: Species name for RepeatMasker (e.g., `human`, `mouse`). This is mandatory.
- `-t`: Number of CPU threads (default is 1).
- `[VCF]`: The input VCF file containing the insertion calls.
- `[working_directory]`: Path where results and intermediate files will be stored.

### Integrated NanoVar Workflow
NanoINSight can be triggered directly during a NanoVar run to provide a seamless "discovery-to-annotation" pipeline.

```bash
nanovar [options] --annotate_ins [species] [sample.fq/bam] [ref.fa] [working_dir]
```

### Manual Path Configuration
If dependencies are not in the system PATH, specify them explicitly to avoid execution errors:
- `-m [path]`: Path to the `mafft` executable.
- `-r [path]`: Path to the `RepeatMasker` executable.

## Input Requirements
To ensure successful annotation, ensure the following NanoVar-specific files are available if running on existing output:
1. **VCF Input**: Must contain insertion (INS) calls.
2. **Sequence Data**: NanoINSight often requires the `ins_seq.fa` (insertion sequences) and `sv_support_reads.tsv` (supporting read information) generated by NanoVar. If these are not in the default working directory, provide them via `-i` and `-u` respectively.

## Output Interpretation
- **Consensus Sequences**: Found in `.ins.con.fasta`. These represent the high-confidence sequence of the inserted element after multiple sequence alignment via MAFFT.
- **Repeat Annotations**: Located in the `rm_output` directory. This contains the standard RepeatMasker output files (.out, .gff, .tbl) detailing the types of repeats identified within the insertions.

## Best Practices
- **Species Selection**: Always verify that the `-s` string matches a valid RepeatMasker species library.
- **Resource Allocation**: RepeatMasker and MAFFT are computationally intensive; always scale `-t` to the available cores on the high-performance computing (HPC) node.
- **VCF Filtering**: For large datasets, run NanoINSight on "PASS" filtered VCFs (e.g., `sample.nanovar.pass.vcf`) to reduce computation time on low-confidence calls.
