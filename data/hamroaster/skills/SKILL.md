# hamroaster

## Overview
hAMRoaster (Harmonized AMR Output compAriSon ToolER) is a specialized benchmarking pipeline for antimicrobial resistance gene detection. It bridges the gap between bioinformatics predictions and phenotypic reality by comparing standardized tool outputs—primarily those processed via hAMRonization—against a "gold standard" key of known AMR phenotypes. This tool is essential for validating the sensitivity and specificity of different AMR callers (such as fARGene or ShortBRED) and understanding how different algorithms perform across various drug classes.

## Installation and Setup
Install hAMRoaster via Bioconda to ensure all dependencies are managed:
```bash
conda create -n hamroaster -c conda-forge -c bioconda hamroaster
conda activate hamroaster
```
Note: The command is case-sensitive: `hAMRoaster`.

## Core CLI Usage
At a minimum, hAMRoaster requires three flags to execute a comparison:

```bash
hAMRoaster --ham_out path/to/hamronized_results.tsv \
           --name my_experiment_v1 \
           --AMR_key path/to/phenotype_key.csv
```

### Required Arguments
- `--ham_out`: The full path to the output file generated by hAMRonization.
- `--name`: A unique identifier for the run. This creates a directory of the same name and prefixes all output files.
- `--AMR_key`: A TSV/CSV file of known AMR phenotypes. It must follow the NCBI BioSample AntiBioGram format: `taxa_name`, `antibiotic tested`, `result`, and `testing standard`.

## Advanced Configuration

### Granular Analysis
By default, hAMRoaster treats the input as one community. To analyze results on a per-sample basis, use the sample grouping flag:
- `--groupby_sample True`: Ensures metrics are calculated for each individual sample. Ensure sample names in the `--AMR_key` match the `--input_file_names` used during hAMRonization.

### Tool-Specific Integrations
While most tools are handled via the hAMRonization input, some require specific flags:
- `--fargene`: Point this to a directory containing subdirectories named after the AMR models analyzed (e.g., `class_a/`).
- `--shortbred`: Point to the directory containing ShortBRED output files.
- `--shortbred_map`: Use this if you need to override the default 2016 ShortBRED metadata mapping file.

### Custom Antibiotic Mapping
If the default drug class classifications (found in `db_files/cleaned_drug_class_key.csv`) are insufficient, provide a custom scheme:
- `--abx_map`: Expects a file where the first column is the drug class and the subsequent columns are the specific antibiotics.

## Interpreting Outputs
All results are stored in the directory named after your `--name` argument:
- `thanksgiving_ham_{name}.csv`: The primary summary containing endpoint performance metrics for every tool included in the run.
- `cooked_ham_w_true_pos_{name}.csv`: A detailed, labeled version of the input data showing specific True Positive, False Positive, and Unknown assignments.
- `grouped_by_tool_drug_class{name}.csv`: A breakdown of AMR gene counts detected per drug class per tool.

## Expert Tips
- **Preprocessing Requirement**: Most AMR detection tools must be run through hAMRonization before they can be ingested by hAMRoaster.
- **fARGene Directory Structure**: hAMRoaster expects a specific hierarchy for fARGene. If you analyzed `class_a` and `class_b`, your input directory should contain `class_a/` and `class_b/` subfolders containing the respective tool outputs.
- **Handling Empty Files**: hAMRoaster is designed to handle empty output files (e.g., when a tool runs but detects zero genes), which is critical for accurate specificity and sensitivity calculations.
