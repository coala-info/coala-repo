# flight-genome

## Overview

flight-genome is a Python-based bioinformatics tool designed for the automated recovery of genomes from metagenomic datasets. It functions as the computational engine for binning platforms like Rosella and Lorikeet. By combining sequence composition and coverage information, it applies UMAP (Uniform Manifold Approximation and Projection) and HDBSCAN (Hierarchical Density-Based Spatial Clustering of Applications with Noise) to identify distinct genomic clusters within complex microbial communities. Use this skill when you have a metagenomic assembly and associated coverage data and need to generate high-quality bins.

## Installation

The most reliable way to install flight-genome is via Bioconda:

```bash
conda install -c bioconda flight-genome
```

Alternatively, for development or specific version requirements, you can install from source:

```bash
git clone https://github.com/rhysnewell/flight.git
cd flight
conda env create -n flight -f flight.yml
conda activate flight
pip install .
```

## Core Usage: MAG Recovery

The primary command for binning contigs is `flight bin`. This process requires an assembly file (FASTA) and a coverage profile (typically generated by tools like CoverM).

### Basic Command Pattern

```bash
flight bin --assembly scaffolds.fasta --input coverage.cov --output output_dir/ --threads 24
```

### Key Arguments

- `--assembly`: Path to your scaffolds or contigs in FASTA format.
- `--input`: The coverage information file. flight-genome is optimized to work with coverage files generated by CoverM.
- `--output`: The directory where the resulting bins and intermediate files will be stored.
- `--threads`: Number of CPU cores to utilize. UMAP and HDBSCAN calculations are computationally intensive and benefit significantly from parallelization.

## Best Practices and Expert Tips

- **Coverage Input**: Ensure your coverage file is formatted correctly. flight-genome expects the output format produced by `coverm contig`.
- **Contig Filtering**: For better binning results, consider removing very short contigs (e.g., <1000bp or <2500bp) from your assembly before running flight-genome, as short sequences often lack sufficient signal for accurate clustering.
- **Resource Allocation**: UMAP can be memory-intensive for very large metagenomes. Ensure your environment has sufficient RAM relative to the number of contigs being processed.
- **Downstream Analysis**: The output of flight-genome consists of clustered contigs. You should validate these bins using tools like CheckM or CheckM2 to assess completeness and contamination.
