# epa-ng

## Overview
EPA-ng (Evolutionary Placement Algorithm - next generation) is a massively parallel tool designed to place query sequences into an existing phylogenetic framework. Instead of inferring a tree from scratch, it evaluates the most likely attachment points for "floating" sequences on a fixed reference tree. This is particularly useful for identifying the origin of environmental DNA samples or adding new sequences to a known phylogeny without altering the established relationships.

## Core Workflow

### 1. Preparation
Before running placement, you must have:
- A reference Multiple Sequence Alignment (MSA) in FASTA format.
- A reference tree in Newick format.
- Query sequences aligned to the reference MSA (using tools like `papara` or `hmmalign`).

### 2. Splitting Alignments
If your query sequences are bundled with the reference alignment (common output from aligners), use the `--split` command to separate them:
```bash
epa-ng --split reference_msa.fasta query_alignment.fasta
```

### 3. Executing Placement
The basic command requires the reference MSA (`-s`), the tree (`-t`), the query file (`-q`), and the substitution model (`--model`).

```bash
epa-ng -s ref.fasta -t tree.nwk -q query.fasta --model GTR+G+F -T 8
```

## Model Specification
EPA-ng requires explicit model parameters. There are two primary ways to provide them:

- **RAxML-ng Style String**: Pass the specific parameters directly.
  `--model GTR{0.7/1.8/1.2/0.6/3.0/1.0}+FU{0.25/0.23/0.30/0.22}+G4{0.47}`
- **Best Model File (Recommended)**: Use the `.bestModel` file generated by `raxml-ng --evaluate`.
  `--model info.raxml.bestModel`
- **Legacy Support**: It can also parse `RAxML_info` files from RAxML 8.x or IQ-TREE report files.

## Performance and Optimization

### Binary Format (bfast)
For very large query sets, convert FASTA files to the binary `bfast` format to enable random access and reduce memory overhead:
```bash
epa-ng -c query.fasta -s ref.fasta
# Then use the resulting .bfast file
epa-ng -q query.bfast ...
```

### Threading
- Use `-T` or `--threads` to specify the core count.
- **Tip**: Do not exceed the number of physical cores. Hyperthreading typically does not provide a speedup for the likelihood calculations used in EPA-ng.

### Heuristics
EPA-ng uses a two-stage approach (preplacement then detailed evaluation).
- **Dynamic Heuristic (Default)**: Controlled by `-g` (or `--dyn-heur`). It adds branches until a Likelihood Weight Ratio (LWR) threshold is met (default 0.99999).
- **Fixed Heuristic**: Controlled by `-G` (or `--fix-heur`). It evaluates a fixed percentage of the top branches.

## Output Handling
The primary output is a `.jplace` file. This file contains the placement coordinates and likelihood weights.
- Use **gappa** or **guppy** for downstream analysis (e.g., taxonomic classification, visualization, or diversity metrics).
- Use the `-w` or `--outdir` flag to specify a results directory; otherwise, outputs are saved to the current working directory.
