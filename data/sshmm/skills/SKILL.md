# sshmm

## Overview

ssHMM (Sequence-structure Hidden Markov Model) is a bioinformatics tool designed to model the relationship between RNA primary sequences and their corresponding secondary structures. By utilizing Hidden Markov Models, it identifies structural motifs and patterns that are often missed when looking at sequence alone. This is particularly valuable for researchers working with RNA-binding protein (RBP) data, CLIP-seq experiments, or any study where RNA folding impacts biological function.

## Command Line Usage and Best Practices

The ssHMM suite consists of several core Python scripts. Ensure your environment has dependencies like `forgi` and `RNAstructure` installed, as these are critical for structure prediction and manipulation.

### Preprocessing Datasets

Before training, raw RNA data must be processed to predict secondary structures and format the input.

*   **Basic Preprocessing**: Use `preprocess_dataset` to prepare your binding sites.
*   **Elongation Span**: Use the elongation span parameter to specify how much the original binding sites should be extended. This improves the accuracy of secondary structure predictions by providing more context for the folding algorithms.
*   **Skip Dependency Checks**: If you are running in a containerized environment where you know dependencies are present, use the `-s` flag to skip the manual check for tools like RNAstructure, which can speed up execution.
*   **Structure Limits**: By default, the tool limits the number of predicted RNAshapes to 10 per nucleotide sequence to maintain computational efficiency.

### Training the Model

The core training logic resides in `train_seqstructhmm`.

*   **Algorithm Selection**: Use the `--baum_welch` parameter for unsupervised training of the HMM parameters. 
*   **Version Note**: Ensure you are using version 1.0.7 or later, as earlier versions had an illogical implementation of the Baum-Welch parameter.
*   **Input Format**: The trainer expects the output generated by the `preprocess_dataset` script.

### Visualization and Analysis

Interpreting the HMM states and transitions is easier through visual graphs.

*   **Model Graphing**: Use `draw_model_graph` to regenerate a model graph (PNG) from a model file in XML format. This is the standard way to inspect the learned architecture of your RNA motifs.
*   **Logging**: Monitor the logs during preprocessing; if binding sites have unknown chromosomes or invalid sequences, the tool is designed to filter these out in the first step to prevent training errors.



## Subcommands

| Command | Description |
|---------|-------------|
| draw_model_graph | Takes a ssHMM model file in XML format and produces a model graph in PNG format. |
| preprocess_dataset | Pipeline for the preparation of a CLIP-Seq dataset in BED format. The pipeline consists of the following steps: 1 - Filter BED file 2 - Elongate BED file for later structure prediction 3 - Fetch genomic sequences for elongated BED file 4 - Produce FASTA file with genomic sequences in viewpoint format 5 - Secondary structure prediction with RNAshapes 6 - Secondary structure prediction with RNAstructures |
| train_seqstructhmm | Trains a Hidden Markov Model for the sequence-structure binding preferences of an RNA-binding protein. The model is trained on sequences and structures from a CLIP-seq experiment given in two FASTA-like files. During the training process, statistics about the model are printed on stdout. In every iteration, the current model and a visualization of the model can be stored in the output directory. The training process terminates when no significant progress has been made for three iterations. |
