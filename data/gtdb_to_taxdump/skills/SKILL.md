# gtdb_to_taxdump

## Overview

The `gtdb_to_taxdump` tool bridges the gap between the GTDB taxonomy and software designed for the NCBI taxonomy ecosystem. It processes GTDB taxonomy TSV files (archaea and bacteria) to generate a local taxdump directory containing `nodes.dmp` and `names.dmp`. This allows researchers to leverage the standardized, genome-based phylogeny of GTDB within popular classification and taxonomic manipulation tools.

## Usage Guidelines

### Basic Conversion
To generate a taxdump from a specific GTDB release, provide the URLs or local paths to the archaea and bacteria taxonomy files.

```bash
gtdb_to_taxdump.py \
  https://data.gtdb.ecogenomic.org/releases/release202/202.0/ar122_taxonomy_r202.tsv.gz \
  https://data.gtdb.ecogenomic.org/releases/release202/202.0/bac120_taxonomy_r202.tsv.gz \
  > taxID_info.tsv
```

### Mapping TaxIDs to Metadata
If you have a GTDB metadata table and want to append the newly generated (arbitrary) TaxIDs to it, use the `--table` flag.

```bash
gtdb_to_taxdump.py \
  --table ar122_metadata_r202.tsv \
  ar122_taxonomy_r202.tsv.gz \
  bac120_taxonomy_r202.tsv.gz \
  > taxID_info.tsv
```

### Key Utilities and Extras
The package includes several auxiliary scripts for downstream analysis:

*   **`ncbi-gtdb_map.py`**: Maps classifications between NCBI and GTDB taxonomies. Use `--query-taxonomy` to specify the direction (NCBI to GTDB or vice versa).
*   **`gtdb_to_diamond.py`**: Creates a DIAMOND database from GTDB genomes that includes the taxonomic information for classification.
*   **`lineage2taxid.py`**: Converts a list of lineages (e.g., from GTDB-Tk) into their corresponding TaxIDs using the generated taxdump.
*   **`acc2gtdb_tax.py`**: Generates an `accession2taxid` mapping file by scanning GTDB genomes.

## Best Practices

*   **TaxID Stability**: Be aware that TaxIDs generated by this tool are **not stable** between different runs or GTDB releases. If your workflow requires persistent TaxIDs, consider using the `gtdb-taxdump` alternative.
*   **Data Directory**: Tools like `taxonkit` often look for a `--data-dir`. Ensure the `.dmp` files created by this tool are in the directory you point those downstream tools toward.
*   **Fuzzy Mapping**: When using `ncbi-gtdb_map.py`, enable fuzzy mapping if you do not expect a perfect 1-to-1 relationship between the two taxonomic systems.
*   **Dependencies**: Ensure `taxonkit`, `numpy`, and `networkx` are installed in your environment for full functionality.

| Command | Description |
|---------|-------------|
| acc2gtdb_tax.py | Create Sequence accession to TAXID mapping file |
| gtdb_to_diamond.py | Convert GTDB taxonomy to input for "diamond makedb --taxonmap" |
| gtdb_to_taxdump.py | Convert Genome Taxonomy Database (GTDB) taxonomy files to NCBI taxdump format (names.dmp & nodes.dmp). |
| lineage2taxid.py | Map taxonomic lineages to taxids |
| ncbi-gtdb_map.py | Map between NCBI & GTDB taxonomies |
