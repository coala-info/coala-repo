# bioconductor-exploremodelmatrix

## Overview
ExploreModelMatrix is a tool for visualizing and interpreting design matrices generated by the `model.matrix()` R function. It helps researchers understand how specific design formulas and sample metadata map to model coefficients and fitted values. It provides both an interactive Shiny interface and static plotting capabilities.

## Core Functions

### ExploreModelMatrix()
Launches an interactive Shiny application for exploring designs.
- **Usage**: `ExploreModelMatrix(sampleData = NULL, designFormula = NULL)`
- **Inputs**: A data frame containing sample metadata and an optional formula starting with `~`.
- **Features**: Interactively change formulas, adjust factor reference levels, drop columns, and view variance inflation factors (VIF).

### VisualizeDesign()
Generates static visualizations of the design.
- **Usage**: `vd <- VisualizeDesign(sampleData, designFormula, dropCols = NULL, ...)`
- **Output**: A list containing `plotlist` (ggplot2 objects) and data frames used for the visualization.
- **Display**: Use `cowplot::plot_grid(plotlist = vd$plotlist)` to render the plots.

## Typical Workflow

1. **Prepare Metadata**: Create or load a data frame where rows are samples and columns are experimental factors (e.g., genotype, treatment, batch).
2. **Define Formula**: Specify the relationship between variables (e.g., `~ genotype + treatment` for additive effects or `~ genotype * treatment` for interactions).
3. **Interactive Exploration**:
   - Run `ExploreModelMatrix(sampleData, designFormula)`.
   - Use the sidebar to test different formulas or reference levels.
   - Examine the "Fitted Values" table to see which coefficients contribute to each sample group's mean.
4. **Static Reporting**: Use `VisualizeDesign()` to create figures for publications or reports that document the model structure.

## Advanced Usage and Tips

- **Reference Levels**: In the Shiny app, use the "Reference levels" input to change which factor level is treated as the intercept. This is crucial for interpreting coefficients as fold-changes relative to a specific control.
- **Dropping Columns**: If a design is not full rank (e.g., due to perfect collinearity), use the `dropCols` argument in `VisualizeDesign()` or the interactive UI to remove redundant columns.
- **Exporting App Data**: To capture the data processed within the Shiny app:
  1. Assign the app call: `app <- ExploreModelMatrix(sampleData, designFormula)`.
  2. Run and capture: `out <- shiny::runApp(app)`.
  3. Click the **"Close app"** button within the UI (do not just close the browser tab) to return the internal data list to the R session.
- **Nested Designs**: For complex designs (e.g., patients nested within groups), use formulas like `~ Group + Group:Patient + Group:Treatment` to correctly partition variance.
