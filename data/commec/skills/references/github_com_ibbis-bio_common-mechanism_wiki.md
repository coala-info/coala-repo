The commec package is a free, open-source tool for DNA sequence screening that is part of the Common Mechanism for DNA Synthesis screening . More information about the Common Mechanism project is available on the International Biosecurity and Biosafety Initiative for Science website, including in the Common Mechanism FAQ . Getting Started This GitHub Wiki has documentation for the package , including information about installing commec and interpreting screening results. Install Guide - quickstart, set up commec and required databases Tutorial - Step-by-step guide to your first screening run Command Line Usage - Detailed reference for all commands Features commec is designed to: Screen sequences down to 50 base pairs in length. Sensitively identify sequences of concern known to contribute to pathogenicity Flag regulated pathogens, including those listed on the Australia Group Common Control List and a variety of national control lists, including those from India, China, and South Africa. The commec screen command runs an input FASTA through the following screening steps: Biorisk search : Fast HMM-based search against curated sequence profiles Taxonomy Search : look for best matches to regulated pathogens using a two-step process: Protein search : BLASTX/DIAMOND search against NCBI nr Nucleotide search : BLASTN search against NCBI core_nt Low concern search : Clear earlier flags based on matches to common or conserved sequences Information about the databases supporting screening can be found in the commec-databases repostiory. Installation See Install Guide for more details. A lightweight version of commec, using only the biorisk search to identify certain sequences of concern, can be run in seconds on a laptop using under 1 Gb of curated databases. This can be used to add resilience to existing screening pipelines, but does not meet full regulatory compliance . Comprehensive screening of all regulated pathogens requires the taxonomy search, and is expected to be run in a high-performance bioinformatics computing environment, requiring 275-650 Gb of reference databases and at least 20 Gb of RAM.
