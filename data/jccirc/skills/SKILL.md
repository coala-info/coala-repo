# jccirc

## Overview
JCcirc is a Perl-based computational tool used to assemble the full-length sequences of circRNAs. It bridges the gap between short-read RNA-seq data and complete circular transcripts by utilizing both junction reads and pre-assembled contigs. You should use this skill when you have identified circRNA candidates and need to reconstruct their full sequences using a combination of reference genomes, gene annotations, and de novo assembled contigs.

## Prerequisites
Before running JCcirc, ensure the following tools and files are available:
- **Software**: Perl (Linux environment) and BWA (for alignment).
- **Upstream Assembly**: You must have already run a de novo transcript assembler such as Trinity, SPAdes, or SOAPdenovo-Trans to generate a contig file.
- **Input Files**:
    - Reference Genome (FASTA)
    - Gene Annotation (GTF)
    - Paired-end RNA-seq reads (FASTQ)
    - circRNA junction list (containing chromosome, start, end, host gene, strand, and junction read IDs)

## Command Line Usage

### Basic Execution
The primary script is `JCcirc.pl`. A standard execution command follows this pattern:

```bash
perl JCcirc.pl -C candidates.circ -G genome.fa -F annotation.gtf -O ./output_dir -P 8 --read1 reads_1.fq --read2 reads_2.fq --contig assembled_contigs.fa -D 0
```

### Parameter Guidance
- **`-C, --circ`**: The input list of circRNAs. Ensure this file includes the junction read IDs required for the assembly process.
- **`--contig`**: Path to the contigs generated by your chosen assembler. Common paths include:
    - Trinity: `trinity/inchworm.DS.fa`
    - SPAdes: `spades/K31/transcripts.fasta`
    - SOAPdenovo-Trans: `SOAP/soap.contig`
- **`-D, --difference`**: Controls the strictness of fragment support when generating isoforms.
    - **0**: Recommended for human data or high-confidence assemblies.
    - **1 or 2**: Recommended for plant data or genomes with shorter intron lengths. Higher values are stricter.
- **`-P, --thread`**: Set this to the number of available CPU cores to enable parallel processing.

## Best Practices and Tips
- **Consistency is Key**: Always use the exact same reference genome and GTF annotation file that were used in the upstream circRNA prediction and de novo assembly steps.
- **Paired-End Data**: JCcirc requires paired-end RNA-seq data. Ensure the FASTQ files provided to `--read1` and `--read2` are the same ones used for the de novo assembly.
- **Output Interpretation**:
    - `circ_full_seq.fa`: This is the primary output containing the reconstructed full-length circRNA sequences.
    - `fragment_final.txt`: A tab-delimited file mapping circRNA locations to their genomic fragments.
- **Simulation**: If you need to simulate circRNA data for testing, use the included `CircSimu.pl` script.
