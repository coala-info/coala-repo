# pangwes

## Overview
The `pangwes` suite enables researchers to perform epistasis and co-selection analysis at the pangenome level. By representing genomic variation as a compacted de Bruijn graph (cdBG), the tool identifies significant associations between unitigs (genomic segments). The workflow involves building a graph from raw assemblies, parsing the graph structure, calculating mutual information scores for unitig pairs, and determining the physical or graph-based distances between those pairs to distinguish between physical linkage and true epistatic interactions.

## Core Workflow and CLI Usage

The `pangwes` pipeline relies on a combination of internal tools and external dependencies (`cuttlefish` and `SpydrPick`).

### 1. Graph Construction
Use `cuttlefish` to build a colored compacted de Bruijn graph from a list of genome assemblies.
```bash
cuttlefish build --list assemblies_list.txt \
  --kmer-len 61 \
  --output cdbg_k61 \
  --work-dir /tmp \
  --threads 8
```

### 2. Graph Parsing
Parse the resulting Graphical Fragment Assembly (GFA) file into a format compatible with the distance calculator.
```bash
gfa1_parser cdbg_k61.gfa1 cdbg_k61
```
*   **Input**: The `.gfa1` file generated by cuttlefish.
*   **Output**: Generates `.unitigs`, `.edges`, and `.paths` files.

### 3. Coupling Analysis
Use `SpydrPick` to calculate mutual information (MI) scores. This identifies the top candidate unitig pairs that show statistical association.
```bash
SpydrPick --alignmentfile cdbg_k61.fasta \
  --maf-threshold 0.05 \
  --mi-values 50000000 \
  --threads 8 \
  --verbose
```

### 4. Distance Calculation
Calculate the graph distances between the candidate unitig pairs identified by SpydrPick. This step is crucial for filtering out pairs that are simply adjacent on the genome.
```bash
unitig_distance --unitigs-file cdbg_k61.unitigs \
  --edges-file cdbg_k61.edges \
  --k-mer-length 61 \
  --sgg-paths-file cdbg_k61.paths \
  --queries-file cdbg.*spydrpick_couplings*edges \
  --threads 8 \
  --queries-one-based \
  --run-sggs-only \
  --output-stem cdbg \
  --verbose
```
*   **--queries-file**: This should point to the output generated by SpydrPick.
*   **--run-sggs-only**: Focuses calculation on single genome subgraphs.

### 5. Visualization
Generate a Manhattan plot to visualize the GWES results.
```bash
Rscript scripts/gwes_plot.r [results_file]
```

## Best Practices and Tips
*   **K-mer Selection**: Ensure the `--kmer-len` used in `cuttlefish` matches the `--k-mer-length` provided to `unitig_distance`. A common default is 61, but this may need adjustment based on the complexity of the pangenome.
*   **Memory Management**: `cuttlefish` and `unitig_distance` can be memory-intensive. Always specify a `--work-dir` on a fast disk (like an SSD or NVMe) with sufficient space for temporary files.
*   **Thread Optimization**: Both `SpydrPick` and `unitig_distance` scale well with threads. Use the maximum available cores to reduce the significant computation time required for large-scale MI calculations.
*   **Input Formatting**: The `assemblies_list.txt` for cuttlefish should contain absolute paths to the FASTA files, one per line.
