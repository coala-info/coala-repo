Download PDF To the editor: Most contemporary methods for multipoint linkage analysis are based on either the Elston-Stewart algorithm 1 or the Lander-Green Hidden Markov model 2 (HMM). The Elston-Stewart algorithm scales exponentially in the number of markers, whereas the Lander-Green HMM scales exponentially in the number of pedigree members. The Vitesse 3 and Superlink 4 software packages are based on the Elston-Stewart algorithm and, with the most recent advancements, can handle a moderate number of markers in large pedigrees. Existing multipoint linkage analysis packages based on the Lander-Green HMM, Genehunter 5 , Allegro 6 and Merlin 7 , can handle arbitrarily many markers but are limited to ∼ 25-bit pedigrees. The bit size of a pedigree is 2 n − f − g , where n is the number of nonfounders, f is the number of founders and g is the number of ungenotyped founder couples. We developed a new software package based on multiterminal binary decision diagrams 8 (MTBDDs). This software package is the second version of our Allegro 6 software package ( http://www.decode.com/software/allegro ) and uses an implementation of MTBDDs from the Cudd package ( http://vlsi.colorado.edu/~fabio/CUDD ). It includes all the main features of the older version: lod scores based on allele sharing models, parametric lod scores, NPL scores, haplotyping, counting of forced recombinations, pairwise IBD sharing calculations, expected crossover counts, all analysis available for the X chromosome and a multitude of scoring functions, including imprinting models and the option to specify a weight for every pair of individuals. MTBDDs offer compact storage of probability distributions through two properties: uniqueness and nonredundancy ( Supplementary Fig. 1 online). Some of the compactness captured by MTBDDs is also captured by the gene flow trees used in Merlin 7 . When the genotypes of a marker make certain inheritance patterns impossible, some of the multipoint calculations, which would otherwise have been required, become redundant. The new implementation takes advantage of this redundancy. During multipoint calculations, individual meioses are convolved separately. The order in which the meioses are convolved can affect the amount of redundancy in calculations. An algorithm akin to bubble sort is used to maximize gains. When the genotypes of a marker are completely uninformative about some meioses, convolving these meioses may be delayed until a marker for which genotypes are informative about the meioses is encountered. The main benefit of delaying convolutions is that probability distributions remain simpler and may therefore be stored in more compact MTBDDs. A secondary benefit is that fewer convolution steps are needed. Delaying of convolutions makes a completion step necessary before analysis is done. Often it is possible to do this step exactly, but it may become expensive. An approximation with arbitrary precision is implemented and may be used to reduce the completion cost substantially. This approximation makes calculations feasible at loci for which no genetic information is available. The Viterbi algorithm is used to haplotype and to count forced recombinations. Counting forced recombinations is an important part of constructing a genetic map 9 . Virtually the same algorithm used for multipoint linkage calculations is used for calculations based on the Viterbi algorithm; the difference is that summation is replaced with maximization. Large pedigrees, with a substantial amount of available genotype information, that were previously impossible to analyze, are now manageable. Table 1 shows the effect of missing genotypes on computational costs and compares the performance of the new software package to existing software packages. The amount of available genotype information has a substantial effect on computational costs, to the point that computational complexity can no longer be predicted by simply counting bits. Computational cost is now a complex function of pedigree structure and the observed data. The new version is able to save some calculations for unphenotyped individuals that are added to the pedigree because of the phase information they provide. Therefore, slightly smaller families can be analyzed when all individuals are phenotyped. For smaller pedigrees, there is often little to be gained by using the new method, and it is sometimes slower than older methods when genotype information is sparse. For these, the new version uses the multipoint engine from the older version.
