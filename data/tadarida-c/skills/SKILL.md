# tadarida-c

## Overview

Tadarida-C (Toolbox for Animal Detection on Acoustic Recordings - Classification part) is an R-based suite of scripts used to identify animal species from acoustic features. It primarily utilizes Random Forest models to classify sound events extracted from recordings. The workflow involves preparing a reference database, training classifiers, and calculating probability thresholds to minimize error rates in automated identification.

## Environment Setup

Tadarida-C requires an R environment with specific packages.

1.  **Install Dependencies**: Ensure `randomForest` and `data.table` are installed.
2.  **Initialization**: Run the provided initialization script to set up the environment:
    ```bash
    Rscript init.r
    ```

## Core Workflow and CLI Patterns

The classification process follows a modular sequence of R scripts.

### 1. Prepare Training Data
Use `write_tabase3HF.r` to generate a table from your Reference Sound DataBase (RSDB).

**Command Pattern:**
```bash
Rscript write_tabase3HF.r <RSDB_PATH> <VARSEL_CSV> [SPECIES_LIST_PATH] [GEO_FILTER]
```
*   `RSDB_PATH`: Path to the database generated by Tadarida-L.
*   `VARSEL_CSV`: A CSV file defining which sound features to subset.
*   `SPECIES_LIST_PATH`: (Optional) Table for grouping taxa or filtering by geography.

### 2. Build the Main Classifier
Use `buildClassif_HF.r` to train the primary Random Forest model.

**Command Pattern:**
```bash
Rscript buildClassif_HF.r <MRF_PATH> <VARSEL_CSV> <GEO_FILTER> <SUBSAMP> <GRADIENT_SAMP>
```
*   `MRF_PATH`: Path to `Modified_randomForest.r`.
*   `SUBSAMP`: Minimum subsampling level (numeric).
*   `GRADIENT_SAMP`: Gradient strength (must be negative).

### 3. Build Secondary Classifier (False Positive Reduction)
Use `buildClassifNbSp.r` to help discriminate false positives caused by heterogeneity or low scores.

**Command Pattern:**
```bash
Rscript buildClassifNbSp.r <PROB_ESP_CSV> <SPECIES_REF_PATH> [ADDITIONAL_REF]
```
*   `PROB_ESP_CSV`: The out-of-bag prediction output from the main classifier.

### 4. Performance Analysis and Thresholding
Use `seuils_analyse.r` to convert confidence scores into success probabilities and establish error risk thresholds (1%, 5%, 10%, etc.).

**Command Pattern:**
```bash
Rscript seuils_analyse.r <PROB_ESP_CSV>
```

## Expert Tips

*   **Memory Management**: Classification of large acoustic datasets is RAM-intensive. Ensure your environment has sufficient memory before running `buildClassif_HF.r`.
*   **Geographical Filtering**: Use the `GeoFilter` argument in `write_tabase3HF.r` to restrict the classifier to species known to occur in your specific study area, which significantly improves accuracy.
*   **Feature Selection**: The `VarSel.csv` file is critical. Tailor your feature selection to the frequency range of your target species (e.g., high-frequency bats vs. lower-frequency insects).
*   **Obsolete Scripts**: Do not use `buildClassif.r` or `TadaridaC.r` for new projects; they are replaced by the modular `HF` (High Frequency) script series.
