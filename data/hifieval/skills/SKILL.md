# hifieval

## Overview

hifieval is a specialized benchmarking tool designed to assess the accuracy of error correction algorithms in long-read sequencing workflows. It operates by analyzing Pairwise Mapping Format (PAF) files to identify three primary metrics: Over-Correction (errors introduced by the EC tool), Under-Correction (original errors remaining after EC), and Correct-Correction (successfully fixed errors). This skill provides the necessary procedures to generate compatible alignment files and execute the evaluation.

## Core Workflow

To evaluate error correction, you must follow a three-step process: aligning raw reads, aligning corrected reads, and running the hifieval script.

### 1. Generate Alignment Files (PAF)
hifieval requires PAF files generated by `minimap2`. You must use the `--cs` flag to include the difference string, which hifieval uses to identify specific base-level changes.

```bash
# Align raw reads to reference
minimap2 -t 8 -cx map-hifi --secondary=no --paf-no-hit --cs reference.fasta raw_reads.fastq > raw.paf

# Align corrected reads to reference
minimap2 -t 8 -cx map-hifi --secondary=no --paf-no-hit --cs reference.fasta corrected_reads.fasta > corrected.paf
```

### 2. Execute Evaluation
Run the main script providing the two PAF files.

```bash
hifieval.py -o output_prefix -r raw.paf -c corrected.paf
```

### 3. Advanced: Homopolymer Evaluation
If the error correction tool does not perform homopolymer compression (HPC), you can evaluate error rates within homopolymer regions by providing the reference fasta.

```bash
hifieval.py -o output_prefix -r raw.paf -c corrected.paf -h reference.fasta
```

## Metric Definitions

*   **OC (Over-correction):** Errors present in the corrected reads that were not present in the raw reads.
*   **UC (Under-correction):** Errors present in the raw reads that persist in the corrected reads.
*   **CC (Correct-correction):** Errors present in the raw reads that were successfully removed in the corrected reads.

## Expert Tips and Best Practices

*   **Mandatory Flag:** Always ensure `minimap2` is run with `--cs`. Without this tag, the PAF file will not contain the alignment details required for hifieval to calculate metrics.
*   **HPC Handling:** If your EC tool produces Homopolymer Compressed (HPC) reads, you must use `seqtk hpc` on both your raw reads and your reference genome before generating the PAF files to ensure a fair comparison.
*   **Secondary Alignments:** Use `--secondary=no` in `minimap2` to simplify the evaluation and focus on the primary alignment for each read.
*   **Output Interpretation:**
    *   `summary.tsv`: Use this for per-read analysis (contains read names and specific error counts).
    *   `metric.eval.tsv`: Use this for high-level reporting of TPR (True Positive Rate) and FPR (False Positive Rate) across chromosomes.
    *   `hp.ErrorRate.tsv`: Use this to identify if your EC tool struggles with specific homopolymer lengths (e.g., poly-A tails).
