# miniprot

## Overview

miniprot is a specialized alignment tool designed to map protein sequences directly to genomic DNA. Unlike traditional protein-protein aligners, miniprot accounts for the biological complexities of eukaryotic genes, including introns (splicing) and sequencing errors (frameshifts). It serves as a significantly faster alternative to legacy tools like GeneWise or Exonerate, making it suitable for modern, large-scale genome annotation workflows.

## Installation

The most efficient way to install miniprot is via Bioconda:

```bash
conda install bioconda::miniprot
```

## Common CLI Patterns

### 1. Basic Alignment
Align a protein set to a genome and output in Protein PAF format (default):

```bash
miniprot -t8 genome.fasta proteins.faa > output.paf
```

### 2. Indexing (Recommended for Large Genomes)
Indexing is memory-intensive and slow but significantly speeds up subsequent alignment steps. Always pre-build the index when working with whole genomes.

```bash
# Create the index
miniprot -t16 -d genome.mpi genome.fasta

# Use the index for alignment
miniprot -t16 genome.mpi proteins.faa > output.paf
```

### 3. Generating GFF3 Output
For gene annotation tasks, GFF3 is the standard output format. Detailed alignment information is typically embedded in `##PAF` lines within the GFF3.

```bash
miniprot --gff genome.mpi proteins.faa > output.gff
```

### 4. Visualizing Residue Alignments
To see the actual amino acid to DNA alignment (including stop codons and frameshifts), use the `--aln` flag:

```bash
miniprot --aln genome.mpi proteins.faa
```

## Expert Tips and Best Practices

*   **Automatic Intron Sizing**: When aligning to a whole genome, always use the `-I` flag. This allows miniprot to automatically set the maximum intron size based on the genome size, which improves accuracy.
*   **Manual Intron Control**: If you have specific knowledge of the target species' biology, use `-G <int>` to explicitly set the maximum intron size (e.g., `-G 50000`).
*   **Performance Tuning**: miniprot performance can vary based on the compiler. If building from source, using a recent version of GCC (v10+) is recommended for stable performance.
*   **Handling Distant Homologs**: By default, miniprot is optimized for relatively close homologs. For more distant homologs, you may need to tune seeding parameters (like reducing k-mer size), though this will decrease performance.
*   **Splice Site Integration**: For higher accuracy in vertebrate or insect genomes, you can provide splice scores generated by `minisplice` using the `--spsc` option.
