## caspeak_peak

### Tool Description
Detects peaks of mobile element insertions in sequencing data.

### Metadata
- **Docker Image**: quay.io/biocontainers/caspeak:1.1.5--pyhdfd78af_0
- **Homepage**: https://github.com/Rye-lxy/CasPeak
- **Package**: https://anaconda.org/channels/bioconda/packages/caspeak/overview
- **Validation**: PASS

### Original Help Text
```text
usage: caspeak peak [-h] --read FILE --ref FILE --insert FILE --target-start
                    START --target-end END [-x] [--bedtools-genome GENOME]
                    [--mask RMSK] [--genome-maf MAF] [--insert-maf MAF]
                    [--workdir DIR] [--thread THREAD] [--min-read-length NUM]
                    [--max-prop NUM] [--min-prop NUM] [--max-trim-length NUM]
                    [--padding NUM] [--min-cov NUM] [-v]

options:
  -h, --help            show this help message and exit
  --read FILE           the read FASTA/FATSQ file (required)
  --ref FILE            the reference genome FASTA file (required)
  --insert FILE         consensus sequence of the MEI FASTA file (required)
  --target-start START  start position of the target in the insert sequence
                        (required)
  --target-end END      end position of the target in the insert sequence
                        (required)
  -x, --exog            set the insertion exogenous
  --bedtools-genome GENOME
                        genome data for bedtools (default: hg38 file in
                        bedtools)
  --mask RMSK           rmsk file on the reference genome generated by
                        RepeatMasker
  --genome-maf MAF      alignment to the genome (default:
                        ./lastal/read_to_ref.maf)
  --insert-maf MAF      alignment to the insertion sequence (default:
                        ./lastal/read_to_insert.maf)
  --workdir DIR         working directory (default: current directory)
  --thread THREAD       number of threads (default: 1)
  -v, --verbose         show progress messages and data

parameters for filtering reads:
  --min-read-length NUM
                        minimum read length (default: 500)
  --max-prop NUM        maximum proportion of the read aligned to the
                        reference genome (default: 1)
  --min-prop NUM        minimum proportion of the read aligned to the
                        reference genome (default: 0)

parameters for trimming reads:
  --max-trim-length NUM
                        maximum length of the adaptor to be trimmed (default:
                        100)
  --padding NUM         padding for the target region (default: 20)

parameters for peak detection:
  --min-cov NUM         minimum coverage for a peak (default: 2)
```

### Original Help Text
```text
usage: caspeak valid [-h] [--trim-read FILE] [--peak-bed FILE] [--workdir DIR]
                     [--thread THREAD] [--sample N] [--min-insprop PROP]
                     [--min-inslen LEN] [--min-asb-seq NUM] [--lib LIB]
                     [--names NAME] [--names-re REGEX] [--vcf] [-v]

options:
  -h, --help          show this help message and exit
  --trim-read FILE    read FASTA file after trimming (default:
                      ./peak/trimmed_reads.fasta)
  --peak-bed FILE     peak BED file (default: ./peak/peaks.bed)
  --workdir DIR       working directory (default: current directory)
  --thread THREAD     number of threads (default: 1)
  -v, --verbose       show progress messages and data

parameters for validation:
  --sample N          at most N reads are selected for assembly (default: 20)
  --min-insprop PROP  minimum proportion of the MEI in the insert sequence
                      (default: 0.2)
  --min-inslen LEN    minimum length of the MEI in the insert sequence
                      (default: 80)
  --min-asb-seq NUM   minimum number of assembled sequences (default: 1)
  --lib LIB           the library (a set of closely related sequences) used to
                      distinguish true insertion from these related sequences
  --names NAME        names of the insertion sequences in the library
  --names-re REGEX    regular expression to match the name of the insertion
                      sequences
  --vcf               output the validated result in VCF format
```

usage: caspeak plot [-h] [-v] --maf MAF [--prefix PREFIX] [-x INT] [-y INT]
                    [-m M] [-1 PATTERN] [-2 PATTERN] [--alignments FILE]
                    [--sort1 N] [--sort2 N] [--strands1 N] [--strands2 N]
                    [--max-gap1 FRAC] [--max-gap2 FRAC] [--pad FRAC] [-j N]
                    [--border-pixels INT] [-a FILE] [-b FILE] [-f FILE]
                    [-s SIZE] [--labels1 N] [--labels2 N] [--rot1 ROT]
                    [--rot2 ROT] [-c COLOR] [-r COLOR] [--border-color COLOR]
                    [--margin-color COLOR] [--exon-color COLOR]
                    [--cds-color COLOR] [--bridged-color COLOR]
                    [--unbridged-color COLOR]

options:
  -h, --help            show this help message and exit
  -v, --verbose         show progress messages and data
  --maf MAF             the maf file to plot (required)
  --prefix PREFIX       prefix of the output figure name (default: fig/peak)
  -x INT, --width INT   maximum width in pixels (default: 1000)
  -y INT, --height INT  maximum height in pixels (default: 1000)
  -m M, --maxseqs M     maximum number of horizontal or vertical sequences
                        (default=100)
  -1 PATTERN, --seq1 PATTERN
                        which sequences to show from the 1st genome
  -2 PATTERN, --seq2 PATTERN
                        which sequences to show from the 2nd genome
  --alignments FILE     secondary alignments
  --sort1 N             genome1 sequence order: 0=input order, 1=name order,
                        2=length order, 3=alignment order (default=1)
  --sort2 N             genome2 sequence order: 0=input order, 1=name order,
                        2=length order, 3=alignment order (default=1)
  --strands1 N          genome1 sequence orientation: 0=forward orientation,
                        1=alignment orientation (default=0)
  --strands2 N          genome2 sequence orientation: 0=forward orientation,
                        1=alignment orientation (default=0)
  --max-gap1 FRAC       maximum unaligned (end,mid) gap in genome1: fraction
                        of aligned length (default=1,4)
  --max-gap2 FRAC       maximum unaligned (end,mid) gap in genome2: fraction
                        of aligned length (default=1,4)
  --pad FRAC            pad length when cutting unaligned gaps: fraction of
                        aligned length (default=0.04)
  -j N, --join N        join: 0=nothing, 1=alignments adjacent in genome1,
                        2=alignments adjacent in genome2 (default=0)
  --border-pixels INT   number of pixels between sequences (default=1)
  -a FILE, --bed1 FILE, --rmsk1 FILE, --genePred1 FILE, --gap1 FILE
                        read genome1 annotations
  -b FILE, --bed2 FILE, --rmsk2 FILE, --genePred2 FILE, --gap2 FILE
                        read genome2 annotations

Text options:
  -f FILE, --fontfile FILE
                        TrueType or OpenType font file
  -s SIZE, --fontsize SIZE
                        TrueType or OpenType font size (default: 14)
  --labels1 N           genome1 labels: 0=name, 1=name:length,
                        2=name:start:length, 3=name:start-end (default=0)
  --labels2 N           genome2 labels: 0=name, 1=name:length,
                        2=name:start:length, 3=name:start-end (default=0)
  --rot1 ROT            text rotation for the 1st genome (default=h)
  --rot2 ROT            text rotation for the 2nd genome (default=v)

Color options:
  -c COLOR, --forwardcolor COLOR
                        color for forward alignments (default: red)
  -r COLOR, --reversecolor COLOR
                        color for reverse alignments (default: blue)
  --border-color COLOR  color for pixels between sequences (default=black)
  --margin-color COLOR  margin color
  --exon-color COLOR    color for exons (default=PaleGreen)
  --cds-color COLOR     color for protein-coding regions (default=LimeGreen)
  --bridged-color COLOR
                        color for bridged gaps (default: yellow)
  --unbridged-color COLOR
                        color for unbridged gaps (default: orange)
