# vgp-hybrid-scaffolding-with-hic-data

## Overview

This workflow is a specialized component of the [Vertebrate Genomes Project (VGP)](https://vertebrategenomesproject.org/) assembly pipeline, designed to perform hybrid scaffolding using HiC data. It is engineered to improve the continuity of genomic assemblies by organizing contigs or existing scaffolds (such as those from Bionano mapping) into larger, chromosome-level structures based on chromatin proximity information.

The process begins by aligning HiC forward and reverse reads to the input assembly using [BWA-MEM2](https://github.com/bwa-mem2/bwa-mem2). These alignments are filtered and merged via [Bellerophon](https://github.com/iuc/bellerophon) before being passed to [SALSA](https://github.com/marbl/salsa), which performs the core scaffolding. The workflow is flexible, accommodating both small and large genomes (>100 Mbp) and optionally utilizing sequence graphs to guide the scaffolding process.

To ensure the integrity of the resulting assembly, the workflow integrates comprehensive evaluation and visualization steps. It utilizes [BUSCO](https://busco.ezlab.org/) to assess gene content completeness and [QUAST](https://quast.sourceforge.net/) for structural assembly metrics. Additionally, it generates HiC contact map visualizations using [PretextMap](https://github.com/wtsi-hpag/PretextMap) and Pretext Snapshot, allowing for the manual or automated inspection of the final scaffolded output.

## Inputs and data preparation

| Step | Label | Type | Notes |
| --- | --- | --- | --- |
| 0 | Sequence graph - Optional | data_input | Sequence graphs encoded in GFA format. (-g) |
| 1 | HiC Forward reads | data_input | HiC forward reads in Fastq Format. The reads must be in the same order as reverse reads. |
| 2 | Scaffolded Assembly | data_input | Scaffold generated by "VGP Bionano" workflows or contigs Generated by the "VGP Hifiasm" workflow. |
| 3 | HiC reverse reads | data_input | HiC reverse reads in Fastq Format. The reads must be in the same order as forward reads. |
| 4 | Estimated genome size - Parameter File | data_input | Parameter file generated in the VGP Hifiasm workflow. Estimated reference genome size (in bp) for computing NGx statistics |
| 5 | Is genome large (> 100 Mbp)? | parameter_input | Quast : Use optimal parameters for evaluation of large genomes. Affects speed and accuracy. In particular, imposes --eukaryote --min-contig 3000 --min-alignment 500 --extensive-mis-size 7000 (can be overridden manually with the corresponding options). In addition, this mode tries to identify misassemblies caused by transposable elements and exclude them from the number of misassemblies. (--large) |


Ensure your HiC forward and reverse reads are in FASTQ format and the scaffolded assembly is in FASTA format. While inputs are listed individually, organizing HiC reads into paired collections can streamline data management before mapping with BWA-MEM2. The estimated genome size should be provided as a single-value text file, and the large genome parameter must be set correctly to optimize SALSA and Bellerophon performance. Consult the README.md for comprehensive instructions on parameter tuning and specific file requirements. You can use `planemo workflow_job_init` to generate a `job.yml` file for automated execution.

## How to run (Planemo)

### Local Galaxy (default engine)

```bash
planemo run hic.ga job.yml --download_outputs --output_directory . --output_json output.json
```

Optional: `--galaxy_root /path/to/galaxy` speeds up spinning up a local Galaxy.

### Docker Galaxy

Append e.g. `--engine docker_galaxy --ignore_dependency_problems` (requires Docker).

### External Galaxy (usegalaxy.org / EU / …)

```bash
planemo run hic.ga job.yml --engine external_galaxy --galaxy_url SERVER_URL --galaxy_user_key YOUR_API_KEY
```

Create a reusable profile: `planemo profile_create myprofile --galaxy_url ... --galaxy_user_key ... --engine external_galaxy` then `planemo run hic.ga job.yml --profile myprofile`.

### Job file

- Generate a template: `planemo workflow_job_init hic.ga -o job.yml`
- Lint: `planemo workflow_lint hic.ga`

See **[Running Galaxy workflows](https://planemo.readthedocs.io/en/master/running.html)** in the Planemo documentation.

## Invocation outputs

After a successful run with `--download_outputs`, files appear under your chosen `--output_directory`. `--output_json` records CWL-style metadata for outputs (paths, checksums).

On an external Galaxy, use `planemo invocation_download INVOCATION_ID --profile ...` to fetch outputs, or track progress with `planemo workflow_track INVOCATION_ID`.

Workflow steps marked as **workflow outputs** in the editor are listed under **Workflow outputs**; exact filenames depend on tools and Galaxy configuration.
