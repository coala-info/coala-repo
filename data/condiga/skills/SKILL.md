# condiga

## Overview
ConDiGA (Contigs Directed Gene Annotation) is a specialized pipeline designed to improve the accuracy of protein sequence databases used in metaproteomics. By integrating taxonomic classification, gene prediction, and contig coverage, it maps predicted genes to the most relevant reference genomes. This process ensures that the resulting protein database is both comprehensive and taxonomically representative of the original metagenomic sample.

## Installation
The recommended installation method is via Bioconda to ensure all dependencies (such as Minimap2 and TaxonKit) are managed correctly:

```bash
conda create -n condiga -c conda-forge -c bioconda condiga
conda activate condiga
```

## Preprocessing Workflow
ConDiGA requires specific inputs generated by external bioinformatics tools. Follow this sequence:

1.  **Assembly**: Assemble reads using MEGAHIT (the only currently supported assembler).
2.  **Taxonomic Annotation**: Annotate contigs using Kraken2, Kaiju, or BLAST.
3.  **Format Conversion**: Use the `convert` utility (bundled with ConDiGA) to standardize the annotation output.
    ```bash
    # Example for Kraken2 results
    convert -i kraken_results.txt -t kraken -o Kraken_Output_Folder
    ```
4.  **Coverage Calculation**: Use CoverM to generate a contig coverage TSV file.
5.  **Gene Prediction**: Use MetaGeneMark (`gmhmmp`) to predict genes within the contigs.
6.  **Reference Summary**: Download the NCBI `assembly_summary.txt` file for the relevant domain (e.g., bacteria).

## Command Line Usage
Once preprocessing is complete, run the main pipeline:

```bash
condiga -c final.contigs.fa \
        -ta Kraken_Output_Folder/converted_results.txt \
        -g final.contigs.fa.lst \
        -cov contig_coverages.tsv \
        -as assembly_summary.txt \
        -o output_results
```

### Key Options and Thresholds
- `-ra, --rel-abundance`: Minimum relative abundance cutoff (default: 0.0001).
- `-gc, --genome-coverage`: Minimum genome coverage cutoff (default: 0.001).
- `-mt, --map-threshold`: Minimum mapping length threshold (default: 0.5).
- `-t, --nthreads`: Number of threads to utilize (default: 8).

## Expert Tips
- **TaxonKit Setup**: If you installed ConDiGA via `pip` or intend to use Kaiju results, you must manually download and configure the NCBI taxdump database for TaxonKit.
- **Input Validation**: The `convert` tool is mandatory for non-standard taxonomic outputs. ConDiGA will fail if the `-ta` input is not in the expected standardized format.
- **Output Files**: 
    - `all_genes.faa`: The primary amino acid sequence database for metaproteomic search.
    - `genes.species.mapped.xlsx`: A spreadsheet detailing the mapping between genes and species.
    - `Assemblies/`: Contains the FASTA files of reference genomes downloaded during the run.
