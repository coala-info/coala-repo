The most basic feature of dominate exposes a class for each HTML element, where the constructor accepts child elements, text, or keyword attributes. dominate nodes return their HTML representation from the __str__ , __unicode__ , and render() methods. print ( html ( body ( h1 ( 'Hello, World!' )))) &lt; html &gt; &lt; body &gt; &lt; h1 &gt; Hello, World! &lt;/ h1 &gt; &lt;/ body &gt; &lt;/ html &gt; Attributes Dominate can also use keyword arguments to append attributes onto your tags. Most of the attributes are a direct copy from the HTML spec with a few variations. For attributes class and for which conflict with Python's reserved keywords , you can use the following aliases: class for _class _for cls fr className htmlFor class_name html_for test = label ( cls = 'classname anothername' , fr = 'someinput' ) print ( test ) &lt; label class =" classname anothername " for =" someinput " &gt; &lt;/ label &gt; Use data_* for custom HTML5 data attributes . test = div ( data_employee = '101011' ) print ( test ) &lt; div data-employee =" 101011 " &gt; &lt;/ div &gt; You can also modify the attributes of tags through a dictionary-like interface: header = div () header [ 'id' ] = 'header' print ( header ) &lt; div id =" header " &gt; &lt;/ div &gt; Complex Structures Through the use of the += operator and the .add() method you can easily create more advanced structures. Create a simple list: list = ul () for item in range ( 4 ): list += li ( 'Item #' , item ) print ( list ) &lt; ul &gt; &lt; li &gt; Item #0 &lt;/ li &gt; &lt; li &gt; Item #1 &lt;/ li &gt; &lt; li &gt; Item #2 &lt;/ li &gt; &lt; li &gt; Item #3 &lt;/ li &gt; &lt;/ ul &gt; dominate supports iterables to help streamline your code: print ( ul ( li ( a ( name , href = link ), __pretty = False ) for name , link in menu_items )) &lt; ul &gt; &lt; li &gt; &lt; a href =" /home/ " &gt; Home &lt;/ a &gt; &lt;/ li &gt; &lt; li &gt; &lt; a href =" /about/ " &gt; About &lt;/ a &gt; &lt;/ li &gt; &lt; li &gt; &lt; a href =" /downloads/ " &gt; Downloads &lt;/ a &gt; &lt;/ li &gt; &lt; li &gt; &lt; a href =" /links/ " &gt; Links &lt;/ a &gt; &lt;/ li &gt; &lt;/ ul &gt; A simple document tree: _html = html () _body = _html . add ( body ()) header = _body . add ( div ( id = 'header' )) content = _body . add ( div ( id = 'content' )) footer = _body . add ( div ( id = 'footer' )) print ( _html ) &lt; html &gt; &lt; body &gt; &lt; div id =" header " &gt; &lt;/ div &gt; &lt; div id =" content " &gt; &lt;/ div &gt; &lt; div id =" footer " &gt; &lt;/ div &gt; &lt;/ body &gt; &lt;/ html &gt; For clean code, the .add() method returns children in tuples. The above example can be cleaned up and expanded like this: _html = html () _head , _body = _html . add ( head ( title ( 'Simple Document Tree' )), body ()) names = [ 'header' , 'content' , 'footer' ] header , content , footer = _body . add ([ div ( id = name ) for name in names ]) print ( _html ) &lt; html &gt; &lt; head &gt; &lt; title &gt; Simple Document Tree &lt;/ title &gt; &lt;/ head &gt; &lt; body &gt; &lt; div id =" header " &gt; &lt;/ div &gt; &lt; div id =" content " &gt; &lt;/ div &gt; &lt; div id =" footer " &gt; &lt;/ div &gt; &lt;/ body &gt; &lt;/ html &gt; You can modify the attributes of tags through a dictionary-like interface: header = div () header [ 'id' ] = 'header' print ( header ) &lt; div id =" header " &gt; &lt;/ div &gt; Or the children of a tag though an array-line interface: header = div ( 'Test' ) header [ 0 ] = 'Hello World' print ( header ) &lt; div &gt; Hello World &lt;/ div &gt; Comments can be created using objects too! print ( comment ( 'BEGIN HEADER' )) &lt;!--BEGIN HEADER--&gt; print ( comment ( p ( 'Upgrade to newer IE!' ), condition = 'lt IE9' )) &lt;!--[if lt IE9]&gt; &lt;p&gt;Upgrade to newer IE!&lt;/p&gt; &lt;![endif]--&gt; Rendering By default, render() tries to make all output human readable, with one HTML element per line and two spaces of indentation. This behavior can be controlled by the __pretty (default: True except for certain element types like pre ) attribute when creating an element, and by the pretty (default: True ), indent (default: ) and xhtml (default: False ) arguments to render() .
