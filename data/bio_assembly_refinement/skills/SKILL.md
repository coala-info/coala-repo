# bio_assembly_refinement

## Overview

The `bio_assembly_refinement` tool is a specialized pipeline designed to transform draft bacterial assemblies into finished, high-quality genomes. It automates the tedious manual steps of the finishing process, including the removal of redundant or short contigs, the identification and trimming of artificial overlaps at contig ends, and the circularization of sequences based on conserved marker genes like `dnaA`. By integrating tools like MUMmer for alignment and Prodigal for gene finding, it prepares assemblies for final polishing and ensures that circular elements are correctly oriented and joined.

## Core Workflow and Best Practices

### 1. Contig Filtering and Redundancy Removal
The tool first cleans the assembly by removing noise.
- **Length Filtering**: Use `cutoff_contig_length` (default 10,000 bp) to discard small, uninformative contigs. If working with small plasmids, ensure this value is lowered.
- **Containment**: It identifies contigs that are completely contained within larger ones using a percent identity threshold (`contained_percent_match`, default 95%). This is critical for removing redundant sequences generated by assembly algorithms.

### 2. Overlap Detection and Trimming
Long-read assemblers often produce contigs with overlapping ends that represent circular molecules.
- **Nucmer Alignment**: The tool uses `nucmer` to find overlaps between the start and end of contigs.
- **Parameter Tuning**: 
    - `overlap_offset`: Defines how far from the ends to look for an overlap (default 1,000 bp).
    - `overlap_min_length`: Minimum overlap size to consider (default 1,000 bp).
    - `overlap_max_length`: Maximum overlap size (default 3,000 bp).
- **Trimming**: Once identified, these overlaps are trimmed to create a clean "join" point for circularization.

### 3. Circularization and Orientation
To standardize the assembly, the tool re-orients circular contigs.
- **Marker Genes**: It searches for `dnaA` (for chromosomes) or other reference sequences (like `repA/B` for plasmids) provided in a fasta file.
- **Standardized Start**: The contig is "rotated" so that the start of the sequence matches the start of the identified marker gene.
- **Gene Finding**: It relies on `prodigal` to identify these genes within the contig. Ensure `prodigal` is in your system PATH.

### 4. Join Refinement (Polishing)
After circularization and trimming, the new "join" point may contain small errors.
- **Quiver Integration**: The tool can automatically run the PacBio `quiver` (or `resequencing`) protocol on the refined contigs.
- **Data Requirements**: This requires the original PacBio `.bax.h5` files and the `pacbio_smrtanalysis` suite to be installed.

## Configuration Parameters

When executing the refinement, consider these key logic parameters:

| Parameter | Default | Description |
| :--- | :--- | :--- |
| `cutoff_contig_length` | 10000 | Minimum length for a contig to be kept. |
| `overlap_percent_identity` | 85 | Minimum identity for end-to-end overlaps. |
| `dnaA_hit_percent_identity` | 80 | Identity threshold for finding the start gene. |
| `dnaA_hit_length_minimum` | 95 | Minimum % length of the reference gene that must be found. |
| `trim` | True | Whether to automatically trim identified overlaps. |

## Expert Tips
- **Reference Genes**: Always provide a comprehensive `dnaA_sequence` file. If you are assembling a specific species, using a species-specific `dnaA` sequence improves the accuracy of the circularization start point.
- **Dependency Check**: Before running, verify that `nucmer` (MUMmer), `prodigal`, and `pyfastaq` are installed and accessible.
- **Working Directory**: Use the `working_directory` parameter to keep your project organized, as the tool generates several intermediate files during the nucmer and re-assembly phases.
- **Debug Mode**: If the circularization fails, set `debug` to true to inspect the temporary files and nucmer output to see why overlaps or genes weren't detected.
