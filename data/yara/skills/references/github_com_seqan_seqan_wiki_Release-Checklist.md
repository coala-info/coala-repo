Then you have to create one package for every app on all supported platforms (linux/64, mac/64, freebsd/64, windows/64 and windows/32). On Linux, make sure that you have cmake-3.1 to get .tar.xz instead of .tar.bz2 . A simple loop for making all the apps on unix could look like this: # regular build rm -rf CMake * for APP in $( cd ../../seqan/apps/ &amp;&amp; ls -d * / | sed -e ' s#\/##g ' ) ; do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP: $APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1 ; make -j 4 ; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64 ; done # optimized build rm -rf CMake * for APP in $( cd ../../seqan/apps/ &amp;&amp; ls -d * / | sed -e ' s#\/##g ' ) ; do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP: $APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1 -DSEQAN_ARCH_SSE4=1 ; make -j 4 ; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64 ; done # 32bit build rm -rf CMake * for APP in $( cd ../../seqan/apps/ &amp;&amp; ls -d * / | sed -e ' s#\/##g ' ) ; do cmake ../../seqan -DSEQAN_BUILD_SYSTEM=APP: $APP -DCMAKE_CXX_COMPILER=g++49 -DSEQAN_STATIC_APPS=1 -DCMAKE_CXX_FLAGS=-m32 ; make -j 4 ; cpack -D CPACK_DEBIAN_PACKAGE_ARCHITECTURE=amd64 ; done Tag release Locally do: git tag -a seqan-v2.x.y As text add Release 2.x.y of the SeqAn library. See https://github.com/seqan/seqan/blob/master/CHANGELOG.rst for a full list of changes. Then eplicity push the tag to upstream: git push upstream seqan-v2.x.y Publish the release on GitHub Go to https://github.com/seqan/seqan/tags and click "Add release notes" beside the new tag. As release title enter seqan-v2.x.y As text enter See https://github.com/seqan/seqan/blob/master/CHANGELOG.rst for a summary of changes. See http://packages.seqan.de for binary downloads of the applications. Select the library source tarballs previously built, i.e. the zip/tar files that are less then 10MB big, and add them to the release (unfittingly called "Attach binaries" but that doesn't matter). Finally click "Publish release". Updating http://packages.seqan.de First copy the package files that you created to /web/packages.seqan.de/htdocs (on any imp.fu-berlin.de ). Assuming you have all the zipballs and tarballs in /tmp/packages , you can: cd /web/packages.seqan.de/htdocs for APP in $( ls -1 | sort -r | sed -e ' s#\/##g ' ) ; do mv /tmp/packages/ ${APP} * ${APP} / ; done Next run the script that generates the static packages site. It can be found in every seqan-git-checkout: git clone https://github.com/seqan/packages.seqan.de.git python /your/seqan-git-checkout/release_page.py -d /web/packages.seqan.de/htdocs/ -o /web/packages.seqan.de/htdocs/ Updating http://docs.seqan.de cd /web/docs.seqan.de/htdocs Edit the index.html by adding a new link with 2.x.y . mkdir -p /web/docs.seqan.de/htdocs/seqan/2.x.y Copy the dox files from the directory where you built the SEQAN_LIBRARY_RELEASE . cp -r /path/to/seqan-2.x.y-package/dox/html/ * /web/docs.seqan.de/htdocs/seqan/2.x.y/ Updating http://seqan.readthedocs.org Log into https://readthedocs.org select SeqAn project select Admin select Versions find the new tag and set the "active"-checkbox Bumping app version This code snippet, run from inside the apps folder increases the last digit of each app's version string: for f in $( find ./ -name CMakeLists.txt ) ; do gawk -i inplace -F \" ' $0 ~ "^set\\ \\(SEQAN_APP_VERSION" { split($2, num, "."); printf $1 "\"" num[1] "." num[2] "." num[3]+1 "\"" $3 "\n"; next }; { print $0 } ' $f ; done You can easily move the +1 to num[2] if you instead want to increase the second digit. Collecting internal app version in a resource.txt file Paste code snippet into a executable, run from inside the seqan root folder to collect versions in a new file called resources.txt: #! /bin/sh if [ " $# " -ne 1 ] ; then echo " [INPUT ERROR] Please specify the current library release (e.g. 2.3.0) as the 2nd command line argument! " return fi # pass current library release as command line argument CURRENT_RELEASE= $1 echo seqan ' \t ' $CURRENT_RELEASE &gt; resources.txt # get app version from CMakeList.txt in each app folder for APP_DIR in $( ls -d apps/ * / ) do VERSION= $( awk ' match($0, /SEQAN_APP_VERSION "(.*)"/, v) {print v[1]} ' $APP_DIR /CMakeLists.txt ) for APP in $( awk ' match($0, /add_executable\s*\(([^ ]+)/, v) {print v[1]} ' $APP_DIR /CMakeLists.txt ) do echo $APP ' \t ' $VERSION &gt;&gt; resources.txt done done Add external apps manually to resources.txt (tab separated file): lambda_indexer 1.0.2 Send resources.txt to the server in Tuebingen.
