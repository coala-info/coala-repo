Download From Github The latest ChIPs release is available on the releases page To compile from the Github repository, run: git clone https://github.com/gymreklab/chips cd chips/ mkdir build cd build/ cmake .. make From source files To compile from source on the releases page , download the file chips-2.2-Source.tar.gz and run: tar -xvf chips-2.4-Source.tar.gz cd chips-2.4-Source/ mkdir build cd build/ cmake .. make This will generate a binary file chips , which you can then copy to a place on your $PATH . There is also a precompiled binary available on the release page . Download and unzip chips-2.2-Linux_x86_64.tar.gz and copy the binary located in chips-2.2-Linux_x86_64/bin/chips somewhere onto your $PATH . 2. Basic usage ChIPs is a single command line tool that contains several modules. To see available modules type: chips The following modules are available: learn : Learn key parameters from existing ChIP-seq datasets. Learn takes in alinged reads (BAM) and peaks (BED) and outputs a model parameters file. simreads : Simulate ChIP-seq reads based on model and experimental parameters. Simulate takes in peaks (BED), model parameters (either user-specified, or learned from an existing dataset using learn ) and outputs raw reads (FASTQ). Basic usage is shown below. See detailed usage below for more info. chips learn \ -b &lt;reads.bam&gt; \ -p &lt;peaks&gt; \ -t &lt;homer|bed&gt; -c &lt;int&gt; \ -o &lt;outprefix&gt; chips simreads \ -p &lt;peaks&gt; \ -f &lt;ref.fa&gt; -t &lt;homer|bed|wce&gt; \ -o &lt;outprefix&gt; 3. Detailed usage chips learn Required parameters: -p &lt;peaks&gt; : file containing peaks. -t &lt;homer|bed&gt; : Specify the format of the peaks file. Options are "bed" or "homer". -o &lt;outprefix&gt; : Prefix to name output files. Outputs file &lt;outprefix&gt;.json with learned model parameters. -b &lt;file.bam&gt; : BAM file containing aligned reads. Should be sorted and indexed. To accurately estimate PCR duplicate rate, duplicates must be flagged e.g. using Picard. Both paired-end or single-end data are supported. -c &lt;int&gt; : The index of the BED or homer peak file column used to score each peak (index starting from 1) Optional parameters: -r &lt;float&gt; : Ignore peaks with top r% of peak scores. Default: 0. --noscale : Don't scale peak scores by the max score. Treat given scores in the input bed file as binding probabilities. Default: false. --scale-outliers : Set all peaks with scores &gt;3x median score to have binding prob 1. Recommended with real data. Default: flase. --region &lt;str&gt; : Only consider peaks from this region chrom:start-end. Default: genome-wide. BAM-file arguments: --paired : Data is paired. Default: false. Fragment length estimation arguments (for single-end data only): --est &lt;int&gt; : Estimated fragment length. Used as a rough guess to guide inference of fragment length distribution from single end data. Please set this number as the loose upper-bound of your best estimation. This can result in more robust estimates especially for data with narrow peaks. Default: 300. --thres &lt;float&gt; : For estimating fragment length distribution from single end data, only consider peaks with scores above this threshold. ChIPs applies --thres or --thres-scale whichever is stricter. Default: 100. --thres-scale &lt;float&gt; : Scale threshold for peak scores. Only consider peaks with at least this score after scaling scores to be between 0-1. ChIPs applies --thres or --thres-scale whichever is stricter. Default: 0. chips simreads Required parameters: -p &lt;peaks&gt; : file containing peaks. -t &lt;homer|bed|wce&gt; : Specify the format of the peaks file. Options are "bed" or "homer" when loading peaks. Specify -t wce and no peaks input file to simulate whole cell extract control data. -f &lt;ref.fa&gt; : Reference genome fasta file. Must be indexed (e.g. samtools faidx &lt;ref.fa&gt; ) -o &lt;outprefix&gt; : Prefix to name output files. Outputs &lt;outprefix&gt;.fastq for single-end data or &lt;outprefix&gt;_1.fastq and &lt;outprefix&gt;_2.fastq for paired-end data. Experiment parameters: --numcopies &lt;int&gt; : Number of simulation rounds (copies of the reference genome) to simulate (Default: 100). Note, this is not directly comparable to the number of input cells. --numreads &lt;int&gt; : Number of reads (or read pairs) to simulate (Default: 1000000) --readlen &lt;int&gt; : Read length to generate (Default: 36bp) --paired : Simulated paired-end reads (by default single-end reads are generated). Model parameters: (either user-specified or learned from chips learn : --model &lt;str&gt; : JSON file with model parameters (e.g. from running learn. Setting parameters with other options overrides anything in the JSON file. --gamma-frag &lt;float&gt;,&lt;float&gt; : Parameters for fragment length distribution (k, theta for Gamma distribution). Default: 15.67,15.49 --spot &lt;float&gt; : SPOT score (fraction of reads in peaks). Default: 0.17594 --frac &lt;float&gt; : Fraction of the genome that is bound. Default: 0.03713 --pcr_rate &lt;float&gt; : The geometric step size paramters for simulating PCR. Default: 1. --recomputeF : Recompute --frac param based on input peaks. Recommended especially when using model parameters that were not learned on real data. Peak scoring: -b &lt;reads.bam&gt; : Use a provided BAM file to obtain scores for each peak (optional). If a BAM is not given, scores in the peak files are used. -c &lt;int&gt; : The index of the BED or homer peak file column used to score each peak (index starting from 1).
