In order to simplify this code base we have rewritten parts of it in kipoi-veff2 . If you must use this code base, make sure the following model_sources.kipoi.auto_update is False in ~/.kipoi/config.yaml pip install kipoi==0.6.35 git checkout 59b0140 at model_sources.kipoi.local_path from ~/.kipoi/config.yaml . This will ensure your local models repo remains compatible with kipoi-veff. An alternative to above step can be to manually copy the postprocessing tag of your chosen model from here to model_sources.kipoi.local_path/&lt;model&gt;/model(model-template).yml . For example, if you would like to work with DeepBind, copy this to model_sources.kipoi.local_path/DeepBind/model-template.yml . The variant effect prediction plug-in bring VCF annotation for DNA-sequence based models to models that are integrated into Kipoi . Additionally it offers visualisation tools to estimate the effect of possible genetic variants in a given area (mutation maps). Installation To install the variant effect prediction plug-in use the following command: pip install kipoi_veff In order to use the variant effect prediction plug-in with Kipoi the --vep argument has to be added when setting up the model environment with kipoi create env &lt;model_name&gt; --vep . Alternatively, you can use conda to install kipoi_veff: conda install -c bioconda kipoi_veff Usage example Main function of this package is score_variants accessible from the command line or python. It annotates the vcf file using model predictions for the reference and alternative alleles.
