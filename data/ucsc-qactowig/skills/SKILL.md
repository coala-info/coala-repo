# ucsc-qactowig

## Overview
The `ucsc-qactowig` skill facilitates the transformation of compressed quality score files (.qac) into a human-readable or browser-compatible wiggle format (.wig). QAC files are a specialized compressed format used by the UCSC Genome Browser utilities to store quality scores efficiently. Converting them to Wiggle format allows researchers to visualize these scores as tracks, showing the distribution of sequencing quality across specific genomic coordinates.

## Usage Instructions

### Basic Command Pattern
The tool is typically invoked using the binary name `qacToWig`. The standard usage follows a simple input-output pattern:

```bash
qacToWig input.qac output.wig
```

### Implementation Details
1.  **Permissions**: If you have downloaded the binary directly from the UCSC servers, ensure it has execution permissions:
    ```bash
    chmod +x qacToWig
    ```
2.  **Input Requirements**: The input must be a valid `.qac` file. These are often generated by other UCSC utilities like `fqToQac`.
3.  **Output Format**: The resulting `.wig` file will contain the quality scores mapped to genomic positions. This file can be further converted to `bigWig` using `wigToBigWig` for more efficient browser performance.

### Expert Tips
*   **Pipeline Integration**: Since Wiggle files can become quite large, it is common practice to pipe the output or immediately convert the resulting `.wig` to a `.bw` (bigWig) file to save disk space and improve visualization speed.
*   **Verification**: If the tool fails, verify the input file integrity. QAC files are binary; you can check if a file is a QAC by running `file input.qac` or attempting to read the header with a hex editor.
*   **No Arguments**: Running the command `qacToWig` with no arguments will typically display the version information and any specific optional flags supported by your current build.
